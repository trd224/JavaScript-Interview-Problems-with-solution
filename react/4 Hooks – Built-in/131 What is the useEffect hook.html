131. What is the useEffect hook?

One-Line Definition

useEffect is a React hook that allows functional components to perform side effects after rendering.

Simple Understanding

useEffect is used when a component needs to do something after rendering.

Examples

  • API calls
  • Timers
  • Event listeners
  • DOM updates
  • Subscriptions
  • Local storage access

Main Idea

Component Renders
↓
useEffect Executes
↓
Side Effect Runs

Basic Syntax

useEffect(() => {

  // Side effect code

}, dependencies);

Basic Example

import { useEffect } from 'react';

function App() {

  useEffect(() => {

    console.log('Component Rendered');

  });

}

What Happens?

This effect runs after every render because no dependency array is provided.

useEffect with Empty Dependency Array

useEffect(() => {

  console.log('Runs Once');

}, []);

What Happens?

This effect runs only once after the first render.

Equivalent Lifecycle Method

componentDidMount()

useEffect with Dependencies

import { useEffect, useState } from 'react';

function App() {

  const [count, setCount] = useState(0);

  useEffect(() => {

    console.log('Count Changed');

  }, [count]);

  return (

    <button onClick={() => setCount(count + 1)}>{count}</button>

  );

}

What Happens?

  • Runs after first render
  • Runs again whenever count changes

Cleanup Function

useEffect(() => {

  console.log('Mounted');

  return () => {

    console.log('Cleanup');

  };

}, []);

When Cleanup Runs

  • Before effect re-runs
  • When component unmounts

Why Cleanup Is Important

  • Prevents memory leaks
  • Stops duplicate listeners
  • Cleans timers and subscriptions

Real Timer Example

import { useEffect, useState } from 'react';

function App() {

  const [count, setCount] = useState(0);

  useEffect(() => {

    const timer = setInterval(() => {

      setCount((prev) => prev + 1);

    }, 1000);

    return () => clearInterval(timer);

  }, []);

  return <h1>{count}</h1>;

}

API Call Example

import { useEffect, useState } from 'react';

function App() {

  const [users, setUsers] = useState([]);

  useEffect(() => {

    fetch('https://api.com/users')
      .then((res) => res.json())
      .then((data) => setUsers(data));

  }, []);

}

Dependency Behavior

Code Behavior
useEffect(fn) Runs after every render
useEffect(fn, []) Runs once after first render
useEffect(fn, [count]) Runs when count changes

Main Dependency Flow

Dependency Changes
↓
useEffect Re-Runs
↓
Side Effect Updates

Before Hooks

// Class Components

componentDidMount();

componentDidUpdate();

componentWillUnmount();

After Hooks

useEffect()

Why useEffect Was Introduced

  • Simplified lifecycle handling
  • Reduced duplicate code
  • Made functional components powerful
  • Cleaner side-effect management

Rules of useEffect

Rule Meaning
Top-level only No loops or conditions
Dependency array controls execution Determines when effect re-runs
Cleanup removes effects Prevents memory leaks

Important Interview Points

  • useEffect handles side effects.
  • Runs after rendering.
  • Dependency array controls execution.
  • Cleanup prevents memory leaks.
  • Replaces lifecycle methods in functional components.
