132. What is the dependency array in useEffect?

One-Line Definition

The dependency array controls when a useEffect hook should re-run.

Simple Understanding

React watches the values inside the dependency array.

If any dependency changes, the effect runs again.

Main Idea

Dependency Changes
↓
React Detects Change
↓
useEffect Re-Runs

Basic Syntax

useEffect(() => {

  // Side effect

}, [dependencies]);

No Dependency Array

useEffect(() => {

  console.log('Runs After Every Render');

});

What Happens?

  • Runs after first render
  • Runs after every re-render

Why?

No dependency array means React has no restriction for re-running the effect.

Empty Dependency Array

useEffect(() => {

  console.log('Runs Once');

}, []);

What Happens?

Effect runs only once after the first render.

Equivalent Lifecycle

componentDidMount()

Dependency Array with Values

import { useEffect, useState } from 'react';

function App() {

  const [count, setCount] = useState(0);

  useEffect(() => {

    console.log('Count Changed');

  }, [count]);

  return (

    <button onClick={() => setCount(count + 1)}>{count}</button>

  );

}

What Happens?

  • Runs after first render
  • Runs whenever count changes

Multiple Dependencies

useEffect(() => {

  console.log('Runs When name or age Changes');

}, [name, age]);

What Happens?

Effect runs if any dependency changes.

How React Checks Dependencies

Previous Dependency Values
↓
Current Dependency Values
↓
React Compares Them
↓
If Changed → Effect Runs

Comparison Type

React uses shallow comparison with Object.is().

Important Problem with Objects and Arrays

useEffect(() => {

  console.log('Runs Again');

}, [{ name: 'Rizwan' }]);

What Happens?

useEffect runs on every render because a new object reference is created each time.

Why Problem?

Objects and arrays create new references on every render.

React compares references, not actual object values. 

{ name: 'Rizwan' } !== { name: 'Rizwan' }

Flow

Component Re-Renders
↓
New Object Created
↓
Previous Reference !== Current Reference
↓
useEffect Runs Again

Correct Solution Using useMemo

const user = useMemo(() => {

  return { name: 'Rizwan' };

}, []);

useEffect(() => {

  console.log('Runs Only Once');

}, [user]);

Important Understanding Example

This example is commonly used for learning purposes because it looks correct at first. 

const user = { name: 'Rizwan' };

useEffect(() => {

  console.log('Runs Only Once');

}, [user]);

Looks Correct But...

If this object is created inside the component, React creates a new object on every render.

Actual Behavior Inside Component

function App() {

  const user = { name: 'Rizwan' };

  useEffect(() => {

    console.log('Runs Again');

  }, [user]);

}

Why?

The component function runs again on every render, so the object gets recreated with a new memory reference. 

{ name: 'Rizwan' } !== { name: 'Rizwan' }
This example is useful for understanding why useMemo is needed for stable object references.
