129. How do you initialize state with useState?

One-Line Definition

State is initialized in React by passing an initial value to the useState() hook.

Basic Syntax

const [state, setState] = useState(initialValue);

What Happens Here?

Part Purpose
state Current state value
setState Function used to update state
initialValue Initial state value

Main Idea

Component Loads
↓
useState(initialValue)
↓
React Stores Initial State
↓
Component Uses That State

Initializing Number State

const [count, setCount] = useState(0);

Initializing String State

const [name, setName] = useState('Rizwan');

Initializing Boolean State

const [isOpen, setIsOpen] = useState(false);

Initializing Array State

const [users, setUsers] = useState([]);

Initializing Object State

const [user, setUser] = useState({

  name: 'Rizwan',
  age: 25

});

Complete Example

import { useState } from 'react';

function App() {

  const [count, setCount] = useState(0);

  return (

    <button onClick={() => setCount(count + 1)}>{count}</button>

  );

}

Important Point

Initial value is used only during the first render.

Example

const [count, setCount] = useState(0);

React does NOT reset count to 0 on every render.

Lazy Initialization

If initial calculation is expensive, use a function.

Example

const [count, setCount] = useState(() => {

  console.log('Runs once');

  return 0;

});

Why Lazy Initialization Is Useful

  • Runs only on first render
  • Improves performance
  • Avoids repeated expensive calculations

Difference Between These Two

Normal Initialization Lazy Initialization
useState(0) useState(() => 0)
Simple value Function returns value
Good for simple data Good for expensive calculations

Example with Local Storage

const [theme, setTheme] = useState(() => {

  return localStorage.getItem('theme') || 'light';

});

Why This Is Better

localStorage is accessed only once during first render.

Multiple States Example

function App() {

  const [name, setName] = useState('Rizwan');

  const [age, setAge] = useState(25);

  const [isAdmin, setIsAdmin] = useState(false);

}

What React Internally Does

useState(initialValue)
↓
React Creates State Slot
↓
Stores Initial Value
↓
Preserves State Between Renders

Important Interview Points

  • Initial state is passed to useState().
  • Initial value runs only during first render.
  • useState supports lazy initialization.
  • Any data type can be used as state.
  • State persists between renders.

Short Interview Answer

State is initialized by passing an initial value to useState(), which React stores during the first render and preserves across future renders.

One-Line Interview Answer

Initialize state by passing an initial value to useState().
