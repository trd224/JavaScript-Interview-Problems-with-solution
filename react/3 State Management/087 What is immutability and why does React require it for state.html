087. What is immutability and why does React require it for state?

One-Line Definition

Immutability means creating new copies of data instead of modifying existing data, and React requires it to detect state changes efficiently.

Simple Understanding

In React:

Never change existing state directly.

Instead:

  • Create a new updated copy
  • Replace old state with new state

Main Idea

Old State
↓
Create New Copy
↓
React Detects New Reference
↓
Component Re-Renders

Mutable Update Example

// Wrong

const user = {

  name: 'Rizwan'

};

user.name = 'Aman';

Why This Is Mutable

Existing object is modified directly.

Immutable Update Example

const user = {

  name: 'Rizwan'

};

const updatedUser = {

  ...user,
  name: 'Aman'

};

Why This Is Immutable

  • Original object stays unchanged
  • New object created
  • New reference generated

Why React Requires Immutability

React checks changes using: 

Reference Comparison

NOT deep object scanning.

Reference Example

const obj1 = { name: 'Rizwan' };

const obj2 = obj1;

console.log(obj1 === obj2);

Output

true

Why?

Both variables point to the same object reference.

New Object Example

const obj3 = {

  ...obj1

};

console.log(obj1 === obj3);

Output

false

Why React Uses Reference Comparison

  • Very fast
  • Better performance
  • Efficient rendering
  • Avoids expensive deep comparisons

Wrong React State Update

// Wrong

user.name = 'Aman';

setUser(user);

Problem

React may not detect any change because object reference remains the same.

Correct Immutable React Update

setUser({

  ...user,
  name: 'Aman'

});

What Happens Now?

  • New object created
  • Reference changes
  • React detects update
  • Component re-renders correctly

Immutable Array Updates

Wrong

// Wrong

users.push('Aman');

setUsers(users);

Correct

setUsers([...users, 'Aman']);

Important Interview Points

  • Immutability means not modifying existing data directly.
  • React uses reference comparison.
  • New references help React detect updates.
  • Spread operator is commonly used for immutable updates.
  • Redux strongly depends on immutability.

Short Interview Answer

Immutability means creating new copies of state instead of mutating existing state, allowing React to efficiently detect changes using reference comparison and re-render components correctly.

One-Line Interview Answer

React requires immutable state updates so it can detect changes using new object references.
