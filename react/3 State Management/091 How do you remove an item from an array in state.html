091. How do you remove an item from an array in state?

One-Line Definition

In React, items are removed from an array in state immutably using methods like filter().

Basic Array State

const [users, setUsers] = useState([

  'Rizwan',
  'Aman',
  'Rahul'

]);

Wrong Way

// Wrong

users.splice(1, 1);

setUsers(users);

Why This Is Wrong

  • Mutates original array
  • Reference remains same
  • React may not detect update
  • UI may not re-render correctly

Correct Way Using filter()

setUsers(

  users.filter((user) => user !== 'Aman')

);

What filter() Does

filter() creates a new array containing only items that match the condition.

Result

[
  'Rizwan',
  'Rahul'
]

Main Flow

Old Array
↓
filter() Creates New Array
↓
Item Removed
↓
New Array Reference
↓
React Re-Renders

Real Example

import { useState } from 'react';

function App() {

  const [users, setUsers] = useState([

    'Rizwan',
    'Aman',
    'Rahul'

  ]);

  const removeUser = () => {

    setUsers(

      users.filter((user) => user !== 'Aman')

    );

  };

  return (

    <div>

      <button onClick={removeUser}>
        Remove Aman
      </button>

      <ul>

        {users.map((user) => (
          <li>{user}</li>
        ))}

      </ul>

    </div>

  );

}

Output Before Click


Rizwan
Aman
Rahul

Output After Click


Rizwan
Rahul

Removing Item by ID

setUsers(

  users.filter((user) => user.id !== 2)

);

Why ID Removal Is Better

  • More reliable
  • Works with duplicate names
  • Common in real projects

Functional Update Version

Recommended when update depends on previous state. 

setUsers((prevUsers) => (

  prevUsers.filter((user) => user.id !== 2)

));

Why Functional Updates Are Better

  • Uses latest state
  • Avoids stale state issues
  • Safer for async updates

Why React Requires Immutable Updates

React uses reference comparison, so immutable operations like filter() create new array references that React can detect efficiently.

Mutable vs Immutable Array Removal

Mutable Immutable
splice() filter()
pop() slice()
shift() filter()

Real Project Use Cases

Feature Remove Operation
Todo app Delete task
Shopping cart Remove product
Notification panel Dismiss alert
Chat app Delete message
OTP logs Remove old entries

Important Interview Points

  • Never mutate arrays directly.
  • Use filter() for immutable removal.
  • filter() creates a new array.
  • Functional updates are safer.
  • React relies on new references for detecting changes.

Short Interview Answer

Items are removed from React state arrays immutably using methods like filter(), which create new arrays without mutating the original array.
