089. How do you update an object inside state?

One-Line Definition

An object inside React state is updated immutably using the spread operator to create a new object with updated values.

Basic State Object

const [user, setUser] = useState({

  name: 'Rizwan',
  age: 25,
  city: 'Jaipur'

});

Wrong Way

// Wrong

user.age = 26;

setUser(user);

Why This Is Wrong

  • Directly mutates existing object
  • Reference remains same
  • React may not detect update
  • UI may fail to re-render

Correct Way

setUser({

  ...user,
  age: 26

});

What Happens Here?

Code Purpose
...user Copies old properties
age: 26 Updates age property

Result

{
  name: 'Rizwan',
  age: 26,
  city: 'Jaipur'
}

Main Flow

Existing Object
↓
Create New Copy
↓
Update Required Property
↓
New Object Reference
↓
React Re-Renders

Updating Multiple Properties

setUser({

  ...user,
  age: 26,
  city: 'Delhi'

});

Functional Update Version

Recommended when update depends on previous state. 

setUser((prevUser) => ({

  ...prevUser,
  age: prevUser.age + 1

}));

Why Functional Updates Are Useful

  • Always uses latest state
  • Avoids stale state problems
  • Safer for async updates

Updating Nested Objects

const [user, setUser] = useState({

  name: 'Rizwan',

  address: {

    city: 'Jaipur',
    state: 'Rajasthan'

  }

});

Correct Nested Update

setUser({

  ...user,

  address: {

    ...user.address,
    city: 'Delhi'

  }

});

Why Nested Spread Is Needed

Nested objects also need immutable updates.

Common Mistake

// Wrong

setUser({

  age: 26

});

Problem

{
  age: 26
}

Other properties are removed because useState replaces the entire object.

Correct Version

setUser({

  ...user,
  age: 26

});

Why React Requires Immutable Updates

React detects state changes using reference comparison, so immutable updates create new references that React can detect efficiently.

Mutable vs Immutable Updates

Mutable Immutable
user.age = 26 { ...user, age: 26 }
array.push() [...array, item]
array.splice() array.filter()

Real Project Use Cases

Feature Object Update
User profile Update user information
OTP form Update verification state
Captcha tracking Update activity object
Theme settings Update config object
API state Update loading/data/error object

Important Interview Points

  • Never mutate state objects directly.
  • Use spread operator for immutable updates.
  • useState replaces entire object.
  • Nested objects require nested spreads.
  • Functional updates are safer for previous-state calculations.

Short Interview Answer

Objects inside React state are updated immutably using the spread operator to create a new object while preserving existing properties.

One-Line Interview Answer

Update React state objects immutably using the spread operator.
