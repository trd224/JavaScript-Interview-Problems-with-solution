085. What happens when you call setState or a state setter function?

One-Line Definition

When setState() or a useState setter function is called, React schedules a state update and re-renders the component with updated data.

Simple Understanding

Calling a state setter does NOT instantly change the UI directly.

React performs several internal steps.

Main Flow

State Setter Called
↓
React Schedules Update
↓
React Re-Renders Component
↓
Virtual DOM Updated
↓
React Compares Changes
↓
Real DOM Updates

Basic Example

import { useState } from 'react';

function App() {

  const [count, setCount] = useState(0);

  const handleClick = () => {

    setCount(count + 1);

  };

  return (

    <button onClick={handleClick}>

      {count}

    </button>

  );

}

What Happens Internally?

Step What React Does
1 setCount() called
2 React schedules state update
3 Component function runs again
4 New Virtual DOM created
5 React compares old vs new Virtual DOM
6 Only changed DOM updates

Very Important Point

State updates are asynchronous.

Example

const handleClick = () => {

  setCount(count + 1);

  console.log(count);

};

Output

Old value prints first

Why?

Because React schedules the update instead of updating immediately.

Functional Update Example

setCount((prev) => prev + 1);

Why Functional Updates Are Better

They always use the latest state value.

Multiple Updates Example

const handleClick = () => {

  setCount((prev) => prev + 1);

  setCount((prev) => prev + 1);

};

Output

Count increases by 2

Incorrect Version

// Wrong for multiple updates

setCount(count + 1);

setCount(count + 1);

Result

Count may increase only once

Why React Schedules Updates

  • Better performance
  • Batch multiple updates
  • Reduce unnecessary rendering
  • Improve UI responsiveness

Batching Updates

React often groups multiple state updates together.

Multiple setState Calls
↓
React Batches Updates
↓
Single Re-Render

Does setState Immediately Update DOM?

No. React first updates Virtual DOM, then efficiently updates the real DOM.

Class Component Version

// Class Component

this.setState({

  count: this.state.count + 1

});

Functional Component Version

// Functional Component

setCount(count + 1);

Re-Rendering Behavior

When state changes:

  • Component function runs again
  • JSX recalculates
  • React compares changes
  • DOM updates efficiently

Important Optimization

React updates only changed parts of the UI.

It does NOT reload the entire page.

Real Project Use Cases

Feature State Update Trigger
OTP input User typing
Captcha validation Validation response
Popup visibility Button click
Theme switching Toggle action
API loading Fetch lifecycle

Important Interview Points

  • State updates trigger re-rendering.
  • Updates are asynchronous.
  • React batches multiple updates.
  • Virtual DOM comparison improves performance.
  • Functional updates are safer for previous-state calculations.

Short Interview Answer

When a state setter function is called, React schedules a state update, re-renders the component, compares Virtual DOM changes, and efficiently updates only necessary parts of the real DOM.

One-Line Interview Answer

Calling a state setter schedules a re-render with updated state and optimized DOM updates.
