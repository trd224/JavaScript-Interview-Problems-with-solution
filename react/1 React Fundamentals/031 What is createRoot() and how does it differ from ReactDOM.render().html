031. What is createRoot() and how does it differ from ReactDOM.render()?

One-Line Definition

createRoot() is the new React 18 API used to create and manage the React application root, replacing the older ReactDOM.render() method.

What Was Used Before React 18?

Before React 18, applications were rendered using: 

import ReactDOM from 'react-dom';
import App from './App';

ReactDOM.render(

  <App />,

  document.getElementById('root')

);

Problem with ReactDOM.render()

ReactDOM.render() uses the old rendering engine and does not support React 18 concurrent features.

New React 18 Way

import { createRoot } from 'react-dom/client';
import App from './App';

const root = createRoot(
  document.getElementById('root')
);

root.render(<App />);

How createRoot() Works

Step Purpose
createRoot() Creates React root container
root.render() Renders React application

Main Difference

createRoot() ReactDOM.render()
Introduced in React 18 Used before React 18
Supports concurrent rendering No concurrent features
Modern rendering engine Legacy rendering engine
Recommended approach Deprecated approach
Uses root.render() Direct render method

What Is Concurrent Rendering?

Concurrent rendering allows React to:

  • Pause rendering
  • Prioritize important updates
  • Improve UI responsiveness
  • Handle large applications more efficiently

Flow Diagram

createRoot() → Create React Root → root.render() → React UI Appears

Why React Changed It

React 18 introduced a new rendering architecture for:

  • Better performance
  • Concurrent rendering
  • Automatic batching
  • Improved user experience

Old Syntax

Where createRoot() Is Usually Used

Normally inside: 

src/index.js

Example with StrictMode

import { createRoot } from 'react-dom/client';
import React from 'react';
import App from './App';

const root = createRoot(
  document.getElementById('root')
);

root.render(

  <React.StrictMode>

    <App />

  </React.StrictMode>

);
