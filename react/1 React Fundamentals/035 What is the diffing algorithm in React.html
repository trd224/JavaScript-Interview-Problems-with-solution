035. What is the diffing algorithm in React?

One-Line Definition

The diffing algorithm is React’s comparison process that finds differences between old and new Virtual DOM trees to efficiently update the real DOM.

Why Diffing Algorithm Is Needed

Updating the full real DOM is slow and expensive.

React improves performance by updating only changed elements.

To know what changed, React compares:

  • Old Virtual DOM
  • New Virtual DOM

This comparison process is called:

Diffing Algorithm

Simple Flow

State/Props Change
↓
New Virtual DOM Created
↓
Diffing Algorithm Compares Trees
↓
Find Differences
↓
Update Only Changed Real DOM Parts

Basic Example

function App() {

  const [count, setCount] = React.useState(0);

  return (

    <div>

      <h1>{count}</h1>

      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>

    </div>

  );

}

What Happens Internally?

Step React Behavior
Button clicked State changes
Component re-renders New Virtual DOM created
Diffing starts Old vs new Virtual DOM compared
Difference found Only h1 text changed
DOM updated Only changed DOM updated

Important Rule 1 - Different Element Types

If element types are different, React replaces the old DOM tree completely. 

// Old

<h1>Hello</h1>

// New

<p>Hello</p>

React destroys h1 and creates p.

Important Rule 2 - Same Element Types

If element types are same, React updates only changed attributes. 

// Old

<h1 className="red">Hello</h1>

// New

<h1 className="blue">Hello</h1>

React updates only: 

className

Important Rule 3 - Lists and Keys

React uses keys to track list items efficiently. 

users.map(user => (
  <li key={user.id}>
    {user.name}
  </li>
));

Why Keys Are Important

  • Correct item identification
  • Faster updates
  • Better rendering performance
  • Prevents unnecessary re-rendering

Without Keys Problem

React may incorrectly update or recreate list items.

Why React’s Diffing Is Fast

Traditional DOM tree comparison is very expensive.

React optimizes comparison using assumptions:

  • Different element types create different trees.
  • Keys identify stable list items.

Complexity Comparison

Algorithm Complexity
Traditional tree diffing O(n³)
React diffing algorithm O(n)

Difference Between Diffing and Reconciliation

Diffing Reconciliation
Comparison algorithm Whole update process
Finds differences Applies updates to DOM
Part of reconciliation Complete React update cycle

Real Project Use Cases

Application Benefit
Live dashboards Fast partial updates
Chat apps Efficient message rendering
Authentication forms Minimal DOM updates
E-commerce sites Optimized UI rendering

Important Interview Points

  • Diffing compares old and new Virtual DOM.
  • React updates only changed DOM nodes.
  • Keys optimize list rendering.
  • React diffing complexity is O(n).
  • Diffing is part of reconciliation.

Short Interview Answer

The diffing algorithm is React’s optimized comparison process that detects changes between old and new Virtual DOM trees and updates only changed parts of the real DOM.
