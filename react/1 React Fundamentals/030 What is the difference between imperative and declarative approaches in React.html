030. What is the difference between imperative and declarative approaches in React?

One-Line Definition

Imperative programming explains HOW to update the UI, while declarative programming describes WHAT the UI should look like.

What Is Imperative Programming?

Imperative programming manually controls the UI step-by-step.

Developers directly manipulate the DOM and tell the browser exactly what to do.

Imperative JavaScript Example

const button = document.getElementById('btn');

button.addEventListener('click', () => {

  const heading = document.getElementById('title');

  heading.innerText = 'Button Clicked';

  heading.style.color = 'cyan';

});

Why This Is Imperative

  • Manually selecting elements
  • Directly changing DOM
  • Updating styles manually
  • Handling UI updates step-by-step

What Is Declarative Programming?

Declarative programming focuses on describing the UI based on state.

React automatically updates the DOM when state changes.

Declarative React Example

import { useState } from 'react';

function App() {

  const [clicked, setClicked] = useState(false);

  return (

    <div>

      <h1 style={{ color: clicked ? 'cyan' : 'white' }}>
        {clicked ? 'Button Clicked' : 'Hello React'}
      </h1>

      <button onClick={() => setClicked(true)}>
        Click
      </button>

    </div>

  );

}

Why This Is Declarative

  • No manual DOM manipulation
  • UI depends on state
  • React handles DOM updates automatically
  • Cleaner and simpler code

Main Difference

Imperative Declarative
Focuses on HOW Focuses on WHAT
Manual DOM updates Automatic DOM updates
Direct DOM manipulation State-driven UI
More complex Cleaner and easier
Common in vanilla JS Core React philosophy

Flow Diagram

State Changes → React Re-renders UI → DOM Updates Automatically

Why React Uses Declarative Approach

  • Less code
  • Better readability
  • Easier maintenance
  • Automatic UI synchronization
  • Reduced DOM manipulation bugs

Real Project Use Cases

Feature Declarative Behavior
OTP Popup Render popup when state becomes true
Loader Show loader when loading=true
Error Message Render error UI from state
Dark Theme Automatically change UI from state

Important Note

React internally performs imperative DOM operations, but developers write declarative code.

Imperative Code in React

Sometimes React still uses imperative operations with refs. 

inputRef.current.focus();

This directly manipulates the DOM.

Important Interview Points

  • React is declarative.
  • Declarative code describes UI from state.
  • Imperative code manually updates the DOM.
  • React automatically handles rendering.
  • Declarative programming improves readability and maintainability.

Short Interview Answer

Imperative programming manually controls UI updates, while declarative React programming describes the UI state and lets React update the DOM automatically.

One-Line Interview Answer

Imperative programming explains HOW to update UI, while declarative React code describes WHAT the UI should display.
