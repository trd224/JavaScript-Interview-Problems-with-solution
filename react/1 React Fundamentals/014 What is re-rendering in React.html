014. What is re-rendering in React?

One-Line Definition

Re-rendering means React updates a component again when its state, props, or parent component changes.

Why Is Re-rendering Needed?

React applications are dynamic.

Whenever data changes, the UI must also update to show the latest information.

Re-rendering allows React to refresh the component UI automatically.

Common Causes of Re-rendering

Cause Description
State change Component state updates using setState or useState
Props change Parent sends new props to child
Parent re-renders Child components may also re-render
Context change Context API values update

Example Using useState

import { useState } from 'react';

function Counter() {

  const [count, setCount] = useState(0);

  console.log('Component Re-rendered');

  return (

    <div>

      <h1>{count}</h1>

      <button onClick={() => setCount(count + 1)} >
        Increment
      </button>

    </div>

  );

}

What Happens Here?

Step Process
1 User clicks button
2 setCount() updates state
3 React re-renders component
4 Updated UI appears in browser

Output

0
1
2
3

Important Point

Re-rendering does NOT mean React reloads the full webpage.

React only updates the changed parts using the Virtual DOM.

How React Optimizes Re-rendering

State Change → Virtual DOM Update → Compare Changes → Update Only Changed UI

Props Re-render Example

Parent Component

function App() {

  const [name, setName] = useState('Rizwan');

  return (
    <User name={name} />
  );

}

Child Component

function User(props) {

  console.log('User Re-rendered');

  return (
    <h1>{props.name}</h1>
  );

}

If the parent sends new props, the child component re-renders.

Real Project Use Case

Re-rendering happens continuously in:

  • OTP verification screens
  • Dashboard updates
  • Live chat applications
  • Notification systems
  • Shopping cart updates

Performance Optimization

React provides optimization tools like:

  • React.memo()
  • useMemo()
  • useCallback()

These help avoid unnecessary re-renders.

Important Interview Points

  • Re-rendering updates component UI.
  • State and props changes trigger re-rendering.
  • React uses Virtual DOM for efficient updates.
  • Re-rendering does not reload the entire page.
  • React optimizes DOM updates internally.

Short Interview Answer

Re-rendering means React updates a component again whenever its state, props, or related data changes.

One-Line Interview Answer

Re-rendering is the process where React updates component UI after data changes.
