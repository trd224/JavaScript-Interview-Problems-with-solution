015. What is the difference between mounting and updating a component?

One-Line Definition

Mounting means a component appears in the DOM for the first time, while updating means the component re-renders because its data changes.

What Is Mounting?

Mounting is the process where React:

  • Creates the component
  • Builds its UI
  • Inserts it into the DOM

This happens only once when the component first appears on the screen.

Mounting Example

function App() {

  return (
    <h1>Hello React</h1>
  );

}

When this component first appears in the browser, mounting occurs.

What Is Updating?

Updating happens when:

  • State changes
  • Props change
  • Parent component re-renders

React re-renders the component and updates the UI.

Updating Example

import { useState } from 'react';

function Counter() {

  const [count, setCount] = useState(0);

  return (

    <div>

      <h1>{count}</h1>

      <button onClick={() => setCount(count + 1)} >
        Increment
      </button>

    </div>

  );

}

Every button click updates the state and causes component updating.

Main Difference

Mounting Updating
Component appears first time Component already exists
Occurs once Can happen many times
Initial rendering Re-rendering
Creates DOM elements Updates existing DOM
First UI load UI refresh after data changes

Lifecycle Flow

Mounting → Updating → Unmounting

useEffect Mounting Example

import { useEffect } from 'react';

function App() {

  useEffect(() => {

    console.log('Mounted');

  }, []);

  return (
    <h1>Hello</h1>
  );

}

Output

Mounted

The empty dependency array means the effect runs only during mounting.

useEffect Updating Example

import { useEffect, useState } from 'react';

function App() {

  const [count, setCount] = useState(0);

  useEffect(() => {

    console.log('Updated');

  }, [count]);

  return (
    <button onClick={() => setCount(count + 1)} >
      {count}
    </button>
  );

}

Every count change triggers updating.

Important Interview Points

  • Mounting is the initial component creation.
  • Updating happens after state or props changes.
  • Mounting occurs once.
  • Updating can happen multiple times.
  • React efficiently updates UI using Virtual DOM.

Short Interview Answer

Mounting means a component loads into the DOM for the first time, while updating means React re-renders the component after data changes.

One-Line Interview Answer

Mounting is the first render of a component, and updating is re-rendering after state or props changes.
