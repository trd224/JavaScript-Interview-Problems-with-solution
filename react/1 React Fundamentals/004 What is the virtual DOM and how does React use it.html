004. What is the Virtual DOM and how does React use it?

One-Line Definition

The Virtual DOM is a lightweight JavaScript copy of the Real DOM that React uses to improve performance and update the UI efficiently.

Why Is It Used?

Updating the Real DOM directly is slow because browsers must:

  • Recalculate layout
  • Repaint UI
  • Re-render elements

React uses the Virtual DOM to minimize direct Real DOM updates, making applications faster and more efficient.

What Is the Real DOM?

The Real DOM is the actual webpage structure inside the browser.

Example Real DOM

<body>

  <h1>Hello</h1>

  <p>Welcome</p>

</body>

What Is the Virtual DOM?

The Virtual DOM is a JavaScript object representation of the Real DOM.

Example Virtual DOM Object

const virtualDOM = {
  type: 'h1',
  props: {
    children: 'Hello'
  }
};

How React Uses the Virtual DOM

Step Process
1 User action changes state/data
2 React creates a new Virtual DOM
3 React compares old and new Virtual DOM
4 React finds only changed elements
5 React updates only those parts in the Real DOM

This Process Is Called

Diffing and Reconciliation

Example

Initial UI

<h1>Hello</h1>

Updated UI

<h1>Hello Rizwan</h1>

React compares both Virtual DOM versions and updates only the changed text, instead of reloading the entire page.

Real Project Use Case

Suppose you are building:

  • Live OTP verification
  • Real-time dashboard
  • Chat application
  • Notification system

UI changes happen frequently. React uses the Virtual DOM to update only changed components, making the application fast and smooth.

Example Code

import { useState } from 'react';

function App() {

  const [count, setCount] = useState(0);

  return (
    <div>

      <h1>{count}</h1>

      <button
        onClick={() => setCount(count + 1)}
      >
        Increment
      </button>

    </div>
  );
}

What Happens Internally?

  • Button click changes state
  • React creates a new Virtual DOM
  • React compares old vs new Virtual DOM
  • Only the count text updates in Real DOM

Important Interview Points

  • Virtual DOM is a copy of Real DOM.
  • React uses Virtual DOM for performance optimization.
  • React updates only changed elements.
  • Diffing compares old and new Virtual DOM.
  • Reconciliation updates Real DOM efficiently.

Short Interview Answer

The Virtual DOM is a lightweight copy of the Real DOM. React compares Virtual DOM changes and updates only modified parts in the Real DOM for better performance.

One-Line Interview Answer

React uses the Virtual DOM to efficiently update only changed parts of the UI.
