013. What does StrictMode do in React?

One-Line Definition

StrictMode is a React development tool that helps detect potential problems and bad practices in a React application.

Why Is It Used?

StrictMode helps developers:

  • Find unsafe code
  • Detect deprecated APIs
  • Identify side effects
  • Improve application quality
  • Prepare apps for future React versions

It only works in development mode and does not affect production builds.

Basic Syntax

import React from 'react';

<React.StrictMode>

  <App />

</React.StrictMode>

Common Usage

import React from 'react';
import { createRoot } from 'react-dom/client';
import App from './App';

const root = createRoot(
  document.getElementById('root')
);

root.render(

  <React.StrictMode>

    <App />

  </React.StrictMode>

);

What StrictMode Does

Feature Purpose
Detects unsafe lifecycle methods Improves future compatibility
Warns about deprecated APIs Encourages modern React code
Detects unexpected side effects Helps debugging
Runs some functions twice in development Finds side effect issues

Important Behavior

In development mode, StrictMode intentionally runs some lifecycle methods and effects twice.

This helps detect accidental side effects.

Example

import { useEffect } from 'react';

function App() {

  useEffect(() => {

    console.log('Effect Executed');

  }, []);

  return (
    <h1>Hello React</h1>
  );

}

Why Double Rendering Happens

React uses double execution to detect:

  • Memory leaks
  • Improper cleanup
  • Unexpected side effects
  • Unsafe state updates

What StrictMode Does NOT Do

  • Does not render UI changes
  • Does not affect production performance
  • Does not add visible UI
  • Does not change application logic

Real Project Use Case

In large applications like:

  • Authentication Portals
  • Admin Dashboards
  • Banking Systems
  • Real-time Applications

StrictMode helps developers identify hidden bugs during development.

Flow Diagram

StrictMode → Detect Problems → Show Warnings → Improve Code Quality
