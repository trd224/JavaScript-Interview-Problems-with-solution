008. What is the difference between a functional and a class component?

One-Line Definition

Functional components are simple JavaScript functions, while class components are ES6 classes that extend React.Component.

Why Is This Important?

React originally used class components for advanced features like state and lifecycle methods.

After Hooks were introduced, functional components became the modern and preferred way to build React applications.

Functional Component Example

function Welcome() {

  return (

    <h1>
      Welcome to React
    </h1>

  );

}

Class Component Example

class Welcome extends React.Component {

  render() {

    return (

      <h1>
        Welcome to React
      </h1>

    );

  }

}

Main Differences

Functional Component Class Component
Simple JavaScript function ES6 class
Uses Hooks Uses lifecycle methods
Less boilerplate code More boilerplate code
No this keyword required Uses this keyword
Easier to read and maintain More complex syntax
Preferred in modern React Older React approach
Uses useState, useEffect, etc. Uses state and lifecycle methods

State Handling Difference

Functional Component State

import { useState } from 'react';

function Counter() {

  const [count, setCount] = useState(0);

  return (

    <button
      onClick={() => setCount(count + 1)}
    >

      {count}

    </button>

  );

}

Class Component State

class Counter extends React.Component {

  constructor() {

    super();

    this.state = {
      count: 0
    };

  }

  render() {

    return (

      <button
        onClick={() =>
          this.setState({
            count: this.state.count + 1
          })
        }
      >

        {this.state.count}

      </button>

    );

  }

}

Lifecycle Difference

Functional Component Class Component
useEffect() componentDidMount()
useEffect cleanup componentWillUnmount()
Hooks-based lifecycle Method-based lifecycle

Real Project Use Case

Modern applications like:

  • Dashboard Systems
  • Authentication Portals
  • Chat Applications
  • Admin Panels

mostly use functional components because they are cleaner, simpler, and easier to maintain.

Why Functional Components Are Preferred Today

  • Cleaner syntax
  • Better readability
  • Hooks simplify state management
  • No this keyword confusion
  • Less code
  • Better reusability

Important Interview Points

  • Functional components are modern React standard.
  • Class components were heavily used before Hooks.
  • Hooks work only in functional components.
  • Functional components are easier to maintain.
  • Class components still exist in older projects.

Short Interview Answer

Functional components are simple JavaScript functions that use Hooks, while class components are ES6 classes that use lifecycle methods and this.state.

One-Line Interview Answer

Functional components use Hooks and simpler syntax, while class components use classes and lifecycle methods.
