009. What is one-way data flow in React?

One-Line Definition

One-way data flow means data moves only from parent component to child component in React.

Why Is It Used?

One-way data flow makes React applications:

  • Predictable
  • Easier to debug
  • Easier to maintain
  • More organized
  • Better for large applications

Since data moves in only one direction, developers can easily track where data comes from and where it changes.

Simple Flow

Parent Component → Child Component

How React Passes Data

React passes data from parent to child using props.

Example

Parent Component

function App() {

  const name = 'Rizwan';

  return (
    <User name={name} />
  );

}

Child Component

function User(props) {

  return (
    <h1>Hello {props.name}</h1>
  );

}

Output

Hello Rizwan

Data Flow Explanation

Step What Happens
1 Parent component creates data
2 Parent sends data using props
3 Child receives props
4 Child displays the data

Important Point

Child components cannot directly modify parent data.

If a child wants to change parent data, the parent must provide a callback function.

Example with Callback

Parent Component

function App() {

  const [count, setCount] = useState(0);

  return (
    <Counter increase={() => setCount(count + 1)} />
  );

}

Child Component

function Counter(props) {

  return (
    <button onClick={props.increase} >
      Increment
    </button>
  );

}

Real Project Use Case

Suppose you are building:

  • Authentication Portal
  • Dashboard
  • Shopping Cart
  • Chat Application

The parent component stores the main application state, while child components only receive and display data.

This keeps the application structure clean and predictable.

One-Way vs Two-Way Data Flow

One-Way Data Flow Two-Way Data Flow
Data moves in one direction Data moves both directions
Easier to debug Harder to track changes
Used in React Common in some other frameworks
More predictable Can become complex

Important Interview Points

  • React follows one-way data binding.
  • Data flows from parent to child.
  • Props are used for passing data.
  • Child components cannot directly change parent state.
  • Callbacks help child components communicate with parent components.

Short Interview Answer

One-way data flow means data moves only from parent to child components in React. This makes applications easier to understand and debug.

One-Line Interview Answer

React follows one-way data flow where parent components pass data to child components using props.
