052. What are default props?

One-Line Definition

Default props are fallback values used when a component does not receive specific props from its parent.

Why Default Props Are Used

Sometimes parent components may not pass all props.

Default props provide safe fallback values to avoid undefined data.

Basic Example

function User({ name = 'Guest' }) {

  return (

    <h1>
      Hello {name}
    </h1>

  );

}

function App() {

  return (

    <User />

  );

}

Output

Hello Guest

How It Works

Since no name prop was passed:

React uses the default value "Guest".

Flow Diagram

Parent Does Not Pass Prop
↓
React Checks Default Value
↓
Default Prop Used

Modern React Approach

Today, default values are commonly handled using:

Destructuring default parameters

Example

function Button({

  text = 'Click Me',
  color = 'blue'

}) {

  return (

    <button>
      {text}
    </button>

  );

}

Using Passed Props

<Button text="Save" />

Output

Save

Passed props override default values.

Old defaultProps Syntax

Older React versions commonly used: 

function User(props) {

  return (

    <h1>
      Hello {props.name}
    </h1>

  );

}

User.defaultProps = {

  name: 'Guest'

};

Important Note

Modern React prefers default parameter values instead of defaultProps for functional components.

Default Props Only Work When Value Is Undefined

Passed Value Result
undefined Default value used
null Default value NOT used
"" Default value NOT used
0 Default value NOT used

Example

<User name={undefined} />

// Uses default value

<User name={null} />

// Does NOT use default value

Why Default Props Are Useful

  • Prevent undefined errors
  • Provide fallback UI
  • Improve component reusability
  • Reduce conditional checks

Real Project Use Cases

Feature Default Prop Example
User avatar Default profile image
Buttons Default button text
Theme Fallback dark/light mode
Authentication popup Default messages

Props vs Default Props

Props Default Props
Passed from parent Fallback values
May be undefined Used if prop missing
Dynamic data Safe defaults

Important Interview Points

  • Default props provide fallback values.
  • Used when props are undefined.
  • Modern React prefers default parameters.
  • Passed props override default values.
  • Useful for reusable components.

Short Interview Answer

Default props are fallback values assigned to component props when the parent component does not provide those props.

One-Line Interview Answer

Default props are fallback values used when React components do not receive specific props.
