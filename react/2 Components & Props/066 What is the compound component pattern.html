066. What is the compound component pattern?

One-Line Definition

The compound component pattern is a React pattern where multiple related components work together while sharing internal state and behavior.

Simple Understanding

Instead of creating one huge component:

  • We split functionality into smaller connected components
  • These components communicate internally
  • They work together as a group

Real-Life Analogy

Think about:

Tabs
├── Tab List
├── Tab Button
└── Tab Panel

All parts are separate components, but they work together.

Main Idea

Parent Component
↓
Shared State
↓
Child Components Access Shared Behavior

Basic Compound Component Example

function Accordion({ children }) {

  return (

    <div>{children}</div>

  );

}

function AccordionItem({ title, children }) {

  return (

    <div>

      <h2>{title}</h2>

      <div>{children}</div>

    </div>

  );

}

Usage

<Accordion>

  <AccordionItem title="What is React?">

    React is a UI library.

  </AccordionItem>

  <AccordionItem title="What is JSX?">

    JSX is syntax for React UI.

  </AccordionItem>

</Accordion>

Why It Is Called Compound Component

Because multiple smaller components combine to create one complete feature.

Most Common Examples

UI Feature Compound Components
Tabs Tabs, TabList, TabPanel
Accordion Accordion, AccordionItem
Dropdown Dropdown, Menu, MenuItem
Modal Modal, Header, Body, Footer

Advanced Compound Pattern

Usually compound components share state using:

  • Context API
  • React.cloneElement()
  • Custom hooks

Example with Context

const TabsContext = createContext();

Parent component stores shared state.

Child components access it using: 

useContext()

Why Compound Components Are Powerful

  • Flexible APIs
  • Cleaner JSX
  • Better reusability
  • Encapsulated logic
  • Customizable layouts

Example of Clean API

<Modal>

  <Modal.Header>
    Login
  </Modal.Header>

  <Modal.Body>
    Form Content
  </Modal.Body>

  <Modal.Footer>
    Buttons
  </Modal.Footer>

</Modal>

How This Helps

Instead of huge prop-based configuration: 

// Hard to manage

<Modal

  header="Login"

  body="..."

  footer="..."

/>

Compound components create cleaner structure.

Compound Components vs Regular Components

Regular Components Compound Components
Independent Connected together
Limited flexibility Highly flexible
More props needed Cleaner API
Simple structures Complex reusable UI

Important Note

Compound components are commonly used in UI libraries like Material UI, Radix UI, and Headless UI.

Important Interview Points

  • Compound components work together as a group.
  • They usually share internal state.
  • Commonly implemented using Context API.
  • Provide flexible and clean APIs.
  • Useful for reusable complex UI components.

Short Interview Answer

The compound component pattern is a React design pattern where multiple related components share internal state and behavior to build flexible and reusable UI structures.

One-Line Interview Answer

Compound components are multiple connected React components that work together using shared internal state.
