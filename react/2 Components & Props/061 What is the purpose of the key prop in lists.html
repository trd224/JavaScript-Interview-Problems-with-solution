061. What is the purpose of the key prop in lists?

One-Line Definition

The key prop helps React uniquely identify list items so it can efficiently update and re-render only changed elements.

Why React Needs key

When rendering lists, React must track:

  • Added items
  • Removed items
  • Updated items
  • Reordered items

React uses:

key

to uniquely identify each item.

Basic Example

function App() {

  const users = [

    { id: 1, name: 'Rizwan' },
    { id: 2, name: 'Aman' },
    { id: 3, name: 'Rahul' }

  ];

  return (

    <ul>
      {users.map((user) => <li key={user.id}>{user.name}</li>)}
    </ul>

  );

}

React Warning

Each child in a list should have a unique "key" prop

Why key Improves Performance

React uses reconciliation and diffing algorithms.

Keys help React quickly detect:

  • Which item changed
  • Which item moved
  • Which item was deleted

Example Without Proper key

Suppose this list changes: 


A
B
C

into: 


X
A
B
C

Without key

React may think:

  • A changed to X
  • B changed to A
  • C changed to B

causing unnecessary re-rendering.

With key

React correctly understands:

Only X is new.

Best Practice

Always use stable unique IDs as keys.

Good key Examples

key={user.id}

key={product._id}

key={email}

Bad key Examples

// Avoid if list changes

key={index}

// Very bad

key={Math.random()}

Why index Is Risky

  • Incorrect UI updates
  • State mismatch issues
  • Problems during sorting/reordering
  • Animation glitches

When index Is Acceptable

Safe Unsafe
Static lists Dynamic lists
No reordering Insert/delete operations
No filtering Sortable lists

Important Note

key is not available inside component props.

Example

// key is used internally by React

<User key={user.id} name={user.name} />

Inside User component: 

props.key  // undefined
