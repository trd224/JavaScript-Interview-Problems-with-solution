057. What is conditional rendering in React?

One-Line Definition

Conditional rendering in React means displaying different UI elements based on conditions.

Simple Understanding

React can show or hide UI depending on:

  • State
  • Props
  • User actions
  • Conditions

This is called:

Conditional Rendering

Real-Life Example

Suppose:

  • User logged in → Show dashboard
  • User not logged in → Show login page

Flow Diagram

Check Condition
↓
True → Render One UI
OR
False → Render Another UI

1. Using if Statement

function App() {

  const isLoggedIn = true;

  if (isLoggedIn) {

    return <h1>Welcome User</h1>;

  }

  return <h1>Please Login</h1>;

}

Output

Welcome User

2. Using Ternary Operator

Most commonly used in JSX. 

function App() {

  const isLoggedIn = false;

  return (

    <div>

      {

        isLoggedIn
          ? <h1>Dashboard</h1>
          : <h1>Login</h1>

      }

    </div>

  );

}

Output

Login

3. Using Short-Circuit Operator

Useful when rendering only if condition is true. 

function App() {

  const isAdmin = true;

  return (

    <div>

      {

        isAdmin && (
          <h1>Admin Panel</h1>
        )

      }

    </div>

  );

}

Output

Admin Panel

How && Works

Condition Result
true && JSX JSX renders
false && JSX Nothing renders

4. Using Variables

function App() {

  const isLoading = true;

  let content;

  if (isLoading) {

    content = <h1>Loading...</h1>;

  } else {

    content = <h1>Data Loaded</h1>;

  }

  return <div>{content}</div>;

}

5. Returning null

Returning null means:

Render nothing 
function Message({ show }) {

  if (!show) {

    return null;

  }

  return <h1>Hello</h1>;

}

Why Conditional Rendering Is Important

  • Dynamic UI
  • User authentication handling
  • Loading states
  • Error handling
  • Permission-based rendering

Real Project Use Cases

Feature Conditional Rendering Usage
OTP popup Show after verification
Loader Show during API calls
Error popup Show only on errors
Role-based dashboard Admin/User UI switching

Most Commonly Used Methods

Method Best Use Case
Ternary Operator Two UI choices
&& Operator Show/hide single UI
if Statement Complex conditions
null Render nothing

Important Interview Points

  • Conditional rendering changes UI based on conditions.
  • Ternary operator is very common in JSX.
  • && is used for show/hide logic.
  • null means render nothing.
  • Used heavily in dynamic applications.

Short Interview Answer

Conditional rendering in React is the technique of rendering different UI elements based on conditions using if statements, ternary operators, logical operators, or null rendering.
