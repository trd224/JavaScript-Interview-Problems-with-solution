#067 What is a Route Guard in Angular

🔹 What is a Route Guard in Angular?

👉 A Route Guard is a feature in Angular used to control access to routes based on conditions like authentication or authorization.

Guards in Angular are interfaces used to control navigation (route access, leaving page, data loading, etc.).

🔥 Types of Guards in Angular

1. CanActivate

👉 CanActivate is a guard that decides whether a route can be activated or not.

Example:

@Injectable({ providedIn: 'root' })
export class AuthGuard implements CanActivate {
  canActivate(): boolean {
    return true; // allow access
  }
}

2. CanActivateChild

👉 CanActivateChild is a guard that controls access to child routes.

Example:

@Injectable({ providedIn: 'root' })
export class ChildGuard implements CanActivateChild {
  canActivateChild(): boolean {
    return true;
  }
}

3. CanDeactivate

👉 CanDeactivate is a guard that decides whether you can leave a route (useful for unsaved changes).

Example:

export interface CanComponentDeactivate {
  canDeactivate: () => boolean;
}

@Injectable({ providedIn: 'root' })
export class ExitGuard implements CanDeactivate<CanComponentDeactivate> {
  canDeactivate(component: CanComponentDeactivate): boolean {
    return confirm('Leave page?');
  }
}

4. CanLoad

👉 CanLoad is a guard that prevents lazy-loaded modules from loading.

Example:

@Injectable({ providedIn: 'root' })
export class LoadGuard implements CanLoad {
  canLoad(): boolean {
    return false; // block module loading
  }
}

5. CanMatch (Modern replacement of CanLoad)

👉 CanMatch is a guard that decides if a route should match before loading.

Example:

@Injectable({ providedIn: 'root' })
export class MatchGuard implements CanMatch {
  canMatch(): boolean {
    return true;
  }
}

🔹 Resolver in Angular

👉 Resolver is a feature that loads data before a route is activated.

👉 Resolver is not a guard; it only prepares data before route activation.

🔥 Full Example

1. Resolver Service

@Injectable({ providedIn: 'root' })
export class UserResolver implements Resolve<any> {

  resolve(): Observable<any> {
    return of({ name: 'Rizwan', age: 25 }); // API call
  }

}

2. Route Configuration

const routes: Routes = [
  {
    path: 'profile',
    component: ProfileComponent,
    resolve: {
      user: UserResolver
    }
  }
];

3. Access Data in Component

constructor(private route: ActivatedRoute) {}

ngOnInit() {
  const data = this.route.snapshot.data['user'];
  console.log(data);
}
