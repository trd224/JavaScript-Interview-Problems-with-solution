What is retryWhen() and how does it give you more control than retry()?

Interview-level answer with full explanation, real Angular-style example, colorful code, and expected output.

Interview-Level Answer

Definition

retryWhen is an advanced retry operator that lets you control retry timing and retry conditions using an error Observable.

Difference From retry

  • retry repeats immediately with simple fixed count.
  • retryWhen can delay retries.
  • retryWhen can implement exponential backoff or custom stop logic.

Use Case

Use retryWhen when you need controlled retry behavior for failed API calls.

Key Points

  • Can delay retries.
  • Can implement exponential backoff.
  • Can stop retry based on custom rules.
Remember: retryWhen is retry with custom timing/control.

Full Understandable Example / Examples

retrywhen-example.service.ts
getUsersWithDelayRetry() {
  return this.http.get<any[]>('/api/users').pipe(
    retryWhen(error$ =>
      error$.pipe(
        tap(() => console.log('Retrying after 1 second...')),
        delay(1000),
        take(3)
      )
    ),
    catchError(error => {
      console.error('Final failure:', error);
      return of([]);
    })
  );
}

Expected Output / Behavior

On failure:
Retrying after 1 second...
Request retries.
After retry limit, fallback [] is returned.
