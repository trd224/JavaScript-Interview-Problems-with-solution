RxJS Mid-Level · Question 70

How does error handling work in an RxJS chain? What happens to the stream after an error?

Definition

In RxJS, an error is a terminal event. Once an Observable errors, that stream stops unless the error is handled.

How To Handle

  • Use catchError to catch the error.
  • Return a fallback Observable from catchError.
  • Use proper placement of catchError to control whether inner or outer stream stops.

Important Interview Point

After an error, the original stream does not continue. catchError replaces it with another Observable.

Key Points

  • Error stops the original stream.
  • catchError can return fallback Observable.
  • Placement of catchError affects whether outer stream continues.
Remember: After error, original stream is dead unless replaced by catchError.

Full Understandable Example / Examples

catchError-outside-example.ts
import { of, throwError } from 'rxjs';
import { catchError } from 'rxjs/operators';

// catchError outside handles the whole chain error.

throwError(() => new Error('API failed')).pipe(
  catchError(error => {
    console.log('Outer catchError:', error.message);
    return of([]);
  })
).subscribe(value => console.log(value));

import { of } from 'rxjs';
import { switchMap, catchError } from 'rxjs/operators';

// catchError inside switchMap handles inner API error,
// so outer stream can continue listening to future searches.

searchControl.valueChanges.pipe(
  switchMap(text =>
    http.get(`/api/search?q=${text}`).pipe(
      catchError(error => {
        console.log('Inner API failed:', error.message);
        return of([]);
      })
    )
  )
).subscribe(result => {
  console.log('Safe result:', result);
});
