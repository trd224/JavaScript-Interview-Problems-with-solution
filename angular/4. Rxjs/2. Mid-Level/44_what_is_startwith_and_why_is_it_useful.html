What is startWith() and why is it useful?

Definition

startWith adds an initial value to an Observable before the source emits its own values.

Why It Is Useful

  • Provides default UI state.
  • Makes streams emit immediately.
  • Helps combineLatest work with Angular form valueChanges.

Angular Use Case

Angular form valueChanges does not emit the current value automatically. startWith provides the first value so the UI can initialize immediately.

Key Points

  • Provides an initial/default emission.
  • Very useful with Angular Reactive Forms valueChanges.
  • Helps combineLatest produce initial output.
Remember: Use startWith when the stream needs a first value immediately.

Full Understandable Example / Examples

search.component.ts
import { Component, OnInit } from '@angular/core';
import { FormControl } from '@angular/forms';
import { startWith } from 'rxjs/operators';

@Component({
  selector: 'app-search',
  template: `&lt;input [formControl]=&quot;searchControl&quot; /&gt;`
})
export class SearchComponent implements OnInit {
  searchControl = new FormControl('');

  ngOnInit() {
    this.searchControl.valueChanges.pipe(
      startWith('default search')
    ).subscribe(value => {
      console.log('Search value:', value);
    });
  }
}
Search value: default search
Then it prints user-entered values.
