What is groupBy() in RxJS?

Definition

groupBy splits a source Observable into multiple grouped Observables based on a key.

How It Works

  • A key selector decides the group.
  • Each group is itself an Observable.
  • Each group has a key property.

Use Case

Use groupBy for category-wise stream processing, such as grouping orders by status, users by role, or logs by severity.

Key Points

  • Creates groups based on a key selector.
  • Each group is itself an Observable.
  • Useful for category-wise stream processing.
Remember: groupBy is like grouping array items, but in an Observable stream.

Full Understandable Example / Examples

groupby-example.ts
import { from } from 'rxjs';
import { groupBy, mergeMap, toArray, map } from 'rxjs/operators';

const orders = [
  { id: 1, status: 'pending' },
  { id: 2, status: 'completed' },
  { id: 3, status: 'pending' }
];

from(orders).pipe(
  groupBy(order => order.status),
  mergeMap(group$ =>
    group$.pipe(
      toArray(),
      map(items => ({
        status: group$.key,
        orders: items
      }))
    )
  )
).subscribe(group => {
  console.log(group);
});
{ status: 'pending', orders: [...] }
{ status: 'completed', orders: [...] }

Step-by-Step Explanation

Step 1 → from(orders)

from(orders) converts the array into an Observable stream. 

{ id: 1, status: 'pending' }

{ id: 2, status: 'completed' }

{ id: 3, status: 'pending' }

Step 2 → groupBy(order => order.status)

groupBy creates separate grouped Observables based on the status.

Group 1 Key: pending 
{ id: 1, status: 'pending' }

{ id: 3, status: 'pending' }
Group 2 Key: completed 
{ id: 2, status: 'completed' }
Important: groupBy() does NOT return grouped arrays directly. It returns grouped inner Observables. 
 Main Stream
      |
      |---- pending$ ----> order1 ----> order3
      |
      |---- completed$ --> order2

Step 3 → mergeMap(group$ => ...)

Each group itself is an Observable, so we use mergeMap() to process every group separately. 

mergeMap(group$ => ...)

Here: 

group$.key === 'pending'

group$.key === 'completed'

Step 4 → toArray()

toArray() collects all values emitted by each group into a single array.

Pending Group After toArray() 
[
  { id: 1, status: 'pending' },
  { id: 3, status: 'pending' }
]
Completed Group After toArray() 
[
  { id: 2, status: 'completed' }
]

Step 5 → map()

map() creates the final object structure. 

map(items => ({
  status: group$.key,
  orders: items
}))

Final Output

{
  status: 'pending',
  orders: [
    { id: 1, status: 'pending' },
    { id: 3, status: 'pending' }
  ]
}

{
  status: 'completed',
  orders: [
    { id: 2, status: 'completed' }
  ]
}

Real Project Use Cases

  • Orders grouped by status
  • Products grouped by category
  • Employees grouped by department
  • Chat messages grouped by roomId
  • Logs grouped by severity

Important Interview Point

groupBy() creates multiple inner Observables. That is why operators like mergeMap(), concatMap(), or switchMap() are commonly used after it.

Short Interview Answer

groupBy() splits Observable values into grouped inner Observables based on a key. Each group must be processed separately.

One-Line Interview Answer

groupBy() groups emitted Observable values by a condition and returns grouped inner Observables.
