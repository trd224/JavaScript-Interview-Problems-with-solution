How would you cancel a pending HTTP request using RxJS?

Interview-level answer with full explanation, real Angular-style example, colorful code, and expected output.

Interview-Level Answer

Definition

A pending Angular HTTP request can be cancelled by unsubscribing from its Observable.

Common RxJS Ways

  • switchMap: cancels previous request when a new value arrives.
  • takeUntil: cancels when a notifier emits, such as a Cancel button or destroy signal.

Angular Interview Point

Angular HttpClient Observables are cancellable. When the subscription is disposed, the underlying request is aborted.

Key Points

  • switchMap cancels old request when a new one starts.
  • takeUntil cancels request based on a cancel signal.
  • Unsubscribing from HttpClient Observable cancels the request.
Remember: Cancellation in RxJS means unsubscribe.

Full Understandable Example / Examples

switchMap-cancel-example.ts
import { FormControl } from '@angular/forms';
import { switchMap } from 'rxjs/operators';

// New search value cancels previous pending HTTP request.

const searchControl = new FormControl('');

searchControl.valueChanges.pipe(
  switchMap(text => http.get(`/api/search?q=${text}`))
).subscribe(result => {
  console.log('Latest result:', result);
});
takeUntil-cancel-example.ts
import { Subject } from 'rxjs';
import { takeUntil } from 'rxjs/operators';

// Cancel button/notifier stops the request.

const cancel$ = new Subject<void>();

http.get('/api/slow-data').pipe(
  takeUntil(cancel$)
).subscribe(data => {
  console.log('Data:', data);
});

// Later, when user clicks cancel
cancel$.next();

Expected Output / Behavior

Click Load: request starts.
Click Cancel before completion: request is cancelled and no data is printed.
