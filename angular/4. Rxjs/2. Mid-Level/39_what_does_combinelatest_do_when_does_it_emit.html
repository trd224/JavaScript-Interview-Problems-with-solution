What does combineLatest() do? When does it emit?

Definition

combineLatest combines multiple Observables and emits the latest value from each source as an array or object.

When It Emits

  • It waits until every source Observable has emitted at least once.
  • After that, whenever any source emits, it emits again with the latest values from all sources.

Angular Use Case

Use combineLatest for live UI state such as search text, selected category, sort order, pagination, route params, or logged-in user state.

Important Interview Point

combineLatest may not emit initially if one source has not emitted. That is why startWith() is commonly used with Angular form valueChanges.

Key Points

  • Waits for all sources to emit once before first output.
  • After first output, any source emission triggers a new combined output.
  • Often used with startWith for Angular form controls.
Remember: combineLatest is best for live latest-value combinations.

Full Understandable Example / Examples

products.component.ts
import { Component, OnInit } from '@angular/core';
import { FormControl } from '@angular/forms';
import { combineLatest } from 'rxjs';
import { startWith } from 'rxjs/operators';

@Component({
  selector: 'app-products',
  template: `
    <input [formControl]="searchControl" placeholder="Search product" />

    <select [formControl]="categoryControl">
      <option value="all">All</option>
      <option value="mobile">Mobile</option>
      <option value="laptop">Laptop</option>
    </select>
  `
})
export class ProductsComponent implements OnInit {
  searchControl = new FormControl('');
  categoryControl = new FormControl('all');

  ngOnInit() {
    combineLatest([
      this.searchControl.valueChanges.pipe(startWith('')),
      this.categoryControl.valueChanges.pipe(startWith('all'))
    ]).subscribe(([search, category]) => {
      console.log('Search:', search);
      console.log('Category:', category);
    });
  }
}
Category: all

Why use combineLatest() in this Angular example?

Simple Explanation

combineLatest() is used here because we need the latest value from both form controls together.

In this example, one value comes from the search input and the other value comes from the category dropdown.

Whenever the user changes search text or category, we want both latest values in one place.

Two Form Control Streams

form-control streams
this.searchControl.valueChanges
this.categoryControl.valueChanges

Both are separate Observables. The search Observable emits when the user types, and the category Observable emits when the user changes the dropdown.

Problem Without combineLatest()

separate subscriptions
this.searchControl.valueChanges.subscribe(search => {
  console.log('Search:', search);
});

this.categoryControl.valueChanges.subscribe(category => {
  console.log('Category:', category);
});
Problem: Search subscription only knows search value, and category subscription only knows category value. But for product filtering, we need both values together.

Solution With combineLatest()

combined latest values
combineLatest([
  this.searchControl.valueChanges.pipe(startWith('')),
  this.categoryControl.valueChanges.pipe(startWith('all'))
]).subscribe(([search, category]) => {
  console.log('Search:', search);
  console.log('Category:', category);
});

Now whenever search changes, category latest value is also available. Whenever category changes, search latest value is also available.

Why startWith() is used?

combineLatest() waits until every Observable emits at least once.

But Angular valueChanges does not emit the initial form value automatically.

initial values
this.searchControl.valueChanges.pipe(startWith(''))
this.categoryControl.valueChanges.pipe(startWith('all'))
Meaning: startWith('') gives initial search value, and startWith('all') gives initial category value. So combineLatest can emit immediately.

Real Product Filter Example

filter products using search + category
products = [
  { name: 'iPhone', category: 'mobile' },
  { name: 'Samsung', category: 'mobile' },
  { name: 'MacBook', category: 'laptop' }
];

combineLatest([
  this.searchControl.valueChanges.pipe(startWith('')),
  this.categoryControl.valueChanges.pipe(startWith('all'))
]).subscribe(([search, category]) => {

  const filtered = this.products.filter(product => {

    const matchSearch = product.name
      .toLowerCase()
      .includes(search.toLowerCase());

    const matchCategory =
      category === 'all' || product.category === category;

    return matchSearch && matchCategory;
  });

  console.log(filtered);
});

Simple Definition

combineLatest() combines multiple Observables and emits the latest values from all Observables whenever any one Observable changes.
