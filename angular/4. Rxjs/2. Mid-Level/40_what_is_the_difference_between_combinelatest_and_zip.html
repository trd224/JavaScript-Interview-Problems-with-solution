What is the difference between combineLatest() and zip()?

Interview-level answer with full explanation, real Angular-style example, colorful code, and expected output.

Interview-Level Answer

Definition

combineLatest and zip both combine Observables, but they combine emissions in different ways.

combineLatest

  • Uses the latest value from each Observable.
  • Emits whenever any source emits after all sources emitted once.
  • Best for live UI state like filters and form controls.

zip

  • Pairs values by index.
  • First value with first value, second with second, and so on.
  • Best when emissions must be matched pair-by-pair.

Key Points

  • combineLatest is latest-value based.
  • zip is pair/index based.
  • combineLatest can emit more frequently than zip.
Remember: combineLatest answers: what are the latest values? zip answers: what are the next matching pairs?

Full Understandable Example / Examples

combineLatest-example.ts
import { combineLatest } from 'rxjs';
import { FormControl } from '@angular/forms';
import { startWith } from 'rxjs/operators';

// combineLatest is best for live/latest UI state.

const searchControl = new FormControl('');
const sortControl = new FormControl('name');

combineLatest([
  searchControl.valueChanges.pipe(startWith('')),
  sortControl.valueChanges.pipe(startWith('name'))
]).subscribe(([search, sort]) => {
  console.log('combineLatest search:', search);
  console.log('combineLatest sort:', sort);
});
zip-example.ts
import { of, zip } from 'rxjs';

// zip is best when values should be paired by position/index.

const names$ = of('Rizwan', 'Aman', 'John');
const scores$ = of(90, 80, 70);

zip(names$, scores$).subscribe(([name, score]) => {
  console.log(`${name} scored ${score}`);
});

Expected Output / Behavior

combineLatest emits latest combinations whenever any source changes.
zip emits paired values like [A,1] and [B,2].
