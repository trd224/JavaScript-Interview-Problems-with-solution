What is exhaustMap and what problem does it solve?

Interview-level answer with full explanation, real Angular-style example, colorful code, and expected output.

Interview-Level Answer

Definition

exhaustMap maps a source value to an inner Observable, but while that inner Observable is running, it ignores all new source emissions.

Problem It Solves

  • Prevents duplicate login requests.
  • Prevents multiple form submissions.
  • Prevents repeated payment API calls caused by double-clicking.

How It Works

The first emission starts the inner Observable. Until that inner Observable completes, all new source emissions are ignored. After completion, the next emission can be accepted.

Angular Use Case

Use exhaustMap for Submit, Login, Save, or Payment buttons where repeated clicks should not create repeated HTTP requests.

Key Points

  • Prevents duplicate submit/login/payment requests.
  • Ignores repeated emissions while the current request is active.
  • Useful for button click streams where repeated clicks should not trigger repeated API calls.
Remember: Use exhaustMap when the first action must finish before another action is accepted.

Full Understandable Example / Examples

login.component.ts
import { Component, AfterViewInit } from '@angular/core';
import { fromEvent } from 'rxjs';
import { exhaustMap } from 'rxjs/operators';

@Component({
  selector: 'app-login',
  template: `&lt;button id=&quot;loginBtn&quot;&gt;Login&lt;/button&gt;`
})
export class LoginComponent implements AfterViewInit {
  constructor(private http: HttpClient) {}

  ngAfterViewInit() {
    const button = document.getElementById('loginBtn')!;

    fromEvent(button, 'click').pipe(
      exhaustMap(() => this.http.post('/api/login', {
        username: 'rizwan',
        password: 'secret'
      }))
    ).subscribe(response => {
      console.log('Login response:', response);
    });
  }
}

Expected Output / Behavior

Click 1: API request starts.
Click 2 and Click 3 during request: ignored.
After request completes, next click can start a new request.
