What is reduce() in RxJS and how does it differ from scan()?

Definition

reduce accumulates all values from a source Observable and emits only the final result when the source completes.

Difference From scan

  • reduce emits only once at the end.
  • scan emits running results after every emission.
  • reduce needs the source Observable to complete.

When To Use

Use reduce for final calculations on finite streams. Use scan for live or continuous state updates.

Key Points

  • reduce emits final accumulated value only after completion.
  • scan emits intermediate values continuously.
  • Do not use reduce on infinite streams unless you limit them.
Remember: reduce is final accumulation; scan is running accumulation.

Full Understandable Example / Examples

reduce-example.ts
import { of } from 'rxjs';
import { reduce } from 'rxjs/operators';

// reduce emits final total only after source completes.

of(100, 200, 300).pipe(
  reduce((total, value) => total + value, 0)
).subscribe(total => console.log('reduce final:', total));
scan: 100
scan: 300
scan: 600
reduce: 600
scan-example.ts
import { of } from 'rxjs';
import { scan } from 'rxjs/operators';

// scan emits running total after every value.

of(100, 200, 300).pipe(
  scan((total, value) => total + value, 0)
).subscribe(total => console.log('scan running:', total));
