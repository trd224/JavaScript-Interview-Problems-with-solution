What is retry() and when would you use it?

Definition

retry resubscribes to a failed Observable for a fixed number of attempts.

When To Use

  • Temporary network failures.
  • Short backend downtime.
  • Retrying safe GET requests.

When Not To Use

Do not blindly retry validation errors, authentication errors, or business rule failures because retrying will not fix them.

Key Points

  • Retries failed Observable a fixed number of times.
  • Useful for temporary network/server errors.
  • Should be combined with catchError for final fallback handling.
Remember: retry is for temporary failures, not permanent/business errors.

Full Understandable Example / Examples

retry-example.service.ts
getUsers() {
  return this.http.get<any[]>('/api/users').pipe(
    retry(3),
    catchError(error => {
      console.error('API failed after retries:', error);
      return of([]);
    })
  );
}

ngOnInit() {
  this.getUsers().subscribe(users => {
    console.log('Users:', users);
  });
}
API fails → retries up to 3 times.
If still failing, catchError returns empty array [].

Retry With 1 Second Delay Between Each Retry

Sometimes retrying immediately is not good because the server may still be temporarily down. In such cases, we can wait before each retry attempt.

retry-delay-example.service.ts
import { timer, throwError } from 'rxjs';
import { mergeMap, retryWhen, delayWhen, take } from 'rxjs/operators';

getUsers() {

  return this.http.get<any[]>('/api/users').pipe(

    retryWhen(errors =>

      errors.pipe(

        // wait 1 second before retry
        delayWhen(() => timer(1000)),

        // retry only 3 times
        take(3)
      )
    )
  );
}

ngOnInit() {

  this.getUsers().subscribe({
    next: users => {
      console.log('Users:', users);
    },

    error: err => {
      console.error('Final Error:', err);
    }
  });

}

API failed
Wait 1 second...

Wait 1 second...

Why use delay between retries? Immediate retries can overload the server again. Waiting 1 second gives the backend time to recover.
Interview Point: retry() retries immediately. retryWhen() gives advanced control like delay, exponential backoff, custom retry conditions, etc.
