What is forkJoin() and when would you use it?

Interview-level answer with full explanation, real Angular-style example, colorful code, and expected output.

Interview-Level Answer

Definition

forkJoin runs multiple Observables in parallel and waits until all of them complete. Then it emits the last value from each Observable once.

When To Use

  • Use it for multiple parallel HTTP calls.
  • Use it when all results are required before updating the UI.
  • Use it for dashboard loading, profile + permissions loading, or master data loading.

Important Behavior

  • If any Observable errors, forkJoin errors.
  • If any Observable never completes, forkJoin never emits.
  • It is best for finite Observables like Angular HttpClient calls.

Key Points

  • Emits only once after all sources complete.
  • Best for parallel HTTP requests.
  • Will not emit if any source never completes.
Remember: forkJoin is like Promise.all for Observables.

Full Understandable Example / Examples

dashboard.component.ts
import { Component, OnInit } from '@angular/core';
import { forkJoin } from 'rxjs';

@Component({
  selector: 'app-dashboard',
  template: `
    <h3>Dashboard</h3>
    <p>Users: {{ users.length }}</p>
    <p>Orders: {{ orders.length }}</p>
  `
})
export class DashboardComponent implements OnInit {
  users: any[] = [];
  orders: any[] = [];

  constructor(private http: HttpClient) {}

  ngOnInit() {
    forkJoin({
      users: this.http.get<any[]>('/api/users'),
      orders: this.http.get<any[]>('/api/orders')
    }).subscribe(result => {
      this.users = result.users;
      this.orders = result.orders;

      console.log('All API calls completed:', result);
    });
  }
}

Expected Output / Behavior

Both API calls start together.
UI updates only after both users and orders responses complete.
