What is the async pipe in Angular and how does it relate to RxJS Observables?

Definition

The async pipe is an Angular template pipe that subscribes to an Observable or Promise and displays the latest emitted value.

Relation With RxJS

  • It works directly with RxJS Observables.
  • It automatically subscribes when the template is rendered.
  • It automatically unsubscribes when the component is destroyed.

Why It Is Useful

It reduces manual subscription code and helps prevent memory leaks in Angular templates.

Key Points

  • async pipe subscribes automatically.
  • It updates template with latest value.
  • It unsubscribes automatically.
Remember: Prefer async pipe when Observable data is only needed in the template.

Full Understandable Example / Examples

component.ts
import { Component } from '@angular/core';
import { HttpClient } from '@angular/common/http';

@Component({
  selector: 'app-users',
  templateUrl: './users.component.html'
})
export class UsersComponent {
  users$ = this.http.get<any[]>('/api/users');

  constructor(private http: HttpClient) {}
}
Template displays user names from the Observable.
Angular handles subscribe/unsubscribe automatically.
component.html
<ul>
  <li *ngFor="let user of users$ | async">
    {{ user.name }}
  </li>
</ul>
manual-subscribe-comparison.ts
// Without async pipe, you manually subscribe and manage cleanup
ngOnInit() {
  this.http.get('/api/users').subscribe(users => {
    this.users = users;
  });
}
