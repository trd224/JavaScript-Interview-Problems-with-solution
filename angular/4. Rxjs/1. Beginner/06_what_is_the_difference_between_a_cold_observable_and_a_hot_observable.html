RxJS Beginner · Question 06

What is the difference between a cold Observable and a hot Observable?

Beginner-friendly interview-level answer with headings, lists, full examples for every concept, colorful code, and expected behavior.

Interview-Level Answer

Cold Observable

A cold Observable starts a new independent execution for each subscriber. Angular HTTP Observables are cold.

Hot Observable

A hot Observable shares the same execution/source among multiple subscribers. Subjects, DOM events, and WebSocket streams are common hot sources.

Interview Difference

  • Cold = each subscriber gets separate producer.
  • Hot = subscribers share same producer.

Key Points

  • Cold Observables are usually unicast.
  • Hot Observables are usually multicast.
  • This affects duplicate API calls and shared events.
Remember: Cold starts per subscriber; hot is already shared.

Full Understandable Example / Examples

cold-observable-example.ts
import { Observable } from 'rxjs';

const cold$ = new Observable<number>(subscriber => {
  const value = Math.random();
  subscriber.next(value);
  subscriber.complete();
});

cold$.subscribe(value => console.log('Cold A:', value));
cold$.subscribe(value => console.log('Cold B:', value));
hot-observable-example.ts
import { Subject } from 'rxjs';

const hot$ = new Subject<number>();

hot$.subscribe(value => console.log('Hot A:', value));
hot$.subscribe(value => console.log('Hot B:', value));

hot$.next(Math.random());

Expected Output / Behavior

Cold A and Cold B may get different values.
Hot A and Hot B receive the same pushed value.
