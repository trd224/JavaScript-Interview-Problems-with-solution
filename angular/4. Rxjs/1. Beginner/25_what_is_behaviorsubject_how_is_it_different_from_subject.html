What is BehaviorSubject? How is it different from Subject?

Beginner-friendly interview-level answer with headings, lists, full examples for every concept, colorful code, and expected behavior.

Interview-Level Answer

BehaviorSubject

BehaviorSubject is a Subject that requires an initial value and always stores the latest value.

Difference From Subject

  • Subject does not store previous values.
  • BehaviorSubject stores the latest value.
  • New subscribers immediately receive the current/latest value from BehaviorSubject.

Angular Use Case

BehaviorSubject is often used for simple shared state such as logged-in user, theme, selected item, or counter state.

Key Points

  • BehaviorSubject needs initial value.
  • It replays latest value to new subscribers.
  • Good for simple state.
Remember: BehaviorSubject = Subject with current value.

Full Understandable Example / Examples

subject-example.ts
import { Subject } from 'rxjs';

const subject$ = new Subject<string>();

subject$.next('Before subscription');

subject$.subscribe(value => {
  console.log('Subject:', value);
});

subject$.next('After subscription');
behavior-subject-example.ts
import { BehaviorSubject } from 'rxjs';

const state$ = new BehaviorSubject<string>('Initial');

state$.next('Before subscription');

state$.subscribe(value => {
  console.log('BehaviorSubject:', value);
});

state$.next('After subscription');

Expected Output / Behavior

Subject: After subscription
BehaviorSubject: Before subscription
BehaviorSubject: After subscription
