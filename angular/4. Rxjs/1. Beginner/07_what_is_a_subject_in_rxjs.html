What is a Subject in RxJS?

Definition

A Subject is both an Observable and an Observer. It can be subscribed to, and it can also emit values using next().

Why It Is Useful

  • It allows multicasting values to many subscribers.
  • It is useful for events and component communication.
  • It can act as a bridge between imperative code and Observable streams.

Best Practice

Keep Subjects private inside services and expose them as Observables.

Key Points

  • Subject can emit and be subscribed to.
  • Subject is multicast.
  • Expose asObservable() to protect it.
Remember: Subject is a manually controlled multicast Observable.

Full Understandable Example / Examples

subject-basic-example.ts 
import { Subject } from 'rxjs';

const message$ = new Subject<string>();

message$.subscribe(message => {
  console.log('Subscriber A:', message);
});

message$.subscribe(message => {
  console.log('Subscriber B:', message);
});

message$.next('Hello Subject');
Subscriber A: Hello Subject
Subscriber B: Hello Subject
angular-service-subject.ts 
import { Injectable } from '@angular/core';
import { Subject } from 'rxjs';

@Injectable({ providedIn: 'root' })

export class MessageService {

  private messageSubject =
  new Subject<string>();

  message$ =
  this.messageSubject.asObservable();

  sendMessage(message: string) {

    this.messageSubject.next(message);

  }

}

Difference Between Subject, BehaviorSubject, ReplaySubject and AsyncSubject

Overview

In RxJS, Subject, BehaviorSubject, ReplaySubject, and AsyncSubject are special types of Observables that can emit values and also be subscribed to.

1. Subject

Subject does not store previous values. Subscribers only receive values emitted after subscription.

subject-example.ts 
import { Subject } from 'rxjs';

const subject =
new Subject<number>();

subject.next(1);

subject.subscribe(value => {

  console.log('A:', value);

});

subject.next(2);

subject.next(3);
A: 2
A: 3
Value 1 was emitted before subscription, so subscriber did not receive it.

2. BehaviorSubject

BehaviorSubject stores the latest value and immediately sends it to new subscribers.

behavior-subject-example.ts 
import { BehaviorSubject } from 'rxjs';

const subject =
new BehaviorSubject<number>(0);

subject.next(1);

subject.subscribe(value => {

  console.log('A:', value);

});

subject.next(2);
A: 1
A: 2
New subscriber immediately receives the latest value.

3. ReplaySubject

ReplaySubject stores multiple previous values and replays them to new subscribers.

replay-subject-example.ts 
import { ReplaySubject } from 'rxjs';

const subject =
new ReplaySubject<number>(2);

subject.next(1);

subject.next(2);

subject.next(3);

subject.subscribe(value => {

  console.log('A:', value);

});

subject.next(4);
A: 2
A: 3
A: 4
ReplaySubject(2) stores the last 2 emitted values.

4. AsyncSubject

AsyncSubject emits only the last value and only after complete() is called.

async-subject-example.ts 
import { AsyncSubject } from 'rxjs';

const subject =
new AsyncSubject<number>();

subject.next(1);

subject.next(2);

subject.next(3);

subject.subscribe(value => {

  console.log('A:', value);

});

subject.complete();
A: 3
AsyncSubject emits only the final value after completion.

Main Difference Table

Type Stores Previous Value? Initial Value Needed? New Subscriber Gets Emits When
Subject No No Nothing old Immediately
BehaviorSubject Latest value only Yes Latest value Immediately
ReplaySubject Multiple values No Buffered values Immediately
AsyncSubject Last value only No Last value Only after complete()

Real-Life Analogy

Subject Live TV
BehaviorSubject Current Score Board
ReplaySubject YouTube History
AsyncSubject Final Exam Result

Why Does BehaviorSubject Require an Initial Value?

Main Reason

BehaviorSubject is designed to always store and provide the current/latest value.

Whenever a new subscriber subscribes, BehaviorSubject immediately emits the latest stored value.

Because of this behavior, it must always have some value available internally.

Example Problem Without Initial Value 

const subject =
new BehaviorSubject();

Now if a subscriber subscribes immediately: 

subject.subscribe(value => {

  console.log(value);

});
What should BehaviorSubject emit immediately? undefined? null? nothing?

That would break the idea of BehaviorSubject always having a current value.

Correct Usage 

const theme$ =
new BehaviorSubject<string>('light');

Now any new subscriber immediately receives: 

light
BehaviorSubject internally stores the latest value.

Internal Concept 

// Internal thinking of BehaviorSubject

currentValue = initialValue;

newSubscriber => send(currentValue);

Difference From Subject

Type Behavior
Subject No stored value
BehaviorSubject Always stores latest value

Real-Life Analogy

Subject Live Radio. If you join late, you miss old audio.
BehaviorSubject Scoreboard. Even if you join late, you immediately see the current score.

Why ReplaySubject Does Not Need an Initial Value

Main Difference

ReplaySubject stores previous emitted history, not a required current value.

If no value has been emitted yet, ReplaySubject can simply have empty history.

ReplaySubject Internal Concept 

// Internal thinking of ReplaySubject

history = [];

next(value) => history.push(value);

If subscriber comes early and history is empty: 

Nothing is emitted
This is completely valid for ReplaySubject.

Example 

const subject =
new ReplaySubject(2);

subject.subscribe(value => {

  console.log(value);

});
Nothing

No error happens because ReplaySubject is allowed to have empty history initially.

Internal Purpose Comparison

Type Main Purpose
BehaviorSubject Always keep current/latest state
ReplaySubject Replay previous emitted history

Real-Life Analogy

BehaviorSubject Scoreboard must always show current score, even before match starts.
ReplaySubject CCTV recording can simply have no previous footage yet.
