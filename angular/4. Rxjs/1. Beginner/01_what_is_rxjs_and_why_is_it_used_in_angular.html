What is RxJS and why is it used in Angular?

Definition

RxJS is a reactive programming library used to work with asynchronous data streams using Observables.

Why Angular Uses It

  • Angular HTTP calls return Observables.
  • Reactive forms expose valueChanges and statusChanges as Observables.
  • Router params, events, timers, and real-time streams can be handled using RxJS.

Interview Explanation

RxJS helps Angular developers transform, combine, cancel, retry, and handle errors in async operations in a clean and composable way.

Key Points

  • RxJS handles async streams.
  • Angular uses RxJS heavily in HttpClient, forms, router, and events.
  • Observable is the core building block.
Remember: RxJS makes async code manageable and powerful in Angular.

Full Understandable Example / Examples

http-rxjs-example.component.ts
import { Component, OnInit } from '@angular/core';
import { HttpClient } from '@angular/common/http';

@Component({
  selector: 'app-users',
  template: `
    <ul>
      <li *ngFor="let user of users">{{ user.name }}</li>
    </ul>
  `
})
export class UsersComponent implements OnInit {
  users: any[] = [];

  constructor(private http: HttpClient) {}

  ngOnInit() {
    this.http.get<any[]>('/api/users').subscribe(users => {
      this.users = users;
      console.log('Users:', users);
    });
  }
}
API users are printed and shown in UI.
form-rxjs-example.ts
import { FormControl } from '@angular/forms';

const searchControl = new FormControl('');

searchControl.valueChanges.subscribe(value => {
  console.log('Search text:', value);
});
every input change emits a value.
