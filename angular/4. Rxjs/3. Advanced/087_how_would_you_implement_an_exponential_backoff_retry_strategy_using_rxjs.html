How would you implement an exponential backoff retry strategy using RxJS?

Advanced interview-level answer with headings, lists, full examples for every concept, colorful code, and expected behavior.

Interview-Level Answer

Definition

Exponential backoff means retrying a failed operation with increasing delay after each failure, such as 1s, 2s, 4s, 8s.

Why It Is Used

  • Prevents hammering a failing server.
  • Gives temporary failures time to recover.
  • Useful for network or 5xx server errors.

RxJS Approach

Use retryWhen with scan to count attempts and delayWhen/timer to delay retries.

Key Points

  • retryWhen gives custom retry logic.
  • scan tracks retry count.
  • timer creates increasing delay.
Remember: Exponential backoff = retry with growing delay.

Full Understandable Example / Examples

exponential-backoff.service.ts
import { throwError, timer } from 'rxjs';
import { retryWhen, scan, delayWhen } from 'rxjs/operators';

getData() {
  return this.http.get('/api/data').pipe(
    retryWhen(errors =>
      errors.pipe(
        scan((retryCount, error) => {
          if (retryCount >= 3) {
            throw error;
          }

          return retryCount + 1;
        }, 0),
        delayWhen(retryCount => {
          const delayMs = Math.pow(2, retryCount) * 1000;
          console.log('Retry after:', delayMs);
          return timer(delayMs);
        })
      )
    )
  );
}

Expected Output / Behavior

Retry 1 after 2000ms
Retry 2 after 4000ms
Retry 3 after 8000ms
Then fail if still unsuccessful.
