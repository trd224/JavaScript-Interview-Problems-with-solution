What is the difference between a pipeable operator and a creation operator?

Advanced with headings, lists, full examples for every concept, colorful code, and expected behavior.

Definition

A creation operator creates a new Observable from scratch. A pipeable operator transforms or controls an existing Observable inside pipe().

Creation Operator

  • Used to create Observables.
  • Examples: of, from, interval, timer, fromEvent, throwError.

Pipeable Operator

  • Used inside pipe() on an existing Observable.
  • Examples: map, filter, switchMap, catchError, takeUntil.

Key Points

  • Creation operators start streams.
  • Pipeable operators modify streams.
  • Both are used together in real RxJS code.
Remember: Creation operator creates; pipeable operator transforms.

Full Understandable Example / Examples

creation-operator-example.ts
import { of, fromEvent } from 'rxjs';

// of() creates an Observable from static values
of(1, 2, 3).subscribe(value => console.log(value));

// fromEvent() creates an Observable from DOM events
fromEvent(document, 'click').subscribe(() => {
  console.log('Clicked');
});
creates streams.
pipeable-operator-example.ts
import { of } from 'rxjs';
import { filter, map } from 'rxjs/operators';

of(1, 2, 3, 4).pipe(
  filter(value => value % 2 === 0),
  map(value => value * 10)
).subscribe(value => console.log(value));
outputs 20 and 40.
