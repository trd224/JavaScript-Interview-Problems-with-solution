What are the key differences between RxJS 6 and RxJS 7?

Advanced with headings, lists, full examples for every concept, colorful code, and expected behavior.

High-Level Difference

RxJS 7 improved typing, APIs, multicasting, deprecations, and compatibility while keeping most RxJS 6 pipeable operator patterns familiar.

Key Differences

  • toPromise is deprecated; use firstValueFrom or lastValueFrom.
  • Improved TypeScript typings.
  • Modern multicasting APIs such as connectable/connect.
  • Some older APIs/operators are deprecated or discouraged.
  • Better configuration options for operators like share.

Interview Point

Most Angular developers mainly notice toPromise deprecation, better typings, and updated multicasting recommendations.

Key Points

  • RxJS 7 keeps pipeable operator style.
  • toPromise deprecation is very important.
  • Multicasting APIs became more modern.
Remember: RxJS 7 is mostly compatible but modernizes APIs and typings.

Full Understandable Example / Examples

rxjs6-old.ts
// Older RxJS style
const user = await this.http.get('/api/user').toPromise();
RxJS 6 style used toPromise.
rxjs7-new.ts
import { firstValueFrom } from 'rxjs';

// Modern RxJS 7 style
const user = await firstValueFrom(
  this.http.get('/api/user')
);
RxJS 7 recommends firstValueFrom/lastValueFrom.
