What are marble diagrams and how do you read them?

Advanced interview-level answer with headings, lists, full examples for every concept, colorful code, and expected behavior.

Interview-Level Answer

Definition

Marble diagrams are visual/string representations of Observable emissions over time.

Common Symbols

  • - time frame
  • a, b, c emitted values
  • | completion
  • # error
  • () grouped simultaneous emissions

Example

The marble string --a-b-| means time passes, then a emits, then b emits, then the stream completes.

Key Points

  • Marbles represent time and emissions.
  • They are used in docs and tests.
  • They help understand complex stream timing.
Remember: Marble diagrams are timelines for Observables.

Full Understandable Example / Examples

marble-reading-example.txt
Marble:
--a-b-c|

Meaning:
--  time passes
a   emit value a
-   time passes
b   emit value b
-   time passes
c   emit value c
|   complete
marble-test-small.spec.ts
scheduler.run(({ cold, expectObservable }) => {
  const source$ = cold('--a-b-|');
  expectObservable(source$).toBe('--a-b-|');
});

Expected Output / Behavior

--a-b-c| means values a, b, c emitted over time, then completed.
