How does connectable() work in RxJS 7+? What replaced publish()?

Advanced with headings, lists, full examples for every concept, colorful code, and expected behavior.

Definition

connectable() creates a connectable Observable that does not start source execution until connect() is called.

Replacement

In RxJS 7+, connectable() is the modern replacement for old publish-style multicasting APIs.

How It Works

  • Create source Observable.
  • Wrap it using connectable().
  • Subscribe observers.
  • Call connect() to start shared execution.

Key Points

  • connectable gives manual control over shared execution.
  • It replaced older publish patterns.
  • Useful when subscribers must be attached before source starts.
Remember: connectable = manual multicast setup in RxJS 7+.

Full Understandable Example / Examples

connectable-example.ts
import { connectable, interval, Subject } from 'rxjs';
import { take } from 'rxjs/operators';

const source$ = interval(1000).pipe(take(3));

const shared$ = connectable(source$, {
  connector: () => new Subject<number>()
});

shared$.subscribe(value => console.log('A:', value));
shared$.subscribe(value => console.log('B:', value));

const connection = shared$.connect();
