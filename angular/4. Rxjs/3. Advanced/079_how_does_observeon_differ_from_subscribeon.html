How does observeOn() differ from subscribeOn()?

Advanced with headings, lists, full examples for every concept, colorful code, and expected behavior.

First Understand the Two Different Phases

An Observable has mainly 2 phases:

Phase Meaning
Subscription Phase When Observable starts executing
Notification Phase When next/error/complete values are delivered

Definition

observeOn and subscribeOn both deal with scheduling, but they affect different parts of Observable execution.

observeOn

  • Controls when emitted notifications are delivered to the observer.
  • Affects next, error, and complete delivery after its position in the pipe.

subscribeOn

  • Controls when subscription to the source happens.
  • Affects source execution setup.

Interview Point

observeOn changes notification delivery timing. subscribeOn changes subscription side-effect timing.

Key Points

  • observeOn schedules emissions to observer.
  • subscribeOn schedules source subscription.
  • They solve different timing problems.
Remember: observeOn = when observer receives; subscribeOn = when source starts.

Full Understandable Example / Examples

observeOn-example.ts
import { of, asyncScheduler } from 'rxjs';
import { observeOn } from 'rxjs/operators';

console.log('Before');

of(1, 2).pipe(
  observeOn(asyncScheduler)
).subscribe(value => console.log('observeOn:', value));

console.log('After');
observeOn: values are delivered asynchronously.
subscribeOn-example.ts
import { Observable, asyncScheduler } from 'rxjs';
import { subscribeOn } from 'rxjs/operators';

const source$ = new Observable(subscriber => {
  console.log('Source subscribed');
  subscriber.next('Data');
  subscriber.complete();
});

console.log('Before subscribe');

source$.pipe(
  subscribeOn(asyncScheduler)
).subscribe(value => console.log(value));

console.log('After subscribe');
subscribeOn: source subscription itself is scheduled asynchronously.

How does observeOn() differ from subscribeOn()?

Biggest Difference

Operator Controls
subscribeOn() Where Observable STARTS execution
observeOn() Where emitted VALUES continue execution
Remember: subscribeOn() affects Observable startup, while observeOn() affects downstream notifications.

Visual Flow

Observable Creation
        │
        │   ← subscribeOn()
        ▼
Source Emits Values
        │
        │   ← observeOn()
        ▼
Subscriber Receives Values

subscribeOn() Example

subscribeOn-example.ts
import { of, asyncScheduler } from 'rxjs';
import { subscribeOn, tap } from 'rxjs/operators';

console.log('Start');

of(1, 2, 3).pipe(

  tap(value => {
    console.log('Before subscribeOn:', value);
  }),

  subscribeOn(asyncScheduler)

).subscribe(value => {
  console.log('Subscriber:', value);
});

console.log('End');
Start
End
Before subscribeOn: 1
Subscriber: 1
Before subscribeOn: 2
Subscriber: 2
Before subscribeOn: 3
Subscriber: 3

Why?

subscribeOn(asyncScheduler) moves the ENTIRE Observable startup to async execution.

Important: subscribeOn() changes when the Observable starts.

observeOn() Example

observeOn-example.ts
import { of, asyncScheduler } from 'rxjs';
import { observeOn, tap } from 'rxjs/operators';

console.log('Start');

of(1, 2, 3).pipe(

  tap(value => {
    console.log('Before observeOn:', value);
  }),

  observeOn(asyncScheduler)

).subscribe(value => {
  console.log('Subscriber:', value);
});

console.log('End');
Start
Before observeOn: 1
Before observeOn: 2
Before observeOn: 3
End
Subscriber: 1
Subscriber: 2
Subscriber: 3

Why?

observeOn(asyncScheduler) changes downstream execution only. Source still emits synchronously, but subscriber receives values asynchronously.

Important: observeOn() affects operators and subscribers AFTER its position in the pipe.

Most Important Difference

subscribeOn() Controls Observable STARTUP execution.
observeOn() Controls notification DELIVERY execution.

observeOn() Affects Only Downstream

observeOn-downstream-example.ts
import { of, asyncScheduler } from 'rxjs';
import { observeOn, tap } from 'rxjs/operators';

of(1, 2, 3).pipe(

  tap(v => console.log('Sync:', v)),

  observeOn(asyncScheduler),

  tap(v => console.log('Async:', v))

).subscribe();

Sync: 2
Sync: 3

Async: 1
Async: 2
Async: 3

Easy Human Example

subscribeOn()

Controls when the kitchen starts cooking food.

observeOn()

Controls how and when the waiter serves food to customers.

Interview Answer

subscribeOn() controls which scheduler is used when the Observable starts executing, while observeOn() controls which scheduler is used for delivering emitted values to downstream operators and subscribers. subscribeOn() affects source subscription timing, whereas observeOn() affects execution after its position in the pipe.
