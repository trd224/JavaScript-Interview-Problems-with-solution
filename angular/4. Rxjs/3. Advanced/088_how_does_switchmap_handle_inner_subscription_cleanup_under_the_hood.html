How does switchMap handle inner subscription cleanup under the hood?

Advanced with headings, lists, full examples for every concept, colorful code, and expected behavior.

Definition

switchMap maps each source value to an inner Observable. When a new source value arrives, it unsubscribes from the previous inner Observable before subscribing to the new one.

Cleanup Behavior

  • Only one inner subscription is active at a time.
  • Previous inner subscription is cancelled on new source emission.
  • For Angular HttpClient, cancellation can abort the HTTP request.

Interview Point

switchMap does not wait for previous inner Observable to complete. It actively unsubscribes from it when switching.

Key Points

  • switchMap keeps latest inner Observable.
  • Old inner subscription is unsubscribed.
  • Useful for cancellation and stale response prevention.
Remember: switchMap cleanup = unsubscribe previous inner stream.

Full Understandable Example / Examples

switchMap-cleanup-example.ts
searchControl.valueChanges.pipe(
  debounceTime(300),
  switchMap(text => {
    console.log('Starting request for:', text);
    return http.get(`/api/search?q=${text}`);
  })
).subscribe(result => {
  console.log('Latest result:', result);
});
When new search text arrives, previous HTTP Observable is unsubscribed and only latest result is used.
manual-cleanup-analogy.ts
// Conceptually similar to:
let innerSubscription: Subscription | null = null;

source$.subscribe(value => {
  if (innerSubscription) {
    innerSubscription.unsubscribe();
  }

  innerSubscription = apiCall(value).subscribe(result => {
    console.log(result);
  });
});
