How would you build a real-time data stream with WebSockets using RxJS?

Advanced with headings, lists, full examples for every concept, colorful code, and expected behavior.

Definition

RxJS provides webSocket from rxjs/webSocket to create a WebSocketSubject. It allows sending and receiving WebSocket messages using Observable patterns.

Architecture

  • Create WebSocketSubject in a service.
  • Expose messages as Observable.
  • Keep the subject private.
  • Use retry/reconnect logic if needed.
  • Clean up connection when no longer needed.

Angular Best Practice

Expose Observable to components, not the raw WebSocketSubject, so components cannot accidentally push invalid messages.

Key Points

  • WebSocketSubject is both Observable and Observer.
  • Use a service wrapper.
  • Add reconnection/error handling for production.
Remember: WebSocket + RxJS gives real-time Observable streams.

Full Understandable Example / Examples

realtime.service.ts
import { Injectable } from '@angular/core';
import { webSocket, WebSocketSubject } from 'rxjs/webSocket';

@Injectable({ providedIn: 'root' })
export class RealtimeService {
  private socket$: WebSocketSubject<any> = webSocket('ws://localhost:8080');

  messages$ = this.socket$.asObservable();

  sendMessage(message: any) {
    this.socket$.next(message);
  }

  close() {
    this.socket$.complete();
  }
}
Incoming WebSocket messages are received as Observable values.
chat.component.ts
ngOnInit() {
  this.realtimeService.messages$.subscribe(message => {
    console.log('New message:', message);
  });

  this.realtimeService.sendMessage({
    type: 'JOIN',
    room: 'general'
  });
}
Outgoing messages are sent using next().
