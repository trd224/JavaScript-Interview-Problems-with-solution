How would you test RxJS code using marble testing?

Advanced with headings, lists, full examples for every concept, colorful code, and expected behavior.

Definition

Marble testing is a way to test Observable behavior over virtual time using string diagrams.

Why It Is Useful

  • Tests async streams synchronously using virtual time.
  • Makes timing-based operators testable.
  • Useful for debounceTime, delay, throttleTime, switchMap, and effects.

Marble Symbols

  • - frame/time passing
  • a,b,c emitted values
  • | completion
  • # error

Key Points

  • Marble tests use virtual time.
  • They make async Observable tests deterministic.
  • Useful for operators and NgRx effects.
Remember: Marble testing = testing Observables with time diagrams.

Full Understandable Example / Examples

marble-test-example.spec.ts
import { TestScheduler } from 'rxjs/testing';
import { map } from 'rxjs/operators';

it('should double values', () => {
  const scheduler = new TestScheduler((actual, expected) => {
    expect(actual).toEqual(expected);
  });

  scheduler.run(({ cold, expectObservable }) => {
    const source$ = cold('-a-b-c|', {
      a: 1,
      b: 2,
      c: 3
    });

    const result$ = source$.pipe(
      map(value => value * 2)
    );

    expectObservable(result$).toBe('-a-b-c|', {
      a: 2,
      b: 4,
      c: 6
    });
  });
});
Virtual stream input: 1,2,3
Expected output: 2,4,6
