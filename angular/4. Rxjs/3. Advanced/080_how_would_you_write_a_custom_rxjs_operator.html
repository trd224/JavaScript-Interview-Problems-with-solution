How would you write a custom RxJS operator?

Advanced with headings, lists, full examples for every concept, colorful code, and expected behavior.

Definition

A custom RxJS operator is a reusable function that takes an Observable as input and returns another Observable. In modern RxJS, custom operators are usually pipeable operators.

How To Create

  • Create a function that returns a function.
  • The inner function receives the source Observable.
  • Return source.pipe(...) with existing operators or return a new Observable manually.

Best Practice

Prefer composing existing operators unless you need full manual control.

Key Points

  • Custom operators improve reuse and readability.
  • Most custom operators compose existing operators.
  • Manual Observable creation is needed only for advanced behavior.
Remember: Custom operator = function(source$) => new/transformed Observable.

Full Understandable Example / Examples

custom-operator-compose.ts
import { Observable } from 'rxjs';
import { filter, map } from 'rxjs/operators';

function activeUserNames() {
  return (source$: Observable<any[]>) =>
    source$.pipe(
      map(users => users.filter(user => user.active)),
      map(users => users.map(user => user.name))
    );
}

this.http.get<any[]>('/api/users').pipe(
  activeUserNames()
).subscribe(names => {
  console.log('Active user names:', names);
});
activeUserNames returns filtered names.
custom-operator-manual.ts
import { Observable } from 'rxjs';

function multiplyBy(multiplier: number) {
  return (source$: Observable<number>) =>
    new Observable<number>(subscriber => {
      const subscription = source$.subscribe({
        next: value => subscriber.next(value * multiplier),
        error: error => subscriber.error(error),
        complete: () => subscriber.complete()
      });

      return () => subscription.unsubscribe();
    });
}
multiplyBy transforms each number and cleans up subscription.
