What is connect() operator (RxJS 7+) and how does it differ from the old multicasting approach?

Advanced interview-level answer with headings, lists, full examples for every concept, colorful code, and expected behavior.

Interview-Level Answer

Definition

The connect operator provides a modern way to multicast a source inside a pipe without manually handling a ConnectableObservable in many cases.

Old Approach

  • Use publish/connectable.
  • Manually call connect().
  • Manage connection lifecycle yourself.

Modern Approach

connect operator allows scoped multicasting logic inside a pipe and is more composable with modern RxJS patterns.

Key Points

  • connect operator supports modern composable multicasting.
  • Old approach required manual connectable handling.
  • share/shareReplay are still simpler for common cases.
Remember: connect operator is modern scoped multicasting.

Full Understandable Example / Examples

connect-operator-concept.ts
import { interval } from 'rxjs';
import { connect, map, take } from 'rxjs/operators';

const result$ = interval(1000).pipe(
  take(3),
  connect(shared$ =>
    shared$.pipe(
      map(value => `Shared value: ${value}`)
    )
  )
);

result$.subscribe(value => console.log('A:', value));
result$.subscribe(value => console.log('B:', value));

Expected Output / Behavior

Both subscribers receive values derived from shared source.
