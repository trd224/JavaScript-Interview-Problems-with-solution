What is forkJoin() behavior when one Observable never completes?

Structured with headings, lists, colorful code, and examples for every operator/concept mentioned.

Definition

If one Observable inside forkJoin never completes, forkJoin never emits because it waits for all sources to complete.

Solution

  • Use take(1) for streams that emit but do not complete.
  • Avoid passing infinite streams like interval directly to forkJoin.

Key Points

  • forkJoin requires all sources to complete.
  • Infinite streams must be limited with take().
Remember: forkJoin is best for finite Observables like HTTP calls.

Full Understandable Example / Examples

forkJoin-never-completes-example.ts
import { forkJoin, interval, of } from 'rxjs';
import { take } from 'rxjs/operators';

forkJoin({
  name: of('Rizwan'),
  tick: interval(1000).pipe(take(1))
}).subscribe(result => {
  console.log(result);
});
{ name: 'Rizwan', tick: 0 }
