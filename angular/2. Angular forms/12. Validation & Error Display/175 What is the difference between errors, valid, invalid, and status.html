What is the difference between errors, valid, invalid, and status?

These properties are used to check the validation state of an Angular form control.

Reactive Form Example


form = this.fb.group({

  username: [

    '',

    [

      Validators.required,

      Validators.minLength(5)

    ]

  ]

});

Main Difference

Property Purpose
errors Contains validation error object
valid Returns true if control is valid
invalid Returns true if control is invalid
status Returns overall status string

1. errors

Returns an object containing validation errors. 


control.errors

Example: 


{
  required: true,
  minlength: {
    requiredLength: 5,
    actualLength: 2
  }
}

If no errors exist: 


null

2. valid

Returns: 


true

when all validations pass. 


control.valid

3. invalid

Returns: 


true

when validation fails. 


control.invalid

4. status

Returns overall validation state as a string. 


control.status

Possible values:

  • VALID
  • INVALID
  • PENDING
  • DISABLED

HTML View


<div>
  Errors:
  {{ form.get('username')?.errors | json }}
</div>

<div>
  Valid:
  {{ form.get('username')?.valid }}
</div>

<div>
  Invalid:
  {{ form.get('username')?.invalid }}
</div>

<div>
  Status:
  {{ form.get('username')?.status }}
</div>

Output Example

Errors: { required: true }
Valid: false
Invalid: true
Status: INVALID

Main Point

  • errors → gives error details object
  • valid → boolean valid state
  • invalid → boolean invalid state
  • status → complete status string
Example: A control can have: 

valid = false
invalid = true
status = "INVALID"
errors = { required: true }
