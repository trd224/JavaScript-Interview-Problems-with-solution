What are the validation states of a form control?

Angular form controls have validation states that show the current validation status of the control.

Main Validation States

  • valid
  • invalid
  • pending

Reactive Form Example


form = this.fb.group({

  username: [

    '',

    [Validators.required],

    [this.usernameValidator()]

  ]

});

Why Validators Come in Separate Blocks?

Angular FormControl structure is: 


[value, syncValidators, asyncValidators]

✔ Correct


username: [

  '',

  [Validators.required],

  [this.usernameValidator()]

]

Validators.required → sync validator

this.usernameValidator() → async validator

✘ Wrong


username: [

  '',

  [

    Validators.required,

    this.usernameValidator()

  ]

]

Angular will treat both as synchronous validators.

Why?

Async validators usually call APIs or servers. 


this.usernameValidator()

That operation takes time, so Angular handles async validators separately.

FormControl Parameter Meaning

Position Meaning
1st Initial Value
2nd Sync Validators
3rd Async Validators

HTML View


<input
  type="text"
  formControlName="username"
>

<div
  class="valid"
  *ngIf="form.get('username')?.valid"
>
  Valid
</div>

<div
  class="invalid"
  *ngIf="form.get('username')?.invalid"
>
  Invalid
</div>

<div
  class="pending"
  *ngIf="form.get('username')?.pending"
>
  Checking...
</div>

Output Example

Checking...
Valid
Invalid

Meaning of Each State

1. valid

The control passed all validators. 


control.valid

2. invalid

The control failed one or more validators. 


control.invalid

3. pending

Async validation is currently running. 


control.pending

Example Flow

  • User types username
  • Angular starts async validator
  • Control becomes pending
  • If validation passes → valid
  • If validation fails → invalid
Main Point: pending state is mostly used with async validators.
