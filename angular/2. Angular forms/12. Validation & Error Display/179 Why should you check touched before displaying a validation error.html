Why should you check touched before displaying a validation error?

You check touched so the error message does not show immediately when the page loads.

HTML View


<input
  type="text"
  formControlName="username"
>

<div
  *ngIf="
    form.get('username')?.invalid &&
    form.get('username')?.touched
  "
>
  Username is required
</div>

Meaning

Error will show only when:

  • Field is invalid
  • User has touched and left the field
Main Point: touched improves user experience because errors appear after user interaction.
