How do you create a custom validator directive for template-driven forms?

In template-driven forms, a custom validator is usually created using a directive and registered with NG_VALIDATORS.

The directive adds custom validation logic directly to HTML elements.

🔹 Step 1: Create Custom Validator Directive


import { Directive } from '@angular/core';

import {
  AbstractControl,
  NG_VALIDATORS,
  ValidationErrors,
  Validator
} from '@angular/forms';

@Directive({
  selector: '[appNoAdmin]',
  providers: [
    {
      provide: NG_VALIDATORS,
      useExisting: NoAdminValidatorDirective,
      multi: true
    }
  ]
})

export class NoAdminValidatorDirective implements Validator {

  validate(
    control: AbstractControl
  ): ValidationErrors | null {

    if (control.value === 'admin') {
      return { noAdmin: true };
    }

    return null;
  }

}

🔹 Explanation of NG_VALIDATORS Provider

This provider object registers the directive as a validator in Angular forms. 


{
  provide: NG_VALIDATORS,
  useExisting: NoAdminValidatorDirective,
  multi: true
}
Property Meaning
provide: NG_VALIDATORS Registers this directive inside Angular's synchronous validator list.
useExisting: NoAdminValidatorDirective Tells Angular to use this existing directive class as the validator.
multi: true Allows this validator to work with other validators like required, instead of replacing them.
Without multi: true, your custom validator may replace other validators.

🔹 Real Example with Multiple Validators


<input
  type="text"
  name="username"
  ngModel
  required
  appNoAdmin
  #username="ngModel"
>

In this example, Angular applies both validators:

  • required validator
  • appNoAdmin custom validator

🔹 Step 2: Use Validator in Template-Driven Form


<form #userForm="ngForm">

  <input
    type="text"
    name="username"
    ngModel
    appNoAdmin
    #username="ngModel"
  >

  <div *ngIf="username.errors?.['noAdmin']">
    Username cannot be admin
  </div>

</form>

🔹 How it Works

  • appNoAdmin is the custom validator directive.
  • Angular calls the validate() method automatically.
  • NG_VALIDATORS tells Angular this directive is a validator.
  • useExisting uses the same directive instance.
  • multi: true allows multiple validators together.
  • If validation fails, an error object is returned.
  • If validation passes, null is returned.

🔹 Validation Result

Input Value Returned Value Status
admin { noAdmin: true } Invalid
john null Valid

🔹 Summary

  • Template-driven custom validators are usually directives.
  • They use NG_VALIDATORS.
  • provide: NG_VALIDATORS registers validator with Angular Forms.
  • useExisting tells Angular to use the directive class.
  • multi: true allows this validator with other validators.
  • Validation logic is written inside validate().
  • Directive is applied directly in HTML.
  • Return null for valid input.
  • Return error object for invalid input.
