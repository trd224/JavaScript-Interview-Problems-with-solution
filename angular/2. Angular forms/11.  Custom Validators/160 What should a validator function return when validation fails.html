What should a validator function return when validation fails?

When validation fails, a validator function should return: 


{ errorName: true }
❌ An error object means the form control is invalid.

🔹 Example Custom Validator


import { AbstractControl, ValidationErrors } from '@angular/forms';

export function noAdminValidator(
  control: AbstractControl
): ValidationErrors | null {

  if (control.value === 'admin') {
    return { noAdmin: true };
  }

  return null;
}

🔹 Return Values Meaning

Return Value Meaning
null Validation passed
{ noAdmin: true } Validation failed

🔹 Real Example

If input value is: 


'admin'

Validator returns: 


{ noAdmin: true }

Angular marks the form control as invalid.

🔹 Accessing Validation Error


<div *ngIf="userForm.get('username')?.errors?.noAdmin">
  Username cannot be admin
</div>

🔹 Summary

  • Validation failure returns an error object.
  • Error object contains custom error key.
  • Angular stores the error inside the errors property.
  • Returned object makes the control invalid.
