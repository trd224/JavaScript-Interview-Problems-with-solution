What should a validator function return when validation passes?

When validation passes, a validator function should return: 


null
✅ null means the form control is valid and has no validation errors.

🔹 Example Custom Validator


import { AbstractControl, ValidationErrors } from '@angular/forms';

export function noAdminValidator(
  control: AbstractControl
): ValidationErrors | null {

  if (control.value === 'admin') {
    return { noAdmin: true };
  }

  return null;
}

🔹 Return Values Meaning

Return Value Meaning
null Validation passed
{ errorName: true } Validation failed

🔹 Real Example

If input value is: 


'john'

Validator returns: 


null

If input value is: 


'admin'

Validator returns: 


{ noAdmin: true }

🔹 Summary

  • null means validation success.
  • Error object means validation failed.
  • Angular checks returned object to mark control invalid.
  • All custom validators follow this pattern.
