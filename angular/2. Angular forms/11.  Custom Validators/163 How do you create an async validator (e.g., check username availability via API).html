Async Validator in Angular

An async validator is used when validation depends on an API or server response.

Example: Check whether a username is already taken or available.

Real Flow

Step What Happens
1 User types username: admin
2 Angular calls async validator
3 Validator calls API/service
4 API checks username in database
5 If username exists, validator returns error

1. User Service

This service simulates an API call.


import { Injectable } from '@angular/core';
import { Observable, of, delay } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class UserService {

  checkUsername(username: string): Observable<boolean> {

    const takenUsers = ['admin', 'rizwan', 'john'];

    const isAvailable = !takenUsers.includes(username);

    return of(isAvailable).pipe(delay(2000));
  }

}

Meaning

Username Result Meaning
admin false Username already taken
sahil true Username available

2. Component TypeScript File


import { Component } from '@angular/core';
import {
  AbstractControl,
  AsyncValidatorFn,
  FormBuilder,
  FormGroup,
  ValidationErrors,
  Validators
} from '@angular/forms';

import { Observable, map } from 'rxjs';
import { UserService } from './user.service';

@Component({
  selector: 'app-register',
  templateUrl: './register.component.html'
})
export class RegisterComponent {

  form: FormGroup;

  constructor(
    private fb: FormBuilder,
    private userService: UserService
  ) {

    this.form = this.fb.group({

      username: [
        '',
        [Validators.required],
        [this.usernameValidator.bind(this)]
      ]

    });

  }

  usernameValidator(control: AbstractControl)
  : Observable<ValidationErrors | null> {

    return this.userService.checkUsername(control.value).pipe(

      map(isAvailable => {

        if (isAvailable) {
          return null;
        }

        return { usernameTaken: true };

      })

    );

  }

  submitForm() {
    console.log(this.form.value);
  }

}

3. HTML Template


<form [formGroup]="form" (ngSubmit)="submitForm()">

  <label>Username</label>

  <input type="text" formControlName="username">

  <div *ngIf="form.get('username')?.pending">
    Checking username...
  </div>

  <div *ngIf="form.get('username')?.errors?.['required']">
    Username is required
  </div>

  <div *ngIf="form.get('username')?.errors?.['usernameTaken']">
    Username already taken
  </div>

  <button type="submit" [disabled]="form.invalid || form.pending">
    Register
  </button>

</form>

4. Important Syntax


username: [
  '',
  [Validators.required],
  [this.usernameValidator.bind(this)]
]
Part Meaning
'' Initial value
[Validators.required] Synchronous validator
[this.usernameValidator.bind(this)] Async validator

5. Why bind(this)?

bind(this) is used because the validator uses: 


this.userService.checkUsername(control.value)

If we do not use bind(this), Angular may call the function without component context. Then this.userService becomes undefined.

Without bind(this)


username: [
  '',
  [Validators.required],
  [this.usernameValidator]
]

This can cause error:


Cannot read properties of undefined

With bind(this)


username: [
  '',
  [Validators.required],
  [this.usernameValidator.bind(this)]
]

Now the validator remembers the current component object. So this works correctly: 


this.userService.checkUsername(control.value)

6. JavaScript Example of bind(this)

Without bind


const person = {
  name: 'Rizwan',

  show() {
    console.log(this.name);
  }
};

const fn = person.show;

fn();

Output:


undefined

With bind


const fn = person.show.bind(person);

fn();

Output:


Rizwan

Short Definition

An async validator in Angular is a validator that checks form value asynchronously, usually by calling an API, and returns null for valid or an error object for invalid.
