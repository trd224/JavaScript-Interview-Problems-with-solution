What is a Custom Validator in Angular?

In Angular, a custom validator is a function you create to define your own validation logic for form controls beyond built-in validators like required, minlength, or email.

A custom validator returns:
  • null → when validation passes
  • Error Object → when validation fails

🔹 Basic Structure of a Custom Validator


import { AbstractControl, ValidationErrors } from '@angular/forms';

export function myCustomValidator(
  control: AbstractControl
): ValidationErrors | null {

  const value = control.value;

  if (value !== 'expectedValue') {
    return { invalidValue: true };
  }

  return null;
}

🔹 Example: Username Cannot Be "admin"


import { AbstractControl, ValidationErrors } from '@angular/forms';

export function forbiddenNameValidator(
  control: AbstractControl
): ValidationErrors | null {

  if (control.value === 'admin') {
    return { forbiddenName: true };
  }

  return null;
}

🔹 Using Custom Validator in Reactive Form


import { FormControl, FormGroup } from '@angular/forms';

this.myForm = new FormGroup({
  username: new FormControl('', [forbiddenNameValidator])
});

🔹 HTML Template


<form [formGroup]="myForm">

  <input type="text" formControlName="username">

  <div *ngIf="myForm.get('username')?.errors?.forbiddenName">
    Username cannot be "admin"
  </div>

</form>

🔹 Custom Validator with Parameters


export function minAgeValidator(minAge: number) {

  return (control: AbstractControl): ValidationErrors | null => {

    return control.value < minAge
      ? { minAge: true }
      : null;

  };

}

🔹 Using Parameterized Validator


age: new FormControl('', [minAgeValidator(18)])

🔹 Cross-Field Validator Example


export function passwordMatchValidator(
  group: AbstractControl
): ValidationErrors | null {

  const password = group.get('password')?.value;
  const confirmPassword = group.get('confirmPassword')?.value;

  return password === confirmPassword
    ? null
    : { passwordMismatch: true };

}

🔹 Summary

  • Custom validators are user-defined validation functions.
  • Return null when valid.
  • Return an error object when invalid.
  • Can validate:
    • Single FormControl
    • Entire FormGroup
  • Useful for business-specific validation rules.
