Angular ngModel & Checkbox

Cheatsheet — two-way binding, forms, custom values, and submit

1. Single Checkbox — Boolean Binding

A single checkbox with ngModel binds to a boolean value representing its checked state.

HTML 
<input type="checkbox" [(ngModel)]="isChecked">
TS 
isChecked = false;
  • Checked → isChecked = true
  • Unchecked → isChecked = false

2. Inside a Form

When placed inside a <form>, the name attribute is required.

HTML 
<form #f="ngForm">
  <input type="checkbox" name="agree" [(ngModel)]="isChecked">
</form>
⚠ The name attribute is required on all inputs inside <form> when using ngModel.

3. Custom True / False Values

Use [trueValue] and [falseValue] to bind custom strings instead of booleans.

HTML 
<input
  type="checkbox"
  [(ngModel)]="status"
  [trueValue]="'YES'"
  [falseValue]="'NO'"
>
TS 
status = 'NO';
Checked → 'YES' Unchecked → 'NO'

4. Full Example — Text Input + Checkbox + Submit

Combining a text input and checkbox inside a form with an ngSubmit handler.

HTML 
<form #myForm="ngForm" (ngSubmit)="onSubmit(myForm)">

  <!-- Text Input -->
  <label>Name:</label>
  <input
    type="text"
    name="username"
    [(ngModel)]="user.name"
    required
  >

  <!-- Checkbox -->
  <label>
    <input
      type="checkbox"
      name="agree"
      [(ngModel)]="user.agree"
    >
    Accept Terms
  </label>

  <button type="submit">Submit</button>

</form>
TS 
user = {
  name: '',
  agree: false
};

onSubmit(form: any) {
  console.log('Form Value:', form.value);
  console.log('User Model:', this.user);
}

5. ngModel Without a Variable (Submit-Only)

Use ngModel alone (without [()]) when you only need the value at submit time and don't need a TypeScript variable.

When to use this

  • You only need the value at submit time
  • You don't need a TS variable like this.username
  • Value is accessed via form.value
HTML 
<form #f="ngForm">
  <input type="text" name="username" ngModel>
</form>
TS 
onSubmit(form: any) {
  console.log(form.value.username);
}

Key difference

  • ✔ Value comes from form.value
  • ✘ No direct TS variable like this.username
ngModel alone → form.value only [(ngModel)] → two-way bound TS variable Missing name → form won't register it
