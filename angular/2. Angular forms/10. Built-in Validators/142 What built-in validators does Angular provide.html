Built-in Validators in Angular

Angular provides built-in validators through the Validators class for reactive forms and HTML validation attributes for template-driven forms.

✅ Common Built-in Validators in Angular

Validator Purpose
required Field cannot be empty
requiredTrue Checkbox must be checked
min Minimum numeric value
max Maximum numeric value
minLength Minimum text length
maxLength Maximum text length
email Valid email format
pattern Custom regex validation
nullValidator No validation
compose() Combine multiple validators

✅ Real Example (Reactive Form)

🔹 TypeScript 
import { FormGroup, FormControl, Validators } from '@angular/forms';

export class AppComponent {

  userForm = new FormGroup({

    username: new FormControl('', [
      Validators.required,
      Validators.minLength(3),
      Validators.maxLength(10)
    ]),

    age: new FormControl('', [
      Validators.required,
      Validators.min(18),
      Validators.max(60)
    ]),

    email: new FormControl('', [
      Validators.required,
      Validators.email
    ]),

    password: new FormControl('', [
      Validators.required,
      Validators.pattern(
        '^(?=.*[A-Z])(?=.*[0-9]).{8,}$'
      )
    ]),

    terms: new FormControl(false, [
      Validators.requiredTrue
    ])

  });

}
🔹 HTML 
<form [formGroup]="userForm">

  <input type="text" formControlName="username">

  <input type="number" formControlName="age">

  <input type="email" formControlName="email">

  <input type="password" formControlName="password">

  <input type="checkbox" formControlName="terms">

</form>

✅ Validator Examples

🔹 required 

Validators.required

Field must not be empty.

🔹 requiredTrue 

Validators.requiredTrue

Mostly used for checkboxes.

🔹 min / max 

Validators.min(18)
Validators.max(60)

Checks numeric range.

🔹 minLength / maxLength 

Validators.minLength(3)
Validators.maxLength(10)

Checks text length.

🔹 email 

Validators.email

Validates email format.

🔹 pattern

✅ compose() Example

Validators.compose() is used to combine multiple validators into a single validator function.

🔹 TypeScript 
import { FormGroup, FormControl, Validators } from '@angular/forms';

export class AppComponent {

  userForm = new FormGroup({

    username: new FormControl(
      '',
      Validators.compose([

        Validators.required,
        Validators.minLength(3),
        Validators.maxLength(10)

      ])
    )

  });

}
🔹 HTML 
<form [formGroup]="userForm">

  <input
    type="text"
    formControlName="username"
    placeholder="Enter username"
  >

</form>

✅ Validation Rules

Rule Meaning
required Cannot be empty
minLength(3) Minimum 3 characters
maxLength(10) Maximum 10 characters

✅ Best Practice

Nowadays Angular developers mostly use: 

[
  Validators.required,
  Validators.email
]

instead of: 

Validators.compose()

because array syntax is shorter and cleaner.

Most commonly used validators: 

Validators.required
Validators.email
Validators.minLength()
Validators.pattern()

These cover most real-world form validation needs.
