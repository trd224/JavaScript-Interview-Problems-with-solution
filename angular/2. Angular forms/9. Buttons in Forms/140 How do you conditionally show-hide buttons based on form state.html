Conditionally Show/Hide Buttons Based on Form State in Angular

In Angular, you can conditionally show or hide buttons using:

  • form.valid
  • form.invalid
  • form.dirty
  • form.pristine
  • form.touched
  • form.untouched
  • form.submitted

with: 

*ngIf

or 

[hidden]

✅ Real Example (Template-Driven Form)

🔹 HTML 
<form #userForm="ngForm">

  <input
    type="text"
    name="username"
    ngModel
    required
    placeholder="Enter username"
  >

  <br><br>

  <!-- Show only when form is valid -->
  <button
    *ngIf="userForm.valid"
    type="submit"
  >
    Submit
  </button>

  <!-- Show only when form is invalid -->
  <button
    *ngIf="userForm.invalid"
    type="button"
  >
    Fill Required Fields
  </button>

</form>

✅ How It Works

🔹 Initially

Input is empty → form becomes invalid

So Angular shows: 

Fill Required Fields

🔹 After Entering Value

If user types: 

Rizwan

Form becomes valid.

Angular hides: 

Fill Required Fields

and shows: 

Submit

✅ Common Form States

Form State Meaning
form.valid All validations passed
form.invalid Form has validation errors
form.dirty User changed value
form.pristine User did not change anything
form.touched Input was focused & left
form.untouched Input never touched
form.submitted Form submitted

✅ Example Using dirty 

<button *ngIf="userForm.dirty">
  Save Changes
</button>

Button appears only after user changes something.

✅ Example Using touched 

<button *ngIf="userForm.touched">
  Continue
</button>

Button appears after user interacts with form.

✅ *ngIf vs [hidden]

Method Removes From DOM Recommended
*ngIf ✅ Yes ✅ Best
[hidden] ❌ No Sometimes

✅ Using [hidden] 

<button [hidden]="userForm.invalid">
  Submit
</button>

Button stays in DOM but becomes hidden.

✅ Best Practice

Usually Angular developers use: 

*ngIf="form.valid"

for conditional button rendering.
