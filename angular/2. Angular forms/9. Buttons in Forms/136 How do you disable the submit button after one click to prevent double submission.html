How Do You Disable the Submit Button After One Click to Prevent Double Submission?

In Angular, double submission is usually prevented using a loading flag like: 

isSubmitting = false;

When the user clicks submit:

  • Button becomes disabled
  • User cannot click again
  • Prevents duplicate API requests

✅ TypeScript Example

isSubmitting = false;

onSubmit() {

  if (this.isSubmitting) {
    return;
  }

  this.isSubmitting = true;

  this.http.post('api/save', data)
    .subscribe({

      next: (res) => {

        console.log(res);

        this.isSubmitting = false;
      },

      error: (err) => {

        console.log(err);

        this.isSubmitting = false;
      }

    });
}

✅ HTML Example

<button
  type="submit"
  [disabled]="isSubmitting"
>

  {{ isSubmitting ? 'Submitting...' : 'Submit' }}

</button>

✅ How It Works

  • First click submits the form
  • Button immediately becomes disabled
  • Second click is blocked
  • Button enabled again after API completes

✅ Live UI Example

Important: Always reset: isSubmitting = false after:
  • success response
  • error response
Otherwise the button may remain disabled forever.

✅ Better Approach Using finalize()

Instead of repeating: this.isSubmitting = false, you can use RxJS finalize(). 

import { finalize } from 'rxjs/operators';

isSubmitting = false;

onSubmit() {

  if (this.isSubmitting) {
    return;
  }

  this.isSubmitting = true;

  this.http.post('api/save', data)
    .pipe(

      finalize(() => {
        this.isSubmitting = false;
      })

    )
    .subscribe({

      next: (res) => {
        console.log(res);
      },

      error: (err) => {
        console.log(err);
      }

    });
}

Result:

  • Cleaner code
  • Prevents duplicate submission
  • Automatically re-enables button

✅ Combined Validation + Submission Check

<button
  type="submit"
  [disabled]="myForm.invalid || isSubmitting"
>

  {{ isSubmitting ? 'Submitting...' : 'Submit' }}

</button>

This disables the button when:

  • Form is invalid
  • OR submission is already in progress
