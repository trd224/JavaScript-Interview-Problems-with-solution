Difference Between type="submit", type="button" and type="reset"

Buttons inside a form can perform different actions depending on their type.

Type Purpose Behavior
submit Submit the form Triggers form submission and ngSubmit
button Normal button Does nothing automatically
reset Reset form fields Restores initial form values

✅ 1. type="submit"

Used to submit the form. 

<form (ngSubmit)="onSubmit()">

  <input type="text">

  <button type="submit">
    Submit
  </button>

</form>

When clicked:

  • Form submits
  • ngSubmit executes
  • Validation runs
Important: <button> without type is automatically type="submit".

✅ 2. type="button"

Used for normal actions. 

<button type="button" (click)="openPopup()">
  Open Popup
</button>

When clicked:

  • Only click event runs
  • Form does NOT submit

Commonly used for:

  • Popup
  • Add row
  • Remove item
  • Custom logic

✅ 3. type="reset"

Used to reset form fields.

If you use: 

<button type="reset">
  Reset
</button>

then you do NOT need any function.

The browser automatically resets the form fields.

✅ Example

<form>

  <input type="text" value="Rizwan">

  <button type="reset">
    Reset
  </button>

</form>

When clicked:

  • Form fields return to initial values
  • No TypeScript function needed
  • No (click) event needed
Angular Note: In Angular Reactive Forms, developers usually prefer using form.reset() because it can also reset validation state, touched/dirty state, and custom form values.

✅ Reactive Form Example

resetForm() {
  this.myForm.reset();
}
<button type="button" (click)="resetForm()">
  Reset
</button>

✅ Complete Example

<form (ngSubmit)="onSubmit()">

  <input type="text">

  <button type="submit">
    Submit
  </button>

  <button type="button" (click)="showMessage()">
    Show Message
  </button>

  <button type="reset">
    Reset
  </button>

</form>
