How Do You Use (click) vs (submit) for Form Submission?

In Angular, form submission is usually handled using: 

(ngSubmit)

instead of: 

(click)

✅ Difference Between (click) and (submit)

Event Used On Purpose
(click) Button Handles button click only
(ngSubmit) Form Handles complete form submission

❌ Using (click) for Form Submission

<form>

  <input type="text">

  <button
    type="submit"
    (click)="onSubmit()"
  >
    Submit
  </button>

</form>

Problems:

  • Only button click is handled
  • Pressing Enter may behave differently
  • Not proper Angular form submission handling

✅ Correct Way Using (ngSubmit)

<form (ngSubmit)="onSubmit()">

  <input type="text">

  <button type="submit">
    Submit
  </button>

</form>

Benefits:

  • Handles complete form submission
  • Works with Enter key
  • Works properly with Angular forms
  • Best practice in Angular

✅ Template-Driven Form Example

<form
  #myForm="ngForm"
  (ngSubmit)="onSubmit()"
>

  <input
    type="text"
    name="username"
    ngModel
    required
  >

  <button type="submit">
    Submit
  </button>

</form>

✅ Reactive Form Example

<form
  [formGroup]="myForm"
  (ngSubmit)="onSubmit()"
>

  <input
    type="text"
    formControlName="username"
  >

  <button type="submit">
    Submit
  </button>

</form>
Important: Use: (click) for normal button actions like:
  • Open popup
  • Add item
  • Delete row
  • Toggle password
Use: (ngSubmit) for form submission.

✅ Example of Normal Button

<button
  type="button"
  (click)="openPopup()"
>
  Open Popup
</button>
