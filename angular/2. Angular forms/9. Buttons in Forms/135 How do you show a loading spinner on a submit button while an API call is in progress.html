How Do You Show a Loading Spinner on a Submit Button While an API Call Is in Progress?

In Angular, a loading spinner is usually shown using a boolean variable like: 

isLoading = false;

When the API starts: 

this.isLoading = true;

When the API completes: 

this.isLoading = false;

✅ TypeScript Example

isLoading = false;

onSubmit() {

  this.isLoading = true;

  this.http.post('api/login', data)
    .subscribe({
      next: (res) => {

        console.log(res);

        this.isLoading = false;
      },

      error: (err) => {

        console.log(err);

        this.isLoading = false;
      }
    });
}

✅ HTML Example

<button
  type="submit"
  [disabled]="isLoading"
>

  <span *ngIf="isLoading" class="spinner"></span>

  {{ isLoading ? 'Loading...' : 'Submit' }}

</button>

✅ How It Works

  • Button becomes disabled during API call
  • Spinner appears
  • Button text changes to Loading...
  • Button enabled again after API completes

✅ Live UI Example

Important: Always stop loading in both:
  • success response
  • error response
Otherwise spinner may continue forever.

✅ Better Approach Using finalize()

Instead of writing: this.isLoading = false multiple times, you can use RxJS finalize(). 

import { finalize } from 'rxjs/operators';

onSubmit() {

  this.isLoading = true;

  this.http.post('api/login', data)
    .pipe(

      finalize(() => {
        this.isLoading = false;
      })

    )
    .subscribe({
      next: (res) => {
        console.log(res);
      },

      error: (err) => {
        console.log(err);
      }
    });
}

Result:

  • Cleaner code
  • Loading automatically stops
  • No duplicate loading code
