Display Error if Textarea is Empty on Form Submit

A validation error for a textarea is displayed on form submit by checking whether the textarea is invalid and the form is submitted.

✅ Template-driven Form Example

HTML

<form #form="ngForm" (ngSubmit)="onSubmit(form)">

  <textarea
    name="message"
    [(ngModel)]="message"
    required
    #msg="ngModel"
  ></textarea>

  <!-- Error Message -->
  <div *ngIf="form.submitted && msg.invalid">
    Message is required
  </div>

  <button type="submit">
    Submit
  </button>

</form>
TS

message = '';

onSubmit(form: any) {

  if (form.valid) {
    console.log(form.value);
  }

}

✅ How it works

  • required → makes textarea mandatory
  • #msg="ngModel" → gives textarea validation state
  • form.submitted → becomes true after submit
  • msg.invalid → checks textarea validation

✅ Output

• Empty textarea → shows error message

• Filled textarea → form submits successfully

✅ Reactive Form Example

HTML

<form [formGroup]="form" (ngSubmit)="onSubmit()">

  <textarea
    formControlName="message"
  ></textarea>

  <!-- Error -->
  <div *ngIf="submitted && form.controls['message'].invalid">
    Message is required
  </div>

  <button type="submit">
    Submit
  </button>

</form>
TS

import {
  FormGroup,
  FormControl,
  Validators
} from '@angular/forms';

submitted = false;

form = new FormGroup({
  message: new FormControl(
    '',
    Validators.required
  )
});

onSubmit() {

  this.submitted = true;

  if (this.form.valid) {
    console.log(this.form.value);
  }

}
