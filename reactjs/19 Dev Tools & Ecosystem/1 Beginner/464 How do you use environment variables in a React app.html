464. How Do You Use Environment Variables in a React App?

One-Line Definition

Environment variables are configuration values stored outside the source code and accessed inside a React application for settings such as API URLs, environment names, and feature flags.

Why Do We Use Environment Variables?

Different environments usually require different configuration values.

Environment API URL
Development http://localhost:5000
Testing https://test-api.example.com
Production https://api.example.com
Instead of changing source code for each environment, we store these values in environment variables.

Using Environment Variables in Vite

Create .env File

VITE_API_URL=https://api.example.com

VITE_APP_NAME=My React App
In Vite, all public environment variables must start with VITE_.

Access Variable

const apiUrl = import.meta.env.VITE_API_URL;

console.log(apiUrl);

Output

https://api.example.com

Using Environment Variables in CRA

Create .env File

REACT_APP_API_URL=https://api.example.com

REACT_APP_APP_NAME=My React App
In Create React App, all public environment variables must start with REACT_APP_.

Access Variable

const apiUrl = process.env.REACT_APP_API_URL;

console.log(apiUrl);

React Example

function App() {

  const apiUrl = import.meta.env.VITE_API_URL;

  return (

    <h1>

      {apiUrl}

    </h1>

  );

}

export default App;

Using Environment Variables in API Calls

const API_URL = import.meta.env.VITE_API_URL;

fetch(`${API_URL}/users`)
  .then(res => res.json())
  .then(data => console.log(data));

Can We Store Secrets?

❌ No.

Environment variables used in React frontend applications are bundled into JavaScript and sent to the browser.

Users can inspect them through browser developer tools.

Do Not Store

DATABASE_PASSWORD=mysecret

JWT_SECRET=mysecret

STRIPE_SECRET_KEY=xxxxx

Safe Examples

VITE_API_URL=https://api.example.com

VITE_APP_NAME=My App

VITE_ENVIRONMENT=production

Real Project Example

In our Aadhaar Authentication application, environment variables are used to store values such as API Base URL, SignalR Hub URL, Authentication Service URL, Environment Name, and Feature Flags so the same codebase can run in UAT, Pre-Production, and Production environments.

Important Interview Points

  • Environment variables store configuration values.
  • Vite uses the VITE_ prefix.
  • CRA uses the REACT_APP_ prefix.
  • Frontend environment variables are publicly visible.
  • Never store secrets in frontend environment variables.

Short Interview Answer

Environment variables allow us to configure React applications without hardcoding values. In Vite they are accessed through import.meta.env, while CRA uses process.env.

One-Line Interview Answer

Environment variables provide configuration values to a React application and are accessed using import.meta.env (Vite) or process.env (CRA).
