299. What are React 19 Form Actions and how do they simplify form handling?

One-Line Answer

React 19 Form Actions allow forms to directly call async functions through the form's action prop, eliminating much of the manual onSubmit, state management, and loading logic.

The Old Way (Before React 19)

Traditionally, forms required:

  • onSubmit handler
  • preventDefault()
  • Loading state
  • Error state
  • Success state
  • Manual API calls
Old Form Handling
function SignupForm(){

  const [loading,setLoading]=useState(false);

  async function handleSubmit(e){

    e.preventDefault();

    setLoading(true);

    const formData=new FormData(e.target);

    await saveUser(formData);

    setLoading(false);

  }

  return (
    <form onSubmit={handleSubmit}>
      <input name="email"/>
      <button disabled={loading}>
        {loading?'Saving...':'Save'}
      </button>
    </form>
  );

}
Lots of boilerplate code just to submit a form.

React 19 Form Actions

Now the form can directly call an async function.

React 19 Form Action
async function saveUser(formData){

  const email=formData.get('email');

  await fetch('/api/users',{
    method:'POST',
    body:JSON.stringify({email})
  });

}

function SignupForm(){

  return (
    <form action={saveUser}>
      <input name="email"/>
      <button>Save</button>
    </form>
  );

}
No onSubmit. No preventDefault. No manual FormData creation.

What Happens Internally?

User Clicks Submit
↓ React Collects FormData
↓ Calls saveUser(formData)
↓ Waits For Promise
↓ Updates UI

Real-World Example

Imagine a login form.

Login Form
async function login(formData){

  const email=formData.get('email');
  const password=formData.get('password');

  await fetch('/api/login',{
    method:'POST',
    body:JSON.stringify({
      email,
      password
    })
  });

}

export default function Login(){

  return (
    <form action={login}>
      <input name="email"/>
      <input type="password" name="password"/>
      <button>Login</button>
    </form>
  );

}

Works Great With useActionState

React 19 introduced useActionState() specifically for Form Actions.

SignupForm.jsx
import {useActionState} from 'react';

async function createUser(prevState,formData){

  const name=formData.get('name');

  if(!name){
    return 'Name is required';
  }

  return 'User created';
}

export default function SignupForm(){

  const [message,formAction]=
    useActionState(createUser,'');

  return (
    <form action={formAction}>
      <input name="name"/>
      <button>Save</button>
      <p>{message}</p>
    </form>
  );

}
Notice how the action returns state directly.

Interview Answer

React 19 Form Actions allow developers to pass async functions directly to a form's action prop. When the form is submitted, React automatically collects the FormData and invokes the action function. This removes much of the traditional form boilerplate such as onSubmit handlers, preventDefault, and manual FormData processing. Form Actions work especially well with hooks like useActionState and Server Actions.
