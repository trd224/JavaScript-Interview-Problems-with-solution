276. What is an Uncontrolled Component in Forms?

One-Line Definition

An uncontrolled component in forms is a form element whose value is managed by the browser DOM instead of React state.

What Does "Uncontrolled" Mean?

In an uncontrolled form component, React does not store the field value in state.

The browser keeps the value internally, and we access it only when needed.

  const nameRef = useRef(null);

How It Works

const nameRef = useRef(null);

React stores a reference to the input element. 

<input ref={nameRef} />

The browser stores whatever the user types. 

nameRef.current.value

Reads the current value from the DOM.

Complete Example

import { useRef } from 'react';

function Form() {

  const nameRef = useRef(null);

  function handleSubmit() {

    alert(nameRef.current.value);

  }

  return (

    <>

      <input ref={nameRef} />

      <button onClick={handleSubmit}>

        Submit

      </button>

    </>

  );

}

Using defaultValue

<input

/>
defaultValue sets the initial value of an uncontrolled input.

Why Doesn't It Re-render?

// No useState

// No setState

// No React update
Since React state is not updated, typing inside an uncontrolled input does not cause component re-renders.

Common Use Cases

  • Simple forms.
  • Reading values only on submit.
  • File upload fields.
  • Third-party DOM integrations.
  • Performance-sensitive forms.

File Input Example

const fileRef =
useRef(null);

<input
  type="file"
  ref={fileRef}
/>
File inputs are typically implemented as uncontrolled components because their value cannot be controlled through React state.

Real Project Example

In our Aadhaar Authentication application, most fields such as Aadhaar Number and OTP are controlled components. However, file upload controls and some third-party integrations may use uncontrolled components with useRef.

Important Interview Points

  • Uncontrolled components use the DOM as the source of truth.
  • They commonly use useRef.
  • Typing does not trigger React re-renders.
  • Values are accessed when needed.
  • File inputs are usually uncontrolled.

Short Interview Answer

An uncontrolled component in forms is a form field whose value is managed by the DOM and accessed through refs rather than React state.

One-Line Interview Answer

An uncontrolled component in forms stores its value in the DOM instead of React state.
