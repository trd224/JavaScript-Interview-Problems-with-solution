280. How Do You Handle a Text Input in React?

One-Line Definition

A text input in React is typically handled using useState and the onChange event to keep the input value synchronized with React state.

Why Do We Need State?

React uses controlled components for text inputs.

Instead of the browser managing the value, React stores the value in state.

Basic Example

import { useState } from 'react';

function UserForm() {

  const [name, setName] = useState('');

  return (

    <input value={name} onChange={e => setName(e.target.value)} placeholder="Enter Name" />

  );

}

Complete Example

import { useState } from 'react';

function UserForm() {

  const [name, setName] = useState('');

  function handleChange(e) {

    setName(e.target.value);

  }

  return (

    <div>

      <input value={name} onChange={handleChange} placeholder="Enter Name" />

      <p>Name: {name}</p>

    </div>

  );

}

export default UserForm;

Why Use Controlled Text Inputs?

  • Easy validation.
  • Easy error handling.
  • Easy form submission.
  • React always knows the current value.
  • Supports dynamic UI updates.

Validation Example

if (name.length < 3) {

  console.log('Name must contain at least 3 characters');

}
Validation is easy because the latest value is already available in React state.

Real Project Example

In our Aadhaar Authentication application, Aadhaar Number, OTP, Department Search, and Purpose fields are handled as controlled text inputs so React can validate and process user input immediately.

Important Interview Points

  • Text inputs are usually controlled components.
  • useState stores the value.
  • onChange updates the value.
  • value binds state to the input.
  • Typing triggers component re-renders.

Short Interview Answer

A text input in React is commonly handled using useState and onChange, where state stores the value and updates whenever the user types.
