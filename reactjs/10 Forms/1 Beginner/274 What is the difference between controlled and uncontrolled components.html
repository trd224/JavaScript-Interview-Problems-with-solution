274. What is the Difference Between Controlled and Uncontrolled Components?

One-Line Definition

A controlled component stores form values in React state, while an uncontrolled component stores form values in the DOM.

What Is the Main Difference?

Controlled Component Uncontrolled Component
React controls the value DOM controls the value
Uses useState Uses useRef
Value stored in React state Value stored in DOM element
Re-renders on every change Does not re-render while typing
Easy validation Validation requires reading DOM value
Recommended for most forms Useful for simple forms and file inputs

Controlled Component Flow

User Types
↓
onChange
↓
setState()
↓
React State Updated
↓
Component Re-renders

Controlled Component Example

import { useState } from 'react';

function Form() {

  const [name, setName] = useState('');

  return (

    <input
      value={name}
      onChange={e => setName(e.target.value)}
    />

  );

}

How Controlled Component Works

// User types "R"

setName('R');

// State becomes

name = 'R';

// Component re-renders

// Input shows "R"

Uncontrolled Component Flow

User Types
↓
DOM Stores Value
↓
React Does Nothing
↓
Read Value Using Ref

Uncontrolled Component Example

import { useRef } from 'react';

function Form() {

  const nameRef = useRef(null);

  return (

    <input ref={nameRef} />

  );

}

How Uncontrolled Component Works

// User types "Rizwan"

// DOM stores value internally

// No React state update

// No component re-render

nameRef.current.value
// "Rizwan"

When Should We Use Controlled Components?

  • Form validation.
  • Real-time error messages.
  • Live search.
  • Character counters.
  • Complex forms.

Example - Live Validation

const [email, setEmail] = useState('');

const isValid = email.includes('@');
This is easy because React already has the current value in state.

When Should We Use Uncontrolled Components?

  • Simple forms.
  • Reading values only on submit.
  • File uploads.
  • Third-party DOM libraries.
  • Performance-sensitive forms.

File Input Example

const fileRef = useRef(null);

<input
  type="file"
  ref={fileRef}
/>
File inputs are usually implemented as uncontrolled components because browsers do not allow React to directly control file values.

Which One Is More Common?

Controlled components are more common in React applications because they make validation, conditional UI updates, and form handling much easier.

Real Project Example

In our Aadhaar Authentication application, Aadhaar Number, OTP, Department, Purpose, and Consent fields are controlled components because React needs to validate and process their values. File uploads and some third-party integrations may use uncontrolled components with useRef.

Important Interview Points

  • Controlled components use useState.
  • Uncontrolled components use useRef.
  • Controlled components re-render when values change.
  • Uncontrolled components keep values inside the DOM.
  • Controlled components are preferred for most React forms.
  • File inputs are usually uncontrolled.

Short Interview Answer

Controlled components store form values in React state and update through onChange, while uncontrolled components store values in the DOM and access them using refs.

One-Line Interview Answer

Controlled components use React state to manage form values, whereas uncontrolled components use the DOM and refs.
