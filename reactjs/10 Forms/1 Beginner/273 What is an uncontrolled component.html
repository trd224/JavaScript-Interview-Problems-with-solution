273. What is an Uncontrolled Component?

One-Line Definition

An uncontrolled component is a form element whose value is managed by the DOM instead of React state.

What Does "Uncontrolled" Mean?

In an uncontrolled component, React does not store the input value in state.

The browser DOM keeps track of the value, and we read it only when needed.

Basic Uncontrolled Input Example

import { useRef } from 'react';

function UserForm() {

  const nameRef = useRef(null);

  return (

    <input ref={nameRef} />

  );

}

How It Works

Unlike controlled components, there is no useState. 

const nameRef = useRef(null);

React stores a reference to the DOM element. 

<input ref={nameRef} />

The browser manages the input value internally.

Reading the Value

function handleSubmit() {

  console.log(nameRef.current.value);

}
The value is read directly from the DOM when needed.

Complete Example

import { useRef } from 'react';

function UserForm() {

  const nameRef = useRef(null);

  function handleSubmit() {

    alert(nameRef.current.value);

  }

  return (

    <>

      <input ref={nameRef} />

      <button onClick={handleSubmit}>Submit</button>

    </>

  );

}

Output Flow

User Types "Rizwan"
↓
DOM Stores Value
↓
User Clicks Submit
↓
nameRef.current.value
↓

Using defaultValue

<input
  defaultValue="Rizwan"
  ref={nameRef}
/>
defaultValue sets the initial value for an uncontrolled component.

Why Doesn't It Re-render?

Because no React state changes. 

// No useState

// No setState

// No re-render
Typing inside an uncontrolled input does not trigger React re-renders.

Controlled vs Uncontrolled

Controlled Component Uncontrolled Component
Value stored in React state Value stored in DOM
Uses useState Uses useRef
Re-renders on every change No re-render while typing
Easy validation Validation requires reading DOM value
React controls input Browser controls input

Controlled Component Example

const [name, setName] = useState('');

<input value={name} onChange={e => setName(e.target.value)} />

Uncontrolled Component Example

const nameRef = useRef(null);

<input ref={nameRef} />

When Should We Use Uncontrolled Components?

  • Simple forms.
  • Reading values only during submit.
  • Interacting with third-party libraries.
  • File inputs.
  • Performance-sensitive forms.

File Input Example

const fileRef = useRef(null);

<input type="file" ref={fileRef} />
File inputs are usually implemented as uncontrolled components.

Real Project Example

In our Aadhaar Authentication application, most form fields such as Aadhaar Number and OTP are controlled components. However, file uploads or certain third-party integrations may use uncontrolled components through useRef.

Important Interview Points

  • Uncontrolled components store values in the DOM.
  • They commonly use useRef.
  • React does not control the input value.
  • Typing does not trigger re-renders.
  • Values are read when required.
  • File inputs are usually uncontrolled.

Short Interview Answer

An uncontrolled component is a form element whose value is managed by the DOM and accessed using refs instead of React state.

One-Line Interview Answer

An uncontrolled component is a form field whose value is controlled by the DOM rather than React state.
