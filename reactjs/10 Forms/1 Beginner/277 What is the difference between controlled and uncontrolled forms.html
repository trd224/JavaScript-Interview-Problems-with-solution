277. What is the Difference Between Controlled and Uncontrolled Forms?

One-Line Definition

A controlled form stores all form values in React state, while an uncontrolled form stores form values in the DOM and accesses them using refs.

What Is the Main Difference?

Controlled Form Uncontrolled Form
Form data stored in React state Form data stored in DOM elements
Uses useState Uses useRef
React controls form values Browser controls form values
Re-renders on every input change No re-render while typing
Easy validation Need to read DOM values first
Most common in React Less common

Controlled Form Example

import { useState } from 'react';

function LoginForm() {

  const [email, setEmail] = useState('');

  const [password, setPassword] = useState('');

  function handleSubmit(e) {

    e.preventDefault();

    console.log(email, password);

  }

  return (

    <form onSubmit={handleSubmit}>

      <input value={email} onChange={e => setEmail(e.target.value)} />

      <input value={password} onChange={e => setPassword(e.target.value)} />

      <button type="submit">Login</button>

    </form>

  );

}
React already has all form values in state, so we can use them directly.

Uncontrolled Form Flow

User Types
↓
Browser Stores Values
↓
No State Updates
↓
Submit Button Clicked
↓
Read Values Using Refs

Uncontrolled Form Example

import { useRef } from 'react';

function LoginForm() {

  const emailRef = useRef(null);

  const passwordRef = useRef(null);

  function handleSubmit(e) {

    e.preventDefault();

    console.log(emailRef.current.value, passwordRef.current.value);

  }

  return (

    <form onSubmit={handleSubmit}>

      <input ref={emailRef} />

      <input ref={passwordRef} />

      <button type="submit">Login</button>

    </form>

  );

}
Values are read directly from DOM elements when the form is submitted.

Validation is easy because React already knows the current value.

Uncontrolled Form

Performance Difference

Controlled Form Uncontrolled Form
Re-renders on every keystroke No re-render while typing
Slightly more React work Less React work
Better control over UI Better raw input performance

When Should We Use Controlled Forms?

  • Form validation.
  • Live search.
  • Dynamic form behavior.
  • Error messages.
  • Most React applications.

When Should We Use Uncontrolled Forms?

  • Simple forms.
  • File uploads.
  • Third-party DOM libraries.
  • Reading values only during submit.

Real Project Example

In our Aadhaar Authentication application, forms such as Aadhaar Number, OTP, Department, Purpose, and Consent are controlled forms because validation and UI updates are required. File upload fields and some third-party integrations may use uncontrolled forms with refs.

Important Interview Points

  • Controlled forms use useState.
  • Uncontrolled forms use useRef.
  • Controlled forms store data in React state.
  • Uncontrolled forms store data in the DOM.
  • Controlled forms are preferred in React applications.
  • File inputs are usually uncontrolled.

Short Interview Answer

Controlled forms manage form values through React state, while uncontrolled forms let the browser manage form values and access them through refs.

One-Line Interview Answer

Controlled forms use React state for form values, whereas uncontrolled forms use the DOM and refs.
