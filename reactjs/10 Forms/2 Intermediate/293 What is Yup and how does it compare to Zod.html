293. What is Yup and How Does it Compare to Zod?

One-Line Definition

Yup is a JavaScript schema validation library used to validate form data. Zod is also a schema validation library, but it is more TypeScript-friendly and modern.

Why Do We Need Yup or Zod?

In forms, we need to validate user input.

  • Email should be valid.
  • Password should have minimum length.
  • Name should be required.
  • Confirm password should match password.
Yup and Zod both help us keep validation rules in one schema instead of writing validation directly inside every input.

Validation Flow

User Submits Form
↓
React Hook Form
↓
Resolver
↓
Yup / Zod Schema
↓
Errors or Valid Data

Yup Schema Example

File: src/schemas/yupLoginSchema.js
import * as yup from 'yup';

export const yupLoginSchema = yup.object({
  email: yup.string().required('Email is required').email('Invalid email'),
  password: yup.string().required('Password is required').min(6, 'Minimum 6 characters')
});

Zod Schema Example

File: src/schemas/zodLoginSchema.js
import { z } from 'zod';

export const zodLoginSchema = z.object({
  email: z.string().min(1, 'Email is required').email('Invalid email'),
  password: z.string().min(6, 'Minimum 6 characters')
});

Using Yup With React Hook Form

File: src/components/LoginForm.jsx
import { useForm } from 'react-hook-form';
import { yupResolver } from '@hookform/resolvers/yup';
import { yupLoginSchema } from '../schemas/yupLoginSchema';

function LoginForm() {
  const { register, handleSubmit, formState: { errors } } = useForm({ resolver: yupResolver(yupLoginSchema) });

  function onSubmit(data) {
    console.log(data);
  }

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input placeholder="Email" {...register('email')} />
      {errors.email && <p className="error">{errors.email.message}</p>}

      <input type="password" placeholder="Password" {...register('password')} />
      {errors.password && <p className="error">{errors.password.message}</p>}

      <button type="submit">Login</button>
    </form>
  );
}

export default LoginForm;

Yup vs Zod Comparison

Feature Yup Zod
Main Purpose Schema validation Schema validation
TypeScript Support Good, but not TypeScript-first Excellent, TypeScript-first
Syntax Older style Cleaner modern style
Type Inference Limited Strong type inference
React Hook Form Works with yupResolver Works with zodResolver
Popularity Older and widely used Modern and increasingly preferred

Password Confirm Example With Yup

File: src/schemas/yupRegisterSchema.js
import * as yup from 'yup';

export const yupRegisterSchema = yup.object({
  password: yup.string().required('Password is required').min(6, 'Minimum 6 characters'),
  confirmPassword: yup.string().required('Confirm password is required').oneOf([yup.ref('password')], 'Passwords do not match')
});

Password Confirm Example With Zod

File: src/schemas/zodRegisterSchema.js
import { z } from 'zod';

export const zodRegisterSchema = z.object({
  password: z.string().min(6, 'Minimum 6 characters'),
  confirmPassword: z.string().min(1, 'Confirm password is required')
}).refine(data => data.password === data.confirmPassword, {
  message: 'Passwords do not match',
  path: ['confirmPassword']
});

Which One Should You Use?

Use Yup When Use Zod When
Project already uses Yup. You are starting a new project.
You are maintaining older code. You want better TypeScript support.
Your team already knows Yup. You want modern schema validation.
For new React + TypeScript projects, Zod is usually preferred. For older projects, Yup is still fine.

Interview Answer

Yup and Zod are schema validation libraries used to validate form data. Yup is older and widely used, while Zod is more modern and TypeScript-first. Both can integrate with React Hook Form using resolvers, but Zod is generally preferred in modern TypeScript projects because it provides better type inference and cleaner schema validation.

One-Line Interview Answer

Yup is an older schema validation library, while Zod is a modern TypeScript-first validation library commonly preferred in new React projects.
