294. What is Formik and How Does it Compare to React Hook Form?

One-Line Definition

Formik is a React form library used to manage form state, validation, and submission. React Hook Form is a newer form library that provides better performance with fewer re-renders.

Why Do We Need Form Libraries?

Managing forms manually becomes repetitive.

  • Form state management
  • Validation
  • Error handling
  • Form submission
  • Reset functionality
Both Formik and React Hook Form solve these problems, but they work differently internally.

How Formik Works

Input Change
↓
Formik State Update
↓
Component Re-render
↓
UI Update

How React Hook Form Works

Input Change
↓
DOM Stores Value
↓
Minimal Re-renders
↓
UI Update

Formik Example

File: src/components/FormikLoginForm.jsx
import { Formik, Form, Field } from 'formik';

function FormikLoginForm() {
  return (
    <Formik
      initialValues={{ email: '', password: '' }}
      onSubmit={values => console.log(values)}
    >
      <Form>
        <Field name="email" placeholder="Email" />
        <Field name="password" type="password" placeholder="Password" />
        <button type="submit">Login</button>
      </Form>
    </Formik>
  );
}

export default FormikLoginForm;

React Hook Form Example

File: src/components/RHFLoginForm.jsx
import { useForm } from 'react-hook-form';

function RHFLoginForm() {
  const { register, handleSubmit } = useForm();

  function onSubmit(data) {
    console.log(data);
  }

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input placeholder="Email" {...register('email')} />
      <input type="password" placeholder="Password" {...register('password')} />
      <button type="submit">Login</button>
    </form>
  );
}

export default RHFLoginForm;

Formik Validation Example

File: src/components/FormikLoginForm.jsx
import { Formik, Form, Field, ErrorMessage } from 'formik';

function FormikLoginForm() {
  return (
    <Formik
      initialValues={{ email: '' }}
      validate={values => {
        const errors = {};

        if (!values.email) {
          errors.email = 'Email is required';
        }

        return errors;
      }}
      onSubmit={values => console.log(values)}
    >
      <Form>
        <Field name="email" placeholder="Email" />
        <ErrorMessage name="email" component="p" />
        <button type="submit">Submit</button>
      </Form>
    </Formik>
  );
}

export default FormikLoginForm;

React Hook Form + Zod Validation

File: src/components/RHFLoginForm.jsx
import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';
import { loginSchema } from '../schemas/loginSchema';

function RHFLoginForm() {
  const { register, handleSubmit, formState: { errors } } = useForm({
    resolver: zodResolver(loginSchema)
  });

  function onSubmit(data) {
    console.log(data);
  }

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input placeholder="Email" {...register('email')} />
      {errors.email && <p className="error">{errors.email.message}</p>}

      <button type="submit">Submit</button>
    </form>
  );
}

export default RHFLoginForm;

When Should You Use Formik?

  • Existing project already uses Formik.
  • Team is familiar with Formik.
  • Maintaining legacy applications.

When Should You Use React Hook Form?

  • New projects.
  • Large forms.
  • Better performance required.
  • Zod integration needed.
  • TypeScript-heavy applications.
Today, React Hook Form + Zod is generally the preferred combination for new React applications.

Interview Answer

Formik is a React form library used for handling form state, validation, and submission. Compared to React Hook Form, Formik relies more on controlled inputs and causes more re-renders. React Hook Form is generally faster, has a smaller bundle size, and integrates very well with validation libraries such as Zod.

One-Line Interview Answer

Formik is an older form library, while React Hook Form is a newer, faster, and more performant alternative that is preferred in most modern React projects.
