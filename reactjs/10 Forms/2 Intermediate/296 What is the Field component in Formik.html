296. What is the Field Component in Formik?

One-Line Definition

Field is a Formik component that automatically connects an input element to Formik form state, handling value, onChange, and onBlur for you.

Why Do We Need Field?

Without Field, we must manually connect inputs to Formik.

File: src/components/LoginForm.jsx
import { Formik } from 'formik';

function LoginForm() {
  return (
    <Formik initialValues={{ email: '' }} onSubmit={values => console.log(values)}>
      {({ values, handleChange, handleBlur }) => (
        <form>
          <input
            name="email"
            value={values.email}
            onChange={handleChange}
            onBlur={handleBlur}
          />
        </form>
      )}
    </Formik>
  );
}

export default LoginForm;
Too much boilerplate code.

Using Field Component

File: src/components/LoginForm.jsx
import { Formik, Form, Field } from 'formik';

function LoginForm() {
  return (
    <Formik initialValues={{ email: '' }} onSubmit={values => console.log(values)}>
      <Form>
        <Field name="email" placeholder="Email" />
      </Form>
    </Formik>
  );
}

export default LoginForm;
Field automatically handles value, onChange, and onBlur.

How Field Works Internally

<Field name="email" />
↓
Formik Finds values.email
↓
Adds value
↓
Adds onChange
↓
Adds onBlur
↓
Connected To Form State

Field vs Normal Input

Normal Input Formik Field
Manual value binding Automatic binding
Manual onChange Automatic onChange
Manual onBlur Automatic onBlur
More code Less code

Basic Example

File: src/components/LoginForm.jsx
import { Formik, Form, Field } from 'formik';

function LoginForm() {
  return (
    <Formik
      initialValues={{ email: '', password: '' }}
      onSubmit={values => console.log(values)}
    >
      <Form>
        <Field name="email" placeholder="Email" />
        <Field name="password" type="password" placeholder="Password" />
        <button type="submit">Login</button>
      </Form>
    </Formik>
  );
}

export default LoginForm;

Field With ErrorMessage

File: src/components/LoginForm.jsx
import { Formik, Form, Field, ErrorMessage } from 'formik';

function LoginForm() {
  return (
    <Formik
      initialValues={{ email: '' }}
      validate={values => {
        const errors = {};

        if (!values.email) {
          errors.email = 'Email is required';
        }

        return errors;
      }}
      onSubmit={values => console.log(values)}
    >
      <Form>
        <Field name="email" placeholder="Email" />
        <ErrorMessage name="email" component="p" />
        <button type="submit">Submit</button>
      </Form>
    </Formik>
  );
}

export default LoginForm;

Real World Example

File: src/components/RegisterForm.jsx
import { Formik, Form, Field, ErrorMessage } from 'formik';

function RegisterForm() {
  return (
    <Formik initialValues={{ name: '', email: '', mobile: '' }} onSubmit={values => console.log(values)}>
      <Form>
        <Field name="name" placeholder="Name" />
        <ErrorMessage name="name" component="p" />

        <Field name="email" placeholder="Email" />
        <ErrorMessage name="email" component="p" />

        <Field name="mobile" placeholder="Mobile" />
        <ErrorMessage name="mobile" component="p" />

        <button type="submit">Register</button>
      </Form>
    </Formik>
  );
}

export default RegisterForm;

Interview Answer

The Field component is a Formik component that automatically connects an input element to Formik's form state. It handles value binding, change events, and blur events automatically, reducing boilerplate code and simplifying form creation.

One-Line Interview Answer
