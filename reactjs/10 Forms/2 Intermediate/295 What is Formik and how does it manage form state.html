295. What is Formik and How Does it Manage Form State?

One-Line Definition

Formik is a React form library that manages form values, validation, errors, touched fields, and submission state using React state internally.

The Problem Formik Solves

Without Formik, we manually create state and handlers for every field.

File: src/components/LoginForm.jsx
import { useState } from 'react';

function LoginForm() {
  const [email, setEmail] = useState('');
  const [password, setPassword] = useState('');

  return (
    <>
      <input value={email} onChange={e => setEmail(e.target.value)} />
      <input value={password} onChange={e => setPassword(e.target.value)} />
    </>
  );
}

export default LoginForm;

function LoginForm() {
  return (
    <Formik
      initialValues={{ email: '', password: '' }}
      onSubmit={values => console.log(values)}
    >
      <Form>
        <Field name="email" placeholder="Email" />
        <Field name="password" type="password" placeholder="Password" />
        <button type="submit">Login</button>
      </Form>
    </Formik>
  );

How Formik Manages State

Formik internally creates a state object similar to: 

{
  values: {
    email: '',
    password: ''
  },

  errors: {},

  touched: {},

  isSubmitting: false
}

values Object

Stores all field values. 

{
  values: {
    email: 'rizwan@gmail.com',
    password: '123456'
  }
}

↓
Formik Updates values
↓
Component Re-renders

Accessing values

File: src/components/LoginForm.jsx
import { Formik } from 'formik';

function LoginForm() {
  return (
    <Formik
      initialValues={{ email: '' }}
      onSubmit={values => console.log(values)}
    >
      {({ values }) => (
        <div>
          {values.email}
        </div>
      )}
    </Formik>
  );
}

export default LoginForm;

errors Object

Stores validation errors. 

{
  errors: {
    email: 'Email is required'
  }
}

touched Object

Tracks fields the user has visited. 

{
  touched: {
    email: true
  }
}
Useful for showing validation messages only after user interaction.

isSubmitting

Tracks form submission status. 

{
  isSubmitting: true
}

Complete Example

File: src/components/LoginForm.jsx
import { Formik, Form, Field, ErrorMessage } from 'formik';

function LoginForm() {
  return (
    <Formik
      initialValues={{ email: '', password: '' }}
      validate={values => {
        const errors = {};

        if (!values.email) {
          errors.email = 'Email is required';
        }

        return errors;
      }}
      onSubmit={values => console.log(values)}
    >
      {({ isSubmitting }) => (
        <Form>
          <Field name="email" placeholder="Email" />
          <ErrorMessage name="email" component="p" />

          <Field name="password" type="password" placeholder="Password" />

          <button type="submit" disabled={isSubmitting}>
            Login
          </button>
        </Form>
      )}
    </Formik>
  );
}

export default LoginForm;

Why Is Formik Slower Than React Hook Form?

Formik stores field values inside React state. 

values.email = 'rizwan@gmail.com';

Every change updates state.

Every state update triggers a re-render.

Typing
↓
State Update
↓
Re-render
↓
UI Update
This is why Formik can become slower in very large forms.

Formik State vs React Hook Form State

Formik React Hook Form
Stores values in React state Stores values in DOM refs
More re-renders Fewer re-renders
Controlled inputs Uncontrolled inputs
Larger bundle Smaller bundle

Interview Answer

Formik is a React form library that manages form state using React state internally. It keeps track of values, errors, touched fields, and submission status. Whenever a field changes, Formik updates its internal state and re-renders the component to keep the UI synchronized.

One-Line Interview Answer

Formik manages form state by storing values, errors, touched fields, and submission status inside React state.
