291. What is Zod and How is it Used for Form Validation?

One-Line Definition

Zod is a TypeScript-first schema validation library used to validate form data, API responses, and objects. It is commonly used with React Hook Form for powerful form validation.

Why Do We Need Zod?

React Hook Form can validate fields using rules like required, minLength, and pattern. 

<input {...register('email', { required: 'Email is required' })} />

But large forms become difficult to manage when validation rules are scattered across many fields.

Zod keeps all validation rules in one central schema.

Step 1 - Create Zod Schema

File: src/schemas/loginSchema.js
import { z } from 'zod';

export const loginSchema = z.object({
  email: z.string().min(1, 'Email is required').email('Invalid email'),
  password: z.string().min(6, 'Minimum 6 characters')
});

Step 2 - Connect Zod with React Hook Form

File: src/components/LoginForm.jsx
import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';
import { loginSchema } from '../schemas/loginSchema';

function LoginForm() {
  const { register, handleSubmit, formState: { errors } } = useForm({
    resolver: zodResolver(loginSchema)
  });

  function onSubmit(data) {
    console.log(data);
  }

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input placeholder="Email" {...register('email')} />
      {errors.email && <p>{errors.email.message}</p>}

      <input type="password" placeholder="Password" {...register('password')} />
      {errors.password && <p>{errors.password.message}</p>}

      <button type="submit">Login</button>
    </form>
  );
}

export default LoginForm;

What Happens During Validation?

If the user enters: 

Email: abc
Password: 123

Zod returns: 

{
  email: "Invalid email",
  password: "Minimum 6 characters"
}

Common Zod Validators

File: src/schemas/userSchema.js
import { z } from 'zod';

export const userSchema = z.object({
  name: z.string().min(3, 'Minimum 3 characters'),
  email: z.string().email('Invalid email'),
  age: z.number().min(18, 'Must be 18+'),
  password: z.string().min(8, 'Minimum 8 characters')
});

Password Confirmation Example

File: src/schemas/registerSchema.js
import { z } from 'zod';

export const registerSchema = z.object({
  password: z.string().min(6, 'Minimum 6 characters'),
  confirmPassword: z.string()
}).refine(data => data.password === data.confirmPassword, {
  message: 'Passwords do not match',
  path: ['confirmPassword']
});
refine() is commonly used for cross-field validation.

Real World Example

A registration form may validate:

  • Name required
  • Email format
  • Password length
  • Password confirmation
  • Age restrictions
  • Terms acceptance

All validation can live inside one Zod schema instead of being spread across inputs.

Why Developers Prefer Zod?

  • Centralized validation
  • Reusable schemas
  • Works perfectly with React Hook Form
  • TypeScript support
  • Easy cross-field validation
  • Cleaner codebase

Interview Answer

Zod is a schema validation library commonly used with React Hook Form. Developers define validation rules inside a Zod schema and connect it using zodResolver(). Zod then validates form data and returns errors that React Hook Form displays in the UI.

One-Line Interview Answer

Zod is a schema validation library that works with React Hook Form to provide centralized, reusable, and type-safe form validation.
