290. What is the formState Object in React Hook Form?

One-Line Definition

formState is an object provided by React Hook Form that contains information about the current state of the form such as errors, touched fields, dirty fields, submission status, and validation status.

Why Do We Need formState?

React Hook Form tracks much more than just field values.

It also tracks:

  • Validation errors
  • Whether a field was touched
  • Whether a field was modified
  • Whether the form is valid
  • Whether the form is submitting
  • Whether submission succeeded

How It Works

User Interacts With Form
↓
React Hook Form Tracks Changes
↓
Updates formState
↓
UI Reacts To Changes

Basic Syntax

File: src/components/LoginForm.jsx
import { useForm } from 'react-hook-form';

function LoginForm() {
  const { register, handleSubmit, formState } = useForm();

  console.log(formState);

  return (
    <form>
      <input {...register('email')} />
    </form>
  );
}

export default LoginForm;

Output: 

{
  isDirty: false,
  dirtyFields: {},
  touchedFields: {},
  errors: {},
  isSubmitting: false,
  isSubmitted: false,
  isSubmitSuccessful: false,
  submitCount: 0,
  isValid: false
}

Most Common Usage

File: src/components/LoginForm.jsx
const { register, handleSubmit, formState: { errors } } = useForm();

Most developers use formState to access validation errors.

Validation Error Example

File: src/components/LoginForm.jsx
import { useForm } from 'react-hook-form';

function LoginForm() {
  const { register, handleSubmit, formState: { errors } } = useForm();

  function onSubmit(data) {
    console.log(data);
  }

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input placeholder="Email" {...register('email', { required: 'Email is required' })} />
      {errors.email && <p>{errors.email.message}</p>}
      <button type="submit">Submit</button>
    </form>
  );
}

export default LoginForm;

What Does formState Contain?

formState = {
  errors,
  touchedFields,
  dirtyFields,
  isDirty,
  isValid,
  isSubmitting,
  isSubmitSuccessful,
  submitCount
}

errors

Contains validation errors. 

const { formState: { errors } } = useForm();

Example: 

{
  email: {
    message: 'Email is required'
  }
}

touchedFields

Tracks fields the user has focused and left. 

const { formState: { touchedFields } } = useForm();

console.log(touchedFields);

Output: 

{
  email: true
}
Useful for showing validation only after user interaction.

dirtyFields

Tracks fields whose values have changed. 

const { formState: { dirtyFields } } = useForm();

Output: 

{
  email: true
}

isDirty

Checks if any field has changed. 

const { formState: { isDirty } } = useForm();

Output: 

true

Real World Example

<button disabled={!isDirty}>Save</button>

Save button becomes enabled only after changes are made.

isValid

Indicates whether the form currently passes validation. 

const { formState: { isValid } } = useForm({
  mode: 'onChange'
});

Example

<button disabled={!isValid}>Submit</button>

isSubmitting

Becomes true while the form is being submitted.

File: src/components/LoginForm.jsx
import { useForm } from 'react-hook-form';

function LoginForm() {
  const { register, handleSubmit, formState: { isSubmitting } } = useForm();

  async function onSubmit(data) {
    await new Promise(resolve => setTimeout(resolve, 2000));
    console.log(data);
  }

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input {...register('email')} />
      <button disabled={isSubmitting}>
        {isSubmitting ? 'Submitting...' : 'Submit'}
      </button>
    </form>
  );
}

export default LoginForm;

isSubmitSuccessful

Becomes true after successful form submission. 

const { formState: { isSubmitSuccessful } } = useForm();

Output: 

true

submitCount

Tracks how many times the form was submitted. 

const { formState: { submitCount } } = useForm();

console.log(submitCount);

Output: 

3

Real World Example

File: src/components/ProfileForm.jsx
import { useForm } from 'react-hook-form';

function ProfileForm() {
  const {
    register,
    handleSubmit,
    formState: { errors, isDirty, isSubmitting }
  } = useForm();

  function onSubmit(data) {
    console.log(data);
  }

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input placeholder="Name" {...register('name', { required: 'Name is required' })} />
      {errors.name && <p>{errors.name.message}</p>}

      <button disabled={!isDirty || isSubmitting}>
        Save Changes
      </button>
    </form>
  );
}

export default ProfileForm;

Most Important formState Properties

Property Purpose
errors Validation errors
touchedFields Fields user interacted with
dirtyFields Fields that changed
isDirty Any field changed?
isValid Form valid?
isSubmitting Currently submitting?
isSubmitSuccessful Submission successful?
submitCount Submission count

Interview Answer

The formState object contains information about the current state of a React Hook Form. It provides properties such as errors, touchedFields, dirtyFields, isDirty, isValid, and isSubmitting, which help developers manage validation, UI behavior, and submission status.

One-Line Interview Answer

formState is an object that provides validation, interaction, and submission status information for a React Hook Form.
