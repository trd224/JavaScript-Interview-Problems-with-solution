289. What is the setValue and getValues API in React Hook Form?

One-Line Definition

setValue() is used to programmatically update a field value, while getValues() is used to read current form values without causing a re-render.

Why Do We Need Them?

Sometimes values must be updated or read outside normal user typing.

Examples:

  • Load user data from API
  • Auto-fill forms
  • Copy address fields
  • Read form values inside custom logic
  • Generate previews

How They Work

API Response
↓
setValue()
↓
Update Form Field

Need Current Value
↓
getValues()
↓
Read Form Data

What is setValue()?

setValue() updates a field programmatically.

File: src/components/ProfileForm.jsx
import { useForm } from 'react-hook-form';

function ProfileForm() {
  const { register, setValue } = useForm();

  function loadData() {
    setValue('name', 'Rizwan');
  }

  return (
    <>
      <input placeholder="Name" {...register('name')} />
      <button onClick={loadData}>Load Data</button>
    </>
  );
}

export default ProfileForm;

When the button is clicked: 

Name Input
↓
Rizwan

setValue() Syntax

setValue(fieldName, value);

Examples

setValue('name', 'Rizwan');
setValue('email', 'rizwan@gmail.com');
setValue('age', 39);

Set Multiple Fields

File: src/components/ProfileForm.jsx
function loadData() {
  setValue('name', 'Rizwan');
  setValue('email', 'rizwan@gmail.com');
  setValue('city', 'Kota');
}

Real World API Example

File: src/components/ProfileForm.jsx
import { useEffect } from 'react';
import { useForm } from 'react-hook-form';

function ProfileForm() {
  const { register, setValue } = useForm();

  useEffect(() => {
    const user = {
      name: 'Rizwan',
      email: 'rizwan@gmail.com'
    };

    setValue('name', user.name);
    setValue('email', user.email);
  }, []);

  return (
    <>
      <input {...register('name')} />
      <input {...register('email')} />
    </>
  );
}

export default ProfileForm;

What is getValues()?

getValues() returns current form values immediately.

File: src/components/ProfileForm.jsx
import { useForm } from 'react-hook-form';

function ProfileForm() {
  const { register, getValues } = useForm();

  function showData() {
    console.log(getValues());
  }

  return (
    <>
      <input placeholder="Name" {...register('name')} />
      <button onClick={showData}>Show Data</button>
    </>
  );
}

export default ProfileForm;

Output: 

{
  name: 'Rizwan'
}

Get Single Field Value

const email = getValues('email');

console.log(email);

Output: 

rizwan@gmail.com

Get Entire Form

const data = getValues();

console.log(data);

Output: 

{
  name: 'Rizwan',
  email: 'rizwan@gmail.com'
}

getValues() vs watch()

getValues() watch()
Snapshot of values Live values
No re-render Can trigger re-render
Used in logic Used in UI
Reads once Continuously watches

Real World Example - Copy Address

File: src/components/CheckoutForm.jsx
import { useForm } from 'react-hook-form';

function CheckoutForm() {
  const { register, getValues, setValue } = useForm();

  function copyAddress() {
    const address = getValues('shippingAddress');
    setValue('billingAddress', address);
  }

  return (
    <>
      <input placeholder="Shipping Address" {...register('shippingAddress')} />
      <input placeholder="Billing Address" {...register('billingAddress')} />
      <button onClick={copyAddress}>Same As Shipping</button>
    </>
  );
}

export default CheckoutForm;

Interview Answer

setValue() is used to programmatically update a field value in React Hook Form, while getValues() is used to retrieve the current form values without triggering a re-render. They are commonly used for auto-filling forms, reading values for custom logic, and handling API data.

One-Line Interview Answer

setValue() writes values into form fields, while getValues() reads current form values.
