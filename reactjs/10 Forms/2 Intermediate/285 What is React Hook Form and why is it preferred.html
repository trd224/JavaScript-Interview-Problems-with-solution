285. What is React Hook Form and Why is it Preferred?

One-Line Definition

React Hook Form (RHF) is a lightweight form library that simplifies form handling, validation, and submission while reducing unnecessary re-renders.

Why Was React Hook Form Created?

Managing forms with useState becomes repetitive for large forms.

Without React Hook Form

File: src/components/LoginForm.jsx
import { useState } from 'react';

function LoginForm() {
  const [email, setEmail] = useState('');
  const [password, setPassword] = useState('');

  function handleSubmit(e) {
    e.preventDefault();
    console.log({ email, password });
  }

  return (
    <form onSubmit={handleSubmit}>
      <input value={email} onChange={e => setEmail(e.target.value)} />
      <input value={password} onChange={e => setPassword(e.target.value)} />
      <button>Login</button>
    </form>
  );
}

export default LoginForm;

Problems:

  • Many state variables
  • Many onChange handlers
  • Manual validation
  • More re-renders
  • More boilerplate code
Large forms become difficult to maintain using only useState.

With React Hook Form

File: src/components/LoginForm.jsx
import { useForm } from 'react-hook-form';

function LoginForm() {
  const { register, handleSubmit } = useForm();

  function onSubmit(data) {
    console.log(data);
  }

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input {...register('email')} />
      <input {...register('password')} />
      <button>Login</button>
    </form>
  );
}

export default LoginForm;

import { useForm } from 'react-hook-form';

function LoginForm() {
  const { register, handleSubmit } = useForm();

  function onSubmit(data) {
    console.log(data);
  }

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input placeholder="Email" {...register('email')} />
      <input type="password" placeholder="Password" {...register('password')} />
      <button type="submit">Login</button>
    </form>
  );
}

export default LoginForm;

What Does register() Do?

<input {...register('email')} />

register():

  • Registers the field
  • Tracks field value
  • Handles validation
  • Collects form data

Validation Example

File: src/components/LoginForm.jsx
import { useForm } from 'react-hook-form';

function LoginForm() {
  const { register, handleSubmit, formState: { errors } } = useForm();

  function onSubmit(data) {
    console.log(data);
  }

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input placeholder="Email" {...register('email', { required: 'Email is required' })} />
      {errors.email && <p>{errors.email.message}</p>}
      <button type="submit">Submit</button>
    </form>
  );
}

export default LoginForm;

Why Is React Hook Form Faster?

Traditional forms store values in React state: 

const [email, setEmail] = useState('');

Every key press causes a component re-render.

React Hook Form primarily uses uncontrolled inputs.

Typing
↓
DOM Stores Value
↓
No State Update
↓
Fewer Re-renders
↓
Better Performance
This performance benefit is the main reason React Hook Form is preferred.

React Hook Form vs useState

useState Forms React Hook Form
Many states Single useForm()
Many handlers register()
Manual validation Built-in validation
More re-renders Fewer re-renders
More code Less code

Interview Answer

React Hook Form is a popular React library used to manage forms, validation, and submission. It is preferred because it reduces boilerplate code, provides built-in validation, uses uncontrolled inputs internally, and minimizes unnecessary re-renders, resulting in better performance.

One-Line Interview Answer

React Hook Form is a lightweight form library that provides simpler form handling and better performance than managing forms entirely with useState.
