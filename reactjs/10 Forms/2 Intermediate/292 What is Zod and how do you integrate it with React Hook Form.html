292. What is Zod and How Do You Integrate It With React Hook Form?

One-Line Definition

Zod is a schema validation library used to define validation rules in one place and integrate them with React Hook Form using zodResolver().

Why Use Zod With React Hook Form?

  • Validation rules stay in one schema file.
  • Code becomes cleaner.
  • Validation is reusable.
  • Works well with TypeScript.
  • Supports advanced validation like password match.
React Hook Form handles the form, while Zod handles the validation rules.

Validation Flow

User Submits Form
↓
React Hook Form
↓
zodResolver()
↓
Zod Schema
↓
Valid Data Or Errors

Step 1 - Create Zod Schema

File: src/schemas/loginSchema.js
import { z } from 'zod';

export const loginSchema = z.object({
  email: z.string().min(1, 'Email is required').email('Invalid email address'),
  password: z.string().min(6, 'Password must be at least 6 characters')
});

Step 2 - Use zodResolver With useForm()

File: src/components/LoginForm.jsx
import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';
import { loginSchema } from '../schemas/loginSchema';

function LoginForm() {
  const { register, handleSubmit, formState: { errors } } = useForm({ resolver: zodResolver(loginSchema) });

  function onSubmit(data) {
    console.log(data);
  }

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input placeholder="Email" {...register('email')} />
      {errors.email && <p className="error">{errors.email.message}</p>}

      <input type="password" placeholder="Password" {...register('password')} />
      {errors.password && <p className="error">{errors.password.message}</p>}

      <button type="submit">Login</button>
    </form>
  );
}

export default LoginForm;

What is zodResolver?

zodResolver() connects a Zod schema with React Hook Form. 
useForm({ resolver: zodResolver(loginSchema) });

This means React Hook Form will validate form data using the Zod schema before calling onSubmit().

What Happens If Validation Fails?

Invalid Email
↓
Zod Checks Schema
↓
Creates Error Message
↓
React Hook Form Stores It In errors
↓

Password Confirm Example

File: src/schemas/registerSchema.js
import { z } from 'zod';

export const registerSchema = z.object({
  password: z.string().min(6, 'Password must be at least 6 characters'),
  confirmPassword: z.string().min(1, 'Confirm password is required')
}).refine(data => data.password === data.confirmPassword, {
  message: 'Passwords do not match',
  path: ['confirmPassword']
});
refine() is used when validation depends on multiple fields.

React Hook Form Validation vs Zod Validation

React Hook Form Rules Zod Schema
Validation is written near inputs. Validation is written in one schema.
Good for small forms. Better for large forms.
Less reusable. Reusable across forms and APIs.
Basic validation. Advanced schema validation.

Interview Answer

Zod is a schema validation library used to define validation rules in one place. It is integrated with React Hook Form using zodResolver from @hookform/resolvers/zod. React Hook Form manages the form, while Zod validates the form data and returns error messages through the errors object.

One-Line Interview Answer

Zod is integrated with React Hook Form by passing zodResolver(schema) to useForm().
