080.1 CAF vs HOC Real World Example

One-Line Difference

CAF means: pass a component to another component for rendering flexibility.
HOC means: pass a component to a function that returns a new enhanced component.

Simple Memory

CAF = Which component should I render?

HOC = How can I add extra behavior to this component?

Side By Side Real World Example

CAF Example

Use CAF when the main logic is same, but UI design can change.

Product Row
function ProductRow({ item }) {
  return <p>{item.name}</p>;
}
Admin Row
function AdminRow({ item }) {
  return <div>{item.name} <button>Delete</button></div>;
}
Reusable Table
function Table({ data, RowComponent }) {
  return <>{data.map(item => <RowComponent key={item.id} item={item} />)}</>;
}
Usage
<Table data={products} RowComponent={ProductRow} />

<Table data={products} RowComponent={AdminRow} />
ProductRow / AdminRow
↓ Passed To Table
↓ Table Renders It

HOC Example

Use HOC when many components need same extra behavior.

Normal Page
function Orders() {
  return <h1>Orders</h1>;
}
HOC Function
function withAuth(Component) {
  return function ProtectedPage(props) {
    if (!isLoggedIn) return <Login />;
    return <Component {...props} />;
  };
}
Create Protected Page
const ProtectedOrders = withAuth(Orders);
Usage
<ProtectedOrders />
Orders
↓ withAuth(Orders)
↓ ProtectedOrders
↓ Login Check Added

Your Confusion: ProtectedPage Comes From Where?

ProtectedPage does not come from React. It is just a function name created by the developer.

This code: 

function withAuth(Component) {
  return function ProtectedPage(props) {
    if (!isLoggedIn) return <Login />;
    return <Component {...props} />;
  };
}

is same as: 

function withAuth(Component) {
  return function ABC(props) {
    if (!isLoggedIn) return <Login />;
    return <Component {...props} />;
  };
}

or even: 

function withAuth(Component) {
  return props => {
    if (!isLoggedIn) return <Login />;
    return <Component {...props} />;
  };
}
The name ProtectedPage is only for readability. It means: this returned component protects the original page.

What Actually Happens In HOC?

Suppose we write: 

const ProtectedOrders = withAuth(Orders);

JavaScript understands it like this: 

const ProtectedOrders = function ProtectedPage(props) {
  if (!isLoggedIn) return <Login />;
  return <Orders {...props} />;
};

Now when React renders: 

<ProtectedOrders />

It runs the returned function and checks login before showing Orders.

withAuth(Orders)
↓ Returns New Function
↓ Stored In ProtectedOrders
↓ <ProtectedOrders /> Runs That Function

Final Interview Answer

CAF is a composition pattern where a component is passed to another component and rendered there. It is useful when the same component logic needs different UI. HOC is a pattern where a component is passed to a function, and that function returns a new component with extra behavior like authentication, permissions, logging, or analytics.
