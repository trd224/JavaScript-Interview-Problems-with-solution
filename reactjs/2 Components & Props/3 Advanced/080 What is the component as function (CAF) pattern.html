080. What is the Component as Function (CAF) Pattern?

One-Line Answer

The Component as Function (CAF) pattern means passing a component as a prop and rendering it like a normal React component.

Why Do We Need It?

Sometimes a component wants to let its parent decide what UI to render.

Instead of hardcoding the UI, we pass a component.

Parent
↓ Pass Component
↓ Child
↓ Render That Component

Basic Example

User Component
function User() {
  return <h3>Rizwan</h3>;
}
Card Component
function Card({ Component }) {
  return (
    <div>
      <Component />
    </div>
  );
}
Usage
<Card Component={User} />
User Component
↓ Passed As Prop
↓ Card Receives It
↓ <Component />
↓ User Renders

Benefits

  • Highly reusable components.
  • Parent controls rendering.
  • Avoids duplicate UI code.
  • Flexible component composition.
  • Cleaner than large conditional rendering blocks.

Interview Answer

The Component as Function (CAF) pattern is a React composition pattern where a component is passed as a prop to another component and rendered dynamically. This allows reusable components to delegate UI rendering to their parents while keeping the underlying logic unchanged.

CAF vs Higher Order Component (HOC)

CAF and HOC are related because both work with components as values, but they solve different problems.

CAF (Component as Function) HOC (Higher Order Component)
Pass component as a prop Pass component into a function
Child renders that component Function returns a new component
Used for composition Used for enhancement

HOC Example

Pass Component To Function

const ProtectedDashboard = withAuth(Dashboard);
Dashboard Component
↓ withAuth()
↓ Creates New Component

CAF is NOT a Higher Order Component.
CAF passes a component to another component as a prop and renders it. HOC passes a component to a function which returns a new enhanced component.
