074. What is forwardRef and why is it needed?

One-Line Definition

forwardRef is a React API that allows a parent component to pass a ref through a custom component and access a DOM element inside that component.

Why Is It Needed?

Normally, refs work directly with DOM elements. 

const inputRef = useRef();

<input ref={inputRef} />

But refs do not automatically work with custom components.

The Problem

Custom Component

function CustomInput() {
  return <input />;
}

Parent Component

function App() {
  const inputRef = useRef();

  return <CustomInput ref={inputRef} />;
}

This does not work.

The ref stops at the CustomInput component and never reaches the actual input element.

Solution: forwardRef

forwardRef allows the child component to receive the ref and forward it to its internal DOM element.

CustomInput Component

import { forwardRef } from 'react';

const CustomInput = forwardRef((props, ref) => {
  return <input ref={ref} {...props} />;
});

export default CustomInput;

Parent Component

import { useRef } from 'react';

function App() {
  const inputRef = useRef();

  return <CustomInput ref={inputRef} />;
}

How It Works

Parent Creates Ref
↓
Passes Ref To Child
↓
forwardRef Receives Ref
↓
Ref Attached To Input
↓
Parent Can Access Input

Real Example - Focus Input

Child Component

import { forwardRef } from 'react';

const CustomInput = forwardRef((props, ref) => {
  return <input ref={ref} {...props} />;
});

Parent Component

import { useRef } from 'react';

function App() {
  const inputRef = useRef();

  const focusInput = () => inputRef.current.focus();

  return (
    <>
      <CustomInput ref={inputRef} />
      <button onClick={focusInput}>Focus Input</button>
    </>
  );
}

Output

When the button is clicked, the parent component focuses the input element inside the child component.

forwardRef Syntax

const MyComponent = forwardRef((props, ref) => {
  return <input ref={ref} {...props} />;
});

Common Use Cases

  • Focus an input field.
  • Scroll to an element.
  • Select text automatically.
  • Integrate with third-party libraries.
  • Build reusable UI components.

forwardRef + useImperativeHandle

forwardRef is often used together with useImperativeHandle. 

useImperativeHandle(ref, () => ({
  focus() {
    inputRef.current.focus();
  }
}));

Instead of exposing the whole DOM element, we expose only specific methods.

Without forwardRef vs With forwardRef

Feature Without forwardRef With forwardRef
Pass ref to custom component ❌ ✅
Access child DOM element ❌ ✅
Focus child input ❌ ✅
Create reusable inputs Difficult Easy

Real Project Example

Suppose our Aadhaar Authentication application has a reusable AadhaarInput component. When validation fails, we may want to automatically focus the Aadhaar field. forwardRef allows the parent component to access the actual input element inside AadhaarInput and call focus().

Important Interview Points

  • Refs do not automatically pass through custom components.
  • forwardRef forwards refs to child DOM elements.
  • Used in reusable component libraries.
  • Commonly used for focus and scroll operations.
  • Often used with useImperativeHandle.

Short Interview Answer

forwardRef is a React API that allows a parent component to pass a ref through a child component and access a DOM element inside that child. It is commonly used for focus management, scrolling, and reusable UI components.

One-Line Interview Answer

forwardRef allows a parent component to forward a ref through a custom component to access a DOM element inside it.
