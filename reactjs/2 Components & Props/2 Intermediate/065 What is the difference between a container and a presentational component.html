065. What is the difference between a container and a presentational component?

One-Line Definition

Container components handle logic and data, while presentational components focus only on UI rendering.

Simple Understanding

React applications are often divided into:

  • Logic components
  • UI components

This separation improves:

  • Code organization
  • Reusability
  • Maintainability

Main Idea

Container Component
↓
Handles Logic + State + API
↓
Passes Data via Props
↓
Presentational Component
↓
Renders UI

Presentational Component

Also called:

  • UI Component
  • Dumb Component

Responsibilities

  • Display UI
  • Receive props
  • Focus on design
  • Reusable rendering

Example

function UserCard({ name }) {

  return (

    <div>

      <h1>{name}</h1>

    </div>

  );

}

Why It Is Presentational

  • No API calls
  • No business logic
  • No complex state handling
  • Only renders UI

Container Component

Also called:

  • Smart Component
  • Logic Component

Responsibilities

  • Manage state
  • Fetch API data
  • Handle business logic
  • Pass data to UI components

Example

import { useState, useEffect } from 'react';

function UserContainer() {

  const [user, setUser] = useState('');

  useEffect(() => {

    setUser('Rizwan');

  }, []);

  return (

    <UserCard name={user} />

  );

}

How It Works

Component Responsibility
UserContainer Logic + state
UserCard UI rendering

Data Flow

API / State
↓
Container Component
↓
Props
↓
Presentational Component

Main Differences

Container Component Presentational Component
Handles logic Handles UI
May use state Usually stateless
Fetches data Receives props
Business logic UI rendering only
Less reusable Highly reusable

Real Project Example

Container Presentational
OTP logic component OTP input UI
Authentication API handler Login form UI
Dashboard data loader Chart component
Redux-connected component Reusable table UI

Why This Pattern Is Useful

  • Cleaner architecture
  • Easier testing
  • Better reusability
  • Separation of concerns
  • Improved maintainability

Modern React Note

With Hooks, the difference is less strict now because one component can handle both logic and UI cleanly.

Example Using Hooks

Modern React often combines:

  • useState
  • useEffect
  • UI rendering

inside a single functional component.

Still Important Concept

Even today, separating logic and UI is considered good architecture in large applications.

Important Interview Points

  • Container components handle logic and data.
  • Presentational components focus on UI.
  • Container passes data through props.
  • Improves separation of concerns.
  • Hooks reduced strict separation in modern React.

Short Interview Answer

Container components manage logic, state, and data fetching, while presentational components focus only on displaying UI using props.

One-Line Interview Answer

Container components handle logic, while presentational components handle UI rendering.
