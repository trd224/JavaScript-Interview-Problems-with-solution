070. What are Render Props and What Problem Do They Solve?

One-Line Definition

A Render Prop is a technique where a component receives a function as a prop and calls that function to decide what UI should be rendered.

What Problem Do Render Props Solve?

Before Hooks existed, React developers often needed to share stateful logic between multiple components.

If the same logic was needed in many places, code duplication became a problem.

Reusable Logic
↓
Used By Multiple Components
↓
Avoid Duplicate Code
↓
Render Props

Without Render Props

Suppose we need mouse position tracking in multiple components. 

function ComponentA() {

  const [position, setPosition] = useState({
    x: 0,
    y: 0
  });

  // Mouse tracking logic

}

Now if another component needs the same logic, we must duplicate the code.

Render Props Solution

We move the reusable logic into a separate component and let consumers decide what UI to render.

MouseTracker Component

import { useState } from 'react';

function MouseTracker({ render }) {

  const [position, setPosition] = useState({
    x: 0,
    y: 0
  });

  const handleMouseMove = event => {

    setPosition({
      x: event.clientX,
      y: event.clientY
    });

  };

  return (
    <div
      style={{ height: '100vh' }}
      onMouseMove={handleMouseMove}
    >
      {render(position)}
    </div>
  );

}

Using the Render Prop

function App() {

  return (
    <MouseTracker
      render={(position) => (
        <h2>
          X: {position.x},
          Y: {position.y}
        </h2>
      )}
    />
  );

}

The MouseTracker component owns the logic, while the App component decides how to display the data.

How It Works

Component Owns Logic
↓
Pass Data To Function
↓
Function Returns UI
↓
React Renders UI

Why Is It Called a Render Prop?

Because the prop contains a function that returns JSX. 

render={(data) => (
  <h2>{data}</h2>
)}

The function controls what gets rendered.

children as a Render Prop

Sometimes the function is passed through the children prop instead of a prop named render. 

function MouseTracker({ children }) {

  return children({
    x: 100,
    y: 200
  });

}

Usage

<MouseTracker>
  {(position) => (
    <h2>
      {position.x}, {position.y}
    </h2>
  )}
</MouseTracker>

Advantages

  • Promotes code reuse.
  • Separates logic from UI.
  • Makes components highly flexible.
  • Avoids code duplication.
  • Allows different UIs to share the same logic.

Real Project Example

Before React Hooks, a reusable Aadhaar Authentication component could have used Render Props to share OTP timer logic, device connection logic, or authentication status logic across multiple screens while allowing each screen to render a different UI.

Why Are Render Props Less Common Today?

After React Hooks were introduced, most logic-sharing use cases moved to Custom Hooks because they provide cleaner and more readable code. 

function useMousePosition() {

  const [position, setPosition] = useState({
    x: 0,
    y: 0
  });

  return position;

}

Today, Custom Hooks are usually preferred over Render Props.

Important Interview Points

  • A Render Prop is a function passed as a prop.
  • The function returns JSX.
  • Used for sharing stateful logic.
  • Separates logic from presentation.
  • Popular before React Hooks.
  • Custom Hooks are now the preferred solution.

Short Interview Answer

Render Props are a React pattern where a component receives a function as a prop and calls it to determine what UI to render. They solve the problem of sharing reusable stateful logic between multiple components while allowing each component to render its own UI.

One-Line Interview Answer

Render Props use a function prop that returns JSX, allowing reusable logic to be shared across different components.
