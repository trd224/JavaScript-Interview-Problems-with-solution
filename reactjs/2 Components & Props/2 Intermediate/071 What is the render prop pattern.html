071. What is the Render Prop Pattern?

One-Line Definition

The Render Prop Pattern is a React pattern where a component receives a function as a prop and calls that function to determine what UI should be rendered.

Why Is It Needed?

Sometimes multiple components need the same stateful logic but different UI.

Instead of duplicating logic, we place the logic inside one component and let consumers decide how the UI should be rendered.

Reusable Logic
↓
One Component Owns Logic
↓
Consumer Provides Render Function
↓
Consumer Controls UI

Basic Render Prop Pattern

The component receives a function prop called render. 

function DataProvider({ render }) {

  const user = {
    name: 'Rizwan',
    age: 39
  };

  return render(user);

}

Usage

function App() {

  return (
    <DataProvider
      render={(user) => (
        <h2>{user.name}</h2>
      )}
    />
  );

}

The DataProvider component provides the data, while the App component decides how to display it.

Real Example - Mouse Tracker

A classic Render Prop Pattern example is mouse position tracking. 

import { useState } from 'react';

function MouseTracker({ render }) {

  const [position, setPosition] = useState({
    x: 0,
    y: 0
  });

  const handleMouseMove = event => {

    setPosition({
      x: event.clientX,
      y: event.clientY
    });

  };

  return (
    <div
      style={{ height: '100vh' }}
      onMouseMove={handleMouseMove}
    >
      {render(position)}
    </div>
  );

}

Usage

function App() {

  return (
    <MouseTracker
      render={(position) => (
        <h2>
          X: {position.x}
          Y: {position.y}
        </h2>
      )}
    />
  );

}

How It Works

Component Owns Logic
↓
Creates Data
↓
Calls Render Function
↓
Function Returns JSX
↓
React Renders UI

Using Children as a Render Prop

Many libraries use the children prop instead of a render prop. 

function DataProvider({ children }) {

  const user = {
    name: 'Rizwan'
  };

  return children(user);

}

Usage

<DataProvider>
  {(user) => (
    <h2>{user.name}</h2>
  )}
</DataProvider>

Advantages

  • Promotes code reuse.
  • Separates logic from UI.
  • Reduces code duplication.
  • Makes components flexible.
  • Allows different UIs to share the same logic.

Disadvantages

  • Can create deeply nested JSX.
  • Harder to read in large applications.
  • Less popular after React Hooks.

Real Project Example

Before React Hooks, our Aadhaar Authentication application could have used the Render Prop Pattern to share OTP timer logic, authentication status logic, or device connection logic across multiple screens while allowing each screen to render a different UI.

Why Is It Less Common Today?

After React Hooks were introduced in React 16.8, most reusable logic moved to Custom Hooks because they provide cleaner and more readable code. 

function useUser() {

  return {
    name: 'Rizwan'
  };

}

Today, Custom Hooks are usually preferred over Render Props.

Important Interview Points

  • A Render Prop is a function passed as a prop.
  • The function returns JSX.
  • Used to share stateful logic.
  • Separates logic from presentation.
  • Popular before React Hooks.
  • Custom Hooks are now the preferred solution.

Short Interview Answer

The Render Prop Pattern is a React pattern where a component receives a function as a prop and calls it to determine what UI should be rendered. It solves the problem of sharing reusable logic while allowing different components to render different UIs.

One-Line Interview Answer

The Render Prop Pattern uses a function prop that returns JSX, allowing reusable logic to be shared across different components.
