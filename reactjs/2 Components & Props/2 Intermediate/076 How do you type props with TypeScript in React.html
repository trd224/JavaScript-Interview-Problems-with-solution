076. How do you type props with TypeScript in React?

One-Line Definition

In React with TypeScript, we define a type or interface for component props so TypeScript can validate the data passed to the component.

Why Is It Needed?

TypeScript helps catch prop-related errors during development instead of at runtime.

Without TypeScript

<User name="Rizwan" age="39" />

React will allow this, but age is actually a string instead of a number.

With TypeScript

TypeScript immediately shows an error because age should be a number.

Typing Props Using type

Create Props Type

type UserProps = {
  name: string;
  age: number;
};

Use Props Type

type UserProps = {
  name: string;
  age: number;
};

function User({ name, age }: UserProps) {
  return (
    <div>
      <h2>{name}</h2>
      <p>{age}</p>
    </div>
  );
}

export default User;

Usage

<User name="Rizwan" age={39} />

TypeScript Error Example

<User name="Rizwan" age="39" />
Error: Type 'string' is not assignable to type 'number'.

Typing Props Using interface

interface UserProps {
  name: string;
  age: number;
}

function User({ name, age }: UserProps) {
  return (
    <div>
      <h2>{name}</h2>
      <p>{age}</p>
    </div>
  );
}

type vs interface

Both are valid and commonly used for React props.

Feature type interface
Props Typing ✅ ✅
Union Types ✅ ❌

Optional Props

Use ? to make a prop optional. 

type UserProps = {
  name: string;
  age?: number;
};

Usage

<User name="Rizwan" />

Now age is optional.

Default Values with Optional Props

type UserProps = {
  name: string;
  age?: number;
};

function User({ name, age = 18 }: UserProps) {
  return <h2>{name} - {age}</h2>;
}

Typing Function Props

type ButtonProps = {
  onClick: () => void;
};

function Button({ onClick }: ButtonProps) {
  return (
    <button onClick={onClick}>
      Click
    </button>
  );
}

Usage

<Button onClick={() => console.log('Clicked')} />

Typing Object Props

type UserProps = {
  user: {
    id: number;
    name: string;
    email: string;
  };
};

function User({ user }: UserProps) {
  return <h2>{user.name}</h2>;
}

Usage

<User
  user={{
    id: 1,
    name: 'Rizwan',
    email: 'rizwan@gmail.com'
  }}
/>

Typing Array Props

type UserListProps = {
  users: string[];
};

function UserList({ users }: UserListProps) {
  return (
    <ul>
      {users.map(user => (
        <li key={user}>{user}</li>
      ))}
    </ul>
  );
}

Typing children

import { ReactNode } from 'react';

type CardProps = {
  children: ReactNode;
};

function Card({ children }: CardProps) {
  return <div>{children}</div>;
}

Usage

<Card>
  <h2>Hello</h2>
</Card>

Most Common Real-World Example

type UserProps = {
  name: string;
  age: number;
  city?: string;
};

function User({ name, age, city }: UserProps) {
  return (
    <div>
      <h2>{name}</h2>
      <p>Age: {age}</p>
      {city && <p>City: {city}</p>}
    </div>
  );
}

Usage

<User
  name="Rizwan"
  age={39}
  city="Kota"
/>

Real Project Example

In our Aadhaar Authentication application, components may receive Aadhaar numbers, department details, authentication status, and callback functions. TypeScript ensures every prop has the correct type before the application is deployed.

Important Interview Points

  • Props are typed using type or interface.
  • TypeScript validates props at compile time.
  • Optional props use ?.
  • Functions, arrays, objects, and children can all be typed.
  • TypeScript helps prevent runtime bugs.
  • Both type and interface are commonly used.

Short Interview Answer

Props are typed in React using TypeScript type aliases or interfaces. This allows TypeScript to validate incoming props and catch type-related errors during development.

One-Line Interview Answer

React props are typed using TypeScript types or interfaces to ensure components receive the correct data types.
