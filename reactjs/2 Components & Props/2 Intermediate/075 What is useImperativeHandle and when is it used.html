075. What is useImperativeHandle and when is it used?

One-Line Definition

useImperativeHandle is a React Hook used with forwardRef to control what a parent component can access through a ref.

Why Is It Needed?

Normally, when we use forwardRef, the parent gets access to the entire DOM element. 

const CustomInput = forwardRef((props, ref) => {
  return <input ref={ref} />;
});

Now the parent can access everything: 

inputRef.current.focus();
inputRef.current.value;
inputRef.current.select();
inputRef.current.style.color = 'red';

Sometimes we do not want to expose the entire DOM element.

We may want to expose only specific methods such as focus() or clear().

Solution: useImperativeHandle

useImperativeHandle lets us customize what is exposed through the ref.

Parent Creates Ref
↓
Child Receives Ref
↓
useImperativeHandle Controls Access
↓
Parent Gets Only Allowed Methods

Without useImperativeHandle

Child Component

const CustomInput = forwardRef((props, ref) => {
  return <input ref={ref} />;
});

Parent Component

const inputRef = useRef();

<CustomInput ref={inputRef} />

Parent receives the complete input element.

With useImperativeHandle

Child Component

import { forwardRef, useRef, useImperativeHandle } from 'react';

const CustomInput = forwardRef((props, ref) => {

  const inputRef = useRef();

  useImperativeHandle(ref, () => ({
    focus() {
      inputRef.current.focus();
    }
  }));

  return <input ref={inputRef} />;

});

Parent Component

function App() {

  const inputRef = useRef();

  return (
    <CustomInput ref={inputRef} />
  );

}

Using the Ref

inputRef.current.focus();

This works because focus() was explicitly exposed.

What Does the Parent Receive?

Instead of receiving the whole input element: 

inputRef.current

The parent receives: 

{
  focus: function
}

Expose Multiple Methods

useImperativeHandle(ref, () => ({
  focus() {
    inputRef.current.focus();
  },

  clear() {
    inputRef.current.value = '';
  },

  select() {
    inputRef.current.select();
  }
}));

Parent Usage

inputRef.current.focus();

inputRef.current.clear();

inputRef.current.select();

How It Works

forwardRef Receives Ref
↓
useImperativeHandle Runs
↓
Creates Custom API
↓
Parent Receives Custom API
↓
Parent Calls Methods

When Is It Used?

  • Reusable input components.
  • Focus management.
  • Custom modals.
  • Custom dropdowns.
  • Video player controls.
  • Canvas and chart components.
  • Third-party library integrations.

Real Example - Modal Component

Child Component

useImperativeHandle(ref, () => ({
  open() {
    setIsOpen(true);
  },

  close() {
    setIsOpen(false);
  }
}));

Parent Component

modalRef.current.open();

modalRef.current.close();

The parent can control the modal without knowing its internal implementation.

forwardRef vs useImperativeHandle

Feature forwardRef useImperativeHandle
Pass Ref To Child ✅ ❌
Customize Ref Value ❌ ✅
Required Together Usually Yes
Controls Exposed Methods ❌ ✅

Real Project Example

In our Aadhaar Authentication application, a reusable AadhaarInput component might expose only focus() and clear() methods. The parent component can call these methods when validation fails, without receiving direct access to the entire input element.

Important Interview Points

  • useImperativeHandle works with forwardRef.
  • It customizes what is exposed through a ref.
  • It helps hide internal implementation details.
  • It improves component encapsulation.
  • Commonly used in reusable component libraries.

Short Interview Answer

useImperativeHandle is a React Hook used with forwardRef to control what a parent component can access through a ref. Instead of exposing the entire DOM element, it allows the child component to expose only selected methods or properties.

One-Line Interview Answer

useImperativeHandle lets a child component customize what is exposed to a parent through a forwarded ref.
