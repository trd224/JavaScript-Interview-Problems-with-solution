077. What is React.FC and should you use it?

One-Line Definition

React.FC (React Functional Component) is a TypeScript type that can be used to define React function components.

What is React.FC?

React.FC is a generic type provided by React for typing functional components. 

type UserProps = {
  name: string;
};

const User: React.FC<UserProps> = ({ name }) => {
  return <h2>{name}</h2>;
};

Here React.FC tells TypeScript that User is a React component and UserProps is the props type.

Without React.FC

Modern React applications usually type props directly. 

type UserProps = {
  name: string;
};

function User({ name }: UserProps) {
  return <h2>{name}</h2>;
}

This is currently the most common approach.

How React.FC Works

Component
↓
React.FC<Props>
↓
TypeScript Validates Props
↓
Component Becomes Fully Typed

Benefits of React.FC

  • Clearly identifies a React component.
  • Provides type checking for props.
  • Provides IntelliSense support.
  • Ensures the component returns valid React elements.

Example

type ButtonProps = {
  text: string;
};

const Button: React.FC<ButtonProps> = ({ text }) => {
  return <button>{text}</button>;
};

Older React.FC Behavior

In older React versions, React.FC automatically included the children prop. 

const Card: React.FC = ({ children }) => {
  return <div>{children}</div>;
};

This was one of the reasons many developers used React.FC.

Current React Behavior

Modern React type definitions no longer automatically include children.

You must explicitly type children yourself. 

import { ReactNode } from 'react';

type CardProps = {
  children: ReactNode;
};

function Card({ children }: CardProps) {
  return <div>{children}</div>;
}

Should You Use React.FC?

Most modern React + TypeScript projects do not use React.FC and instead type props directly.

Recommended Modern Approach

type UserProps = {
  name: string;
  age: number;
};

function User({ name, age }: UserProps) {
  return (
    <div>
      <h2>{name}</h2>
      <p>{age}</p>
    </div>
  );
}

Why Many Developers Avoid React.FC?

  • Adds extra typing that is often unnecessary.
  • Direct prop typing is simpler.
  • Can make generic components harder to type.
  • Modern React documentation does not recommend it.
  • Most new codebases prefer direct props typing.

Important Interview Points

  • React.FC stands for React Functional Component.
  • It is used to type functional components.
  • Older versions automatically included children.
  • Modern React types no longer include children automatically.
  • Most modern projects prefer direct props typing.
  • React.FC is still valid but less commonly used today.

Short Interview Answer

React.FC is a TypeScript type for React functional components. While it provides prop type checking and component typing, most modern React projects prefer direct props typing because it is simpler and more explicit.
