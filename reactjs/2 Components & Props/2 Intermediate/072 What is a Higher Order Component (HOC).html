072. What is a Higher Order Component (HOC)?

One-Line Definition

A Higher Order Component (HOC) is a function that takes a component as input and returns a new enhanced component with additional functionality.

Why Is It Needed?

Sometimes multiple components need the same logic.

Instead of duplicating code, we can place the shared logic inside a Higher Order Component and reuse it across many components.

Shared Logic
↓
Higher Order Component
↓
Wrap Existing Component
↓
Return Enhanced Component

What Does "Higher Order" Mean?

A Higher Order Function is a function that works with other functions.

Similarly, a Higher Order Component is a function that works with React components. 

function withSomething(Component) {

  return function EnhancedComponent() {

    return <Component />;

  };

}

The HOC receives a component and returns a new component.

In an HOC you can replace the named function with an anonymous function because it is just returning a component, but the function will lose a readable name in debugging.

Simple Example

User Component

function User() {

  return <h2>Rizwan</h2>;

}

HOC

function withBorder(Component) {

  return function EnhancedComponent() {

    return (
      <div style={{ border: '2px solid blue' }}>
        <Component />
      </div>
    );

  };

}

Create Enhanced Component

const UserWithBorder = withBorder(User);

Usage

<UserWithBorder />

Output

A blue border is added around the User component without modifying the original component.

Real Example - Authentication

A common use case is route protection.

Protected HOC

function withAuth(Component) {

  return function AuthenticatedComponent(props) {

    const isLoggedIn = true;

    if (!isLoggedIn) {
      return <h2>Access Denied</h2>;
    }

    return <Component {...props} />;

  };

}

Dashboard Component

function Dashboard() {

  return <h2>Dashboard</h2>;

}

Wrap Component

const ProtectedDashboard = withAuth(Dashboard);

Passing Props Through HOC

The HOC should forward props to the wrapped component. 


function withSomething(Component) {
  return function EnhancedComponent(props) {
    return <Component {...props} />;
  };
}

function Hello(props) {
  return <h1>Hello {props.name}</h1>;
}

const EnhancedHello = withSomething(Hello);

export default function App() {
  return <EnhancedHello name="Rizwan" />;
}

Advantages

  • Promotes code reuse.
  • Separates concerns.
  • Adds functionality without modifying components.
  • Keeps business logic centralized.
  • Useful for authentication, logging, permissions, and analytics.

Disadvantages

  • Can create deeply nested wrappers.
  • Makes component trees harder to debug.
  • Prop collisions can occur.
  • Less common after React Hooks.

Popular HOC Examples

  • connect() from Redux
  • withRouter() in older React Router versions
  • withAuth()
  • withLoading()
  • withPermissions()

Real Project Example

Before React Hooks, our Aadhaar Authentication application could have used HOCs such as withAuth() to protect screens, withPermissions() to control access, or withAuditLogs() to automatically log user actions across multiple components.

Why Are HOCs Less Common Today?

After React Hooks were introduced, most reusable logic moved into Custom Hooks because Hooks provide cleaner, simpler, and more readable code. 

function useAuth() {

  const isLoggedIn = true;

  return isLoggedIn;

}

Today, Custom Hooks are usually preferred over HOCs for sharing logic.

Important Interview Points

  • A HOC is a function that receives a component and returns a new component.
  • Used for sharing reusable logic.
  • Does not modify the original component.
  • Common before React Hooks.
  • connect() from Redux is a popular HOC example.
  • Custom Hooks are now the preferred solution.

Short Interview Answer

A Higher Order Component (HOC) is a function that takes a component and returns an enhanced version of that component with additional functionality. It is commonly used for logic reuse, authentication, permissions, and analytics.
