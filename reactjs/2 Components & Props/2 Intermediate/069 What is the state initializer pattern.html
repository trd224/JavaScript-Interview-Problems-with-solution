069. What is the State Initializer Pattern?

One-Line Definition

The State Initializer Pattern is a React pattern that allows consumers to provide the initial state value and reset the state back to that initial value when needed.

Why Is It Used?

Sometimes reusable components or custom hooks should not decide their own initial state.

Instead, we allow the consumer to provide the initial value.

This gives more flexibility and reusability.

Simple Example

function Counter() {

  const [count, setCount] = useState(0);

  return (
    <h2>{count}</h2>
  );

}

Here the initial value is always 0.

Consumers cannot customize it.

State Initializer Pattern

Instead, we accept an initial value from the consumer. 

import { useState } from 'react';

function useCounter(initialCount = 0) {

  const [count, setCount] = useState(initialCount);

  return {
    count,
    setCount
  };

}

Usage

function App() {

  const { count } = useCounter(100);

  return <h2>{count}</h2>;

}

Now the consumer controls the initial state value.

Main Requirements of State Initializer Pattern

  • Allow configurable initial state.
  • Provide a way to reset state.
  • Reset back to the original initial value.

Complete State Initializer Pattern

import { useState } from 'react';

function useCounter(initialCount = 0) {

  const [count, setCount] = useState(initialCount);

  const reset = () => {
    setCount(initialCount);
  };

  return {
    count,
    setCount,
    reset
  };

}

Usage

function App() {

  const { count, setCount, reset } = useCounter(10);

  return (
    <div>
      <h2>{count}</h2>

      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>

      <button onClick={reset}>
        Reset
      </button>
    </div>
  );

}

How It Works

Consumer Provides Initial State
↓
Hook Uses Initial Value
↓
State Changes
↓
Reset Called
↓
Original Initial Value Restored

Example Output

Why Not Hardcode State?

Hardcoded State State Initializer Pattern
Fixed initial value Consumer decides initial value
Less reusable Highly reusable
Harder to customize Easy to customize

Real Project Example

Suppose our Aadhaar Authentication application contains a reusable OTP timer hook. Different screens may require different starting times such as 30 seconds, 60 seconds, or 120 seconds. The State Initializer Pattern allows the consumer to provide the initial timer value and reset it whenever needed.

Important Interview Points

  • The consumer provides the initial state value.
  • The hook or component uses that value to initialize state.
  • A reset function usually resets state to the original value.
  • Improves reusability and flexibility.
  • Commonly used in custom hooks.

Short Interview Answer

The State Initializer Pattern is a React pattern where consumers provide the initial state value. The component or hook uses that value to initialize state and usually provides a reset function that restores the original initial value.
