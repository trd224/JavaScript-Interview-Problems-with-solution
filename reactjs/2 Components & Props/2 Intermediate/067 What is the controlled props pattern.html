067. What is the controlled props pattern?

One-Line Definition

The controlled props pattern allows parent components to fully control a child component’s state using props and callback functions.

Simple Understanding

Normally a component manages its own state internally.

But in the controlled props pattern:

Parent Controls State
↓
Child Receives State via Props
↓
Child Requests Updates via Callback

Why It Is Called Controlled

Because the parent component controls the child component’s behavior and state.

Basic Example

function Input({ value, onChange }) {

  return (

    <input

      value={value}

      onChange={(e) => onChange(e.target.value)}

    />

  );

}

Parent Component

import { useState } from 'react';

function App() {

  const [name, setName] = useState('');

  return (

    <Input

      value={name}

      onChange={setName}

    />

  );

}

How It Works

Part Responsibility
Parent Stores state
Child Displays state
Callback Requests state update

Data Flow

Parent State
↓
Passed as Props
↓
Child Renders UI
↓
User Interaction
↓
Child Calls Callback
↓
Parent Updates State

Most Common Example

Controlled form inputs. 

<input

  value={name}

  onChange={(e) => setName(e.target.value)}

/>

Why This Pattern Is Useful

  • Centralized state management
  • Predictable UI behavior
  • Easier validation
  • Better debugging
  • Reusable components

Controlled vs Uncontrolled Components

Controlled Uncontrolled
State managed by React State managed by DOM
Uses value prop Uses ref
Predictable Less controlled
Easier validation Simple for small forms

Example of Uncontrolled Input

<input ref={inputRef} />

Advanced Controlled Props Pattern

Some reusable components support both:

  • Internal state
  • External controlled state

Example

function Toggle({ checked, onChange }) {

  return (

    <button onClick={() => onChange(!checked)}>

      {checked ? 'ON' : 'OFF'}

    </button>

  );

}

Parent Controls Toggle

function App() {

  const [enabled, setEnabled] = useState(false);

  return (

    <Toggle

      checked={enabled}

      onChange={setEnabled}

    />

  );

}

Real Project Use Cases

Feature Controlled State
OTP input OTP value

Important Benefit

Controlled props make components reusable because external components can fully manage their behavior.

Important Interview Points

  • Parent controls child state using props.
  • Child updates state through callbacks.
  • Commonly used with form inputs.
  • Provides predictable behavior.
  • Improves component reusability.

Short Interview Answer

The controlled props pattern is a React pattern where parent components manage child component state through props and callback functions.

One-Line Interview Answer

Controlled props allow parent components to fully control child component state and behavior.
