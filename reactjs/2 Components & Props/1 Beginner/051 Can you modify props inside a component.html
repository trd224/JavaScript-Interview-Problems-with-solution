051. Can you modify props inside a component?

One-Line Definition

No, props should never be modified inside a component because props are read-only in React.

Why Props Cannot Be Modified

React follows:

One-Way Data Flow

Data always moves:

Parent Component
↓
Child Component

The child component receives props but should not change them.

Props Are Immutable

Immutable means:

Cannot be changed directly.

Wrong Example

function User(props) {

  // Wrong

  props.name = 'Aman';

  return (

    <h1>
      {props.name}
    </h1>

  );

}

Why This Is Wrong

  • Breaks React data flow
  • Makes UI unpredictable
  • Can cause rendering bugs
  • Violates React principles

Correct Approach

If data needs to change:

  • Use state
  • Or ask parent component to update props

Correct Example Using State

import { useState } from 'react';

function User({ initialName }) {

  const [name, setName] = useState(initialName);

  return (

    <button onClick={() => setName('Aman')}>
      {name}
    </button>

  );

}

What Happens Here?

Item Purpose
initialName prop Initial value from parent
state variable Local editable value
setName() Updates local state

Flow Diagram

Parent Passes Props
↓
Child Receives Props
↓
Child Uses Props
↓
State Handles Changes

What If Parent Needs Updated Data?

Child components should notify parent using callback functions.

Example

function Child({ changeName }) {

  return (

    <button onClick={() => changeName('Aman')}>
      Change Name
    </button>

  );

}

Parent Updates the Data

Parent owns the state.

Child only requests updates.

Important Difference

Props State
Read-only Mutable using setState
Passed from parent Managed inside component
External data Internal component data

Real Project Use Cases

Scenario Correct Approach
User profile update Update parent state
OTP verification Use state for changes
Theme switch Parent controls theme props
Form editing Use local state

Important React Principle

Props are inputs, not editable variables.

Important Interview Points

  • Props are read-only.
  • Props should never be modified directly.
  • React follows one-way data flow.
  • Use state for editable data.
  • Parent components own props data.

Short Interview Answer

No, props cannot be modified inside a component because they are immutable and follow React’s one-way data flow. State should be used for mutable data.
