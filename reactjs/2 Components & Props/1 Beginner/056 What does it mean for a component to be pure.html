056. What does it mean for a component to be "pure"?

One-Line Definition

A pure component is a component that always returns the same output for the same props and state without causing side effects.

Simple Understanding

A pure component behaves predictably.

If input stays same:

Same Props + Same State
↓
Same UI Output

Real-Life Analogy

Think about a calculator. 

2 + 2 = 4

Every time you give same input: 

2 + 2

You always get: 

4

This predictable behavior is purity.

Pure Component Example

function Greeting({ name }) {

  return (

    <h1>
      Hello {name}
    </h1>

  );

}

Why This Is Pure

  • Same name prop gives same output
  • No external changes
  • No side effects
  • Predictable rendering

Example

<Greeting name="Rizwan" />

Output

Hello Rizwan

What Makes a Component Impure?

A component becomes impure if it:

  • Modifies external variables
  • Changes props directly
  • Produces side effects during rendering
  • Returns different output for same input

Impure Example

let count = 0;

function Counter() {

  count++;

  return (

    <h1>
      {count}
    </h1>

  );

}

Why This Is Impure

Same component call gives different output every time. 

1
2
3
4

Pure Components and Performance

Pure components help React optimize rendering.

React can skip unnecessary re-renders when props/state have not changed.

Flow Diagram

React Receives New Props/State
↓
Compare Old vs New Values
↓
Same Values?
↓
Skip Re-render

React.PureComponent

React provides: 

React.PureComponent

for class components.

Example

import React from 'react';

class User extends React.PureComponent {

  render() {

    return (

      <h1>
        {this.props.name}
      </h1>

    );

  }

}

What PureComponent Does

It automatically performs:

Shallow comparison of props and state

Functional Component Alternative

Modern React commonly uses: 

React.memo()

Example

const User = React.memo(({ name }) => {

  return (

    <h1>
      {name}
    </h1>

  );

});

Important Note About Shallow Comparison

React compares:

  • Primitive values directly
  • Object references, not deep object values

Example Problem

const user = { name: 'Rizwan' };

New object references can still trigger re-rendering.

Real Project Use Cases

Feature Benefit of Pure Components
Dashboard widgets Prevent unnecessary renders
Large tables Improve rendering performance
Authentication forms Efficient UI updates
Live charts Reduce rendering cost

Pure vs Impure Component

Pure Component Impure Component
Predictable output Unpredictable output
No side effects May create side effects
Optimizable Harder to optimize
Same input → same output Same input → different output

Important Interview Points

  • Pure components return same output for same input.
  • No side effects during rendering.
  • React.PureComponent uses shallow comparison.
  • React.memo is used for functional components.
  • Pure components improve performance.

Short Interview Answer

A pure component is a component that produces the same UI output for the same props and state and avoids side effects, allowing React to optimize rendering performance.

One-Line Interview Answer

A pure component always returns the same output for the same props and state.

React.memo in React

One-Line Definition

React.memo is a higher-order component used to prevent unnecessary re-rendering of functional components.

Syntax

const MemoizedComponent = React.memo(Component);

Simple Understanding

Normally when a parent component re-renders, all child components also re-render.

React.memo helps skip child rendering if props have not changed.

Parent Re-render
↓
React.memo Checks Props
↓
Same Props?
↓
Skip Re-render

Example

import React, { useState } from 'react';

const Child = React.memo(({ count }) => {

  console.log('Child Rendered');

  return (
    <h1>{count}</h1>
  );

});

function App() {

  const [count, setCount] = useState(0);
  const [text, setText] = useState('');

  return (

    <div>

      <Child count={count} />

      <button
        onClick={() => setCount(count + 1)}
      >
        Increment
      </button>

      <input
        value={text}
        onChange={(e) => setText(e.target.value)}
      />

    </div>

  );

}

How It Works

  • If props stay same → component does not re-render
  • If props change → component re-renders
  • Uses shallow comparison internally

Output Behavior

Typing in input re-renders App component but Child component does not re-render because count prop is unchanged.

Shallow Comparison

React.memo compares previous props and new props using shallow comparison. 

prevProps === nextProps

Primitive Value Example

<Child value={10} />

Object Reference Problem

<Child data={{ name: 'Rizwan' }} />

New object references cause re-render even if values are same.

Using with useCallback

Functions also create new references during every render.

Without useCallback

<Child
  onClick={() => console.log('click')}
/>

With useCallback

const handleClick = useCallback(() => {

  console.log('click');

}, []);

<Child onClick={handleClick} />

React.memo vs useMemo vs useCallback

Hook/API Purpose
React.memo Memoize component rendering
useMemo Memoize calculated values
useCallback Memoize functions

Important Interview Points

  • React.memo works only for functional components.
  • It performs shallow comparison of props.
  • Helps optimize rendering performance.
  • Objects/functions may still trigger re-renders.
  • Often used with useCallback and useMemo.

Short Interview Answer

React.memo is used to memoize functional components and prevent unnecessary re-rendering when props remain unchanged.

One-Line Interview Answer

React.memo prevents unnecessary re-rendering of functional components.
