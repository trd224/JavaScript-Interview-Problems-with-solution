059. What is the difference between onClick in HTML and React?

One-Line Definition

In HTML, onclick uses lowercase and executes JavaScript strings directly, while React uses camelCase onClick and passes function references.

HTML onClick Example

<button onclick="showMessage()">
  Click
</button>

How HTML Works

  • Uses lowercase onclick
  • Executes JavaScript code as a string
  • Direct DOM-based event handling

React onClick Example

function App() {

  const showMessage = () => {

    console.log('Clicked');

  };

  return (

    <button onClick={showMessage}>
      Click
    </button>

  );

}

How React Works

  • Uses camelCase onClick
  • Passes function references
  • Uses SyntheticEvent system
  • Integrated with React rendering system

Main Difference

HTML React
onclick onClick
Lowercase camelCase
String execution Function reference
Native DOM events SyntheticEvents
Manual DOM handling React-managed rendering

Important React Rule

React expects a function, not a string.

Wrong React Example

// Wrong

<button onClick="showMessage()">
  Click
</button>

Correct React Example

<button onClick={showMessage}>
  Click
</button>

Passing Parameters in React

<button onClick={() => showMessage('Rizwan')}>
  Click
</button>

Why React Uses Function References

  • Better performance
  • Cleaner architecture
  • Safer code
  • Works with React rendering lifecycle

HTML Event Flow

React Event Flow

React SyntheticEvent System
↓
Function Handler Executes
↓

React Synthetic Events

React wraps browser events inside:

SyntheticEvent

This provides:

  • Cross-browser compatibility
  • Consistent behavior
  • Optimized performance

Another Important Difference

HTML

<button onclick="alert('Hello')">
  Click
</button>

React

<button onClick={() => alert('Hello')}>
  Click
</button>

Why React Avoids Strings

  • Avoids inline JavaScript strings
  • Improves maintainability
  • Prevents unsafe patterns
  • Keeps logic inside JavaScript

Important Interview Points

  • HTML uses onclick, React uses onClick.
  • React uses camelCase event names.
  • HTML executes strings, React passes functions.
  • React uses SyntheticEvents.
  • React integrates events with component rendering.

Short Interview Answer

HTML uses lowercase onclick with JavaScript strings, while React uses camelCase onClick with function references and SyntheticEvents for better architecture and performance.

One-Line Interview Answer

HTML onclick executes strings, while React onClick uses function references with SyntheticEvents.
