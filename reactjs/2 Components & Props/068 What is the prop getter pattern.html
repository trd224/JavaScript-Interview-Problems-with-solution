068. What is the prop getter pattern?

One-Line Definition

The prop getter pattern is a React pattern where helper functions return props that should be applied to UI elements.

Simple Understanding

Instead of manually attaching:

  • event handlers
  • attributes
  • accessibility props
  • logic-related props

a component provides helper functions that generate these props automatically.

Main Idea

Component Logic
↓
Prop Getter Function
↓
Returns Ready-to-Use Props
↓
Applied to JSX Elements

Basic Example

function useToggle() {

  const [on, setOn] = useState(false);

  const getTogglerProps = () => ({

    onClick: () => setOn(!on),
    'aria-pressed': on

  });

  return {

    on,
    getTogglerProps

  };

}

Usage

function App() {

  const {

    on,
    getTogglerProps

  } = useToggle();

  return (

    <button {...getTogglerProps()}>

      {on ? 'ON' : 'OFF'}

    </button>

  );

}

What Happens Internally

Step Description
1 Hook stores logic
2 Getter creates props object
3 Props spread into element
4 Element receives logic automatically

Why This Pattern Is Useful

  • Encapsulates logic
  • Improves reusability
  • Supports accessibility
  • Reduces repeated code
  • Creates flexible APIs

Common Real-World Usage

Popular libraries use this pattern:

  • Downshift
  • React Aria
  • Headless UI
  • Custom hooks libraries

Advanced Example

const getInputProps = () => ({

  value,
  onChange: handleChange,
  placeholder: 'Enter name',
  'aria-label': 'Name'

});

Usage

<input {...getInputProps()} />

Without Prop Getter Pattern

<input

  value={value}

  onChange={handleChange}

  placeholder="Enter name"

  aria-label="Name"

/>

With Prop Getter Pattern

<input {...getInputProps()} />

Main Benefit

Consumers do not need to know internal implementation details.

Combining User Props

Good prop getters allow custom props too.

Example

const getButtonProps = (props = {}) => ({

  ...props,

  onClick: (event) => {

    props.onClick?.(event);

    toggle();

  }

});

Why This Is Important

Users can still add their own:

  • onClick
  • className
  • style
  • custom attributes

Prop Getter vs Controlled Props

Prop Getter Pattern Controlled Props Pattern
Generates props automatically Parent controls state
Focuses on reusable behavior Focuses on external control
Returns prop objects Uses state props + callbacks

Real Project Use Cases

Feature Prop Getter Usage
Dropdowns Keyboard + accessibility props
OTP input Input handlers and validation
Custom toggles State + click handlers
Reusable forms Input configuration

Important Interview Points

  • Prop getters return props objects.
  • Often used with custom hooks.
  • Improves reusable logic.
  • Supports accessibility patterns.
  • Common in headless component libraries.

Short Interview Answer

The prop getter pattern is a React pattern where helper functions return preconfigured props that can be spread into UI elements to simplify reusable logic and behavior.

One-Line Interview Answer

The prop getter pattern returns reusable props objects for JSX elements.
