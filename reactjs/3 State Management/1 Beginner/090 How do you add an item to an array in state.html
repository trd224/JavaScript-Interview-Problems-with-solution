090. How do you add an item to an array in state?

One-Line Definition

In React, items are added to an array in state immutably using the spread operator.

Basic Array State

const [users, setUsers] = useState([

  'Rizwan',
  'Aman'

]);

Wrong Way

// Wrong

users.push('Rahul');

setUsers(users);

Why This Is Wrong

  • Mutates existing array directly
  • Reference remains same
  • React may not detect update
  • UI may not re-render properly

Correct Way

setUsers([

  ...users,
  'Rahul'

]);

What Happens Here?

Code Purpose
...users Copies old array items
'Rahul' Adds new item

Result

[
  'Rizwan',
  'Aman',
  'Rahul'
]

Main Flow

Old Array
↓
Create New Array Copy
↓
Add New Item
↓
New Array Reference
↓
React Re-Renders

Real Example

import { useState } from 'react';

function App() {

  const [users, setUsers] = useState([

    'Rizwan'

  ]);

  const addUser = () => {

    setUsers([

      ...users,
      'Aman'

    ]);

  };

  return (

    <div>

      <button onClick={addUser}>
        Add User
      </button>

      <ul>

        {users.map((user) => (
          <li>{user}</li>
        ))}

      </ul>

    </div>

  );

}

Output Before Click

Rizwan

Output After Click


Rizwan
Aman

Functional Update Version

Recommended when update depends on previous state. 

setUsers((prevUsers) => ([

  ...prevUsers,
  'Rahul'

]));

Why Functional Updates Are Better

  • Always uses latest state
  • Avoids stale state problems
  • Safer for async updates

Adding Object to Array

setUsers([

  ...users,

  {
    id: 1,
    name: 'Rizwan'
  }

]);

Adding Item at Beginning

setUsers([

  'Rahul',
  ...users

]);

Why React Requires Immutable Updates

React detects changes using reference comparison, so immutable updates create new array references that React can detect efficiently.

Mutable vs Immutable Array Updates

Mutable Immutable
push() [...array, item]
splice() filter()
sort() [...array].sort()
reverse() [...array].reverse()

Real Project Use Cases

Feature Array Update
Todo app Add new task
Shopping cart Add product
Chat app Add new message
Notifications Add alerts
OTP logs Add verification history

Important Interview Points

  • Never mutate arrays directly.
  • Use spread operator for immutable updates.
  • push() mutates original array.
  • Functional updates are safer.
  • React depends on new references for change detection.

Short Interview Answer

Items are added to React state arrays immutably using the spread operator to create a new array with existing items and the new item.

One-Line Interview Answer

Add items to React state arrays using immutable updates with the spread operator.
