092. What is lifting state up?

One-Line Definition

Lifting state up means moving shared state from child components to their closest common parent component.

Simple Understanding

Sometimes multiple components need the same data.

Instead of storing separate states in each child:

Move the state to the parent and share it using props.

Main Idea

Shared State Needed
↓
Move State to Parent
↓
Parent Controls State
↓
Pass Data & Functions via Props

Problem Without Lifting State Up

Two child components have separate states: 

// Child A state

const [count, setCount] = useState(0);

// Child B state

const [count, setCount] = useState(0);

Problem

  • States are not synchronized
  • Data becomes inconsistent
  • Components cannot share updates

Correct Approach

Move state to parent component.

Parent Component

import { useState } from 'react';

function App() {

  const [count, setCount] = useState(0);

  return (

    <div>

      <ChildA

        count={count}
        setCount={setCount}

      />

      <ChildB

        count={count}

      />

    </div>

  );

}

Child A

function ChildA({ count, setCount }) {

  return (

    <button

      onClick={() => setCount(count + 1)}

    >

      Increment

    </button>

  );

}

Child B

function ChildB({ count }) {

  return (

    <h1>{count}</h1>

  );

}

What Happens Now?

  • Parent owns state
  • Children receive same data
  • Updates stay synchronized
  • Single source of truth created

Data Flow

Parent State
↓
Props Passed to Children
↓
Child Updates Parent State
↓
Parent Re-Renders All Children

Why It Is Called "Lifting" State Up

Because state moves upward:

Parent Component
↑
Child Component

Main Benefits

  • Shared synchronized data
  • Single source of truth
  • Easier debugging
  • Better state management
  • Predictable application behavior

When To Lift State Up

Situation Need Lifting?
Multiple components use same data Yes
Only one component uses data No
Components must stay synchronized Yes

Real Project Use Cases

Feature Shared State
OTP verification OTP value
Theme switching Dark/light mode
Search filtering Search text
Authentication popup User authentication state
Shopping cart Cart items

Lifting State Up vs Global State

Lifting State Up Global State
Parent-child sharing Application-wide sharing
Local component hierarchy Redux/Context/Zustand
Simpler More scalable

Important Interview Points

  • State moves to closest common parent.
  • Creates single source of truth.
  • Data shared using props.
  • Helps synchronize components.
  • Very common React architecture pattern.

Short Interview Answer

Lifting state up is the process of moving shared state from child components to their closest common parent so multiple components can access synchronized data.

One-Line Interview Answer

Lifting state up means moving shared state to a common parent component.
