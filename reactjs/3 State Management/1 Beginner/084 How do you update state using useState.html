084. How do you update state using useState?

One-Line Definition

In React, state is updated using the setter function returned by useState().

Basic Syntax

const [state, setState] = useState(initialValue);
Part Meaning
state Current state value
setState Function used to update state
initialValue Initial state value

Basic Example

import { useState } from 'react';

function App() {

  const [count, setCount] = useState(0);

  return (

    <div>

      <h1>{count}</h1>

      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>

    </div>

  );

}

Output Before Click

0

Output After Click

1

How State Update Works

User Action
↓
setCount() Called
↓
React Updates State
↓
Component Re-Renders
↓
Updated UI Appears

Important Rule

Never update state directly.

Wrong Example

// Wrong

count = count + 1;

Correct Example

setCount(count + 1);

Updating String State

const [name, setName] = useState('Rizwan');

setName('Aman');

Updating Boolean State

const [isOpen, setIsOpen] = useState(false);

setIsOpen(true);

Toggle Example

const [darkMode, setDarkMode] = useState(false);

<button onClick={() => setDarkMode(!darkMode)}>
  Toggle
</button>

Updating Object State

const [user, setUser] = useState({

  name: 'Rizwan',
  age: 25

});

Correct Object Update

setUser({

  ...user,
  age: 26

});

Why Spread Operator Is Needed

useState replaces the entire object.

Without spread operator: 

// Wrong

setUser({

  age: 26

});

Result: 

{ age: 26 }

name property gets removed.

Updating Array State

const [users, setUsers] = useState([]);

setUsers([...users, 'Rizwan']);

Functional State Update

Recommended when new state depends on previous state. 

setCount((prevCount) => prevCount + 1);

Why Functional Update Is Important

React state updates may be asynchronous.

Functional updates ensure latest state is used.

Example

<button

  onClick={() => {

    setCount((prev) => prev + 1);
    setCount((prev) => prev + 1);

  }}

>

  +2

</button>

Output

Count increases by 2

Real Project Use Cases

Feature State Update
OTP input setOtp()
Popup visibility setIsOpen()
Theme switching setDarkMode()
Loading spinner setLoading()
API response setData()

Important Interview Points

  • useState returns state and setter function.
  • Setter function updates state.
  • State updates trigger re-rendering.
  • Never mutate state directly.
  • Use functional updates when depending on previous state.

Short Interview Answer

State is updated in React using the setter function returned by useState(), which updates the value and triggers component re-rendering.

One-Line Interview Answer

React state is updated using the setter function returned by useState().
