081. What is state in React?

One-Line Definition

State in React is a built-in object used to store and manage dynamic data that changes over time inside a component.

Simple Understanding

State stores values that can change while the application is running.

When state changes:

React automatically re-renders the component UI.

Examples of State

  • Counter value
  • Form input text
  • OTP value
  • Theme mode
  • Popup open/close status
  • API response data

Main Idea

State Changes
↓
React Detects Change
↓
Component Re-Renders
↓
UI Updates Automatically

Basic Example Using useState

import { useState } from 'react';

function App() {

  const [count, setCount] = useState(0);

  return (

    <div>

      <h1>{count}</h1>

      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>

    </div>

  );

}

Output Before Click

0

Output After Click

1

How useState Works

const [count, setCount] = useState(0);
Part Meaning
count Current state value
setCount Function to update state
0 Initial state value

Important Rule

Never modify state directly.

Wrong Example

// Wrong

count = count + 1;

Correct Example

setCount(count + 1);

Why Direct Modification Is Wrong

React will not detect state changes if state is modified directly.

Therefore:

UI will not update properly.

State Can Store Different Data Types

Type Example
String useState('Rizwan')
Number useState(0)
Boolean useState(false)
Array useState([])
Object useState({})

Example with Object State

const [user, setUser] = useState({

  name: 'Rizwan',
  age: 25

});

Updating Object State

setUser({

  ...user,
  age: 26

});

Why State Is Important

  • Creates interactive UI
  • Handles dynamic data
  • Controls application behavior
  • Updates UI automatically

State vs Props

State Props
Managed inside component Passed from parent
Mutable using setter Read-only
Dynamic data External data
Can change over time Controlled by parent

Real Project Use Cases

Feature State Usage
OTP authentication Store OTP value
Captcha system Track validation state
Popup modal Open/close status
Theme switching Dark/light mode
API loading Loading indicators

Important Interview Points

  • State stores dynamic component data.
  • Updating state triggers re-rendering.
  • useState is commonly used in functional components.
  • Never mutate state directly.
  • State makes React applications interactive.

Short Interview Answer

State in React is a mechanism used to store and update dynamic data inside components, where state changes automatically trigger UI re-rendering.

One-Line Interview Answer

State in React stores dynamic data that updates the UI when changed.
