086. Why must state updates be immutable?

One-Line Definition

State updates must be immutable because React detects changes by comparing object references, and direct mutation can prevent proper re-rendering.

What Does Immutable Mean?

Immutable means:

Do not modify existing state directly. Create a new updated copy instead.

Main Idea

Old State
↓
Create New Copy
↓
React Detects New Reference
↓
Component Re-Renders

Why React Needs Immutability

React mainly checks: 

Reference Equality

NOT deep object comparison.

Simple Example

const obj1 = { name: 'Rizwan' };

const obj2 = obj1;

obj2.name = 'Aman';

console.log(obj1.name);

Output

Aman

Why?

Because both variables point to the same object reference.

Wrong React State Update

// Wrong

user.name = 'Aman';

setUser(user);

Problem

React may not detect any change because the object reference is still the same.

Correct Immutable Update

setUser({

  ...user,
  name: 'Aman'

});

What Happens Here?

  • Spread operator creates new object
  • New reference created
  • React detects change
  • Component re-renders correctly

Reference Comparison

Update Type Reference Changes? React Detects Change?
Direct mutation No Sometimes No
Immutable update Yes Yes

Array Mutation Problem

// Wrong

users.push('Aman');

setUsers(users);

Correct Immutable Array Update

setUsers([...users, 'Aman']);

Why Direct Mutation Is Dangerous

  • UI may not update
  • React optimization breaks
  • Debugging becomes difficult
  • Unexpected bugs appear
  • Pure components stop working correctly

React Optimization Depends on Immutability

Features relying on immutability:

  • React.memo()
  • PureComponent
  • useMemo()
  • useCallback()
  • Redux state management

Flow of Immutable Updates

Create New State Object
↓
New Reference Generated
↓
React Detects Difference
↓
Efficient Re-Render Happens

Mutable vs Immutable Example

Mutable Immutable
array.push() [...array, item]
object.name = 'x' { ...object, name: 'x' }
array.splice() array.filter()
array.sort() [...array].sort()

Real Project Use Cases

Feature Immutable Update Example
OTP form Update OTP state
User profile Update user object
Shopping cart Add/remove items
Theme settings Update config object
Redux store Immutable reducers

Important Interview Points

  • React uses reference comparison.
  • Direct mutation can prevent re-rendering.
  • Immutable updates create new references.
  • Spread operator is commonly used.
  • Redux strongly depends on immutability.

Short Interview Answer

State updates must be immutable because React detects changes using reference comparison, and immutable updates create new references that allow React to properly re-render components.

One-Line Interview Answer

React state updates must be immutable so React can detect changes using new object references.
