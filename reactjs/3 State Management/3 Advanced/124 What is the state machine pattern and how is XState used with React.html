124. What is the State Machine Pattern and How is XState Used with React?

One-Line Answer

A State Machine Pattern models an application as a collection of states and valid transitions between those states. XState is a library that implements this pattern and integrates it with React.

First Understand The Problem

Many React developers start with multiple booleans:

Login.jsx
const [loading, setLoading] = useState(false);
const [success, setSuccess] = useState(false);
const [error, setError] = useState(false);

Initially it looks simple.

Later:

  • Loading
  • Success
  • Error
  • Retry
  • Cancel
  • Timeout

Things become messy.

Impossible states become possible.

Impossible State Example

{
  loading: true,
  success: true,
  error: true
}
Loading?
Yes

Success?
Yes

Error?
Yes
This state should never exist.

State Machine Pattern Solution

Instead of many booleans: 

{
  loading: true,
  success: false,
  error: false
}

We define valid states:

idle
↓ loading
↓ success
or
error
Only one state can be active at a time.

What Is A State Machine?

A state machine has:

  • State
  • Event
  • Transition
Current State
+ Event
↓ Next State

↓ TIMER

↓ TIMER

Only valid transitions are allowed.

Where Does XState Fit?

XState is a library that helps us build state machines in JavaScript and React.

Creating A Machine In XState

loginMachine.js
import { createMachine } from 'xstate';

export const loginMachine =
  createMachine({

    initial: 'idle',

    states: {

      idle: {
        on: {
          SUBMIT: 'loading'
        }
      },

      loading: {
        on: {
          SUCCESS: 'success',
          FAILURE: 'error'
        }
      },

      success: {},

      error: {
        on: {
          RETRY: 'loading'
        }
      }

    }

  });

How To Read This Machine?

Current State = idle
↓ SUBMIT Event
↓ loading

Current State = loading
↓ SUCCESS Event
↓ success

Current State = loading
↓ FAILURE Event
↓ error

Using XState With React

Login.jsx
import { useMachine } from '@xstate/react';
import { loginMachine } from './loginMachine';

function Login() {

  const [state, send] =
    useMachine(loginMachine);

  return (
    <>

      {state.matches('idle') &&
        <button
          onClick={() => send({ type:'SUBMIT' })}
        >
          Login
        </button>
      }

      {state.matches('loading') &&
        <h2>Loading...</h2>
      }

      {state.matches('success') &&
        <h2>Welcome</h2>
      }

      {state.matches('error') &&
        <h2>Login Failed</h2>
      }

    </>
  );

}

What Does useMachine Return?

Value Purpose
state Current machine state
send Send events to machine

XState vs Redux

Redux XState

Action → Reducer Event → Transition
Stores state Controls state flow
What data exists? What state am I in?
Redux answers: "What data do I have?"

XState answers: "What state am I currently in and where can I go next?"

Interview Answer

The State Machine Pattern models an application as a collection of states, events, and transitions. Instead of using many independent boolean flags, it defines explicit states and valid transitions between them. This prevents impossible states and makes complex workflows easier to manage. XState is a JavaScript library that implements state machines and statecharts. In React, XState is commonly used through the useMachine hook, which provides the current state and a send function for dispatching events. It is especially useful for authentication flows, multi-step forms, checkout processes, file uploads, and other complex UI workflows.
