117. What is the flux architecture and how does Redux implement it?

One-Line Answer

Flux is an architecture pattern for one-way data flow. Redux is a state management library inspired by Flux, but Redux implements it differently using dispatch(), reducers, and a single store.

First Clear The Main Confusion

Flux is not a second name of Redux.

Flux is not an extra feature inside Redux.

Redux does not internally use Flux code.
Correct understanding: Redux follows Flux architecture principles.

Simple Meaning

Term Meaning
Flux Architecture / pattern / blueprint
Redux Actual JavaScript state management library
Flux Main Rule Data should move in one direction
Redux Main Idea Implement one-way data flow with action, dispatch, reducer, and store

Best Memory Example

Flux = Blueprint

Redux = House Built Using That Blueprint Idea

A blueprint is not the house. But the house follows the blueprint.

Similarly, Flux is not Redux. Redux follows the Flux idea.

Why Flux Was Created?

Before Flux, large applications could have messy two-way state updates.

Component A
↔ Component B
↔ Component C
↔ Component D

This made state hard to debug because data could move everywhere.

Flux solved this by enforcing one-way data flow.

Original Flux Flow

User Interaction
↓ Action
↓ Dispatcher
↓ Store
↓ View

Flux Parts

Flux Part Meaning
Action Plain object that describes what happened
Dispatcher A separate central object that sends actions to stores
Store Holds state and update logic
View UI that displays state

Flux Real-World Example

User Clicks Add To Cart
↓ Action Created
↓ Dispatcher Receives Action
↓ Dispatcher Sends Action To CartStore
↓ CartStore Updates
↓ Cart View Updates

Important: Original Flux Has A Real Dispatcher

In original Flux, Dispatcher is a separate object/class.
Flux Concept
Dispatcher.dispatch({
  type: 'ADD_TO_CART',
  payload: product
});
Action
↓ Dispatcher Object
↓ CartStore
UserStore
ProductStore

Flux needed Dispatcher because Flux could have multiple stores.

Then Redux Came

Redux liked Flux's one-way data flow idea, but simplified the design.

Redux removed the separate Dispatcher object, used reducers for update logic, and used one single store.

Redux Flow

User Interaction
↓ dispatch(action)
↓ Reducer
↓ Single Store
↓ React UI

Redux Real-World Example

React Component
function ProductButton({ product }) {
  const dispatch = useDispatch();

  return <button onClick={() => dispatch(addToCart(product))}>Add To Cart</button>;
}
Action
const action = { type: 'ADD_TO_CART', payload: product };
Reducer
function cartReducer(state, action) {
  switch (action.type) {
    case 'ADD_TO_CART':
      return { ...state, items: [...state.items, action.payload] };
    default:
      return state;
  }
}
dispatch(addToCart(product))
↓ Reducer Receives Action
↓ Reducer Returns New State
↓ Redux Store Updates
↓ UI Re-renders

Your Confusion: Redux Has dispatch(), So Is That Dispatcher?

No. Redux dispatch() is not the same as Flux Dispatcher.

The name looks similar, but architecture is different.

Flux Dispatcher Redux dispatch()
Separate object/class Function on Redux store
Routes action to multiple stores Sends action to reducers
Needed because Flux can have many stores Used with one Redux store
Store contains update logic Reducer contains update logic

Why Redux Has No Separate Dispatcher?

Flux has many stores.

Action
↓ Dispatcher
↓ CartStore
UserStore
ProductStore

So Dispatcher is like traffic police.

Dispatcher
↓ Send Action To Correct Stores

Redux has one store.

Action
↓ Single Store
↓ Reducers

So Redux does not need a separate Dispatcher object.

Redux still uses the word dispatch because dispatch means "send an action", but it is not the Flux Dispatcher component.

Flux vs Redux Flow

Flux Redux
Action → Dispatcher → Store → View Action → dispatch() → Reducer → Store → View
Multiple stores possible Single store
Dispatcher routes actions dispatch sends action to reducer
Store updates itself Reducer calculates new state

Complete Redux Flow With React-Redux

User Clicks Button
↓ useDispatch()
↓ dispatch(action)
↓ Reducer Runs
↓ Redux Store Updates
↓ useSelector() Gets New State
↓ React Component Re-renders
React-Redux Example
function CartButton({ product }) {
  const dispatch = useDispatch();
  const count = useSelector(state => state.cart.items.length);

  return <button onClick={() => dispatch(addToCart(product))}>Cart ({count})</button>;
}

Final Correct Understanding

Redux does not internally use Flux. Redux is a separate library inspired by Flux architecture. It keeps Flux's one-way data flow idea, but replaces the separate Dispatcher with dispatch() + reducers and uses one single store instead of multiple stores.

Interview Answer

Flux is an architectural pattern created by Facebook for predictable one-way data flow. Original Flux uses Actions, a Dispatcher, Stores, and Views. Redux is a state management library inspired by Flux, but it simplifies Flux by removing the separate Dispatcher, using a single store, and moving update logic into pure reducers. Redux still uses actions and dispatch(), but Redux dispatch() is not the same as the original Flux Dispatcher object. Redux follows Flux principles; it does not internally use Flux code.
