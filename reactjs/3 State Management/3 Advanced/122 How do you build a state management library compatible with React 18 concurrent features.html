122. How do you build a state management library compatible with React 18 concurrent features?

One-Line Answer

To build a React 18 concurrent-compatible state library, keep state outside React, expose getSnapshot() and subscribe(), and connect components using useSyncExternalStore().

Why React 18 Needs Special Care?

React 18 can pause and resume rendering.

Render Starts
↓ React Can Pause
↓ External Store Can Change
↓ React Can Resume
If components read external state directly, different components may see different state versions. This is called tearing.

Main Rule

Do not connect external stores to React using only useEffect + setState. Use useSyncExternalStore.

What A Concurrent-Safe Store Needs

Function Purpose
getSnapshot() Returns current state
subscribe(listener) Registers listener for updates
setState() Updates store and notifies listeners
useStore() React hook that uses useSyncExternalStore

Step 1: Create External Store

createStore.js
export function createStore(initialState) {
  let state = initialState;
  const listeners = new Set();

  function getSnapshot() {
    return state;
  }

  function subscribe(listener) {
    listeners.add(listener);
    return () => listeners.delete(listener);
  }

  function setState(updater) {
    state = typeof updater === 'function' ? updater(state) : updater;
    listeners.forEach(listener => listener());
  }

  return {
    getSnapshot,
    subscribe,
    setState
  };
}

How This Store Works

state Lives Outside React
↓ getSnapshot Reads State
↓ setState Updates State
↓ subscribe Notifies React

Step 2: Create Store Instance

counterStore.js
import { createStore } from './createStore';

export const counterStore = createStore({
  count: 0
});
This is an external store because state lives outside React.

Step 3: Build React Hook Using useSyncExternalStore

useStore.js
import { useSyncExternalStore } from 'react';

export function useStore(store) {
  return useSyncExternalStore(
    store.subscribe,
    store.getSnapshot
  );
}
useStore()
↓ useSyncExternalStore()
↓ subscribe + getSnapshot
↓ Concurrent Safe Updates

Step 4: Use Store In Component

Counter.jsx
import { useStore } from './useStore';
import { counterStore } from './counterStore';

function Counter() {
  const state = useStore(counterStore);

  return (
    <>
      <h2>Count: {state.count}</h2>

      <button onClick={() => counterStore.setState(prev => ({ ...prev, count: prev.count + 1 }))}>
        Increment
      </button>
    </>
  );
}

Complete Flow

User Clicks Button
↓ counterStore.setState()
↓ Store State Changes
↓ Listeners Notify React
↓ useSyncExternalStore Gets Snapshot
↓ Component Re-renders Safely

Problem With Basic useStore

This works, but every component using the store re-renders whenever any part of the store changes.

We need selector support like Redux useSelector().

Step 5: Add Selector Support

useStoreSelector.js
import { useSyncExternalStore } from 'react';

export function useStoreSelector(store, selector) {
  return useSyncExternalStore(
    store.subscribe,
    () => selector(store.getSnapshot())
  );
}

Use Selector In Component

CounterValue.jsx
function CounterValue() {
  const count = useStoreSelector(
    counterStore,
    state => state.count
  );

  return <h2>Count: {count}</h2>;
}
Now component reads only the selected part of the store.

Real World Example: Auth Store

authStore.js
export const authStore = createStore({
  user: null,
  loading: false,
  error: null
});
authActions.js
export async function login(email, password) {
  authStore.setState(prev => ({ ...prev, loading: true, error: null }));

  try {
    const user = await api.login(email, password);
    authStore.setState(prev => ({ ...prev, user, loading: false }));
  } catch {
    authStore.setState(prev => ({ ...prev, loading: false, error: 'Login failed' }));
  }
}
LoginStatus.jsx
function LoginStatus() {
  const user = useStoreSelector(authStore, state => state.user);
  const loading = useStoreSelector(authStore, state => state.loading);
  const error = useStoreSelector(authStore, state => state.error);

  if (loading) return <h2>Loading...</h2>;
  if (error) return <h2>{error}</h2>;

  return <h2>{user ? user.name : 'Guest'}</h2>;
}

Why This Is React 18 Compatible?

Because React reads the external store through useSyncExternalStore, React can keep snapshots consistent during concurrent rendering.
External Store
↓ getSnapshot()
↓ React Reads Same Snapshot
↓ No Tearing

What Not To Do

Do not build a React 18 state library using only useEffect + subscribe + setState.
Old Unsafe Pattern
useEffect(() => {
  return store.subscribe(() => {
    setState(store.getSnapshot());
  });
}, []);
This can be unsafe with Concurrent Rendering because React may pause and resume renders.

Better Pattern

Concurrent Safe Pattern
const state = useSyncExternalStore(
  store.subscribe,
  store.getSnapshot
);

Library Design Checklist

Requirement Why Important
External store has subscribe() React needs update notification
External store has getSnapshot() React needs current value
Use useSyncExternalStore() Prevents tearing
Support selectors Avoid unnecessary re-renders
Return unsubscribe function Prevent memory leaks
Avoid mutation surprises Keep snapshots predictable

Redux Comparison

Redux store already has:

  • store.subscribe()
  • store.dispatch()

React Redux connects Redux to React using useSyncExternalStore internally.

Redux Store

↓ useSyncExternalStore
↓ useSelector
↓ Your Component

Interview Answer

To build a state management library compatible with React 18 concurrent features, the store should expose a subscribe method for listening to changes and a getSnapshot method for reading the current state. React components should subscribe to the store using useSyncExternalStore instead of manually using useEffect and setState. This ensures React reads consistent snapshots during concurrent rendering and prevents tearing. A good library also supports selectors to avoid unnecessary re-renders and returns unsubscribe functions to clean up subscriptions.
