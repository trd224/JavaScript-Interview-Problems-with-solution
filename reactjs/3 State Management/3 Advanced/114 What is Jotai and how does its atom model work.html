114. What is Jotai and how does its atom model work?

One-Line Answer

Jotai is a lightweight React state management library where state is stored in small independent units called atoms instead of one large global store.

Why Jotai Was Created?

Redux stores everything in one big store.

Redux Store
const store = {
  user: {},
  cart: [],
  theme: 'dark',
  notifications: []
};
One Big Store
↓ Reducers
↓ Actions
↓ Dispatch

Jotai takes a different approach.

❌

✅

What Is An Atom?

An atom is a single piece of global state.
countAtom.js
import { atom } from 'jotai';

export const countAtom = atom(0);
countAtom
↓ Stores
0

Think of an atom as a globally shared useState.

Using An Atom

Counter.jsx
import { useAtom } from 'jotai';
import { countAtom } from './countAtom';

function Counter() {

  const [count, setCount] =
    useAtom(countAtom);

  return (
    <>
      <h2>{count}</h2>

      <button
        onClick={() => setCount(count + 1)}
      >
        Increment
      </button>
    </>
  );

}

How The Atom Model Works

Imagine an application has:

Independent State Pieces

Each atom is independent.

Components subscribe only to the atoms they use.

themeAtom Changes
↓ Only Theme Components Re-render
Unlike Context, unrelated components do not re-render.

Derived Atoms

Atoms can depend on other atoms.

cartTotalAtom.js
import { atom } from 'jotai';
import { cartAtom } from './cartAtom';

export const cartTotalAtom =
  atom(get =>
    get(cartAtom).reduce(
      (sum, item) =>
        sum + item.price,
      0
    )
  );
cartAtom
↓ Derived Atom
↓ Total Price

This is similar to Redux selectors.

Writable Derived Atom

countAtom.js
export const countAtom = atom(0);

export const incrementAtom = atom(
  null,
  (get, set) => {
    set(
      countAtom,
      get(countAtom) + 1
    );
  }
);

↓ Inside Atom
↓ Reusable Everywhere

How Jotai Updates Components

Atom Created
↓ Component Uses Atom
↓ Atom Changes
↓ Only Subscribers Re-render
This is the biggest advantage of the atom model.

Jotai solves Redux complexity by allowing state to be split into small independent atoms that can be read and updated directly using React hooks.

Interview Answer

Jotai is a lightweight React state management library based on atoms. An atom represents a single piece of global state that can be shared across components. Components subscribe only to the atoms they use, so when an atom changes, only those components re-render. Jotai also supports derived atoms and writable atoms, providing powerful state management with very little boilerplate.
