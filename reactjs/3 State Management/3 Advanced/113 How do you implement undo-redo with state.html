113. How do you implement undo-redo with state?

One-Line Answer

Undo/redo is usually implemented by storing state history in three parts: past, present, and future.

Main Idea

Past States
↓ Current State
↓ Future States

When user changes data, current state moves into past.

When user clicks undo, current state moves into future and previous state becomes current.

When user clicks redo, current state moves into past and next future state becomes current.

State Shape

Undo Redo State
const history = {
  past: [],
  present: '',
  future: []
};
Part Meaning
past Old states for undo
present Current state
future Undone states for redo

Example Flow

Type A
↓ present = A

Type AB
↓ past = [A]
present = AB

Type ABC
↓ past = [A, AB]
present = ABC

When Undo Clicked

past = [A, AB]
present = ABC
future = []

Undo
↓ past = [A]
present = AB
future = [ABC]

When Redo Clicked

past = [A]
present = AB
future = [ABC]

Redo
↓ past = [A, AB]
present = ABC
future = []

Full React Example

App.jsx
import { useState } from 'react';

function App() {
  const [history, setHistory] = useState({ past: [], present: '', future: [] });

  function updateValue(newValue) {
    setHistory(prev => ({
      past: [...prev.past, prev.present],
      present: newValue,
      future: []
    }));
  }

  function undo() {
    setHistory(prev => {
      if (prev.past.length === 0) return prev;

      const previous = prev.past[prev.past.length - 1];
      const newPast = prev.past.slice(0, -1);

      return {
        past: newPast,
        present: previous,
        future: [prev.present, ...prev.future]
      };
    });
  }

  function redo() {
    setHistory(prev => {
      if (prev.future.length === 0) return prev;

      const next = prev.future[0];
      const newFuture = prev.future.slice(1);

      return {
        past: [...prev.past, prev.present],
        present: next,
        future: newFuture
      };
    });
  }

  return (
    <div>
      <input value={history.present} onChange={e => updateValue(e.target.value)} />

      <button onClick={undo} disabled={history.past.length === 0}>Undo</button>

      <button onClick={redo} disabled={history.future.length === 0}>Redo</button>

      <h3>Current Value: {history.present}</h3>
    </div>
  );
}

export default App;

Why Future Is Cleared On New Typing?

A → AB → ABC
↓ Undo To AB
↓ Type ABD
↓ ABC Redo Is No Longer Valid
After a new change, redo history should be removed because the user has created a new branch of state.

Interview Answer

Undo/redo can be implemented by storing state history as past, present, and future. On every new update, the current present value is pushed into past and future is cleared. On undo, the last past value becomes present and the old present moves into future. On redo, the first future value becomes present and the old present moves back into past.

Easy Way To Remember

New Change
↓ present → past

Undo
↓ past → present → future

Redo
↓ future → present → past
