123. What is XState and When Would You Use a State Machine?

One-Line Answer

XState is a library for building applications using finite state machines and statecharts. It helps manage complex UI flows by allowing only valid state transitions.

First Understand The Problem

Many React applications start simple:

Simple State
const [loading, setLoading] = useState(false);
const [error, setError] = useState(null);
const [data, setData] = useState(null);

Looks fine initially.

But after adding:

  • Loading
  • Success
  • Error
  • Retry
  • Timeout
  • Cancel
  • Authentication
  • Multi-step forms

State becomes difficult to manage.

Impossible states can appear.

Example Of Impossible State

{
  loading: true,
  error: "Server Error",
  data: {
    name: "Rizwan"
  }
}

Questions:

Loading?
↓ Yes

Error?
↓ Yes

Success Data?
↓ Yes

All three are active simultaneously.

This state should never exist.

State Machine Solution

Instead of many booleans: 

{
  loading: true,
  error: null,
  success: false
}

We define valid states:

idle
↓ loading
↓ success
or
error
Only one state can be active at a time.

Real World Example - Login Flow

Without state machine: 

{
  loading: true,
  error: true,
  authenticated: true
}
Impossible combination.

With state machine:

idle
↓ submitting
↓ success
or
failure

Only one state exists at a time.

What Is A Finite State Machine?

A finite state machine (FSM) has:

  • States
  • Events
  • Transitions
Current State
+ Event
↓ Next State

Traffic Signal Example

Red
↓ TIMER
Green
↓ TIMER
Yellow
↓ TIMER
Red
You cannot jump directly from Red to Yellow.

Only valid transitions are allowed.

Login Example As State Machine

idle
↓ SUBMIT
loading
↓ SUCCESS
success

idle
↓ SUBMIT
loading
↓ FAILURE
error

How XState Represents This

loginMachine.js
import { createMachine } from 'xstate';

const loginMachine = createMachine({

  initial: 'idle',

  states: {

    idle: {
      on: {
        SUBMIT: 'loading'
      }
    },

    loading: {
      on: {
        SUCCESS: 'success',
        FAILURE: 'error'
      }
    },

    success: {},

    error: {
      on: {
        RETRY: 'loading'
      }
    }

  }

});

How To Read It?

Current State = idle
↓ SUBMIT Event
↓ loading

Current State = loading
↓ SUCCESS Event
↓ success

Current State = loading
↓ FAILURE Event
↓ error

Using XState In React

Login.jsx
import { useMachine } from '@xstate/react';

function Login() {

  const [state, send] =
    useMachine(loginMachine);

  return (
    <>

      {state.matches('idle') &&
        <button onClick={() => send({ type:'SUBMIT' })}>
          Login
        </button>
      }

      {state.matches('loading') &&
        <h2>Loading...</h2>
      }

      {state.matches('success') &&
        <h2>Welcome</h2>
      }

      {state.matches('error') &&
        <h2>Login Failed</h2>
      }

    </>
  );

}

Interview Answer

XState is a state management library based on finite state machines and statecharts. Instead of managing many independent boolean flags, it models an application as a set of states and valid transitions between those states. This prevents impossible states and makes complex workflows easier to understand. XState is commonly used for authentication flows, multi-step forms, checkout processes, wizards, and other applications where state transitions are important. Unlike Redux, which focuses on storing data, XState focuses on modeling how state changes over time.
