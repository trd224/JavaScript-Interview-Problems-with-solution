116. How does MobX differ from Redux in its reactive approach?

One-Line Answer

Redux uses an explicit action → reducer → store update flow, while MobX automatically tracks dependencies and updates components reactively whenever observable data changes.

The Core Difference

Redux
↓ Tell State What To Do

MobX
↓ State Changes
↓ UI Automatically Reacts

Redux Approach

Redux follows a strict unidirectional data flow.

Redux Update
dispatch(addToCart(product));
dispatch()
↓ Action
↓ Reducer
↓ Store Updated
↓ Component Re-renders

Redux requires:

  • Store
  • Actions
  • Reducers
  • Dispatch

MobX Approach

MobX uses observable state.

CartStore.js
import { makeAutoObservable } from 'mobx';

class CartStore {

  items = [];

  constructor() {
    makeAutoObservable(this);
  }

  addToCart(product) {
    this.items.push(product);
  }

}

export default new CartStore();

Update state directly: 

cartStore.addToCart(product);
Observable State
↓ State Changes
↓ MobX Detects Change
↓ UI Updates Automatically

What Problem Does MobX Solve?

Redux
↓ Action
↓ Reducer
↓ Store
↓ Component

MobX
↓ Observable State
↓ Component
MobX removes much of Redux's boilerplate by making state observable and automatically updating components that depend on that state.

Interview Answer

Redux uses an explicit action-based architecture where actions are dispatched, reducers calculate new state, and the store updates predictably. MobX uses a reactive model where state is observable and components automatically re-render when the observable data they use changes. Redux emphasizes predictability and explicit data flow, while MobX emphasizes simplicity and automatic reactivity.
