115. What is Recoil and what problem does it solve over Redux?

One-Line Answer

Recoil is a React state management library from Meta that uses atoms and selectors to manage global state with less boilerplate and more granular re-rendering than Redux.

First Understand The Redux Problem

Suppose an e-commerce application has:

User State
↓ Cart State
↓ Theme State
↓ Notification State

Redux stores everything inside one central store.

Redux Store
const store = {
  user: {},
  cart: [],
  theme: 'light',
  notifications: []
};
One Big Store
↓ Actions
↓ Reducers
↓ Dispatch
↓ Update State
Redux is powerful, but even simple state often needs actions, reducers, dispatch calls, and store configuration.

How Recoil Solves This

Instead of one big store, Recoil splits state into many small atoms.
Redux
↓ One Large Store

Recoil
↓ Many Small Atoms

What Is An Atom?

An atom is a single piece of global state.

themeAtom.js
import { atom } from 'recoil';

export const themeAtom = atom({
  key: 'themeAtom',
  default: 'light'
});
themeAtom
↓ Stores
light

Think of an atom as a global useState.

Using An Atom

ThemeSwitcher.jsx
import { useRecoilState } from 'recoil';
import { themeAtom } from './themeAtom';

function ThemeSwitcher() {

  const [theme, setTheme] =
    useRecoilState(themeAtom);

  return (
    <button
      onClick={() => setTheme('dark')}
    >
      Dark Mode
    </button>
  );

}
useRecoilState()
↓ Read Atom
↓ Update Atom

What Are Selectors?

Selectors are derived state.

cartTotalSelector.js
import { selector } from 'recoil';
import { cartAtom } from './cartAtom';

export const cartTotalSelector =
  selector({

    key: 'cartTotalSelector',

    get: ({ get }) =>
      get(cartAtom).reduce(
        (sum, item) =>
          sum + item.price,
        0
      )

  });
cartAtom
↓ Selector
↓ Calculated Value

Similar to Redux selectors.

Why Recoil Feels Simpler Than Redux

Redux Update
dispatch(addToCart(product));
Requires:
  • Action
  • Reducer
  • Dispatch
  • Store Setup
Recoil Update
setCart([...cart, product]);
Recoil feels much closer to useState.

How Recoil Re-Renders Components

Atom Created
↓ Component Subscribes
↓ Atom Changes
↓ Only Subscribers Re-render
Unlike Context, components that don't use that atom do not re-render.

What Problem Does Recoil Solve Over Redux?

Redux
↓ Store
↓ Action
↓ Reducer
↓ Dispatch

Recoil
↓ Atom
↓ Hook
↓ Update State
Recoil solves Redux's boilerplate and complexity by replacing actions, reducers, and dispatch calls with simple atom-based state management while still providing global shared state and efficient re-rendering.

Interview Answer

Recoil is a React state management library that uses atoms and selectors instead of a centralized Redux store. Atoms represent individual pieces of global state, while selectors derive computed state from atoms. Recoil solves Redux's boilerplate by eliminating actions, reducers, and dispatch calls, making global state management feel much closer to React's useState while still supporting shared state and efficient updates.
