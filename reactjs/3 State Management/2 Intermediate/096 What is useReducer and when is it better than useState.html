096. What is useReducer and when is it better than useState?

One-Line Definition

useReducer is a React Hook that manages state using a reducer function and actions, making it useful for complex state logic and multiple related state values.

What is useReducer?

useReducer is an alternative to useState.

Instead of updating state directly, we dispatch actions and let a reducer function decide how the state should change.

dispatch(action)
↓
Reducer Receives Action
↓
Returns New State
↓
React Updates State
↓
Component Re-renders

useReducer Syntax

const [state, dispatch] =
  useReducer(reducer, initialState);

Parameters

  • reducer → Function that updates state.
  • initialState → Initial value of state.

Returns

  • state → Current state.
  • dispatch → Function used to send actions.

Simple Counter Example

import { useReducer } from 'react';

const initialState = { count: 0 };

function reducer(state, action) {
  switch (action.type) {
    case 'INCREMENT':
      return { count: state.count + 1 };

    case 'DECREMENT':
      return { count: state.count - 1 };

    default:
      return state;
  }
}

function Counter() {
  const [state, dispatch] = useReducer(reducer, initialState);

  return (
    <>
      <h2>{state.count}</h2>

      <button onClick={() => dispatch({ type: 'INCREMENT' })}>
        +
      </button>

      <button onClick={() => dispatch({ type: 'DECREMENT' })}>
        -
      </button>
    </>
  );
}

How It Works Internally

Suppose the user clicks the Increment button. 

dispatch({ type: 'INCREMENT' });

React automatically calls: 

reducer(
  { count: 0 },
  { type: 'INCREMENT' }
);

Reducer returns: 

{ count: 1 }

React stores the new state and re-renders the component.

Complete Real-World Example

This example shows why useReducer becomes useful when a form contains multiple related fields. 

import { useReducer } from 'react';

const initialState = {
  firstName: '',
  lastName: '',
  email: '',
  mobile: '',
  loading: false,
  error: ''
};

function reducer(state, action) {
  switch (action.type) {

    case 'UPDATE_FIELD':
      return {
        ...state,
        [action.field]: action.value,
        error: ''
      };

    case 'SUBMIT_START':
      return {
        ...state,
        loading: true,
        error: ''
      };

    case 'SUBMIT_SUCCESS':
      return {
        ...state,
        loading: false
      };

    case 'SET_ERROR':
      return {
        ...state,
        loading: false,
        error: action.message
      };

    case 'RESET_FORM':
      return initialState;

    default:
      return state;

  }
}

function UserForm() {

  const [state, dispatch] = useReducer(reducer, initialState);

  const handleChange = event => {
    dispatch({
      type: 'UPDATE_FIELD',
      field: event.target.name,
      value: event.target.value
    });
  };

  const handleSubmit = event => {

    event.preventDefault();

    if (!state.firstName || !state.email || !state.mobile) {
      dispatch({
        type: 'SET_ERROR',
        message: 'Please fill required fields'
      });
      return;
    }

    dispatch({ type: 'SUBMIT_START' });

    setTimeout(() => {
      console.log('Form Submitted', state);
      dispatch({ type: 'SUBMIT_SUCCESS' });
    }, 1000);

  };

  return (
    <form onSubmit={handleSubmit}>

      <input
        name="firstName"
        placeholder="First Name"
        value={state.firstName}
        onChange={handleChange}
      />

      <input
        name="lastName"
        placeholder="Last Name"
        value={state.lastName}
        onChange={handleChange}
      />

      <input
        name="email"
        placeholder="Email"
        value={state.email}
        onChange={handleChange}
      />

      <input
        name="mobile"
        placeholder="Mobile"
        value={state.mobile}
        onChange={handleChange}
      />

      <button
        type="submit"
        disabled={state.loading}
      >
        {state.loading ? 'Submitting...' : 'Submit'}
      </button>

      <button
        type="button"
        disabled={state.loading}
        onClick={() => dispatch({ type: 'RESET_FORM' })}
      >
        Reset
      </button>

      {state.error && <p>{state.error}</p>}

    </form>
  );

}

Real Project Example

In our Aadhaar Authentication application, useState is perfect for simple values such as OTP input or popup visibility. However, useReducer is better for managing Aadhaar number, OTP, consent, loading, authentication type, API response, and error messages together in a single state object.

Important Interview Points

  • useReducer is an alternative to useState.
  • State updates happen through actions.
  • dispatch sends actions to the reducer.
  • Reducer decides how state changes.
  • Best for complex and related state values.
  • Similar to Redux-style state management.

Short Interview Answer

useReducer is a React Hook that manages state using a reducer function and actions. It is better than useState when state logic becomes complex, contains multiple related values, or requires predictable state transitions.

One-Line Interview Answer

useReducer manages state through actions and reducers and is best suited for complex state logic.
