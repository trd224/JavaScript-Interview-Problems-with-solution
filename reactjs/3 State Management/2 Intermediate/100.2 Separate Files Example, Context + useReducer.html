Separate Files Example: Context + useReducer With Backend Login

Why Split Into Multiple Files?

In large React applications, keeping Context, Reducer, Provider, Custom Hooks, and API services in separate files makes the code easier to maintain, test, and scale.

In real projects, login credentials should not be verified directly in the React component. The component sends username and password to the backend, and the backend verifies them.

Real Login Flow

User enters username and password
↓
User clicks Login
↓
React dispatches LOGIN_START
↓
Frontend sends credentials to backend API
↓
Backend verifies credentials
↓
Backend sends success or error response
↓
React dispatches LOGIN_SUCCESS or LOGIN_ERROR
↓
Global auth state updates
↓
Dashboard appears or error message shows

1. Create Context

context/AuthContext.jsx

import { createContext } from 'react';

export const AuthContext = createContext();
This file only creates and exports AuthContext.

2. Create Reducer

reducers/authReducer.js

export const initialState = {
  user: null,
  loading: false,
  error: ''
};

export function authReducer(state, action) {
  switch (action.type) {
    case 'LOGIN_START':
      return {
        ...state,
        loading: true,
        error: ''
      };

    case 'LOGIN_SUCCESS':
      return {
        ...state,
        loading: false,
        user: action.payload,
        error: ''
      };

    case 'LOGIN_ERROR':
      return {
        ...state,
        loading: false,
        error: action.payload
      };

    case 'LOGOUT':
      return {
        ...state,
        user: null,
        error: ''
      };

    default:
      return state;
  }
}
The reducer controls authentication state like user, loading, and error.

3. Create Provider

providers/AuthProvider.jsx

import { useReducer } from 'react';
import { AuthContext } from '../context/AuthContext';
import { authReducer, initialState } from '../reducers/authReducer';

export function AuthProvider({ children }) {
  const [state, dispatch] = useReducer(authReducer, initialState);

  return (
    <AuthContext.Provider value={{ state, dispatch }}>
      {children}
    </AuthContext.Provider>
  );
}
AuthProvider creates state using useReducer and shares state + dispatch globally.

4. Create Custom Hook

hooks/useAuth.js

import { useContext } from 'react';
import { AuthContext } from '../context/AuthContext';

export function useAuth() {
  return useContext(AuthContext);
}
useAuth hides useContext(AuthContext), so components can access auth state easily.

5. Create Backend API Service

services/authService.js

export async function loginUser(credentials) {
  const response = await fetch('https://your-api.com/api/login', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify(credentials)
  });

  const data = await response.json();

  if (!response.ok) {
    throw new Error(data.message || 'Login failed');
  }

  return data;
}
This service sends username and password to the backend. The backend verifies credentials and returns success or error.

Request Sent To Backend

}

    "name": "Rizwan",
    "role": "Admin"
  },
  "token": "jwt_token_here"
}

Backend Error Response

{
  "success": false,
  "message": "Invalid username or password"
}

6. Login Component

components/Login.jsx

import { useState } from 'react';
import { useAuth } from '../hooks/useAuth';
import { loginUser } from '../services/authService';

function Login() {
  const { state, dispatch } = useAuth();

  const [username, setUsername] = useState('');
  const [password, setPassword] = useState('');

  async function handleLogin(e) {
    e.preventDefault();

    dispatch({
      type: 'LOGIN_START'
    });

    try {
      const data = await loginUser({ username, password });

      dispatch({
        type: 'LOGIN_SUCCESS',
        payload: data.user
      });

      localStorage.setItem('token', data.token);
    } catch (error) {
      dispatch({
        type: 'LOGIN_ERROR',
        payload: error.message
      });
    }
  }

  return (
    <form onSubmit={handleLogin}>
      <input
        value={username}
        onChange={(e) => setUsername(e.target.value)}
        placeholder="Username"
      />

      <input
        type="password"
        value={password}
        onChange={(e) => setPassword(e.target.value)}
        placeholder="Password"
      />

      {state.error && <p>{state.error}</p>}

      <button disabled={state.loading}>
        {state.loading ? 'Logging In...' : 'Login'}
      </button>
    </form>
  );
}

export default Login;
Here username and password are local state because only Login component needs them. loading, error, and user are global auth state because other components may need them.

7. Dashboard Component

components/Dashboard.jsx

import { useAuth } from '../hooks/useAuth';

function Dashboard() {
  const { state, dispatch } = useAuth();

  function handleLogout() {
    localStorage.removeItem('token');

    dispatch({
      type: 'LOGOUT'
    });
  }

  return (
    <div>
      <h2>Dashboard</h2>

      <p>Welcome {state.user.name}</p>

      <p>Role: {state.user.role}</p>

      <button onClick={handleLogout}>
        Logout
      </button>
    </div>
  );
}

export default Dashboard;

8. App Component

App.jsx

import Login from './components/Login';
import Dashboard from './components/Dashboard';
import { useAuth } from './hooks/useAuth';

function App() {
  const { state } = useAuth();

  return (
    <>
      {state.user ? <Dashboard /> : <Login />}
    </>
  );
}

export default App;
App checks global auth state. If user exists, Dashboard is shown. If user is null, Login is shown.

10. Where Should We Use useEffect In Authentication?

Many beginners think login API calls should be placed inside useEffect.

Login is a user action, so the login API should be called inside handleLogin(), not inside useEffect.

Wrong Example

useEffect(() => {
  loginUser({
    username,
    password
  });
}, []);

This runs immediately when the page loads, before the user enters credentials.

Another Wrong Example

useEffect(() => {
  loginUser({
    username,
    password
  });
}, [username, password]);

Now every keystroke triggers an API call.

Type "a"
↓
API Call

Type "ad"
↓
API Call

Type "admin"
↓
API Call
This is not how login forms should work.

Correct Login Approach

async function handleLogin(e) {
  e.preventDefault();

  const data = await loginUser({
    username,
    password
  });
}

The API call happens only when the user clicks Login.

Real Place To Use useEffect

The best authentication use case for useEffect is checking whether the user is already logged in when the application starts.

services/authService.js

export async function getCurrentUser(token) {

  const response = await fetch(
    'https://your-api.com/api/me',
    {
      headers: {
        Authorization: `Bearer ${token}`
      }
    }
  );

  return response.json();

}

App.jsx

import { useEffect } from 'react';
import { useAuth } from './hooks/useAuth';
import { getCurrentUser } from './services/authService';

function App() {

  const { state, dispatch } = useAuth();

Other Real Authentication useEffect Examples

Feature Use useEffect? Reason
Login Button Click ❌ No User Action
Logout Button Click ❌ No User Action
Submit Form ❌ No User Action
Check Existing Session ✅ Yes Automatic On App Start
Load User Profile ✅ Yes Automatic Data Fetch
Auto Logout ✅ Yes Automatic Timer
Refresh Notifications ✅ Yes Automatic Background Task
WebSocket Connection ✅ Yes Automatic Connection Setup

Easy Rule

If something should happen because the user clicked a button, use an event handler such as onClick or onSubmit.

If something should happen automatically when a component loads or when data changes, use useEffect.

Important Interview Points

  • React should not verify username and password directly.
  • React sends credentials to the backend API.
  • The backend verifies credentials and returns user data or error.
  • useReducer manages loading, user, and error state.
  • Context shares auth state globally.
  • Custom hook makes Context usage cleaner.
  • API logic should be kept in a separate service file.

One-Line Interview Answer

In a real login flow, React collects credentials, sends them to the backend API, stores the backend response in global auth state using useReducer, and shares that state across the app using Context.
