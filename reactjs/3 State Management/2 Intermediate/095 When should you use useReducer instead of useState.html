095. When should you use useReducer instead of useState?

One-Line Definition

useReducer should be used instead of useState when state has multiple related values, complex update logic, or different actions that update the same state.

Simple Rule

State Type Use
Simple state useState
Complex or related state useReducer

When useState Is Enough

For simple values, useState is better and easier. 

function Counter() {
  const [count, setCount] = useState(0);

  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}

Good useState Examples

  • Counter value
  • Input value
  • Checkbox state
  • Modal open/close
  • Dropdown visibility

Why useReducer Is Needed?

When we have many related states, multiple useState hooks can become messy. 

const [firstName, setFirstName] = useState('');
const [lastName, setLastName] = useState('');
const [email, setEmail] = useState('');
const [mobile, setMobile] = useState('');
const [error, setError] = useState('');
Instead of many separate states and setters, useReducer keeps related state in one object and updates it through actions.

How useReducer Works

dispatch(action)
↓
Reducer Receives Current State + Action
↓
Reducer Returns New State
↓
React Updates State
↓
Component Re-renders

Important Mental Model

Term Meaning
state Current data
dispatch Sends an action
action Describes what happened
reducer Decides how state should change

Simple Working Example

const initialState = { count: 0 };

function reducer(state, action) {
  switch (action.type) {
    case 'INCREMENT':
      return { count: state.count + 1 };
    default:
      return state;
  }
}

function Counter() {
  const [state, dispatch] = useReducer(reducer, initialState);

  return <button onClick={() => dispatch({ type: 'INCREMENT' })}>{state.count}</button>;
}

here { type: 'INCREMENT' } is a action

What Happens Internally?

Initial State: { count: 0 }
↓
User Clicks Button
↓
dispatch({ type: 'INCREMENT' })
↓
reducer({ count: 0 }, { type: 'INCREMENT' })
↓
Reducer Returns { count: 1 }
↓
React Re-renders UI

Complete Multi-Input Form Example

This is a common real-world example where useReducer is better than many useState hooks. 

import { useReducer } from 'react';

const initialState = {
  firstName: '',
  lastName: '',
  email: '',
  mobile: '',
  loading: false,
  error: ''
};

function reducer(state, action) {
  switch (action.type) {
    case 'UPDATE_FIELD':
      return { ...state, [action.field]: action.value, error: '' };

    case 'SUBMIT_START':
      return { ...state, loading: true, error: '' };

    case 'SUBMIT_SUCCESS':
      return { ...state, loading: false };

    case 'SET_ERROR':
      return { ...state, loading: false, error: action.message };

    case 'RESET_FORM':
      return initialState;

    default:
      return state;
  }
}

function UserForm() {
  const [state, dispatch] = useReducer(reducer, initialState);

  const handleChange = event => {
    dispatch({ type: 'UPDATE_FIELD', field: event.target.name, value: event.target.value });
  };

  const handleSubmit = event => {
    event.preventDefault();

    if (!state.firstName || !state.email || !state.mobile) {
      dispatch({ type: 'SET_ERROR', message: 'Please fill required fields' });
      return;
    }

    dispatch({ type: 'SUBMIT_START' });

    setTimeout(() => {
      console.log('Form Submitted:', state);
      dispatch({ type: 'SUBMIT_SUCCESS' });
    }, 1000);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input name="firstName" placeholder="First Name" value={state.firstName} onChange={handleChange} />
      <input name="lastName" placeholder="Last Name" value={state.lastName} onChange={handleChange} />
      <input name="email" placeholder="Email" value={state.email} onChange={handleChange} />
      <input name="mobile" placeholder="Mobile" value={state.mobile} onChange={handleChange} />

      <button type="submit" disabled={state.loading}>
        {state.loading ? 'Submitting...' : 'Submit'}
      </button>

      <button type="button" disabled={state.loading} onClick={() => dispatch({ type: 'RESET_FORM' })}>
        Reset
      </button>

      {state.error && <p>{state.error}</p>}
    </form>
  );
}

export default UserForm;

How Multi-Input Example Works

When the user types in the first name input: 

dispatch({ type: 'UPDATE_FIELD', field: 'firstName', value: 'Rizwan' });

Reducer receives the action and updates only that field: 

return { ...state, [action.field]: action.value, error: '' };

So the new state becomes: 

{
  firstName: 'Rizwan',
  lastName: '',
  email: '',
  mobile: '',
  error: ''
}

Why useReducer Is Better in This Form?

Multiple useState useReducer
Many state variables One state object
Many setter functions One dispatch function
Update logic spread everywhere Update logic inside reducer
Harder to manage large forms Easier to manage large forms

When Should We Use useReducer?

  • When state has many related fields.
  • When one action updates multiple values.
  • When state update logic is complex.
  • When form logic has validation, reset, error, and submit states.
  • When we want predictable state transitions.

useState vs useReducer

Feature useState useReducer
Best for Simple state Complex state
Update method Setter function dispatch action
Logic location Inside component Inside reducer function
Multiple related values Harder Easier
Learning Easy Medium

Real Project Example

In our Aadhaar Authentication application, useState is good for simple values like OTP input, captcha value, or popup visibility. But useReducer is better for a complete authentication form where Aadhaar number, OTP, consent, loading, error, authentication type, and API response are managed together.

Important Interview Points

  • useState is best for simple independent state.
  • useReducer is best for complex related state.
  • dispatch sends an action.
  • Reducer decides how state changes.
  • Reducer returns the new state.
  • React stores the new state and re-renders the component.

Short Interview Answer

We should use useReducer instead of useState when state logic becomes complex, when multiple related values are updated together, or when different actions affect the same state. useReducer keeps state updates centralized and predictable.

One-Line Interview Answer

Use useReducer for complex, related, or action-based state updates, and use useState for simple state.
