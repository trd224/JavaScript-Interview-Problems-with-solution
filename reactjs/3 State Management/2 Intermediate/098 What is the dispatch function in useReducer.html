098. What is the dispatch function in useReducer?

One-Line Definition

dispatch is the function returned by useReducer that sends an action to the reducer so the state can be updated.

Simple Explanation

In useState, we update state using a setter function. 

const [count, setCount] = useState(0);

setCount(count + 1);

In useReducer, we do not update state directly.

Instead, we send an action using dispatch. 

const [state, dispatch] = useReducer(reducer, initialState);

dispatch({ type: 'INCREMENT' });
Think of dispatch as a messenger. It sends an action to the reducer and the reducer decides how the state should change.

How dispatch Works

User Action
↓
dispatch(action)
↓
Reducer Receives Action
↓
Returns New State
↓
React Updates State
↓
Component Re-renders

Basic Syntax

dispatch({
  type: 'ACTION_NAME'
});

The action object usually contains a type property.

Simple Counter Example

const initialState = {
  count: 0
};

function reducer(state, action) {

  switch (action.type) {

    case 'INCREMENT':
      return {
        count: state.count + 1
      };

    default:
      return state;

  }

}

function Counter() {

  const [state, dispatch] =
    useReducer(reducer, initialState);

  return (
    <button
      onClick={() => dispatch({ type: 'INCREMENT' })}
    >
      {state.count}
    </button>
  );

}

What Happens Internally?

Suppose the current state is: 

{ count: 0 }

User clicks the button: 

dispatch({ type: 'INCREMENT' });

React automatically calls: 

reducer(
  { count: 0 },
  { type: 'INCREMENT' }
);

Reducer returns: 

{ count: 1 }

React stores the new state and re-renders the component.

dispatch Can Send Additional Data

Actions can contain extra information. 

dispatch({
  type: 'SET_USERNAME',
  value: 'Rizwan'
});

Reducer receives: 

action.type
action.value

And updates the state accordingly.

Reducer Example with Payload

function reducer(state, action) {

  switch (action.type) {

    case 'SET_USERNAME':
      return {
        ...state,
        username: action.value
      };

    default:
      return state;

  }

}

Real Form Example

const handleChange = event => {

  dispatch({
    type: 'UPDATE_FIELD',
    field: event.target.name,
    value: event.target.value
  });

};

If the user types: 

Rizwan

inside: 

<input name="firstName" />

dispatch sends: 

{
  type: 'UPDATE_FIELD',

  value: 'Rizwan'

Common dispatch Actions

dispatch({ type: 'INCREMENT' });

dispatch({ type: 'DECREMENT' });

dispatch({ type: 'RESET' });

dispatch({
  type: 'SET_ERROR',
  message: 'Invalid Login'
});

dispatch({
  type: 'UPDATE_FIELD',
  field: 'email',
  value: 'abc@gmail.com'
});

Real Project Example

In our Aadhaar Authentication application, dispatch can send actions such as UPDATE_OTP, SET_ERROR, START_LOADING, AUTH_SUCCESS, RESET_FORM, or UPDATE_AADHAAR. The reducer then decides how the authentication state should be updated.

Important Interview Points

  • dispatch is returned by useReducer.
  • dispatch sends actions to the reducer.
  • dispatch does not update state directly.
  • Reducer receives the action and returns the new state.
  • Actions usually contain a type property.
  • Actions may also contain additional data.

Short Interview Answer

dispatch is the function returned by useReducer that sends an action object to the reducer. The reducer processes the action and returns the next state, which React then stores and renders.

One-Line Interview Answer

dispatch sends an action to the reducer so React can calculate and update the next state.
