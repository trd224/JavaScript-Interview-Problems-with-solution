108. What are useSelector and useDispatch Hooks in React Redux?

One-Line Definition

useSelector is used to read data from the Redux store, while useDispatch is used to send actions to the Redux store.

The Two Most Important Redux Hooks

Almost every React Redux component uses these two hooks:

  • useSelector() → Read data from Redux Store
  • useDispatch() → Update data in Redux Store
Read State
↓
useSelector()

Update State
↓
useDispatch()

First Understand the Redux Store

Suppose our Redux store looks like this: 

{
  user: {
    name: 'Rizwan',
    email: 'rizwan@gmail.com'
  },

  cart: {
    items: [
      { id: 1, name: 'Laptop' },
      { id: 2, name: 'Mouse' }
    ]
  }
}

Components need two things:

  • Read data from this store
  • Update data inside this store

What is useSelector?

useSelector is used to read data from the Redux store.

Example

import { useSelector } from 'react-redux';

function Navbar() {

  const items = useSelector(
    state => state.cart.items
  );

  return (
    <h3>Cart Items: {items.length}</h3>
  );
}

What Happens?

Redux Store
      ↓
state.cart.items
      ↓
useSelector()
      ↓
Navbar Component

The component automatically receives the latest Redux state.

Reading Different Parts of State

Read User Name

const name = useSelector(
  state => state.user.name
);

Read User Email

const email = useSelector(
  state => state.user.email
);

Read Cart Items

const items = useSelector(
  state => state.cart.items
);

Read Entire Cart Slice

const cart = useSelector(
  state => state.cart
);
useSelector only reads state. It cannot update state.

What is useDispatch?

useDispatch is used to send actions to Redux so that state can be updated.

Example

import { useDispatch } from 'react-redux';
import { addToCart } from './cartSlice';

function ProductCard({ product }) {

  const dispatch = useDispatch();

  return (
    <button
      onClick={() => dispatch(addToCart(product))}
    >
      Add To Cart
    </button>
  );
}

What Happens?

Button Click
      ↓
dispatch(addToCart(product))
      ↓
Action Created
      ↓
Reducer Runs
      ↓
Redux Store Updated

Understanding dispatch()

This line: 

dispatch(addToCart(product));

actually performs two steps:

Step 1

addToCart(product)

Returns an action object: 

{
  type: 'cart/addToCart',
  payload: product
}

Step 2

dispatch(action)

Sends the action to Redux.

dispatch does not update state directly. It sends an action to Redux reducers.

Complete Example

cartSlice.js

import { createSlice } from '@reduxjs/toolkit';

const cartSlice = createSlice({

  name: 'cart',

  initialState: {
    items: []
  },

  reducers: {

    addToCart(state, action) {
      state.items.push(action.payload);
    }

  }

});

export const { addToCart } = cartSlice.actions;

export default cartSlice.reducer;

ProductCard.jsx

import { useDispatch } from 'react-redux';
import { addToCart } from '../features/cart/cartSlice';

function ProductCard({ product }) {

  const dispatch = useDispatch();

  return (
    <button
      onClick={() => dispatch(addToCart(product))}
    >
      Add To Cart
    </button>
  );
}

Navbar.jsx

import { useSelector } from 'react-redux';

function Navbar() {

  const items = useSelector(
    state => state.cart.items
  );

  return (
    <h3>Cart Items: {items.length}</h3>
  );
}

Complete Flow

ProductCard
↓
useDispatch()
↓
dispatch(addToCart(product))
↓

↓

↓

↓

↓
UI Re-renders

Most Common Beginner Mistake

Many beginners think: 

const items = useSelector(
  state => state.cart.items
);

creates state.

useSelector does not create state.

It only reads existing Redux state.

Similarly: 

const dispatch = useDispatch();

does not update state.

dispatch only sends actions.

Reducers actually update the state.

Easy Way To Remember

Hook Purpose
useSelector Read data from Redux Store
useDispatch Send actions to Redux Store
useSelector()
↓
Read State

useDispatch()
↓
Send Action

Interview Answer

useSelector is a React Redux hook used to read data from the Redux store, while useDispatch is a hook used to dispatch actions that update the Redux store state.

One-Line Interview Answer

useSelector reads Redux state and useDispatch sends actions to update Redux state.
