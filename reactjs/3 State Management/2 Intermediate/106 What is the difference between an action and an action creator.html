106. What is the Difference Between an Action and an Action Creator?

One-Line Definition

An action is a plain JavaScript object that describes what happened, while an action creator is a function that creates and returns an action object.

Think of It Like an Order Slip

Imagine a restaurant.

  • Action = The written food order.
  • Action Creator = The waiter who writes the order.
Action Creator
↓
Creates Action
↓
Dispatch Sends Action
↓
Reducer Updates State

What Is an Action?

An action is a plain JavaScript object.

It describes what happened in the application.

Example Action

{
  type: 'ADD_TO_CART',
  payload: {
    id: 1,
    name: 'Laptop'
  }
}
The action is the object itself.

Redux reducers read this object and decide how state should change.

What Is an Action Creator?

An action creator is a function that creates and returns an action object.

Example Action Creator

function addToCart(product) {

  return {
    type: 'ADD_TO_CART',
    payload: product
  };

}

Calling the function: 

addToCart({
  id: 1,
  name: 'Laptop'
});

returns: 

{
  type: 'ADD_TO_CART',
  payload: {
    id: 1,
    name: 'Laptop'
  }
}
The function is the action creator. The returned object is the action.

Traditional Redux Example

Action Creator

export const addToCart = product => {

  return {
    type: 'ADD_TO_CART',
    payload: product
  };

};

Dispatching Action

dispatch(
  addToCart(product)
);

Step-by-step: 

addToCart(product)
        ↓
Returns Action Object
        ↓
dispatch(action)
        ↓
Reducer Receives Action

Redux Toolkit Example

Redux Toolkit automatically creates action creators for us.

Slice

const cartSlice = createSlice({

  name: 'cart',

  initialState: {
    items: []
  },

  reducers: {

    addToCart(state, action) {
      state.items.push(action.payload);
    }

  }

});

Generated Action Creator

export const {
  addToCart
} = cartSlice.actions;

Redux Toolkit automatically creates: 

function addToCart(payload) {

  return {
    type: 'cart/addToCart',
    payload
  };

}

We don't write this ourselves.

What Actually Gets Dispatched?

Code We Write

dispatch(
  addToCart(product)
);

What Happens Internally

dispatch({

  type: 'cart/addToCart',

  payload: product

});
dispatch never receives the action creator function. It receives the action object returned by the action creator.

Real Project Example

ProductCard.jsx

import { useDispatch } from 'react-redux';
import { addToCart } from '../features/cart/cartSlice';

function ProductCard({ product }) {

  const dispatch = useDispatch();

  return (
    <button
      onClick={() => dispatch(addToCart(product))}
    >
      Add To Cart
    </button>
  );
}

↓

↓

↓

↓
Store Updates

Most Common Interview Confusion

Many developers think this is the action: 

addToCart(product)

Actually:

  • addToCart = Action Creator
  • addToCart(product) = Function Call
  • Returned Object = Action

Example

const action = addToCart(product);

console.log(action);

Output: 

{
  type: 'cart/addToCart',
  payload: product
}

This output object is the actual action.

Interview Answer

An action is a plain JavaScript object that describes what happened in the application, while an action creator is a function that creates and returns that action object. Redux Toolkit automatically generates action creators when we use createSlice().

One-Line Interview Answer

An action is an object, and an action creator is the function that creates that object.
