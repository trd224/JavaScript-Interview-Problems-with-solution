093. What is Derived State?

One-Line Definition

Derived state is state that can be calculated from existing props or state instead of being stored separately.

What Does "Derived" Mean?

Derived means "calculated from something else".

If a value can be computed from existing data, we usually do not store it in state.

Existing State / Props
↓
Calculate New Value
↓
Derived State

Simple Example

Suppose we have a shopping cart. 

const [items] = useState([
  { id: 1, price: 100 },
  { id: 2, price: 200 }
]);

The total price can be calculated from items. 

const total = items.reduce(
  (sum, item) => sum + item.price,
  0
);

Here total is derived state.

Good Example

function Cart() {

  const [items] = useState([
    { id: 1, price: 100 },
    { id: 2, price: 200 }
  ]);

  const total = items.reduce(
    (sum, item) => sum + item.price,
    0
  );

  return <h2>Total: {total}</h2>;

}

The total is calculated during rendering and is always correct.

Bad Example

Many beginners unnecessarily store derived values in state. 

const [items, setItems] = useState([]);

const [total, setTotal] = useState(0);

Now we must manually keep total synchronized whenever items change. 

setItems(newItems);

setTotal(
  newItems.reduce(
    (sum, item) => sum + item.price,
    0
  )
);
This creates duplicate state and can easily lead to bugs.

Why Is This a Problem?

If we forget to update total, the UI becomes incorrect.

items Updated
↓
Forgot To Update total
↓
Incorrect UI

Derived State from Props

Derived values can also come from props. 

function User({ firstName, lastName }) {

  const fullName =
    `${firstName} ${lastName}`;

  return <h2>{fullName}</h2>;

}

fullName is derived from props.

Derived State with useMemo

If the calculation is expensive, we can use useMemo. 

const total = useMemo(() => {
  return items.reduce(
    (sum, item) => sum + item.price,
    0
  );
}, [items]);

useMemo avoids unnecessary recalculations.

When Should We Store State?

Store state only when the value can change independently.

Value Store in State?
User Input ✅ Yes
API Response ✅ Yes
Total Price from Items ❌ No
Full Name from Props ❌ No
Filtered List from Existing Data ❌ Usually No

Real Project Example

In our Aadhaar Authentication application, if we receive firstName and lastName from an API, we should not create a separate state variable for fullName. We can derive it directly from the existing data during rendering.

Common Derived State Examples

  • Full name from first and last name.
  • Total price from cart items.
  • Completed task count from tasks list.
  • Filtered products from product list.
  • Authentication status from user data.

Advantages

  • Less state to manage.
  • Fewer bugs.
  • No synchronization issues.
  • Cleaner code.
  • Better maintainability.

Important Interview Points

  • Derived state is calculated from existing props or state.
  • Avoid storing derived values in state when possible.
  • Duplicate state can cause synchronization bugs.
  • Calculate derived values during rendering.
  • Use useMemo for expensive calculations.

Short Interview Answer

Derived state is data that can be calculated from existing props or state. Instead of storing it separately, React applications usually compute it during rendering to avoid duplicate state and synchronization issues.
