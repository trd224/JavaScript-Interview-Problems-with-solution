097. What is a Reducer Function?

One-Line Definition

A reducer function is a function that receives the current state and an action, then returns the next state.

Why Is It Called a Reducer?

The term comes from JavaScript's Array.reduce() method, where multiple values are reduced into a single result.

In React, a reducer takes the current state and an action and reduces them into a new state.

Reducer Function Syntax

function reducer(state, action) {
  return newState;
}

Parameters

  • state → Current state.
  • action → Information about what happened.

Return Value

  • Returns the next state.

Simple Example

function reducer(state, action) {

  switch (action.type) {

    case 'INCREMENT':
      return {
        count: state.count + 1
      };

    default:
      return state;

  }

}

Step-by-Step Example

Current state: 

{ count: 0 }

Action dispatched: 

dispatch({ type: 'INCREMENT' });

React calls: 

reducer(
  { count: 0 },
  { type: 'INCREMENT' }
);

Reducer returns: 

{ count: 1 }

React stores the new state and re-renders the component.

Reducer Inside useReducer

const initialState = {
  count: 0
};

function reducer(state, action) {

  switch (action.type) {

    case 'INCREMENT':
      return {
        count: state.count + 1
      };

    case 'DECREMENT':
      return {
        count: state.count - 1
      };

    default:
      return state;

  }

}

function Counter() {

  const [state, dispatch] =
    useReducer(reducer, initialState);

  return (
    <>
      <h2>{state.count}</h2>

      <button
        onClick={() => dispatch({ type: 'INCREMENT' })}
      >
        +
      </button>

      <button
        onClick={() => dispatch({ type: 'DECREMENT' })}
      >
        -
      </button>
    </>
  );

}

Reducer Is Like a Decision Maker

Think of the reducer as a security guard that checks the action and decides how state should change.

↓
Reducer Checks Action
↓
Returns Updated State

Multiple Actions Example

function reducer(state, action) {

  switch (action.type) {

    case 'ADD':
      return {
        count: state.count + 1
      };

    case 'SUBTRACT':
      return {
        count: state.count - 1
      };

    case 'RESET':
      return {
        count: 0
      };

    default:
      return state;

  }

}

One reducer can handle many actions.

Real-World Form Reducer

function reducer(state, action) {

  switch (action.type) {

    case 'UPDATE_FIELD':
      return {
        ...state,
        [action.field]: action.value
      };

    case 'SET_ERROR':
      return {
        ...state,
        error: action.message
      };

    case 'RESET_FORM':
      return initialState;

    default:
      return state;

  }

}

This reducer can update form fields, show errors, and reset the form.

Rules of a Reducer Function

  • Must be a pure function.
  • Must not modify the existing state directly.
  • Must return a new state object.
  • Same input should always produce the same output.

Bad Example

Never mutate state directly. 

function reducer(state, action) {

  state.count++;

  return state;

}
This mutates the existing state object and should be avoided.

Good Example

function reducer(state, action) {

  return {
    count: state.count + 1
  };

}
Always return a new state object.

Real Project Example

In our Aadhaar Authentication application, a reducer can manage Aadhaar number, OTP, loading state, error messages, consent, and authentication type. Different actions such as UPDATE_OTP, SET_ERROR, START_LOADING, and RESET_FORM can all be handled by a single reducer function.

Reducer vs Dispatch

Reducer Dispatch
Decides how state changes Sends an action
Receives state and action Receives an action object
Returns new state Triggers reducer execution

Important Interview Points

  • A reducer receives state and action.
  • A reducer returns the next state.
  • Reducers are used with useReducer.
  • Reducers should be pure functions.
  • Reducers must not mutate state directly.
  • One reducer can handle multiple actions.

Short Interview Answer

A reducer function is a function that receives the current state and an action, then returns the next state. It is commonly used with useReducer to manage complex state updates in a predictable way.

One-Line Interview Answer

A reducer function takes the current state and an action and returns the new state.
