099. How does useReducer compare to Redux?

One-Line Definition

useReducer is a built-in React Hook for component-level state management, while Redux is a separate library used for managing application-wide global state.

The Most Important Thing to Understand

Both useReducer and Redux use the same core concepts:

  • State
  • Actions
  • Reducers
  • Dispatch

In fact, Redux was the inspiration for React's useReducer Hook.

Both Follow the Same Flow

dispatch(action)
↓
Reducer Receives Action
↓
Returns New State
↓
UI Updates

useReducer Example

const initialState = {
  count: 0
};

function reducer(state, action) {
  switch (action.type) {
    case 'INCREMENT':
      return {
        count: state.count + 1
      };

    default:
      return state;
  }
}

function Counter() {

  const [state, dispatch] =
    useReducer(reducer, initialState);

  return (
    <button
      onClick={() => dispatch({ type: 'INCREMENT' })}
    >
      {state.count}
    </button>
  );

}

Redux Example

const initialState = {
  count: 0
};

function counterReducer(state = initialState, action) {

  switch (action.type) {

    case 'INCREMENT':
      return {
        count: state.count + 1
      };

    default:
      return state;

  }

}

Notice that the reducer looks almost identical.

Main Difference

useReducer Redux
Inside a React component Outside React components
Local or shared state Global application state
Built into React Separate library
No installation needed Requires installation

How Data Is Accessed

useReducer

function UserForm() {

  const [state, dispatch] = useReducer(reducer, initialState);

}

Only this component and its children can easily access the state.

Redux

const user = useSelector(state => state.user);

const dispatch = useDispatch();

Any component in the application can access the Redux store.

Real Example

Imagine an e-commerce application.

Good useReducer Use Cases

  • Checkout form
  • Login form
  • Wizard form
  • Complex modal workflow

Good Redux Use Cases

  • Logged-in user
  • Shopping cart
  • Theme
  • Language
  • Notifications
  • Application-wide settings

Complex Form Example

A form with multiple related fields is a great use case for useReducer. 

const initialState = {
  firstName: '',
  lastName: '',
  email: '',
  loading: false,
  error: ''
};

const [state, dispatch] = useReducer(reducer, initialState);

The state belongs only to that form, so Redux would be unnecessary.

Authentication Example

Suppose the logged-in user must be accessible everywhere.

In that case Redux is a better choice. 

{
  user: {
    id: 1,
    name: 'Rizwan'
  }
}

Many pages may need access to the same user information.

Can useReducer Replace Redux?

Sometimes yes.

Many small and medium React applications use: 

useReducer + Context API

instead of Redux.

For many projects, useReducer combined with Context API can provide Redux-like state management without installing Redux.

useReducer + Context API Example

const UserContext = createContext();

function App() {

  const [state, dispatch] = useReducer(reducer, initialState);

  return (
    <UserContext.Provider value={{ state, dispatch }}>
        <Dashboard />
    </UserContext.Provider>
  );

}

Now multiple components can access the reducer state.

Advantages of useReducer

  • Built into React.
  • No extra dependency.
  • Easier to learn.
  • Perfect for component-level state.
  • Great for complex forms.

Advantages of Redux

  • Centralized global store.
  • Excellent DevTools.
  • Middleware support.
  • Better for large applications.
  • Works outside React as well.

Real Project Example

In our Aadhaar Authentication application, useReducer is a great choice for managing the authentication form because Aadhaar number, OTP, loading state, consent, and errors are closely related. However, if user information, department details, permissions, and session data need to be shared across the entire application, Redux would be a better choice.

Important Interview Points

  • useReducer and Redux use the same concepts.
  • Both use actions, reducers, and dispatch.
  • useReducer is built into React.
  • Redux is a separate state management library.
  • useReducer is best for local complex state.
  • Redux is best for global application state.
  • useReducer + Context API can sometimes replace Redux.

Short Interview Answer

useReducer and Redux both use reducers, actions, and dispatch. The main difference is that useReducer is a built-in React Hook for component-level state management, while Redux is a separate library designed for managing global application state.

One-Line Interview Answer

useReducer manages complex local state inside React components, while Redux manages global state across the entire application.
