Traditional Redux (Professional Architecture)

Redux is a predictable state container that allows us to manage global state inside a single centralized store.

Why Redux?

Without Redux, passing data from parent to deeply nested child components leads to prop drilling.

Step 1 — Install Redux

npm install redux react-redux
Package Purpose
redux Creates and manages the Redux Store.
react-redux Connects React Components with Redux.

Step 2 — Create Action Constants

redux/constants/counterConstants.js

export const INCREMENT = "INCREMENT";
export const DECREMENT = "DECREMENT";
Using constants prevents spelling mistakes in action types.

Step 3 — Create Action Creators

redux/actions/counterActions.js

import { INCREMENT, DECREMENT } from "../constants/counterConstants";

export const increment = () => {
  return { type: INCREMENT };
};

export const decrement = () => {
  return { type: DECREMENT };
};
Action creators return action objects. Instead of writing dispatch({ type: "INCREMENT" }) everywhere, we simply write dispatch(increment())

Step 4 — Create Reducer

redux/reducers/counterReducer.js

import { INCREMENT, DECREMENT } from "../constants/counterConstants";

const initialState = { count: 0 };

export default function counterReducer(state = initialState, action) {
  switch (action.type) {
    case INCREMENT:
      return { ...state, count: state.count + 1 };
    case DECREMENT:
      return { ...state, count: state.count - 1 };
    default:
      return state;
  }
}
Reducers never modify the existing state. They always return a brand-new state object.

Step 5 — Create Root Reducer

redux/reducers/rootReducer.js

import { combineReducers } from "redux";
import counterReducer from "./counterReducer";

const rootReducer = combineReducers({
  counter: counterReducer
});

export default rootReducer;
Even if there is only one reducer today, using a Root Reducer makes scaling much easier.

Step 6 — Create Store

app/store.js

import { createStore } from "redux";
import rootReducer from "../redux/reducers/rootReducer";

const store = createStore(rootReducer);

export default store;
The Store keeps the entire application's global state.

Step 7 — Connect Redux with React

index.js

import React from "react";
import ReactDOM from "react-dom/client";
import { Provider } from "react-redux";
import store from "./app/store";
import App from "./App";

const root = ReactDOM.createRoot(document.getElementById("root"));

root.render(
  <Provider store={store}>
    <App />
  </Provider>
);
The Provider makes the Redux Store available to every React component.

Step 8 — Read Data using useSelector()

features/counter/Counter.js

import { useSelector } from "react-redux";

export default function Counter() {
  const count = useSelector(state => state.counter.count);
  return <h1>{count}</h1>;
}
useSelector() allows us to read data from the Redux Store. Whenever the selected state changes, React automatically re-renders the component.

Step 9 — Update State using useDispatch()

features/counter/CounterButtons.js

import { useDispatch } from "react-redux";
import { increment, decrement } from "../../redux/actions/counterActions";

export default function CounterButtons() {
  const dispatch = useDispatch();

  return (
    <>
      <button onClick={() => dispatch(increment())}>+</button>
      <button onClick={() => dispatch(decrement())}>-</button>
    </>
  );
}
useDispatch() allows us to send actions to Redux. The reducer receives the action and updates the Store.

Step 10 — App.js

import Counter from "./features/counter/Counter";
import CounterButtons from "./features/counter/CounterButtons";

export default function App() {
  return (
    <>
      <Counter />
      <CounterButtons />
    </>
  );
}

Why Reducers Return a New Object?

Redux compares object references to detect state changes. If we modify the existing object directly, Redux cannot reliably detect the update. Therefore reducers must always return a new object.

❌ Wrong

state.count++;
return state;

✅ Correct

return { ...state, count: state.count + 1 };

Advantages of Using Constants

  • Avoids spelling mistakes.
  • Provides auto-completion in editors.
  • Easy to reuse across multiple reducers.
  • Central place for action names.

Advantages of Action Creators

  • Reusable.
  • Cleaner components.
  • Easy to add payloads later.
  • Business logic stays outside UI components.

Advantages of Root Reducer

  • Supports multiple reducers.
  • Scales well for enterprise applications.
  • Keeps store configuration clean.

Traditional Redux Interview Questions

1. What is Redux?

Redux is a predictable state container that allows us to manage global state in a single centralized store.

2. What is a Store?

A Store is an object that holds the application's global state.

3. What is an Action?

An Action is a plain JavaScript object that describes what happened.

4. What is an Action Creator?

A function that returns an Action object.

5. What is a Reducer?

A pure function that receives the current state and an action, then returns a new state.

6. What is useSelector()?

A React-Redux hook used to read data from the Redux Store.

7. What is useDispatch()?

A React-Redux hook used to dispatch actions to Redux.

8. Why should reducers never mutate state?

Because Redux depends on immutable updates to detect changes and trigger UI re-rendering correctly.

Traditional Redux Flow in One Line

Component → Dispatch → Action Creator → Action → Store → Root Reducer → Reducer → New State → Store Update → useSelector → UI Re-render
