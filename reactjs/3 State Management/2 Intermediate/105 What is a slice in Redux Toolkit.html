105. What is a Slice in Redux Toolkit?

One-Line Definition

A slice is a feature-specific part of the Redux store that contains its own state, reducers, and automatically generated actions.

Why Is It Called a Slice?

Imagine our Redux store is one big object. 

{
  user: {
    name: 'Rizwan'
  },

  cart: {
    items: []
  },

  orders: {
    list: []
  }
}

This store is divided into smaller parts.

Redux Store
↓
user slice
cart slice
orders slice
Each small feature-based part of the Redux store is called a slice.

Simple Meaning

A slice means one feature's Redux logic in one file.

For example:

  • cartSlice handles cart state and cart actions.
  • userSlice handles user state and user actions.
  • orderSlice handles order state and order actions.

Cart Slice Example

features/cart/cartSlice.js

import { createSlice } from '@reduxjs/toolkit';

const initialState = {
  items: []
};

const cartSlice = createSlice({
  name: 'cart',
  initialState,
  reducers: {
    addToCart(state, action) {
      state.items.push(action.payload);
    },

    removeFromCart(state, action) {
      state.items = state.items.filter(item => item.id !== action.payload);
    }
  }
});

export const { addToCart, removeFromCart } = cartSlice.actions;

export default cartSlice.reducer;

What Does This Slice Contain?

Part Meaning
name Name of the slice, like cart
initialState Default state of this slice
reducers Functions that update this slice state
actions Automatically created action creators
reducer Reducer exported to the Redux store

1. Slice State

This is the state owned by the cart slice. 

const initialState = {
  items: []
};
This state belongs only to the cart slice.

2. Slice Reducers

Reducers define how this slice state will change. 

reducers: {
  addToCart(state, action) {
    state.items.push(action.payload);
  },

  removeFromCart(state, action) {
    state.items = state.items.filter(item => item.id !== action.payload);
  }
}
Reducers are the update logic of the slice.

3. Slice Actions

Redux Toolkit automatically creates actions from reducer names. 

export const { addToCart, removeFromCart } = cartSlice.actions;

Now components can dispatch these actions. 

dispatch(addToCart(product));

dispatch(removeFromCart(productId));

What Redux Toolkit Generates Internally

When we write this: 

reducers: {
  addToCart(state, action) {},
  removeFromCart(state, action) {}
}

Redux Toolkit automatically creates: 

Action Types:

cart/addToCart
cart/removeFromCart


Action Creators:

addToCart()
removeFromCart()


Reducer Logic:

case cart/addToCart
case cart/removeFromCart
We do not need to manually create action types, action creators, or switch cases.

Connect Slice With Store

app/store.js

import { configureStore } from '@reduxjs/toolkit';
import cartReducer from '../features/cart/cartSlice';
import userReducer from '../features/user/userSlice';

export const store = configureStore({
  reducer: {
    cart: cartReducer,
    user: userReducer
  }
});

Now Redux store will look like this: 

{
  cart: {
    items: []
  },

  user: {
    name: '',
    email: ''
  }
}
The key inside configureStore becomes the key inside Redux state.

How Component Uses Slice

components/ProductCard.jsx

import { useDispatch } from 'react-redux';
import { addToCart } from '../features/cart/cartSlice';

function ProductCard({ product }) {
  const dispatch = useDispatch();

  return (
    <button onClick={() => dispatch(addToCart(product))}>
      Add To Cart
    </button>
  );
}

export default ProductCard;

components/Navbar.jsx

import { useSelector } from 'react-redux';

function Navbar() {
  const items = useSelector(state => state.cart.items);

  return <h3>Cart Items: {items.length}</h3>;
}

export default Navbar;

Why Slice Is Useful?

  • It keeps state and update logic together.
  • It reduces boilerplate code.
  • It automatically creates actions.
  • It removes manual switch statements.
  • It keeps each feature organized separately.

Without Slice vs With Slice

Traditional Redux Redux Toolkit Slice
Separate action types Automatically generated
Separate action creators Automatically generated
Separate reducer file Reducer inside slice
Switch statement No switch statement
More boilerplate Less boilerplate

Interview Answer

A slice in Redux Toolkit is a feature-based section of the Redux store created using createSlice(). It contains the initial state, reducer functions, and automatically generated action creators for that specific feature.

One-Line Interview Answer

A slice is one feature's Redux state and update logic combined together using Redux Toolkit's createSlice() function.
