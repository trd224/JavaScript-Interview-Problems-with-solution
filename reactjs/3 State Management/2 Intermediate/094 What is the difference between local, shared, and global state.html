094. What is the Difference Between Local, Shared, and Global State?

One-Line Definition

Local state belongs to one component, shared state belongs to multiple related components, and global state belongs to the entire application.

Why Do We Need Different Types of State?

Not every piece of data should be accessible everywhere.

The scope of the data determines where the state should live.

1. Local State

Local state is owned and used by a single component.

It is usually created using useState.

Example

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <button onClick={() => setCount(count + 1)}>
      {count}
    </button>
  );
}

Only the Counter component can access count.

Common Examples

  • Input field value
  • Dropdown open/close state
  • Modal visibility
  • Component loading state

2. Shared State

Shared state is used when multiple related components need access to the same data.

The state is usually lifted to the nearest common parent.

Example

function App() {
  const [search, setSearch] = useState('');

  return (
    <>
      <SearchBox
        search={search}
        setSearch={setSearch}
      />

      <ProductList search={search} />
    </>
  );
}
Parent Component
↓
Stores State
↓
Passes Props
↓
Multiple Children Use Same Data

Common Examples

  • Search filters
  • Selected user
  • Current tab
  • Multi-step form data

3. Global State

Global state is accessible throughout the application.

Any component can access it without prop drilling.

Example Using Context API

const UserContext = createContext();

function App() {
  const [user] = useState('Rizwan');

  return (
    <UserContext.Provider value={user}>
      <Dashboard />
    </UserContext.Provider>
  );
}

Nested Component

function Profile() {
  const user = useContext(UserContext);

  return <h2>{user}</h2>;
}

Common Examples

  • Logged-in user
  • Theme (Dark/Light)
  • Language
  • Authentication status
  • Shopping cart

When Should We Use Each?

Situation Recommended State
Input Value Local State
Modal Open/Close Local State
Search Filter Shared State
Wizard Form Data Shared State
Logged-In User Global State
Theme Global State

Important Rule

Start with local state. Move it to shared state only when multiple components need it. Use global state only when the data is needed across many unrelated parts of the application.

Real Project Example

In our Aadhaar Authentication application:
  • Local State: OTP input value, captcha value, popup visibility.
  • Shared State: Authentication form data shared between multiple child components.
  • Global State: Logged-in user details, department information, session data, theme settings.

Important Interview Points

  • Local state belongs to one component.
  • Shared state is lifted to a common parent.
  • Global state is available application-wide.
  • Do not put everything into global state.
  • Prefer local state whenever possible.
  • Context API, Redux, and Zustand are common global state solutions.

Short Interview Answer

Local state is used by a single component, shared state is used by multiple related components through a common parent, and global state is available throughout the entire application using tools such as Context API, Redux, or Zustand.

One-Line Interview Answer

Local state belongs to one component, shared state belongs to related components, and global state belongs to the entire application.
