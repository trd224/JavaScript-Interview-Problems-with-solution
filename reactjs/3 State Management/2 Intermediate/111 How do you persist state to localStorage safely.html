111. How Do You Persist State to localStorage Safely?

One-Line Definition

Persisting state to localStorage means saving selected React state in the browser so it remains available after page refresh, but it must be done safely with JSON parsing, error handling, browser checks, and without storing sensitive data.

Why Do We Persist State?

Normally React state is lost when the page refreshes. 

const [theme, setTheme] = useState('dark');

If the user changes the theme and refreshes the page, React starts again from the initial value.

User Changes State
↓
Page Refresh
↓
React State Resets

To keep some values after refresh, we can save them in localStorage.

What Should You Store in localStorage?

Good To Store Do Not Store
Theme preference Password
Language preference Access token if avoidable
Cart items Credit card details
Sidebar open/close preference Private user documents
Non-sensitive filters Highly sensitive personal data
localStorage is readable by JavaScript, so never store passwords, secret keys, or highly sensitive data in it.

Unsafe Example

const savedCart = JSON.parse(localStorage.getItem('cart'));

const [cart, setCart] = useState(savedCart);

This can break because:

  • localStorage may not exist during server-side rendering.
  • Stored JSON may be invalid.
  • localStorage access can throw errors.
  • Initial value may become null unexpectedly.

Safe Local Storage Helpers

Create reusable helper functions.

utils/storage.js

export function getStorageValue(key, fallbackValue) {
  try {
    if (typeof window === 'undefined') return fallbackValue;

    const savedValue = window.localStorage.getItem(key);

    if (savedValue === null) return fallbackValue;

    return JSON.parse(savedValue);
  } catch (error) {
    console.error('Error reading localStorage:', error);
    return fallbackValue;
  }
}

export function setStorageValue(key, value) {
  try {
    if (typeof window === 'undefined') return;

    window.localStorage.setItem(key, JSON.stringify(value));
  } catch (error) {
    console.error('Error writing localStorage:', error);
  }
}

export function removeStorageValue(key) {
  try {
    if (typeof window === 'undefined') return;

    window.localStorage.removeItem(key);
  } catch (error) {
    console.error('Error removing localStorage:', error);
  }
}
Safe localStorage code should always use try/catch and check if window exists.

React Example: Persist Theme

Use lazy initial state so localStorage is read only once on first render. 

import { useEffect, useState } from 'react';
import { getStorageValue, setStorageValue } from './utils/storage';

function ThemeSwitcher() {
  const [theme, setTheme] = useState(() => getStorageValue('theme', 'dark'));

  useEffect(() => {
    setStorageValue('theme', theme);
  }, [theme]);

  return (
    <button onClick={() => setTheme(theme === 'dark' ? 'light' : 'dark')}>
      Current Theme: {theme}
    </button>
  );
}

export default ThemeSwitcher;

Why Lazy Initial State?

This is better: 

const [theme, setTheme] = useState(() => getStorageValue('theme', 'dark'));

Than this: 

const [theme, setTheme] = useState(getStorageValue('theme', 'dark'));
Lazy initial state runs only during the initial render, not on every re-render.

Custom Hook: useLocalStorage

A reusable hook is the cleanest solution.

hooks/useLocalStorage.js

import { useEffect, useState } from 'react';

function readValue(key, initialValue) {
  try {
    if (typeof window === 'undefined') return initialValue;

    const item = window.localStorage.getItem(key);

    return item ? JSON.parse(item) : initialValue;
  } catch (error) {
    console.error('localStorage read error:', error);
    return initialValue;
  }
}

export function useLocalStorage(key, initialValue) {
  const [value, setValue] = useState(() => readValue(key, initialValue));

  useEffect(() => {
    try {
      window.localStorage.setItem(key, JSON.stringify(value));
    } catch (error) {
      console.error('localStorage write error:', error);
    }
  }, [key, value]);

  return [value, setValue];
}

Using the Hook

import { useLocalStorage } from './hooks/useLocalStorage';

function CartPage() {
  const [cart, setCart] = useLocalStorage('cart', []);

  function addToCart(product) {
    setCart(prevCart => [...prevCart, product]);
  }

  return (
    <button onClick={() => addToCart({ id: 1, name: 'Laptop' })}>
      Add Laptop
    </button>
  );
}

export default CartPage;

Persist Redux State Safely

Do not persist the whole Redux store blindly.

Only persist selected non-sensitive slices like cart, theme, or filters.

Example: Persist Only Cart

const preloadedState = {
  cart: getStorageValue('cart', { items: [] })
};

store.js

import { configureStore } from '@reduxjs/toolkit';
import cartReducer from '../features/cart/cartSlice';
import { getStorageValue, setStorageValue } from '../utils/storage';

export const store = configureStore({
  reducer: {
    cart: cartReducer
  },
  preloadedState: {
    cart: getStorageValue('cart', { items: [] })
  }
});

store.subscribe(() => {
  const state = store.getState();
  setStorageValue('cart', state.cart);
});
preloadedState loads saved state when the Redux store is created.

Important Safety Rules

  • Use try/catch when reading or writing localStorage.
  • Check typeof window before accessing localStorage.
  • Use JSON.stringify before saving objects or arrays.
  • Use JSON.parse when reading objects or arrays.
  • Use fallback values if saved data is missing or broken.
  • Do not store passwords, tokens, or sensitive data.
  • Persist only the required state, not the complete app state.

Common Mistakes

Mistake Problem
Reading localStorage directly in SSR window is not defined
No try/catch around JSON.parse App can crash if stored data is invalid
Saving sensitive data Security risk
Saving entire Redux store Unnecessary data and possible stale state
No fallback value State can become null or undefined

↓

↓
Use Fallback If Missing
↓

↓

↓
Save Updated State to localStorage

Interview Answer

To persist state to localStorage safely, we should read the saved value inside a lazy initializer, wrap JSON.parse and localStorage access in try/catch, check that window exists for SSR safety, provide a fallback value, stringify data before saving, and avoid storing sensitive information.

One-Line Interview Answer

Persist state safely by reading localStorage with fallback and try/catch, writing with JSON.stringify inside useEffect, checking for window, and never storing sensitive data.
