107. What is Middleware in Redux? Name a Common Middleware.

One-Line Definition

Middleware is a function that runs between dispatching an action and the action reaching the reducer. It allows Redux to perform extra work such as API calls, logging, analytics, and error handling.

Normal Redux Flow

Without middleware, Redux follows a simple flow:

Component
↓
dispatch(action)
↓
Reducer
↓
Store Updated

The action goes directly to the reducer.

Redux Flow With Middleware

Middleware sits between dispatch and reducer.

Component
↓
dispatch(action)
↓
Middleware
↓
Reducer
↓
Store Updated
Middleware gets a chance to inspect, modify, delay, log, or perform async work before the action reaches the reducer.

Why Do We Need Middleware?

Reducers must be pure functions.

Reducers should:

  • Return new state.
  • Not call APIs.
  • Not use setTimeout.
  • Not perform side effects.

Wrong Example

function cartReducer(state, action) {

  fetch('/products');

  return state;
}
Reducers should never contain API calls or side effects.

Instead, Redux middleware handles those tasks.

Real Example - API Call

Suppose we need to load products from the server.

User Opens Page
↓
Dispatch Action
↓
Middleware Calls API
↓
Receives Data
↓
Dispatch Success Action
↓
Reducer Updates Store

This is one of the most common uses of middleware.

How Middleware Works

Middleware receives an action before the reducer. 

const logger = store => next => action => {

  console.log('Action:', action);

  next(action);

};

When an action is dispatched: 

dispatch(action)
       ↓
logger middleware runs
       ↓
next(action)
       ↓
reducer runs
Calling next(action) passes the action to the next middleware or reducer.

Common Middleware: Redux Thunk

The most common Redux middleware is Redux Thunk.

Redux normally expects: 

dispatch({
  type: 'LOAD_PRODUCTS'
});

A plain object action.

Redux Thunk allows: 

dispatch(async function() {

  const response = await fetch('/api/products');

});

Now we can perform asynchronous operations before updating Redux state.

Traditional Redux Thunk Example

export const fetchProducts = () => {

  return async dispatch => {

    const response = await fetch('/api/products');

    const data = await response.json();

    dispatch({
      type: 'PRODUCTS_LOADED',
      payload: data
    });

  };

};

Flow:

dispatch(fetchProducts())
↓
Redux Thunk Runs
↓
API Call
↓
Data Received
↓
Dispatch Success Action
↓
Reducer Updates Store

Redux Toolkit and Middleware

Redux Toolkit automatically includes useful middleware.

configureStore()

import { configureStore } from '@reduxjs/toolkit';

const store = configureStore({
  reducer: {
    cart: cartReducer
  }
});

Redux Toolkit automatically adds:

  • Redux Thunk middleware
  • Immutability check middleware
  • Serializable state check middleware
In Redux Toolkit, Redux Thunk is already included by default.

Interview Answer

Middleware is a function that runs between dispatching an action and the action reaching the reducer. It is commonly used for API calls, logging, analytics, and other side effects. Redux Thunk is the most commonly used Redux middleware.

One-Line Interview Answer

Middleware intercepts actions before they reach reducers, and Redux Thunk is the most common middleware used for asynchronous operations.
