Before Step 2: Why createSlice Is Used?

Before learning Redux Toolkit code, first understand that Redux already worked before createSlice() existed.

The old Redux way required us to write three things separately for every update:

  • Action Type
  • Action Creator
  • Reducer Case
createSlice() does not create a new Redux concept. It mainly removes repeated boilerplate code by keeping state, reducers, and actions together in one place.

Step A: Create Action Types

src/redux/constants/cartConstants.js

export const ADD_TO_CART = 'ADD_TO_CART';
export const REMOVE_FROM_CART = 'REMOVE_FROM_CART';

Action type tells Redux what happened. 

{
  type: 'ADD_TO_CART'
}
ADD_TO_CART means: user wants to add a product to cart.

Step B: Create Action Creators

src/redux/actions/cartActions.js

import { ADD_TO_CART, REMOVE_FROM_CART } from '../constants/cartConstants';

export function addToCart(product) {
  return {
    type: ADD_TO_CART,
    payload: product
  };
}

export function removeFromCart(id) {
  return {
    type: REMOVE_FROM_CART,
    payload: id
  };
}

Action creator is a normal function that returns an action object. 

dispatch(addToCart(product));

This creates this action object: 

{
  type: 'ADD_TO_CART',
  payload: product
}
Component does not directly send a plain object every time. Usually, component calls an action creator like addToCart(product).

Step C: Create Reducer

src/redux/reducers/cartReducer.js

import { ADD_TO_CART, REMOVE_FROM_CART } from '../constants/cartConstants';

const initialState = {
  items: []
};

export default function cartReducer(state = initialState, action) {
  switch (action.type) {
    case ADD_TO_CART:
      return {
        ...state,
        items: [...state.items, action.payload]
      };

    case REMOVE_FROM_CART:
      return {
        ...state,
        items: state.items.filter(item => item.id !== action.payload)
      };

    default:
      return state;
  }
}

Reducer receives current state and action, then returns the new state.

In traditional Redux, we must return a new object manually. We should not directly mutate Redux state.

Step D: Create Store

There are two cases when creating a Redux store in traditional Redux.

Case 1: Store With Single Reducer

If our application has only cart state, we can directly pass cartReducer to createStore().

src/redux/store.js

import { createStore } from 'redux';
import cartReducer from './reducers/cartReducer';

const store = createStore(cartReducer);

export default store;

In this case, Redux state directly looks like this: 

{
  items: []
}

So we read cart items like this: 

const cartItems = useSelector(state => state.items);
This works only when the store has one reducer directly, like createStore(cartReducer).

Case 2: Store With Multiple Reducers

Real applications usually have multiple reducers such as cart, auth, wishlist, and orders. 

cartReducer
authReducer
wishlistReducer
orderReducer

In this case, we cannot directly pass only one reducer. We combine all reducers using combineReducers().

src/redux/store.js

import { createStore, combineReducers } from 'redux';
import cartReducer from './reducers/cartReducer';
import authReducer from './reducers/authReducer';
import wishlistReducer from './reducers/wishlistReducer';

const rootReducer = combineReducers({
  cart: cartReducer,
  auth: authReducer,
  wishlist: wishlistReducer
});

const store = createStore(rootReducer);

export default store;

Now Redux state becomes an object with separate sections. 

{
  cart: {
    items: []
  },

  auth: {
    user: null
  },

  wishlist: {
    items: []
  }
}

So we read data using the reducer keys. 

const cartItems = useSelector(state => state.cart.items);
const user = useSelector(state => state.auth.user);
const wishlistItems = useSelector(state => state.wishlist.items);
Important: When we use combineReducers(), the keys we write inside combineReducers become the keys in Redux state. If we write cart: cartReducer, then cart data is read using state.cart.

Single Reducer vs Multiple Reducers

Case Store Code State Reading
Single reducer createStore(cartReducer) state.items
Multiple reducers createStore(rootReducer) state.cart.items

Problem With Traditional Redux

For every cart operation, we have to create action type, action creator, and reducer case separately. 

ADD_TO_CART
REMOVE_FROM_CART
INCREASE_QUANTITY
DECREASE_QUANTITY
CLEAR_CART

For each action above, old Redux needs repeated code.

Requirement Traditional Redux Work
Add product Write action type, action creator, reducer case
Remove product Write action type, action creator, reducer case
Increase quantity Write action type, action creator, reducer case
Clear cart Write action type, action creator, reducer case
Traditional Redux is powerful, but it creates too much repeated code for simple state updates.

Solution 2: Redux Toolkit With createSlice

Redux Toolkit says: instead of writing action types, action creators, and reducers separately, write them together inside one slice. 

import { createSlice } from '@reduxjs/toolkit';

const initialState = {
  items: [],
  totalItems: 0,
  totalAmount: 0,
  loading: false,
  error: ''
};

const cartSlice = createSlice({
  name: 'cart',
  initialState,
  reducers: {
    addToCart(state, action) {
      const product = action.payload;

      state.items.push({
        ...product,
        quantity: 1
      });

      state.totalItems += 1;
      state.totalAmount += product.price;
    },

    removeFromCart(state, action) {
      const item = state.items.find(item => item.id === action.payload);

      if (item) {
        state.totalItems -= item.quantity;
        state.totalAmount -= item.price * item.quantity;
      }

      state.items = state.items.filter(item => item.id !== action.payload);
    }
  }
});

export const { addToCart, removeFromCart } = cartSlice.actions;
export default cartSlice.reducer;
Here items, totalItems, totalAmount, loading, and error all belong to one feature: cart. This is why they live together inside one cart slice initialState.

What createSlice Automatically Creates

You Write Redux Toolkit Creates
name: 'cart' Action type prefix: cart
addToCart(state, action) Action type: cart/addToCart
removeFromCart(state, action) Action type: cart/removeFromCart
cartSlice.actions Action creators
cartSlice.reducer Reducer function for the store

Multiple State Values Inside One Slice

A slice is not limited to one value. One slice usually manages a complete feature state. 

const initialState = {
  items: [],
  totalItems: 0,
  totalAmount: 0,
  loading: false,
  error: ''
};
State Value Meaning
items Products added to cart.
totalItems Total product quantity in cart.
totalAmount Total cart price.
loading Used while cart-related API work is running.
error Stores cart error message.
Do not confuse multiple reducers with multiple state values. cart is one reducer/slice, but inside cart initialState we can keep many related state values.

Side-by-Side Comparison

Traditional Redux Redux Toolkit createSlice
Write action type manually Action type generated automatically
Write action creator manually Action creator generated automatically
Write switch case manually Reducer functions written directly
Need immutable updates manually Can write simpler update logic
More boilerplate Less boilerplate

Important: Why We Can Write state.items.push()

In normal Redux, state should not be mutated directly. 

return {
  ...state,
  items: [...state.items, product]
};

But inside createSlice(), this is allowed: 

state.items.push(product);

It looks like direct mutation, but Redux Toolkit uses Immer internally. Immer converts this simple-looking code into safe immutable updates behind the scenes.

Main Point: createSlice is used because it keeps state, reducers, and actions together, automatically creates action creators and action types, reduces boilerplate, and makes Redux code easier to read.

Step 4: Create Cart Slice

src/features/cart/cartSlice.js

import { createSlice } from '@reduxjs/toolkit';

const initialState = {
  items: [],
  totalItems: 0,
  totalAmount: 0,
  loading: false,
  error: ''
};

function calculateCartTotals(state) {
  state.totalItems = state.items.reduce((total, item) => total + item.quantity, 0);
  state.totalAmount = state.items.reduce((total, item) => total + item.price * item.quantity, 0);
}

const cartSlice = createSlice({
  name: 'cart',
  initialState,
  reducers: {
    addToCart(state, action) {
      const product = action.payload;
      const existingItem = state.items.find(item => item.id === product.id);

      if (existingItem) {
        existingItem.quantity += 1;
      } else {
        state.items.push({ ...product, quantity: 1 });
      }

      calculateCartTotals(state);
    },

    increaseQuantity(state, action) {
      const id = action.payload;
      const item = state.items.find(item => item.id === id);

      if (item) {
        item.quantity += 1;
      }

      calculateCartTotals(state);
    },

    decreaseQuantity(state, action) {
      const id = action.payload;
      const item = state.items.find(item => item.id === id);

      if (item && item.quantity > 1) {
        item.quantity -= 1;
      } else {
        state.items = state.items.filter(item => item.id !== id);
      }

      calculateCartTotals(state);
    },

    removeFromCart(state, action) {
      state.items = state.items.filter(item => item.id !== action.payload);
      calculateCartTotals(state);
    },

    clearCart(state) {
      state.items = [];
      state.totalItems = 0;
      state.totalAmount = 0;
      state.loading = false;
      state.error = '';
    },

    cartLoading(state) {
      state.loading = true;
      state.error = '';
    },

    cartError(state, action) {
      state.loading = false;
      state.error = action.payload;
    }
  }
});

export const { addToCart, increaseQuantity, decreaseQuantity, removeFromCart, clearCart, cartLoading, cartError } = cartSlice.actions;
export default cartSlice.reducer;

Why This Cart Slice Is Complete

A real cart usually needs more than only add and remove. It also needs quantity handling and clear cart logic.

State / Reducer Real Project Use
items Stores cart product list.
totalItems Stores total quantity shown in Navbar.
totalAmount Stores final cart amount shown in CartPage and CheckoutPage.
loading Used when cart data is loading from API or local storage.
error Stores cart-related error message.
addToCart Add product. If product already exists, increase quantity.
increaseQuantity User clicks plus button in cart.
decreaseQuantity User clicks minus button. If quantity becomes zero, remove item.
removeFromCart User removes product from cart.
clearCart Cart clears after successful order.
This shows two levels clearly: Redux store can have multiple reducers like cart and auth, and each reducer can have multiple state values inside its own initialState.
Now cart update logic is not written separately in ProductDetails, CartPage, WishlistPage, and CheckoutPage. All cart update logic lives in one file: cartSlice.js.

Step 5: Create Auth Slice

src/features/auth/authSlice.js

import { createSlice } from '@reduxjs/toolkit';

const initialState = {
  user: null,
  loading: false,
  error: ''
};

const authSlice = createSlice({
  name: 'auth',
  initialState,
  reducers: {
    loginStart(state) {
      state.loading = true;
      state.error = '';
    },

    loginSuccess(state, action) {
      state.loading = false;
      state.user = action.payload;
      state.error = '';
    },

    loginError(state, action) {
      state.loading = false;
      state.error = action.payload;
    },

    logout(state) {
      state.user = null;
      state.loading = false;
      state.error = '';
    }
  }
});

export const { loginStart, loginSuccess, loginError, logout } = authSlice.actions;
export default authSlice.reducer;

Auth slice stores logged-in user information, loading state, and error message.

Step 10: Login With Redux

src/components/Login.jsx

import { useState } from 'react';
import { useSelector, useDispatch } from 'react-redux';
import { loginStart, loginSuccess, loginError } from '../features/auth/authSlice';

function Login() {
  const dispatch = useDispatch();
  const { loading, error } = useSelector(state => state.auth);
  const [email, setEmail] = useState('');
  const [password, setPassword] = useState('');

  async function handleLogin(e) {
    e.preventDefault();
    dispatch(loginStart());

    try {
      const response = await fetch('/api/login', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ email, password })
      });

      const data = await response.json();
      if (!response.ok) throw new Error(data.message || 'Login failed');
      dispatch(loginSuccess(data.user));
    } catch (error) {
      dispatch(loginError(error.message));
    }
  }

  return (
    <form onSubmit={handleLogin}>
      <input value={email} onChange={(e) => setEmail(e.target.value)} placeholder="Email" />
      <input type="password" value={password} onChange={(e) => setPassword(e.target.value)} placeholder="Password" />
      {error && <p>{error}</p>}
      <button disabled={loading}>{loading ? 'Logging in...' : 'Login'}</button>
    </form>
  );
}

export default Login;

import { useSelector, useDispatch } from 'react-redux';

WishlistPage dispatch(addToCart(wishlistProduct)) cartSlice.addToCart
CartPage dispatch(increaseQuantity(id)) cartSlice.increaseQuantity
CartPage dispatch(removeFromCart(id)) cartSlice.removeFromCart
CheckoutPage dispatch(clearCart()) cartSlice.clearCart
Without Redux, these update rules may be written inside many components. With Redux, all cart rules stay in cartSlice.js.

Why Redux Is Useful Here

Problem Redux Solution
Many components need cart data Store cart in Redux store
Many components update cart Use dispatch and reducers
Cart update logic repeated in many files Keep cart logic in cartSlice.js
Navbar needs user and cart count Read both using useSelector
Checkout needs cart and user Read both from one central store
Debugging is difficult Redux DevTools can show every dispatched action

Final Simple Understanding

Redux is not mainly used because React cannot share state. React can share state using props or Context. Redux is used when shared state becomes large, many components read it, many components update it, and we need one predictable place for state and update logic.

Important Interview Points

  • Redux is used for centralized state management.
  • Redux solves prop drilling, but that is not its only purpose.
  • Redux is useful when many components need the same state.
  • Redux keeps update logic inside reducers.
  • Redux Toolkit is the modern recommended way to use Redux.
  • useSelector reads data from Redux.
  • useDispatch sends actions to Redux.
  • Redux is useful in large applications like e-commerce, banking, food delivery, and social media apps.

Short Interview Answer

Redux is a centralized state management library. It solves problems that happen when the same shared state is needed and updated by many components, such as prop drilling, scattered state, duplicated update logic, and difficult debugging. Redux keeps shared state in one store and updates it through predictable actions and reducers.

One-Line Interview Answer

Redux solves large-scale state management problems by keeping shared state in one centralized store and updating it through predictable actions.
