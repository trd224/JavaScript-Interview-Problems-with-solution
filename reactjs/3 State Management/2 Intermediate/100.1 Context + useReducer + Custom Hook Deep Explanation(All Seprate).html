1. Context Only

Problem Without Context

Imagine our application has this component structure: 

App
├── Header
│   └── UserMenu
│       └── Avatar
│
└── Dashboard

Logged-in user data is stored in App. 

const user = {
  name: 'Rizwan'
};

Now Avatar needs this user data. Without Context, we have to pass user through every component. 

App
↓
Header
↓
UserMenu
↓
Avatar
<Header user={user} />

<UserMenu user={user} />

<Avatar user={user} />
This is called prop drilling. We pass props through components that may not even need the data.

Context Solution

Context creates a shared container. 

const AuthContext = createContext();

Provide value once: 

<AuthContext.Provider value={user}>
  <App />
</AuthContext.Provider>

Read anywhere: 

const user = useContext(AuthContext);
Now Avatar can directly read the user from Context without passing props through Header and UserMenu.

Real-World Use Cases

Use Case Example State
Authentication { user: { id: 1, name: 'Rizwan', role: 'Admin' } }
Theme { theme: 'dark' }
Language { language: 'en' }
Shopping Cart { items: [] }

2. Reducer Only

A reducer is simply a state update machine.

Current State
+
Action
↓
Reducer
↓
New State

Simple Reducer Example

const state = {
  count: 0
};

const action = {
  type: 'INCREMENT'
};

function reducer(state, action) {
  switch (action.type) {
    case 'INCREMENT':
      return {
        count: state.count + 1
      };

    default:
      return state;
  }
}

Result: 

{
  count: 1
}

Real-World Authentication Reducer

Current state: 

{
  user: null,
  loading: false
}

Action: 

{
  type: 'LOGIN_START'
}

Reducer update: 

return {
  ...state,
  loading: true
};

New state: 

{
  user: null,
  loading: true
}
The reducer decides how state changes based on action type.

3. useReducer Only

useReducer is the React Hook that uses a reducer to create state and dispatch. 

const [state, dispatch] = useReducer(reducer, initialState);
Part Meaning
state Current data
dispatch Function used to send an action
reducer Function that updates state
initialState Starting state

Counter Example

const initialState = {
  count: 0
};

function counterReducer(state, action) {
  switch (action.type) {
    case 'INCREMENT':
      return {
        count: state.count + 1
      };

    default:
      return state;
  }
}
const [state, dispatch] = useReducer(counterReducer, initialState);
<button onClick={() => dispatch({ type: 'INCREMENT' })}>
  Add
</button>

↓

↓

↓

dispatch({
  type: 'LOGIN_START'

After API success: 

dispatch({

});

4. Custom Hook Only

A custom hook is reusable React logic.

A custom hook is just a JavaScript function whose name starts with use and can use React Hooks inside it.

Problem Without Custom Hook

If two components need the same counter logic, we may repeat the same useState logic in both components. 

const [count, setCount] = useState(0);

Create Custom Hook

import { useState } from 'react';

function useCounter() {
  const [count, setCount] = useState(0);

  function increment() {
    setCount(prev => prev + 1);
  }

  function decrement() {
    setCount(prev => prev - 1);
  }

  return {
    count,
    increment,
    decrement
  };
}

Use Custom Hook

const { count, increment, decrement } = useCounter();
Custom hooks help us reuse logic between components.

Real-World Custom Hook Example

For example, instead of writing window resize logic in every component, we can create: 

useWindowSize()

Then use it anywhere: 

const { width, height } = useWindowSize();

5. Context + useReducer Together

Now we combine Context and useReducer.

useReducer creates state and dispatch: 

const [state, dispatch] = useReducer(authReducer, initialState);

Context shares state and dispatch: 

<AuthContext.Provider value={{ state, dispatch }}>
  {children}
</AuthContext.Provider>
Context + useReducer gives us global state with organized update logic.

Real-World Authentication State

{
  user: null,
  loading: false,
  error: ''
}

This state can be used in:

  • Header
  • Dashboard
  • Sidebar
  • Profile Page
  • Protected Routes

Understanding children in Context Provider

Sometimes we write: 

<AuthContext.Provider value={{ state, dispatch }}>
  <App />
</AuthContext.Provider>

And sometimes we write: 

<AuthContext.Provider value={{ state, dispatch }}>
  {children}
</AuthContext.Provider>
Both are related. The difference is that <App /> is a fixed component, while {children} means whatever component is passed inside.

Case 1: Direct App Component

<AuthContext.Provider value={{ state, dispatch }}>
  <App />
</AuthContext.Provider>

Here we are directly wrapping App with AuthContext.Provider.

AuthContext.Provider
↓
App

This means only App is placed inside the Provider.

Case 2: Using children

function AuthProvider({ children }) {
  return (
    <AuthContext.Provider value={{ state, dispatch }}>
      {children}
    </AuthContext.Provider>
  );
}

Here AuthProvider is a reusable wrapper component.

children means whatever we put between <AuthProvider> and </AuthProvider>.

How React Understands children

When we write: 

<AuthProvider>
  <App />
</AuthProvider>

React internally treats it like this: 

AuthProvider({
  children: <App />
});

So inside AuthProvider: 

function AuthProvider({ children }) {
  // children = <App />
}

Then this: 

<AuthContext.Provider value={{ state, dispatch }}>
  {children}
</AuthContext.Provider>

becomes: 

<AuthContext.Provider value={{ state, dispatch }}>
  <App />
</AuthContext.Provider>

Why We Prefer children

Not Reusable

function AuthProvider() {
  return (
    <AuthContext.Provider value={{ state, dispatch }}>
      <App />
    </AuthContext.Provider>
  );
}
This is not reusable because AuthProvider can only wrap App.

Reusable

function AuthProvider({ children }) {
  return (
    <AuthContext.Provider value={{ state, dispatch }}>
      {children}
    </AuthContext.Provider>
  );
}

Now AuthProvider can wrap anything. 

<AuthProvider>
  <App />
</AuthProvider>
<AuthProvider>
  <Dashboard />
</AuthProvider>
<AuthProvider>
  <Header />
  <Sidebar />
  <Dashboard />
</AuthProvider>

Simple Box Example

Forget Context for a moment. Think about a normal Box component. 

function Box({ children }) {
  return (
    <div className="box">
      {children}
    </div>
  );
}

Usage: 

<Box>
  <h1>Hello Rizwan</h1>
</Box>

React sends this to Box: 

Box({
  children: <h1>Hello Rizwan</h1>
});

Then: 

{children}

renders: 

<h1>Hello Rizwan</h1>

Final Comparison

Code Meaning
<App /> Only App is rendered here
{children} Whatever is passed inside the component is rendered here
<AuthProvider><App /></AuthProvider> App becomes children of AuthProvider
<AuthProvider><Dashboard /></AuthProvider> Dashboard becomes children of AuthProvider

One-Line Explanation

{children} is a placeholder that renders whatever we put between the opening and closing tags of a component.

6. Context + useReducer + Custom Hook

Without Custom Hook

import { useContext } from 'react';
import { AuthContext } from '../context/AuthContext';

const { state, dispatch } = useContext(AuthContext);

With Custom Hook

function useAuth() {
  return useContext(AuthContext);
}

Now component code becomes: 

const { state, dispatch } = useAuth();
useAuth hides useContext(AuthContext) and makes component code cleaner.

7. Single File Authentication Example

In a small or medium app, we can keep Context, Reducer, Provider, and Custom Hook in one file.

context/AuthContext.jsx

import { createContext, useContext, useReducer } from 'react';

const AuthContext = createContext();

const initialState = {
  user: null,
  loading: false,
  error: ''
};

function authReducer(state, action) {
  switch (action.type) {
    case 'LOGIN_START':
      return {
        ...state,
        loading: true,
        error: ''
      };

    case 'LOGIN_SUCCESS':
      return {
        ...state,
        loading: false,
        user: action.payload
      };

    case 'LOGIN_ERROR':
      return {
        ...state,
        loading: false,
        error: action.payload
      };

    case 'LOGOUT':
      return {
        ...state,
        user: null
      };

    default:
      return state;
  }
}

export function AuthProvider({ children }) {
  const [state, dispatch] = useReducer(authReducer, initialState);

  return (
    <AuthContext.Provider value={{ state, dispatch }}>
      {children}
    </AuthContext.Provider>
  );
}

export function useAuth() {
  return useContext(AuthContext);
}
This single-file approach is good for learning, interviews, and small projects.

components/Login.jsx

import { useState } from 'react';
import { useAuth } from '../hooks/useAuth';

function Login() {
  const { state, dispatch } = useAuth();

  const [username, setUsername] = useState('');
  const [password, setPassword] = useState('');

  function handleLogin(e) {
    e.preventDefault();

    dispatch({ type: 'LOGIN_START' });

    setTimeout(() => {
      if (username === 'admin' && password === '123') {
        dispatch({
          type: 'LOGIN_SUCCESS',
          payload: {
            id: 1,
            name: 'Rizwan',
            role: 'Admin'
          }
        });
      } else {
        dispatch({
          type: 'LOGIN_ERROR',
          payload: 'Invalid Username or Password'
        });
      }
    }, 1500);
  }

  return (
    <form onSubmit={handleLogin}>
      <input value={username} onChange={(e) => setUsername(e.target.value)} placeholder="Username" />

      <input type="password" value={password} onChange={(e) => setPassword(e.target.value)} placeholder="Password" />

      {state.error && <p>{state.error}</p>}

      <button disabled={state.loading}>
        {state.loading ? 'Logging In...' : 'Login'}
      </button>
    </form>
  );
}

export default Login;

components/Dashboard.jsx

import { useAuth } from '../hooks/useAuth';

function Dashboard() {
  const { state, dispatch } = useAuth();

  return (
    <div>
      <h2>Dashboard</h2>

      <p>Welcome {state.user.name}</p>

      <p>Role: {state.user.role}</p>

      <button onClick={() => dispatch({ type: 'LOGOUT' })}>
        Logout
      </button>
    </div>
  );
}

export default Dashboard;

App.jsx

import Login from './components/Login';
import Dashboard from './components/Dashboard';
import { useAuth } from './hooks/useAuth';

function App() {
  const { state } = useAuth();

  return (
    <>
      {state.user ? <Dashboard /> : <Login />}
    </>
  );
}

export default App;

main.jsx

import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App';
import { AuthProvider } from './providers/AuthProvider';

ReactDOM.createRoot(document.getElementById('root')).render(
  <AuthProvider>
    <App />
  </AuthProvider>
);

Final Summary

Concept Main Purpose
Context Shares data globally
Reducer Controls how state changes
useReducer Creates state and dispatch
Custom Hook Reuses logic and makes code cleaner
Provider Makes state available to child components
dispatch Sends an action to the reducer

One-Line Interview Answer

Context shares global state, useReducer controls state updates through actions, and a custom hook makes that global state easy to access from any component.
