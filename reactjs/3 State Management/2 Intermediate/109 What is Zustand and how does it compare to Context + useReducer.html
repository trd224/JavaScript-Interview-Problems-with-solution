109. What is Zustand and How Does It Compare to Context + useReducer?

One-Line Definition

Zustand is a lightweight state management library for React that provides a global store without requiring Context Providers, reducers, or boilerplate code.

What Problem Does Zustand Solve?

As React applications grow, many components need access to the same shared state.

Examples:

  • User information
  • Shopping cart
  • Theme settings
  • Notifications
  • Authentication state

React provides Context and useReducer to solve this problem.

However, Context + useReducer can become verbose in large applications.

Zustand was created to provide global state management with much less code than Context + useReducer or Redux.

Context + useReducer Approach

To create global state using React only, we usually need:

  • Create Context
  • Create Reducer
  • Create Provider
  • Wrap App with Provider
  • Create Custom Hook

Step 1 - Create Context

import { createContext } from 'react';

export const CartContext = createContext();

Step 2 - Create Reducer

function cartReducer(state, action) {

  switch(action.type) {

    case 'ADD_TO_CART':
      return {
        ...state,
        items: [...state.items, action.payload]
      };

    default:
      return state;

  }

}

Step 3 - Create Provider

export function CartProvider({ children }) {

  const [state, dispatch] = useReducer(
    cartReducer,
    { items: [] }
  );

  return (
    
      {children}
    
  );

}

Step 4 - Wrap App

<CartProvider>
  <App />
</CartProvider>

Step 5 - Consume Context

const { state, dispatch } =
  useContext(CartContext);
Context + useReducer works very well, but it requires several setup steps.

Zustand Approach

With Zustand, everything can live in a single file.

cartStore.js

import { create } from 'zustand';

const useCartStore = create(set => ({

  items: [],

  addToCart: product =>
    set(state => ({
      items: [...state.items, product]
    }))

}));

export default useCartStore;
No Context.
No Provider.
No useReducer.
No dispatch.

Using Zustand State

ProductCard.jsx

import useCartStore from './cartStore';

function ProductCard({ product }) {

  const addToCart =
    useCartStore(state => state.addToCart);

  return (
    <button
      onClick={() => addToCart(product)}
    >
      Add To Cart
    </button>
  );
}

Navbar.jsx

import useCartStore from './cartStore';

function Navbar() {

  const items =
    useCartStore(state => state.items);

  return (
    <h3>Cart Items: {items.length}</h3>
  );
}

How Zustand Updates State

Redux uses: 

dispatch(addToCart(product));

Context + useReducer uses: 

dispatch({
  type: 'ADD_TO_CART',
  payload: product
});

Zustand uses: 

addToCart(product);
Zustand directly calls store functions instead of dispatching actions.

Performance Difference

Context Problem

When Context value changes, every component consuming that Context may re-render. 

Context Updated
      ↓
All Consumers Re-render

Zustand

Zustand subscribes only to the selected state. 

const items =
  useCartStore(state => state.items);

Only components using items will update.

Zustand usually causes fewer unnecessary re-renders than Context.

When Should You Use Zustand?

  • Global state is needed.
  • You want less boilerplate.
  • You do not want Redux complexity.
  • You want better performance than large Context trees.
  • You want a simple API.

Interview Answer

Zustand is a lightweight React state management library that provides global state without requiring Context Providers, reducers, or dispatch functions. Compared to Context + useReducer, Zustand requires less boilerplate, offers simpler APIs, and often provides better performance through selective subscriptions.

One-Line Interview Answer

Zustand is a lightweight global state management library that achieves the same goal as Context + useReducer but with less code and simpler state updates.
