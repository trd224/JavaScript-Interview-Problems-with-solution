100. How do you combine Context + useReducer for global state?

For more understanding see example 100.1 and 100.2

One-Line Definition

Context + useReducer is a React pattern where Context shares state globally and useReducer manages complex state updates through actions.

Why Combine Context and useReducer?

Context alone only shares data. It does not organize complex update logic.

useReducer organizes state updates, but by itself the state is local to one component.

Context shares the state globally, and useReducer controls how the state changes.

Simple Flow

useReducer
↓
Creates state and dispatch
↓
Context Provider
↓
Shares state and dispatch
↓
Any Child Component Can Use Them

When Should We Use This Pattern?

  • Authentication state
  • Logged-in user details
  • Theme state
  • Language state
  • Cart state
  • Application settings
  • Small to medium global state management

Step 1: Create Context and Reducer

context/AppContext.jsx

import { createContext, useContext, useReducer } from 'react';

const AppContext = createContext();

const initialState = {
  user: null,
  theme: 'dark',
  loading: false,
  error: ''
};

function appReducer(state, action) {
  switch (action.type) {
    case 'LOGIN_START':
      return { ...state, loading: true, error: '' };

    case 'LOGIN_SUCCESS':
      return { ...state, user: action.payload, loading: false };

    case 'LOGIN_ERROR':
      return { ...state, loading: false, error: action.message };

    case 'LOGOUT':
      return { ...state, user: null };

    case 'TOGGLE_THEME':
      return { ...state, theme: state.theme === 'dark' ? 'light' : 'dark' };

    default:
      return state;
  }
}

export function AppProvider({ children }) {
  const [state, dispatch] = useReducer(appReducer, initialState);

  return (
    <AppContext.Provider value={{ state, dispatch }}>
      {children}
    </AppContext.Provider>
  );
}

export function useAppContext() {
  return useContext(AppContext);
}

What Is Happening Here?

Part Purpose
createContext() Creates global context
initialState Stores initial global state
appReducer() Handles actions and returns new state
useReducer() Creates state and dispatch
AppProvider Shares state and dispatch globally
useAppContext() Custom hook to consume global state

Step 2: Wrap App with Provider

main.jsx

import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App';
import { AppProvider } from './context/AppContext';

ReactDOM.createRoot(document.getElementById('root')).render(
  <AppProvider>
    <App />
  </AppProvider>
);

Now every component inside App can access global state and dispatch.

Step 4: Update Global State Using dispatch

components/LoginButton.jsx

import { useAppContext } from '../context/AppContext';

function LoginButton() {
  const { state, dispatch } = useAppContext();

  const handleLogin = () => {
    dispatch({ type: 'LOGIN_START' });

    setTimeout(() => {
      dispatch({ type: 'LOGIN_SUCCESS', payload: { id: 1, name: 'Rizwan' } });
    }, 1000);
  };

  return (
    <button onClick={handleLogin} disabled={state.loading}>
      {state.loading ? 'Logging in...' : 'Login'}
    </button>
  );
}

export default LoginButton;

Step 5: Toggle Theme from Any Component

components/ThemeButton.jsx

import { useAppContext } from '../context/AppContext';

function ThemeButton() {
  const { state, dispatch } = useAppContext();

  return (
    <button onClick={() => dispatch({ type: 'TOGGLE_THEME' })}>
      Current Theme: {state.theme}
    </button>
  );
}

export default ThemeButton;

Real-World Authentication Example (Recommended)

In real projects, we usually create Contexts based on business domains instead of generic names like AppContext.

Authentication is one of the most common use cases for combining Context and useReducer.

├── App.jsx

AuthContext.jsx

import { createContext, useContext, useReducer } from 'react';

const AuthContext = createContext();

const initialState = {
  user: null,
  loading: false,
  error: ''
};

function authReducer(state, action) {
  switch (action.type) {
    case 'LOGIN_START':
      return {
        ...state,
        loading: true,
        error: ''
      };

    case 'LOGIN_SUCCESS':
      return {
        ...state,
        loading: false,
        user: action.payload
      };

    case 'LOGIN_ERROR':
      return {
        ...state,
        loading: false,
        error: action.payload
      };

    case 'LOGOUT':
      return {
        ...state,
        user: null
      };

    default:
      return state;
  }
}

export function AuthProvider({ children }) {
  const [state, dispatch] = useReducer(authReducer, initialState);

  return (
    <AuthContext.Provider value={{ state, dispatch }}>
      {children}
    </AuthContext.Provider>
  );
}

export function useAuth() {
  return useContext(AuthContext);
}

Login.jsx

import { useState } from 'react';
import { useAuth } from '../context/AuthContext';

function Login() {
  const { state, dispatch } = useAuth();

  const [username, setUsername] = useState('');
  const [password, setPassword] = useState('');

  function handleSubmit(e) {
    e.preventDefault();

    dispatch({ type: 'LOGIN_START' });

    setTimeout(() => {
      if (username === 'admin' && password === '123') {
        dispatch({
          type: 'LOGIN_SUCCESS',
          payload: {
            id: 1,
            name: 'Rizwan',
            role: 'Admin'
          }
        });
      } else {
        dispatch({
          type: 'LOGIN_ERROR',
          payload: 'Invalid Username or Password'
        });
      }
    }, 1500);
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        value={username}
        onChange={(e) => setUsername(e.target.value)}
        placeholder="Username"
      />

      <input
        type="password"
        value={password}
        onChange={(e) => setPassword(e.target.value)}
        placeholder="Password"
      />

      {state.error && <p>{state.error}</p>}

      <button disabled={state.loading}>
        {state.loading ? 'Logging In...' : 'Login'}
      </button>
    </form>
  );
}

export default Login;

Dashboard.jsx

import { useAuth } from '../context/AuthContext';

function Dashboard() {
  const { state, dispatch } = useAuth();

  return (
    <div>
      <h2>Dashboard</h2>

      <p>Welcome {state.user.name}</p>
      <p>Role: {state.user.role}</p>

      <button onClick={() => dispatch({ type: 'LOGOUT' })}>
        Logout
      </button>
    </div>
  );
}

export default Dashboard;

App.jsx

import { useAuth } from './context/AuthContext';
import Login from './components/Login';
import Dashboard from './components/Dashboard';

function App() {
  const { state } = useAuth();

  return (
    <div>
      {state.user ? <Dashboard /> : <Login />}
    </div>
  );
}

export default App;
This example is closer to what you will see in production React applications. Instead of generic names like AppContext and AppProvider, we use AuthContext, AuthProvider, and useAuth because the Context manages authentication-related state.

Shares values Shares values and update logic
Good for simple data Good for complex global state
Update logic can become scattered Update logic stays inside reducer
No action-based flow Uses dispatch and actions

Real Project Example

In our Aadhaar Authentication application, Context + useReducer can manage global data like logged-in user details, department information, selected theme, authentication session, and permissions. Context makes this data available everywhere, while useReducer keeps updates predictable through actions like LOGIN_SUCCESS, LOGOUT, SET_DEPARTMENT, and SET_ERROR.

Important Interview Points

  • Context provides global access to state.
  • useReducer manages complex state updates.
  • The Provider wraps the application.
  • Components consume state using useContext or a custom hook.
  • Components update state using dispatch.
  • This pattern can replace Redux in small to medium apps.
  • Redux is still better for large apps with advanced global state needs.

Short Interview Answer

We combine Context and useReducer by creating a context, managing state with useReducer inside a Provider, and passing state and dispatch through the context value. Any child component can then read global state and update it using dispatch.

One-Line Interview Answer

Context shares global state, while useReducer controls how that state changes through actions.
