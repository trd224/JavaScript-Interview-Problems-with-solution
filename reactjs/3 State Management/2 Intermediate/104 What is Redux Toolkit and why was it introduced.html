104. What is Redux Toolkit and Why Was It Introduced?

One-Line Definition

Redux Toolkit (RTK) is the official recommended way to write Redux code. It simplifies Redux setup and removes most of the boilerplate code required in traditional Redux.

What Problem Did Traditional Redux Have?

Redux itself is powerful, but developers often complained that writing Redux required too much code.

Traditional Redux Required Many Files

redux/
│
├── actions/
│   └── cartActions.js
│
├── reducers/
│   └── cartReducer.js
│
├── constants/
│   └── actionTypes.js
│
└── store.js

Even a simple Add To Cart feature required multiple files.

Step 1 - Create Action Types

const ADD_TO_CART = 'ADD_TO_CART';
const REMOVE_FROM_CART = 'REMOVE_FROM_CART';

Step 2 - Create Action Creators

export const addToCart = product => ({
  type: ADD_TO_CART,
  payload: product
});

Step 3 - Create Reducer

const initialState = {
  items: []
};

function cartReducer(state = initialState, action) {

  switch (action.type) {

    case ADD_TO_CART:
      return {
        ...state,
        items: [...state.items, action.payload]
      };

    default:
      return state;
  }

}

Step 4 - Configure Store

import { createStore } from 'redux';

const store = createStore(cartReducer);
A simple feature required action types, action creators, reducers, switch statements, and store configuration.

Why Redux Toolkit Was Introduced

The Redux team introduced Redux Toolkit to solve the most common Redux problems.

Problem Redux Toolkit Solution
Too much boilerplate createSlice()
Complex store setup configureStore()
Manual immutable updates Immer integration
Too many files Actions and reducers together
Async logic complexity createAsyncThunk()

Redux Toolkit Approach

Instead of creating action types, action creators, and reducers separately, Redux Toolkit combines them into one slice.

Single Slice Example

import { createSlice } from '@reduxjs/toolkit';

const initialState = {
  items: []
};

const cartSlice = createSlice({

  name: 'cart',

  initialState,

  reducers: {

    addToCart(state, action) {
      state.items.push(action.payload);
    },

    removeFromCart(state, action) {
      state.items = state.items.filter(
        item => item.id !== action.payload
      );
    }

  }

});

export const {
  addToCart,
  removeFromCart
} = cartSlice.actions;

export default cartSlice.reducer;
One file automatically creates action types, action creators, and reducers.

What Does createSlice() Automatically Generate?

When Redux Toolkit sees: 

reducers: {

  addToCart(state, action) {},

  removeFromCart(state, action) {}

}

It automatically generates: 

Action Type:
cart/addToCart

Action Creator:
addToCart()

Reducer Case:
addToCart reducer logic

You no longer write those manually.

Store Configuration Becomes Easier

Traditional Redux

import { createStore } from 'redux';

const store = createStore(rootReducer);

Redux Toolkit

import { configureStore } from '@reduxjs/toolkit';

const store = configureStore({

  reducer: {
    cart: cartReducer,
    user: userReducer
  }

});
configureStore automatically adds Redux DevTools support and useful middleware.

Immer Makes State Updates Easier

Traditional Redux required manual immutable updates.

Traditional Redux

return {
  ...state,
  items: [...state.items, action.payload]
};

Redux Toolkit

state.items.push(action.payload);

This looks like mutation but Redux Toolkit uses Immer internally and safely creates a new immutable state behind the scenes.

Redux state is still immutable. Redux Toolkit simply makes immutable updates easier to write.

Why Almost Every Modern Redux Project Uses Redux Toolkit

  • Less code
  • Less boilerplate
  • Simpler store setup
  • Built-in Immer support
  • Built-in Redux DevTools support
  • Official Redux recommendation
  • Easier maintenance in large applications
Today, when developers say "Redux", they almost always mean Redux Toolkit because it is the officially recommended way to use Redux.

Interview Answer

Redux Toolkit is the official Redux package that simplifies Redux development by reducing boilerplate code, simplifying store configuration, automatically generating actions and reducers, and providing built-in support for immutable state updates through Immer.

One-Line Interview Answer

Redux Toolkit is the official recommended way to write Redux because it makes Redux simpler, cleaner, and requires much less code than traditional Redux.
