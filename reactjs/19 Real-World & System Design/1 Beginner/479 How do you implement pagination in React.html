479. How Do You Implement Pagination in React?

One-Line Definition

Pagination in React means dividing a large list of data into smaller pages and showing only one page of records at a time.

Why Do We Use Pagination?

  • To avoid showing too many records at once.
  • To improve page performance.
  • To make large lists easier to read.
  • To reduce API and UI load.

Pagination Flow

Full Data List
↓
Current Page
↓
Items Per Page
↓
Slice Data
↓
Show Current Page Data

Complete React Pagination Example

import { useState } from 'react';

function UserList() {
  const users = [
    'Rizwan', 'Amit', 'Rahul', 'Sohan', 'Imran',
    'Karan', 'Anil', 'Deepak', 'Mohit', 'Ravi',
    'Salman', 'Akash'
  ];

  const [currentPage, setCurrentPage] = useState(1);
  const itemsPerPage = 5;

  const lastIndex = currentPage * itemsPerPage;
  const firstIndex = lastIndex - itemsPerPage;
  const currentUsers = users.slice(firstIndex, lastIndex);

  const totalPages = Math.ceil(users.length / itemsPerPage);

  function goToNextPage() {
    if (currentPage < totalPages) setCurrentPage(currentPage + 1);
  }

  function goToPreviousPage() {
    if (currentPage > 1) setCurrentPage(currentPage - 1);
  }

  return (
    <div>
      <h2>User List</h2>

      <ul>
        {currentUsers.map((user, index) => <li key={index}>{user}</li>)}
      </ul>

      <button onClick={goToPreviousPage} disabled={currentPage === 1}>Previous</button>

      <span> Page {currentPage} of {totalPages} </span>

      <button onClick={goToNextPage} disabled={currentPage === totalPages}>Next</button>
    </div>
  );
}

export default UserList;

How the Calculation Works

Formula Meaning
lastIndex = currentPage * itemsPerPage Finds the ending index of current page
firstIndex = lastIndex - itemsPerPage Finds the starting index of current page
slice(firstIndex, lastIndex) Returns only current page records

Example Calculation

If currentPage = 2 and itemsPerPage = 5:

lastIndex = 2 × 5 = 10
firstIndex = 10 - 5 = 5

So React shows records from index 5 to index 9.

Page Number Buttons Example

const pageNumbers = [];

for (let i = 1; i <= totalPages; i++) {
  pageNumbers.push(i);
}

<div>
  {pageNumbers.map(page => (
    <button key={page} onClick={() => setCurrentPage(page)} disabled={currentPage === page}>
      {page}
    </button>
  ))}
</div>

Client-Side vs Server-Side Pagination

Client-Side Pagination Server-Side Pagination
All data is already loaded in frontend Only current page data is fetched from API
Good for small data Good for large data
Uses slice() Uses API query params like page and limit

Server-Side Pagination API Example

fetch(`/api/users?page=${currentPage}&limit=${itemsPerPage}`)
  .then(res => res.json())
  .then(data => console.log(data));

Important Interview Points

  • Pagination divides large data into smaller pages.
  • useState stores the current page number.
  • slice() is used for client-side pagination.
  • Server-side pagination is better for large datasets.
  • Previous and Next buttons update the current page.

Short Interview Answer

Pagination in React is implemented by storing the current page in state, calculating start and end indexes, slicing the data, and rendering only the current page records.

One-Line Interview Answer

React pagination is implemented by using current page state and displaying only a sliced portion of the data.
