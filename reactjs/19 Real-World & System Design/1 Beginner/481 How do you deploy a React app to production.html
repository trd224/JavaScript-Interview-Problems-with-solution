481. How do you deploy a React app to production?

One-Line Definition

Deploying a React app to production means creating an optimized build and uploading it to a server or hosting platform so users can access it through a live URL.

Simple Flow

Write React Code
↓
Create Production Build
↓
Upload Build Files
↓
Configure Server
↓
React App Goes Live

Step 1: Set Production Environment Variables

Before creating the production build, we should make sure production environment variables are correct.

CRA Example

.env.production

REACT_APP_API_URL=https://api.myapp.com

Vite Example

.env.production

VITE_API_URL=https://api.myapp.com

Step 2: Create Production Build

Create React App

npm run build

This creates a build folder.

Vite React

npm run build

This creates a dist folder.

What Happens During Build?

React converts our application into optimized static files like HTML, CSS, and JavaScript.
  • JavaScript is minified.
  • CSS is optimized.
  • Unused code is removed where possible.
  • Files are prepared for fast browser loading.

Build Output

Tool Build Command Output Folder
Create React App npm run build build
Vite npm run build dist

Step 3: Test Production Build Locally

Before deployment, we can test the production build locally.

CRA

npx serve -s build

Vite

npm run preview

Step 4: Deploy Build Folder

After the build is created, we upload the build output folder to a hosting platform.

Platform What To Upload / Configure
Netlify Upload build/dist folder or connect GitHub repo
Vercel Connect GitHub repo and deploy automatically
GitHub Pages Deploy static build files
AWS S3 Upload build/dist folder and enable static website hosting
Nginx / Apache Place build/dist files inside server root folder

Example: Deploy on Nginx Server

For CRA, copy the build folder content to the server. 

npm run build

scp -r build/* user@server:/var/www/my-react-app

Nginx Configuration

server {
  listen 80;
  server_name myapp.com;

  root /var/www/my-react-app;
  index index.html;

  location / {
    try_files $uri /index.html;
  }
}
The try_files $uri /index.html; line is important for React Router. It sends all routes back to index.html.

Why React Router Needs Server Configuration?

React is a Single Page Application. Routes like /dashboard or /profile are handled by React in the browser.

If the user refreshes /dashboard, the server may look for a real /dashboard file and show 404.

User opens /dashboard
↓
Server sends index.html
↓
React Router displays Dashboard component

Important Deployment Checklist

  • Use correct production API URL.
  • Run npm run build.
  • Test the build locally.
  • Upload build folder for CRA.
  • Upload dist folder for Vite.
  • Configure fallback to index.html for React Router.
  • Enable HTTPS in production.
  • Do not store secret keys in frontend environment files.

Real Project Example

In our Aadhaar Authentication React application, we first set production API URLs in .env.production, then run npm run build, and finally deploy the generated build files to the production server.

Short Interview Answer

To deploy a React app to production, we set production environment variables, run npm run build, test the generated build, and upload the optimized static files to a hosting platform like Netlify, Vercel, AWS S3, or an Nginx/Apache server.

One-Line Interview Answer

A React app is deployed by creating an optimized production build using npm run build and hosting the generated static files on a production server or hosting platform.
