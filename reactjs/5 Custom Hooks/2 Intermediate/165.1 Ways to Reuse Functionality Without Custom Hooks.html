Ways to Reuse Functionality Without Custom Hooks

Question

What are other ways to reuse the same functionality across multiple React components without using a Custom Hook?

1. Utility Functions

Utility functions are used to reuse pure JavaScript logic. 

function calculateTax(amount) {
  return amount * 0.18;
}

Used for:

  • Calculations
  • Formatting
  • Validation
  • Helper Functions
Utility functions cannot use React hooks like useState or useEffect.

2. Services

Services are used to reuse business logic and API calls. 

// userService.js

export async function getUsers() {

  const response =
    await fetch('/api/users');

  return response.json();

}

Used for:

  • API Calls
  • Business Logic
  • Data Transformation
  • Local Storage Access
  • Authentication Logic
Services are one of the most common patterns used in large React applications.

3. Context API

Context API allows us to share state across multiple components. 

<UserContext.Provider
  value={user}
>
  {children}
</UserContext.Provider>

Usage: 

const user =
  useContext(UserContext);

Used for:

  • Authentication
  • Theme
  • Language
  • Shopping Cart
  • Global State

4. Reusable Components

Sometimes functionality can be reused through components. 

<Button variant="primary" />

<Button variant="danger" />

Used for:

  • Buttons
  • Forms
  • Tables
  • Cards
  • Modals
Reusable components allow us to reuse both UI and behavior.

5. Higher Order Components (HOC)

A Higher Order Component wraps another component and adds functionality. 

function withLoading(Component) {

  return function(props) {

    if (props.loading) {
      return <p>Loading...</p>;
    }

    return <Component {...props} />;

  };

}

Usage: 

const UserPage =
  withLoading(UserPage);
HOCs were very popular before React Hooks were introduced.

6. Render Props

Render Props allow components to share logic through a function prop. 

<DataFetcher
  render={(data) => (
    <UserList data={data} />
  )}
/>
Render Props were also commonly used before Hooks became available.

Comparison Table

Technique Purpose Can Use Hooks?
Utility Function Reuse JavaScript Logic ❌ No
Service Reuse Business Logic & API Calls ❌ No
Context API Share State ✅ Yes
Reusable Component Reuse UI + Behavior ✅ Yes
HOC Reuse Component Logic ✅ Yes
Render Props Reuse Component Logic ✅ Yes
Custom Hook Reuse Hook Logic ✅ Yes

In modern React applications, Services and Custom Hooks usually work together rather than replacing each other.

Interview Answer

Without custom hooks, functionality can be reused using utility functions, services, Context API, reusable components, Higher Order Components (HOCs), and Render Props. Modern React applications most commonly use services for business logic, Context for shared state, and reusable components for UI reuse.

One-Line Interview Answer

Besides custom hooks, React functionality can be reused through services, utility functions, Context API, reusable components, HOCs, and Render Props.
