167. Write a Custom Hook useLocalStorage

One-Line Definition

useLocalStorage is a custom hook that works like useState, but it also saves the value in browser localStorage.

Why Do We Need It?

Normal React state is temporary. When the page refreshes, the state becomes default again. 

const [theme, setTheme] = useState('light');
theme = light
↓
User changes theme to dark
↓
theme = dark
↓
Refresh page
↓
theme becomes light again
This happens because useState stores data only in React memory.

What localStorage Does

localStorage saves data in the browser. So even after refresh, the value is still available. 

localStorage.setItem('theme', 'dark');
localStorage remembers the value even after page refresh.

Create useLocalStorage Hook

import { useState, useEffect } from 'react';

function useLocalStorage(key, initialValue) {

  const [value, setValue] = useState(() => {

    const storedValue = localStorage.getItem(key);

    return storedValue ? JSON.parse(storedValue) : initialValue;

  });

  useEffect(() => {

    localStorage.setItem(key, JSON.stringify(value));

  }, [key, value]);

  return [value, setValue];

}

export default useLocalStorage;

Using useLocalStorage

import useLocalStorage from './useLocalStorage';

function App() {

  const [theme, setTheme] = useLocalStorage('theme', 'light');

  return (
    <button onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}>
      Current Theme: {theme}
    </button>
  );

}

export default App;

Understand This Line

const [theme, setTheme] = useLocalStorage('theme', 'light');

Read this line like this:

Hey useLocalStorage hook, create one state for theme. The localStorage key name is theme. If no old value exists, use light as default value.

Step 1: App Component Starts

useLocalStorage('theme', 'light');

So inside the hook: 

key = 'theme'
initialValue = 'light'

Step 2: First Time Page Loads

Suppose localStorage is empty. 

localStorage = {}

This code runs: 

const storedValue = localStorage.getItem(key);

Because key is theme, it becomes: 

const storedValue = localStorage.getItem('theme');

Nothing is saved yet, so: 

storedValue = null

Step 3: Initial Value Is Used

return storedValue ? JSON.parse(storedValue) : initialValue;

Now it becomes: 

return null ? JSON.parse(null) : 'light';

So result is: 

return 'light';
Now React state value becomes light. 
value = 'light'

Step 4: useEffect Saves Value

useEffect(() => {
  localStorage.setItem(key, JSON.stringify(value));
}, [key, value]);

Current values are: 

key = 'theme'
value = 'light'

So this runs: 

localStorage.setItem('theme', JSON.stringify('light'));

Browser stores: 

theme = "light"

Step 5: Hook Returns Value

return [value, setValue];

At this time: 

value = 'light'

So hook returns: 

return ['light', setValue];

Step 6: App Receives It

const [theme, setTheme] = useLocalStorage('theme', 'light');

Actually becomes: 

const [theme, setTheme] = ['light', setValue];

So: 

theme = 'light'
setTheme = setValue
Now button shows Current Theme: light

Step 7: User Clicks Button

<button onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}>
  Current Theme: {theme}
</button>

Current theme is: 

theme = 'light'

So this condition: 

theme === 'light' ? 'dark' : 'light'

returns: 

'dark'

So this runs: 

setTheme('dark');

Step 8: State Changes

Because setTheme is actually setValue, value changes: 

value = 'dark'

Step 9: useEffect Runs Again

Because value changed from light to dark, useEffect runs again. 

localStorage.setItem('theme', JSON.stringify('dark'));

Browser storage becomes: 

theme = "dark"
Now button shows Current Theme: dark

Step 10: Page Refresh

After refresh, App starts again. 

useLocalStorage('theme', 'light');

Now localStorage already has: 

theme = "dark"

So this line: 

const storedValue = localStorage.getItem('theme');

returns: 

storedValue = '"dark"'

Now this condition runs: 

return storedValue ? JSON.parse(storedValue) : initialValue;

Because storedValue exists, it becomes: 

return JSON.parse('"dark"');

Result: 

return 'dark';
After refresh, theme is still dark.

Complete Flow

First Load
↓
localStorage empty
↓
theme = light
↓
User clicks button
↓
theme = dark
↓
useEffect saves dark in localStorage
↓
Refresh page
↓
Hook reads dark from localStorage
↓
theme = dark

Why JSON.stringify?

localStorage stores only strings. 

localStorage.setItem('theme', JSON.stringify('dark'));

It saves: 

"dark"

Why JSON.parse?

When we read from localStorage, we convert the stored string back to real JavaScript value. 

JSON.parse('"dark"')

Result: 

'dark'

useState vs useLocalStorage

Hook What It Does
useState Stores value in React memory only.
useLocalStorage Stores value in React memory and browser localStorage.

Important Point

useLocalStorage behaves like useState. The difference is that it automatically reads from localStorage first and saves changes back to localStorage.

Easy Interview Answer

useLocalStorage is a custom React hook that stores state in localStorage so the value is not lost after page refresh.

One-Line Interview Answer

useLocalStorage is like useState with browser persistence.
