166. Write a Custom Hook useFetch for Data Fetching

One-Line Definition

useFetch is a custom hook that reusable API fetching logic using useState and useEffect.

Why Do We Need useFetch?

Many components need API data.

Without a custom hook, every component repeats the same logic:

  • data state
  • loading state
  • error state
  • useEffect
  • fetch call
  • try/catch
If the same API loading and error logic is repeated in many components, we can move it into a custom hook.

Basic useFetch Hook

import { useState, useEffect } from 'react';

function useFetch(url) {

  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {

    async function fetchData() {

      try {
        setLoading(true);
        setError(null);

        const response = await fetch(url);

        if (!response.ok) {
          throw new Error('Failed to fetch data');
        }

        const result = await response.json();

        setData(result);
      } catch (err) {
        setError(err.message);
      } finally {
        setLoading(false);
      }

    }

    fetchData();

  }, [url]);

  return { data, loading, error };

}

export default useFetch;

How to Use useFetch

import useFetch from './useFetch';

function Users() {

  const { data: users, loading, error } = useFetch('https://jsonplaceholder.typicode.com/users');

  if (loading) {
    return <p>Loading...</p>;
  }

  if (error) {
    return <p>Error: {error}</p>;
  }

  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );

}

export default Users;

Important Point

Custom hooks reuse logic, not state.

If two components call useFetch, both get their own independent state. 

const users = useFetch('/api/users');

const products = useFetch('/api/products');
Each call has its own data, loading, and error state.

But In Real Projects, We Often Use Service + Custom Hook

In professional applications, we usually keep API calls inside a service file.

Service handles API logic. Custom hook handles React state and lifecycle.

Step 1 - Service File

// services/userService.js

export async function getUsers() {

  const response = await fetch('/api/users');

  if (!response.ok) {
    throw new Error('Failed to fetch users');
  }

  return response.json();

}

Step 2 - Custom Hook Uses Service

// hooks/useUsers.js

import { useState, useEffect } from 'react';
import { getUsers } from '../services/userService';

function useUsers() {

  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {

    async function loadUsers() {

      try {
        setLoading(true);
        setError(null);

        const data = await getUsers();

        setUsers(data);
      } catch (err) {
        setError(err.message);
      } finally {
        setLoading(false);
      }

    }

    loadUsers();

  }, []);

  return { users, loading, error };

}

export default useUsers;

Step 3 - Component Uses Custom Hook

import useUsers from './useUsers';

function UsersPage() {

  const { users, loading, error } = useUsers();

  if (loading) {
    return <p>Loading...</p>;
  }

  if (error) {
    return <p>Error: {error}</p>;
  }

  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );

}

export default UsersPage;

When Should You Write useFetch?

  • When many components need the same data-fetching pattern.
  • When loading logic is repeated.
  • When error handling is repeated.
  • When API calls follow a common pattern.

When Should You NOT Write useFetch?

  • When the API call is used only once.
  • When every API has very different logic.
  • When using React Query or SWR.
  • When the logic belongs only in a service.
