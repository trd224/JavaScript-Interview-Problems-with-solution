176. How do you type a custom hook return value with TypeScript?

One-Line Definition

In TypeScript, we can explicitly define the type of data returned by a custom hook so TypeScript knows exactly what values and functions the hook provides.

Why Is It Needed?

Consider this custom hook: 

function useCounter() {

  const [count, setCount] = useState(0);

  function increment() {
    setCount(c => c + 1);
  }

  return {
    count,
    increment
  };

}

The hook returns: 

{
  count,
  increment
}

TypeScript tries to infer the type automatically.

But in larger applications, explicitly defining the return type makes the code easier to understand and safer.

Typing the return value clearly documents what the hook returns.

Method 1 - Type the Return Object

Create Return Type

type UseCounterReturn = {
  count: number;
  increment: () => void;
};

Apply It To The Hook

function useCounter(): UseCounterReturn {

  const [count, setCount] = useState(0);

  function increment() {
    setCount(c => c + 1);
  }

  return {
    count,
    increment
  };

}
Now TypeScript knows exactly what the hook returns.

How TypeScript Reads This

function useCounter(): UseCounterReturn

Meaning: 

This function must return:

{
  count: number;
  increment: () => void;
}

Otherwise TypeScript will show an error.

Using The Hook

const { count, increment } = useCounter();

TypeScript automatically knows: 

count      // number

increment  // function

Method 2 - Return a Tuple

Some hooks return arrays instead of objects.

Example: 

return [count, increment];

Type The Tuple

type UseCounterReturn = [
  number,
  () => void
];

Apply It

function useCounter(): UseCounterReturn {

  const [count, setCount] = useState(0);

  function increment() {
    setCount(c => c + 1);
  }

  return [count, increment];

}

How TypeScript Reads This

[
  number,
  () => void
]

Meaning: 

Index 0 -> number

Index 1 -> function

Usage: 

const [count, increment] = useCounter();

Real-World Example

type UseAuthReturn = {
  user: string | null;
  login: () => void;
  logout: () => void;
};

function useAuth(): UseAuthReturn {

  const user = 'Rizwan';

  function login() {}

  function logout() {}

  return {
    user,
    login,
    logout
  };

}

Now every developer immediately knows what the hook provides.

Why Not Let TypeScript Infer Everything?

TypeScript can often infer the return type automatically. 

function useCounter() {

  return {
    count: 0,
    increment: () => {}
  };

}

This works.

However explicit return types:

  • Improve readability.
  • Act as documentation.
  • Prevent accidental return changes.
  • Make large codebases easier to maintain.
Small projects often rely on inference. Large projects usually define explicit return types.

Important Interview Points

  • Custom hooks can return objects or tuples.
  • Return types can be explicitly defined.
  • Type aliases are commonly used.
  • Tuple types are used for array returns.
  • TypeScript can infer types but explicit types improve maintainability.

Short Interview Answer

A custom hook return value can be typed by defining a return type or interface and applying it to the hook function's return type.

One-Line Interview Answer

Type a custom hook return value by defining the shape of the returned object or tuple and using it as the hook's return type.
