165. What is a Custom Hook and When Should You Write One?

One-Line Definition

A custom hook is a JavaScript function that uses React hooks and allows us to reuse stateful logic across multiple components.

Why Do We Need Custom Hooks?

Many components often need the same logic.

Examples:

  • API fetching logic
  • Window resize tracking
  • Authentication logic
  • Form handling logic
  • Local storage logic
  • Online/offline detection

Without a custom hook, we would duplicate the same useState and useEffect code in multiple components.

Custom hooks help us avoid duplicated React hook logic.

The Problem Without a Custom Hook

Suppose two components need the current window width.

Component A

useEffect(() => {

  function handleResize() {
    setWidth(window.innerWidth);
  }

  window.addEventListener('resize', handleResize);

  return () => {
    window.removeEventListener('resize', handleResize);
  };

}, []);

Component B

useEffect(() => {

  function handleResize() {
    setWidth(window.innerWidth);
  }

  window.addEventListener('resize', handleResize);

  return () => {
    window.removeEventListener('resize', handleResize);
  };

}, []);

The same logic is repeated.

A custom hook allows us to write the logic once and reuse it everywhere.

Basic Syntax

function useSomething() {

  // React hook logic

  return something;

}

Custom hook names must always start with: 

use

Examples: 

useAuth()
useFetch()
useCounter()
useWindowSize()
useLocalStorage()

Simple Custom Hook Example

Step 1 - Create Custom Hook

import { useState } from 'react';

function useCounter() {

  const [count, setCount] = useState(0);

  function increment() {
    setCount(c => c + 1);
  }

  return { count, increment };

}

export default useCounter;

Step 2 - Use Custom Hook

import useCounter from './useCounter';

function App() {

  const { count, increment } = useCounter();

  return (
    <>
      <h1>{count}</h1>

      <button onClick={increment}>
        Increment
      </button>
    </>
  );

}

export default App;
The component receives state and functions from the custom hook.

Real Project Example - Window Width Hook

Create Hook

import { useState, useEffect } from 'react';

function useWindowWidth() {

  const [width, setWidth] = useState(window.innerWidth);

  useEffect(() => {

    function handleResize() {
      setWidth(window.innerWidth);
    }

    window.addEventListener('resize', handleResize);

    return () => {
      window.removeEventListener('resize', handleResize);
    };

  }, []);

  return width;

}

export default useWindowWidth;

Use Hook

import useWindowWidth from './useWindowWidth';

function App() {

  const width = useWindowWidth();

  return 
Width: {width}
;

}
All resize logic is hidden inside the custom hook.

Important Understanding

Custom hooks reuse logic, not state.

Example

const counter1 = useCounter();

const counter2 = useCounter();

Each hook call creates a separate state.

useCounter()
↓
Own State

useCounter()
↓
Another Own State
A custom hook does not automatically create shared state.

Custom Hook vs Service

Custom Hook Service
Can use React hooks. Cannot use React hooks.
Uses useState, useEffect, useContext. Usually contains API calls and business logic.
Knows about React rendering. Independent of React.
Example: useUsers() Example: getUsers()

Custom Hook vs Normal Function

Normal Function Custom Hook
Cannot use React hooks. Can use React hooks.
General JavaScript logic. Reusable React logic.
Any function name. Name must start with use.
No React lifecycle access. Has access to React lifecycle.

When Should You Write a Custom Hook?

  • When the same hook logic is repeated.
  • When multiple components use identical state logic.
  • When API-fetching hook logic is duplicated.
  • When form logic is reused.
  • When event listener logic is reused.
If you copy the same useState + useEffect logic into multiple components, consider creating a custom hook.

When Should You NOT Write a Custom Hook?

  • For one-time component logic.
  • When logic is used in only one place.
  • For simple utility functions.
  • When React hooks are not involved.

Interview Answer

A custom hook is a reusable function that uses React hooks to share stateful logic between components. We should create a custom hook whenever the same hook logic is needed in multiple places.

One-Line Interview Answer

A custom hook lets us reuse React hook logic across multiple components.
