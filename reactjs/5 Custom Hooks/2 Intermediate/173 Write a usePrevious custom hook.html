173. Write a usePrevious Custom Hook

One-Line Definition

usePrevious is a custom hook that remembers and returns the value from the previous render.

Why Do We Need It?

React easily gives us the current value of state or props. 

const [count, setCount] = useState(0);

We can access the current value: 

count

But React does not automatically provide the previous value: 

previousCount

If we need to compare the old value with the new value, we can create a custom hook called usePrevious.

usePrevious solves the problem of tracking the previous value from the last render.

Custom Hook Code

import { useRef, useEffect } from 'react';

function usePrevious(value) {

  const ref = useRef();

  useEffect(() => {

    ref.current = value;

  });

  return ref.current;

}

export default usePrevious;

Important Confusion

You may think: 

const [count, setCount] = useState(0);

const previousCount = usePrevious(count);

Because count is 0 on the first render, previousCount should also become 0.

But previousCount does not become 0 on the first render. It becomes undefined.

The reason is simple: usePrevious does not return count directly. It returns ref.current. 

return ref.current;

Why Are We Using useRef Without Any HTML Element?

Most developers first learn useRef like this: 

const inputRef = useRef();

return (
  <input ref={inputRef} />
);

Here React stores the HTML input element inside: 

inputRef.current

Then we can use it like this: 

inputRef.current.focus();
Many developers think useRef can only be used with HTML elements. This is not true.

useRef Has Two Uses

Use Case Purpose
DOM Ref Stores a reference to an HTML element.
Value Ref Stores any JavaScript value between renders.

DOM Ref Example

const inputRef = useRef();

return (
  <input ref={inputRef} />
);

React stores: 

inputRef.current = HTMLInputElement

Value Ref Example

const ref = useRef();

ref.current = 10;
ref.current = 'Rizwan';
ref.current = true;

No HTML element is attached here. We are just using ref as a storage box.

In usePrevious, useRef is used as a storage box, not as an HTML element reference.

Easy Difference Between useState and useRef

useState useRef
Stores data. Stores data.
Changing it causes re-render. Changing it does not cause re-render.
Used for UI values. Used for values we want to remember silently.
For usePrevious, we only need to remember the old value. We do not want an extra re-render. That is why useRef is the best choice.

What Actually Happens Line By Line?

const [count, setCount] = useState(0);

const previousCount = usePrevious(count);

First Render

This line runs first: 

const [count, setCount] = useState(0);

So count becomes: 

count = 0

Now count goes into the hook: 

const previousCount = usePrevious(count);

Inside the hook, value is 0: 

function usePrevious(value) {
  // value = 0
}

Then this line creates a ref: 

const ref = useRef();

Initially: 

ref.current = undefined

Now this useEffect is registered, but it does not run immediately: 

useEffect(() => {
  ref.current = value;
});

React continues and reaches the return line: 

return ref.current;

At this exact moment: 

ref.current = undefined

So: 

previousCount = undefined
The hook returns ref.current first. After the render finishes, useEffect stores the current count for the next render.

After First Render Finishes

Now React runs useEffect: 

useEffect(() => {
  ref.current = value;
});

Since value is 0: 

ref.current = 0

This 0 is now saved for the next render.

Second Render

User clicks the button: 

setCount(count + 1)

Now count becomes: 

count = 1

Again this line runs: 

const previousCount = usePrevious(count);

Inside hook: 

value = 1

But ref.current already has the value stored from the previous render: 

ref.current = 0

So the hook returns: 

return ref.current;

Therefore: 

previousCount = 0

After this render finishes, useEffect runs again and stores the current value: 

ref.current = 1

Third Render

User clicks again. Now: 

count = 2

The hook returns the old stored value: 

previousCount = 1

After render finishes: 

ref.current = 2

Visual Timeline

Render #1

count = 0
ref.current = undefined
previousCount = undefined

After render: ref.current = 0

Render #2

count = 1
ref.current = 0
previousCount = 0

After render: ref.current = 1

Render #3

count = 2
ref.current = 1
previousCount = 1

After render: ref.current = 2

Complete usePrevious Hook

import { useRef, useEffect } from 'react';

function usePrevious(value) {

  const ref = useRef();

  useEffect(() => {

    ref.current = value;

  });

  return ref.current;

}

export default usePrevious;

Complete App Example

import { useState } from 'react';
import usePrevious from './usePrevious';

function App() {

  const [count, setCount] = useState(0);

  const previousCount = usePrevious(count);

  return (
    <div>
      <h2>Current Count: {count}</h2>

      <h2>Previous Count: {previousCount}</h2>

      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
    </div>
  );

}

export default App;

Output

Initial Render

Current Count: 0
Previous Count: undefined

After First Click

Current Count: 1
Previous Count: 0

After Second Click

Current Count: 2
Previous Count: 1

Real-World Use Cases

  • Compare old and new values.
  • Track previous props.
  • Show whether a value increased or decreased.
  • Build undo functionality.
  • Run logic when a value changes from one value to another.

Example - Detect Price Change

const previousPrice = usePrevious(price);

if (price > previousPrice) {
  console.log('Price Increased');
}

if (price < previousPrice) {
  console.log('Price Decreased');
}

Important Interview Points

  • usePrevious is not built into React.
  • It is a custom hook.
  • It uses useRef internally.
  • useRef is not attached to any HTML element in this hook.
  • useRef is used as a storage box.
  • useEffect runs after render.
  • That is why first previous value is undefined.
  • It does not cause extra re-renders.

Short Interview Answer

usePrevious is a custom hook that uses useRef to store the current value after rendering and returns the value from the previous render.

One-Line Interview Answer

usePrevious lets us access the previous value of state or props in React.
