168. Write a Custom Hook useDebounce

One-Line Definition

useDebounce is a custom React hook that delays updating a value until the user stops changing it for a specified amount of time.

Why Is It Used?

Imagine a search box where the user types very fast.

R
↓
Re
↓
Rea
↓
Reac
↓
React

Without debounce, every letter can trigger an API call.

Without debounce: typing React can create 5 API calls.

With debounce, the API call happens only after the user stops typing.

With debounce: typing React creates only 1 API call after a small delay.

Custom Hook: useDebounce

import { useState, useEffect } from 'react';

function useDebounce(value, delay) {

  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {

    const timer = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => clearTimeout(timer);

  }, [value, delay]);

  return debouncedValue;

}

export default useDebounce;

Understanding Step-by-Step

Step 1 - Store Delayed Value

const [debouncedValue, setDebouncedValue] = useState(value);

This state stores the final delayed value.

value changes immediately, but debouncedValue changes after delay.

Step 2 - Run Effect When Value Changes

useEffect(() => {

}, [value, delay]);

This effect runs whenever value or delay changes.

Step 3 - Start Timer

const timer = setTimeout(() => {
  setDebouncedValue(value);
}, delay);

React waits for the delay time before updating debouncedValue.

value changes
↓
Timer starts
↓
Wait 500ms
↓
debouncedValue updates

Step 4 - Clear Old Timer

return () => clearTimeout(timer);

This is the most important part.

If the user types again before the delay ends, the old timer is removed.

Simple Timeline

User types R → timer starts
↓
User types Re → old timer removed, new timer starts
↓
User types Rea → old timer removed, new timer starts
↓
User stops typing
↓
500ms later
↓
debouncedValue becomes final value

Using useDebounce

import { useState } from 'react';
import useDebounce from './useDebounce';

function App() {

  const [search, setSearch] = useState('');

  const debouncedSearch = useDebounce(search, 500);

  return (
    <div>
      <input
        type="text"
        value={search}
        onChange={(e) => setSearch(e.target.value)}
        placeholder="Search..."
      />

      <p>Typing: {search}</p>

      <p>Debounced: {debouncedSearch}</p>
    </div>
  );

}

export default App;

What Happens Here?

Value Meaning
search Updates immediately on every key press.
debouncedSearch Updates only after the user stops typing for 500ms.

Real API Example

import { useState, useEffect } from 'react';
import useDebounce from './useDebounce';

function App() {

  const [search, setSearch] = useState('');

  const debouncedSearch = useDebounce(search, 500);

  useEffect(() => {

    if (!debouncedSearch) return;

    console.log('API Call:', debouncedSearch);

    // fetch(`/api/users?q=${debouncedSearch}`)

  }, [debouncedSearch]);

  return (
    <input
      value={search}
      onChange={(e) => setSearch(e.target.value)}
      placeholder="Search Users"
    />
  );

}

export default App;
Now the API call runs only when debouncedSearch changes, not on every key press.

Bad Way - API Directly Inside onChange

onChange={(e) => {
  setSearch(e.target.value);
  fetch('/api/search');
}}
This can call the API again and again while the user is typing.

Good Way - API After Debounce

const debouncedSearch = useDebounce(search, 500);
This waits until the user stops typing, then runs the final logic.

Real-World Use Cases

Important Interview Points

  • useDebounce delays value updates.
  • It uses useState to store the debounced value.
  • It uses useEffect to start and clean timers.
  • clearTimeout prevents old timers from running.
  • It is mostly used to reduce unnecessary API calls.

Short Interview Answer

useDebounce is a custom hook that delays updating a value until a specified time has passed without further changes. It is commonly used to reduce unnecessary API calls while typing in search fields.

One-Line Interview Answer

useDebounce delays value updates and helps optimize performance by preventing frequent operations such as API requests.
