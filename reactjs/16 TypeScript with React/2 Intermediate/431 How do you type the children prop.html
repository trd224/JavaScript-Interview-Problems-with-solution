431. How do you type the children prop?

One-Line Definition

In React TypeScript, the children prop is usually typed using ReactNode because it can represent anything React can render.

What is children?

Anything placed between a component's opening and closing tags becomes the children prop.

File: src/App.tsx
<Card>
  <h2>Hello</h2>
</Card>

React converts this into: 

<Card
  children={<h2>Hello</h2>}
/>

Project Structure

src/
│
├── App.tsx
└── components/
    └── Card.tsx

Recommended Approach - ReactNode

File: src/components/Card.tsx
import { ReactNode } from 'react';

interface CardProps {
  children: ReactNode;
}

function Card({ children }: CardProps) {
  return <div>{children}</div>;
}

export default Card;
This is the most common and recommended approach.

Using The Component

File: src/App.tsx
import Card from './components/Card';

function App() {
  return (
    <Card>
      <h2>Welcome</h2>
    </Card>
  );
}

export default App;

What Can ReactNode Contain?

Value Supported
String ✅
Number ✅
JSX ✅
Array of JSX ✅
null ✅
undefined ✅
boolean ✅

Examples Of Valid ReactNode Values

children: 'Hello'
children: 100
children: <h1>Title</h1>
children: [
  <h1 key="1">One</h1>,
  <h1 key="2">Two</h1>
]

Using PropsWithChildren

File: src/components/Card.tsx
import { PropsWithChildren } from 'react';

type CardProps = PropsWithChildren<{
  title: string;
}>;

function Card({ title, children }: CardProps) {
  return (
    <div>
      <h2>{title}</h2>
      {children}
    </div>
  );
}

export default Card;

PropsWithChildren automatically adds: 

children?: ReactNode;

Equivalent Version

File: src/components/Card.tsx
import { ReactNode } from 'react';

interface CardProps {
  title: string;
  children?: ReactNode;
}

Typing Only JSX Elements

File: src/components/Layout.tsx
import { JSX } from 'react';

interface LayoutProps {
  children: JSX.Element;
}

Only a single JSX element is allowed.

This is more restrictive than ReactNode.

Typing Multiple JSX Elements

File: src/components/Layout.tsx
interface LayoutProps {
  children: JSX.Element[];
}

Typing Function As Children

File: src/components/DataProvider.tsx
interface DataProviderProps {
  children: (name: string) => React.ReactNode;
}

Usage: 

<DataProvider>
  {name => <h2>{name}</h2>}
</DataProvider>

Most Common Pattern

File: src/components/Card.tsx
import { ReactNode } from 'react';

interface CardProps {
  children: ReactNode;
}

function Card({ children }: CardProps) {
  return <div>{children}</div>;
}

export default Card;
This is what you will see in most React TypeScript codebases.

React.FC And children

File: src/components/Card.tsx
const Card: React.FC = ({ children }) => {
  return <div>{children}</div>;
};
Modern React projects usually avoid React.FC and type children explicitly.

Recommended Approach

Interview Answer

The children prop is typically typed as ReactNode because it can represent anything React can render, including strings, numbers, JSX elements, arrays, null, and undefined. Another common approach is using PropsWithChildren, which automatically adds a typed children prop.

One-Line Interview Answer

Type the children prop as ReactNode or use PropsWithChildren for automatic children typing.
