426. How do you type useState?

One-Line Definition

In TypeScript, useState can automatically infer the type from the initial value, or you can explicitly provide the type using generics.

Basic Syntax

const [state, setState] = useState<Type>(initialValue);
useState
↓
Provide Type
↓
Type Safe State

Type Inference (Most Common)

File: src/App.tsx
import { useState } from 'react';

function App() {
  const [count, setCount] = useState(0);

  return (
    <button onClick={() => setCount(count + 1)}>
      {count}
    </button>
  );
}

export default App;

TypeScript automatically infers: 

count: number
No generic needed because TypeScript already knows the type.

Explicit Type

File: src/App.tsx
const [count, setCount] = useState<number>(0);

Equivalent to: 

const [count, setCount] = useState(0);

String State

File: src/App.tsx
const [name, setName] = useState<string>('Rizwan');

Boolean State

File: src/App.tsx
const [loading, setLoading] = useState<boolean>(false);

Why Generics Are Sometimes Required?

Problem: 

const [user, setUser] = useState(null);

TypeScript infers: 

user: null

Later: 

setUser({ id: 1, name: 'Rizwan' });

Error: 

Type '{ id: number; name: string; }'
is not assignable to type 'null'

Correct Solution

File: src/App.tsx
interface User {
  id: number;
  name: string;
}

const [user, setUser] = useState<User | null>(null);
Now user can be either a User object or null.

Object State

File: src/components/UserProfile.tsx
interface User {
  name: string;
  age: number;
}

const [user, setUser] = useState<User>({
  name: 'Rizwan',
  age: 39
});

Array State

File: src/App.tsx
const [users, setUsers] = useState<string[]>([]);

Array Of Objects

File: src/App.tsx
interface User {
  id: number;
  name: string;
}

const [users, setUsers] = useState<User[]>([]);

Union Types

File: src/App.tsx
const [status, setStatus] = useState<'loading' | 'success' | 'error'>('loading');

Optional State

File: src/App.tsx
const [token, setToken] = useState<string | undefined>();

Real World Example

File: src/components/Login.tsx
import { useState } from 'react';

interface User {
  id: number;
  name: string;
}

function Login() {
  const [email, setEmail] = useState<string>('');
  const [loading, setLoading] = useState<boolean>(false);
  const [user, setUser] = useState<User | null>(null);

  return (
    <div>
      <input
        value={email}
        onChange={e => setEmail(e.target.value)}
      />
    </div>
  );
}

export default Login;

Interview Answer

useState is typed using TypeScript generics. In simple cases, TypeScript can infer the type from the initial value. For nullable, optional, object, array, or union states, you should explicitly provide a type such as useState<User | null>(null).

One-Line Interview Answer

Type useState using generics like useState<Type>(initialValue), especially when the state can be null, undefined, or a complex object.
