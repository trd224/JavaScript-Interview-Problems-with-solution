425. What is the Difference Between interface and type for Props?

One-Line Definition

Both interface and type can be used to define React component props. For most props they work the same, but interface supports declaration merging while type supports unions, intersections, and more advanced type operations.

Basic Example Using interface

File: src/components/UserCard.tsx
interface UserCardProps {
  name: string;
  age: number;
}

function UserCard({ name, age }: UserCardProps) {
  return <h2>{name} - {age}</h2>;
}

export default UserCard;

Basic Example Using type

File: src/components/UserCard.tsx
type UserCardProps = {
  name: string;
  age: number;
};

function UserCard({ name, age }: UserCardProps) {
  return <h2>{name} - {age}</h2>;
}

export default UserCard;
For normal React props, both approaches behave almost identically.

Key Differences

Feature interface type
Object Shapes ✅ Yes ✅ Yes
Extending ✅ extends ✅ &
Union Types ❌ No ✅ Yes
Tuple Types ❌ No ✅ Yes
Primitive Aliases ❌ No ✅ Yes
Declaration Merging ✅ Yes ❌ No

Extending With interface

File: src/types/user.ts
interface Person {
  name: string;
}

interface UserProps extends Person {
  age: number;
}

Extending With type

File: src/types/user.ts
type Person = {
  name: string;
};

type UserProps = Person & {
  age: number;
};

Union Types (Only type)

File: src/types/status.ts
type Status = 'loading' | 'success' | 'error';
interface cannot create union types.

Tuple Types (Only type)

File: src/types/point.ts
type Point = [number, number];

Primitive Aliases (Only type)

File: src/types/id.ts
type UserId = string;

Declaration Merging (Only interface)

File: src/types/user.ts
interface User {
  name: string;
}

interface User {
  age: number;
}

TypeScript automatically merges them: 

interface User {
  name: string;
  age: number;
}
type aliases cannot be merged.

React Props Example

File: src/components/Button.tsx
interface ButtonProps {
  text: string;
  onClick: () => void;
}

function Button({ text, onClick }: ButtonProps) {
  return <button onClick={onClick}>{text}</button>;
}

export default Button;
This is the style most React TypeScript projects use.

Recommended Rule

  • Use interface for React component props.
  • Use type for unions, tuples, and complex type operations.
  • Either is acceptable for props.

Interview Answer

Both interface and type can be used to define React props. interface is commonly preferred for component props because it supports extension and declaration merging. type is more flexible because it can represent unions, tuples, primitive aliases, and advanced type compositions.

One-Line Interview Answer

Use interface for most React props and type for unions or advanced TypeScript type operations.
