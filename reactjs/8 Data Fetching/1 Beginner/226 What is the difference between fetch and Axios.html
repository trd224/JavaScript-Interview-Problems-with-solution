226. What is the Difference Between fetch and Axios?

One-Line Definition

fetch is a built-in browser API for making HTTP requests, whereas Axios is a third-party library that provides additional features and a simpler API.

What Are fetch and Axios?

Both fetch and Axios are used to communicate with APIs and servers.

Using fetch

fetch('/api/users')
  .then(res => res.json())
  .then(data => {

    console.log(data);

  });

Using Axios

import axios from 'axios';

axios.get('/api/users')
  .then(res => {

    console.log(res.data);

  });

Main Differences

Feature fetch Axios
Installation Required ❌ No ✅ Yes
Built Into Browser ✅ Yes ❌ No
Automatic JSON Parsing ❌ No ✅ Yes
Request Timeout Support ❌ Manual ✅ Built-in
Interceptors ❌ No ✅ Yes
Error Handling More Manual Simpler
Request Cancellation Manual Built-in

JSON Parsing Difference

fetch

const response = await fetch(url);

const data = await response.json();

Axios

const response = await axios.get(url);

const data = response.data;
Axios automatically converts JSON responses into JavaScript objects.

Error Handling Difference

fetch

const response = await fetch(url);

if (!response.ok) {

  throw new Error(
    'Request Failed'
  );

}

Axios

try {

  const response = await axios.get(url);

}

catch (err) {

  console.log(err);

}
Axios automatically throws errors for HTTP status codes such as 404 and 500, while fetch requires manual checking of response.ok.

Request Timeout

Axios

axios.get(url, {

  timeout: 5000

});

fetch

// Requires AbortController

const controller = new AbortController();

fetch(url, {

  signal: controller.signal

});

Interceptors (Axios Only)

Axios supports interceptors that allow us to modify requests and responses globally. 
axios.interceptors.request.use(config => {

  config.headers.Authorization = 'Bearer token';

  return config;

});

Which One Should You Use?

Use Case Recommended
Simple API Calls fetch
No Extra Dependencies fetch
Large Applications Axios
Authentication Tokens Axios
Global Error Handling Axios

Real Project Example

In our Aadhaar Authentication application, Axios is often more convenient because we can configure a common base URL, automatically attach authentication headers, handle token expiration globally, and centralize error handling using interceptors.

Important Interview Points

  • fetch is built into modern browsers.
  • Axios is a third-party library.
  • Axios automatically parses JSON responses.
  • Axios automatically throws for HTTP errors.
  • Axios supports interceptors.
  • fetch requires more manual configuration.

Short Interview Answer

fetch is a native browser API for HTTP requests, while Axios is a third-party library that provides features such as automatic JSON parsing, interceptors, timeout support, and simplified error handling.

One-Line Interview Answer

fetch is built into the browser, whereas Axios is a feature-rich HTTP client library with simpler API handling.
