223. How Do You Fetch Data Inside a React Component?

One-Line Definition

Data is typically fetched inside a React component using the fetch() API within useEffect() and stored using useState().

Why Do We Fetch Data?

React components often need data from a backend API, database, or external service.

Examples include:

  • User lists
  • Products
  • Orders
  • Dashboard statistics
  • Authentication details

Why Use useEffect?

API calls are side effects, and side effects should usually be handled inside useEffect().

↓

↓

Basic Example

import { useEffect, useState } from 'react';

function Users() {

  const [users, setUsers] = useState([]);

  useEffect(() => {

    fetch('https://jsonplaceholder.typicode.com/users')
      .then(res => res.json())
      .then(data => setUsers(data));

  }, []);

  return (

    <ul>

      {users.map(user => (

        <li key={user.id}>

          {user.name}

        </li>

      ))}

    </ul>

  );

}

Using async/await

import { useEffect, useState } from 'react';

function Users() {

  const [users, setUsers] = useState([]);

  useEffect(() => {

    async function fetchUsers() {

      const response = await fetch(
        'https://jsonplaceholder.typicode.com/users'
      );

      const data = await response.json();

      setUsers(data);

    }

    fetchUsers();

  }, []);

}

Loading State

const [loading, setLoading] = useState(true);

useEffect(() => {

  fetch(url)
    .then(res => res.json())
    .then(data => {

      setUsers(data);

      setLoading(false);

    });

}, []);
if (loading) {

  return <p>Loading...</p>;

}

Error Handling

const [error, setError] = useState(null);

useEffect(() => {

  fetch(url)
    .then(res => res.json())
    .then(data => setUsers(data))
    .catch(err => setError(err.message));

}, []);
if (error) {

  return <p>{error}</p>;

}

Why useState is Required?

The API response must be stored somewhere. We usually use useState so React can update the UI when the data arrives. 
const [users, setUsers] = useState([]);

What Triggers the Re-render?

API Response Received
↓
setUsers(data)
↓
State Changes
↓
React Re-renders

Can We Fetch Data Without useEffect?

Yes. If the API call happens because of a user action such as a button click, useEffect is usually not required. 
function Search() {

  const handleSearch = () => {

    fetch('/api/users');

  };

  return (

    <button onClick={handleSearch}>

      Search

    </button>

  );

}

Real Project Example

In our Aadhaar Authentication application, useEffect can be used to fetch audit records, department lists, service configurations, dashboard data, and user details when a page loads. When the API response arrives, useState updates the UI automatically.

Important Interview Points

  • Data fetching is a side effect.
  • useEffect is commonly used for API calls on component mount.
  • useState stores the fetched data.
  • Updating state triggers a re-render.
  • Loading and error states should be handled.
  • Button-click API calls usually do not need useEffect.

Short Interview Answer

Data is typically fetched inside useEffect(), stored in useState(), and displayed after React re-renders the component with the received data.

One-Line Interview Answer

We usually fetch data inside useEffect() and store the response in useState().
