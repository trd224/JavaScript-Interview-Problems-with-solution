225. How Do You Handle Errors When Fetching Data?

One-Line Definition

Errors are typically handled by using a try/catch block or .catch() method and storing the error message in state using useState().

Why Do We Need Error Handling?

API requests can fail for many reasons:

  • No internet connection
  • Server is down
  • Invalid API URL
  • Network timeout
  • Authentication failure
  • Server-side errors
Without error handling, users may see a broken page or no feedback when something goes wrong.

Step 1 - Create Error State

const [error, setError] = useState(null);
The error state stores the error message if the request fails.

Using .catch()

import { useEffect, useState } from 'react';

function Users() {

  const [users, setUsers] = useState([]);
  const [error, setError] = useState(null);

  useEffect(() => {

    fetch('https://api.example.com/users')
      .then(res => res.json())
      .then(data => {

        setUsers(data);

      })
      .catch(err => {

        setError(err.message);

      });

  }, []);

}

Display Error Message

if (error) {

  return (

    <p>

      Error: {error}

    </p>

  );

}

Using async/await with try/catch

useEffect(() => {

  async function fetchUsers() {

    try {

      const response = await fetch(
        'https://api.example.com/users'
      );

      const data = await response.json();

      setUsers(data);

    }

    catch (err) {

      setError(err.message);

    }

  }

  fetchUsers();

}, []);

Loading + Error Handling Together

const [users, setUsers] = useState([]);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);

useEffect(() => {

  fetch(url)
    .then(res => res.json())
    .then(data => {

      setUsers(data);

      setLoading(false);

    })
    .catch(err => {

      setError(err.message);

      setLoading(false);

    });

}, []);

Conditional Rendering

if (loading) {

  return <p>Loading...</p>;

}

if (error) {

  return <p>{error}</p>;

}

Checking HTTP Errors

fetch() only throws for network errors. HTTP errors such as 404 or 500 must be checked manually. 
const response = await fetch(url);

if (!response.ok) {

  throw new Error(
    'Failed to fetch data'
  );

}

Complete Example

import { useEffect, useState } from 'react';

function Users() {

  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {

    fetch('https://api.example.com/users')
      .then(res => res.json())
      .then(data => {

        setUsers(data);

        setLoading(false);

      })
      .catch(err => {

        setError(err.message);

        setLoading(false);

      });

  }, []);

  if (loading) {

    return <p>Loading...</p>;

  }

  if (error) {

    return <p>Error: {error}</p>;

  }

  return (

    <ul>

      {users.map(user => (

        <li key={user.id}>

          {user.name}

        </li>

      ))}

    </ul>

  );

}

Real Project Example

In our Aadhaar Authentication application, API calls for audit records, user details, authentication logs, and department data should handle errors gracefully. If a request fails, an error message can be displayed instead of showing an empty or broken screen.

Important Interview Points

  • Error messages are usually stored using useState().
  • Use .catch() or try/catch for error handling.
  • Loading and error states are often used together.
  • fetch() does not automatically throw for 404 or 500 responses.
  • Check response.ok for HTTP errors.

Short Interview Answer

Errors are handled by catching exceptions, storing the error message in state, and displaying a user-friendly error message when the request fails.
