249. What is a React Server Component and how does it handle data fetching differently?

One-Line Answer

A React Server Component (RSC) is a component that runs only on the server. It can fetch data directly while rendering using await, so data is loaded before the UI is sent to the browser.

Why Was It Introduced?

Traditionally React components run in the browser. To get data they usually use useEffect().

Page Loads
↓ Download JS
↓ Hydration
↓ useEffect Runs
↓ Fetch Data
↓ Show Data
The browser must wait for JavaScript and hydration before data fetching starts.

Traditional Client Component Fetching

Users.jsx (Client Component)
'use client';

import {useState,useEffect} from 'react';

export default function Users(){

  const [users,setUsers]=useState([]);

  useEffect(()=>{
    fetch('/api/users')
      .then(res=>res.json())
      .then(data=>setUsers(data));
  },[]);

  return (
    <ul>
      {users.map(user=>
        <li key={user.id}>{user.name}</li>
      )}
    </ul>
  );

}

What Happens?

Browser Gets HTML
↓ JS Downloads
↓ Hydration Starts
↓ useEffect Executes
↓ API Call Starts
↓ Data Arrives
↓ UI Updates

React Server Component Fetching

Users.jsx (Server Component)
export default async function Users(){

  const users=await fetch(
    'https://api.example.com/users'
  ).then(res=>res.json());

  return (
    <ul>
      {users.map(user=>
        <li key={user.id}>{user.name}</li>
      )}
    </ul>
  );

}
No second .then() is needed because await already waits for the entire Promise chain and gives the final users data.

What Happens?

User Requests Page
↓ Server Component Runs
↓ Fetch Data
↓ Generate UI
↓ Send Ready Result
↓ Browser Displays Data
Data fetching happens on the server before the page reaches the browser.

Relation To Our SSR Discussion

Earlier we discussed SSR:

Server Generates HTML
↓ Browser Receives HTML
↓ Hydration Starts
↓ Page Becomes Interactive

In SSR the HTML comes from the server, but React components may still need JavaScript hydration.

With React Server Components:

Server Component Runs
↓ Fetch Data
↓ Generate UI
↓ Send Result
↓ No Component JS Sent
A Server Component never runs in the browser.

Can We Use Hooks?

Feature Server Component
useState ❌ No
useEffect ❌ No
Browser APIs ❌ No
Database Access ✅ Yes
API Calls ✅ Yes
async/await ✅ Yes

How Interactivity Is Added?

Server Components are often combined with Client Components.

Page.jsx
import SearchBox from './SearchBox';

export default async function Page(){

  const products=await getProducts();

  return (
    <>
      <SearchBox />
      <ProductsList products={products} />
    </>
  );

}
Component Type
Page Server Component
ProductsList Server Component
SearchBox Client Component

Data Fetching Difference

Client Component Server Component
Fetches after hydration Fetches before sending UI
Usually uses useEffect Uses await directly
Runs in browser Runs on server
More JS shipped Less JS shipped
Extra network round trip No extra round trip

Easy Way To Remember

Client Component
↓ Hydrate First
↓ Fetch Later

Server Component
↓ Fetch First
↓ Send Ready UI

Interview Answer

A React Server Component is a component that executes only on the server. Unlike Client Components that usually fetch data in useEffect after hydration, Server Components can fetch data directly during rendering using async/await. The server retrieves the data, renders the UI, and sends the result to the browser, reducing JavaScript bundle size and eliminating the extra fetch-after-hydration step.
