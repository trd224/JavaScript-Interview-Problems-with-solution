234. What is the Difference Between useQuery and useMutation in React Query?

One-Line Definition

useQuery is used to fetch data, while useMutation is used to create, update, or delete data.

The Easiest Way To Remember

GET Request
↓
useQuery

POST / PUT / PATCH / DELETE
↓
useMutation

Think About a Real Application

Suppose we have a Users page.

  • Load all users → useQuery
  • Create user → useMutation
  • Update user → useMutation
  • Delete user → useMutation

useQuery Purpose

useQuery is responsible for:

  • Fetching data
  • Caching data
  • Refetching data
  • Keeping data fresh

Example

const {data,isLoading,error}=useQuery({
  queryKey:['users'],
  queryFn:fetchUsers
});

API: 

GET /api/users
useQuery is mainly for reading data.

useMutation Purpose

useMutation is responsible for:

  • Creating data
  • Updating data
  • Deleting data
  • Triggering server-side changes

Example

const mutation = useMutation({
  mutationFn: createUser
});

API: 

POST /api/users

Execute: 

mutation.mutate({
  name: 'Rizwan'
});
useMutation changes data on the server.

Visual Difference

Read Data
↓
GET
↓
useQuery

Change Data
↓
POST / PUT / PATCH / DELETE
↓
useMutation

Example - Fetch Users

API Function

async function fetchUsers() {
  const res = await fetch('/api/users');

  return res.json();
}

Query

const {data,isLoading}=useQuery({
  queryKey:['users'],
  queryFn:fetchUsers
});

React Query automatically:

  • Caches response.
  • Refetches when needed.
  • Manages loading state.

useMutation Example - POST (Create User)

API Function

async function createUser(user) {
  const res = await fetch('/api/users', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify(user)
  });

  return res.json();
}

Mutation

const createUserMutation = useMutation({
  mutationFn: createUser
});

Execute

createUserMutation.mutate({
  name: 'Rizwan',
  email: 'rizwan@gmail.com'
});
POST creates new data on the server.

useMutation Example - PUT (Update User)

API Function

async function updateUser(user) {
  const res = await fetch(`/api/users/${user.id}`, {
    method: 'PUT',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify(user)
  });

  return res.json();
}

Mutation

const updateUserMutation = useMutation({
  mutationFn: updateUser
});

Execute

updateUserMutation.mutate({
  id: 1,
  name: 'Mohammad Rizwan',
  email: 'rizwan@gmail.com'
});
PUT usually replaces the entire resource.

useMutation Example - PATCH (Partial Update)

API Function

async function updateUserStatus(user) {
  const res = await fetch(`/api/users/${user.id}`, {
    method: 'PATCH',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      status: user.status
    })
  });

  return res.json();
}

Mutation

const updateStatusMutation = useMutation({
  mutationFn: updateUserStatus
});

Execute

updateStatusMutation.mutate({
  id: 1,
  status: 'active'
});
PATCH updates only specific fields instead of replacing the whole object.

useMutation Example - DELETE (Delete User)

API Function

async function deleteUser(id) {
  const res = await fetch(`/api/users/${id}`, {
    method: 'DELETE'
  });

  return res.json();
}

Mutation

const deleteUserMutation = useMutation({
  mutationFn: deleteUser
});

Execute

deleteUserMutation.mutate(1);
DELETE removes data from the server.

Most Common Real-World Pattern

const queryClient = useQueryClient();

const createUserMutation = useMutation({
  mutationFn: createUser,

  onSuccess: () => {
    queryClient.invalidateQueries({
      queryKey: ['users']
    });
  }
});

Flow: 

Create User
    ↓
POST Success
    ↓
Invalidate ['users']
    ↓
Refetch Users List
    ↓
UI Updates Automatically

Example - Create User

API Function

async function createUser(user) {
  const res = await fetch('/api/users', {
    method: 'POST',
    body: JSON.stringify(user)
  });

  return res.json();
}

Mutation

const createUserMutation = useMutation({
  mutationFn: createUser
});

Run Mutation

createUserMutation.mutate({
  name: 'Rizwan'
});

Why Not Use useQuery For POST?

Because queries are designed for reading.

Queries may:

  • Refetch automatically.
  • Run on mount.
  • Run on reconnect.
  • Run on window focus.
You do not want a POST request running automatically.

Imagine: 

POST /api/users

running every time the page regains focus.

That would create duplicate users.

Why Not Use useMutation For GET?

You can technically do it.

But then you lose:

  • Caching.
  • Automatic refetching.
  • Background updates.
  • Query invalidation support.
GET requests belong in useQuery.

Loading State Comparison

useQuery

const { data, isLoading, error } = useQuery(...);

useMutation

const { mutate, isPending, error } = useMutation(...);

Caching ✅ Yes ❌ No
Refetching ✅ Yes ❌ No
GET Requests ✅ Yes ❌ No
POST Requests ❌ No ✅ Yes
PUT Requests ❌ No ✅ Yes
DELETE Requests ❌ No ✅ Yes

Using Both Together

This is the most common pattern.

Load Users

const usersQuery = useQuery({
  queryKey: ['users'],
  queryFn: fetchUsers
});

Create User

const createUserMutation = useMutation({
  mutationFn: createUser,
  onSuccess: () => {
    queryClient.invalidateQueries({
      queryKey: ['users']
    });
  }
});

Real Project Example

Suppose our Aadhaar Audit System has:

Load Audit Plans

useQuery({
  queryKey: ['auditPlans'],
  queryFn: fetchAuditPlans
});

API: 

GET /api/auditPlans

Create Audit Plan

useMutation({
  mutationFn: createAuditPlan
});

API: 

POST /api/auditPlans
Fetching uses useQuery. Creating uses useMutation.

Simple Rule

If the API reads data → useQuery.
If the API changes data → useMutation.

Important Interview Points

  • useQuery is for GET requests.
  • useMutation is for POST, PUT, PATCH, DELETE requests.
  • useQuery automatically caches data.
  • useMutation runs only when mutate() is called.
  • Queries are for reading.
  • Mutations are for changing server state.
  • Mutations often invalidate related queries after success.

Short Interview Answer

useQuery is used for fetching and caching server data, typically through GET requests. useMutation is used for creating, updating, or deleting server data through POST, PUT, PATCH, or DELETE requests. Queries automatically manage caching and refetching, while mutations are manually triggered using mutate().

One-Line Interview Answer

useQuery reads and caches server data, while useMutation changes server data and is triggered manually.
