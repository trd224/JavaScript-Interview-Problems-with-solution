238. What is Optimistic Update in React Query?

One-Line Definition

An optimistic update updates the UI immediately before the server confirms the change, making the application feel much faster.

Why Is It Called "Optimistic"?

Because we are optimistic that the server request will succeed.

Instead of waiting for:

User Clicks Save
↓
API Call
↓
Server Response
↓
Update UI

We do:

User Clicks Save
↓
Update UI Immediately
↓
API Call In Background
↓
Server Confirms
The UI feels instant because users don't wait for the server.

Normal Update vs Optimistic Update

Normal Update Optimistic Update
User waits for API response. UI updates immediately.
Feels slower. Feels instant.
No rollback needed. Rollback may be needed.

How React Query Implements Optimistic Updates

React Query provides:

  • onMutate
  • onError
  • onSettled

Basic Mutation

const addTodoMutation = useMutation({
  mutationFn: addTodo
});

Optimistic Update Example

const queryClient = useQueryClient();

const addTodoMutation = useMutation({
  mutationFn: addTodo,

  onMutate: async (newTodo) => {

    await queryClient.cancelQueries({
      queryKey: ['todos']
    });

    const previousTodos =
      queryClient.getQueryData(['todos']);

    queryClient.setQueryData(
      ['todos'],
      (old = []) => [
        ...old,
        newTodo
      ]
    );

    return {
      previousTodos
    };
  },

  onError: (
    error,
    newTodo,
    context
  ) => {

    queryClient.setQueryData(
      ['todos'],
      context.previousTodos
    );

  },

  onSettled: () => {

    queryClient.invalidateQueries({
      queryKey: ['todos']
    });

  }
});

What Happens Here?

Step 1 - User Clicks Add

User Clicks Add
↓
onMutate Runs

Step 2 - Update Cache Immediately

queryClient.setQueryData(
  ['todos'],
  (old = []) => [
    ...old,
    newTodo
  ]
);

UI updates instantly.

Step 3 - API Call Runs

Updated UI
↓
API Request
↓
Waiting For Server

Step 4 - Server Success

Nothing special needed. UI already shows the new item.

Step 5 - Server Failure

React Query restores previous data: 

queryClient.setQueryData(
  ['todos'],
  context.previousTodos
);
This is called rollback.

When Should You Use Optimistic Updates?

  • Like buttons
  • Todo lists
  • Chat messages
  • Comments
  • Bookmarks
  • Follow/Unfollow
Perfect when failures are rare and fast UI is important.

When Should You Avoid It?

  • Bank transfers
  • Payment processing
  • Wallet balance updates
  • Critical financial operations
Never show money transferred before the server actually confirms it.

Difference Between Background Refetching and Optimistic Updates

Background Refetching Optimistic Update
Shows cached data. Creates new UI state immediately.
Usually for GET requests. Usually for POST/PUT/DELETE.
Updates when fresh data arrives. Updates before server response.
No rollback needed. Rollback may be required.

Interview Answer

An optimistic update is a technique where React Query updates the UI immediately before the server confirms a mutation. It improves perceived performance by making the application feel instant. If the server request fails, React Query can roll back the cache to its previous state using onError.

One-Line Interview Answer

Optimistic updates update the UI before the server responds and roll back if the request fails.
