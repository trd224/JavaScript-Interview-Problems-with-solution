242. What is SWR and How Does It Compare to React Query?

One-Line Definition

SWR is a lightweight data-fetching library from Vercel that focuses on caching, revalidation, and background data updates using the Stale-While-Revalidate strategy.

What Does SWR Mean?

SWR stands for:

Stale
While
Revalidate

This is the same concept we discussed in React Query.

Simple Meaning

Instead of showing a loading spinner every time:

Show Cached Data
↓
Fetch Fresh Data In Background
↓
Update UI

SWR follows this pattern by default.

Example

const { data, error, isLoading } = useSWR(
  '/api/users',
  fetcher
);

When the page loads:

Check Cache
↓
Show Cached Data If Available
↓
Background Request
↓
Update Cache
↓
Update UI

What Problem Does SWR Solve?

Without SWR: 

useEffect(() => {
  fetch('/api/users')
    .then(res => res.json())
    .then(setUsers);
}, []);
Problems:
  • No caching.
  • No deduplication.
  • No background refresh.
  • No automatic revalidation.
  • Repeated boilerplate code.

SWR provides these features automatically.

Basic SWR Example

Fetcher Function

const fetcher = url =>
  fetch(url).then(res => res.json());

Use SWR

const {
  data,
  error,
  isLoading
} = useSWR(
  '/api/users',
  fetcher
);

Render Data

if (isLoading) {
  return <h2>Loading...</h2>;
}

return (
  <ul>
    {data.map(user => (
      <li key={user.id}>
        {user.name}
      </li>
    ))}
  </ul>
);

How SWR Works Internally

Check Cache
↓
Cached Data Found?
↓
Show Cached Data
↓
Background Request
↓
Fresh Data Arrives
↓
Update UI
This is why SWR applications feel very fast.

Wait... Isn't That Similar To React Query?

Yes. Both libraries solve the same problem: server state management.

React Query Example

const { data } = useQuery({
  queryKey: ['users'],
  queryFn: fetchUsers
});

SWR Example

const { data } = useSWR(
  '/api/users',
  fetcher
);

Both fetch data and cache it.

Major Differences

Feature SWR React Query
Learning Curve Simple More Advanced
Caching ✅ ✅
Background Refetch ✅ ✅
Mutations Basic Advanced
Optimistic Updates Limited Excellent
Infinite Queries Basic Built-In
DevTools Basic Excellent
Query Invalidation Manual Powerful
Pagination Basic Advanced
Enterprise Features Fewer Many

The Biggest Difference

SWR focuses mainly on data fetching.
React Query focuses on complete server-state management.

Mutation Example

React Query: 

const mutation = useMutation({
  mutationFn: addUser
});

Built-in:

  • Optimistic updates
  • Rollback
  • Invalidation
  • Mutation states

SWR: 

mutate('/api/users');

Much simpler but less powerful.

Connection To staleTime

We learned:

Cached Data
↓
Background Refetch
↓
Fresh Data

SWR was built around this exact concept.

React Query later expanded the idea with:

  • staleTime
  • cacheTime
  • Mutations
  • Optimistic Updates
  • Infinite Queries
  • Query Invalidation

Real Project Example

SWR Is Great For

  • User profile
  • Settings
  • Dashboard widgets
  • Read-only APIs

React Query Is Better For

  • E-commerce
  • Admin panels
  • CRUD applications
  • Complex server-state management
  • Heavy mutations

Which One Should You Choose?

Situation Recommended
Simple data fetching SWR
Next.js read-heavy app SWR
Large React application React Query
Lots of mutations React Query
Optimistic updates required React Query
Complex caching rules React Query

Important Interview Points

  • SWR stands for Stale-While-Revalidate.
  • Created by Vercel.
  • Uses cache-first + background revalidation.
  • Very lightweight and easy to learn.
  • Excellent for read-heavy applications.
  • React Query provides more advanced server-state features.
  • React Query is usually preferred for complex CRUD applications.

Short Interview Answer

SWR is a lightweight data-fetching library from Vercel based on the Stale-While-Revalidate pattern. It shows cached data immediately and fetches fresh data in the background. Compared to SWR, React Query provides more advanced features such as mutations, optimistic updates, query invalidation, infinite queries, and sophisticated caching controls.

One-Line Interview Answer

SWR is a lightweight cache-and-revalidate library, while React Query is a full server-state management solution.
