244. How do you set global headers in Axios?

One-Line Definition

Global headers in Axios allow us to automatically send the same HTTP headers with every request without repeating them.

Why Are Global Headers Used?

Many applications need to send headers with every API request.

  • Authorization Token
  • Content-Type
  • Language Preference
  • API Key
  • Application Version

Without global headers, we repeat the same header again and again. 

axios.get('/users', {
  headers: { Authorization: 'Bearer token' }
});

axios.get('/products', {
  headers: { Authorization: 'Bearer token' }
});
This creates duplicate code and makes maintenance harder.

Setting Global Headers

Axios provides a global configuration object. 

axios.defaults.headers

Set Authorization Header Globally

import axios from 'axios';

axios.defaults.headers.common['Authorization'] = 'Bearer my-token';

Now every Axios request automatically sends this header. 

Authorization: Bearer my-token

Example

import axios from 'axios';

axios.defaults.headers.common['Authorization'] = 'Bearer my-token';

axios.get('/users');
axios.get('/products');
axios.post('/orders');
All requests will automatically include the Authorization header.

Set Content-Type Globally

axios.defaults.headers.common['Content-Type'] = 'application/json';

Multiple Global Headers

axios.defaults.headers.common['Authorization'] = 'Bearer my-token';
axios.defaults.headers.common['X-App-Version'] = '1.0.0';
axios.defaults.headers.common['Accept-Language'] = 'en';

Real Project Example

api.js

import axios from 'axios';

axios.defaults.baseURL = 'https://api.example.com';
axios.defaults.headers.common['Authorization'] = `Bearer ${localStorage.getItem('token')}`;
axios.defaults.headers.common['Content-Type'] = 'application/json';

export default axios;

Using Axios Anywhere

import axios from './api';

axios.get('/users');

axios.post('/orders', { productId: 1 });
Now we do not need to add headers manually in every API call.

Better Approach - Axios Instance

In large applications, we usually create a custom Axios instance instead of changing the global Axios object directly. 

import axios from 'axios';

const api = axios.create({
  baseURL: 'https://api.example.com',
  headers: {
    Authorization: 'Bearer my-token',
    'Content-Type': 'application/json'
  }
});

export default api;

Usage

import api from './api';

api.get('/users');
api.post('/orders', { productId: 1 });
Axios instance keeps API configuration separate, clean, and reusable.

Dynamic Token Problem

Sometimes the token changes after login, logout, or refresh token flow. 

axios.defaults.headers.common['Authorization'] = 'Bearer old-token';
If the token changes, the old token may still be used unless we update it manually.

Best Solution - Request Interceptor

A request interceptor runs before every request. So we can read the latest token every time. 

import axios from 'axios';

const api = axios.create({
  baseURL: 'https://api.example.com'
});

api.interceptors.request.use(config => {
  const token = localStorage.getItem('token');

  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }

  return config;
});

export default api;

↓
Send API Request

Global Headers vs Request Headers

Global Headers Request Headers
Applied to all requests. Applied to one request only.
Defined once. Defined every time.
Good for auth tokens, content type, language. Good for special request-specific headers.
Reduces repeated code. Gives more control for one API call.

Important Interview Points

  • axios.defaults.headers.common sets headers globally.
  • Global headers are automatically sent with every request.
  • Common use cases are Authorization, Content-Type, and language headers.
  • Large applications usually use axios.create().
  • Request interceptors are better for dynamic tokens.
  • Global headers help avoid repetitive code.

Short Interview Answer

Global headers in Axios are configured using axios.defaults.headers.common. They allow us to automatically send headers such as Authorization token or Content-Type with every request. In larger applications, we commonly use Axios instances and request interceptors for better control and dynamic token handling.
