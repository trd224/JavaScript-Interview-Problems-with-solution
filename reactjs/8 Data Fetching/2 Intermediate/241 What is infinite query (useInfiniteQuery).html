241. What is Infinite Query (useInfiniteQuery)?

One-Line Definition

useInfiniteQuery is a React Query hook used for loading paginated data page-by-page, typically for "Load More" buttons and infinite scrolling.

Why Do We Need It?

A normal query loads data only once. 

useQuery({
  queryKey: ['users'],
  queryFn: fetchUsers
});

Result:

API Call
↓
Page 1 Data

But what if we have:

  • 10,000 Products
  • 50,000 Users
  • Infinite Social Media Feed
  • Chat History
Loading everything at once would be very slow.

Solution

Load Page 1
↓
User Scrolls
↓
Load Page 2
↓
User Scrolls
↓
Load Page 3

This is exactly what useInfiniteQuery is designed for.

Real World Examples

  • Facebook Feed
  • Instagram Feed
  • X (Twitter) Timeline
  • YouTube Comments
  • E-commerce Product Lists
  • Chat Applications

Traditional Pagination

Page 1
↓
Next
↓
Page 2
↓
Next
↓
Page 3

Infinite Query

Page 1
↓
Scroll
↓
Page 2 Appended
↓
Scroll
↓
Page 3 Appended
Users see one continuously growing list.

API Example

Suppose API supports page numbers: 

/api/users?page=1
/api/users?page=2
/api/users?page=3

Basic Example

const {
  data,
  fetchNextPage,
  hasNextPage,
  isFetchingNextPage
} = useInfiniteQuery({
  queryKey: ['users'],

  queryFn: ({ pageParam = 1 }) =>
    fetchUsers(pageParam),

  getNextPageParam: (lastPage) =>
    lastPage.nextPage
});

Understanding pageParam

This is the most confusing part for beginners. 

queryFn: ({ pageParam = 1 }) =>
  fetchUsers(pageParam)

First request: 

pageParam = 1

API call: 

/api/users?page=1

When next page loads: 

pageParam = 2

API call: 

/api/users?page=2

What Does getNextPageParam Do?

React Query must know: 

What is the next page number?

Example API response: 

{
  users: [...],
  nextPage: 2
}

Then: 

getNextPageParam: (lastPage) =>
  lastPage.nextPage
React Query automatically passes the returned value as the next pageParam.

Flow Example

Initial Load
↓
page=1
↓
Response nextPage=2

fetchNextPage()
↓
page=2
↓
Response nextPage=3

fetchNextPage()
↓
page=3

What Does data Look Like?

Unlike useQuery, infinite queries store multiple pages. 

data = {
  pages: [
    {
      users: ['A', 'B']
    },
    {
      users: ['C', 'D']
    },
    {
      users: ['E', 'F']
    }
  ]
}

Rendering Pages

{data?.pages.map(page =>
  page.users.map(user => (
    <div key={user.id}>
      {user.name}
    </div>
  ))
)}

Load More Button

<button
  onClick={() => fetchNextPage()}
  disabled={!hasNextPage}
>
  Load More
</button>

Useful Properties

Property Purpose
fetchNextPage Loads next page.
hasNextPage Checks if more pages exist.
isFetchingNextPage Loading state for next page.
data.pages Contains all loaded pages.

Loading Next Page

{isFetchingNextPage &&
  <p>Loading More...</p>
}

User experience:

Page 1 Loaded
↓
Click Load More
↓
Loading More...
↓
Page 2 Added

Infinite Scroll Example

Instead of a button, we can load automatically when user reaches the bottom.

User Scrolls Down
↓
Bottom Reached
↓
fetchNextPage()
↓
More Data Loaded
This is how Instagram, Facebook and Twitter feeds work.

How Is It Different From useQuery?

useQuery useInfiniteQuery
Single result set. Multiple pages.
One API request. Page-by-page requests.
No pagination support. Built for pagination.
Returns data. Returns pages array.

Real Project Example

Suppose our application has 50,000 audit records.

Bad approach: 

/api/audits
Loads all records at once.

Better approach: 

/api/audits?page=1
/api/audits?page=2
/api/audits?page=3
Load records only when the user needs them.

Common Interview Question

Why not use useQuery with page state? 

const [page, setPage] =
  useState(1);

useQuery({
  queryKey: ['users', page],
  queryFn: () => fetchUsers(page)
});

This creates classic pagination:

Page 1
↓
Page 2
↓
Page 3

But: 

useInfiniteQuery()
creates:
Page 1
↓
Page 1 + Page 2
↓
Page 1 + Page 2 + Page 3

Important Interview Points

  • useInfiniteQuery is used for infinite scrolling and load-more functionality.
  • Uses pageParam to determine which page to load.
  • getNextPageParam tells React Query what page to fetch next.
  • Data is stored inside data.pages.
  • fetchNextPage() loads the next page.
  • hasNextPage indicates whether more data exists.
  • Commonly used in feeds, chats, comments, and product listings.

Short Interview Answer

useInfiniteQuery is a React Query hook used for paginated and infinite-scroll data. It loads data page-by-page using pageParam and stores all loaded pages in data.pages. The fetchNextPage function loads additional pages while preserving previously loaded data.

One-Line Interview Answer

useInfiniteQuery loads paginated data incrementally for infinite scrolling and load-more interfaces.
