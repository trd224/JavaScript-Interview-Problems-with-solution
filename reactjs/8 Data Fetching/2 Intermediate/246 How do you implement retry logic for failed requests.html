246. How do you implement Retry Logic for Failed Requests?

One-Line Definition

Retry logic means automatically sending the same API request again when it fails due to a temporary problem.

Why Is Retry Logic Used?

Sometimes API requests fail because of temporary issues: 

Internet problem
Server busy
Timeout
Network error
Temporary 500 error

In these cases, retrying the request can fix the problem without requiring the user to click again.

Simple Retry Logic

async function fetchWithRetry(url, retries = 3) {

  try {

    const response = await axios.get(url);

    return response.data;

  } catch (error) {

    if (retries === 0) {
      throw error;
    }

    return fetchWithRetry(url, retries - 1);

  }

}

Usage

fetchWithRetry('/users')
  .then(data => console.log(data))
  .catch(error => console.log(error));

Retry Flow

API Request
↓
Failed
↓
Retry Again
↓
Failed Again
↓
Retry Again
↓
Still Failed
↓
Show Error

Better Retry With Delay

function delay(ms) {
  return new Promise(resolve =>
    setTimeout(resolve, ms)
  );
}

async function fetchWithRetry(url, retries = 3, delayTime = 1000) {

  try {

    const response = await axios.get(url);

    return response.data;

  } catch (error) {

    if (retries === 0) {
      throw error;
    }

    await delay(delayTime);

    return fetchWithRetry(url, retries - 1, delayTime);

  }

}
Adding a delay prevents the client from immediately hammering the server with repeated requests.

Retry Only Specific Errors

We should not retry every failed request.

Usually Retry

Network error
Timeout
500 Internal Server Error
502 Bad Gateway
503 Service Unavailable
504 Gateway Timeout

Usually Do Not Retry

400 Bad Request
401 Unauthorized
403 Forbidden
404 Not Found
These client-side errors are usually not temporary, so retrying rarely helps.

Axios Interceptor Retry Example

import axios from 'axios';

const api = axios.create({
  baseURL: 'https://api.example.com'
});

api.interceptors.response.use(
  response => response,
  async error => {
    const config = error.config;

    if (!config) return Promise.reject(error);

    config.retryCount = config.retryCount || 0;

    const shouldRetry = !error.response || [500, 502, 503, 504].includes(error.response.status);

    if (shouldRetry && config.retryCount < 3) {
      config.retryCount++;

      await new Promise(resolve =>
        setTimeout(resolve, 1000)
      );

      return api(config);
    }

    return Promise.reject(error);
  }
);

export default api;

How This Works

Request Failed
↓
Interceptor Catches Error
↓
Check Status Code
↓
Check Retry Count
↓
Wait 1 Second
↓
Send Request Again

Real Project Usage

import api from './api';

async function getUsers() {

  try {

    const response =
      await api.get('/users');

    return response.data;

  } catch (error) {

    console.log(
      'Request failed after retries'
    );

  }

}

Important Point

Retry logic is mostly safe for: 

GET Requests

Be careful with: 

POST
PUT
PATCH
DELETE

Because retrying them may perform the same action multiple times.

Example Problem

POST /orders

If the request is retried incorrectly, the server may create two orders.

Best Practice

For POST requests, use idempotency keys when the backend supports them. 

api.post('/orders', orderData, {
  headers: {
    'Idempotency-Key':
      uniqueRequestId
  }
});
Idempotency keys help the backend identify duplicate requests and prevent duplicate actions.

Important Interview Points

  • Retry logic automatically retries failed requests.
  • Useful for temporary network and server failures.
  • Do not retry 400, 401, 403, or 404 errors.
  • GET requests are generally safe to retry.
  • Be careful when retrying POST requests.
  • Axios interceptors are commonly used for global retry logic.
  • Always maintain a retry count.
  • Delays help avoid server overload.

Short Interview Answer

Retry logic automatically resends failed requests when temporary issues occur. In Axios, it is commonly implemented using a response interceptor, a retry counter, and a delay to prevent infinite retry loops.

One-Line Interview Answer

Retry logic automatically retries failed API requests using a retry count, delay, and typically an Axios response interceptor.
