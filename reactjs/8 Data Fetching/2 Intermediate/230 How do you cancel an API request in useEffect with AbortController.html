230. How do you cancel an API request in useEffect with AbortController?

One-Line Definition

AbortController allows us to cancel an ongoing fetch request when a component unmounts or when a new request starts.

Why Do We Need AbortController?

Suppose a component starts an API request.

Before the response arrives:

  • User navigates away.
  • Component unmounts.
  • A new request starts.

The old request may still complete.

The old response can update state with outdated data or cause memory leaks.

Problem Example

useEffect(() => {
  fetch('/api/user')
    .then(res => res.json())
    .then(data => setUser(data));
}, []);

What if:

Component Mounted
↓
Request Started
↓
User Leaves Page
↓
Component Unmounted
↓
Response Arrives
↓
setUser(data)
The request continues even though the component is gone.

Solution - AbortController

Create an AbortController inside useEffect. 

useEffect(() => {
  const controller = new AbortController();

  fetch('/api/user', {
    signal: controller.signal
  })
    .then(res => res.json())
    .then(data => setUser(data))
    .catch(error => console.log(error));

  return () => controller.abort();
}, []);
When the component unmounts, controller.abort() cancels the request.

Understanding Each Line

Create Controller

const controller = new AbortController();

Creates an object capable of cancelling requests.

Pass Signal To Fetch

signal: controller.signal

Connects fetch to the controller.

Abort Request

controller.abort();

Immediately cancels the request.

Handling Abort Errors

When a request is aborted, fetch throws an error.

We usually ignore that specific error. 

useEffect(() => {
  const controller = new AbortController();

  fetch('/api/user', {
    signal: controller.signal
  })
    .then(res => res.json())
    .then(data => setUser(data))
    .catch(error => {
      if (error.name !== 'AbortError') {
        console.log(error);
      }
    });

  return () => controller.abort();
}, []);
AbortError is expected when a request is cancelled.

Async/Await Version

This is the version most projects use. 

useEffect(() => {
  const controller = new AbortController();

  async function getUser() {
    try {
      const res = await fetch('/api/user', {
        signal: controller.signal
      });

      const data = await res.json();

      setUser(data);
    } catch (error) {
      if (error.name !== 'AbortError') {
        console.log(error);
      }
    }
  }

  getUser();

  return () => controller.abort();
}, []);

Race Condition Example

Imagine a search box: 

const [search, setSearch] = useState('');

User types quickly:

r
↓
Request 1

ri
↓
Request 2

riz
↓
Request 3

Without AbortController:

  • All requests continue.
  • Older requests may finish later.
  • Old data may overwrite new data.

Fix

useEffect(() => {
  const controller = new AbortController();

  async function searchUsers() {
    try {
      const res = await fetch(
        `/api/users?search=${search}`,
        { signal: controller.signal }
      );

      const data = await res.json();

      setUsers(data);
    } catch (error) {
      if (error.name !== 'AbortError') {
        console.log(error);
      }
    }
  }

  searchUsers();

  return () => controller.abort();
}, [search]);
Each new search cancels the previous request.

Real Project Example

Suppose our Aadhaar Authentication page loads user details after selecting a department. 

useEffect(() => {
  const controller = new AbortController();

  async function loadDepartmentData() {
    try {
      const res = await fetch(
        `/api/department/${departmentId}`,
        { signal: controller.signal }
      );

      const data = await res.json();

      setDepartmentData(data);
    } catch (error) {
      if (error.name !== 'AbortError') {
        console.log(error);
      }
    }
  }

  loadDepartmentData();

  return () => controller.abort();
}, [departmentId]);

If the user changes department quickly:

  • Old request is cancelled.
  • Only latest request updates UI.

Common Interview Question

Why not just ignore the response?

Because:

  • Network request still consumes resources.
  • Server still processes it.
  • Browser still waits for it.
AbortController actually stops the request instead of merely ignoring the result.

Important Interview Points

  • AbortController cancels fetch requests.
  • Pass controller.signal to fetch.
  • Call controller.abort() inside cleanup.
  • Useful for preventing memory leaks.
  • Useful for preventing race conditions.
  • AbortError should usually be ignored.
  • Commonly used inside useEffect.

Short Interview Answer

To cancel an API request in useEffect, create an AbortController, pass its signal to fetch, and call controller.abort() inside the cleanup function. This prevents memory leaks, avoids stale updates, and helps fix race conditions.

One-Line Interview Answer

Use AbortController inside useEffect, pass its signal to fetch, and call abort() in cleanup to cancel pending requests.
