240. What is Query Invalidation and When Do You Use It?

One-Line Definition

Query invalidation tells React Query that cached data may no longer be correct and should be refetched.

Simple Meaning

React Query caches data.

Example: 

['users']

Cached users: 

[
  { id: 1, name: 'Rizwan' },
  { id: 2, name: 'Raziq' }
]

Now suppose we add a new user. 

POST /users

The server now contains: 

[
  { id: 1, name: 'Rizwan' },
  { id: 2, name: 'Raziq' },
  { id: 3, name: 'John' }
]
But React Query cache still contains the old data.

The Problem

Server Data Changed
↓
React Query Cache Is Old
↓
UI Shows Old Data

We need a way to tell React Query: 

This cache is no longer trustworthy.
Please get fresh data.

That is called:

Query Invalidation

Basic Syntax

queryClient.invalidateQueries({
  queryKey: ['users']
});

What Does invalidateQueries Do?

invalidateQueries does NOT immediately delete cache.
invalidateQueries does NOT directly update cache.

It simply marks the query as stale.

invalidateQueries
↓
Mark Query As Stale
↓
Refetch Fresh Data
↓
Update Cache
↓
Update UI

Connection With staleTime

Earlier we learned: 

staleTime expired
↓
Data becomes stale

Query invalidation does almost the same thing manually. 

invalidateQueries
↓
Data becomes stale immediately
Think of invalidateQueries as manually forcing staleTime to expire.

Example - Add User

Mutation

const addUserMutation = useMutation({
  mutationFn: addUser,

  onSuccess: () => {
    queryClient.invalidateQueries({
      queryKey: ['users']
    });
  }
});

Flow

Why Not Just Update State Manually?

Without React Query: 

POST User
↓
Update Local State
↓
Update User List State
↓
Update Dashboard Count
↓
Update Other Components

With React Query: 

POST User
↓
invalidateQueries()
↓
React Query Handles Everything
React Query automatically synchronizes all components using the same query.

Invalidate Specific Query

queryClient.invalidateQueries({
  queryKey: ['users']
});

Only users query is affected.

Invalidate Dynamic Query

Suppose: 

useQuery({
  queryKey: ['user', userId],
  queryFn: () => fetchUser(userId)
});

Invalidate specific user: 

queryClient.invalidateQueries({
  queryKey: ['user', userId]
});
Only that user's cache will be refetched.

Invalidate Multiple Queries

queryClient.invalidateQueries({
  queryKey: ['users']
});
This invalidates: 
['users']
['users', 1]
['users', 2]
['users', 'active']
because they all start with: 
['users']

Common Interview Question

What is the difference between: 

invalidateQueries()
and 
refetch()
invalidateQueries refetch
Marks query as stale. Immediately refetches.
Works from anywhere. Works on current query.
Used after mutations. Used for manual refresh.

Query Invalidation vs Optimistic Update

Optimistic Update Query Invalidation
Update cache immediately. Refetch fresh server data.
Fast UI. Accurate UI.
May require rollback. No rollback needed.

When Should You Use Query Invalidation?

  • After creating data (POST)
  • After updating data (PUT/PATCH)
  • After deleting data (DELETE)
  • Whenever server data changes
  • Whenever cached data may be outdated
A good rule is: after most successful mutations, invalidate related queries.

Important Interview Points

  • invalidateQueries marks queries as stale.
  • It does not delete cache.
  • It does not directly update cache.
  • Usually used after mutations.
  • Triggers refetch of fresh server data.
  • Keeps UI synchronized with server state.
  • Can target specific query keys.
  • Can target query key groups.

Short Interview Answer

Query invalidation is a React Query mechanism used to mark cached data as stale so it can be refetched. It is commonly used after successful mutations such as create, update, or delete operations to ensure the UI stays synchronized with the latest server data.

One-Line Interview Answer

Query invalidation tells React Query that cached data may be outdated and should be refetched.
