227. Why should you not call async functions directly in useEffect?

One-Line Definition

You should not make the useEffect callback async because React expects useEffect to return either nothing or a cleanup function, but an async function always returns a Promise.

The Common Mistake

Many developers try this: 

useEffect(async () => {
  const res = await fetch('/api/users');
  const data = await res.json();

  setUsers(data);
}, []);
This is incorrect and React will show a warning.

Why Is It Wrong?

React expects useEffect to return:

  • Nothing (undefined)
  • A cleanup function

Valid Returns

useEffect(() => {
  console.log('Effect');
}, []);
useEffect(() => {
  return () => {
    console.log('Cleanup');
  };
}, []);

But Async Functions Return A Promise

async function getUsers() {
  return [];
}

Actually returns: 

Promise<Array>

Therefore: 

useEffect(async () => {
  ...
}, []);

becomes: 

useEffect(() => {
  return Promise;
}, []);
React cannot use a Promise as a cleanup function.

Problem

The Correct Pattern

Create an async function inside useEffect and call it. 

useEffect(() => {

  async function getUsers() {
    const res = await fetch('/api/users');
    const data = await res.json();

    setUsers(data);
  }

  getUsers();

}, []);
The effect itself is not async. Only the inner function is async.

Compact Version

useEffect(() => {
  async function getUsers() {
    const res = await fetch('/api/users');
    const data = await res.json();

    setUsers(data);
  }

  getUsers();
}, []);

Even More Compact

useEffect(() => {
  (async () => {
    const res = await fetch('/api/users');
    const data = await res.json();

    setUsers(data);
  })();
}, []);

This works but many teams prefer the named function version because it is easier to read.

What About Cleanup?

One big reason React requires a normal function is cleanup support. 

useEffect(() => {
  const timerId = setInterval(() => {
    console.log('Running...');
  }, 1000);

  return () => clearInterval(timerId);
}, []);

React knows exactly how to execute the cleanup function.

But if the effect returned a Promise: 

return Promise;

React would not know what cleanup to perform.

Real API Example

Wrong

useEffect(async () => {
  const res = await fetch('/api/user');
  const data = await res.json();

  setUser(data);
}, []);

Correct

useEffect(() => {
  async function getUser() {
    const res = await fetch('/api/user');
    const data = await res.json();

    setUser(data);
  }

  getUser();
}, []);

Correct Pattern With Cleanup

This is what you will commonly use in real projects: 

useEffect(() => {
  const controller = new AbortController();

  async function getUser() {
    try {
      const res = await fetch('/api/user', {
        signal: controller.signal
      });

      const data = await res.json();

      setUser(data);
    } catch (error) {
      console.log(error);
    }
  }

  getUser();

  return () => controller.abort();
}, []);
This handles both async work and cleanup correctly.

Real Project Example

Suppose our Aadhaar Authentication page loads:

useEffect(() => {

    const res = await fetch('/api/config');

instead of:

  ...

Interview Trap

Many interviewers ask: 

Can useEffect be async?

Correct answer:

No. The useEffect callback itself should not be async.

Instead:

  • Create an async function inside useEffect.
  • Call that function.
  • Return a cleanup function if needed.

Important Interview Points

  • useEffect expects undefined or a cleanup function.
  • Async functions always return a Promise.
  • React cannot use a Promise as a cleanup function.
  • Do not make the useEffect callback async.
  • Create an async function inside useEffect instead.
  • AbortController is commonly used for cleanup of fetch requests.
  • This is one of the most common React interview questions.

Short Interview Answer

You should not call async functions directly in useEffect because the effect callback must return either nothing or a cleanup function, while an async function always returns a Promise. The correct approach is to define an async function inside useEffect and then call it.

One-Line Interview Answer

The useEffect callback should not be async because React expects a cleanup function, but async functions return Promises.
