237. What is Background Refetching in React Query?

One-Line Definition

Background refetching is when React Query fetches fresh data from the server while continuing to display existing cached data to the user.

Why Does It Exist?

React Query prefers:

Show Something Immediately
↓
Fetch Fresh Data In Background
↓
Update UI If Needed

instead of:

Show Loading Spinner
↓
Wait For API
↓
Show Data
This makes applications feel much faster.

Basic Example

useQuery({
  queryKey: ['users'],
  queryFn: fetchUsers,
  staleTime: 60000
});

Suppose:

  • Data was fetched at 12:00 PM.
  • staleTime is 1 minute.
  • Data becomes stale at 12:01 PM.
  • User returns at 12:04 PM.

What Happens?

Cached Data Exists
↓
Show Cached Data Immediately
↓
Background Refetch API
↓
Fresh Data Arrives
↓
Update Cache
↓
UI Updates

Real Example

Cached Data: 

[
  { id: 1, name: 'Rizwan' }
]

User opens page:

Show Cached Data
↓
Rizwan

Meanwhile API runs in background: 

[
  { id: 1, name: 'Mohammad Rizwan' }
]

React Query updates the cache:

Old Data
↓
Rizwan

Fresh Data Arrives
↓
Mohammad Rizwan

Important Observation

Yes, background refetching can cause two UI updates. 
UI Update #1
↓
Show Cached Data

UI Update #2
↓
Show Fresh Data

This is exactly the behavior we discussed while learning staleTime and cacheTime.

When Does Background Refetching Happen?

1. User Revisits A Page With Stale Data

Cached Data Exists
↓
Data Is Stale
↓
User Returns
↓
Background Refetch

2. Window Gets Focus Again

Switch To Another Tab
↓
Come Back
↓
Data Is Stale
↓
Background Refetch

3. Internet Reconnects

Offline
↓
Online
↓
Data Is Stale
↓
Background Refetch

4. Query Invalidation

queryClient.invalidateQueries({
  queryKey: ['users']
});
Invalidate Query
↓
Refetch In Background

How To Know A Background Refetch Is Running?

React Query provides: 

const {
  data,
  isLoading,
  isFetching
} = useQuery({
  queryKey: ['users'],
  queryFn: fetchUsers
});

isLoading

First request. 

Nothing In Cache
↓
API Call
↓
isLoading = true

isFetching

Any fetch, including background refetches. 

Cached Data Exists
↓
Background Refetch
↓
isFetching = true

Very Important Difference

Property Meaning
isLoading First load, no data available yet.
isFetching Any fetch, including background refetches.

Example UI

const {
  data,
  isLoading,
  isFetching
} = useQuery({
  queryKey: ['users'],
  queryFn: fetchUsers
});

if (isLoading) {
  return <h2>Loading...</h2>;
}

return (
  <>
    {isFetching && <p>Refreshing...</p>}

    {data.map(user => (
      <div key={user.id}>
        {user.name}
      </div>
    ))}
  </>
);

Result:

Show Existing Data
+
Show "Refreshing..."
↓
Fresh Data Arrives
↓
Remove Refresh Message

Can Background Refetching Be Bad UX?

Sometimes yes.

Example: 

Bank Balance
↓
₹10,000

Background Refetch
↓
₹2,000

User briefly sees old data before new data arrives.

For highly sensitive or real-time data, showing stale data first may not be ideal.

Can We Disable It?

Yes, by increasing staleTime. 

useQuery({
  queryKey: ['departments'],
  queryFn: fetchDepartments,
  staleTime: 30 * 60 * 1000
});

For 30 minutes:

Data Is Fresh
↓
User Returns
↓
Use Cache Only
↓
No Background Refetch

Real Project Example

Department Master Data: 

useQuery({
  queryKey: ['departments'],
  queryFn: fetchDepartments,
  staleTime: 30 * 60 * 1000
});

Background refetching is usually unnecessary because department data changes very rarely.

User List: 

useQuery({
  queryKey: ['users'],
  queryFn: fetchUsers,
  staleTime: 60000
});

Background refetching is useful because users may have been added or updated.

Interview Answer

Background refetching is a React Query feature where cached data is shown immediately while a new request is sent to the server in the background. When fresh data arrives, React Query updates the cache and re-renders the UI. This improves perceived performance by avoiding unnecessary loading screens.

One-Line Interview Answer

Background refetching means React Query shows cached data immediately and fetches fresh data in the background.
