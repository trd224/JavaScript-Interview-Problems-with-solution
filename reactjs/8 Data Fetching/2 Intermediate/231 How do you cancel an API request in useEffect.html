231. How do you cancel an API request in useEffect?

One-Line Definition

An API request can be cancelled in useEffect by cleaning it up when the component unmounts or when the effect runs again.

Why Cancel API Requests?

Suppose a component starts an API request.

Before the response arrives:

  • User leaves the page.
  • Component unmounts.
  • A new request starts.

The old request may still finish.

This can cause memory leaks, stale data, and race conditions.

Visual Problem

Component Mounted
↓
API Request Started
↓
User Navigates Away
↓
Component Unmounted
↓
Response Arrives Later
↓
State Update Attempt

Most Common Solution - AbortController

The standard way is using AbortController. 

useEffect(() => {
  const controller = new AbortController();

  fetch('/api/user', {
    signal: controller.signal
  })
    .then(res => res.json())
    .then(data => setUser(data))
    .catch(error => {
      if (error.name !== 'AbortError') {
        console.log(error);
      }
    });

  return () => controller.abort();
}, []);
When the component unmounts, the request is cancelled automatically.

How It Works

Step Description
1 Create AbortController.
2 Pass controller.signal to fetch.
3 Start API request.
4 Cleanup function runs.
5 controller.abort() cancels request.

Compact Async/Await Version

useEffect(() => {
  const controller = new AbortController();

  async function getUser() {
    try {
      const res = await fetch('/api/user', {
        signal: controller.signal
      });

      const data = await res.json();

      setUser(data);
    } catch (error) {
      if (error.name !== 'AbortError') {
        console.log(error);
      }
    }
  }

  getUser();

  return () => controller.abort();
}, []);

Search Example

Race conditions commonly happen in search inputs. 

useEffect(() => {
  const controller = new AbortController();

  async function searchUsers() {
    try {
      const res = await fetch(
        `/api/users?q=${search}`,
        { signal: controller.signal }
      );

      const data = await res.json();

      setUsers(data);
    } catch (error) {
      if (error.name !== 'AbortError') {
        console.log(error);
      }
    }
  }

  searchUsers();

  return () => controller.abort();
}, [search]);
Each new search cancels the previous request.

Alternative Solution - Ignore Old Responses

Sometimes requests cannot be cancelled.

In that case we ignore stale responses. 

useEffect(() => {
  let ignore = false;

  async function loadUser() {
    const res = await fetch('/api/user');
    const data = await res.json();

    if (!ignore) {
      setUser(data);
    }
  }

  loadUser();

  return () => {
    ignore = true;
  };
}, []);
The request still runs, but old responses cannot update state.

AbortController vs Ignore Flag

Method Cancels Request? Prevents Stale Updates?
AbortController ✅ Yes ✅ Yes
Ignore Flag ❌ No ✅ Yes
AbortController is the preferred solution because it actually stops the network request.

Real Project Example

In an Aadhaar Authentication screen:

  • User selects Department A.
  • API request starts.
  • User quickly selects Department B.
  • Another request starts.

Without cancellation:

  • Department A response may arrive later.
  • Old data may overwrite Department B data.

Using AbortController: 

return () => controller.abort();

ensures only the latest request survives.

Important Interview Points

  • API requests should be cleaned up in useEffect.
  • AbortController is the standard solution.
  • Pass controller.signal to fetch.
  • Call controller.abort() in cleanup.
  • Useful for preventing memory leaks.
  • Useful for preventing race conditions.
  • Ignore-flag pattern is a fallback when cancellation is not possible.

Short Interview Answer

API requests can be cancelled in useEffect using AbortController. Create a controller, pass its signal to fetch, and call controller.abort() inside the cleanup function. This prevents memory leaks, stale updates, and race conditions.

One-Line Interview Answer
