245. What is an Axios Interceptor and When is it Useful?

One-Line Definition

An Axios interceptor is a function that runs automatically before an API request is sent or after an API response is received.

Simple Meaning

Interceptor means Axios gives us a common place where we can control all API requests and responses.

Component Calls API
↓
Request Interceptor Runs
↓
API Request Goes To Server
↓
Server Sends Response
↓
Response Interceptor Runs
↓
Data Reaches Component

Why Two Interceptors?

Interceptor When It Runs Common Use
Request Interceptor Before API request goes to server Show loader, add token, add headers
Response Interceptor After server sends response Hide loader, handle errors, handle logout
Request interceptor is for before API call. Response interceptor is for after API call.

File 1: src/api/axios.js

This file creates one reusable Axios instance. We add interceptors here only once. 

import axios from 'axios';

const api = axios.create({
  baseURL: 'https://jsonplaceholder.typicode.com'
});

function showLoader() {
  console.log('Loader Started');
}

function hideLoader() {
  console.log('Loader Stopped');
}

//=====================
//For request
//=====================

api.interceptors.request.use(config => {
    showLoader();

    const token = localStorage.getItem('token');

    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }

    return config;
  },
  error => {
    hideLoader();
    return Promise.reject(error);
  }
);

//=====================
//For response
//=====================

api.interceptors.response.use(response => {
    hideLoader();
    return response;
  },
  error => {
    hideLoader();

    if (error.response?.status === 401) {
      console.log('Unauthorized user');
    }

    return Promise.reject(error);
  }
);

export default api;

File 2: src/components/Users.jsx

This component only calls the API. Loader and token logic is already handled by interceptor. 

import { useEffect, useState } from 'react';
import api from '../api/axios';

function Users() {
  const [users, setUsers] = useState([]);

  useEffect(() => {
    api.get('/users')
      .then(res => setUsers(res.data))
      .catch(error => console.log(error));
  }, []);

  return (
    <div>
      <h2>Users List</h2>

      <ul>
        {users.map(user => (
          <li key={user.id}>{user.name}</li>
        ))}
      </ul>
    </div>
  );
}

export default Users;

File 3: src/App.jsx

import Users from './components/Users';

function App() {
  return <Users />;
}

export default App;

Why is Axios Interceptor Useful?

  • We can show loader globally.
  • We can hide loader globally.
  • We can add JWT token in every request automatically.
  • We can handle 401 unauthorized error globally.
  • We can avoid repeating same code in every component.

Without Interceptor

Without interceptor, we have to repeat loader and error handling code in every API call. 
showLoader();

axios.get('/users')
  .then(res => {
    hideLoader();
    setUsers(res.data);
  })
  .catch(error => {
    hideLoader();
    console.log(error);
  });

With Interceptor

With interceptor, component code becomes clean. Common API logic is handled globally. 
api.get('/users')
  .then(res => setUsers(res.data))
  .catch(error => console.log(error));

Real Project Use Case

Suppose our application has many API calls. 

api.get('/users');
api.get('/products');
api.post('/orders');
api.get('/profile');

Instead of adding token and loader manually in every API call, we add that logic once inside Axios interceptor.

This makes our application clean, reusable, and easy to maintain.

Important Interview Points

  • Axios interceptor can intercept request and response.
  • Request interceptor runs before request goes to server.
  • Response interceptor runs after response comes from server.
  • Request interceptor is useful for adding token and headers.
  • Response interceptor is useful for global error handling.
  • Interceptors reduce duplicate API-related code.

Short Interview Answer

Axios interceptor is used to run common logic globally before requests or after responses. Request interceptor is useful for adding tokens, headers, and showing loaders. Response interceptor is useful for hiding loaders, handling errors, and managing unauthorized responses.

One-Line Interview Answer

Axios interceptor helps us handle common API request and response logic globally from one place.
