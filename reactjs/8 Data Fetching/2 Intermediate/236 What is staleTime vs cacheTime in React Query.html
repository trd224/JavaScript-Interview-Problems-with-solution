236. What is staleTime vs cacheTime in React Query?

One-Line Difference

staleTime controls when data becomes stale (old), while cacheTime controls when unused cached data is deleted from memory.

The Simplest Memory Trick

staleTime
=
Freshness Timer

cacheTime
=
Deletion Timer

Think About Two Questions

Question Answer
How long should I trust this data? staleTime
How long should I keep unused data? cacheTime

Example

useQuery({
  queryKey: ['users'],
  queryFn: fetchUsers,
  staleTime: 60000,
  cacheTime: 300000
});

Meaning:

  • staleTime = 1 minute
  • cacheTime = 5 minutes

What Happens When staleTime Expires?

At 12:01 PM: 

staleTime expired
↓
Data becomes stale
Data is NOT deleted.
API is NOT called immediately.

React Query simply says: 

I still have the data,
but I am not sure it is fresh anymore.

Case 1 - User Returns While Data Is Still Fresh

12:00 API Call
12:00:30 User Returns
Data Still Fresh
↓
Show Cached Data
↓
No API Call
Because staleTime has not expired.

Case 2 - User Returns After staleTime

12:00 API Call
12:01 Data Stale
12:04 User Returns
Cache Still Exists
↓
Show Cached Data Immediately
↓
Background Refetch API
↓
Fresh Data Arrives
↓
UI Updates Again
This may cause two UI updates.

Example: 

Old Cached Data
↓
User Sees It
↓
Fresh API Data Arrives
↓
UI Changes

Why Is This Done?

React Query prefers: 

Fast UI First
↓
Fresh Data Second

instead of: 

Loading Spinner
↓
Wait For Server
↓
Show Data

When Is This Good UX?

  • Department List
  • Country List
  • Categories
  • Product List
  • Static Master Data
Showing slightly old data for a moment is usually acceptable.

When Can This Be Bad UX?

  • Bank Balance
  • Wallet Amount
  • Payment Status
  • Stock Price
  • Live Score
Users may briefly see an outdated value and then a new value.

What Happens When cacheTime Expires?

Suppose: 

12:01 User Leaves
12:06 cacheTime Expires
12:10 User Returns
Cache Deleted
↓
User Returns
↓
No Cache Available
↓
Show Loading
↓
API Call
↓
Show Fresh Data
Cache is deleted only if nobody uses that query until cacheTime finishes.

Case 3 - User Returns Before cacheTime Expires

12:01 User Leaves
12:04 User Returns
Cache Still Exists
↓
Query Becomes Active Again
↓
cacheTime Stops / Resets
↓
Cache Can Be Reused
Because cacheTime did not finish, the cached data survives.

Interview Answer

staleTime controls how long React Query considers data fresh. After staleTime expires, the data becomes stale but remains in cache. If the user revisits while the cache still exists, React Query can show cached data immediately and refetch fresh data in the background. cacheTime controls how long unused cached data remains in memory. If nobody uses the query before cacheTime expires, the cache is deleted and the next visit requires a fresh API call.

One-Line Interview Answer

staleTime controls freshness, while cacheTime controls how long unused cached data stays before deletion.
