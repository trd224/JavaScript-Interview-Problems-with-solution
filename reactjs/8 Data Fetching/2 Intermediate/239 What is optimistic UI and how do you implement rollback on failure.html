239. What is Optimistic UI and How Do You Implement Rollback on Failure?

One-Line Definition

Optimistic UI means updating the UI immediately before the server confirms the action, and rollback means restoring the old UI if the server request fails.

Simple Meaning

Normally, when a user performs an action, we wait for the server response first.

User Clicks Button
↓
API Request
↓
Wait For Server
↓
Update UI

With optimistic UI, we update the UI first.

User Clicks Button
↓
Update UI Immediately
↓
API Request In Background
↓
If Success → Keep UI
If Failure → Rollback UI
This makes the application feel very fast.

What Is Rollback?

Rollback means undoing the optimistic UI change when the server request fails.

Example:

100 Likes
↓
Optimistic Update
↓
101 Likes
↓
API Failed
↓
Rollback
↓
100 Likes

How React Query Helps

React Query provides mutation lifecycle callbacks:

Callback Use
onMutate Runs before mutation request. Used for optimistic update.
onError Runs if mutation fails. Used for rollback.
onSettled Runs after success or failure. Used for refetch/invalidation.

Complete Optimistic UI Example

API Function

async function addTodo(newTodo) {
  const res = await fetch('/api/todos', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(newTodo)
  });

  if (!res.ok) {
    throw new Error('Failed to add todo');
  }

  return res.json();
}

useMutation With Rollback

const queryClient = useQueryClient();

const addTodoMutation = useMutation({
  mutationFn: addTodo,

  onMutate: async (newTodo) => {
    await queryClient.cancelQueries({ queryKey: ['todos'] });

    const previousTodos = queryClient.getQueryData(['todos']);

    queryClient.setQueryData(['todos'], (old = []) => [...old, {
      id: Date.now(),
      title: newTodo.title,
      pending: true
    }]);

    return { previousTodos };
  },

  onError: (error, newTodo, context) => {
    queryClient.setQueryData(['todos'], context.previousTodos);
  },

  onSettled: () => {
    queryClient.invalidateQueries({ queryKey: ['todos'] });
  }
});

How This Code Works

Step 1 - Cancel Existing Refetch

await queryClient.cancelQueries({ queryKey: ['todos'] });

This prevents an old background refetch from overwriting our optimistic update.

Step 2 - Save Previous Data

const previousTodos = queryClient.getQueryData(['todos']);

We save old data so we can restore it if the API fails.

Step 3 - Update UI Immediately

queryClient.setQueryData(['todos'], (old = []) => [
  ...old,
  {
    id: Date.now(),
    title: newTodo.title,
    pending: true
  }
]);

React Query updates the cache immediately, so the UI updates immediately.

Step 4 - Return Old Data

return { previousTodos };

The returned object becomes context inside onError.

Step 5 - Rollback on Failure

onError: (error, newTodo, context) => {
  queryClient.setQueryData(['todos'], context.previousTodos);
}

If API fails, we restore old todos.

Step 6 - Refetch Final Data

onSettled: () => {
  queryClient.invalidateQueries({ queryKey: ['todos'] });
}

After success or failure, we refetch to make sure cache matches server.

Example Usage

function AddTodo() {
  const [title, setTitle] = useState('');

  function handleAdd() {
    addTodoMutation.mutate({ title });
    setTitle('');
  }

  return (
    <>
      <input value={title} onChange={e => setTitle(e.target.value)} />
      <button onClick={handleAdd}>Add Todo</button>
    </>
  );
}

When Optimistic UI Is Good

  • Like button
  • Bookmark
  • Todo add/remove
  • Follow/unfollow
  • Comments
  • Chat messages
Use optimistic UI when failure is rare and instant feedback improves UX.

When Optimistic UI Is Risky

  • Payment success
  • Bank balance
  • Wallet transfer
  • Order confirmation
  • Critical financial status
Do not show critical success before the server confirms it.

Optimistic UI vs Background Refetching

Optimistic UI Background Refetching
Used mostly with mutations. Used mostly with queries.
UI updates before server response. Cached UI shows while fresh data loads.
Rollback may be needed. Rollback usually not needed.
Example: Like button updates instantly. Example: User list refreshes in background.

Important Interview Points

  • Optimistic UI updates the UI before server confirmation.
  • It improves perceived performance.
  • Rollback restores previous state on failure.
  • React Query uses onMutate for optimistic updates.
  • onError is used for rollback.
  • onSettled is used to refetch or invalidate the query.
  • Save previous cache before modifying it.
  • Avoid optimistic UI for critical financial operations.

Short Interview Answer

Optimistic UI means updating the UI immediately before the server confirms the mutation. In React Query, we implement it in onMutate by saving the previous cache and updating the cache immediately. If the request fails, onError restores the previous cache as rollback. Finally, onSettled usually invalidates the query to sync with the server.

One-Line Interview Answer

Optimistic UI updates the interface before server confirmation and rolls back to previous data if the request fails.
