243. What is Axios and How Does It Differ From Fetch?

One-Line Definition

Axios is a popular third-party HTTP client library used to make API requests in JavaScript applications.

fetch is the built-in browser API used for making HTTP requests.

Why Are They Used?

Both Axios and fetch allow our application to communicate with backend APIs.

  • Call REST APIs
  • Send GET, POST, PUT, DELETE requests
  • Send request data to servers
  • Receive response data from servers

Common API Request Types

GET /users
POST /login
PUT /profile
DELETE /cart/1

Fetch Example

fetch('/api/users')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.log(error));

Axios Example

import axios from 'axios';

axios.get('/api/users')
  .then(response => console.log(response.data))
  .catch(error => console.log(error));
Axios gives response data directly inside response.data, while fetch requires response.json().

Main Differences

Feature Fetch Axios
Installation Required No Yes, npm install axios
Built Into Browser Yes No
Automatic JSON Parsing No Yes
Request Timeout Manual Built-in
Request Interceptors No Yes
Response Interceptors No Yes
Error Handling Less convenient More convenient
Request Cancellation Possible, but more manual Easier
Upload Progress More work Built-in support

JSON Parsing Difference

Fetch

In fetch, we must manually convert the response into JSON. 

const response = await fetch('/api/users');
const data = await response.json();

Axios

Axios automatically parses JSON response data. 

const response = await axios.get('/api/users');
const data = response.data;

Error Handling Difference

Fetch

Fetch only throws an error for network failures. Even if the API returns 404, 401, or 500, fetch does not automatically go to catch. 

const response = await fetch('/users');

if (!response.ok) {
  throw new Error('Request Failed');
}

const data = await response.json();
With fetch, we usually need to manually check response.ok.

Axios

Axios automatically treats failed HTTP status codes like 404, 401, and 500 as errors. 

try {
  await axios.get('/users');
} catch (error) {
  console.log(error.response.status);
}
Axios error handling is easier in real projects because failed status codes go directly into catch.

Interceptors in Axios

Axios can run code before every request and after every response. 

axios.interceptors.request.use(config => {
  config.headers.Authorization = `Bearer ${token}`;
  return config;
});

Interceptors are useful for:

  • Adding JWT tokens
  • Authentication handling
  • Logging requests
  • Handling global API errors
  • Refreshing expired tokens

Timeout Difference

Fetch

Fetch needs extra code for timeout. 

const controller = new AbortController();

setTimeout(() => {
  controller.abort();
}, 5000);

fetch('/users', {
  signal: controller.signal
});

Axios

Axios has timeout support built in. 

axios.get('/users', {
  timeout: 5000
});

React Example Using Axios

import { useEffect, useState } from 'react';
import axios from 'axios';

function Users() {
  const [users, setUsers] = useState([]);

  useEffect(() => {
    axios.get('/api/users')
      .then(res => setUsers(res.data))
      .catch(err => console.log(err));
  }, []);

  return (
    <ul>
      {users.map(user => <li key={user.id}>{user.name}</li>)}
    </ul>
  );
}

export default Users;
Axios URL

Frontend Port 3000 and Backend Port 4000

Suppose our React frontend is running on:

http://localhost:3000

And our backend API is running on:

http://localhost:4000

If we write:

axios.get('/api/users');

Then Axios will call the frontend origin, because /api/users is a relative URL.

Important: It will call http://localhost:3000/api/users, not http://localhost:4000/api/users. 
axios.get('/api/users')
↓
http://localhost:3000/api/users

Option 1: Use Full Backend URL

axios.get('http://localhost:4000/api/users');

This directly calls the backend server.

Option 2: Configure Axios Base URL

import axios from 'axios';

const api = axios.create({
  baseURL: 'http://localhost:4000'
});

export default api;

Now call API like this:

api.get('/api/users');

Axios will convert it to:

http://localhost:4000/api/users

Option 3: Use Proxy in Development

In development, we can configure a proxy so our frontend dev server forwards API requests to the backend. 

{
  "proxy": "http://localhost:4000"
}

Then this code:

axios.get('/api/users');

Works like this:

Browser
↓
localhost:3000/api/users
↓
React Dev Server Proxy
↓
localhost:4000/api/users
Browser thinks it called port 3000, but the dev server forwards the request to port 4000.

How To Check?

Open browser DevTools → Network tab.

  • If you see http://localhost:3000/api/users, then relative URL is being used.
  • If you see http://localhost:4000/api/users, then backend is being called directly.

Interview Answer

axios.get('/api/users') uses the current website origin. If frontend and backend run on different ports, we need full backend URL, Axios baseURL, or development proxy.

Which One Should You Use?

Use Fetch When

  • Our project is small.
  • We do not want an extra dependency.
  • We only need basic API calls.
  • We are comfortable handling JSON and errors manually.

Use Axios When

  • Our project is medium or large.
  • We need authentication tokens.
  • We need interceptors.
  • We need better error handling.
  • We need timeout support.
  • We want cleaner API request code.
Small Project
↓
fetch is enough

Large Project with Auth / Interceptors / Timeout
↓
Axios is more convenient

Important Interview Points

  • fetch is built into browsers.
  • Axios is a third-party HTTP client library.
  • Axios automatically parses JSON.
  • Axios automatically throws errors for failed HTTP status codes.
  • Axios supports request and response interceptors.
  • Axios has built-in timeout support.
  • fetch requires more manual code for common networking tasks.

Short Interview Answer

Axios is a third-party HTTP client library, while fetch is the browser's built-in API for making HTTP requests. Axios provides automatic JSON parsing, better error handling, interceptors, request cancellation, and timeout support, whereas fetch requires more manual code for these features.

One-Line Interview Answer

Axios is a feature-rich HTTP client library, whereas fetch is the browser's native API that requires more manual handling for common networking tasks.
