177. What Triggers a Re-render in React?

One-Line Definition

A React component re-renders whenever React detects that its state, props, context value, or parent render output has changed.

What is a Re-render?

A re-render means React executes the component function again to calculate the latest UI.

Data Changes
↓
Component Function Runs Again
↓
React Compares Old and New UI
↓
DOM Updates If Needed

Main Things That Trigger Re-renders

Trigger Causes Re-render?
State Change (useState) ✅ Yes
Props Change ✅ Yes
Context Value Change ✅ Yes
Parent Component Re-renders ✅ Usually Yes
useRef.current Change ❌ No

1. State Changes Trigger Re-renders

import { useState } from 'react';

function Counter() {

  const [count, setCount] = useState(0);

  return (
    <button onClick={() => setCount(count + 1)}>
      {count}
    </button>
  );

}
Calling setCount() changes state and triggers a re-render.

2. Props Changes Trigger Re-renders

function Parent() {

  const [name, setName] = useState('Rizwan');

  return <Child name={name} />;

}

function Child({ name }) {

  return <h1>{name}</h1>;

}
When the name prop changes, Child re-renders.

3. Context Changes Trigger Re-renders

const UserContext = createContext();

function Profile() {

  const user = useContext(UserContext);

  return <h1>{user}</h1>;

}
If the Provider value changes, all consuming components re-render.

4. Parent Re-renders Can Trigger Child Re-renders

function Parent() {

  const [count, setCount] = useState(0);

  return (
    <>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>

      <Child />
    </>
  );

}
When Parent re-renders, Child also re-renders by default.

What Does NOT Trigger a Re-render?

Action Triggers Re-render?
Changing useRef.current ❌ No
Changing Normal Variables ❌ No
Console Logs ❌ No
API Calls Alone ❌ No

Example: useRef Does Not Re-render

import { useRef } from 'react';

function Counter() {

  const countRef = useRef(0);

  const increment = () => {

    countRef.current++;

  };

  return (

    <button onClick={increment}>
      Increment
    </button>

  );

}
countRef.current changes, but React does not re-render the component.

Important Note About useEffect

useEffect itself does not trigger a re-render.

However, if useEffect updates state using setState(), that state update causes a re-render. 
useEffect(() => {

  setCount(10);

}, []);

Real Project Example

In our Aadhaar Authentication application:

Updating OTP, loader state, authentication response, consent status, or popup visibility using useState causes re-renders because the UI changes.

Updating timer IDs, input references, or SignalR connection references using useRef does not cause re-renders because the UI does not depend on those values.

Important Interview Points

  • State changes trigger re-renders.
  • Props changes trigger re-renders.
  • Context changes trigger re-renders.
  • Parent re-renders usually cause child re-renders.
  • useRef changes do not trigger re-renders.
  • useEffect does not directly trigger re-renders.

Short Interview Answer

React re-renders a component when its state changes, props change, context value changes, or its parent re-renders. Changes to useRef values do not trigger re-renders.

One-Line Interview Answer

State changes, props changes, context updates, and parent re-renders trigger React re-renders, while useRef changes do not.
