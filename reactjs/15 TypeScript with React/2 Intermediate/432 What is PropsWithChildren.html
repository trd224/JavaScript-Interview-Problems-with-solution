432. What is PropsWithChildren?

One-Line Definition

PropsWithChildren is a utility type provided by React that automatically adds a typed children prop to your component props.

Why Do We Need It?

Normally, if a component accepts children, we must type it manually.

File: src/components/Card.tsx
import { ReactNode } from 'react';

interface CardProps {
  title: string;
  children?: ReactNode;
}

PropsWithChildren saves us from writing: 

children?: ReactNode;

What Does PropsWithChildren Actually Do?

Internally it is roughly equivalent to: 

type PropsWithChildren<P> = P & {
  children?: React.ReactNode;
};
Your Props
+
children?: ReactNode
↓
PropsWithChildren

Project Structure

src/
│
├── App.tsx
└── components/
    └── Card.tsx

Without PropsWithChildren

File: src/components/Card.tsx
import { ReactNode } from 'react';

interface CardProps {
  title: string;
  children?: ReactNode;
}

function Card({ title, children }: CardProps) {
  return (
    <div>
      <h2>{title}</h2>
      {children}
    </div>
  );
}

export default Card;

With PropsWithChildren

File: src/components/Card.tsx
import { PropsWithChildren } from 'react';

type CardProps = PropsWithChildren<{
    title: string;
  }>;

function Card({ title, children }: CardProps) {
  return (
    <div>
      <h2>{title}</h2>
      {children}
    </div>
  );
}

export default Card;
children is automatically available and correctly typed.

Using The Component

File: src/App.tsx
import Card from './components/Card';

function App() {
  return (
    <Card title="Profile">
      <h3>Rizwan</h3>
    </Card>
  );
}

export default App;

What Type Does children Get?

PropsWithChildren automatically adds: 

children?: ReactNode;

ReactNode supports:

Value Supported
String ✅
Number ✅
JSX ✅

null ✅
undefined ✅
boolean ✅

Real World Layout Example

File: src/components/Layout.tsx
import { PropsWithChildren } from 'react';

type LayoutProps = PropsWithChildren<{
    title: string;
  }>;

function Layout({ title, children }: LayoutProps) {
  return (
    <>
      <header>{title}</header>
      <main>{children}</main>
    </>
  );
}

export default Layout;

Equivalent Manual Version

File: src/components/Layout.tsx
import { ReactNode } from 'react';

interface LayoutProps {
  title: string;
  children?: ReactNode;
}

Both versions are exactly the same.

Should You Use It?

Approach Recommendation
children: ReactNode ✅ Most Common
PropsWithChildren ✅ Also Good
React.FC ⚠ Less Preferred

What Do Most Teams Prefer?

Most React TypeScript projects today prefer:

File: src/components/Card.tsx
interface CardProps {
  title: string;
  children: React.ReactNode;
}

because it is more explicit.

PropsWithChildren is useful, but many teams prefer explicitly declaring children so it is visible immediately in the props definition.

Interview Answer

PropsWithChildren is a React utility type that automatically adds a typed children prop to a component's props. It is equivalent to combining your props with children?: ReactNode and is useful when creating components that render nested content.

One-Line Interview Answer

PropsWithChildren automatically adds a typed children prop (children?: ReactNode) to your component props.
