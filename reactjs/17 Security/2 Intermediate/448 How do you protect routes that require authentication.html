448. How do you protect routes that require authentication?

One-Line Definition

In React, protected routes check whether the user is authenticated before showing a page. If the user is not authenticated, the app redirects them to the login page.

Very Important Point

ProtectedRoute in React is mainly for user experience. Real security must be done on the backend.

Why? Because frontend code runs in the user's browser. A user can open DevTools and manually create a fake token.

Basic Protected Route Example

File: src/components/ProtectedRoute.jsx
import { Navigate } from 'react-router-dom';

function ProtectedRoute({ children }) {
  const token = localStorage.getItem('token');

  if (!token) {
    return <Navigate to="/login" replace />;
  }

  return children;
}

export default ProtectedRoute;
This checks if token exists. If token does not exist, user goes to login page.

But There Is A Problem

A user can open browser DevTools and add a fake token.

Browser Console
localStorage.setItem('token', 'fake-token');

Now this condition becomes true:

ProtectedRoute Check
const token = localStorage.getItem('token');

if (token) {
  // React thinks user is logged in
}
So checking only token existence on frontend is not real security.

What Happens With Fake Token?

User Creates Fake Token
↓
ProtectedRoute Sees Token Exists
↓
Dashboard Page Opens
↓
Dashboard Calls Protected API
↓
Backend Verifies Token
↓
Fake Token Fails
↓

So What Is Real Security?

Real security means backend must verify the token/session before sending protected data.
Backend Example
app.get('/api/profile', authenticate, (req, res) => {
  res.json({ name: 'Rizwan', role: 'user' });
});
Backend Auth Middleware
function authenticate(req, res, next) {
  const token = req.headers.authorization;

  if (!token || !verifyToken(token)) {
    return res.status(401).json({ message: 'Unauthorized' });
  }

  next();
}
Even if the dashboard opens using a fake token, backend will not return protected data.

Why Use ProtectedRoute Then?

ProtectedRoute gives better user experience.

Without ProtectedRoute With ProtectedRoute
User opens dashboard User opens dashboard
Dashboard loads first Auth checked first
API fails with 401 User redirected to login
Bad UX Better UX

Better Production Approach

Instead of trusting only localStorage, call a backend API like /me when the app starts.

↓

↓

↓
ProtectedRoute Uses User State

Auth Context Example

File: src/context/AuthContext.jsx
import { createContext, useContext, useEffect, useState } from 'react';

const AuthContext = createContext(null);

export function AuthProvider({ children }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    fetch('/api/me')
      .then(res => res.ok ? res.json() : null)
      .then(data => setUser(data))
      .finally(() => setLoading(false));
  }, []);

  return <AuthContext.Provider value={{ user, loading, setUser }}>{children}</AuthContext.Provider>;
}

export function useAuth() {
  return useContext(AuthContext);
}

ProtectedRoute Using Auth Context

File: src/components/ProtectedRoute.jsx
import { Navigate } from 'react-router-dom';
import { useAuth } from '../context/AuthContext';

function ProtectedRoute({ children }) {
  const { user, loading } = useAuth();

  if (loading) {
    return <h2>Checking authentication...</h2>;
  }

  if (!user) {
    return <Navigate to="/login" replace />;
  }

  return children;
}

export default ProtectedRoute;
Now React trusts the backend response, not only localStorage token existence.

Use ProtectedRoute In App

File: src/App.jsx
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import { AuthProvider } from './context/AuthContext';
import Login from './pages/Login';
import Dashboard from './pages/Dashboard';
import ProtectedRoute from './components/ProtectedRoute';

function App() {
  return (
    <AuthProvider>
      <BrowserRouter>
        <Routes>
          <Route path="/login" element={<Login />} />
          <Route path="/dashboard" element={<ProtectedRoute><Dashboard /></ProtectedRoute>} />
        </Routes>
      </BrowserRouter>
    </AuthProvider>
  );
}

export default App;

Preserve Requested Page After Login

If user tries to open /dashboard without login, redirect to login and remember the requested page.

File: src/components/ProtectedRoute.jsx
import { Navigate, useLocation } from 'react-router-dom';
import { useAuth } from '../context/AuthContext';

function ProtectedRoute({ children }) {
  const { user, loading } = useAuth();
  const location = useLocation();

  if (loading) {
    return <h2>Checking authentication...</h2>;
  }

  if (!user) {
    return <Navigate to="/login" state={{ from: location }} replace />;
  }

  return children;
}

export default ProtectedRoute;
File: src/pages/Login.jsx
import { useLocation, useNavigate } from 'react-router-dom';

function Login() {
  const navigate = useNavigate();
  const location = useLocation();
  const from = location.state?.from?.pathname || '/dashboard';

  function handleLogin() {
    // call login API first
    navigate(from, { replace: true });
  }

  return <button onClick={handleLogin}>Login</button>;
}

export default Login;

Role-Based Route Protection

For admin pages, check the role returned by the backend.

File: src/components/AdminRoute.jsx
import { Navigate } from 'react-router-dom';
import { useAuth } from '../context/AuthContext';

function AdminRoute({ children }) {
  const { user, loading } = useAuth();

  if (loading) {
    return <h2>Checking authentication...</h2>;
  }

  if (!user) {
    return <Navigate to="/login" replace />;
  }

  if (user.role !== 'admin') {
    return <Navigate to="/dashboard" replace />;
  }

  return children;
}

export default AdminRoute;

Frontend vs Backend Responsibility

Layer Purpose Security Level
ProtectedRoute Redirect user if not logged in UX only
Auth Context Store current user in React state UX + app state
Backend Middleware Verify token/session Real security
Database/API Return protected data only after auth check Real security

Important Warning

Never depend only on frontend route protection. Users can change localStorage, modify frontend state, or directly call APIs. Always protect backend APIs.

Interview Answer

In React, protected routes are usually implemented using a ProtectedRoute component. It checks whether the user is authenticated and either renders the page or redirects to login using Navigate from React Router. However, checking only localStorage token existence is not real security because users can create fake tokens in DevTools. ProtectedRoute is mainly for user experience. Real security must be enforced on the backend by verifying the token or session before returning protected data.

One-Line Interview Answer

Use ProtectedRoute for frontend UX redirection, but always verify authentication on the backend because frontend checks can be bypassed.
