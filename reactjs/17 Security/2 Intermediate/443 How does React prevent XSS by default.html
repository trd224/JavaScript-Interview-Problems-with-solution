443. How does React prevent XSS by default?

One-Line Definition

React prevents XSS by default by escaping dynamic values before putting them into the DOM, so user input is treated as text, not executable HTML or JavaScript.

Simple Meaning

If a user enters normal text, React displays normal text. If an attacker enters HTML or JavaScript code, React does not run it. React shows it as plain text.

Attacker Input Example

Suppose an attacker writes this in a comment box:

<script>alert('Hacked')</script>

Safe React Rendering

File: Comment.jsx
function Comment({ comment }) {
  return <div>{comment}</div>;
}

export default Comment;
This is safe because React automatically escapes the comment value.

What React Actually Sends To Browser

React converts dangerous characters like < and > into safe HTML entities.

<div>
  &lt;script&gt;alert('Hacked')&lt;/script&gt;
</div>

What User Sees

<script>alert('Hacked')</script>
The browser displays the script as text. It does not execute it.

Why Script Does Not Run

Because the browser does not receive a real script tag. It receives escaped text.

Unsafe Way

React protection can be bypassed if we use dangerouslySetInnerHTML.

File: UnsafeComment.jsx
function UnsafeComment({ comment }) {
  return <div dangerouslySetInnerHTML={{ __html: comment }} />;
}

export default UnsafeComment;
This is dangerous because React inserts the HTML directly without escaping it.

Important Point

React protects only values rendered inside JSX using curly braces. React does not protect you if you manually inject raw HTML into the page.

How To Stay Safe

  • Render user input using JSX curly braces.
  • Avoid dangerouslySetInnerHTML for user-generated data.
  • If HTML must be displayed, sanitize it first using a trusted sanitizer.
  • Never trust data coming from forms, URL params, APIs, comments, or chat messages.

Interview Answer

React prevents XSS by default by escaping dynamic values rendered inside JSX. If a user input contains HTML or JavaScript, React converts special characters into safe text, so the browser displays the content instead of executing it. React can still become vulnerable if developers use dangerouslySetInnerHTML with untrusted data.

One-Line Interview Answer

React prevents XSS by escaping JSX values by default, but raw HTML injection with dangerouslySetInnerHTML can still create XSS risk.
