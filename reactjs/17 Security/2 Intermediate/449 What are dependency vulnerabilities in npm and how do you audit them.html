449. What are dependency vulnerabilities in npm and how do you audit them?

One-Line Definition

Dependency vulnerabilities are security issues found inside npm packages used by your project. You audit them using commands like npm audit and fix them using updates, patches, or package replacement.

Simple Meaning

In a React or Node project, we install many packages from npm.

Example
npm install axios react-router-dom lodash

These packages may also install their own internal packages.

Your Project
↓
Direct Dependencies
↓
Indirect Dependencies
↓
Hundreds Of Packages

What Is A Dependency Vulnerability?

A dependency vulnerability means one of your installed npm packages has a known security problem.

Example problems:

XSS Prototype Pollution Remote Code Execution ReDoS Path Traversal

Direct vs Transitive Dependency

Type Meaning Example
Direct Dependency You installed it manually axios
Transitive Dependency Installed by another package small package used inside axios or build tools
You Install Package A
↓
Package A Installs Package B
↓
Package B Has Vulnerability
↓
Your Project Is Affected

Why Are They Dangerous?

A vulnerable package can affect your application depending on where and how it is used.

Vulnerability Possible Risk
XSS Malicious script can run in browser
Prototype Pollution Object behavior can be changed unexpectedly
Remote Code Execution Attacker may run code on server
ReDoS Regex can slow or crash app
Path Traversal Unauthorized file access

How To Audit npm Dependencies

Terminal
npm audit

This scans your installed packages and reports known vulnerabilities.

Example npm audit Output

Terminal Output
3 vulnerabilities found

Severity:
1 moderate
1 high
1 critical

Run npm audit fix to fix them.

Severity Levels

Severity Meaning
Low Small risk, usually not urgent
Moderate Should be reviewed
High Should be fixed soon
Critical Fix immediately if your project is affected

How To Fix Vulnerabilities

1. Automatic Safe Fix

Terminal
npm audit fix
This applies compatible package updates without breaking major versions.

2. Force Fix

Terminal
npm audit fix --force
Be careful. This can upgrade major versions and may break your project.

3. Manually Update Package

Terminal
npm install package-name@latest

4. Check Outdated Packages

Terminal
npm outdated

5. Replace Unsafe Package

If a package is abandoned or not fixed, replace it with a safer maintained package.

Package Vulnerable
↓
No Fix Available
↓
Package Not Maintained
↓
Replace Package

Useful npm Commands

Command Use
npm audit Show vulnerabilities
npm audit fix Apply safe fixes
npm audit fix --force Apply aggressive fixes, may break app
npm outdated Show old packages
npm ls package-name Find why package is installed

Find Which Package Is Bringing Vulnerable Dependency

Terminal
npm ls vulnerable-package-name
Example Output
my-app
└─┬ react-scripts
  └── vulnerable-package-name

This means the vulnerable package came through react-scripts.

Important: Not Every Audit Warning Means Immediate Danger

A package may be vulnerable, but your app may not be affected if the vulnerable code path is never used.

Still, you should review and fix high/critical issues seriously.

Good Security Practice

  • Run npm audit regularly.
  • Keep dependencies updated.
  • Commit package-lock.json.
  • Avoid abandoned packages.
  • Use trusted packages with active maintenance.
  • Review high and critical vulnerabilities first.
  • Run tests after updating packages.

CI/CD Example

In real projects, npm audit can run during build pipeline.

Terminal
npm audit --audit-level=high
This can fail the build only for high or critical vulnerabilities.

Interview Answer

Dependency vulnerabilities in npm are known security issues in packages installed by a project, including both direct and transitive dependencies. They can cause issues like XSS, prototype pollution, ReDoS, or remote code execution. We audit them using npm audit, fix safe updates with npm audit fix, manually update or replace vulnerable packages, and carefully test the application after updates.

One-Line Interview Answer

npm dependency vulnerabilities are security flaws in installed packages, and we audit them using npm audit, then fix them by updating, patching, or replacing affected packages.
