445. What is CSRF and does React protect against it?

One-Line Definition

CSRF means Cross-Site Request Forgery. It is an attack where a malicious website tricks a logged-in user's browser into sending a request to another website.

Simple Meaning

CSRF does not mean the attacker steals your password directly. The attacker uses your already logged-in browser to perform an action.

User Logged In
↓
Browser Has Login Cookie
↓
User Opens Evil Website
↓
Evil Website Sends Request
↓
Browser Automatically Sends Cookie
↓
Real Website Thinks Request Came From User

Simple Real-Life Example

Imagine you are already logged in to a banking website. Your browser has a session cookie for that bank.

Bank website remembers you because your browser has a login cookie.

Now you open another dangerous website in a different tab. That dangerous website secretly tries to submit a money transfer request to the bank.

The dangerous website cannot directly see your bank data, but it may try to make your browser send a request to the bank.

CSRF Attack Example

Malicious Website HTML
<form action="https://bank.com/transfer" method="POST">
  <input name="amount" value="10000">
  <input name="to" value="attacker">
</form>

<script>
document.forms[0].submit();
</script>

If the user is logged in to the bank, the browser may automatically attach the bank cookie with this request.

Why Does CSRF Work?

Request Sent To Bank
+
Browser Automatically Adds Cookie
↓
Bank Sees Valid Login Session
↓
Bank May Accept Request
Important: CSRF happens mainly when authentication depends on cookies that browsers automatically send with requests.

Does React Protect Against CSRF?

No. React does not automatically protect against CSRF.

React is a frontend UI library. It renders components and handles UI logic. CSRF protection is mainly handled by the server, backend API, and cookie configuration.

Why React Cannot Fully Protect It Alone

React Responsibility Backend Responsibility
Render UI Validate requests
Call API Check CSRF token
Manage frontend state Set secure cookies
Show forms/buttons Reject fake requests

How To Protect Against CSRF

Common protections are:

CSRF Token SameSite Cookie Origin Check Referer Check

1. CSRF Token

A CSRF token is a secret random value generated by the server. React sends it with the request, and the server verifies it.
Server Creates Token
↓
React Receives Token
↓
React Sends Token With POST Request
↓
Server Checks Token
↓
Request Accepted Or Rejected
React Fetch Example
fetch('/api/update-profile', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json', 'X-CSRF-Token': csrfToken },
  body: JSON.stringify({ name: 'Rizwan' })
});

2. SameSite Cookie

SameSite tells the browser when cookies should be sent with cross-site requests.

Server Cookie Example
Set-Cookie: sessionId=abc123; HttpOnly; Secure; SameSite=Lax
SameSite=Lax or SameSite=Strict can reduce CSRF risk because cookies are not freely sent in every cross-site situation.

3. Origin Validation

The backend can check where the request came from.

Backend Logic Idea
if (request.headers.origin !== 'https://myapp.com') {
  return response.status(403).json({ message: 'Invalid origin' });
}

CSRF vs XSS

Topic XSS CSRF
Main Idea Attacker injects JavaScript into page Attacker tricks browser to send request
Problem Malicious script runs Unauthorized request happens
React Default Protection Yes, React escapes JSX values No, React does not automatically stop CSRF
Common Protection Avoid raw HTML, sanitize HTML CSRF token, SameSite cookie, origin check

Easy Way To Remember

XSS
↓
Attacker Runs Code In Your Page

CSRF
↓
Attacker Makes Your Browser Send A Request

Important React Point

React protects against many XSS cases by escaping JSX values, but React does not automatically protect against CSRF. CSRF protection must be added in backend/API security.

Interview Answer

CSRF, or Cross-Site Request Forgery, is an attack where a malicious website tricks a logged-in user's browser into sending an unwanted request to another website. It works because browsers automatically send cookies with requests. React does not automatically protect against CSRF because React is only a UI library. CSRF protection should be implemented using server-side CSRF tokens, SameSite cookies, and origin validation.

One-Line Interview Answer

CSRF tricks a logged-in browser into sending unwanted requests, and React does not protect against it by default; backend CSRF tokens and cookie security are needed.
