446. How do you safely render user-generated HTML in React?

One-Line Definition

To safely render user-generated HTML in React, first sanitize the HTML using a trusted sanitizer like DOMPurify, then render the sanitized output with dangerouslySetInnerHTML.

Simple Meaning

User-generated HTML means HTML that comes from users, database, CMS, blog editor, comment box, or API response.

Never trust HTML that comes from users directly.

Why Is It Dangerous?

An attacker can submit HTML that contains JavaScript.

Attacker Input
<img src="x" onerror="alert('Hacked')">

If you render this directly, the browser may execute the JavaScript.

Directly rendering user HTML can create XSS vulnerabilities.

Unsafe Way

Bad React Example
function Comment({ html }) {
  return <div dangerouslySetInnerHTML={{ __html: html }} />;
}
This is unsafe if html comes from a user, database, CMS, or external API without sanitization.

Safe Flow

User HTML
↓
Sanitize HTML
↓
Remove Dangerous Code
↓
Render Sanitized HTML
↓
Safe Output

Install DOMPurify

Terminal
npm install dompurify

Safe Way Using DOMPurify

File: SafeHtml.jsx
import DOMPurify from 'dompurify';

function SafeHtml({ html }) {
  const cleanHtml = DOMPurify.sanitize(html);
  return <div dangerouslySetInnerHTML={{ __html: cleanHtml }} />;
}

export default SafeHtml;
DOMPurify removes dangerous attributes like onerror, onclick, and script tags.

Example

Input From User
<h2>Hello</h2>
<img src="x" onerror="alert('Hacked')">
<script>alert('Bad')</script>
After Sanitization
<h2>Hello</h2>
<img src="x">

Dangerous JavaScript is removed.

Use Component

File: App.jsx
import SafeHtml from './SafeHtml';

function App() {
  const blogContent = '<h2>Hello</h2><p>This is safe after sanitize.</p>';
  return <SafeHtml html={blogContent} />;
}

export default App;

When Should You Use This?

Use sanitized HTML rendering only when you really need to show HTML formatting.

Use Case Need Sanitization?
Blog content from CMS Yes
Rich text editor content Yes
User bio with HTML Yes
Comment with plain text only No, use normal JSX

Best Option For Plain Text

If you do not need HTML formatting, do not use dangerouslySetInnerHTML.

Safe Plain Text Rendering
function Comment({ text }) {
  return <p>{text}</p>;
}
This is safest because React escapes text automatically.

Safe vs Unsafe

Approach Safe? Reason
<div>{text}</div> Yes React escapes it automatically
dangerouslySetInnerHTML with raw user HTML No Can execute malicious HTML/JS
dangerouslySetInnerHTML with sanitized HTML Better Dangerous parts are removed first

Extra Safety Tips

  • Sanitize HTML before rendering.
  • Prefer plain text rendering when possible.
  • Do not trust API, database, or CMS content blindly.
  • Use Content Security Policy for extra protection.
  • Validate and sanitize on backend also.

Interview Answer

To safely render user-generated HTML in React, avoid rendering raw HTML directly. If HTML rendering is required, sanitize the content using a library like DOMPurify and then pass the sanitized HTML to dangerouslySetInnerHTML. For plain text, use normal JSX interpolation because React escapes it automatically.

One-Line Interview Answer

Use normal JSX for plain text, and if HTML is required, sanitize it first with DOMPurify before using dangerouslySetInnerHTML.
