442. What is dangerouslySetInnerHTML and Why Is It Dangerous?

One-Line Definition

dangerouslySetInnerHTML is a React prop used to insert raw HTML directly into the DOM, and it is dangerous because untrusted HTML can execute malicious JavaScript and cause XSS attacks.

First Understand Normal React Rendering

In React, when you render text inside JSX, React treats it as text, not real HTML.

Safe Example
const comment = "<script>alert('Hacked')</script>";

function App() {
  return <div>{comment}</div>;
}

React escapes the HTML characters.

<div>
  &lt;script&gt;alert('Hacked')&lt;/script&gt;
</div>
The browser displays the script as plain text. It does not execute it.

What dangerouslySetInnerHTML Does

dangerouslySetInnerHTML tells React:

Do not escape this value. Insert it as real HTML inside the page.
Unsafe Example
const comment = "<script>alert('Hacked')</script>";

function App() {
  return <div dangerouslySetInnerHTML={{ __html: comment }} />;
}

Now React directly inserts the HTML.

<div>
  <script>alert('Hacked')</script>
</div>
Now the browser can treat the input as real HTML or JavaScript. This can create an XSS vulnerability.

Why Is It Called Dangerous?

React intentionally uses a long and scary name:

dangerouslySetInnerHTML

Because it is similar to JavaScript's innerHTML:

element.innerHTML = userInput;
Both can be dangerous when the value comes from users, APIs, comments, reviews, profiles, or database content.

When Do Developers Use It?

Sometimes data already contains HTML and developers want to render it as HTML.

  • Blog content from CMS
  • Rich text editor output
  • Formatted article content
  • Email template preview
  • HTML returned from trusted backend
But before rendering HTML, the content must be sanitized.

What Is Sanitization?

Sanitization means removing dangerous HTML and JavaScript before rendering.

Example With DOMPurify
import DOMPurify from 'dompurify';

function Article({ html }) {
  const cleanHtml = DOMPurify.sanitize(html);
  return <div dangerouslySetInnerHTML={{ __html: cleanHtml }} />;
}
DOMPurify removes dangerous scripts, event handlers, and unsafe HTML before rendering.

Simple Rule

Use <div>{value}</div> for normal text. Use dangerouslySetInnerHTML only when you must render real HTML, and only after sanitizing the content.

Interview Answer

dangerouslySetInnerHTML is a React prop that allows developers to insert raw HTML directly into the DOM. It is dangerous because React does not escape that HTML, so if the content contains malicious scripts or unsafe attributes from user input, it can lead to XSS attacks. React normally escapes JSX values, but dangerouslySetInnerHTML bypasses that protection. It should only be used with trusted or sanitized HTML.

One-Line Interview Answer

dangerouslySetInnerHTML renders raw HTML in React, and it is dangerous because untrusted HTML can execute malicious JavaScript and cause XSS.
