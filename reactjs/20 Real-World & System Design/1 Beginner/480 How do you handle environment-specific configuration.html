480. How do you handle environment-specific configuration in React?

One-Line Definition

Environment-specific configuration means using different configuration values for development and production environments without changing the React code.

Why Is It Used?

Different environments usually need different values.

  • Different API URLs
  • Different application names
  • Different feature flags
  • Different analytics settings

Simple Environment Files

.env
.env.development
.env.production
For most React projects, these three files are enough.

Why Do We Need Both Files?

The .env file stores common values used in every environment.

The .env.development file stores values used only during development.

The .env.production file stores values used only in production.

.env (Common Configuration)

REACT_APP_NAME=Aadhaar Authentication
REACT_APP_TIMEOUT=30000
These values are shared by both development and production.

.env.development

REACT_APP_API_URL=https://uat.aadhaarauth.rajasthan.gov.in
Used only when running the application locally with npm start.

.env.production

REACT_APP_API_URL=https://aadhaarauth.rajasthan.gov.in
Used only when creating the production build with npm run build.

What Happens During Build?

When we run: 

npm run build

React first reads: 

.env

Then React reads: 

.env.production

Finally, both files are merged together to create the production configuration.

Variable Override Example

.env

REACT_APP_API_URL=https://common-api.com

.env.production

REACT_APP_API_URL=https://production-api.com

Final value after build: 

REACT_APP_API_URL=https://production-api.com
If the same variable exists in both files, the environment-specific file overrides the value from .env.

Create React App (CRA)

In Create React App, environment variables must start with: 

REACT_APP_

Example

REACT_APP_API_URL=https://api.myapp.com

Usage

const apiUrl = process.env.REACT_APP_API_URL;

Vite

In Vite, environment variables must start with: 

VITE_

Example

VITE_API_URL=https://api.myapp.com

Usage

const apiUrl = import.meta.env.VITE_API_URL;

CRA vs Vite

Feature CRA Vite
Variable Prefix REACT_APP_ VITE_
Access Method process.env import.meta.env

Real Project Example

.env 
REACT_APP_NAME=Aadhaar Authentication
REACT_APP_TIMEOUT=30000
.env.development 
REACT_APP_API_URL=https://uat.aadhaarauth.rajasthan.gov.in
.env.production

This avoids repeating common values in every environment file.

Important Interview Points

  • Use .env for common configuration.
  • Use .env.development for development-only values.
  • Use .env.production for production-only values.
  • React merges .env with the environment-specific file.
  • Environment-specific files override duplicate values.
  • CRA uses REACT_APP_ variables.
  • Vite uses VITE_ variables.
  • Restart the server after changing environment files.

Short Interview Answer

We handle environment-specific configuration using .env files. Common values are stored in .env, while environment-specific values are stored in .env.development and .env.production. React merges these files automatically and environment-specific values override common values when necessary.

One-Line Interview Answer

Environment-specific configuration in React is managed using .env files that provide different settings for development and production environments.
