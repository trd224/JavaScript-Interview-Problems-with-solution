278. What is a Controlled Form Versus an Uncontrolled Form?

One-Line Definition

A controlled form stores form values in React state, whereas an uncontrolled form stores form values in the DOM and accesses them using refs.

Why Do We Have Two Approaches?

React forms can be managed in two different ways:

  • Controlled Form → React manages form values.
  • Uncontrolled Form → Browser DOM manages form values.

Controlled Form Flow

User Types
↓
onChange Event
↓
setState()
↓
React State Updated
↓
Form Re-renders

Controlled Form Example

import { useState } from 'react';

function LoginForm() {

  const [email, setEmail] = useState('');

  return (

    <input
      value={email}
      onChange={e => setEmail(e.target.value)}
    />

  );

}
React always knows the latest value because it is stored in state.

Uncontrolled Form Flow

User Types
↓
DOM Stores Value
↓
No State Updates
↓
Read Value Using Ref

Uncontrolled Form Example

import { useRef } from 'react';

function LoginForm() {

  const emailRef = useRef(null);

  return (

    <input ref={emailRef} />

  );

}
React does not know the current value until we read it from the DOM.

Controlled vs Uncontrolled Forms

Feature Controlled Form Uncontrolled Form
Source of Truth React State DOM
Hook Used useState useRef
Value Access Directly from state Using ref.current.value
Re-render While Typing ✅ Yes ❌ No
Validation Easy Requires DOM access
React Knows Current Value ✅ Yes ❌ No
Most Common in React ✅ Yes ❌ Less Common

Controlled Form Submission

const [email, setEmail] = useState('');

function handleSubmit() {

  console.log(email);

}
React already has the value in state, so no DOM access is required.

Uncontrolled Form Submission

const emailRef = useRef(null);

function handleSubmit() {

  console.log(emailRef.current.value);

}
We must read the value from the DOM during submission.

Validation Example

Controlled Form

const isValid =
email.includes('@');

Uncontrolled Form

const email =
emailRef.current.value;

const isValid =
email.includes('@');
Controlled forms make validation much easier because React always has the latest value.

When Should We Use Controlled Forms?

  • Form validation.
  • Real-time error messages.
  • Live search.
  • Dynamic UI updates.
  • Most React forms.

When Should We Use Uncontrolled Forms?

  • Simple forms.
  • File uploads.
  • Third-party DOM libraries.
  • Reading values only on submit.

File Input Example

const fileRef = useRef(null);

<input
  type="file"
  ref={fileRef}
/>
File inputs are usually implemented as uncontrolled forms.

Real Project Example

In our Aadhaar Authentication application, Aadhaar Number, OTP, Department, Purpose, and Consent forms are controlled because React must validate and process values continuously. File upload fields and some third-party integrations may use uncontrolled forms.

Important Interview Points

  • Controlled forms use useState.
  • Uncontrolled forms use useRef.
  • Controlled forms store values in React state.
  • Uncontrolled forms store values in the DOM.
  • Controlled forms are preferred in React applications.
  • File inputs are usually uncontrolled.

Short Interview Answer

A controlled form manages form values through React state, while an uncontrolled form lets the browser manage values and accesses them through refs.

One-Line Interview Answer

Controlled forms use React state for form values, whereas uncontrolled forms use the DOM and refs.
