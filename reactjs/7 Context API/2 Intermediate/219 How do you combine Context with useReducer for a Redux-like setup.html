219. How do you combine Context with useReducer for a Redux-like setup?

One-Line Definition

We combine Context with useReducer by keeping state update logic inside a reducer and sharing state plus dispatch through Context.

Simple Meaning

Context helps us share data globally.

useReducer helps us manage complex state updates in a predictable way.

Together, Context + useReducer gives a simple Redux-like pattern without installing Redux.

Why Combine Context With useReducer?

  • Context shares state across components.
  • useReducer keeps update logic in one place.
  • dispatch sends actions to update state.
  • Components do not directly modify state.
  • State changes become predictable.

Step 1 - Create Initial State

const initialState = {
  cart: [],
  user: null,
  loading: false
};

This is similar to Redux initial state.

Step 2 - Create Reducer Function

function appReducer(state, action) {
  switch (action.type) {
    case 'ADD_TO_CART':
      return { ...state, cart: [...state.cart, action.payload] };

    case 'REMOVE_FROM_CART':
      return {
        ...state,
        cart: state.cart.filter(item => item.id !== action.payload)
      };

    case 'SET_USER':
      return { ...state, user: action.payload };

    case 'SET_LOADING':
      return { ...state, loading: action.payload };

    default:
      return state;
  }
}
Reducer receives current state and action, then returns new state.

Step 3 - Create Context

import { createContext } from 'react';

export const AppContext = createContext();

Step 4 - Create Provider

import { createContext, useContext, useReducer } from 'react';

const AppContext = createContext();

const initialState = {
  cart: [],
  user: null,
  loading: false
};

function appReducer(state, action) {
  switch (action.type) {
    case 'ADD_TO_CART':
      return { ...state, cart: [...state.cart, action.payload] };

    case 'REMOVE_FROM_CART':
      return { ...state, cart: state.cart.filter(item => item.id !== action.payload) };

    case 'SET_USER':
      return { ...state, user: action.payload };

    case 'SET_LOADING':
      return { ...state, loading: action.payload };

    default:
      return state;
  }
}

export function AppProvider({ children }) {
  const [state, dispatch] = useReducer(appReducer, initialState);

  return (
    <AppContext.Provider value={{ state, dispatch }}>
      {children}
    </AppContext.Provider>
  );
}

export function useAppContext() {
  return useContext(AppContext);
}

Step 5 - Wrap Application With Provider

import { AppProvider } from './AppContext';

function App() {
  return (
    <AppProvider>
      <Dashboard />
    </AppProvider>
  );
}

export default App;

Step 6 - Use State In Component

import { useAppContext } from './AppContext';

function Cart() {
  const { state } = useAppContext();

  return <h2>Cart Items: {state.cart.length}</h2>;
}

export default Cart;

Step 7 - Dispatch Action From Component

import { useAppContext } from './AppContext';

function Product({ product }) {
  const { dispatch } = useAppContext();

  return (
    <button onClick={() => dispatch({ type: 'ADD_TO_CART', payload: product })}>
      Add To Cart
    </button>
  );
}

export default Product;
Component sends action. Reducer decides how state should change.

Complete Small Example

import { createContext, useContext, useReducer } from 'react';

const CartContext = createContext();

const initialState = {
  items: []
};

function cartReducer(state, action) {
  switch (action.type) {
    case 'ADD_ITEM':
      return { ...state, items: [...state.items, action.payload] };

    case 'REMOVE_ITEM':
      return { ...state, items: state.items.filter(item => item.id !== action.payload) };

    case 'CLEAR_CART':
      return { ...state, items: [] };

    default:
      return state;
  }
}

export function CartProvider({ children }) {
  const [state, dispatch] = useReducer(cartReducer, initialState);

  return (
    <CartContext.Provider value={{ state, dispatch }}>
      {children}
    </CartContext.Provider>
  );
}

export function useCart() {
  return useContext(CartContext);
}

Using The Cart Context

function CartPage() {
  const { state, dispatch } = useCart();

  return (
    <>
      <h2>Items: {state.items.length}</h2>

      <button onClick={() => dispatch({ type: 'CLEAR_CART' })}>
        Clear Cart
      </button>
    </>
  );
}

Why This Is Redux-Like?

Because state changes happen through actions. 

dispatch({ type: 'ADD_ITEM', payload: product });

The reducer handles the update. 

case 'ADD_ITEM':
  return { ...state, items: [...state.items, action.payload] };
Components do not directly update complex state. They only dispatch actions.

Real Project Example

Suppose our Aadhaar Authentication module has many related states:

  • Aadhaar number
  • OTP value
  • Selected auth method
  • Loader status
  • Error popup
  • Consent status

Instead of many separate useState calls: 

const [aadhaar, setAadhaar] = useState('');
const [otp, setOtp] = useState('');
const [loading, setLoading] = useState(false);
const [error, setError] = useState(null);

We can manage related state through reducer actions: 

dispatch({ type: 'SET_AADHAAR', payload: value });
dispatch({ type: 'SET_OTP', payload: otp });
dispatch({ type: 'SET_LOADING', payload: true });
dispatch({ type: 'SET_ERROR', payload: message });
This keeps complex state update logic organized and predictable.

When Should You Use Context + useReducer?

Use It When Reason
State is shared by many components Context can share it globally
State update logic is complex Reducer organizes update rules
Many actions update same state dispatch pattern is cleaner
You want Redux-like flow without Redux No external library needed

When Not To Use It

  • For simple local state.
  • For one input field.
  • For state used by only one component.
  • For very large apps where Redux Toolkit/Zustand is better.
  • For server state where React Query/SWR is better.

Important Interview Points

  • Context shares state globally.
  • useReducer manages complex state transitions.
  • dispatch sends actions to reducer.
  • Reducer returns new state.
  • This pattern is similar to Redux.
  • Good for medium-level shared client state.
  • Not a full replacement for Redux in very large apps.

Short Interview Answer

To combine Context with useReducer, create a reducer and initial state, use useReducer inside a Provider, and pass state and dispatch through Context. Components can then read state and dispatch actions, creating a Redux-like predictable state flow without installing Redux.

One-Line Interview Answer

Context provides global access to state, while useReducer provides Redux-like state updates through actions and dispatch.
