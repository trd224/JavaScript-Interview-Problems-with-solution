220. What is the Context Selector Pattern?

One-Line Definition

The Context Selector Pattern is a React optimization pattern that allows a component to subscribe only to the specific part of a Context value it needs, preventing unnecessary re-renders.

Why Do We Need It?

Normally, whenever a Context value changes, every component using useContext() re-renders even if it only needs one property.

const AppContext = createContext();

function UserName() {
  const { user } = useContext(AppContext);
  return <h1>{user.name}</h1>;
}

function ThemeButton() {
  const { theme } = useContext(AppContext);
  return <button>{theme}</button>;
}
If only theme changes, both components re-render because React tracks the entire context value.

Context Selector Pattern

Context
├── user
├── theme
└── cart

UserName → user only
ThemeButton → theme only
CartCount → cart only
  • Changing theme re-renders only ThemeButton.
  • Changing user re-renders only UserName.
  • Changing cart re-renders only CartCount.

Using use-context-selector

npm install use-context-selector
// import { createContext } from 'react';
import { createContext, useContextSelector } from "use-context-selector"; 

const AppContext = createContext(null);

function UserName() {
  const user = useContextSelector(AppContext, value => value.user);
  return <h1>{user.name}</h1>;
}

function ThemeButton() {
  const theme = useContextSelector(AppContext, value => value.theme);
  return <button>{theme}</button>;
}

When Should You Use It?

  • Large contexts with many values.
  • Different components need different data.
  • Frequent updates cause unnecessary re-renders.
  • Performance optimization is important.
Example: If a context contains user, theme, notifications, cart, and language, changing only cart re-renders only the CartBadge component.

One-Line Interview Answer

The Context Selector Pattern lets components subscribe only to the specific context value they need, reducing unnecessary re-renders and improving performance.
