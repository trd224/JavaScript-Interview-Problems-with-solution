221. How does use-context-selector library solve the re-render problem?

Definition

use-context-selector is a React library that allows us to subscribe to only the specific part of a Context value that our component needs. As a result, components re-render only when their selected value changes instead of whenever any value inside the Context changes.

Why does React Context re-render?

Suppose our Context contains multiple values. 


const value = {
  user,
  theme,
  cart
};

Now imagine three components.

Component Uses
UserProfile user
ThemeButton theme
CartBadge cart

When cart changes... 


setCart(newCart);

React creates a new Context value object. 


{
  user,
  theme,
  cart: newCart
}

Since the Context value reference changed, every component using useContext() re-renders, even if it only uses theme or user.

This happens because useContext() subscribes to the entire Context object, not to individual properties.

How does use-context-selector solve this?

Instead of subscribing to the whole Context, each component selects only the value it needs. 


const theme = useContextSelector(
  AppContext,
  value => value.theme
);

Now this component only watches theme.

If cart changes, the selector still returns the same theme value. 


Old:
theme = "dark"

New:
theme = "dark"

Since the selected value didn't change, React skips the re-render.

Example

Create Context


import { createContext } from "use-context-selector";

export const AppContext = createContext(null);

Provider


const [user, setUser] = useState("John");
const [theme, setTheme] = useState("dark");
const [cart, setCart] = useState([]);


  {children}

User Component


import { useContextSelector } from "use-context-selector";

const user = useContextSelector(
  AppContext,
  value => value.user
);

Theme Component


const theme = useContextSelector(
  AppContext,
  value => value.theme
);

Cart Component


const cart = useContextSelector(
  AppContext,
  value => value.cart
);

What happens when cart changes?

Component Selected Value Re-render?
User Component user ❌ No
Theme Component theme ❌ No
Cart Component cart ✅ Yes

How does it work internally?

  1. The Provider stores the Context value.
  2. Each consumer registers its selector function.
  3. When the Provider value changes, the library executes each selector.
  4. It compares the previous selected value with the new selected value using Object.is().
  5. If the selected value is unchanged, the component is not re-rendered.
  6. If the selected value changes, only that component re-renders.

Comparison

useContext() use-context-selector
Subscribes to entire Context Subscribes to selected value
All consumers re-render Only affected consumers re-render
Simple API Slightly more setup
May cause unnecessary renders Avoids unnecessary renders
Interview Tip:

Normal useContext() subscribes to the entire Context object, so every consumer re-renders whenever the Provider value reference changes. The use-context-selector library lets each component subscribe only to the specific value it needs using a selector function. It compares the previously selected value with the new one using Object.is() and re-renders only the components whose selected value has changed.
