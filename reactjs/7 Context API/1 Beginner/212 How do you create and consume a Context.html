212. How Do You Create and Consume a Context?

One-Line Definition

A Context is created using createContext(), provided using Context.Provider, and consumed using useContext().

Three Steps to Use Context

Create Context
↓
Provide Context Value
↓
Consume Context Value

Project Structure

src/

├── context/
│   └── UserContext.js

├── components/
│   └── Profile.js

└── App.js

Step 1 - Create Context

First, create a context using createContext(). 

// context/UserContext.js

import { createContext } from 'react';

export const UserContext = createContext();

What Happens Here?

createContext() creates a Context object that can store shared data for multiple components.

Step 2 - Provide Context Value

Wrap components with UserContext.Provider and pass the value. 

// App.js

import { useState } from 'react';
import { UserContext } from './context/UserContext';
import Profile from './components/Profile';

function App() {

  const [user] = useState('Rizwan');

  return (

    <UserContext.Provider value={user}>

      <Profile />

    </UserContext.Provider>

  );

}

export default App;

What Happens Here?

UserContext.Provider makes the value available to all descendant components.

Step 3 - Consume Context Value

Use useContext() to access the value inside any descendant component. 

// components/Profile.js

import { useContext } from 'react';
import { UserContext } from '../context/UserContext';

function Profile() {

  const user = useContext(UserContext);

  return (

    <h1>{user}</h1>

  );

}

export default Profile;

How Data Flows

App
↓
UserContext.Provider
↓
Profile
↓
useContext(UserContext)
↓
Receives "Rizwan"

Using Context in Deeply Nested Components

App
│
├── Dashboard
│
├── Sidebar
│
└── Profile
Profile can directly access the context value without Dashboard and Sidebar passing props.

What Happens When Context Value Changes?

const [user, setUser] = useState('Rizwan');

setUser('Ali');
Context Value Changes
↓
React Updates Context
↓
Components Using useContext()
↓
Re-render Automatically

Common Mistake

// ❌ Error

const user = useContext(UserContext);

// No Provider exists above this component
A component can consume a context only if a matching Provider exists above it in the component tree.

Real Project Example

In our Aadhaar Authentication application, we can create an AuthContext to store logged-in user details, department information, permissions, and authentication settings. Any component can access this data directly using useContext() without prop drilling.

Important Interview Points

  • Context is created using createContext().
  • Data is provided using Context.Provider.
  • Data is consumed using useContext().
  • Provider must be above the consuming component.
  • Context helps avoid prop drilling.
  • Context updates re-render consuming components.

Short Interview Answer

To use Context API, first create a context using createContext(), provide data through Context.Provider, and access the data using useContext() in descendant components.

One-Line Interview Answer

Context is created with createContext(), shared through Provider, and consumed using useContext().
