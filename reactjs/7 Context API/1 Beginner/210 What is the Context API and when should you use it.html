210. What is the Context API and When Should You Use It?

One-Line Definition

Context API is a React feature that allows us to share data across multiple components without passing props manually through every level of the component tree.

Why Was Context API Introduced?

In React, data normally flows from parent to child using props.

When deeply nested components need the same data, we may end up passing props through many intermediate components.

This problem is called Prop Drilling.

Problem Without Context API

App
↓
Dashboard
↓
Sidebar
↓
UserProfile

If UserProfile needs the user data, every component in between must receive and pass the prop. 

<App user={user} />

      ↓

<Dashboard user={user} />

      ↓

<Sidebar user={user} />

      ↓

<UserProfile user={user} />

Solution With Context API

Context Provider
↓
Stores Shared Data
↓
Any Child Component
↓
Accesses Data Directly

Creating Context

import { createContext } from 'react';

export const UserContext = createContext();

Step 1 - Create Provider

import { useState } from 'react';
import { UserContext } from './UserContext';

function App() {

  const [user] = useState('Rizwan');

  return (

    <UserContext.Provider value={user}>

      <Profile />

    </UserContext.Provider>

  );

}

Step 2 - Consume Context

import { useContext } from 'react';
import { UserContext } from './UserContext';

function Profile() {

  const user = useContext(UserContext);

  return (

    <h1>{user}</h1>

  );

}

What Happens Internally?

Provider Value Changes
↓
React Updates Context
↓
Components Using useContext
↓
Re-render Automatically

When Should You Use Context API?

Good Use Cases Example
User Information Logged-in User
Theme Dark / Light Mode
Language English / Hindi
Authentication User Token
Application Settings Global Config

When NOT To Use Context API?

Context API is not intended for large and frequently changing application state.

For complex state management, libraries such as Redux, Zustand, or MobX may be more suitable.

Context API vs Props

Props Context API
Parent → Child Only Any Descendant Component
Can Cause Prop Drilling Avoids Prop Drilling
Best For Local Data Best For Shared Data

Real Project Example

In our Aadhaar Authentication application, Context API can be used to share the logged-in user information, department details, theme settings, language settings, and authentication configuration across many components without passing props through every level.

Important Interview Points

  • Context API helps avoid prop drilling.
  • Created using createContext().
  • Data is provided using Provider.
  • Data is consumed using useContext().
  • Context updates re-render consuming components.
  • Not a full replacement for Redux or Zustand.

Short Interview Answer

Context API is React's built-in mechanism for sharing data across multiple components without manually passing props through every level of the component tree.

One-Line Interview Answer

Context API allows us to share global data between components without prop drilling.
