214. How Do You Consume Context with useContext?

One-Line Definition

useContext() is a React hook that allows us to read and use a Context value directly inside a component.

Why Do We Need useContext?

A Context Provider makes data available to child components.

To access that shared data, we use the useContext() hook.

Context Provider
↓
Stores Value
↓
useContext()
↓
Reads Value

Prerequisite

A Context must already exist and be provided by a Provider. 

// UserContext.js

import { createContext } from 'react';

export const UserContext = createContext();
// App.js

<UserContext.Provider value={'Rizwan'}>

  <Profile />

</UserContext.Provider>

Step 1 - Import useContext

import { useContext } from 'react';

Step 2 - Import the Context

import { UserContext } from '../context/UserContext';

Step 3 - Consume the Context

function Profile() {

  const user = useContext(UserContext);

  return (

    <h1>{user}</h1>

  );

}

What Happens Internally?

useContext(UserContext)
↓
React Finds Nearest Provider
↓
Reads Provider Value
↓
Returns Value

Complete Example

// context/UserContext.js

import { createContext } from 'react';

export const UserContext = createContext();
// App.js

import { UserContext } from './context/UserContext';
import Profile from './Profile';

function App() {

  return (

    <UserContext.Provider value={'Rizwan'}>

      <Profile />

    </UserContext.Provider>

  );

}
// Profile.js

import { useContext } from 'react';
import { UserContext } from './context/UserContext';

function Profile() {

  const user = useContext(UserContext);

  return (

    <h1>Hello {user}</h1>

  );

}

export default Profile;

Hello Rizwan

What Happens When Context Changes?

setUser('Ali');
Provider Value Changes
↓
Context Updates
↓
useContext Receives New Value
↓
Component Re-renders

Common Mistake

// ❌ Provider does not exist

const user = useContext(UserContext);
useContext() only works when a matching Provider exists above the component in the component tree.

Real Project Example

In our Aadhaar Authentication application, components such as Header, Dashboard, Audit Forms, and Reports can access logged-in user information, department details, permissions, and theme settings directly using useContext() without receiving props from every intermediate component.

Important Interview Points

  • useContext() is used to consume Context values.
  • React automatically finds the nearest matching Provider.
  • It eliminates the need for prop drilling.
  • Context updates cause consuming components to re-render.
  • useContext() works only inside React components or custom hooks.

Short Interview Answer

useContext() allows a component to access data from the nearest matching Context Provider without manually passing props through intermediate components.

One-Line Interview Answer

useContext() reads and uses the value stored in a Context Provider.
