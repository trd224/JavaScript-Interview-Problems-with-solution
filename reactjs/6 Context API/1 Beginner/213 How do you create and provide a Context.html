213. How Do You Create and Provide a Context?

One-Line Definition

A Context is created using createContext() and provided to child components using Context.Provider.

Why Do We Create a Context?

Context allows us to share data with multiple components without passing props through every level of the component tree.

↓

↓

↓

Step 1 - Create a Context

First, create a context using createContext(). 

// context/UserContext.js

import { createContext } from 'react';

export const UserContext = createContext();

What Does createContext() Return?

createContext() returns a Context object that contains a Provider and allows components to share data. 
const UserContext = createContext();

UserContext.Provider

Step 2 - Import the Context

// App.js

import { UserContext } from './context/UserContext';

Step 3 - Provide a Value

Wrap child components inside the Provider and pass the value prop. 

import { useState } from 'react';
import { UserContext } from './context/UserContext';
import Profile from './Profile';

function App() {

  const [user] = useState('Rizwan');

  return (

    <UserContext.Provider value={user}>

      <Profile />

    </UserContext.Provider>

  );

}

export default App;

Provider Can Wrap Multiple Components

<UserContext.Provider value={user}>

  <Header />

  <Sidebar />

  <Profile />

</UserContext.Provider>
Every component inside the Provider can access the context value.

Provider Can Pass Objects Too

const user = {

  name: 'Rizwan',
  role: 'Admin'

};

<UserContext.Provider value={user}>

  <Profile />

</UserContext.Provider>

Provider Can Pass State and Functions

const [theme, setTheme] = useState('dark');

<ThemeContext.Provider

  value={{ theme, setTheme }}

>

  <App />

</ThemeContext.Provider>
A Provider can share values, objects, state variables, and functions.

What Happens When Provider Value Changes?

Real Project Example

In our Aadhaar Authentication application, we can create an AuthContext and provide logged-in user information, department details, permissions, language settings, and theme settings to all child components from a single Provider.

Important Interview Points

  • Context is created using createContext().
  • Provider shares the value with descendant components.
  • The shared data is passed through the value prop.
  • A Provider can share primitive values, objects, state, and functions.
  • When the Provider value changes, consuming components re-render.

Short Interview Answer

We create a Context using createContext() and provide data to child components by wrapping them inside Context.Provider and passing the value prop.
