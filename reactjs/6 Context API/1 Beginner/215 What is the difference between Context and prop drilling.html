215. What is the Difference Between Context and Prop Drilling?

One-Line Definition

Prop drilling is the process of passing props through multiple intermediate components, whereas Context allows us to share data directly with descendant components without manually passing props.

What is Prop Drilling?

Prop drilling occurs when data must travel through several components before reaching the component that actually needs it.

App
↓
Dashboard
↓
Sidebar
↓
Profile

If Profile needs user data, every component in the chain must receive and pass the prop. 

// App.js

<Dashboard user={user} />
// Dashboard.js

<Sidebar user={user} />
// Sidebar.js

<Profile user={user} />
// Profile.js

<h1>{user}</h1>

Problems with Prop Drilling

  • Extra code in intermediate components.
  • Difficult to maintain large component trees.
  • Components receive props they do not use.
  • Refactoring becomes harder.

How Context Solves This Problem

Context stores shared data in a Provider so descendant components can access it directly.

Context Provider
↓
Shared Data
↓
Profile
↓
useContext() 
// App.js

<UserContext.Provider value={user}>

  <Dashboard />

</UserContext.Provider>
// Profile.js

const user = useContext(UserContext);

return (

  <h1>{user}</h1>

);

Should Context Replace Props?

No.

Props are still the preferred way to pass data between closely related parent and child components.

Context should be used when many components across the application need the same data.

When to Use Props?

  • Parent and child are directly connected.
  • Only a few components need the data.
  • Data is local to a feature.

When to Use Context?

  • User information.
  • Authentication state.
  • Theme settings.
  • Language settings.
  • Global application configuration.

Real Project Example

In our Aadhaar Authentication application, passing logged-in user details from App → Dashboard → Audit → Report → Header would be prop drilling.

Using AuthContext allows Header, Audit, Report, and other components to access user information directly through useContext() without passing props through intermediate components.

Important Interview Points

  • Prop drilling is a problem, not a feature.
  • Context API helps solve prop drilling.
  • Props are still useful for local component communication.
  • Context is best for shared application-wide data.
  • Context updates re-render consuming components.

Short Interview Answer

Prop drilling means passing data through multiple intermediate components using props, while Context allows components to access shared data directly without manually passing props through every level.

One-Line Interview Answer

Prop drilling passes data through many components, whereas Context shares data directly with descendant components.
