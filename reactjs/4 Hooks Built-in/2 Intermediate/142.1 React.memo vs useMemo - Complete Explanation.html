143. React.memo vs useMemo (Complete Explanation)

One-Line Definition

useMemo caches a value between renders, while React.memo caches an entire component and skips re-rendering it when its props stay the same. They solve two different problems, which is why beginners often confuse them.

This guide explains, step by step:

  • Why useMemo does not stop a component from re-rendering.
  • Why we still need React.memo even after using useMemo.
  • How both work together in real applications.

Step 1 — What is Re-rendering?

Whenever React executes a component function again, we say that the component has re-rendered. 

function App() {
  console.log('App Rendered');
  return <h1>Hello</h1>;
}

Whenever React calls App(), the component renders. If React calls App() again, the component re-renders.

Step 2 — When Does React Re-render a Component?

A component usually re-renders when:

  • State changes
  • Props change
  • Context changes
  • Its parent component re-renders
const [count, setCount] = useState(0);

When setCount(count + 1) is executed, React immediately runs App() again.

Notice that React does not ask whether useMemo exists. It simply re-renders because the component's state changed.

Step 3 — What Does useMemo() Actually Do?

Suppose we write: 

const users = ['Rizwan', 'Ali', 'John'];

Every render creates a brand new array.

Render 1
↓
users → Array A

Render 2
↓
users → Array B

Although the values are identical (["Rizwan","Ali","John"]), the references are different.

Using useMemo()

const users = useMemo(() => {
  return ['Rizwan', 'Ali', 'John'];
}, []);

Now:

Render 1
↓
users → Array A

Render 2
↓
users → Array A
useMemo() prevents creating a new array. It keeps the same reference until dependencies change.

Step 4 — Does useMemo Stop the Component From Re-rendering?

No. It only memoizes the value. It does not stop the component itself from rendering. 

function App() {
  const [count, setCount] = useState(0);

  const users = useMemo(() => {
    console.log('Creating Array');
    return ['Rizwan', 'Ali'];
  }, []);

  console.log('App Rendered');

  return (
    <button onClick={() => setCount(count + 1)}>
      {count}
    </button>
  );
}

Initial Render

Creating Array
App Rendered

Click Button

App Rendered

Notice that Creating Array did not execute again, because useMemo reused the previous array. However, App Rendered still appeared.

This proves that useMemo does not stop App from re-rendering. It only prevents recreating the memoized value.

Step 5 — Now Let's Add a Child Component

function UserList({ users }) {
  console.log('UserList Rendered');
  return <h1>{users.length}</h1>;
}

Parent: 

<UserList users={users} />

Now click the Increment button.

App Rendered
↓
React renders UserList 
Console:

App Rendered
UserList Rendered
Even though useMemo returned the same array, the child still rendered.

Why?

Because React's default behavior is:

Parent Rendered
↓
Render Every Child

React does not compare props automatically.

Step 6 — Then Why Do We Need React.memo()?

At this point, a common question is:

Question: We already used useMemo(). It keeps the same array reference. So why do we still need React.memo()?

The answer is simple.

useMemo() only creates (or reuses) the value. It never tells React to skip rendering a component.

Without React.memo(), React's rendering algorithm is simply:

Parent Rendered
↓
Render Child
↓
Done

Notice something important — there is no props comparison. React simply executes the child component again.

Step 7 — What Does React.memo() Do?

Now wrap the child: 

const UserList = React.memo(function UserList({ users }) {
  console.log('UserList Rendered');
  return <h1>{users.length}</h1>;
});

Now the rendering process changes.

Parent Rendered
↓
React.memo
↓
Compare Previous Props
↓
Same?
↓
Yes
↓
Skip Rendering
React.memo adds a new rule: before rendering this component, compare its previous props with its current props.

Step 8 — How React.memo() Compares Props

Suppose during the first render:

users → Array A

After clicking Increment, because of useMemo():

users → Array A

Again React reaches <UserList users={users} />. This time React.memo internally performs something similar to: 

previousProps.users === nextProps.users

which becomes: 

ArrayA === ArrayA // true

Since nothing changed, React skips rendering the child.

App Rendered
↓
React.memo
↓
Props Same
↓
Skip UserList()

Step 9 — What Happens Without useMemo()?

Suppose we remove useMemo: 

const users = ['Rizwan', 'Ali', 'John'];

Now every render creates a new array.

Render 1 → Array A

Render 2 → Array B

Now React.memo compares: 

previousProps.users === nextProps.users
// ArrayA === ArrayB → false

React thinks the prop changed. Therefore:

Render Child Again

↓
useMemo()
↓
Same Array Returned

React.memo()
↓

↓

This is why these two APIs are commonly used together.

Step 11 — Responsibilities

API Responsibility
useMemo() Keeps the same object, array, or calculated value until dependencies change.
React.memo() Uses the stable props to decide whether the component should render again.

Step 12 — Teacher & Student Analogy

Imagine:

  • Teacher = Parent Component
  • Student = Child Component
  • Notebook = users Array

Without React.memo()

Teacher
↓
Calls Student
↓
Student Comes
↓
Reads Notebook

Even if the notebook is exactly the same, the teacher still calls the student every time.

With React.memo()

Teacher
↓
Calls Security Guard
↓
Guard Checks Notebook
↓
Same Notebook?
↓
Yes
↓
Don't Call Student

Here, useMemo() keeps the notebook the same, and React.memo() is the security guard that checks whether the notebook changed.

Step 13 — Why Can't useMemo() Skip Rendering by Itself?

Because useMemo() has only one responsibility:

Create Value
or
Reuse Previous Value

It does not know anything about rendering child components. It never tells React "Don't render UserList." That responsibility belongs to React.memo().

Final Summary

Question Answer
Does useMemo stop App from rendering? No.
Does useMemo keep the same reference? Yes.
Does React compare props by default? No.
Who compares props? React.memo()
Who skips child rendering? React.memo()
Why use useMemo then? To provide stable references (objects, arrays, calculations) so React.memo() can detect that the props haven't changed.

Easy Way To Remember

useMemo → Keeps the Value Stable

React.memo → Skips Re-render When Props Are Stable

Interview Answer

useMemo() memoizes a value and keeps its reference stable, whereas React.memo() memoizes a component and skips its re-render when its props remain unchanged. They are often used together because useMemo provides stable prop references, and React.memo uses those references to determine whether the child component should render again.

One-Line Interview Answer

useMemo caches a value so it doesn't change between renders; React.memo uses that stable value to skip re-rendering a child component whose props haven't actually changed.
