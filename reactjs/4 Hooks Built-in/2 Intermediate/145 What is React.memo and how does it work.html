145. What is React.memo and How Does It Work?

One-Line Definition

React.memo is a Higher Order Component (HOC) that prevents a component from re-rendering when its props have not changed.

Why Do We Need React.memo?

Normally, when a parent component re-renders, all child components re-render too.

Even if the child's props are exactly the same.

Example

function Child() {

  console.log('Child Rendered');

  return <h2>Child Component</h2>;

}
import { useState } from 'react';

export default function App() {

  const [count, setCount] = useState(0);

  return (
    <>
      <h2>Count: {count}</h2>

      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>

      <Child />
    </>
  );

}

What Happens?

Count Changes
↓
App Re-renders
↓
Child Re-renders

Even though Child uses no state and receives no props.

React automatically re-renders child components whenever the parent re-renders.

How React.memo Helps

Wrap the component: 

const Child = React.memo(function Child() {

  console.log('Child Rendered');

  return <h2>Child Component</h2>;

});

Now What Happens?

Count Changes
↓
App Re-renders
↓
React.memo Checks Props
↓
Props Same?
↓
Yes
↓
Skip Child Re-render
React.memo remembers the previous props and compares them with the new props.

How React.memo Works Internally

Imagine: 

<Child name="Rizwan" />

First render: 

Previous Props

{
  name: "Rizwan"
}

Second render: 

New Props

{
  name: "Rizwan"
}

React.memo performs a shallow comparison: 

"Rizwan" === "Rizwan"

true

Since props are equal: 

Skip Re-render

Real Example

Without React.memo

function Child({ name }) {

  console.log('Child Rendered');

  return <h2>{name}</h2>;

}
import { useState } from 'react';

export default function App() {

  const [count, setCount] = useState(0);

  return (
    <>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>

      <Child name="Rizwan" />
    </>
  );

}

Console: 

Child Rendered
Child Rendered
Child Rendered
Child Rendered

With React.memo

const Child = React.memo(function Child({ name }) {

  console.log('Child Rendered');

  return <h2>{name}</h2>;

});

Console: 

Child Rendered

Only once.

The Most Important Limitation

React.memo only compares props.

If component state changes: 

const Child = React.memo(function Child() {

  const [count, setCount] = useState(0);

  return (
    <button onClick={() => setCount(count + 1)}>
      {count}
    </button>
  );

});

React.memo cannot stop: 

State Update
↓
Component Re-render
React.memo only skips renders caused by unchanged props.

The Function Prop Problem

Many developers use React.memo but still see re-renders.

Example

const Child = React.memo(function Child({ onClick }) {

  console.log('Child Rendered');

  return (
    <button onClick={onClick}>
      Click
    </button>
  );

});
function App() {

  const [count, setCount] = useState(0);

  const handleClick = () => {
    console.log('Clicked');
  };

  return (
    <Child onClick={handleClick} />
  );

}

Every render creates: 

New Function Reference

React.memo sees: 

Old Function !== New Function

Result: 

Child Re-renders

Solution: useCallback

const handleClick = useCallback(() => {
  console.log('Clicked');
}, []);
React.memo and useCallback are commonly used together.

Shallow Comparison

React.memo performs shallow comparison.

Value Comparison Result
"Rizwan" Compared by value
100 Compared by value
true Compared by value
{} Compared by reference
[] Compared by reference
function(){} Compared by reference

Common Mistake

const users = [
  'Rizwan',
  'Ali'
];

<Child users={users} />

Each render creates: 

New Array Reference

React.memo sees: 

Old Array !== New Array

Child re-renders.

Fix

const users = useMemo(() => {
  return ['Rizwan', 'Ali'];
}, []);
React.memo often works together with useMemo and useCallback.

When Should You Use React.memo?

  • Expensive child components.
  • Large lists.
  • Complex UI trees.
  • Frequently re-rendering parent components.

When Should You NOT Use React.memo?

  • Very small components.
  • Components that always receive changing props.
  • Components that rarely re-render.
React.memo itself performs prop comparisons. Using it everywhere can actually reduce performance.

Easy Way To Remember

Parent Re-render
↓
React.memo Checks Props
↓
Props Changed?
↓
Yes → Re-render

No → Skip Re-render

Interview Answer

React.memo is a Higher Order Component that memoizes a component and skips re-rendering when its props remain unchanged. It improves performance by avoiding unnecessary renders caused by parent component updates.

One-Line Interview Answer

React.memo prevents unnecessary component re-renders by performing a shallow comparison of props.
