151. How Do You Avoid Infinite Loops in useEffect?

One-Line Definition

Infinite loops in useEffect happen when the effect updates something that causes the effect to run again repeatedly. To avoid them, use correct dependency arrays and avoid unnecessary state updates inside effects.

What Is An Infinite Loop?

An infinite loop happens when:

useEffect Runs
↓
Updates State
↓
Component Re-renders
↓
useEffect Runs Again
↓
Updates State Again
↓
Forever...
This can crash the application or cause the browser to freeze.

Most Common Infinite Loop

Wrong Example

function App() {

  const [count, setCount] = useState(0);

  useEffect(() => {

    setCount(count + 1);

  });

}

What Happens?

Render
↓
useEffect Runs
↓
setCount()
↓
Re-render
↓
useEffect Runs Again
↓
setCount()
↓
Repeat Forever

Because: 

useEffect(...)

without a dependency array runs after every render.

Fix #1 - Use Dependency Array

useEffect(() => {

  setCount(count + 1);

}, []);

Now:

First Render
↓
Effect Runs Once
↓
State Updates
↓
One More Render
↓
Effect Does Not Run Again
The effect runs only once after component mount.

Another Common Infinite Loop

Wrong Example

function App() {

  const [count, setCount] = useState(0);

  useEffect(() => {

    setCount(count + 1);

  }, [count]);

}

What Happens?

count Changes
↓
Dependency Changed
↓
Effect Runs
↓
setCount()
↓
count Changes Again
↓
Repeat Forever
The effect is updating the same dependency it is watching.

Fix #2 - Add A Condition

useEffect(() => {

  if (count < 5) {
    setCount(count + 1);
  }

}, [count]);

Now: 

0 → 1 → 2 → 3 → 4 → 5

Then it stops.

Function Dependency Loop

Wrong Example

function App() {

  const fetchUsers = () => {
    console.log('Fetch');
  };

  useEffect(() => {

    fetchUsers();

  }, [fetchUsers]);

}

Problem

Every render creates: 

New Function Reference

React sees: 

Dependency Changed

Effect runs again.

Functions are recreated on every render.

Fix #3 - useCallback

const fetchUsers = useCallback(() => {

  console.log('Fetch');

}, []);

useEffect(() => {

  fetchUsers();

}, [fetchUsers]);
The function reference remains stable.

Object Dependency Loop

Wrong Example

const options = {
  page: 1
};

useEffect(() => {

  console.log(options);

}, [options]);

Problem

Every render creates: 

New Object Reference

React sees: 

Dependency Changed

Effect runs again.

Fix #4 - useMemo

const options = useMemo(() => {

  return {
    page: 1
  };

}, []);

useEffect(() => {

  console.log(options);

}, [options]);
The object reference remains stable.

Array Dependency Loop

Wrong Example

const users = [
  'Rizwan',
  'Ali'
];

useEffect(() => {

  console.log(users);

}, [users]);

Each render creates: 

New Array Reference

Fix #5 - useMemo

const users = useMemo(() => {

  return [
    'Rizwan',
    'Ali'
  ];

}, []);

useEffect(() => {

  console.log(users);

}, [users]);

API Fetch Loop

Wrong Example

useEffect(() => {

  fetch('/api/users')
    .then(res => res.json())
    .then(data => {
      setUsers(data);
    });

}, [users]);

Problem

Fetch Data
↓
setUsers()
↓
users Changes
↓
Dependency Changed
↓
Fetch Again
↓
Repeat Forever

Fix #6

useEffect(() => {

  fetch('/api/users')
    .then(res => res.json())
    .then(data => {
      setUsers(data);
    });

}, []);
Fetch only once when the component mounts.

Golden Rule

Never update a dependency inside an effect unless you intentionally want the effect to run again.

How To Detect Infinite Loops

Common symptoms:

  • Console logs continuously.
  • API requests repeat forever.
  • CPU usage becomes high.
  • Browser freezes.
  • "Maximum update depth exceeded" error appears.

Most Common React Error

Maximum update depth exceeded.

This usually means:

State Update
↓
Re-render
↓
State Update
↓
Re-render
↓
Infinite Loop

Interview Answer

Infinite loops in useEffect usually occur when an effect updates state that is also listed in its dependency array, causing the effect to continuously re-run. They can be avoided by using correct dependency arrays, stabilizing functions with useCallback, stabilizing objects and arrays with useMemo, and avoiding unnecessary state updates inside effects.

One-Line Interview Answer

Avoid infinite loops in useEffect by ensuring effects do not continuously update the dependencies that trigger them.
