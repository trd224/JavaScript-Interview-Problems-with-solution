143. What is useCallback and Why is it Needed?

One-Line Definition

useCallback is a React hook that remembers a function and returns the same function reference between renders until its dependencies change.

Why Most Developers Get Confused

Most tutorials say: 

const handleClick = useCallback(() => {
  console.log('Clicked');
}, []);

But they never explain what problem this actually solves.

The real purpose of useCallback is NOT to make a function run faster.

The real purpose is to keep the same function reference between renders.

First Understand the Real Problem

Consider this simple application: 

import { useState } from 'react';

function Child({ onClick }) {

  console.log('Child Rendered');

  return (
    <button onClick={onClick}>
      Child Button
    </button>
  );

}

export default function App() {

  const [count, setCount] = useState(0);

  const handleClick = () => {
    console.log('Clicked');
  };

  return (
    <>
      <h2>{count}</h2>

      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>

      <Child onClick={handleClick} />
    </>
  );

}

What Happens During First Render?

React creates: 

handleClick → Function #101

Child receives: 

onClick = Function #101

User Clicks Increment

setCount(count + 1);

App re-renders.

React runs the entire component again.

This line runs again: 

const handleClick = () => {
  console.log('Clicked');
};

React creates a completely new function: 

Function #102

Next Render

Function #103

Next Render

Function #104
Even though the code is identical, React creates a new function object every render.

Why Is This Bad?

Normally it is not a problem.

But now let's optimize Child with React.memo. 

const Child = React.memo(function Child({ onClick }) {

  console.log('Child Rendered');

  return (
    <button onClick={onClick}>
      Child Button
    </button>
  );

});

React.memo says:

Props Changed?
↓
Yes → Re-render

No → Skip Re-render

Problem

First render: 

onClick = Function #101

Second render: 

onClick = Function #102

React compares: 

#101 === #102

Result: 

false

React thinks: 

Props Changed

Therefore: 

Child Re-renders
React.memo cannot help because the function reference changes every render.

How useCallback Fixes It

const handleClick = useCallback(() => {
  console.log('Clicked');
}, []);

First Render

Function #101

Second Render

Dependencies: 

[]

Changed? 

No

React returns: 

Function #101

Third Render

Function #101

Fourth Render

Function #101
React keeps returning the same function reference.

Now React.memo Works

React compares: 

#101 === #101

Result: 

true

Props did not change.

Child skips re-render.

Parent Re-render
↓
Same Function Reference
↓
Props Unchanged
↓
React.memo Works
↓
Child Skips Re-render

Another Common Use Case

Function Inside useEffect

function App() {

  const fetchUsers = () => {
    console.log('Fetch Users');
  };

  useEffect(() => {
    fetchUsers();
  }, [fetchUsers]);

}

Problem:

Render
↓
New Function
↓
Dependency Changed
↓
useEffect Runs Again

Possibly forever.

Solution

const fetchUsers = useCallback(() => {
  console.log('Fetch Users');
}, []);

useEffect(() => {
  fetchUsers();
}, [fetchUsers]);
Now the function reference remains stable.

When Should You Use useCallback?

  • Passing functions to React.memo components.
  • Function is used inside useEffect dependencies.
  • Function is used inside useMemo dependencies.
  • Large applications with expensive child renders.

When Should You NOT Use useCallback?

Do not wrap every function. 

const sayHello = useCallback(() => {
  alert('Hello');
}, []);

Usually unnecessary.

useCallback itself has overhead. Use it only when stable function references actually matter.

The Biggest Interview Point

Many developers incorrectly say: 

useCallback prevents re-renders

Wrong.

useCallback does NOT prevent component re-renders.

It only preserves the same function reference between renders.

Interview Answer

useCallback is a React hook that memoizes a function and returns the same function reference until its dependencies change. It is mainly used when passing functions to React.memo components or when functions are used in dependency arrays such as useEffect.

One-Line Interview Answer

useCallback caches a function reference so React can reuse the same function between renders until dependencies change.
