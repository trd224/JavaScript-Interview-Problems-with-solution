142. What is useMemo and When Should You Use It?

One-Line Definition

useMemo is a React hook that caches (memorizes) the result of a calculation and only recalculates it when its dependencies change.

Simple Meaning

Normally React re-runs the entire component whenever state changes.

That means every variable, function call, filter, sort, or calculation inside the component runs again. 

function App() {

  const total = calculateTotal();

  return <h2>{total}</h2>;

}

Every re-render calls: 

calculateTotal();

even if the data used by the calculation has not changed.

This can become expensive when calculations are heavy.

What Problem Does useMemo Solve?

Suppose we have 10,000 products. 

const filteredProducts = products.filter(
  product => product.price > 1000
);

Every component re-render runs the filter again.

Component Re-renders
↓
Filter Runs Again
↓
Filter Runs Again
↓
Filter Runs Again

Even if products never changed.

useMemo can cache the result.

Syntax

const memoizedValue = useMemo(() => {
    return expensiveCalculation();
  },
  [dependencies]
);

Parameters

Parameter Purpose
Callback Function Returns the value to cache
Dependency Array Determines when recalculation should happen

Basic Example

import { useMemo } from 'react';

function App() {

  const total = useMemo(() => {
    return 10 + 20;
  }, []);

  return <h2>{total}</h2>;

}

The value is calculated only once.

Real Example Without useMemo

function ProductList({ products }) {

  const expensiveProducts = products.filter(
    product => product.price > 1000
  );

  return (
    <div>
      {expensiveProducts.length}
    </div>
  );

}

Every re-render: 

Component Re-renders
       ↓
filter()
       ↓
Runs Again

Same Example With useMemo

import { useMemo } from 'react';

function ProductList({ products }) {

  const expensiveProducts = useMemo(() => {

    return products.filter(
      product => product.price > 1000
    );

  }, [products]);

  return (
    <div>
      {expensiveProducts.length}
    </div>
  );

}

Now the filter runs only when: 

products changes
If products stay the same, React reuses the cached value.

Real Search Example

function Search({ products, search }) {

  const filteredProducts = useMemo(() => {

    return products.filter(product => product.name.toLowerCase().includes(search.toLowerCase()));

  }, [products, search]);

  return (
    <ul>
      {filteredProducts.map(product => (
        <li key={product.id}>
          {product.name}
        </li>
      ))}
    </ul>
  );

}

Filtering only happens when:

  • products changes
  • search changes

Common useMemo Use Cases

  • Filtering large arrays
  • Sorting large arrays
  • Complex calculations
  • Generating expensive derived data
  • Preventing unnecessary re-renders with React.memo

Important Example: React.memo + useMemo

Parent Component

const users = useMemo(() => {
  return [
    'Rizwan',
    'Ali',
    'John'
  ];
}, []);

Child Component

const UserList = React.memo(
  function UserList({ users }) {
    return <div>{users.length}</div>;
  }
);
useMemo keeps the same array reference, helping React.memo avoid unnecessary child re-renders.

When Should You Use useMemo?

Use useMemo when:

  • Calculation is expensive.
  • Large arrays are filtered or sorted.
  • Derived data is expensive to compute.
  • React.memo optimization needs stable references.

When Should You NOT Use useMemo?

Do not use useMemo for simple calculations.

Bad Example

const total = useMemo(() => {
  return 5 + 10;
}, []);
useMemo itself has overhead. Simple calculations are usually faster than memoization.

Better

const total = 5 + 10;

Most Common Beginner Mistake

Many developers think useMemo prevents component re-renders.

useMemo does NOT stop re-renders.

It only prevents recalculating a value.

Wrong Thinking

useMemo
     ↓
Stops Component Re-render

Correct Thinking

useMemo
     ↓
Caches Calculation Result

How useMemo Compares to useCallback

Hook Returns
useMemo Memoized Value
useCallback Memoized Function

useMemo

const total = useMemo(() => {
  return price * quantity;
}, [price, quantity]);

useCallback

const handleClick = useCallback(() => {
  console.log('Clicked');
}, []);

Interview Answer

useMemo is a React hook that memoizes the result of a calculation and returns the cached value until one of its dependencies changes. It is mainly used to optimize expensive calculations and prevent unnecessary recomputation during re-renders.

One-Line Interview Answer

useMemo caches a calculated value and recomputes it only when its dependencies change.
