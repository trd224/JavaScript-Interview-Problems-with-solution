149. What is the Difference Between useLayoutEffect and useEffect?

One-Line Definition

Both hooks run after React updates the DOM, but useLayoutEffect runs before the browser paints the screen, while useEffect runs after the browser paints the screen.

The only major difference is WHEN they execute.

Understanding Browser Paint

Paint means: 

Browser draws pixels on screen

At that moment the user can finally see the UI.

DOM Ready
↓
Browser Paint
↓
User Sees UI

useEffect Timing

useEffect(() => {
  console.log('Effect');
}, []);

Execution order:

React Render
↓
DOM Updated
↓
Browser Paint
↓
useEffect Runs

The user sees the page first.

Then useEffect runs.

useEffect never blocks rendering.

useLayoutEffect Timing

useLayoutEffect(() => {
  console.log('Layout Effect');
}, []);

Execution order:

React Render
↓
DOM Updated
↓
useLayoutEffect Runs
↓
Browser Paint

The browser must wait.

useLayoutEffect blocks the browser from painting until it finishes.

Visual Example

Suppose we render: 

<div>Tooltip</div>

Then we calculate its position.

Using useEffect

Render Tooltip
↓
Browser Paint
↓
User Sees Wrong Position
↓
useEffect Runs
↓
Position Fixed

User may see a jump.

Using useLayoutEffect

Render Tooltip
↓
Measure Position
↓
Correct Position Applied
↓
Browser Paint

User only sees the final result.

This is the main reason useLayoutEffect exists.

Real Example: Measuring Element Height

const ref = useRef();

useLayoutEffect(() => {

  const height =
    ref.current.getBoundingClientRect().height;

  console.log(height);

}, []);

We measure before paint.

No visual flicker occurs.

Real Example: API Call

Correct

useEffect(() => {

  fetch('/api/users')
    .then(res => res.json())
    .then(data => console.log(data));

}, []);

Wrong

useLayoutEffect(() => {

  fetch('/api/users');

}, []);

Network requests do not affect layout.

Never block browser painting for API calls.

Most Common useEffect Use Cases

  • API calls.
  • Subscriptions.
  • Timers.
  • Analytics.
  • Logging.
  • WebSocket connections.
  • Event listeners.

Most Common useLayoutEffect Use Cases

  • Measuring element size.
  • Reading getBoundingClientRect().
  • Tooltip positioning.
  • Modal positioning.
  • Scrolling before paint.
  • Animation preparation.
  • Preventing layout flicker.

Performance Difference

Hook Paint Blocking
useEffect ❌ No
useLayoutEffect ✅ Yes

Because useLayoutEffect blocks paint:

  • It is slower.
  • It should be used rarely.
  • Only use it when layout timing matters.
If useEffect works, useEffect is usually the correct choice.

Interview Trick Question

Question: 

Can useLayoutEffect replace useEffect?

Yes

It would unnecessarily block browser painting and hurt performance.

Interview Answer

Both useEffect and useLayoutEffect run after React updates the DOM. The difference is that useLayoutEffect runs before the browser paints the screen and blocks painting until it finishes, while useEffect runs after the browser has painted. useEffect is used for most side effects such as API calls, while useLayoutEffect is used for DOM measurements and preventing visual flicker.

One-Line Interview Answer

useLayoutEffect runs before browser paint, while useEffect runs after browser paint.
