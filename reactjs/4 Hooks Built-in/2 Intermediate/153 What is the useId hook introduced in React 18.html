153. What is the useId Hook Introduced in React 18?

One-Line Definition

useId is a React 18 hook that generates stable, unique IDs for accessibility attributes and works correctly with both Server-Side Rendering (SSR) and Client-Side Rendering (CSR).

Why Was useId Added?

Many HTML elements need matching IDs.

Example

<label for="email">
  Email
</label>

<input id="email" />

The label and input must share the same ID.

In reusable React components we cannot hardcode: 

id="email"

because multiple copies of the component would create duplicate IDs.

HTML IDs must be unique on the page.

The Problem Before React 18

Developers often used: 

const id = Math.random();

or: 

const id = Date.now();

Example

const id = Math.random();

return (
  <>
    <label htmlFor={id}>
      Email
    </label>

    <input id={id} />
  </>
);

This seems fine until SSR is involved.

The SSR Hydration Problem

Server Generates

ID = 12345

Browser Generates

ID = 98765

React compares: 

12345 !== 98765

Result: 

Hydration Warning
Server HTML and client HTML no longer match.

Solution: useId

import { useId } from 'react';

function LoginForm() {

  const id = useId();

  return (
    <>
      <label htmlFor={id}>
        Email
      </label>

      <input id={id} />
    </>
  );

}
React guarantees that the generated ID matches on both the server and client.

Basic Syntax

import { useId } from 'react';

function App() {

  const id = useId();

  return (
    <input id={id} />
  );

}

Most Common Real-World Example

Connecting labels and inputs. 

import { useId } from 'react';

function InputField() {

  const id = useId();

  return (
    <>
      <label htmlFor={id}>
        Username
      </label>

      <input
        id={id}
        type="text"
      />
    </>
  );

}

Now:

Label
↓
htmlFor
↓
ID
↓
Input

Screen readers understand the relationship correctly.

Accessibility Benefits

useId is mainly designed for accessibility.

It helps with:

  • htmlFor
  • aria-labelledby
  • aria-describedby
  • Form controls
  • Screen readers

Generating Multiple Related IDs

One ID can generate several related IDs. 

const id = useId();

const inputId = `${id}-input`;

const errorId = `${id}-error`;

Usage

<input
  id={inputId}
  aria-describedby={errorId}
/>

<p id={errorId}>
  Invalid email address
</p>
This keeps all related IDs unique and connected.

What Does useId Return?

React generates an internal string similar to: 

:r0:
:r1:
:r2:

The exact format may change.

Never depend on the actual value returned by useId. Treat it as an opaque string.

Can useId Be Used As a React Key?

No

{users.map(user => (

  <div key={useId()}>
    {user.name}
  </div>

))}
This is incorrect.

React keys should come from data.

Correct

{users.map(user => (

  <div key={user.id}>
    {user.name}
  </div>

))}
useId is for DOM IDs, not React keys.

Can useId Be Used As a Database ID?

No.

Do not use useId for:

  • User IDs
  • Database IDs
  • Order IDs
  • API IDs
  • Business identifiers

Wrong

const userId = useId();

saveUser({
  id: userId
});
useId is only for UI element identification.

Common Interview Confusion

Question: 

Does useId generate globally unique IDs?

Answer:

No.

It generates IDs that are unique within the React tree and safe for accessibility and hydration.

When Should You Use useId?

  • Labels and inputs.
  • ARIA attributes.
  • Reusable form components.
  • SSR-safe IDs.
  • Accessibility features.

When Should You NOT Use useId?

  • React list keys.
  • Database IDs.
  • User IDs.
  • API identifiers.
  • Business logic identifiers.

Interview Answer

useId is a React 18 hook that generates stable, unique IDs that work correctly with both server-side rendering and client-side hydration. It is primarily used for accessibility features such as linking labels, inputs, and ARIA attributes.

One-Line Interview Answer

useId generates stable, SSR-safe unique IDs for accessibility and form elements.
