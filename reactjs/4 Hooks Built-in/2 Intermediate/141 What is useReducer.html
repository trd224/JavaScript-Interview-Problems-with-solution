141. What is useReducer?

One-Line Definition

useReducer is a React hook used to manage complex state logic by updating state through actions and a reducer function instead of directly using multiple useState hooks.

Simple Meaning

With useState, we directly update state: 

const [count, setCount] = useState(0);

setCount(count + 1);

With useReducer, we do not update state directly.

Instead, we send an action. 

dispatch({
  type: 'INCREMENT'
});

A reducer function decides how state should change.

dispatch(action)
↓
Reducer Function
↓
New State
↓
Component Re-renders

Why Does useReducer Exist?

useState works great for simple state. 

const [count, setCount] = useState(0);

But as state grows, update logic becomes harder to manage.

Example Using Multiple useState

const [username, setUsername] = useState('');
const [password, setPassword] = useState('');
const [loading, setLoading] = useState(false);
const [error, setError] = useState('');
const [isLoggedIn, setIsLoggedIn] = useState(false);

Now state logic is scattered across many setters.

When multiple pieces of state are related, useReducer can make the logic easier to organize.

Syntax

const [state, dispatch] = useReducer(reducer, initialState);

Returns

Value Purpose
state Current state object
dispatch Function used to send actions

The Three Parts of useReducer

Initial State
↓
Reducer Function

1. Initial State

const initialState = {
  count: 0

2. Reducer Function

function reducer(state, action) {

  switch (action.type) {

    case 'INCREMENT':
      return {
        count: state.count + 1
      };

    case 'DECREMENT':
      return {
        count: state.count - 1
      };

    default:
      return state;
  }

}

3. useReducer Hook

const [state, dispatch] = useReducer(reducer,initialState);

Complete Counter Example

import { useReducer } from 'react';

const initialState = {
  count: 0
};

function reducer(state, action) {

  switch (action.type) {

    case 'INCREMENT':
      return {
        count: state.count + 1
      };

    case 'DECREMENT':
      return {
        count: state.count - 1
      };

    default:
      return state;
  }

}

export default function Counter() {

  const [state, dispatch] = useReducer(reducer, initialState);

  return (
    <>
      <h2>Count: {state.count}</h2>

      <button
        onClick={() => dispatch({ type: 'INCREMENT' })}
      >
        Increment
      </button>

      <button
        onClick={() => dispatch({ type: 'DECREMENT' })}
      >
        Decrement
      </button>
    </>
  );
}

How It Works Internally

User clicks Increment: 

dispatch({
  type: 'INCREMENT'
});

Reducer receives: 

state = { count: 0 }

action = {
  type: 'INCREMENT'
}

Reducer returns: 

{
  count: 1
}

Real Login Form Example

This is where useReducer becomes more useful. 

const initialState = {
  username: '',
  password: '',
  loading: false,
  error: '',
  isLoggedIn: false
};

Reducer

function reducer(state, action) {

  switch (action.type) {

    case 'SET_USERNAME':
      return {
        ...state,
        username: action.payload
      };

    case 'SET_PASSWORD':
      return {
        ...state,
        password: action.payload
      };

    case 'LOGIN_START':
      return {
        ...state,
        loading: true
      };

    case 'LOGIN_SUCCESS':
      return {
        ...state,
        loading: false,
        isLoggedIn: true
      };

    case 'LOGIN_ERROR':
      return {
        ...state,
        loading: false,
        error: action.payload
      };

    default:
      return state;
  }

}
Instead of managing many useState hooks, all related state updates are centralized inside one reducer.

How useReducer Compares to useState

useState useReducer
Simple state Complex state
Direct updates Action-based updates
Easy to learn More structured
Few state values Many related state values
Simple forms Complex forms

Relationship with Redux

Redux is heavily inspired by useReducer.

Both use the same idea:

Action
↓
Reducer
↓
New State

The difference:

useReducer Redux
Component-level state Global application state
Built into React External library
No Provider required Store and Provider required

When Should You Use useReducer?

  • When state has multiple related values.
  • When update logic becomes complicated.
  • When many state changes depend on actions.
  • For forms with many fields.
  • For state machines or workflows.

When Should You NOT Use useReducer?

Do not use useReducer for simple state. 

const [isOpen, setIsOpen] = useState(false);

This is simpler than creating a reducer.

If a single useState solves the problem clearly, useState is usually the better choice.

Interview Answer

useReducer is a React hook used for managing complex state logic. It stores state in a reducer function and updates state through dispatched actions, making related state changes easier to organize and maintain.

One-Line Interview Answer

useReducer is a React hook that manages state through actions and a reducer function instead of directly updating state with setters.
