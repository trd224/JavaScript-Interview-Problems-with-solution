useTransition vs useDeferredValue

Main Difference

useTransition delays a state update, while useDeferredValue creates a delayed version of a value.

useTransition

In useTransition, we usually create two states. 

const [input, setInput] = useState('');
const [query, setQuery] = useState('');

Then we manually tell React which update is urgent and which update is non-urgent. 

setInput(value);

startTransition(() => {
  setQuery(value);
});
State Purpose
input Controls the textbox and updates immediately.
query Controls expensive filtering and updates with lower priority.
In useTransition, we manually manage urgent and non-urgent state updates.

useDeferredValue

In useDeferredValue, we usually need only one real state. 

const [input, setInput] = useState('');

Then React creates a delayed version of that value. 

const deferredInput = useDeferredValue(input);
We do not need to create a second state because React automatically gives us a deferred copy of the original value.

Why We Do Not Need Two States in useDeferredValue?

Because useDeferredValue already gives us the second value automatically.

When we write: 

const [input, setInput] = useState('');

const deferredInput = useDeferredValue(input);

React internally treats deferredInput like a delayed version of input.

input
↓
Current Value

deferredInput
↓
Delayed Value

So we already have two values:

  • input = latest value typed by the user.
  • deferredInput = delayed value used for expensive rendering.
That is why we do not need const [query, setQuery] = useState('') when using useDeferredValue.

Compare Side By Side

useTransition useDeferredValue
Delays a state update. Delays a value.
Usually needs two states. Usually needs one state.
We manually use startTransition(). React automatically creates a deferred value.
Provides isPending. Does not provide isPending.
Good when we want to control update priority. Good when we only need a delayed value.
Example: tab switching, heavy state update, route update. Example: search box, large list filtering, autocomplete.

useTransition Example

import { useState, useTransition } from 'react';

function App() {

  const [input, setInput] = useState('');
  const [query, setQuery] = useState('');

  const [isPending, startTransition] = useTransition();

  function handleChange(e) {

    const value = e.target.value;

    setInput(value);

    startTransition(() => {
      setQuery(value);
    });

  }

  return (
    <>
      <input value={input} onChange={handleChange} />

      {isPending && <p>Searching...</p>}

      <ProductList query={query} />
    </>
  );

}
Here, input updates immediately and query updates with lower priority.

useDeferredValue Example

import { useState, useDeferredValue } from 'react';

function App() {

  const [input, setInput] = useState('');

  const deferredInput = useDeferredValue(input);

  return (
    <>
      <input
        value={input}
        onChange={e => setInput(e.target.value)}
      />

      <ProductList query={deferredInput} />
    </>
  );

}
Here, we have only one state. React automatically creates deferredInput as a delayed version of input.

What Happens During Typing?

Suppose user types: 

React

Immediately: 

input = "React"

But for a short time: 

deferredInput = "Reac"

After React finishes the expensive rendering: 

deferredInput = "React"
The textbox stays fast because it uses input, while the heavy list uses deferredInput.

When Should We Use useTransition?

  • When we want to control which update is urgent and which update is non-urgent.
  • When we need isPending.
  • When switching tabs renders heavy UI.
  • When updating a large component tree.
  • When route or page update feels slow.

When Should We Use useDeferredValue?

  • When we already have a value and only need its delayed version.
  • Search input.
  • Large list filtering.
  • Autocomplete.
  • Live search result rendering.

Short Interview Answer

useTransition delays state updates and usually requires two states because we manually separate urgent and non-urgent updates. useDeferredValue creates a delayed version of a value automatically, so we usually need only one state because React provides the deferred copy for us.

One-Line Interview Answer

useTransition delays state updates, while useDeferredValue delays value consumption.
