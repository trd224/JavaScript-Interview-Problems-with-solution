154. What is useTransition?

One-Line Definition

useTransition is a React 18 hook that lets us mark non-urgent state updates as low priority so urgent updates remain responsive.

The Problem Before React 18

Suppose we have a search box. 

function App() {

  const [search, setSearch] = useState('');

  return (
    <input value={search} onChange={e => setSearch(e.target.value)} />
  );

}

Now imagine:

  • 10,000 products
  • Heavy filtering
  • Complex rendering

Every keystroke triggers:

User Types
↓
State Update
↓
Heavy Re-render
↓
UI Becomes Slow
Typing can become laggy because React treats all updates equally.

What React 18 Introduced

React now supports update priorities.

Urgent Updates
↓
High Priority

Non-Urgent Updates
↓
Low Priority

useTransition allows us to tell React: 

This update is not urgent.

Basic Syntax

const [isPending, startTransition] = useTransition();
Value Purpose
isPending Shows whether transition is running
startTransition Marks updates as low priority

Basic Example

import { useTransition } from 'react';

function App() {

  const [isPending, startTransition] = useTransition();

  const [text, setText] = useState('');

  function handleChange(e) {

    startTransition(() => {

      setText(e.target.value);

    });

  }

}
Updates inside startTransition become non-urgent.

Real Search Example

Without useTransition: 

import { useState } from "react";

const products = [
    { id: 1, name: "Laptop" },
    { id: 2, name: "Mobile" },
    { id: 3, name: "Keyboard" },
    { id: 4, name: "Mouse" }
];

function Search() {

    const [query, setQuery] = useState("");

    const filteredProducts = products.filter(product =>
        product.name.toLowerCase().includes(query.toLowerCase())
    );

    //Here, filteredProducts is derived from query and products, so 
    //there is no need to add derived state to React state.

    return (

        <div>

            <input
                type="text"
                placeholder="Search product..."
                value={query}
                onChange={(e) => setQuery(e.target.value)}
            />

            <ul>
                {filteredProducts.map(product => (
                    <li key={product.id}>
                        {product.name}
                    </li>
                ))}
            </ul>

        </div>

    );

}

export default Search;

What Happens?

User Types
↓
Update Query
↓
Filter 10,000 Products
↓
Render Results
↓
Typing Feels Slow

Using useTransition

import { useState, useTransition } from "react";

const products = [
    { id: 1, name: "Laptop" },
    { id: 2, name: "Mobile" },
    { id: 3, name: "Keyboard" },
    { id: 4, name: "Mouse" },
    { id: 5, name: "Monitor" },
    { id: 6, name: "Printer" }
];

export default function Search() {

    const [query, setQuery] = useState('');
    const [results, setResults] = useState([]);

    const [isPending, startTransition] = useTransition();

    function handleChange(e) {

        const value = e.target.value;

        setQuery(value);

        startTransition(() => {

            setResults(
                products.filter(product =>
                    product.name.toLowerCase().includes(value.toLowerCase())
                )
            );

        });

    }

    return (

        <div>

            <input
                type="text"
                value={query}
                onChange={handleChange}
                placeholder="Search product..."
            />

            {isPending && <p>Searching...</p>}

            <ul>
                {results.map(product => <li key={product.id}>{product.name}</li>)}
            </ul>

        </div>

    );

}

Now What Happens?

User Types
↓
Update Input Immediately
↓
Render Input
↓
Process Heavy Filter Later
The input remains responsive while React works on the expensive update.

Understanding Urgent vs Non-Urgent Updates

Urgent Update

setQuery(value);

User expects this instantly.

Non-Urgent Update

startTransition(() => {

  setResults(filteredData);

});

The results can appear slightly later.

useTransition does not make updates faster.

It makes important updates happen first.

Showing Loading State

isPending becomes true while React is processing the transition. 

const [isPending, startTransition] = useTransition();

Example

{isPending && (
  <p>Loading...</p>
)}

React automatically updates: 

false
↓
true
↓
false

Another Real Example

Switching large tabs. 

startTransition(() => {

  setActiveTab('reports');

});

The tab switch becomes low priority.

Urgent interactions remain responsive.

What useTransition Does NOT Do

Many developers think: 

useTransition
↓
Makes Code Faster

Wrong.

useTransition does not reduce computation.

It only changes update priority.

Comparison With Normal State Update

Normal

setResults(filteredProducts);

Priority: 

High

Transition

startTransition(() => {

  setResults(filteredProducts);

});

Priority: 

Low

When Should You Use useTransition?

  • Large search results.
  • Heavy filtering.
  • Heavy sorting.
  • Large table rendering.
  • Complex dashboard updates.
  • Tab switching with lots of content.

When Should You NOT Use useTransition?

  • Simple forms.
  • Small components.
  • Normal state updates.
  • Updates that must happen immediately.
Most state updates do not need useTransition.

How It Compares to useDeferredValue

Hook Purpose
useTransition Delay state updates
useDeferredValue Delay value updates

Both help keep the UI responsive.

Easy Way To Remember

Urgent Update?
↓
Normal setState()

Can Wait?
↓
startTransition()

Interview Answer

useTransition is a React 18 hook that allows us to mark state updates as non-urgent. React gives priority to urgent updates such as typing and user interactions, while transition updates run with lower priority, keeping the UI responsive during expensive renders.

One-Line Interview Answer

useTransition lets React prioritize urgent updates while delaying non-urgent state updates to keep the UI responsive.
