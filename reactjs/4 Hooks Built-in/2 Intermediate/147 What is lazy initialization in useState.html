147. What is Lazy Initialization in useState?

One-Line Definition

Lazy initialization is a useState technique where we pass a function instead of a value, allowing React to calculate the initial state only once during the first render.

Why Does Lazy Initialization Exist?

Normally we initialize state like this: 

const [count, setCount] = useState(0);

For simple values this is perfectly fine.

But sometimes calculating the initial value is expensive.

Example

const [users, setUsers] = useState(
  generateLargeUserList()
);

The function: 

generateLargeUserList()

runs every time the component renders.

Even though React only uses the initial value once, JavaScript still executes the function on every render.

The Problem

function App() {

  const [count, setCount] = useState(0);

  const [users, setUsers] = useState(
    generateLargeUserList()
  );

  return (
    <button onClick={() => setCount(count + 1)}>
      Increment
    </button>
  );

}

First Render

generateLargeUserList()
↓
Runs

Second Render

generateLargeUserList()
↓
Runs Again

Third Render

generateLargeUserList()
↓
Runs Again

React ignores the returned value after the first render, but the function still executes.

This can waste CPU time if the calculation is expensive.

Solution: Lazy Initialization

Instead of passing the result: 

useState(generateLargeUserList())

Pass a function: 

useState(() => generateLargeUserList())

Example

const [users, setUsers] = useState(() => {
  return generateLargeUserList();
});
React calls this function only during the first render.

Simple Example

Without Lazy Initialization

function App() {

  console.log('Component Rendered');

  const [value] = useState(Math.random());

  return <h2>{value}</h2>;

}

Every render: 

Math.random()
↓
Runs Again

With Lazy Initialization

function App() {

  console.log('Component Rendered');

  const [value] = useState(() => {
    return Math.random();
  });

  return <h2>{value}</h2>;

}

Now: 

Math.random()
↓
Runs Once

Most Common Real-World Example

Reading from localStorage.

Bad

const [theme, setTheme] = useState(
  localStorage.getItem('theme')
);

Every render: 

localStorage.getItem()
↓
Runs Again

Good

const [theme, setTheme] = useState(() => {
  return localStorage.getItem('theme');
});

Now: 

localStorage.getItem()
↓
Runs Once
This is the most common interview example of lazy initialization.

How React Handles It Internally

When React sees: 

useState(100)

it stores: 

100

When React sees: 

useState(() => 100)

React executes the function once: 

() => 100
↓
100

and stores the result.

Future renders reuse the stored state.

Important Interview Confusion

Many developers think: 

useState(() => value)
↓
Runs every render

Wrong.

The function passed to useState for lazy initialization runs only during the first render.

When Should You Use Lazy Initialization?

  • Reading localStorage.
  • Reading sessionStorage.
  • Large array generation.
  • Expensive calculations.
  • Complex initial state creation.

When Should You NOT Use It?

Simple values do not need it. 

const [count, setCount] = useState(0);
const [name, setName] = useState('Rizwan');

These are already fast.

Lazy initialization should be used only when calculating the initial state is expensive.

Normal vs Lazy Initialization

Normal Initialization Lazy Initialization
useState(value) useState(() => value)
Expression runs every render Function runs once
Good for simple values Good for expensive values

Interview Answer

Lazy initialization in useState means passing a function instead of a value. React executes that function only during the first render and uses its return value as the initial state. It is commonly used for expensive calculations and reading values from localStorage.

One-Line Interview Answer

Lazy initialization allows useState to compute its initial value only once by passing a function instead of a direct value.
