144. What is the Difference Between useMemo and useCallback?

One-Line Definition

useMemo memoizes a value, while useCallback memoizes a function.

The Biggest Difference

Hook What It Caches
useMemo Value
useCallback Function
useMemo
↓
Caches Value

useCallback
↓
Caches Function

Why Do We Need Them?

React re-runs the entire component whenever state changes.

That means:

  • Variables are recalculated.
  • Functions are recreated.
  • Arrays and objects are recreated.
Sometimes these recalculations are expensive or cause unnecessary child re-renders.

That's why React provides:

  • useMemo → Cache values.
  • useCallback → Cache functions.

useMemo Example

Suppose filtering products is expensive. 

const expensiveProducts = useMemo(() => {

  return products.filter(
    product => product.price > 1000
  );

}, [products]);

Result: 

products unchanged
        ↓
cached value returned


products changed
        ↓
recalculate filter
useMemo stores the filtered array result.

useCallback Example

Suppose a function is passed to a React.memo component. 

const handleAddToCart = useCallback(() => {

  console.log('Added');

}, []);

Result: 

dependencies unchanged
        ↓
same function returned


dependencies changed
        ↓
new function created
useCallback stores the function reference.

Side-by-Side Comparison

useMemo

const total = useMemo(() => {

  return price * quantity;

}, [price, quantity]);

Returns: 

500

A value is returned.

useCallback

const handleClick = useCallback(() => {

  console.log('Clicked');

}, []);

Returns: 

function handleClick() { ... }

A function is returned.

How useCallback Is Actually Implemented

Many developers don't know this: 

useCallback(fn, deps)

is roughly equivalent to: 

useMemo(() => fn, deps)
useCallback is essentially a specialized version of useMemo designed specifically for functions.

Real Project Example

Problem

function App() {

  const [count, setCount] = useState(0);

  const users = [
    'Rizwan',
    'Ali',
    'John'
  ];

  const handleClick = () => {
    console.log('Clicked');
  };

  return (
    
  );

}

Every render creates:

  • New users array
  • New handleClick function

Optimized Version

const users = useMemo(() => {

  return [
    'Rizwan',
    'Ali',
    'John'
  ];

}, []);

const handleClick = useCallback(() => {

  console.log('Clicked');

}, []);
useMemo stabilizes the array.

useCallback stabilizes the function.

When Should You Use useMemo?

  • Expensive calculations.
  • Filtering large arrays.
  • Sorting large arrays.
  • Creating stable object references.
  • Creating stable array references.

When Should You Use useCallback?

  • Passing functions to React.memo components.
  • Functions used inside dependency arrays.
  • Preventing unnecessary child re-renders.
  • Creating stable function references.

Common Interview Trick Question

Question: 

Can useCallback replace useMemo?

Answer:

No.

useCallback caches functions.
useMemo caches values.

Question: 

Can useMemo replace useCallback?

Technically: 

const handleClick = useMemo(() => {

  return () => {
    console.log('Clicked');
  };

}, []);

works.

But:

useCallback is clearer and should be used for functions.

Most Common Beginner Mistake

Many developers think: 

useMemo
↓
Prevents Re-renders


useCallback
↓
Prevents Re-renders

This is incorrect.

Neither hook prevents component re-renders.

They only preserve references between renders.

Interview Answer

useMemo memoizes a calculated value and recomputes it only when dependencies change, whereas useCallback memoizes a function and returns the same function reference until dependencies change. useMemo is used for expensive calculations, while useCallback is used for stable function references.

One-Line Interview Answer

useMemo caches values, while useCallback caches functions.
