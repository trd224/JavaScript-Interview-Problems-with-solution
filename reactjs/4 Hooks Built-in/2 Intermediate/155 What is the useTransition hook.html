155. What is the useTransition Hook?

One-Line Definition

useTransition is a React 18 hook that allows us to mark state updates as non-urgent so React can keep important user interactions responsive.

Why Was useTransition Introduced?

Before React 18, all state updates had the same priority. 

setSearch(value);
setResults(filteredData);

React processed both updates immediately.

If filtering or rendering was expensive, the UI could freeze.

Large updates could make typing, clicking, and scrolling feel slow.

The Main Idea

React now supports update priorities.

Urgent Updates
↓
High Priority
↓
Run Immediately

Non-Urgent Updates
↓
Low Priority
↓
Can Wait

useTransition lets us tell React: 

This update is important.
Run it now.

This update is not important.
Run it later.

Syntax

const [isPending, startTransition] = useTransition();
Value Purpose
isPending Indicates whether a transition is running
startTransition Marks updates as low priority

Basic Example

import { useState, useTransition } from 'react';

function App() {

  const [text, setText] = useState('');

  const [isPending, startTransition] = useTransition();

  function handleChange(e) {

    startTransition(() => {

      setText(e.target.value);

    });

  }

}

Everything inside: 

startTransition(() => {

  ...

});

becomes a low-priority update.

Real Search Example

Imagine:

  • 10,000 products
  • Heavy filtering
  • Large UI updates

Without useTransition

function Search() {

  const [query, setQuery] = useState('');

  const filteredProducts = products.filter(product =>
      product.name.includes(query)
    );

}

What Happens?

User Types
↓
Update Query
↓
Filter 10,000 Products
↓
Render Results
↓
Input Feels Slow
Every keystroke waits for the expensive update.

Using useTransition

function Search() {

  const [query, setQuery] = useState('');

  const [results, setResults] = useState([]);

  const [isPending, startTransition] = useTransition();

  function handleChange(e) {

    const value = e.target.value;

    setQuery(value);

    startTransition(() => {

      setResults(
        products.filter(product =>
          product.name.includes(value)
        )
      );

    });

  }

}

What Happens Now?

User Types
↓
Update Input Immediately
↓
Show Typed Character
↓
Run Heavy Filter Later
↓
Display Results
Typing stays smooth because React prioritizes the input update.

Understanding Urgent vs Non-Urgent Updates

Urgent Update

setQuery(value);

The user expects this immediately.

Non-Urgent Update

startTransition(() => {

  setResults(filteredData);

});

The results can appear a little later.

useTransition does not make updates faster.

It makes important updates happen first.

Showing Loading State

isPending becomes true while React is processing the transition. 

const [isPending, startTransition] =
  useTransition();

Example

{isPending && (
  <p>Loading Results...</p>
)}

Timeline: 

false
↓
true
↓
false

Another Real-World Example

Large Dashboard Filter

startTransition(() => {

  setFilteredReports(data);

});

Users can still interact with the UI while React processes the expensive update.

What useTransition Does NOT Do

Many developers think: 

useTransition
↓
Makes Code Faster

Wrong.

useTransition does not reduce computation.

It only changes the priority of updates.

Comparison With Normal State Update

Normal Update

setResults(filteredProducts);

Priority: 

High Priority

Transition Update

startTransition(() => {

  setResults(filteredProducts);

});

Priority: 

Low Priority

Most Common Interview Confusion

Question: 

Should we wrap every setState in startTransition?

Answer: 

No
Most state updates should remain urgent.

Examples:

  • Typing in inputs.
  • Checkbox changes.
  • Button clicks.
  • Form values.

These should update immediately.

When Should You Use useTransition?

  • Large search results.
  • Heavy filtering.
  • Heavy sorting.
  • Complex dashboards.
  • Large tables.
  • Tab switches with expensive rendering.

When Should You NOT Use useTransition?

  • Simple forms.
  • Small state updates.
  • Normal UI interactions.
  • Updates that must happen instantly.

useTransition vs useDeferredValue

Hook Purpose
useTransition Delay state updates
useDeferredValue Delay a value

Both are React 18 performance hooks.

Easy Way To Remember

Must Happen Immediately?
↓
Normal setState()

Can Wait A Little?
↓
startTransition()

Interview Answer

useTransition is a React 18 hook that allows us to mark state updates as non-urgent. React prioritizes urgent updates such as typing and clicking while processing transition updates in the background, helping keep the UI responsive during expensive renders.

One-Line Interview Answer

useTransition lets React prioritize urgent updates while delaying non-urgent updates to improve UI responsiveness.
