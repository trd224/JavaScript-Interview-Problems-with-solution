161. What is the use() Hook in React 19 and How Does It Replace useContext and Data Unwrapping?

One-Line Answer

React 19's use() hook can read Context and Promise values during rendering. For Context, it can replace useContext(). For Promise data, it unwraps the resolved value and works with Suspense.

Simple Meaning

use(Context)
↓ Read Context Value

use(Promise)
↓ Read Async Data
↓ If Pending, Suspend

1. Replacing useContext()

Before - useContext
import { useContext } from 'react';

function Child() {
  const theme = useContext(ThemeContext);

  return <h2>{theme}</h2>;
}
React 19 - use()
import { use } from 'react';

function Child() {
  const theme = use(ThemeContext);

  return <h2>{theme}</h2>;
}
For Context, use(ThemeContext) works like useContext(ThemeContext).

2. Replacing Data Unwrapping

Data unwrapping means getting the final value from a Promise.

Promise Created Outside Component
const userPromise = fetch('/api/user').then(res => res.json());
React 19 use()
import { use } from 'react';

function User({ userPromise }) {
  const user = use(userPromise);

  return <h2>{user.name}</h2>;
}
use(userPromise)
↓ Promise Pending?
↓ Suspend Component
↓ Suspense Shows Loading
↓ Promise Resolves
↓ user Contains Data

Important: Fetch Without useEffect

Yes, with React 19 use(), data can be fetched without useEffect in Suspense-based data loading.
Complete Example
import { Suspense, use } from 'react';

const userPromise = fetch('/api/user').then(res => res.json());

function User({ userPromise }) {
  const user = use(userPromise);

  return <h2>{user.name}</h2>;
}

export default function App() {
  return (
    <Suspense fallback={<h2>Loading...</h2>}>
      <User userPromise={userPromise} />
    </Suspense>
  );
}

How This Works?

JS File Loads
↓ fetch('/api/user') Starts
↓ Promise Created
↓ App Renders
↓ User Calls use(userPromise)
↓ Promise Pending
↓ React Suspends
↓ Suspense Shows Loading...
↓ Promise Resolves
↓ User Renders Data

Old React Way

Before React 19
function User() {
  const [user, setUser] = useState(null);

  useEffect(() => {
    fetch('/api/user')
      .then(res => res.json())
      .then(data => setUser(data));
  }, []);

  if (!user) return <h2>Loading...</h2>;

  return <h2>{user.name}</h2>;
}
Render
↓ useEffect Runs
↓ Fetch Starts
↓ setUser()
↓ Re-render

React 19 use() Way

Fetch Starts
↓ Component Reads Promise
↓ use(promise)
↓ React Waits With Suspense
↓ Render Continues With Data

Very Important Warning

Do not create a new Promise directly inside the component on every render.
Bad Pattern
function User() {
  const user = use(fetch('/api/user').then(res => res.json()));

  return <h2>{user.name}</h2>;
}
Render
↓ Creates New Promise
↓ Render Again
↓ Creates New Promise Again
↓ Possible Re-fetch Problem
Better: create the Promise outside the component or pass it from a framework/server layer.

use() Does Not Fully Replace useEffect

use() replaces many data-fetching useEffect patterns, but not all useEffect usage.
Still Need useEffect For Example
Subscriptions socket.connect()
Timers setInterval()
Event Listeners window.addEventListener()
Manual DOM side effects document.title

How React Handles use()

Input React Behavior
Context Returns current context value
Resolved Promise Returns resolved data
Pending Promise Suspends component
Rejected Promise Throws error to Error Boundary

Interview Answer

React 19's use() hook lets components read Context values and Promises during rendering. For Context, use(Context) can replace useContext(Context). For Promises, use(promise) unwraps the resolved value. If the Promise is pending, React suspends the component and shows the nearest Suspense fallback. This removes much of the boilerplate previously handled with useEffect, useState, and manual loading state for Suspense-based data loading, but useEffect is still needed for subscriptions, timers, event listeners, and other side effects.
