162. What is useOptimistic in React 19?

One-Line Answer

useOptimistic() lets React update the UI immediately before the server request finishes. If the server succeeds, the UI stays the same. If the server fails, React can roll back to the previous state.

Why Was It Introduced?

Normally:

User Clicks Like
↓ Send API Request
↓ Wait
↓ Server Response
↓ Update UI

This feels slow.

User may wait 1–2 seconds before seeing any change.

Optimistic UI Idea

User Clicks Like
↓ Update UI Immediately
↓ Send API Request
↓ Server Success
↓ Keep UI
UI feels instant.

Real World Example

Instagram Like Button:

Tap ❤️
↓ Heart Turns Red Immediately
↓ Request Sent
↓ Server Updates Database

Instagram doesn't wait for the server before updating the heart.

Before React 19

Manual Optimistic Update
const [likes,setLikes] = useState(10);

async function handleLike() {

  setLikes(prev => prev + 1);

  try {

    await api.likePost();

  } catch {

    setLikes(prev => prev - 1);

  }

}
You manually handle rollback logic.

React 19 useOptimistic()

Basic Syntax
const [optimisticState, addOptimistic] = useOptimistic(state, updateFn);
Value Meaning
state Real state
optimisticState Temporary UI state
addOptimistic() Apply optimistic update
updateFn How optimistic value is calculated

Like Button Example

LikeButton.jsx
import { useState,useOptimistic } from 'react';

export default function LikeButton() {

  const [likes,setLikes] = useState(10);

  const [optimisticLikes,addLike] = useOptimistic(likes,(current,value) => current + value);

  async function handleLike() {

    addLike(1);

    try {

      const updatedLikes = await fakeLikeApi();

      setLikes(updatedLikes);

    } catch(error) {

      console.log(error);

    }

  }

  return (
    <button onClick={handleLike}>
      ❤️ {optimisticLikes}
    </button>
  );

}

function fakeLikeApi() {

  return new Promise(resolve => {
    setTimeout(() => resolve(11),2000);
  });

}

What Happens Internally?

Likes = 10
↓ User Clicks Like
↓ addLike(1)
↓ UI Shows 11
↓ Request Sent
↓ Server Success
↓ UI Stays 11

If Server Fails

Likes = 10
↓ User Clicks Like
↓ UI Shows 11
↓ Request Sent
↓ Server Error
↓ Rollback
↓ UI Shows 10
React automatically drops the optimistic state when the real state doesn't update.

Works Great With Server Actions

Server Action Example
const [optimisticTodos,addTodo] =
  useOptimistic(
    todos,
    (current,text) => [
      ...current,
      { id: Date.now(), text }
    ]
  );

async function formAction(formData) {

  const text =
    formData.get('todo');

Interview Answer

useOptimistic is a React 19 hook that allows components to show optimistic UI updates before a server request completes. It immediately updates the UI with an expected result, improving responsiveness. If the server request succeeds, the UI remains unchanged. If it fails, React can roll back to the original state. It is commonly used for likes, comments, chat messages, todos, and works especially well with React 19 Server Actions.
