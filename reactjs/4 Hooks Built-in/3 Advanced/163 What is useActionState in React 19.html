163. What is useActionState in React 19?

One-Line Answer

useActionState is a React 19 hook that connects an action with state, so after a form is submitted, React can automatically update the result state and also give you a pending/loading state.

Simple Meaning

Normally, when we submit a form, we manually manage many things:

  • Form submit function
  • Loading state
  • Error message
  • Success message
  • Returned response
useActionState helps manage these form action results in one clean place.

Basic Syntax

Syntax
const [state, formAction, isPending] = useActionState(actionFunction, initialState);
Value Meaning
state Current result returned by the action
formAction Function passed to the form action attribute
isPending true while the action is running

Without useActionState

Old Manual Way
import { useState } from 'react';

function SignupForm() {
  const [message, setMessage] = useState('');
  const [isLoading, setIsLoading] = useState(false);

  async function handleSubmit(e) {
    e.preventDefault();
    setIsLoading(true);
    const formData = new FormData(e.target);
    const name = formData.get('name');
    await new Promise(resolve => setTimeout(resolve, 1000));
    setMessage(`User ${name} created successfully`);
    setIsLoading(false);
  }

  return (
    <form onSubmit={handleSubmit}>
      <input name="name" placeholder="Enter name" />
      <button disabled={isLoading}>{isLoading ? 'Creating...' : 'Create User'}</button>
      <p>{message}</p>
    </form>
  );
}
This works, but we manually handled submit, loading, FormData, and message state.

With useActionState

SignupForm.jsx
import { useActionState } from 'react';

async function createUser(prevState, formData) {
  const name = formData.get('name');
  await new Promise(resolve => setTimeout(resolve, 1000));

  if (!name) {
    return { message: 'Name is required', success: false };
  }

  return { message: `User ${name} created successfully`, success: true };
}

function SignupForm() {
  const [state, formAction, isPending] = useActionState(createUser, { message: '', success: false });

  return (
    <form action={formAction}>
      <input name="name" placeholder="Enter name" />
      <button disabled={isPending}>{isPending ? 'Creating...' : 'Create User'}</button>
      <p>{state.message}</p>
    </form>
  );
}

export default SignupForm;

What Happens Step By Step?

User Enters Name
↓ Clicks Submit
↓ formAction Runs
↓ React Sends formData To createUser
↓ isPending Becomes true
↓ createUser Returns New State
↓ state.message Updates
↓ isPending Becomes false

Important Point: Why prevState?

async function createUser(prevState, formData) {
  console.log(prevState);
}

The first parameter is the previous result of the action. This is useful when the new result depends on the old result.

Think of prevState like: “What was the last result before this submit?”

Example With Error Message

LoginForm.jsx
import { useActionState } from 'react';

async function login(prevState, formData) {
  const email = formData.get('email');
  const password = formData.get('password');

  if (!email) {
    return { error: 'Email is required' };
  }

  if (!password) {
    return { error: 'Password is required' };
  }

  return { error: '', message: 'Login successful' };
}

function LoginForm() {
  const [state, formAction, isPending] = useActionState(login, { error: '', message: '' });

  return (
    <form action={formAction}>
      <input name="email" placeholder="Email" />
      <input name="password" type="password" placeholder="Password" />
      <button disabled={isPending}>{isPending ? 'Logging in...' : 'Login'}</button>
      {state.error && <p>{state.error}</p>}
      {state.message && <p>{state.message}</p>}
    </form>
  );
}

export default LoginForm;

Important Point: What is prevState?

prevState is the previous state returned by the last execution of the action function. React automatically passes it as the first argument whenever the action runs again.

Example

LoginForm.jsx
const [state, formAction, isPending] =
  useActionState(login, { error: '', message: '' });

async function login(prevState, formData) {
  console.log(prevState);
}

The initial state passed to useActionState is: 

{
  error: '',
  message: ''
}

Therefore, during the first form submission, prevState will be: 

{
  error: '',
  message: ''
}

First Submission

Suppose the user clicks Login without entering an email. 

async function login(prevState, formData) {
  const email = formData.get('email');

  if (!email) {
    return {
      error: 'Email is required'
    };
  }
}

The action returns: 

{
  error: 'Email is required'
}

React automatically stores this returned object as the new state.

Second Submission

When the form is submitted again, React passes the last returned state as prevState. 

prevState

{
  error: 'Email is required'
}

Suppose this time both fields are filled correctly. 

return {
  error: '',
  message: 'Login successful'
};

Now React updates the state again.

Third Submission

On the next submit, prevState becomes: 

{
  error: '',
  message: 'Login successful'
}

Execution Flow

Initial State
↓ { error: '', message: '' }

First Submit
↓ prevState = { error: '', message: '' }
↓ Returns
↓ { error: 'Email is required' }

Second Submit
↓ prevState = { error: 'Email is required' }
↓ Returns
↓ { error: '', message: 'Login successful' }

Third Submit
↓ prevState = { error: '', message: 'Login successful' }

When is prevState Useful?

prevState is useful whenever the next state depends on the previous state.

Counter Example
import { useActionState } from 'react';

async function increment(prevState) {
  return {
    count: prevState.count + 1
  };
}

function Counter() {
  const [state, formAction] = useActionState(increment, { count: 0 });

  return (
    <form action={formAction}>
      <button>Increment</button>
      <p>Count: {state.count}</p>
    </form>
  );
}
Initial State
↓ { count: 0 }

First Click
↓ prevState = { count: 0 }
↓ Returns { count: 1 }

Second Click
↓ prevState = { count: 1 }
↓ Returns { count: 2 }

Third Click
↓ prevState = { count: 2 }
↓ Returns { count: 3 }
Remember: prevState always contains the state returned by the previous execution of the action function. This allows the next action to calculate a new state based on the previous one.

Why It Is Useful In React 19?

Problem useActionState Help
Need loading state Gives isPending automatically
Need success/error message Action return value becomes state
Need FormData Form action receives formData directly
Too much form boilerplate Keeps form handling cleaner

useActionState vs useState

useState useActionState
General purpose state Mainly useful for action/form result state
You update manually Action return value updates state
Loading handled manually isPending is built-in
Uses onSubmit often Works cleanly with form action

Easy Real-Life Thinking

Form Submit
↓ Action Function Runs
↓ Action Returns Result
↓ React Stores Result In state
↓ UI Shows Message

Interview Answer

useActionState is a React 19 hook used to manage state that changes after an action runs, especially form actions. It accepts an action function and an initial state, then returns the current state, a form action function, and an isPending boolean. When the form is submitted, React calls the action function with the previous state and formData. Whatever the action returns becomes the new state. It reduces boilerplate because we do not need separate useState hooks for loading, success, and error handling.

Quick Summary

Use useActionState when a form submission should return a result like success message, validation error, or server response, and you also want loading state without writing extra useState code.
