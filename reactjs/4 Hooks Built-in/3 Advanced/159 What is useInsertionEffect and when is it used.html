159. What is useInsertionEffect and When is it Used?

One-Line Answer

useInsertionEffect is a React 18 hook designed primarily for CSS-in-JS libraries. It runs after React updates the DOM but before layout effects and before the browser paints the screen, allowing styles to be inserted at exactly the right time.

Why Was It Introduced?

React 18 introduced Concurrent Rendering.

With concurrent rendering, React can:

Start Rendering
↓ Pause Rendering
↓ Resume Rendering
↓ Commit DOM

CSS-in-JS libraries such as:

  • Emotion
  • Styled Components
  • Stitches

need to inject CSS into the page.

If CSS is inserted too late, users may briefly see unstyled content.

The Problem Before React 18

Many CSS-in-JS libraries used: 

useLayoutEffect(() => {
  insertStyles();
}, []);

But React 18 concurrent rendering made timing more sensitive.

DOM Updated
↓ Browser Preparing Layout
↓ Styles Not Inserted Yet
↓ Flash Of Incorrect Styles

React Added useInsertionEffect

useInsertionEffect gives style libraries a special place in React's lifecycle where they can inject styles before layout calculations happen.

Execution Order

Render
↓ DOM Updated
↓ useInsertionEffect
↓ useLayoutEffect
↓ Browser Paint
↓ useEffect

Most Important Interview Point

Hook When It Runs
useInsertionEffect Before layout effects
useLayoutEffect Before paint
useEffect After paint

Visual Timeline

React Render
↓ DOM Commit
↓ useInsertionEffect
↓ Insert CSS
↓ useLayoutEffect
↓ Measure Layout
↓ Paint Screen
↓ useEffect

Basic Example

Example.jsx
import { useInsertionEffect } from 'react';

function App() {

  useInsertionEffect(() => {

    console.log('Insert Styles');

  }, []);

  return <h2>Hello</h2>;

}

Output order:

DOM Updated
↓ Insert Styles
↓ Layout Effects
↓ Paint

Real CSS-In-JS Example

Style Injection
useInsertionEffect(() => {

  const style =
    document.createElement('style');

  style.textContent =
    '.title{color:red;}';

  document.head.appendChild(style);

  return () => {
    style.remove();
  };

}, []);
Styles are available before layout calculations happen.

Why Not useEffect?

useEffect(() => {
  insertStyles();
}, []);

Timeline:

DOM Updated
↓ Paint Screen
↓ User Sees Page
↓ Styles Inserted
User may see a flash of unstyled content.

Why Not useLayoutEffect?

useLayoutEffect(() => {
  insertStyles();
}, []);

Better than useEffect.

But:

DOM Updated
↓ Layout Effects Start
↓ Insert Styles
↓ Layout Recalculated

React wanted style libraries to inject styles even earlier.

Relationship With useLayoutEffect

You already learned:

useLayoutEffect
↓ Read DOM
↓ Measure Width
↓ Measure Height
↓ Position Tooltip

React expects layout effects to measure the DOM.

For accurate measurements:

Styles Must Already Exist
↓ Then Measure Layout
That is exactly why useInsertionEffect runs first.

Real World Example

Imagine Emotion library.

Render Button
↓ Generate CSS
↓ Insert CSS
↓ Measure Layout
↓ Paint

Without useInsertionEffect:

Render Button
↓ Measure Layout
↓ Paint
↓ Insert CSS
Possible flicker or incorrect layout measurements.

Can Application Developers Use It?

Almost never.

Most React developers should use:

  • useEffect
  • useLayoutEffect

Only library authors usually need:

  • useInsertionEffect

What React Documentation Says

useInsertionEffect is primarily intended for CSS-in-JS library authors and is not commonly needed in application code.

Comparison Table

Hook Main Purpose Runs
useInsertionEffect Insert CSS Before layout effects
useLayoutEffect Read/measure DOM Before paint
useEffect API calls, subscriptions After paint

Easy Way To Remember

Need CSS?
↓ useInsertionEffect

Need DOM Measurement?
↓ useLayoutEffect

Need Side Effects?
↓ useEffect

Interview Answer

useInsertionEffect is a React 18 hook primarily designed for CSS-in-JS libraries. It runs after React commits DOM updates but before layout effects and before the browser paints. This allows style libraries to inject CSS before layout calculations occur, preventing flashes of unstyled content and ensuring accurate measurements. Most application developers rarely need it; it is mainly intended for library authors. The execution order is useInsertionEffect → useLayoutEffect → paint → useEffect.
