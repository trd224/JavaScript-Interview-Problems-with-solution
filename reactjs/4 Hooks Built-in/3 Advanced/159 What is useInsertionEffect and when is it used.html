159. What is useInsertionEffect and When is it Used?

One-Line Answer

useInsertionEffect is a React 18 hook designed primarily for CSS-in-JS libraries. It runs immediately after React commits DOM updates but before the browser calculates styles and layout, allowing styles to be inserted at exactly the right time.

Why Was It Introduced?

React 18 introduced Concurrent Rendering.

With concurrent rendering, React can:

Start Rendering
↓ Pause Rendering
↓ Resume Rendering
↓ Commit DOM

CSS-in-JS libraries such as:

  • Emotion
  • Styled Components
  • Stitches

generate CSS dynamically while components render.

If CSS is inserted too late, the browser may calculate the layout without those styles, causing incorrect layout calculations or a flash of unstyled content.

The Problem Before React 18

Many CSS-in-JS libraries used: 

useLayoutEffect(() => {
  insertStyles();
}, []);

This worked, but it wasn't the ideal place for inserting styles.

DOM Commit
↓ Browser Begins Preparing Layout
↓ useLayoutEffect Inserts CSS
↓ Browser May Need Another Layout Calculation
React wanted CSS libraries to inject styles even earlier so the browser could calculate the layout correctly the first time.

React Added useInsertionEffect

useInsertionEffect provides a dedicated lifecycle phase for CSS-in-JS libraries to insert styles before the browser performs its layout calculation.

Execution Order

React Render
↓ DOM Commit
↓ useInsertionEffect
↓ Insert CSS (if needed)
↓ Browser Calculates Styles & Layout
↓ useLayoutEffect
↓ Read / Measure Computed Layout
↓ Make Synchronous DOM Updates (if needed)
↓ Paint Screen
↓ useEffect

Important Difference

Browser Calculates Styles & Layout
  • The browser computes the final width, height, position, margins, padding and other layout information for every element.
  • This work is done automatically by the browser.
Read / Measure Computed Layout
  • Our JavaScript reads the values that the browser has already calculated.
  • Common APIs include:
    • offsetWidth
    • offsetHeight
    • clientWidth
    • clientHeight
    • scrollHeight
    • getBoundingClientRect()

The browser performs the calculations. Our code simply reads the computed values.

Basic Example

Example.jsx
import { useInsertionEffect } from "react";

function App() {

  useInsertionEffect(() => {

    console.log("Insert Styles");

  }, []);

  return <h2>Hello</h2>;

}

export default App;

Execution order:

React Render
↓ DOM Commit
↓ useInsertionEffect
↓ Insert Styles
↓ Browser Calculates Styles & Layout
↓ Paint
This example simply demonstrates when useInsertionEffect runs. Normally, only CSS-in-JS libraries use this hook.

Real CSS-in-JS Example

Style Injection
import { useInsertionEffect } from "react";

function App() {

  useInsertionEffect(() => {

    const style =
      document.createElement("style");

    style.textContent =
      ".title{color:red;font-size:32px;}";

    document.head.appendChild(style);

    return () => {

      style.remove();

    };

  }, []);

  return <h1 className="title">Hello React</h1>;

}
The CSS is inserted before the browser computes the layout, so the browser calculates the correct width, height, and position the first time.

Why Not useEffect?

useEffect(() => {

  insertStyles();

}, []);

Timeline:

React Render
↓ DOM Commit
↓ Browser Calculates Layout
↓ Paint Screen
↓ useEffect
↓ Insert CSS
↓ Browser Recalculates Layout
The user may briefly see unstyled content (FOUC), and the browser may need another layout calculation after the CSS is inserted.

Why Not useLayoutEffect?

useLayoutEffect(() => {

  insertStyles();

}, []);

useLayoutEffect runs before the browser paints, so it is much better than useEffect.

However, React expects useLayoutEffect to read the layout that the browser has already computed—not to insert CSS.

DOM Commit
↓ Browser Calculates Styles & Layout
↓ useLayoutEffect
↓ Insert New CSS
↓ Browser Calculates Layout Again
↓ Paint
Because new CSS changes the appearance and size of elements, the browser may need to perform another layout calculation.
useInsertionEffect avoids this by inserting CSS before the browser performs its initial layout calculation.

Relationship With useLayoutEffect

The browser first computes the layout of every element.

Then useLayoutEffect runs, allowing our code to read the computed layout and make synchronous DOM updates before the browser paints.

Browser Calculates Styles & Layout
↓ useLayoutEffect
↓ Read offsetWidth
↓ Read getBoundingClientRect()
↓ Optional DOM Updates
↓ Paint
Important Difference

Browser Calculates Styles & Layout
  • The browser computes each element's width, height, position, margins, padding, and other layout information.
  • This calculation is performed automatically by the browser.
Read / Measure Computed Layout
  • Our JavaScript simply reads the values the browser has already calculated.
  • Examples:
    • offsetWidth
    • offsetHeight
    • clientWidth
    • clientHeight
    • scrollHeight
    • getBoundingClientRect()

The browser performs the calculations. Our code only reads the computed results.

Since useLayoutEffect depends on accurate layout information, all required CSS must already exist. That is exactly why useInsertionEffect runs first.

Real World Example

Imagine a CSS-in-JS library like Emotion or Styled Components. It generates CSS dynamically while React renders components.

Using useInsertionEffect (Correct)

React Render
↓ DOM Commit
↓ Generate CSS
↓ useInsertionEffect
↓ Insert CSS
↓ Browser Calculates Styles & Layout
↓ useLayoutEffect Reads Layout
↓ Paint Screen
The browser performs the layout calculation only once because all required CSS already exists.

React Render
↓ DOM Commit
↓ Browser Calculates Styles & Layout
↓ useLayoutEffect
↓ Insert CSS
↓ Browser Calculates Styles & Layout Again
↓ Paint Screen
Because new CSS was inserted after the browser had already computed the layout, the browser may need to calculate the layout again.

React Render
↓ DOM Commit
↓ Browser Calculates Styles & Layout
↓ Paint Screen
↓ useEffect
↓ Insert CSS
↓ Browser Calculates Styles & Layout Again
Users may briefly see unstyled content (Flash Of Unstyled Content - FOUC), and the browser performs another layout calculation after the CSS is inserted.

Can Application Developers Use It?

Almost never.

useInsertionEffect is not intended for normal application logic.

Most React developers should continue using:

  • useEffect for API calls, subscriptions, timers and other side effects.
  • useLayoutEffect for reading layout information and making synchronous DOM updates before paint.

Only library authors usually need:

  • useInsertionEffect to insert dynamically generated CSS.

What React Documentation Says

useInsertionEffect is primarily intended for CSS-in-JS library authors. It is rarely needed in normal React applications.

Key Takeaways

  • useInsertionEffect is designed primarily for CSS-in-JS libraries.
  • It runs immediately after React commits DOM updates but before the browser performs its layout calculation.
  • Its purpose is to ensure all required CSS exists before the browser computes element sizes and positions.
  • The browser performs the layout calculation automatically.
  • useLayoutEffect does not calculate layout. It reads the layout that the browser has already computed using APIs like offsetWidth and getBoundingClientRect().
  • If CSS is inserted inside useLayoutEffect, the browser may need to calculate the layout again because the styles have changed.
  • Using useEffect to insert CSS is even later, which can cause a Flash Of Unstyled Content (FOUC).
  • Most React application developers rarely need useInsertionEffect. It is mainly intended for library authors.

Interview Answer

useInsertionEffect is a React 18 hook primarily designed for CSS-in-JS libraries such as Emotion and Styled Components. It runs immediately after React commits DOM updates but before the browser calculates styles and layout. This allows libraries to inject CSS before the browser computes element sizes and positions, preventing unnecessary layout recalculations and flashes of unstyled content.

After the browser computes the layout, useLayoutEffect runs, allowing our code to read the computed layout using APIs like offsetWidth and getBoundingClientRect(), and make synchronous DOM updates before the screen is painted.

The execution order is:

React Render → DOM Commit → useInsertionEffect → Insert CSS → Browser Calculates Styles & Layout → useLayoutEffect → Read / Measure Computed Layout → Optional DOM Updates → Paint Screen → useEffect

Most application developers rarely use useInsertionEffect; it is mainly intended for CSS-in-JS library authors.
