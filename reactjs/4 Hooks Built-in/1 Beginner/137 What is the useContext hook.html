137. What is the useContext hook?

One-Line Definition

useContext is a React hook used to read and access Context values directly inside components.

Simple Understanding

useContext allows components to access shared/global data without manually passing props through multiple component levels.

Common Shared Data

  • Theme
  • Authenticated user
  • Language
  • Settings
  • Permissions

Main Problem Without useContext

Normally data is passed using props from parent to child.

Prop Drilling Problem

App
↓ props
Header
↓ props
Navbar
↓ props
Profile

Problem

Passing props through many components is called prop drilling.

How useContext Solves This

Context Provider Stores Shared Data
↓
Any Child Component Can Access It Directly

Basic Syntax

const value = useContext(MyContext);

Project Structure

src/
├── App.jsx
├── context/
│   └── ThemeContext.jsx
├── components/
│   └── Navbar.jsx

Step 1 - Create Context

context/ThemeContext.jsx

import { createContext } from 'react';

export const ThemeContext = createContext();

Step 2 - Consume Context Using useContext

components/Navbar.jsx

import { useContext } from 'react';
import { ThemeContext } from '../context/ThemeContext';

function Navbar() {

  const theme = useContext(ThemeContext);

  return <h1>{theme}</h1>;

}

export default Navbar;

Step 3 - Provide Context Value

App.jsx

import Navbar from './components/Navbar';
import { ThemeContext } from './context/ThemeContext';

function App() {

  const theme = 'dark';

  return (

    <ThemeContext.Provider value={theme}>

      <Navbar />

    </ThemeContext.Provider>

  );

}

export default App;

Output

dark

Complete Flow

createContext()
↓
Provider Stores Value
↓
useContext Reads Value

Why useContext Is Useful

  • Avoids prop drilling
  • Cleaner components
  • Easier global state sharing
  • Improves maintainability

Important Rule

useContext must be used inside the matching Context Provider.

Default Value Example

const ThemeContext = createContext('light');

Without Provider

light

Real Project Use Cases

Feature Shared Data

Props vs useContext

Props useContext
Manual passing Direct access
Can create prop drilling Avoids prop drilling
Good for local data Good for shared/global data

Important Interview Points

  • useContext reads Context data.
  • Used to avoid prop drilling.
  • Works with Context Provider.
  • Used for shared/global state.
  • Commonly used for themes and authentication.

Short Interview Answer

useContext is a React hook used to directly access shared Context data inside components without manually passing props through multiple levels.
