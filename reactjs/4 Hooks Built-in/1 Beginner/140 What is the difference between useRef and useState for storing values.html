140. What is the Difference Between useRef and useState for Storing Values?

One-Line Definition

useState stores values that affect the UI and trigger re-renders, while useRef stores values without triggering re-renders.

Quick Comparison

Feature useState useRef
Triggers Re-render ✅ Yes ❌ No
Updates UI ✅ Yes ❌ No
Stores Value Between Renders ✅ Yes ✅ Yes
Used For DOM Access ❌ No ✅ Yes
Causes Component Function to Run Again ✅ Yes ❌ No

useState Example

When the value changes, React re-renders the component and updates the UI. 

import { useState } from 'react';

function Counter() {

  const [count, setCount] = useState(0);

  return (
    <>
      <h1>{count}</h1>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
    </>
  );

}

Flow of useState

setCount(1)
↓
State Changes
↓
Component Re-renders
↓
UI Updates

useRef Example

When the value changes, React does not re-render the component. 

import { useRef } from 'react';

function Counter() {

  const countRef = useRef(0);

  const increment = () => {

    countRef.current++;

    console.log(countRef.current);

  };

  return (

    <button onClick={increment}>
      Increment
    </button>

  );

}

Flow of useRef

countRef.current++
↓
Ref Value Changes
↓
No Re-render
↓
UI Remains Same

Important Example

import { useState, useRef } from 'react';

function App() {

  const [stateCount, setStateCount] = useState(0);

  const refCount = useRef(0);

  return (
    <>
      <h2>State: {stateCount}</h2>
      <h2>Ref: {refCount.current}</h2>

      <button onClick={() => setStateCount(stateCount + 1)}>
        State +
      </button>

      <button onClick={() => refCount.current++}>
        Ref +
      </button>
    </>
  );

}

What Happens?

  • Clicking State + updates the screen immediately.
  • Clicking Ref + changes the value internally.
  • The screen does not update because no re-render occurs.

When To Use useState

  • Counter values
  • Form fields
  • API response data
  • Modal or popup visibility
  • Anything displayed on the screen

When To Use useRef

  • DOM element references
  • Input focus
  • Timers
  • Previous values
  • Values that should not trigger UI updates

Real Project Example

In our Aadhaar Authentication application:

useState is used for OTP values, loader state, consent state, and authentication results because the UI must update.

useRef is used for input references, timer IDs, SignalR connections, and tracking values because changing them should not re-render the entire component.

Important Interview Points

  • Both persist values between renders.
  • Only useState triggers re-renders.
  • useRef is commonly used for DOM access.
  • Changing ref.current does not update the UI.
  • UI-related data should usually be stored in useState.

Short Interview Answer

useState is used for values that affect the UI and should trigger re-renders, while useRef is used for values that should persist between renders without causing a re-render.

One-Line Interview Answer

useState updates the UI by re-rendering the component, while useRef stores values without triggering a re-render.
