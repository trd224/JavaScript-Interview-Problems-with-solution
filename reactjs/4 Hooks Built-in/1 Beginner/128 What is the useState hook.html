128. What is the useState hook?

One-Line Definition

useState is a React hook used to add and manage state inside functional components.

Simple Understanding

useState allows a component to:

  • Store data
  • Update data
  • Re-render UI when data changes

Main Idea

User Action
↓
State Updates
↓
Component Re-Renders
↓
UI Updates Automatically

Basic Syntax

const [state, setState] = useState(initialValue);

What useState Returns

Part Purpose
state Current state value
setState Function used to update state
initialValue Initial state value

Basic Example

import { useState } from 'react';

function App() {

  const [count, setCount] = useState(0);

  return (

    <button onClick={() => setCount(count + 1)}>{count}</button>

  );

}

What Happens Internally?

setCount()
↓
React Updates State
↓
Component Function Runs Again
↓
Updated UI Appears

Updating String State

const [name, setName] = useState('Rizwan');

setName('Aman');

Updating Boolean State

const [isOpen, setIsOpen] = useState(false);

setIsOpen(true);

Updating Object State

const [user, setUser] = useState({

  name: 'Rizwan',
  age: 25

});

setUser({

  ...user,
  age: 26

});

Updating Array State

const [users, setUsers] = useState([]);

setUsers([

  ...users,
  'Aman'

]);

Functional Update Version

setCount((prev) => prev + 1);

Why Functional Updates Are Useful

  • Uses latest state value
  • Avoids stale state problems
  • Safer for async updates

Multiple useState Hooks

function App() {

  const [name, setName] = useState('Rizwan');

  const [age, setAge] = useState(25);

}

Why useState Was Introduced

Before hooks, state was mainly available in class components.

Before Hooks

// Class Component

class Counter extends React.Component {

  state = {

    count: 0

  };

}

After Hooks

// Functional Component

function Counter() {

  const [count, setCount] = useState(0);

}

Benefits of useState

  • Simpler code
  • No this keyword
  • Cleaner functional components
  • Easy state management
  • Better readability

Rules of useState

Rule Meaning
Top-level only No loops or conditions
Only inside React functions Components or custom hooks

Real Project Use Cases

Feature State Managed
OTP input OTP value
Popup visibility Open/close state
Theme switch Dark/light mode
Captcha tracking Captcha state
API loading Loading/error state

Important Interview Points

  • useState works only in functional components.
  • It returns current state and setter function.
  • Updating state triggers re-render.
  • Multiple useState hooks are allowed.
  • State updates are asynchronous.

Short Interview Answer

useState is a React hook that allows functional components to store and update state, automatically re-rendering the UI when state changes.

One-Line Interview Answer

useState is a hook used to manage state inside functional components.
