130. What is the useEffect hook used for?

One-Line Definition

useEffect is a React hook used to handle side effects in functional components.

What Is a Side Effect?

A side effect means work that happens outside normal UI rendering.

Examples of Side Effects

  • API calls
  • Timers
  • Event listeners
  • DOM updates
  • Local storage access
  • Subscriptions

Do All API Calls Need useEffect?

API calls are side effects, but useEffect is mainly used for API calls that should run automatically after rendering. API calls triggered by user actions such as button clicks or form submissions are usually placed directly inside event handlers, not inside useEffect.

User Action → No useEffect Needed

function SaveButton() {

  const saveUser = async () => {

    await fetch('/api/user', {
      method: 'POST'
    });

  };

  return (
    <button onClick={saveUser}>
      Save
    </button>
  );

}

The API call runs only when the user clicks the button, so useEffect is not needed.

Automatic API Call → useEffect Needed

useEffect(() => {

  fetch('/api/users')
    .then(res => res.json())
    .then(data => setUsers(data));

}, []);

The API call runs automatically after the component renders, so useEffect is needed.

Interview Point

  • API calls are side effects.
  • GET requests on page load are commonly placed inside useEffect.
  • POST requests triggered by user actions are usually placed inside event handlers.
  • useEffect is for automatic side effects, not user-triggered actions.

Basic Syntax

useEffect(() => {

  // Side effect code

}, dependencies);

useEffect Without Dependency Array

import { useEffect } from 'react';

function App() {

  useEffect(() => {

    console.log('Runs After Every Render');

  });

}

What Happens?

This useEffect runs after every render because no dependency array is provided.

useEffect with Empty Dependency Array

useEffect(() => {

  console.log('Runs Once After First Render');

}, []);

What Happens?

This useEffect runs only once after the first render.

Equivalent Class Lifecycle

componentDidMount()

useEffect with Dependencies

import { useEffect, useState } from 'react';

function App() {

  const [count, setCount] = useState(0);

  useEffect(() => {

    console.log('Count Changed');

  }, [count]);

  return (

    <button onClick={() => setCount(count + 1)}>{count}</button>

  );

}

What Happens?

  • Runs after first render
  • Runs again whenever count changes

Main Dependency Flow

↓

useEffect Dependency Behavior

Code Behavior
useEffect(fn) Runs after every render
useEffect(fn, []) Runs once after first render
useEffect(fn, [count]) Runs when count changes

Cleanup Function

useEffect(() => {

  console.log('Mounted');

  return () => {

    console.log('Cleanup');

  };

}, []);

When Cleanup Runs

  • Before effect re-runs
  • When component unmounts

Why Cleanup Is Important

  • Prevents memory leaks
  • Stops duplicate listeners
  • Cleans subscriptions/timers

Real Timer Example

import { useEffect, useState } from 'react';

function App() {

  const [count, setCount] = useState(0);

  useEffect(() => {

    const timer = setInterval(() => {

      setCount((prev) => prev + 1);

    }, 1000);

    return () => clearInterval(timer);

  }, []);

  return <h1>{count}</h1>;

}

API Call Example

import { useEffect, useState } from 'react';

function App() {

  const [users, setUsers] = useState([]);

  useEffect(() => {

    fetch('https://api.com/users')
      .then((res) => res.json())
      .then((data) => setUsers(data));

  }, []);

}

Why useEffect Was Introduced

  • Simplified lifecycle handling
  • Reduced duplicate code
  • Made functional components powerful
  • Cleaner side-effect management

Rules of useEffect

Rule Meaning
Top-level only No loops or conditions
Dependency array controls execution Determines when effect re-runs
Cleanup removes effects Prevents memory leaks

Important Interview Points

  • useEffect handles side effects.
  • Runs after render.
  • Dependency array controls execution.
  • Cleanup prevents memory leaks.
  • Replaces lifecycle methods in functional components.
