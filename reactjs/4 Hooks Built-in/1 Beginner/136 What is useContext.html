136. What is useContext?

One-Line Definition

useContext is a React hook used to access shared Context data directly inside components.

Simple Understanding

useContext helps components access common/shared data without passing props manually through multiple component levels.

Common Shared Data Examples

  • Theme
  • Logged-in user
  • Language
  • Authentication state
  • Settings

Main Problem Without useContext

Normally data is passed from parent to child using props.

Prop Drilling Example

App
↓ props
Parent
↓ props
Child
↓ props
GrandChild

Problem

Passing props through many components is called prop drilling.

How useContext Solves This

Context Provider Stores Data
↓
Any Child Component Can Access It Directly

Basic Syntax

const value = useContext(MyContext);

Project Structure

src/
├── App.jsx
├── context/
│   └── UserContext.jsx
├── components/
│   └── Profile.jsx

Step 1 - Create Context

context/UserContext.jsx

import { createContext } from 'react';

export const UserContext = createContext();

Step 2 - Access Context Using useContext

components/Profile.jsx

import { useContext } from 'react';
import { UserContext } from '../context/UserContext';

function Profile() {

  const user = useContext(UserContext);

  return <h1>Hello {user}</h1>;

}

export default Profile;

Step 3 - Provide Context Value

App.jsx

import { useState } from 'react';
import { UserContext } from './context/UserContext';
import Profile from './components/Profile';

function App() {

  const [user] = useState('Rizwan');

  return (

    <UserContext.Provider value={user}>

      <Profile />

    </UserContext.Provider>

  );

}

export default App;

Output

Hello Rizwan

Provider Can Be Placed at Any Level

Context data flows from the nearest Provider to all nested children below it. The Provider does not have to be placed in the top parent component.

Mid Child → Nested Child Example

import { useState } from 'react';
import { UserContext } from './context/UserContext';
import NestedChild from './NestedChild';

function MidChild() {

  const [user] = useState('Rizwan');

  return (

    <UserContext.Provider value={user}>

      <NestedChild />

    </UserContext.Provider>

  );

}

Nested Child Reads Context

import { useContext } from 'react';
import { UserContext } from './context/UserContext';

function NestedChild() {

  const user = useContext(UserContext);

  return <h2>{user}</h2>;

}

Important Point

  • Provider can be placed in App, Parent, Child, or any component.
  • Only components below that Provider can access the Context value.
  • useContext always reads from the nearest matching Provider.

Complete Flow

createContext()
↓
Provider Stores Value
↓
useContext Reads Value

Why useContext Is Useful

  • Avoids prop drilling
  • Cleaner component structure
  • Easier state sharing
  • Useful for global/shared data

Real Project Example

Feature Shared Through Context
Authentication User data
Theme Dark/light mode
Language Translation settings
Shopping cart Cart items
Permissions User roles

Important Rule

useContext must be used inside a matching Context Provider.

Default Value Example

const UserContext = createContext('Guest');

Without Provider

Guest

Props vs useContext

Props useContext
Pass manually Access directly
Can create prop drilling Avoids prop drilling
Good for local data Good for shared/global data

Important Interview Points

  • useContext accesses Context data.
  • Used to avoid prop drilling.
  • Works with Context Provider.
  • Useful for shared/global state.
  • Often used for authentication and themes.

Short Interview Answer

useContext is a React hook that allows components to directly access shared Context data without manually passing props through multiple levels.
