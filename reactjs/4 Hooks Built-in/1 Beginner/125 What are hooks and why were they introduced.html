125. What are hooks and why were they introduced?

One-Line Definition

Hooks are special React functions that allow functional components to use state, lifecycle features, refs, context, and reusable logic.

Simple Understanding

Before hooks:

Functional components could only render UI.

Features like:

  • State
  • Lifecycle methods
  • Side effects
  • Refs
  • Context

were available mainly in class components.

Before Hooks

// Class Component

class Counter extends React.Component {

  state = {

    count: 0

  };

  render() {

    return (

      <h1>{this.state.count}</h1>

    );

  }

}

After Hooks

import { useState } from 'react';

function Counter() {

  const [count, setCount] = useState(0);

  return (

    <h1>{count}</h1>

  );

}

Why Hooks Were Introduced

Problem Before Hooks Hooks Solution
Complex class components Simpler functional components
this keyword confusion No this keyword needed
Lifecycle duplication useEffect simplified logic
Difficult logic reuse Custom hooks
Large boilerplate code Cleaner syntax

Most Common Hooks

Hook Purpose
useState State management
useEffect Side effects / lifecycle
useRef DOM references
useContext Context access
useMemo Performance optimization
useCallback Function memoization

Example of useState

import { useState } from 'react';

function App() {

  const [count, setCount] = useState(0);

  return (

    <button

      onClick={() => setCount(count + 1)}

    >

      {count}

    </button>

  );

}

Example of useEffect

import { useEffect } from 'react';

function App() {

  useEffect(() => {

    console.log('Component Mounted');

  }, []);

}

Custom Hooks

Hooks also introduced reusable logic.

Custom Hook Example

import { useState } from 'react';

function useCounter() {

  const [count, setCount] = useState(0);

  const increment = () => {

    setCount((prev) => prev + 1);

  };

  return {

    count,
    increment

  };

}

How To Use This Custom Hook

function App() {

  const { count, increment } = useCounter();

  return (

    <div>

      <h1>{count}</h1>

      <button onClick={increment}>
        Increment
      </button>

    </div>

  );

}

Why Custom Hooks Are Powerful

  • Reusable logic
  • Cleaner components
  • Better architecture
  • Easier maintenance

Rules of Hooks

Rule Meaning
Only call hooks at top level No loops or conditions
Only call hooks in React functions Components or custom hooks only
Hook names start with use Example: useState

Why Hooks Became Popular

Hooks made React applications simpler, cleaner, and easier to maintain.

Class Components vs Hooks

Class Components Hooks
Uses this keyword No this keyword
Complex lifecycle methods Simple useEffect
More boilerplate Cleaner syntax
Harder logic reuse Custom hooks

Real Project Use Cases

Feature Hook Used
OTP form useState
API calls useEffect
Captcha tracking useRef + useEffect
Theme management useContext
Performance optimization useMemo/useCallback

Important Interview Points

  • Hooks work only in functional components.
  • Introduced in React 16.8.
  • Removed need for most class components.
  • Custom hooks allow reusable logic.
  • Hooks simplified React architecture.

Short Interview Answer

Hooks are special React functions introduced to allow functional components to use state, lifecycle features, side effects, refs, and reusable logic without using class components.

One-Line Interview Answer

Hooks allow functional components to use React features like state and lifecycle behavior.
