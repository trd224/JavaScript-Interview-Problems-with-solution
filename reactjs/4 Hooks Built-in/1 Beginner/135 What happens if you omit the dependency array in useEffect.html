135. What happens if you omit the dependency array in useEffect?

One-Line Definition

If you omit the dependency array, useEffect runs after every render.

Simple Understanding

React executes the effect again whenever the component re-renders.

Main Idea

Component Renders
↓
useEffect Runs
↓
Component Re-Renders
↓
useEffect Runs Again

Basic Syntax

useEffect(() => {

  // Side effect

});

Basic Example

import { useEffect } from 'react';

function App() {

  useEffect(() => {

    console.log('Effect Executed');

  });

}

What Happens?

  • Effect runs after first render
  • Effect runs after every re-render

Example with State Updates

import { useEffect, useState } from 'react';

function App() {

  const [count, setCount] = useState(0);

  useEffect(() => {

    console.log('Effect Runs');

  });

  return (

    <button onClick={() => setCount(count + 1)}>{count}</button>

  );

}

What Happens?

Action Effect Runs?
Initial render ✅ Yes
Button click ✅ Yes
Every re-render ✅ Yes

Render Flow

First Render
↓
Effect Runs
↓
State Changes
↓
Re-Render Happens
↓
Effect Runs Again

Cleanup Behavior

useEffect(() => {

  console.log('Effect Started');

  return () => {

    console.log('Cleanup');

  };

});

What Happens?

  • Cleanup runs before every next effect execution
  • Cleanup runs on component unmount

Important Problem

useEffect(() => {

  setCount((prev) => prev + 1);

});

What Happens?

This creates an infinite render loop.

Why?

Effect Runs
↓
State Updates
↓
Component Re-Renders
↓
Effect Runs Again
↓
Infinite Loop

Difference Between Dependency Types

Code Behavior
useEffect(fn) Runs after every render
useEffect(fn, []) Runs once after first render
useEffect(fn, [count]) Runs when count changes

When Is Omitting Dependency Array Useful?

  • Rarely used in real projects
  • Useful when effect must run after every render
  • Mostly used for debugging or logging

Real Example

useEffect(() => {

  console.log('Current Render Completed');

});

Use Case

Logging render activity during development/debugging.

Real Project Problems

Problem Reason
Too many API calls Effect runs after every render
Performance issues Repeated side effects
Infinite loops State updates inside effect
Duplicate listeners Repeated registrations

Important Interview Points

  • Omitting dependency array means run after every render.
  • Cleanup runs before every next effect.
  • Can easily create infinite loops.
  • Rarely used in production.
  • Different from empty dependency array.

Short Interview Answer

If the dependency array is omitted, useEffect executes after every render and re-render of the component.

One-Line Interview Answer

Omitting the dependency array makes useEffect run after every render.
