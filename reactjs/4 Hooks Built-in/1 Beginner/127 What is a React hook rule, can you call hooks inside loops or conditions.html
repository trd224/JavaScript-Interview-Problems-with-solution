127. What is a React hook rule — can you call hooks inside loops or conditions?

One-Line Definition

No, hooks must never be called inside loops, conditions, or nested functions because React depends on consistent hook execution order. Hooks execute in the same order during every render.

Main Rule

Hooks must always be called at the top level of React components or custom hooks.

Correct Example

function App() {

  const [count, setCount] = useState(0);

  useEffect(() => {

    console.log(count);

  }, [count]);

}

Why This Is Correct

Hooks execute in the same order during every render.

Wrong Example - Inside Condition

// Wrong

function App() {

  if (true) {

    useEffect(() => {

      console.log('Hello');

    }, []);

  }

}

Why This Is Wrong

Conditional hooks may run in one render but not another render, breaking hook order.

Wrong Example - Inside Loop

// Wrong

function App() {

  for (let i = 0; i < 3; i++) {

    useEffect(() => {

      console.log(i);

    }, []);

  }

}

Why This Is Wrong

Number of hook calls may change dynamically.

Wrong Example - Nested Function

// Wrong

function App() {

  function test() {

    useEffect(() => {

      console.log('Hello');

    }, []);

  }

}

Why This Is Wrong

React cannot guarantee consistent hook execution order.

Main Internal Concept

React Render
↓
React Reads Hooks Sequentially
↓
React Maps Hook State By Order

How React Tracks Hooks

React does NOT identify hooks using variable names.

React tracks hooks using: 

Execution Order

Example

// First Render

useState();   // Slot 1
useEffect();  // Slot 2

// Second Render

useState();   // Slot 1

Problem

React expects useEffect() in slot 2, but it disappeared.

Result

  • Broken state mapping
  • Unexpected behavior
  • Invalid hook call errors
  • Incorrect state values

Correct Pattern for Conditions

Put conditions INSIDE the hook.

Correct Example

function App() {

  useEffect(() => {

    if (true) {

      console.log('Hello');

    }

  }, []);

}

Why This Works

Hook itself still executes in consistent order.

Custom Hook Example

function useCounter() {

  const [count, setCount] = useState(0);

  return {

    count,
    setCount

  };

}

How To Use This Hook

function App() {

  const { count, setCount } = useCounter();

  return (

    <button onClick={() => setCount(count + 1)}>{count}</button>

  );

}

Rules of Hooks Summary

Allowed? Situation
✅ Yes Top-level component code
✅ Yes Inside custom hooks
❌ No Inside if conditions
❌ No Inside loops
❌ No Inside nested functions
❌ No Inside normal JS functions

Important Interview Points

  • Hooks must always execute in same order.
  • Never call hooks inside conditions or loops.
  • React tracks hooks using execution order.
  • Conditions should go inside hooks.
  • Hooks work only inside React functions or custom hooks.

Short Interview Answer

Hooks cannot be called inside loops, conditions, or nested functions because React relies on consistent hook execution order to correctly map hook state during renders.

One-Line Interview Answer

Hooks must only be called at the top level of React components or custom hooks.
