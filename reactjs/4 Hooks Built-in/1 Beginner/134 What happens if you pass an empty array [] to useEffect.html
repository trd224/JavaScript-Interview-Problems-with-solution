134. What happens if you pass an empty array [] to useEffect?

One-Line Definition

Passing an empty dependency array [] makes useEffect run only once after the first render.

Simple Understanding

React runs the effect one time because there are no dependencies to watch.

Main Idea

Component Mounts
↓
useEffect Runs Once
↓
No Dependency Changes
↓
Effect Does Not Re-Run

Basic Syntax

useEffect(() => {

  // Side effect

}, []);

Basic Example

import { useEffect } from 'react';

function App() {

  useEffect(() => {

    console.log('Runs Once');

  }, []);

}

What Happens?

  • Component renders
  • useEffect runs once
  • Effect does not run again

Equivalent Class Lifecycle

componentDidMount()

Why?

Empty dependency array means React has no changing dependency to monitor.

Render Flow

First Render
↓
Effect Runs
↓
Future Re-Renders
↓
Effect Skipped

Example with State Updates

import { useEffect, useState } from 'react';

function App() {

  const [count, setCount] = useState(0);

  useEffect(() => {

    console.log('Runs Once');

  }, []);

  return (

    <button onClick={() => setCount(count + 1)}>{count}</button>

  );

}

What Happens?

  • Effect runs only after first render
  • Clicking button re-renders component
  • Effect does not run again

Cleanup with Empty Array

useEffect(() => {

  console.log('Mounted');

  return () => {

    console.log('Cleanup');

  };

}, []);

What Happens?

Phase Behavior
Component mounts Effect runs once
Component unmounts Cleanup runs once

Real Example with API Call

import { useEffect, useState } from 'react';

function App() {

  const [users, setUsers] = useState([]);

  useEffect(() => {

    fetch('https://api.com/users')
      .then((res) => res.json())
      .then((data) => setUsers(data));

  }, []);

}

Why Use Empty Array Here?

We usually want the API call only once when the component loads.

Common Real-World Uses

Feature Reason
API calls Load data once
Authentication check Run once on startup
Initialize WebSocket Create single connection
Load theme Read localStorage once
OTP timer setup Start timer once

Difference Between Dependency Types

Code Behavior
useEffect(fn) Runs after every render
useEffect(fn, []) Runs once after first render
useEffect(fn, [count]) Runs when count changes

Important Interview Points

  • Empty dependency array means run once.
  • Effect runs after first render.
  • Cleanup runs on unmount.
  • Equivalent to componentDidMount().
  • Commonly used for API calls and initialization.

Short Interview Answer

Passing an empty dependency array [] makes useEffect run only once after the component’s first render.

One-Line Interview Answer

[] makes useEffect run only once after initial render.
