126. What are the rules of hooks?

One-Line Definition

Rules of Hooks are special rules that ensure React hooks work correctly and maintain proper state behavior.

Main Rules of Hooks

Rule Meaning
Only call hooks at top level Do not call hooks inside loops, conditions, or nested functions
Only call hooks in React functions Hooks can be used only inside components or custom hooks
Custom hooks must start with use Example: useCounter, useAuth

Why These Rules Exist

React depends on hook call order to correctly preserve state between renders.

Main Internal Concept

Component Render
↓
React Reads Hooks In Order
↓
React Matches State Correctly

Rule 1 - Only Call Hooks at Top Level

Hooks should always run in the same order.

Correct Example

function App() {

  const [count, setCount] = useState(0);

  useEffect(() => {

    console.log(count);

  }, [count]);

}

Wrong Example

// Wrong  

function App() {

  if (true) {

    useEffect(() => {

      console.log('Hello');

    }, []);

  }

}

Why This Is Wrong

Conditional hooks can break hook order between renders.

Problem Example

// First render

useState();
useEffect();

// Second render

useState();

Problem

React becomes confused because hook order changed.

Rule 2 - Only Call Hooks Inside React Functions

Hooks can only be used inside:

  • React functional components
  • Custom hooks

Correct Example

function App() {

  const [count, setCount] = useState(0);

}

Wrong Example

// Wrong - Can not use inside a normal function

function calculate() {

  const [count, setCount] = useState(0);

}

Why?

React hooks require React rendering context.

Rule 3 - Custom Hooks Must Start With "use"

Correct Example

function useCounter() {

  const [count, setCount] = useState(0);

}

Wrong Example

// Wrong naming

function counterHook() {

  const [count, setCount] = useState(0);

}

Why Naming Matters

  • React ESLint rules detect hooks
  • Improves readability
  • Helps developers identify hooks easily

Custom Hook Example

import { useState } from 'react';

function useCounter() {

  const [count, setCount] = useState(0);

  const increment = () => {

    setCount((prev) => prev + 1);

  };

  return {

    count,
    increment

  };

}

How To Use This Hook

function App() {

  const {

    count,
    increment

  } = useCounter();

  return (

    <button onClick={increment}>

      {count}

    </button>

  );

}

What React Internally Does

First Hook → State Slot 1
↓
Second Hook → State Slot 2
↓
Third Hook → State Slot 3

Why Hook Order Is Important

React identifies hooks by their execution order.

NOT by variable names.

Rules of Hooks ESLint Plugin

React provides ESLint rules to automatically detect hook mistakes. 

eslint-plugin-react-hooks

Common Hook Mistakes

Mistake Problem
Hooks inside if statements Hook order breaks
Hooks inside loops Inconsistent execution
Hooks inside event handlers Invalid hook call
Hooks inside normal functions No React context

Real Project Use Cases

Feature Hook Usage
OTP forms useState

Authentication system Custom auth hooks

Important Interview Points

  • Hooks must always run in same order.
  • Never call hooks inside conditions or loops.
  • Hooks work only in React functions.
  • Custom hooks start with use.
  • React tracks hooks using execution order.

Short Interview Answer

Rules of Hooks ensure hooks execute in a consistent order by allowing hooks only at the top level of React components or custom hooks, preventing broken state behavior.

One-Line Interview Answer

Hooks must be called only at the top level of React components or custom hooks.
