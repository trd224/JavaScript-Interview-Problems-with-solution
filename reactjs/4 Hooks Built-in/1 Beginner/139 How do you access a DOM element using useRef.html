139. How do you access a DOM element using useRef in React?

One-Line Definition

useRef is a React hook used to directly access and interact with a DOM element without re-rendering the component.

Why Is It Used?

We use useRef when our application needs direct access to a real DOM element.

  • Focus an input field
  • Scroll to an element
  • Play or pause a video
  • Access an element's width or height
  • Modify a DOM element directly

Basic Example

import { useRef } from 'react';

function App() {
  const inputRef = useRef(null);

  const focusInput = () => {
    inputRef.current.focus();
  };

  return (
    <>
      <input ref={inputRef} type="text" />
      <button onClick={focusInput}>Focus Input</button>
    </>
  );
}

export default App;

How It Works

First, we create a ref object using useRef(null). 

const inputRef = useRef(null);

It creates an object like this: 

{
  current: null
}

Then we connect that ref object with the input element. 

<input ref={inputRef} type="text" />

After rendering, React stores the real DOM element inside the current property. 

inputRef.current

Now inputRef.current points to the actual input element. 

<input type="text">

Main Flow

useRef(null)
↓
Connect with DOM element using ref
↓
React stores DOM element in current
↓
Access DOM using ref.current

Common DOM Methods

inputRef.current.focus();

inputRef.current.value = 'Hello';

inputRef.current.scrollIntoView();

Real-World Example

In our application, we can automatically focus the Aadhaar input when the popup opens. 

import { useEffect, useRef } from 'react';

function AadhaarPopup() {
  const aadhaarRef = useRef(null);

  useEffect(() => {
    aadhaarRef.current.focus();
  }, []);

  return <input ref={aadhaarRef} placeholder="Enter Aadhaar Number" />;
}

export default AadhaarPopup;

Output


Popup Opens

↓

Aadhaar input automatically gets focus

Important Point

Updating ref.current does not re-render our component, because refs are mainly used for storing mutable values or DOM references.

useRef vs useState

useRef useState
Stores DOM reference or mutable value Stores UI state
Does not cause re-render Causes re-render
Access value using ref.current Access value directly from state variable
Best for DOM access Best for UI updates

Important Interview Points

  • useRef returns an object with a current property.
  • The DOM element is accessed using ref.current.
  • Changing ref.current does not re-render the component.
  • It is commonly used for focus, scroll, video control, and DOM measurement.

Short Interview Answer

We access a DOM element in React by creating a ref with useRef(null), attaching it to the element using the ref attribute, and then accessing the element through ref.current.

One-Line Interview Answer

useRef allows us to get a reference to a DOM element through the current property and interact with it directly without causing a component re-render.
