258. How Do You Create a 404 Not Found Page?

One-Line Definition

A 404 Not Found page is created by defining a catch-all route that renders a custom component when no route matches the current URL.

What Is a 404 Page?

A 404 page is shown when a user visits a URL that does not exist in our application.

Example: 

/about     ✅ Exists

/contact   ✅ Exists

/xyz       ❌ Does Not Exist

/abc       ❌ Does Not Exist
Instead of showing a blank page, React Router can display a custom "Page Not Found" screen.

How React Router Handles 404 Pages

User Opens URL
↓
React Router Checks Routes
↓
Route Found ?
↓
Yes → Render Component

No
↓
Render 404 Page

Step 1 - Create a NotFound Component

function NotFound() {

  return (

    <h1>

      404 - Page Not Found

    </h1>

  );

}

Step 2 - Add a Catch-All Route

import { Routes, Route } from 'react-router-dom';

<Routes>

  <Route path="/" element={<Home />} />

  <Route path="/about" element={<About />} />

  <Route path="/contact" element={<Contact />} />

  <Route path="*" element={<NotFound />} />

</Routes>
The "*" route matches any URL that does not match another route.

Why Must the "*" Route Be Last?

The wildcard route should always be placed at the end because it catches all unmatched URLs. 
<Routes>

  <Route path="/" element={<Home />} />

  <Route path="/users" element={<Users />} />

  <Route path="*" element={<NotFound />} />

</Routes>

Complete Example

import { BrowserRouter, Routes, Route } from 'react-router-dom';

function NotFound() {

  return (

    <h1>

      404 - Page Not Found

    </h1>

  );

}

function App() {

  return (

    <BrowserRouter>

      <Routes>

        <Route path="/" element={<Home />} />

        <Route path="/about" element={<About />} />

        <Route path="/contact" element={<Contact />} />

        <Route path="*" element={<NotFound />} />

      </Routes>

    </BrowserRouter>

  );

}

Adding a Home Link

import { Link } from 'react-router-dom';

function NotFound() {

}

Real Project Example

In our Aadhaar Authentication application, if a user manually enters an invalid URL such as /audit/unknown-page or /invalid-route, a custom 404 page can be displayed with a button that redirects them back to the Dashboard.

Important Interview Points

  • A 404 page handles invalid URLs.
  • React Router v6 uses path="*" for unmatched routes.
  • The wildcard route should be placed after all valid routes.
  • A custom NotFound component is commonly used.
  • We can provide navigation back to the home page.

Short Interview Answer

A 404 page is created by defining a NotFound component and mapping it to a wildcard route using path="*" in React Router v6.

One-Line Interview Answer

Create a 404 page by defining a wildcard route with path="*" that renders a NotFound component.
