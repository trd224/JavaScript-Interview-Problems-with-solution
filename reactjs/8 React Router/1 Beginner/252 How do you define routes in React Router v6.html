252. How Do You Define Routes in React Router v6?

One-Line Definition

In React Router v6, routes are defined using the Routes component and one or more Route components.

Why Do We Define Routes?

React Router needs to know which component should be displayed for a particular URL.

Installation

npm install react-router-dom

Step 1 - Wrap the Application with BrowserRouter

import { BrowserRouter } from 'react-router-dom';
import App from './App';

export default function Root() {

  return (
    <BrowserRouter>
      <App />
    </BrowserRouter>
  );

}

Step 2 - Import Routes and Route

import { Routes, Route } from 'react-router-dom';

Step 3 - Define Routes

import { Routes, Route } from 'react-router-dom';

function App() {

  return (

    <Routes>

      <Route path="/" element={<Home />} />

      <Route path="/about" element={<About />} />

      <Route path="/contact" element={<Contact />} />

    </Routes>

  );

}

Important Interview Points

  • Routes contains all Route components.
  • Route maps a URL path to a React component.
  • BrowserRouter enables routing.
  • React Router v6 uses element instead of component.
  • React Router v6 uses Routes instead of Switch.
  • The element prop receives JSX.

Short Interview Answer

In React Router v6, routes are defined using the Routes component and Route components. Each Route maps a URL path to a React component through the element prop.

One-Line Interview Answer

Routes in React Router v6 are defined using Routes and Route components with the element prop.
