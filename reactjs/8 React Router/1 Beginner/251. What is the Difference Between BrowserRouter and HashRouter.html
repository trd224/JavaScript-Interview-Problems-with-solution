251. What is the Difference Between BrowserRouter and HashRouter?

One-Line Definition

BrowserRouter uses clean URLs like /about and needs server configuration, while HashRouter uses URLs like /#/about and works without special server configuration.

Main Confusion

In a React Single Page Application, the server usually has only one main HTML file. 

/* React build output */

build/
  index.html
  main.js
  style.css
There is no real about.html, no real contact.html, and no real dashboard.html. These pages are React components, not server files.

Important Point

Before React can run, the browser must first receive index.html from the server.

BrowserRouter Example

https://myapp.com/about

When a user opens this URL directly or refreshes the page, the browser sends this request to the server: 

GET /about

By default, the server looks for a real file or folder: 

/about
/about.html
/about/index.html

But in React SPA, these files do not exist.

Without Server Configuration

Browser Requests /about
↓
Server Looks for /about File
↓
File Not Found
↓
404 Not Found
↓
React Never Loads

With Server Configuration

We configure the server like this:

If the requested file exists, send that file. If the file does not exist, send index.html.
Browser Requests /about
↓
Server Sends index.html
↓
React Loads
↓
React Router Reads /about
↓
About Component Renders

How React Identifies Which Page to Render?

The server sends the same index.html, but the browser URL remains the same. 

https://myapp.com/about

After React loads, React Router reads: 

window.location.pathname

Result: 

"/about"

Then React Router checks the route configuration: 

<Routes>
  <Route path="/" element={<Home />} />
  <Route path="/about" element={<About />} />
  <Route path="/contact" element={<Contact />} />
</Routes>
Because the current path is /about, React Router renders the About component.

Does Server Send index.html Every Time?

Action Server Sends index.html? Reason
User opens website first time ✅ Yes Browser requests the server
User refreshes /about ✅ Yes Browser sends GET /about
User pastes /contact in new tab ✅ Yes Direct URL request goes to server
User clicks React Router Link ❌ No Navigation happens inside React

Client-Side Navigation Example

import { Link } from 'react-router-dom';

function Navbar() {
  return (
    <nav>
      <Link to="/">Home</Link>
      <Link to="/about">About</Link>
      <Link to="/contact">Contact</Link>
    </nav>
  );
}
When we click Link, React Router changes the URL without asking the server for index.html again.

HashRouter Example

https://myapp.com/#/about

In HashRouter, everything after # stays inside the browser.

Browser sends only this to the server: 

GET /

The server never receives: 

#/about
Browser Requests /
↓
Server Sends index.html
↓
React Loads
↓
HashRouter Reads #/about
↓
About Component Renders

BrowserRouter vs HashRouter

Feature BrowserRouter HashRouter
URL Format /about /#/about
Server Sees Route ✅ Yes, server sees /about ❌ No, server only sees /
Server Configuration Required ✅ Yes ❌ No
URL Look Clean and professional Contains # symbol
Refresh Problem Can happen if server is not configured Usually no refresh problem
Best Use Production apps with server control Static hosting or no server control

BrowserRouter Setup

import { BrowserRouter, Routes, Route } from 'react-router-dom';

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </BrowserRouter>
  );
}

HashRouter Setup

import { HashRouter, Routes, Route } from 'react-router-dom';

function App() {
  return (
    <HashRouter>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </HashRouter>
  );
}

Nginx Setup for BrowserRouter

# React SPA fallback configuration

location / {
  try_files $uri $uri/ /index.html;
}
This means: first try to find a real file. If not found, return index.html so React Router can handle the route.

Real Project Understanding

In our Aadhaar Authentication application, routes like /dashboard, /audit, /reports, and /settings are React routes. The server should return index.html, then React Router reads the URL and renders the correct component.

Important Interview Points

  • BrowserRouter uses clean URLs like /about.
  • HashRouter uses URLs like /#/about.
  • BrowserRouter requires server fallback to index.html.
  • HashRouter does not need server fallback because # routes are not sent to the server.
  • React Router identifies the page using the current browser URL.
  • During Link navigation, index.html is not requested again.

Short Interview Answer

BrowserRouter uses clean URLs and requires the server to return index.html for unknown routes, while HashRouter stores routes after the # symbol, so the server only receives the root URL and no special configuration is needed.

One-Line Interview Answer

BrowserRouter needs server fallback for clean URLs, while HashRouter avoids server fallback by keeping routes after # in the browser.
