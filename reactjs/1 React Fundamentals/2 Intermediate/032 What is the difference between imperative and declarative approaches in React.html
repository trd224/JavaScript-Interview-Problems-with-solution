030. What is the Difference Between Imperative and Declarative Approaches in React?

One-Line Definition

Imperative programming describes how to perform a task, while declarative programming describes what the final result should be.

What is Imperative Programming?

In an imperative approach, we manually write every step required to achieve the result.

Tell the Computer
HOW
to Perform the Task

Imperative JavaScript Example

const button = document.getElementById('btn');

button.style.background = 'blue';

button.innerText = 'Clicked';

Here we manually tell JavaScript exactly what to do:

  • Find the button
  • Change its background color
  • Change its text

What is Declarative Programming?

In a declarative approach, we describe the desired UI and let React handle the updates.

Tell React
WHAT
the UI Should Look Like

Declarative React Example

function App() {

  const isClicked = true;

  return (
    <button>
      {isClicked ? 'Clicked' : 'Click Me'}
    </button>
  );

}

We simply describe the UI and React updates the DOM automatically.

Real-Life Analogy

Imperative Declarative
Go to the kitchen, take a cup, pour water, add tea, boil it, and serve it. Please bring me a cup of tea.

Imperative explains every step. Declarative describes only the desired result.

Counter Example

Imperative Approach

let count = 0;

const button = document.getElementById('btn');

button.addEventListener('click', () => {

  count++;

  document.getElementById('count').innerText = count;

});

We manually update the DOM after every click.

Declarative React Approach

import { useState } from 'react';

function Counter() {

  const [count, setCount] = useState(0);

  return (
    <div>
      <h2>{count}</h2>

      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
    </div>
  );

}

export default Counter;

React automatically updates the UI when state changes.

How React Works Internally

State Changes
↓
React Re-renders Component
↓
Virtual DOM Updated
↓
Real DOM Updated

Because React handles DOM updates, we rarely need direct DOM manipulation.

Imperative vs Declarative

Feature Imperative Declarative
Focus How to do it What the result should be
DOM Updates Manual Automatic
Code Size More Less
Readability Lower Higher
Maintenance Harder Easier
React Style Rarely Used Preferred

Can React Use Imperative Code?

Yes. React is primarily declarative, but sometimes imperative code is needed.

Example Using useRef

import { useRef } from 'react';

function Input() {

  const inputRef = useRef();

  const focusInput = () => {
    inputRef.current.focus();
  };

  return (
    <>
      <input ref={inputRef} />
      <button onClick={focusInput}>
        Focus
      </button>
    </>
  );

}

This is an imperative action because we directly tell the DOM element to focus.

Real Project Example

In our Aadhaar Authentication application, we usually update React state and let React update the UI automatically. This is a declarative approach. However, when focusing an input, opening a device bridge popup, or accessing a DOM element through useRef, we use an imperative approach.

Important Interview Points

  • Imperative = How to perform a task.
  • Declarative = What the final result should be.
  • React is primarily declarative.
  • State changes automatically update the UI.
  • useRef is commonly used for imperative DOM operations.
  • Declarative code is easier to read and maintain.

Short Interview Answer

Imperative programming focuses on describing the steps needed to perform a task, while declarative programming focuses on describing the desired result. React follows a declarative approach where developers describe the UI and React handles DOM updates automatically.

One-Line Interview Answer

Imperative programming tells the computer how to do something, while declarative programming tells it what the final result should be.
