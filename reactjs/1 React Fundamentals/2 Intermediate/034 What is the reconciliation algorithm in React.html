034. What is the reconciliation algorithm in React?

One-Line Definition

The reconciliation algorithm in React is the diffing algorithm that compares old and new Virtual DOM trees to efficiently update the real DOM.

Simple Understanding

Whenever state or props change:

  • React creates a new Virtual DOM.
  • React compares it with the previous Virtual DOM.
  • React finds differences.
  • Only changed parts are updated in the real DOM.

This comparison system is called:

Reconciliation Algorithm

Main Goal of Reconciliation

React wants to avoid expensive full DOM updates.

Instead of replacing the whole UI, React updates only necessary DOM nodes.

Flow Diagram

State/Props Change
↓
New Virtual DOM Created
↓
Diffing Algorithm Runs
↓
Compare Old vs New Virtual DOM
↓
Find Changes
↓
Update Only Changed Real DOM Nodes

Example

function App() {

  const [count, setCount] = React.useState(0);

  return (

    <div>

      <h1>{count}</h1>

      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>

    </div>

  );

}

What Happens Internally?

Action React Behavior
Button clicked State changes
Component re-renders New Virtual DOM created
Diffing starts Old vs new Virtual DOM compared
Difference found Only h1 text changed
DOM updated Only h1 updated

Important Rule 1 - Different Element Types

If element types are different, React destroys the old tree and creates a new one. 

// Old

<h1>Hello</h1>

// New

<p>Hello</p>

React removes h1 and creates p.

Important Rule 2 - Same Element Types

If element types are the same, React updates only changed attributes. 

// Old

<h1 className="red">Hello</h1>

// New

<h1 className="blue">Hello</h1>

React updates only: 

className

Important Rule 3 - List Reconciliation

React uses keys to identify changed list items. 

users.map(user => (
  <li key={user.id}>
    {user.name}
  </li>
));

Why Keys Are Important

  • Faster list updates
  • Correct item tracking
  • Better performance
  • Prevents UI bugs

Without Keys Problem

React may unnecessarily re-render list items.

Why React’s Algorithm Is Fast

A complete DOM comparison would be very slow.

React uses optimized assumptions:

  • Different element types produce different trees.
  • Keys identify stable list items.

Complexity Advantage

Approach Complexity
Traditional tree diffing O(n³)
React diffing algorithm O(n)

Real Project Use Cases

Feature Benefit
Chat applications Fast message updates
Dashboards Efficient data refresh
OTP forms Minimal DOM changes
Large tables Optimized rendering

Important Interview Points

  • React uses a diffing algorithm for reconciliation.
  • Compares old and new Virtual DOM trees.
  • Updates only changed real DOM nodes.
  • Keys optimize list reconciliation.
  • React’s diffing complexity is O(n).

Short Interview Answer

The reconciliation algorithm is React’s diffing process that compares old and new Virtual DOM trees to efficiently update only changed parts of the real DOM.
