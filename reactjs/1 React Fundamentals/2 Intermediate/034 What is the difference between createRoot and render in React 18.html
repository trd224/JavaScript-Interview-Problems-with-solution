032. What is the difference between createRoot and render in React 18?

One-Line Definition

createRoot() creates and manages the React root container, while render() displays React elements inside that root.

Simple Understanding

In React 18:

  • createRoot() prepares the React application root.
  • render() renders the UI into that root.

Basic Example

import { createRoot } from 'react-dom/client';
import App from './App';

const root = createRoot(
  document.getElementById('root')
);

root.render(<App />);

What Happens Internally?

Code Purpose
createRoot() Create React root container
root.render() Render React UI inside root

Real-Life Analogy

React Real-Life Example
createRoot() Create a TV screen
render() Display a movie on the TV

Detailed Difference

createRoot() render()
Creates root object Renders UI
Called once usually Can be called multiple times
Connects React to DOM container Displays components
Setup phase Rendering phase
Returns root instance No root creation

Flow Diagram

HTML root div → createRoot() → React Root Created → render() → UI Displayed

Why React 18 Uses This Pattern

React 18 introduced a new rendering architecture for:

  • Concurrent rendering
  • Better performance
  • Automatic batching
  • Improved rendering control

Important Concept

createRoot() itself does NOT display UI. render() actually renders the UI.

What Does createRoot() Return?

const root = createRoot(
  document.getElementById('root')
);

It returns a React root object.

That object contains methods like: 

root.render()
root.unmount()

render() Example

root.render(<App />);

This tells React:

"Render the App component inside the created root container."

Can render() Be Called Again?

Yes. 

root.render(<Login />);

root.render(<Dashboard />);

React updates the UI accordingly.

Real Project Use Cases

Feature Role
Application startup createRoot()
Rendering dashboard render()
Theme switching render() updates UI
Authentication app render() updates components

Important Interview Points

  • createRoot() creates React root container.
  • render() displays React UI.
  • Both are used together in React 18.
  • createRoot() supports concurrent rendering features.
  • render() can re-render updated UI.

Short Interview Answer

In React 18, createRoot() creates the React root container, while render() renders React components into that root.
