033. What is the reconciliation process in React?

One-Line Definition

Reconciliation is the React process of comparing old and new Virtual DOM trees to update only the changed parts in the real DOM.

Why Reconciliation Is Needed

Updating the real DOM directly is slow and expensive.

React improves performance by updating only the necessary DOM elements instead of reloading the whole page.

Simple Understanding

Whenever:

  • state changes
  • props change
  • component re-renders

React creates a new Virtual DOM and compares it with the previous Virtual DOM.

This comparison process is called:

Reconciliation

Flow Diagram

State Change
↓
New Virtual DOM Created
↓
Compare Old vs New Virtual DOM
↓
Find Differences
↓
Update Only Changed Real DOM Parts

Example

function App() {

  const [count, setCount] = React.useState(0);

  return (

    <div>

      <h1>{count}</h1>

      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>

    </div>

  );

}

What Happens Internally?

Step React Behavior
Button clicked State changes
Component re-renders New Virtual DOM created
Comparison happens Old vs new Virtual DOM
Difference found Only h1 text changed
Real DOM updated Only h1 updated

Important Point

React does NOT replace the entire DOM during re-rendering.

React updates only the changed elements.

What Algorithm React Uses?

React uses a fast comparison algorithm called:

Diffing Algorithm

Diffing Algorithm Rules

Rule Behavior
Different element types Destroy old tree and create new tree
Same element type Update only changed attributes
List rendering Uses key prop for optimization

Example of Same Element Type

// Old

<h1 className="red">Hello</h1>

// New

<h1 className="blue">Hello</h1>

React only updates: 

className

Example of Different Element Type

// Old

<h1>Hello</h1>

// New

<p>Hello</p>

React destroys old h1 and creates new p element.

Role of Keys in Reconciliation

Keys help React identify changed list items efficiently. 

users.map(user => (
  <li key={user.id}>
    {user.name}
  </li>
));

Without Keys Problem

  • Incorrect DOM updates
  • Poor performance
  • UI bugs in lists

Why Reconciliation Is Fast

  • Virtual DOM comparison is lightweight
  • Only changed nodes are updated
  • Minimizes expensive real DOM operations

Real Project Use Cases

Feature Reconciliation Benefit
Live dashboards Efficient updates
Chat applications Fast UI rendering
OTP forms Minimal input updates
Large tables Optimized list rendering

Important Interview Points

  • Reconciliation compares old and new Virtual DOM.
  • React uses the diffing algorithm.
  • Only changed DOM parts are updated.
  • Keys help optimize list reconciliation.
  • Improves React application performance.

Short Interview Answer

Reconciliation is React’s process of comparing old and new Virtual DOM trees to efficiently update only the changed parts in the real DOM.
