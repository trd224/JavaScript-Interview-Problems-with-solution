046. What are double buffering and current/work-in-progress trees in Fiber?

One-Line Definition

React Fiber uses a technique called double buffering. It maintains two Fiber trees: the Current Tree (what the user sees) and the Work-In-Progress Tree (what React is preparing next).

Why Does React Need Two Trees?

React should not change the visible UI while it is still calculating updates.

What Is Double Buffering?

Double buffering means React keeps two copies of the Fiber tree and switches between them when updates are ready.

Interview Answer

Double buffering in React Fiber means React maintains two Fiber trees simultaneously: the Current Tree and the Work-In-Progress Tree. The Current Tree represents the UI currently displayed to the user, while the Work-In-Progress Tree represents the next version of the UI being prepared during the Render Phase. Once React finishes building the WIP tree, the Commit Phase swaps it with the Current Tree. This architecture allows React to prepare updates safely, support scheduling, pause and resume work, and enable concurrent rendering features.

One-Line Interview Answer

Double buffering means React keeps both a Current Tree and a Work-In-Progress Tree, then swaps them during commit when the new UI is ready.
