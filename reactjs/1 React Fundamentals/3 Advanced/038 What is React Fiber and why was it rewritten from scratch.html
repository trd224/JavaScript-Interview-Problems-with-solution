038. What is React Fiber and why was it rewritten from scratch?

One-Line Definition

React Fiber is React’s rewritten rendering engine that allows React to pause, prioritize, reuse, and manage rendering work more efficiently.

What Was Before Fiber?

Before Fiber, React used the old:

Stack Reconciler

The old system rendered everything synchronously.

Problem with Old React Engine

Once rendering started:

  • React could not pause rendering
  • React could not prioritize updates
  • Large UI updates could block the browser
  • Animations and interactions could lag

Simple Example of Old Problem

Suppose a huge dashboard is rendering.

Meanwhile the user clicks a button.

Old React would:

Finish the entire rendering first before handling the button click.

This caused UI lag and poor responsiveness.

Why Fiber Was Created

Facebook rewrote React from scratch to support:

  • Smoother UI rendering
  • Interruptible rendering
  • Prioritized updates
  • Concurrent rendering
  • Better animations
  • Improved responsiveness

What Fiber Actually Is

Fiber is:

A new reconciliation engine and internal tree structure.

Each component becomes a Fiber node.

Fiber Tree Example

App Fiber
↓
Dashboard Fiber
↓
Sidebar Fiber
↓
Content Fiber

Main Features of Fiber

Feature Purpose
Pause rendering Prevent browser blocking
Resume rendering Continue work later
Prioritize updates Handle important UI first
Abort rendering Skip outdated work
Concurrent rendering Smoother UI experience

Important Concept

Fiber breaks rendering work into small units.

This allows React to work incrementally instead of blocking the browser.

How Fiber Works

Update Happens
↓
React Creates Fiber Work Units
↓
React Processes Small Tasks
↓
Browser Gets Time to Respond
↓
React Continues Remaining Work

Why This Is Better

  • Responsive UI
  • Better animations
  • Smoother scrolling
  • Less browser freezing
  • Efficient large application rendering

Fiber and Concurrent Rendering

Modern React features like:

  • Suspense
  • Transitions
  • Concurrent rendering
  • Streaming SSR

are possible because of Fiber.

Real Project Example

Fiber vs Old Stack Reconciler

Old Stack Reconciler React Fiber
Synchronous rendering Interruptible rendering
No prioritization Priority scheduling
Could block browser Better responsiveness
Single large task Small incremental tasks
Limited scalability Optimized for large apps

Important Note

Fiber is completely internal. Developers usually do not directly interact with Fiber APIs.

Important Interview Points

  • Fiber is React’s rewritten rendering engine.
  • Introduced to solve UI blocking problems.
  • Supports concurrent rendering.
  • Breaks rendering into small tasks.
  • Allows pausing, resuming, and prioritizing updates.

Short Interview Answer

React Fiber is the rewritten React reconciliation engine designed to support interruptible, prioritized, and concurrent rendering for better UI responsiveness and performance.

One-Line Interview Answer

React Fiber is React’s modern rendering engine that enables efficient, interruptible, and prioritized UI rendering.
