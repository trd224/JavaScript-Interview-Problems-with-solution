043. How does React Fiber enable time slicing and interruptible rendering?

One-Line Definition

React Fiber stores rendering work inside Fiber objects, allowing React to split large rendering tasks into small units. This architecture made time slicing and interruptible rendering possible in React 18 concurrent rendering.

Important Correction

Fiber was introduced in React 16.
Time slicing and interruptible rendering became available through concurrent rendering features built on top of Fiber in React 18.

Interview Answer

React Fiber enables time slicing and interruptible rendering by representing rendering work as Fiber objects instead of relying solely on the JavaScript call stack. This allows React to break rendering into smaller units, track progress, and schedule work more flexibly. Fiber was introduced in React 16 as the foundation, and React 18 used that foundation to provide concurrent rendering features such as time slicing, interruptible rendering, transitions, and better update prioritization.

One-Line Interview Answer

Fiber enables time slicing and interruptible rendering by storing work in Fiber objects that React can pause, prioritize, and resume, a capability fully utilized by React 18 concurrent rendering.
