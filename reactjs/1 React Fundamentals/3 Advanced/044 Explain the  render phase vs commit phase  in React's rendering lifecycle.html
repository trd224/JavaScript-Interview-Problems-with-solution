044. Explain the render phase vs commit phase in React's rendering lifecycle.

One-Line Definition

React rendering happens in two phases: Render Phase calculates what should change, and Commit Phase applies those changes to the real DOM.

Render Phase

During Render Phase, React runs components, builds the Fiber tree, compares old UI with new UI, and determines what changes are needed.

Example

  return <h1>{count}</h1>;

Render Phase does not update the real DOM. It only calculates what should change.

Commit Phase

Commit Phase applies the work prepared during Render Phase to the real DOM.

↓

↓

↓

Users see UI changes only after Commit Phase completes.

Render Phase discovers the difference. Commit Phase updates the actual DOM node.

Can Render Phase Be Interrupted?

Yes. In React 18 concurrent rendering, Render Phase can be paused, interrupted, resumed, or restarted.

Can Commit Phase Be Interrupted?

No. Once Commit Phase starts, React must finish it because it is modifying the real DOM.

Interview Answer

React's rendering lifecycle has two phases. During the Render Phase, React runs components, builds the Fiber tree, compares old and new UI, and calculates what changes are needed. No DOM updates happen here. During the Commit Phase, React applies those calculated changes to the real DOM. In React 18, the Render Phase can be interrupted and resumed, but the Commit Phase cannot be interrupted because it modifies the DOM.

One-Line Interview Answer

Render Phase calculates changes; Commit Phase applies those changes to the real DOM.
