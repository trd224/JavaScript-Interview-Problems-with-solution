041. What is React Fiber and why was it rewritten from scratch?

One-Line Definition

React Fiber is the rewritten reconciliation architecture(React's reconciliation engine) introduced in React 16. It replaced the old Stack Reconciler and created the foundation for better error handling, scheduling, prioritization, and future concurrent rendering features.

Very Important Correction

React Fiber came in React 16. Full Concurrent Rendering behavior came later in React 18.
React 16
↓
Fiber Architecture Introduced
↓
Rendering Work Became Fiber-Based
↓
Foundation Ready

React 18
↓
Concurrent Rendering Built On Fiber
↓
Interruptible / Prioritized Rendering Features Exposed

Why Was React Fiber Needed?

Before Fiber, React used the old Stack Reconciler. It used a recursive call stack style rendering approach.

State Change
↓
React Starts Rendering
↓
Old Stack Reconciler Processes Tree
↓
Must Finish Work
↓
Cannot Easily Pause Or Resume
The old architecture was rigid. It was hard to pause work, resume work, assign priorities, or recover from errors properly.

Main Purpose Of Fiber In React 16

The main purpose of Fiber in React 16 was not to give developers full Concurrent Rendering immediately. Its main purpose was to replace the old reconciliation engine with a more flexible architecture.
Old Stack Reconciler
↓
Hard To Pause / Resume / Prioritize
↓
React 16 Fiber Rewrite
↓
Work Stored As Fiber Objects
↓
Future Scheduling And Concurrent Features Became Possible

Fiber Engine vs Fiber Object

React Fiber, the engine, and a Fiber object are related, but they are not the same thing.
Term Meaning
React Fiber The rewritten reconciliation architecture / engine
Fiber Object A JavaScript object representing one component and its work
Fiber Tree A tree made from many Fiber objects
React Fiber Engine
↓
Uses Many Fiber Objects
↓
Builds Fiber Tree
↓
Performs Reconciliation

What Is A Fiber Object?

A Fiber object is an internal JavaScript object that stores information about a component, its props, state, child, sibling, parent, and pending work.
Simplified Fiber Object
{
  type: Component,
  props: {},
  stateNode: {},
  child: Fiber,
  sibling: Fiber,
  return: Fiber
}

Component Tree To Fiber Tree

React Component Tree
<App>
  <Header />
  <Dashboard />
</App>
Fiber(App)
│
├── Fiber(Header)
│
└── Fiber(Dashboard)
React creates Fiber objects for components. Many Fiber objects together create the Fiber tree.

Why Fiber Objects Helped

Old React depended heavily on the JavaScript call stack. Fiber gave React its own internal work structure.

What React 16 Got Immediately From Fiber

1. Better Error Handling

React 16 introduced Error Boundaries. Fiber made it easier for React to recover from rendering errors.

Error Boundary Example
class ErrorBoundary extends React.Component {
  state = { hasError: false };

  static getDerivedStateFromError() {
    return { hasError: true };
  }

  render() {
    if (this.state.hasError) {
      return <h2>Something went wrong</h2>;
    }

    return this.props.children;
  }
}
Component Crashes
↓
Error Boundary Catches Error
↓
Fallback UI Shows
↓
Whole App Does Not Crash

2. More Flexible Reconciliation

Fiber changed reconciliation from one rigid recursive process into a system where work could be represented, tracked, and managed as units.
Rendering Work
↓
Fiber Object
↓
Track What To Do
↓
Move To Next Fiber

3. Foundation For Priority System

Fiber introduced the foundation for prioritizing different types of work.

Urgent Work
↓
User Typing / Clicking

Less Urgent Work
↓
Large UI Update
In React 16, this was mostly internal foundation. React 18 later exposed modern concurrent behavior more clearly.

4. Foundation For Future Features

Many modern React features became possible because of Fiber architecture.

  • Concurrent Rendering
  • Suspense improvements
  • Transitions
  • Selective Hydration
  • Better scheduling
  • Improved error recovery

React 16 Fiber vs React 18 Concurrent Rendering

Version Main Point
React 16 Introduced Fiber architecture and replaced Stack Reconciler
React 17 Mostly a stepping-stone release with gradual upgrade improvements
React 18 Used Fiber foundation for Concurrent Rendering features
Simple line: Fiber was the foundation in React 16; React 18 used that foundation for Concurrent Rendering.

Most Important Interview Point

Do not say React 16 Fiber and React 18 Concurrent Rendering are exactly the same thing.

Say: Fiber was introduced in React 16 as the internal architecture. React 18 later exposed concurrent rendering capabilities built on top of Fiber.

Interview Answer

React Fiber is the rewritten reconciliation architecture introduced in React 16. It replaced the old Stack Reconciler, which was rigid and stack-based. Fiber represents rendering work using Fiber objects, allowing React to track work more flexibly, improve error handling, and build a foundation for scheduling and prioritization. The main use of Fiber in React 16 was to create this new internal architecture, while React 18 later used the Fiber foundation to enable Concurrent Rendering features like interruptible rendering, transitions, and better prioritization.

One-Line Interview Answer

React Fiber is React 16's rewritten reconciliation architecture that replaced the old Stack Reconciler and created the foundation for React 18 concurrent rendering.
