045. What is a Work-In-Progress (WIP) Tree in React Fiber?

One-Line Definition

A Work-In-Progress (WIP) Tree is a temporary Fiber tree that React builds while calculating the next UI update. The currently visible UI stays unchanged until React finishes the WIP tree and commits it.

Why Does React Need It?

React should not modify the screen while it is still calculating updates.

After commit, the old current tree is replaced by the completed Work-In-Progress Tree.

Why Is WIP Tree Important For Fiber?

Fiber can pause, resume, restart, or discard work because updates are stored inside the Work-In-Progress Tree instead of directly changing the DOM.

React 16 vs React 18 Connection

React 16 introduced Fiber and the Work-In-Progress Tree architecture.

React 18 uses this architecture to support concurrent rendering features.

Interview Answer

A Work-In-Progress Tree is the temporary Fiber tree React builds during the Render Phase. It contains the next version of the UI while the current tree continues to represent what the user sees. Once React finishes building the WIP tree, the Commit Phase swaps it with the current tree, making the new UI visible. This architecture allows React Fiber to support scheduling, pausing, resuming, and concurrent rendering features.

One-Line Interview Answer

The Work-In-Progress Tree is React Fiber's temporary tree that stores the next UI state before it is committed to the screen.
