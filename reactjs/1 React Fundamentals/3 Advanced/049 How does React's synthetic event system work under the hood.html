049. How does React's synthetic event system work under the hood?

One-Line Simple Answer

React's Synthetic Event System is a cross-browser event handling system that wraps native browser events inside a SyntheticEvent object and dispatches them to the correct React event handlers using event delegation.

First, Forget React

Start with normal HTML.

Plain HTML
<div id="root">
  <button id="save">Save</button>
  <button id="delete">Delete</button>
</div>

There are two buttons: Save and Delete.

Step 1 - Browser Creates An Event

When you click Save, the browser creates a native click event.

User Clicks Save Button
↓
Browser Creates Click Event

When you click Delete, browser again creates a click event.

User Clicks Delete Button
↓
Browser Creates Click Event
Yes, every click creates a new event object. But this is different from creating many event listeners.

Event Object vs Event Listener

Thing Meaning
Event Object Created every time user clicks
Event Listener Function attached to DOM to listen for events
Many clicks create many events, but React can still use one real listener to catch them.

Step 2 - Event Bubbling

After the click event is created, it travels upward.

Clicked Button
↓
Root Div
↓
Body
↓
Document
This upward movement is called event bubbling.

Step 3 - Event Delegation

Instead of attaching listeners to both buttons, we can attach one listener to the root.

Only One Real Listener
root.addEventListener('click', event => {
  console.log('clicked element:', event.target.id);
});
Save Button Clicked
↓
Event Bubbles To Root
↓
Root Listener Runs
↓
event.target = save

Delete Button Clicked
↓
Event Bubbles To Root
↓
Same Root Listener Runs
↓
event.target = delete
This technique is called event delegation.

Most Important Difference

Concept Simple Meaning
Bubbling Event moves from child to parent
Delegation Parent/root catches child events using bubbling
SyntheticEvent React wrapper around browser event

Now Come To React

You write this:

React Code
<button onClick={save}>Save</button>
<button onClick={deleteItem}>Delete</button>

It looks like there are two browser listeners, but React mainly stores these handlers internally.

Save Button
↓
React stores: onClick → save()

Delete Button
↓
React stores: onClick → deleteItem()
React handler and browser listener are not the same thing.

React's Real Listener Concept

React attaches a delegated browser listener near the root container.

Conceptual Internal Idea
root.addEventListener('click', reactInternalClickHandler);
React Root
↓
One Real Click Listener
↓
Handles Clicks From Many Children

Clear The Main Confusion

Every click creates a new event object, but React does not need a separate real browser listener for every button.
Question Answer
Does clicking Save create an event? Yes
Does clicking Delete create an event? Yes
Does React need separate real browser listeners for both? No
How does React know which handler to run? By checking event.target and React's internal handler mapping

What Is SyntheticEvent?

SyntheticEvent is React's normalized wrapper around the browser's native event.
Example
function handleClick(event) {
  console.log(event); // React SyntheticEvent
  console.log(event.nativeEvent); // Real browser event
}

React gives you event, but the real browser event is still available as event.nativeEvent.

Why React Creates SyntheticEvent?

  • To give consistent event behavior.
  • To normalize browser differences.
  • To connect events with React state updates.
  • To support React's bubbling and capturing behavior.
  • To batch state updates triggered inside event handlers.

React 16 vs React 17+

React Version Where Delegated Listener Is Attached
React 16 and before document
React 17+ React root container
Modern React attaches delegated listeners to the React root container instead of the document.

Stop Bubbling

Example
function App() {
  return (
    <div onClick={() => console.log('Parent')}>
      <button onClick={e => {
        e.stopPropagation();
        console.log('Child');
      }}>Click</button>
    </div>
  );
}
Output
Child
e.stopPropagation() stops the event from reaching the parent handler.

Interview Answer

React's synthetic event system works using event delegation. When we write handlers like onClick, React stores those handlers internally instead of attaching separate real browser listeners to every element. A native browser event is created whenever the user interacts with the page, and that event bubbles up to the React root. React catches it using a delegated root listener, wraps it in a SyntheticEvent, checks the event target, finds the matching React handler, and executes it. This provides consistent event behavior and integrates events with React's update system.

One-Line Interview Answer

React catches bubbling native events at the root, wraps them in SyntheticEvent, and dispatches them to the correct stored React handler.
