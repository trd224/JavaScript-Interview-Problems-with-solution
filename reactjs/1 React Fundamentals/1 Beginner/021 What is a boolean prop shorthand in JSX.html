021. What is a boolean prop shorthand in JSX?

One-Line Definition

Boolean prop shorthand in JSX means writing a prop without assigning true explicitly.

Why Is It Used?

JSX allows a shorter syntax for boolean props to make code cleaner and easier to read.

Normal Boolean Prop

<User isAdmin={true} />

Boolean Prop Shorthand

<User isAdmin />

Important Point

Both syntaxes are exactly the same in React.
Syntax Meaning
<User isAdmin={true} /> Passes true
<User isAdmin /> Also passes true

Component Example

function User(props) {

  return (

    <h1>

      {
        props.isAdmin
          ? 'Admin User'
          : 'Normal User'
      }

    </h1>

  );

}

Using Boolean Shorthand

<User isAdmin />

Output

Admin User

Passing false

Shorthand only works for true.

To pass false, you must write it explicitly. 

<User isAdmin={false} />

Flow Diagram

Prop Without Value → React Treats It As true

Real HTML Similarity

This behavior is similar to HTML boolean attributes.

HTML Example

<input disabled />

Here, disabled automatically means: 

<input disabled="true" />

Real Project Use Cases

Scenario Usage
Disabled button <Button disabled />
Admin access <User isAdmin />
OTP verification <Input required />
Dark mode <Theme darkMode />

Important Note

Boolean shorthand only works when the value should be true.

If you need false or dynamic values, use curly braces.

Dynamic Boolean Example

const isAdmin = true;

<User isAdmin={isAdmin} />

Important Interview Points

  • Boolean shorthand automatically passes true.
  • <Comp active /> means active={true}.
  • Cleaner and shorter JSX syntax.
  • Cannot use shorthand for false.
  • Similar to HTML boolean attributes.

Short Interview Answer

Boolean prop shorthand is a JSX feature where writing a prop without a value automatically passes true.

One-Line Interview Answer

In JSX, a prop written without a value automatically becomes true.
