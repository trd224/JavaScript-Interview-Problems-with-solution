003. Can browsers read JSX directly? What transforms it?

One-Line Definition

Browsers cannot understand JSX directly because JSX is not valid JavaScript. JSX must first be transformed into normal JavaScript.

Why Is It Used?

JSX makes React code easier to write and understand, but browsers only understand pure JavaScript.

Therefore, JSX must be converted before the browser can execute it.

Who Transforms JSX?

Babel transforms JSX into normal JavaScript.

Real Project Flow

Step Process
1 Developer writes JSX
2 Babel converts JSX into JavaScript
3 Browser reads the converted JavaScript

Example JSX Code

const element = (
  <h1>Hello React</h1>
);

Browser Cannot Understand This

The browser does not understand:

  • <h1>
  • JSX syntax
  • HTML inside JavaScript

Babel Transforms It Into

const element =
  React.createElement(
    'h1',
    null,
    'Hello React'
  );

Now Browser Can Understand It

After transformation, the browser executes normal JavaScript code.

Real Project Use Case

Suppose you create:

  • Login Form
  • OTP Popup
  • Dashboard
  • Profile Component

You write all UI using JSX, and tools like Babel + Webpack/Vite automatically convert JSX into browser-readable JavaScript.

Important Interview Point

JSX is syntactic sugar over React.createElement().

Detailed Understanding

JSX

<button>
  Click Me
</button>

Converted JavaScript

React.createElement(
  'button',
  null,
  'Click Me'
);

Important Interview Points

  • Browsers cannot read JSX directly.
  • JSX is not valid JavaScript.
  • Babel converts JSX into JavaScript.
  • React.createElement() is generated internally.
  • Build tools like Vite/Webpack use Babel automatically.

Short Interview Answer

Browsers cannot understand JSX directly. Babel converts JSX into normal JavaScript that browsers can execute.

One-Line Interview Answer

Babel transforms JSX into browser-readable JavaScript.
