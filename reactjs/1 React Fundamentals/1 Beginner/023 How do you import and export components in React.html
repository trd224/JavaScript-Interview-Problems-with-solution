023. How do you import and export components in React?

One-Line Definition

Import and export are used to share React components between different files.

Why Is It Used?

Large React applications contain many components.

Export allows a component to be shared, and import allows it to be used in another file.

Types of Export

Type Description
Default Export Exports one main value from a file
Named Export Exports multiple values from a file

Default Export

User.js

function User() {

  return (
    <h1>User Component</h1>
  );

}

export default User;

App.js

import User from './User';

function App() {

  return (
    <User />
  );

}

Output

User Component

Important Point About Default Export

A file can have only one default export.

Named Export

utils.js

export function add(a, b) {

  return a + b;

}

export function subtract(a, b) {

  return a - b;

}

App.js

import { add, subtract } from './utils';

console.log(add(10, 5));
console.log(subtract(10, 5));

Output

15
5

Important Point About Named Export

Named exports must use curly braces during import.

Default vs Named Export

Default Export Named Export
Only one per file Multiple allowed
No curly braces during import Requires curly braces
Can rename freely during import Name must match export

Importing Multiple Named Exports

import { add, subtract } from './utils';

Renaming Named Imports

import { add as sum } from './utils';

console.log(sum(5, 5));

Import All Named Exports

import * as math from './utils';

console.log(math.add(5, 5));

Flow Diagram

Export Component → Import Component → Reuse Component Anywhere

Real Project Use Cases

File Purpose
Login.js Export Login component
Dashboard.js Import Navbar component
api.js Export API helper functions
utils.js Export reusable functions

Why Import/Export Is Important

  • Code reusability
  • Better project structure
  • Cleaner code organization
  • Easier maintenance
  • Modular application architecture

Important Interview Points

  • Default export exports one main value.
  • Named export exports multiple values.
  • Named imports require curly braces.
  • React applications heavily use modular imports.
  • Import/export is based on JavaScript ES Modules.

Short Interview Answer

Import and export are used to share components and functions between files. React mainly uses default exports for components and named exports for utilities or multiple values.

One-Line Interview Answer

Import and export allow React components and functions to be reused across files.
