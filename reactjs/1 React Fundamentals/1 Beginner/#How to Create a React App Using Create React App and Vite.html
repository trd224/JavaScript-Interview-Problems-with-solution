Install React App: Create React App vs Vite, npm vs npx

One-Line Answer

For a new React app, use Vite: npm create vite@latest my-app. Create React App works, but it is old and can create dependency problems like the AJV error.

What Is -g?

-g means global install. The package is installed on your computer and its command becomes available everywhere.
Global CRA Install
npm install -g create-react-app
create-react-app my-app
Problem: global tools can become old. If you installed CRA globally one year ago, your computer may still use that old version.

Your CRA Error: AJV

Your error was: Cannot find module 'ajv/dist/compile/codegen'. This usually happens because CRA uses older tooling like react-scripts, webpack-dev-server, schema-utils, ajv, and ajv-keywords. One package expects another package version, but the installed version does not match.

Fix For CRA AJV Error

Try This First Inside CRA Project
npm install --save-dev ajv
npm start
If Still Error Comes
rmdir /s /q node_modules
del package-lock.json
npm install
npm install --save-dev ajv
npm start
Do not run npm audit fix --force immediately. It can upgrade breaking dependencies and make CRA worse.

Why CRA Uses npm start But Vite Uses npm run dev?

These commands come from the scripts section of package.json.
CRA package.json Scripts
{
  "scripts": {
    "start": "react-scripts start",
    "build": "react-scripts build",
    "test": "react-scripts test"
  }
}
Vite package.json Scripts
{
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview"
  }
}
So CRA uses npm start because it has a start script. Vite uses npm run dev because it has a dev script.

Final Interview Answer

To create a React app, older tutorials use Create React App. With npm, CRA was often installed globally using npm install -g create-react-app, but this can become outdated. A better CRA method is npx create-react-app my-app, which runs the creator temporarily. However, CRA is now old and can cause dependency issues like AJV errors. For modern React apps, Vite is preferred: npm create vite@latest my-app. Vite does not need -g because npm create downloads and runs the latest creator package temporarily.
