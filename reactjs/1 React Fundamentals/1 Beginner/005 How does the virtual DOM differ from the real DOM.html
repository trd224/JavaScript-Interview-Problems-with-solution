005. How does the Virtual DOM differ from the Real DOM?

One-Line Definition

The Real DOM is the actual webpage inside the browser, while the Virtual DOM is a lightweight JavaScript copy used by React for faster updates.

What Is the Real DOM?

The Real DOM is the actual HTML structure rendered inside the browser.

Example Real DOM

<body>

  <h1>Hello</h1>

  <p>Welcome</p>

</body>

Every change in the Real DOM directly affects the browser UI.

What Is the Virtual DOM?

The Virtual DOM is a JavaScript object representation of the Real DOM.

Example Virtual DOM

const virtualDOM = {
  type: 'h1',
  props: {
    children: 'Hello'
  }
};

React uses this lightweight copy to compare UI changes before updating the Real DOM.

Main Difference

Virtual DOM Real DOM
Lightweight JavaScript object Actual browser DOM
Fast updates Slow updates
Updates only changed parts May re-render larger sections
Exists in memory Exists in browser
Used by React internally Used by browser rendering engine
Improves performance Direct manipulation is expensive

How React Uses the Virtual DOM

Step Process
1 User action changes state
2 React creates a new Virtual DOM
3 React compares old vs new Virtual DOM
4 React finds changed elements
5 Only changed parts update in Real DOM

Example

Initial UI

<h1>Hello</h1>

Updated UI

<h1>Hello Rizwan</h1>

React compares the old and new Virtual DOM and updates only the changed text, instead of reloading the entire page.

Real Project Use Case

Suppose you are building:

  • Live Chat Application
  • OTP Verification Screen
  • Stock Market Dashboard
  • Notification System

Data changes continuously. React uses the Virtual DOM to efficiently update only changed UI elements, making applications fast and smooth.

Example Code

import { useState } from 'react';

function Counter() {

  const [count, setCount] = useState(0);

  return (
    <div>

      <h1>{count}</h1>

      <button
        onClick={() => setCount(count + 1)}
      >

        Increment

      </button>

    </div>
  );
}

What Happens Internally?

  • Button click changes state
  • React creates a new Virtual DOM
  • React compares old vs new Virtual DOM
  • Only the changed count updates in Real DOM

Important Interview Points

  • Virtual DOM is a copy of the Real DOM.
  • Real DOM updates are slower.
  • Virtual DOM improves performance.
  • React uses diffing and reconciliation.
  • Only changed elements update in the Real DOM.

Short Interview Answer

The Real DOM is the actual browser DOM, while the Virtual DOM is a lightweight copy used by React to optimize UI updates and improve performance.

One-Line Interview Answer

React uses the Virtual DOM to compare changes and efficiently update the Real DOM.
