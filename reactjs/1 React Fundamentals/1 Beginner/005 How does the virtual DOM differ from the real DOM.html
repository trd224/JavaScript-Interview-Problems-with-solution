005. How does the Virtual DOM differ from the Real DOM?

One-Line Definition

Both Real DOM and Virtual DOM are object trees. The Real DOM is created by the browser and is connected to the actual screen, layout, CSS, paint, and events. The Virtual DOM is a lightweight JavaScript object tree created by React to calculate what changed before updating the Real DOM.

Important Correction

Do not say: “Real DOM is HTML and Virtual DOM is an object.”
Correct answer: Real DOM is also exposed as JavaScript objects by the browser, but it is heavier because it is connected to rendering.

What Is the Real DOM?

The Real DOM is the browser-created object tree for your HTML document. It represents the actual webpage that the user sees.

HTML Written by Developer

<body>
  <h1>Hello</h1>
  <p>Welcome</p>
</body>

Browser Creates Real DOM Objects

const heading = document.querySelector('h1');

console.log(typeof heading); // object

So yes, the Real DOM is also an object. But it is not just a simple object. It is connected to the browser rendering system.

Real DOM Object Is Connected To

  • Browser screen
  • CSS engine
  • Layout calculation
  • Paint and repaint
  • Browser event system

Why Real DOM Updates Can Be Expensive

When you directly change the Real DOM, the browser may need to recalculate styles, update layout, and repaint pixels on the screen. 

heading.innerText = 'Hello Rizwan';

This change looks small, but the browser may need to do extra rendering work behind the scenes.

What Is the Virtual DOM?

The Virtual DOM is React's lightweight JavaScript object representation of the UI.

Example Virtual DOM

const virtualDOM = {
  type: 'h1',
  props: {
    children: 'Hello'
  }
};

This object is only data. It is not directly connected to the browser screen.

Virtual DOM Is Not Connected To

  • No direct screen rendering
  • No layout calculation
  • No repaint
  • No browser CSS engine connection

Main Difference

Virtual DOM Real DOM
Created by React Created by the browser
Plain lightweight JavaScript object tree Browser DOM object tree
Not directly connected to the screen Connected to the actual webpage screen
Cheap to create and compare Expensive to update frequently
Used for diffing and reconciliation Used for actual rendering
React updates it first React updates it only where needed

How React Uses the Virtual DOM

Step Process
1 State or props change
2 React creates a new Virtual DOM tree
3 React compares old Virtual DOM with new Virtual DOM
4 React finds the exact UI difference
5 React updates only the required part of the Real DOM

Example

Old UI

<h1>Hello</h1>

New UI

<h1>Hello Rizwan</h1>

React Comparison

Old Virtual DOM: children = 'Hello'
New Virtual DOM: children = 'Hello Rizwan'

Result: only text changed

React does not reload the full page. It updates only the changed text node in the Real DOM.

Example Code

import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h1>{count}</h1>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;

What Happens Internally?

  • Button click changes React state.
  • React creates a new Virtual DOM.
  • React compares old Virtual DOM and new Virtual DOM.
  • React finds that only count text changed.
  • React updates only that text in the Real DOM.

Real Project Use Case

Suppose you are building a live chat, OTP screen, stock dashboard, or notification system. Data changes again and again. React uses Virtual DOM comparison to avoid unnecessary Real DOM work and update only the changed UI parts.

Important Interview Points

  • Real DOM is also an object tree, created by the browser.
  • Virtual DOM is a lightweight object tree, created by React.
  • Real DOM is connected to rendering, layout, paint, CSS, and events.
  • Virtual DOM is only an in-memory UI representation.
  • React compares old and new Virtual DOM using diffing.
  • React updates the Real DOM using reconciliation.
  • Only changed Real DOM parts are updated.

Short Interview Answer

The Real DOM is the browser-created object tree connected to the actual webpage rendering, while the Virtual DOM is React's lightweight JavaScript object tree used to calculate UI changes before updating the Real DOM.

One-Line Interview Answer

Both are object trees, but the Real DOM belongs to the browser and affects the screen, while the Virtual DOM belongs to React and is used to efficiently find and apply changes.
