031. How do you prevent default browser behavior in React events?

One-Line Definition

In React, default browser behavior is prevented using event.preventDefault().

What Is Default Browser Behavior?

Browsers automatically perform some built-in actions when events occur.

Examples

Element Default Browser Behavior
form Page reload on submit
a tag Navigate to another page
checkbox Toggle checked state
right click Open context menu

How React Prevents Default Behavior

React provides: 

event.preventDefault()

This stops the browser’s default action.

Most Common Example - Form Submit

function App() {

  const handleSubmit = (event) => {

    event.preventDefault();

    console.log('Form Submitted');

  };

  return (

    <form onSubmit={handleSubmit}>

      <input type="text" />

      <button>
        Submit
      </button>

    </form>

  );

}

Without preventDefault()

Form Submit
↓
Browser Reloads Page

With preventDefault()

Form Submit
↓
Browser Reload Stopped
↓
React Handles Logic

Why React Applications Use It

React applications are usually:

Single Page Applications (SPA)

React handles UI updates itself without full page reloads.

Anchor Tag Example

function App() {

  const handleClick = (event) => {

    event.preventDefault();

    console.log('Navigation Stopped');

  };

  return (

    <a
      href="https://google.com"
      onClick={handleClick}
    >

      Open Google

    </a>

  );

}

Output

Navigation Stopped

Browser will NOT open Google.

Important Difference

preventDefault() stopPropagation()
Stops browser default behavior Stops event propagation
Prevents reload/navigation Prevents parent handlers

Can Both Be Used Together?

Yes. 

const handleClick = (event) => {

  event.preventDefault();

  event.stopPropagation();

};

Real Project Use Cases

Feature Why preventDefault() Is Used
Login form Prevent page reload
OTP verification Handle API calls manually
Custom routing Prevent full navigation
Popup forms Prevent browser submission

Important Point About Synthetic Events

React events are SyntheticEvents.

But preventDefault() works the same way as native browser events.

Check If Default Was Prevented

event.defaultPrevented

Example

const handleClick = (event) => {

  event.preventDefault();

  console.log(event.defaultPrevented);

};

Output

true

Important Interview Points

  • preventDefault() stops browser default behavior.
  • Commonly used with forms and links.
  • Very important in SPA applications.
  • Different from stopPropagation().
  • Works with React SyntheticEvents.

Short Interview Answer

In React, default browser actions like form submission or link navigation are prevented using event.preventDefault(), allowing React to control application behavior manually.

One-Line Interview Answer

React prevents default browser behavior using event.preventDefault().
