264. How do you implement a protected/private route?

One-Line Definition

A protected/private route is implemented by checking whether a user is authenticated before rendering a page. If not authenticated, the user is redirected to the login page.

Implementation Flow

User Opens Route
↓
ProtectedRoute Component
↓
Check Authentication
↓
Logged In ?
↓
YES → Render Page
↓
NO → Redirect Login

src/

Step 1 - Create ProtectedRoute Component

File: src/components/ProtectedRoute.jsx 
import { Navigate } from 'react-router-dom';

function ProtectedRoute({ children }) {
  const isAuthenticated = localStorage.getItem('token');

  return isAuthenticated
    ? children
    : <Navigate to="/login" replace />;
}

export default ProtectedRoute;

This component checks whether the user is logged in.

If logged in: 

return children;

If not logged in: 

return <Navigate to="/login" replace />;

Step 2 - Create Dashboard Page

File: src/pages/Dashboard.jsx 
function Dashboard() {
  return <h1>Dashboard Page</h1>;
}

export default Dashboard;

Step 3 - Create Login Page

File: src/pages/Login.jsx 
function Login() {

  function handleLogin() {
    localStorage.setItem('token', 'abc123');
    window.location.href = '/dashboard';
  }

  return (
    <>
      <h1>Login Page</h1>

      <button onClick={handleLogin}>
        Login
      </button>
    </>
  );
}

export default Login;

Step 4 - Configure Routes

File: src/App.jsx 
import { BrowserRouter, Routes, Route } from 'react-router-dom';

import Home from './pages/Home';
import Login from './pages/Login';
import Dashboard from './pages/Dashboard';
import ProtectedRoute from './components/ProtectedRoute';

function App() {
  return (
    <BrowserRouter>
      <Routes>

        <Route path="/" element={<Home />} />

        <Route path="/login" element={<Login />} />

        <Route
          path="/dashboard"
          element={
            <ProtectedRoute>
              <Dashboard />
            </ProtectedRoute>
          }
        />

      </Routes>
    </BrowserRouter>
  );
}

export default App;

Step 5 - Home Page

File: src/pages/Home.jsx 
function Home() {
  return <h1>Home Page</h1>;
}

export default Home;

How ProtectedRoute Works

User Visits:
/dashboard

       ↓

ProtectedRoute

       ↓

localStorage.getItem('token')

       ↓

Token Exists ?
       ↓

YES → Dashboard

NO → Login Page
The page component never renders unless authentication succeeds.

Why Use a Separate ProtectedRoute Component?

Without a separate component, every protected page would need to repeat authentication checks.

Bad: 

if (!token) {
  return <Navigate to="/login" />;
}

Repeated in every page.

Good: 

<ProtectedRoute>
  <Dashboard />
</ProtectedRoute>

Authentication logic is written once and reused everywhere.

Production Example Using Context

File: src/components/ProtectedRoute.jsx 
import { Navigate } from 'react-router-dom';
import { useAuth } from '../context/AuthContext';

function ProtectedRoute({ children }) {
  const { user } = useAuth();

  return user
    ? children
    : <Navigate to="/login" replace />;
}

export default ProtectedRoute;

Most real applications use Context, Redux, Zustand, or React Query instead of reading localStorage directly.

Protected Route vs Private Route

Term Meaning
Protected Route Authenticated users only.
Private Route Same thing. Different naming.
Protected Route and Private Route mean the same thing in React applications.

Interview Answer

A protected/private route is implemented by creating a wrapper component that checks whether the user is authenticated. If authentication succeeds, the requested page is rendered. Otherwise, the user is redirected to the login page using React Router's Navigate component.

One-Line Interview Answer

A protected route checks authentication before rendering a page and redirects unauthenticated users to the login page.
