263. What is a Protected Route and How Do You Implement One?

One-Line Definition

A Protected Route restricts access to specific pages and allows only authenticated users to view them.

Why Do We Need Protected Routes?

Some pages should only be accessible after login.

Examples:

  • User Dashboard
  • Profile Page
  • Order History
  • Admin Panel
  • Settings Page

If a user is not logged in, they should be redirected to the login page.

Protected Routes are a frontend convenience only. Real security must always be enforced by the backend API.

How It Works

User Opens Protected Page
↓
Check Authentication
↓
Logged In ?
↓
YES → Show Page
↓
NO → Redirect To Login

Complete Example With Separate Files

File: src/App.jsx 
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import Home from './pages/Home';
import Login from './pages/Login';
import Dashboard from './pages/Dashboard';
import ProtectedRoute from './components/ProtectedRoute';

function App() {
  return (
    <BrowserRouter>
      <Routes>

        <Route path="/" element={<Home />} />

        <Route path="/login" element={<Login />} />

        <Route
          path="/dashboard"
          element={
            <ProtectedRoute>
              <Dashboard />
            </ProtectedRoute>
          }
        />

      </Routes>
    </BrowserRouter>
  );
}

export default App;
File: src/components/ProtectedRoute.jsx 
import { Navigate } from 'react-router-dom';

function ProtectedRoute({ children }) {
  const isAuthenticated = localStorage.getItem('token');

  if (!isAuthenticated) {
    return <Navigate to="/login" replace />;
  }

  return children;
}

export default ProtectedRoute;
File: src/pages/Login.jsx 
function Login() {

  function handleLogin() {
    localStorage.setItem('token', 'abc123');
    window.location.href = '/dashboard';
  }

  return (
    <>
      <h1>Login Page</h1>

      <button onClick={handleLogin}>
        Login
      </button>
    </>
  );
}

export default Login;
File: src/pages/Dashboard.jsx 
function Dashboard() {
  return <h1>Dashboard Page</h1>;
}

export default Dashboard;
File: src/pages/Home.jsx 
function Home() {
  return <h1>Home Page</h1>;
}

export default Home;

What Does Navigate Do?

return <Navigate to="/login" replace />;

Navigate redirects the user to another route.

If the user is not authenticated: 

/dashboard
    ↓
ProtectedRoute
    ↓
Not Logged In
    ↓
/login
Navigate is the modern replacement for Redirect in React Router v6.

Real-World Example

Most e-commerce websites protect pages like: 

/profile
/orders
/wishlist
/checkout

Unauthenticated users are redirected to: 

/login

Better Production Example

Instead of localStorage directly, authentication state is usually stored in:

  • Context API
  • Redux
  • Zustand
  • React Query
const { user } = useAuth();

if (!user) {
  return <Navigate to="/login" replace />;
}

return children;

Protected Route vs Public Route

Public Route Protected Route
/home /dashboard
/about /profile
/contact /orders
Accessible by everyone. Requires login.

Interview Answer

A Protected Route is a route that only authenticated users can access. In React Router v6, it is commonly implemented using a wrapper component that checks authentication status and either renders the requested page or redirects the user to the login page using Navigate.

One-Line Interview Answer

A Protected Route restricts access to authenticated users and redirects unauthenticated users to the login page.
