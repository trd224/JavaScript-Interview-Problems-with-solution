266. What is Route-Based Code Splitting?

One-Line Definition

Route-based code splitting is a technique where each route gets its own JavaScript bundle and is downloaded only when the user visits that route.

Why Do We Need Route-Based Code Splitting?

Large React applications often contain many pages:

  • Dashboard
  • Products
  • Orders
  • Reports
  • Admin Panel
  • Settings

Without code splitting, all page code is included in a single large bundle.

Users may download code for pages they never visit.

Without Route-Based Code Splitting

Application Starts
↓
Download Dashboard
↓
Download Products
↓
Download Orders
↓
Download Reports
↓
Download Settings
↓
Large Bundle

With Route-Based Code Splitting

Application Starts
↓
Download Current Route Only
↓
User Visits Products
↓
Download Products Bundle
↓
User Visits Reports
↓
Download Reports Bundle
Only the routes users actually visit are downloaded.

Route-Based Code Splitting vs Lazy Loading

Lazy Loading Route-Based Code Splitting
General concept. Specific implementation.
Can lazy load any component. Lazy loads route components.
Works anywhere. Usually used with React Router.
Route-based code splitting is usually implemented using React.lazy + React Router.

Step 1 - Create Lazy Imports

File: src/App.jsx 
import { lazy } from 'react';

const Dashboard = lazy(() => import('./pages/Dashboard'));
const Products = lazy(() => import('./pages/Products'));
const Orders = lazy(() => import('./pages/Orders'));
const Reports = lazy(() => import('./pages/Reports'));

Each page becomes a separate bundle (chunk).

Step 2 - Configure Routes

File: src/App.jsx 
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import { lazy, Suspense } from 'react';

const Dashboard = lazy(() => import('./pages/Dashboard'));
const Products = lazy(() => import('./pages/Products'));
const Orders = lazy(() => import('./pages/Orders'));
const Reports = lazy(() => import('./pages/Reports'));

function App() {
  return (
    <BrowserRouter>

      <Suspense fallback={<h2>Loading...</h2>}>

        <Routes>

          <Route path="/" element={<Dashboard />} />

          <Route path="/products" element={<Products />} />

          <Route path="/orders" element={<Orders />} />

          <Route path="/reports" element={<Reports />} />

        </Routes>

      </Suspense>

    </BrowserRouter>
  );
}

export default App;

Real World Example

Admin panels often contain heavy pages:

  • Reports
  • Analytics
  • Charts
  • User Management
  • Audit Logs

Most users never visit all of these pages.

Route-based code splitting prevents downloading unnecessary code.

Advantages

  • Smaller initial bundle.
  • Faster page load.
  • Better performance.
  • Less bandwidth usage.
  • Improved user experience.

Interview Answer

Route-based code splitting is a performance optimization technique where each route is placed into a separate JavaScript bundle. The bundle is downloaded only when the user visits that route. It is commonly implemented using React.lazy, dynamic import(), Suspense, and React Router.

One-Line Interview Answer

Route-based code splitting loads route code only when the user navigates to that route instead of downloading all routes during the initial load.
