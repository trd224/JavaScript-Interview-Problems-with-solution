269. How do you pass state through navigation with useNavigate?

One-Line Definition

useNavigate allows us to navigate to another route and pass temporary state data along with the navigation.

Why Pass State Through Navigation?

Sometimes we need to send data to another page without:

  • Using URL parameters
  • Using query strings
  • Using Context
  • Using Redux

Examples:

  • Selected user details
  • Product information
  • Order summary
  • Success messages
Navigation state is temporary. It disappears after a page refresh.

How It Works

Current Page
↓
useNavigate()
↓
Pass State Object
↓
Navigate To Route
↓
useLocation()
↓
Read State

Project Structure

src/
│

└── pages/

Step 1 - Configure Routes

File: src/App.jsx 
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import Home from './pages/Home';
import Profile from './pages/Profile';

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/profile" element={<Profile />} />
      </Routes>
    </BrowserRouter>
  );
}

export default App;

Step 2 - Pass State Using useNavigate

File: src/pages/Home.jsx 
import { useNavigate } from 'react-router-dom';

function Home() {
  const navigate = useNavigate();

  function openProfile() {
    navigate('/profile', {
      state: {
        id: 101,
        name: 'Rizwan',
        role: 'Software Engineer'
      }
    });
  }

  return (
    <button onClick={openProfile}>
      Open Profile
    </button>
  );
}

export default Home;

State Being Passed

{
  id: 101,
  name: 'Rizwan',
  role: 'Software Engineer'
}

Step 3 - Read State Using useLocation

File: src/pages/Profile.jsx 
import { useLocation } from 'react-router-dom';

function Profile() {
  const location = useLocation();

  const { id, name, role } = location.state;

  return (
    <>
      <h2>ID: {id}</h2>
      <h2>Name: {name}</h2>
      <h2>Role: {role}</h2>
    </>
  );
}

export default Profile;

What Actually Happens?

Home Page
↓
navigate('/profile', { state: user })
↓
Profile Route
↓
location.state
↓
User Data Available

Real URL

Notice that the URL remains: 

/profile

No query string appears: 

/profile?id=101

No route parameter appears: 

/profile/101
The data is passed internally through React Router navigation state.

Safer Production Example

location.state may be undefined if the user directly opens the URL or refreshes the page.

File: src/pages/Profile.jsx 
import { useLocation } from 'react-router-dom';

function Profile() {
  const location = useLocation();

  const user = location.state;

  if (!user) {
    return <h2>No User Data Found</h2>;
  }

  return <h2>{user.name}</h2>;
}

export default Profile;
Always handle the case where location.state is undefined.

useNavigate State vs URL Params

Navigation State URL Params
Hidden from URL. Visible in URL.
Temporary. Persistent.
Lost on refresh. Available after refresh.
Good for temporary data. Good for shareable URLs.

Interview Answer

State can be passed through navigation by using the state option of useNavigate. The receiving component can access that state using useLocation(). This is useful for passing temporary data between routes without exposing it in the URL.

One-Line Interview Answer

Pass state using navigate('/route', { state: data }) and read it using useLocation().state.
