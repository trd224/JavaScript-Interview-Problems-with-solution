267. What is the Difference Between Client-Side and Server-Side Routing?

One-Line Definition

Client-side routing is handled by JavaScript in the browser, while server-side routing is handled by the web server.

Why Does Routing Exist?

Routing determines which page should be displayed when a user visits a URL.

Examples: 

/
/products
/orders
/contact

The question is:

Who decides what page to show?

Browser JavaScript?
OR
Server?

Client-Side Routing

In React applications, routing is usually handled by React Router inside the browser.

User Clicks Link
↓
React Router Handles URL
↓
Browser Does NOT Reload
↓
Component Changes

Example

File: src/App.jsx 
import { BrowserRouter, Routes, Route } from 'react-router-dom';

function App() {
  return (
    <Routes>
      <Route path="/" element={<Home />} />
      <Route path="/products" element={<Products />} />
      <Route path="/orders" element={<Orders />} />
    </Routes>
  );
}

React Router decides which component should render.

How Client-Side Routing Works

User Opens App
↓
index.html Downloads
↓
React App Loads
↓
User Navigates
↓
React Router Updates UI
↓
No Full Page Reload
Only the UI changes. The browser does not request a new HTML page from the server.

Server-Side Routing

In traditional websites, every route is handled by the server.

User Opens URL
↓
Request Sent To Server
↓
Server Chooses Page
↓
Returns New HTML
↓
Browser Reloads

Example

File: server.js (Express)
app.get('/', (req, res) => {
  res.send('Home Page');
});

app.get('/products', (req, res) => {
  res.send('Products Page');
});

app.get('/orders', (req, res) => {
  res.send('Orders Page');
});

The server decides which page to return.

React Router Example

File: Navigation.jsx 
import { Link } from 'react-router-dom';

function Navigation() {
  return (
    <>
      <Link to="/">Home</Link>
      <Link to="/products">Products</Link>
    </>
  );
}

Clicking Link updates the URL without refreshing the page.

Server Refresh Problem

You previously asked: 

Why does BrowserRouter need server support?

Because when the user refreshes: 

/products
Browser Refresh
↓
Request Goes To Server
↓
Server Must Return index.html
↓
React Router Takes Over
If the server is not configured correctly, refreshing a React route often results in a 404 error.

Advantages of Client-Side Routing

  • Fast navigation.
  • No full page reload.
  • Better user experience.
  • Less server work.
  • Feels like a desktop app.

Advantages of Server-Side Routing

  • Works without JavaScript.
  • Better initial page load.
  • SEO friendly by default.
  • Simple server control.

Interview Answer

Client-side routing is handled by JavaScript in the browser using libraries such as React Router. Navigation updates the UI without reloading the page. Server-side routing is handled by the web server, which returns a new HTML page for every route request. Client-side routing provides faster navigation, while server-side routing relies on the server to determine which page to render.

One-Line Interview Answer

Client-side routing is handled by the browser using JavaScript, while server-side routing is handled by the web server and usually causes a full page reload.
