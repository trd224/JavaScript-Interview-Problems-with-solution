261. What is useSearchParams?

One-Line Definition

useSearchParams is a React Router hook used to read and update query string parameters in the URL.

Why Do We Need useSearchParams?

Sometimes application state should be stored in the URL instead of React state.

Examples:

  • Pagination
  • Search keywords
  • Product filters
  • Sorting options
  • Category selection

This allows users to refresh the page, bookmark it, or share the URL without losing state.

Without useSearchParams, you would need to manually read and update the query string using the browser APIs.

Example URL

/products?page=2&category=mobile&sort=price

Complete Example With Separate Files

File: src/App.jsx 
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import Products from './pages/Products';

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/products" element={<Products />} />
      </Routes>
    </BrowserRouter>
  );
}

export default App;
File: src/pages/Products.jsx 
import { useSearchParams } from 'react-router-dom';

function Products() {
  const [searchParams, setSearchParams] = useSearchParams();

  const page = searchParams.get('page');
  const category = searchParams.get('category');
  const sort = searchParams.get('sort');

  function changeFilters() {
    setSearchParams({
      page: '3',
      category: 'laptop',
      sort: 'rating'
    });
  }

  return (
    <>
      <h2>Page: {page}</h2>
      <h2>Category: {category}</h2>
      <h2>Sort: {sort}</h2>

      <button onClick={changeFilters}>
        Change Filters
      </button>
    </>
  );
}

export default Products;

What Does useSearchParams Return?

const [searchParams, setSearchParams] = useSearchParams();
Variable Purpose
searchParams Read query string values.
setSearchParams Update query string values.

Reading Query Parameters

const page = searchParams.get('page');
const category = searchParams.get('category');
const sort = searchParams.get('sort');

For URL: 

/products?page=2&category=mobile&sort=price

Values: 

page === '2'
category === 'mobile'
sort === 'price'

Updating Query Parameters

setSearchParams({
  page: '3',
  category: 'laptop',
  sort: 'rating'
});

URL becomes: 

/products?page=3&category=laptop&sort=rating
React Router updates the URL without refreshing the page.

How It Works

URL
↓
/products?page=2&category=mobile
↓
useSearchParams()
↓
Read Query Values
↓
Display Data

Useful Methods

searchParams.get('page');

searchParams.has('page');

setSearchParams({ page: '2' });

setSearchParams({
  page: '2',
  sort: 'price'
});

Real World Example

E-commerce websites commonly store filters inside query parameters. 

/products?
category=laptop&
brand=dell&
price=50000&
sort=rating

When users share the URL, other users see the same filtered results.

useSearchParams vs useParams

useParams useSearchParams
/users/101 ?page=2
Reads route params. Reads query params.
Path based. URL query string based.

Interview Answer

useSearchParams is a React Router hook that allows us to read and update query string parameters in the URL. It is commonly used for pagination, filters, sorting, and search functionality while keeping application state synchronized with the URL.
