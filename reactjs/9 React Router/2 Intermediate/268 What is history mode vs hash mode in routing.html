268. What is History Mode vs Hash Mode in Routing?

One-Line Definition

History Mode uses normal URLs and requires server support, while Hash Mode uses a # in the URL and works without server configuration.

What Problem Are They Solving?

Single Page Applications (SPA) need routing without reloading the page.

Examples: 

/products
/orders
/profile

The browser URL changes, but React stays on the same page.

There are two common ways to achieve this:

  • History Mode
  • Hash Mode

History Mode

History Mode uses the browser History API.

URLs look like normal website URLs. 

https://example.com/products

https://example.com/orders

https://example.com/profile
Clean and professional URLs.

React Router Example

File: src/main.jsx 
import ReactDOM from 'react-dom/client';
import { BrowserRouter } from 'react-router-dom';
import App from './App';

ReactDOM.createRoot(
  document.getElementById('root')
).render(
  <BrowserRouter>
    <App />
  </BrowserRouter>
);

BrowserRouter uses History Mode.

How History Mode Works

User Opens /products
↓
Browser Sends Request
↓
Server Must Return index.html
↓
React Router Handles Route
History Mode requires server configuration.

Refresh Problem

Suppose the user refreshes: 

https://example.com/products
Refresh
↓
Request Goes To Server
↓
Server Looks For /products
↓
May Return 404

To fix this, the server must return: 

index.html

for every route.

Hash Mode

Hash Mode stores the route after the # symbol. 

https://example.com/#/products

https://example.com/#/orders

https://example.com/#/profile
No server configuration required.

React Router Example

File: src/main.jsx 
import ReactDOM from 'react-dom/client';
import { HashRouter } from 'react-router-dom';
import App from './App';

ReactDOM.createRoot(
  document.getElementById('root')
).render(
  <HashRouter>
    <App />
  </HashRouter>
);

HashRouter uses Hash Mode.

How Hash Mode Works

User Opens /#/products
↓
Server Ignores Everything After #
↓
Returns index.html
↓
React Router Reads Hash
↓
Products Page

The server never sees: 

#/products

because hashes are handled entirely by the browser.

Why Hash Mode Never Gets 404?

Server only receives: 

https://example.com/

Everything after # stays inside the browser.

https://example.com/#/products
↓
Server Receives
↓
https://example.com/
↓
Always Returns index.html

App Component

File: src/App.jsx 
import { Routes, Route } from 'react-router-dom';
import Home from './pages/Home';
import Products from './pages/Products';

function App() {
  return (
    <Routes>
      <Route path="/" element={<Home />} />
      <Route path="/products" element={<Products />} />
    </Routes>
  );
}

export default App;

Which One Should You Use?

Use BrowserRouter (History Mode) whenever possible.

Use HashRouter only when you cannot configure the server.

Interview Answer

History Mode uses the browser History API to create clean URLs such as /products. It requires server support because refreshing a page sends the route to the server. Hash Mode stores the route after the # symbol, such as #/products. Since the server never receives the hash portion, it works without server configuration but produces less clean URLs.

One-Line Interview Answer

History Mode uses clean URLs and requires server support, while Hash Mode uses # in the URL and works without server configuration.
