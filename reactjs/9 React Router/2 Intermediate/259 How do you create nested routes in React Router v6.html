259. How Do You Create Nested Routes in React Router v6?

One-Line Definition

Nested routes allow child pages to render inside a parent layout using the <Outlet /> component.

Why Do We Use Nested Routes?

Nested routes are useful when multiple pages share the same layout, such as a dashboard, admin panel, profile section, or settings area.

The parent route renders the common layout, and the child route renders inside <Outlet />.

Project Structure

src/
│
├── App.jsx
├── layouts/
│   └── DashboardLayout.jsx
│
└── pages/
    ├── DashboardHome.jsx
    ├── Profile.jsx
    └── Settings.jsx

Complete Example With Separate Files

File: src/App.jsx
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import DashboardLayout from './layouts/DashboardLayout';
import DashboardHome from './pages/DashboardHome';
import Profile from './pages/Profile';
import Settings from './pages/Settings';

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/dashboard" element={<DashboardLayout />}>
          <Route index element={<DashboardHome />} />
          <Route path="profile" element={<Profile />} />
          <Route path="settings" element={<Settings />} />
        </Route>
      </Routes>
    </BrowserRouter>
  );
}

export default App;
File: src/layouts/DashboardLayout.jsx
import { Link, Outlet } from 'react-router-dom';

function DashboardLayout() {
  return (
    <>
      <h1>Dashboard Layout</h1>

      <nav>
        <Link to="/dashboard">Home</Link>
        {' | '}
        <Link to="/dashboard/profile">Profile</Link>
        {' | '}
        <Link to="/dashboard/settings">Settings</Link>
      </nav>

      <hr />

      <Outlet />
    </>
  );
}

export default DashboardLayout;
File: src/pages/DashboardHome.jsx
function DashboardHome() {
  return <h2>Dashboard Home</h2>;
}

export default DashboardHome;
File: src/pages/Profile.jsx
function Profile() {
  return <h2>Profile Page</h2>;
}

export default Profile;
File: src/pages/Settings.jsx
function Settings() {
  return <h2>Settings Page</h2>;
}

export default Settings;

What is <Outlet />?

<Outlet /> is a placeholder where React Router renders the matched child route component.

Example: Outlet Placeholder
function DashboardLayout() {
  return (
    <>
      <h1>Dashboard</h1>
      <Outlet />
    </>
  );
}
Without <Outlet />, nested child routes will match, but their components will not be visible on the page.

↓
DashboardLayout
↓
Outlet
↓
DashboardHome
/dashboard/profile
↓
DashboardLayout
↓
Outlet
↓
Profile
/dashboard/settings
↓
DashboardLayout
↓
Outlet
↓
Settings

URL Mapping

URL Components Rendered
/dashboard DashboardLayout + DashboardHome
/dashboard/profile DashboardLayout + Profile
/dashboard/settings DashboardLayout + Settings

Important Point About Child Path

Inside a parent route, child route paths are usually written as relative paths.

Use path="profile", not path="/dashboard/profile", inside the child route. 
<Route path="/dashboard" element={<DashboardLayout />}>
  <Route path="profile" element={<Profile />} />
</Route>

What is Index Route?

An index route is the default child route. It renders when the parent path matches exactly. 

<Route path="/dashboard" element={<DashboardLayout />}>
  <Route index element={<DashboardHome />} />
</Route>
When the user visits /dashboard, React Router renders DashboardHome inside DashboardLayout.

Why Nested Routes Are Better

Without Nested Routes

<Route path="/dashboard" element={<Dashboard />} />
<Route path="/dashboard/profile" element={<Profile />} />
<Route path="/dashboard/settings" element={<Settings />} />

In this approach, the common dashboard layout may need to be repeated in multiple pages.

With Nested Routes

<Route path="/dashboard" element={<DashboardLayout />}>
  <Route index element={<DashboardHome />} />
  <Route path="profile" element={<Profile />} />
  <Route path="settings" element={<Settings />} />
</Route>
The layout is written only once, and only the child page changes inside <Outlet />.

Real-World Example

/admin
/admin/users
/admin/products
/admin/orders

All these pages can share one AdminLayout. Sidebar, header, and footer remain common. Only the content section changes.

AdminLayout
↓
Header + Sidebar
↓
Outlet
↓
Users / Products / Orders

Interview Answer

Nested routes in React Router v6 allow child routes to render inside a parent route. The parent component uses <Outlet /> as a placeholder where the matched child component is displayed. This is useful for shared layouts like dashboards, admin panels, profile pages, and settings sections.

One-Line Interview Answer

Nested routes render child pages inside a parent layout using React Router v6's <Outlet /> component.
