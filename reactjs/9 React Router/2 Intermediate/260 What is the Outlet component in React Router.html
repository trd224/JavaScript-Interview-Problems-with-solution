260. What is the Outlet Component in React Router?

One-Line Definition

The <Outlet /> component is a placeholder where React Router renders matching child routes inside a parent route.

Why Do We Need Outlet?

When we create nested routes, the parent component renders first. But React Router needs a place inside the parent component where the child page should appear.

That place is called <Outlet />. Without Outlet, nested child routes will match the URL but will not display on the screen.

Complete Example With Separate Files

File: src/App.jsx
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import DashboardLayout from './layouts/DashboardLayout';
import DashboardHome from './pages/DashboardHome';
import Profile from './pages/Profile';
import Settings from './pages/Settings';

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/dashboard" element={<DashboardLayout />}>
          <Route index element={<DashboardHome />} />
          <Route path="profile" element={<Profile />} />
          <Route path="settings" element={<Settings />} />
        </Route>
      </Routes>
    </BrowserRouter>
  );
}

export default App;
File: src/layouts/DashboardLayout.jsx
import { Link, Outlet } from 'react-router-dom';

function DashboardLayout() {
  return (
    <>
      <h1>Dashboard Layout</h1>

      <nav>
        <Link to="/dashboard">Home</Link>
        {' | '}
        <Link to="/dashboard/profile">Profile</Link>
        {' | '}
        <Link to="/dashboard/settings">Settings</Link>
      </nav>

      <hr />

      <Outlet />
    </>
  );
}

export default DashboardLayout;
File: src/pages/DashboardHome.jsx
function DashboardHome() {
  return <h2>Dashboard Home Page</h2>;
}

export default DashboardHome;
File: src/pages/Profile.jsx
function Profile() {
  return <h2>Profile Page</h2>;
}

export default Profile;
File: src/pages/Settings.jsx
function Settings() {
  return <h2>Settings Page</h2>;
}

export default Settings;

How Outlet Works

/dashboard/profile
↓
DashboardLayout renders
↓
React Router finds <Outlet />
↓
Profile component renders inside Outlet

URL Mapping

URL Parent Component Child Component Rendered Inside Outlet
/dashboard DashboardLayout DashboardHome
/dashboard/profile DashboardLayout Profile
/dashboard/settings DashboardLayout Settings

Important Point

The parent route component must contain <Outlet />. Otherwise, child routes will not be visible.

Wrong Example - No Outlet

function DashboardLayout() {
  return (
    <>
      <h1>Dashboard Layout</h1>
    </>
  );
}

In this case, /dashboard/profile may match the route, but the Profile component will not appear because there is no Outlet.

Correct Example - With Outlet

import { Outlet } from 'react-router-dom';

function DashboardLayout() {
  return (
    <>
      <h1>Dashboard Layout</h1>
      <Outlet />
    </>
  );
}
Now child pages can render inside the parent layout.

Real-World Use Case

Outlet is commonly used when many pages share the same layout. 

Admin Layout
│
├── Dashboard Page
├── Users Page
├── Products Page
└── Orders Page

The sidebar, navbar, and layout remain the same. Only the child page inside Outlet changes.

Easy Way To Remember

Parent Route
↓
Layout Component
↓
Outlet
↓
Child Route Component

Interview Answer

The Outlet component in React Router is used inside a parent route component to render matching nested child routes. It works like a placeholder. When the URL matches a child route, React Router displays that child component inside the parent component's Outlet.

One-Line Interview Answer

Outlet is a placeholder used to render nested child routes inside a parent route component.
