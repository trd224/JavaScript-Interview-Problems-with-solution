262. What is useLocation?

One-Line Definition

useLocation is a React Router hook that gives access to the current URL information.

Why Do We Need useLocation?

Sometimes a component needs to know:

  • Current pathname
  • Current query string
  • Current hash value
  • Navigation state data

useLocation provides all of this information.

Without useLocation, you would need to manually read information from the browser URL.

Example URL

http://localhost:3000/products?page=2#reviews

Project Structure

src/
│
├── App.jsx
│
└── pages/
    └── Products.jsx

Complete Example With Separate Files

File: src/App.jsx 
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import Products from './pages/Products';

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/products" element={<Products />} />
      </Routes>
    </BrowserRouter>
  );
}

export default App;
File: src/pages/Products.jsx 
import { useLocation } from 'react-router-dom';

function Products() {
  const location = useLocation();

  return (
    <>
      <h2>Pathname: {location.pathname}</h2>
      <h2>Search: {location.search}</h2>
      <h2>Hash: {location.hash}</h2>
    </>
  );
}

export default Products;

What Does useLocation Return?

const location = useLocation();

Location object: 

{
  pathname: '/products',
  search: '?page=2',
  hash: '#reviews',
  state: null,
  key: 'abc123'
}

Important Properties

Property Description
pathname Current route path.
search Query string.
hash URL fragment.
state Data passed during navigation.
key Unique route identifier.

Reading Current Path

const location = useLocation();

console.log(location.pathname);

Output: 

/products

Reading Query String

const location = useLocation();

console.log(location.search);

Output: 

?page=2

Reading Hash

const location = useLocation();

console.log(location.hash);

Output: 

#reviews

Using Navigation State

File: src/pages/Home.jsx 
import { useNavigate } from 'react-router-dom';

function Home() {
  const navigate = useNavigate();

  return (
    <button
      onClick={() =>
        navigate('/profile', {
          state: {
            username: 'Rizwan'
          }
        })
      }
    >
      Open Profile
    </button>
  );
}

export default Home;
File: src/pages/Profile.jsx 
import { useLocation } from 'react-router-dom';

function Profile() {
  const location = useLocation();

  return (
    <h2>
      {location.state.username}
    </h2>
  );
}

export default Profile;

Common Use Cases

  • Highlight active menu links
  • Analytics tracking
  • Breadcrumb generation
  • Route-based UI changes
  • Reading navigation state

Real World Example

Most dashboards highlight the active menu item based on the current pathname. 

const location = useLocation();

const isUsersPage =
  location.pathname === '/users';
This is one of the most common real-world uses of useLocation.

useLocation vs useSearchParams

useLocation useSearchParams
Returns complete URL information. Works specifically with query params.
pathname, search, hash, state. Read/update query strings.
Read-only URL details. Can update query params.

Interview Answer

useLocation is a React Router hook that provides information about the current URL. It returns a location object containing pathname, search query, hash, and navigation state, which is useful for route-based UI logic and reading URL details.

One-Line Interview Answer

useLocation is used to access information about the current URL in React Router.
