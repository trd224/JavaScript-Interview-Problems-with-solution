265. What is Lazy Loading a Route?

One-Line Definition

Lazy loading a route means loading a page's JavaScript code only when the user visits that route instead of downloading it during the initial application load.

Why Do We Need Lazy Loading?

By default, React bundles all pages together.

Suppose your application has:

  • Dashboard
  • Products
  • Reports
  • Settings
  • Admin Panel

Without lazy loading, all page code is downloaded immediately.

Users may never visit some pages, but they still download their code.

Step 1 - Import React.lazy

File: src/App.jsx 
import { lazy } from 'react';

const Dashboard = lazy(() => import('./pages/Dashboard'));
const Products = lazy(() => import('./pages/Products'));
const Reports = lazy(() => import('./pages/Reports'));

React.lazy loads components only when needed.

Step 2 - Wrap Routes With Suspense

File: src/App.jsx 
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import { lazy, Suspense } from 'react';

const Dashboard = lazy(() => import('./pages/Dashboard'));
const Products = lazy(() => import('./pages/Products'));
const Reports = lazy(() => import('./pages/Reports'));

function App() {
  return (
    <BrowserRouter>

      <Suspense fallback={<h2>Loading...</h2>}>

        <Routes>

          <Route path="/" element={<Dashboard />} />

          <Route path="/products" element={<Products />} />

          <Route path="/reports" element={<Reports />} />

        </Routes>

      </Suspense>

    </BrowserRouter>
  );
}

export default App;

Step 3 - Create Route Components

File: src/pages/Dashboard.jsx 
function Dashboard() {
  return <h1>Dashboard Page</h1>;
}

export default Dashboard;
File: src/pages/Products.jsx 
function Products() {
  return <h1>Products Page</h1>;
}

export default Products;
File: src/pages/Reports.jsx 
function Reports() {
  return <h1>Reports Page</h1>;
}

export default Reports;

What Does Suspense Do?

<Suspense fallback={<h2>Loading...</h2>}>
  ...
</Suspense>

Suspense shows fallback content while React downloads the lazy-loaded page.

Real World Example

Large pages are ideal candidates for lazy loading:

  • Admin Panel
  • Reports
  • Analytics
  • Charts
  • Settings

Users may never open these pages during a session.

Advantages

  • Smaller initial bundle size
  • Faster first load
  • Better performance
  • Less JavaScript downloaded
  • Improved user experience

Interview Answer

Lazy loading a route means loading a page component only when the user navigates to that route. React Router commonly uses React.lazy and Suspense to split route code into separate chunks, reducing the initial bundle size and improving application performance.

One-Line Interview Answer

Lazy loading a route loads the route's code only when the user visits that route instead of during the initial application load.
