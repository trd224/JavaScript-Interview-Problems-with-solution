271. How do you handle scroll restoration in React Router?

One-Line Answer

React Router v6.4+ provides the <ScrollRestoration /> component, which automatically remembers and restores scroll positions when users navigate back and forward.

The Problem

Imagine an e-commerce website with hundreds of products.

Open Products Page
↓ Scroll To Product #80
↓ Click Product Details
↓ Press Browser Back

What should happen?

User should return to Product #80.
Without scroll restoration, the user returns to the top of the page and must scroll again.

Without Scroll Restoration

Products Page
↓ Scroll To Product #80
↓ Open Product Detail
↓ Back Button
↓ Returned To Top ❌

With Scroll Restoration

Products Page
↓ Scroll To Product #80
↓ Open Product Detail
↓ Back Button
↓ Return To Product #80 ✅

Using ScrollRestoration

Place it once inside your root layout route.

AppLayout.jsx
import {Outlet,ScrollRestoration} from 'react-router-dom';

export default function AppLayout(){

  return (
    <>
      <Header />

      <Outlet />

      <Footer />

      <ScrollRestoration />
    </>
  );

}

Router Setup

router.jsx
import {createBrowserRouter} from 'react-router-dom';
import AppLayout from './AppLayout';

export const router=createBrowserRouter([
  {
    path:'/',
    element:<AppLayout />,
    children:[
      {index:true,element:<Home />},
      {path:'products',element:<Products />},
      {path:'products/:id',element:<ProductDetail />}
    ]
  }
]);

How It Works Internally

User Scrolls
↓ React Router Saves Position
↓ Navigate To New Route
↓ Position Stored
↓ User Clicks Back
↓ Position Restored

Understanding AppLayout, Outlet and ScrollRestoration

AppLayout
├─ Header
├─ Outlet
└─ ScrollRestoration

If the user visits: 

https://site.com/products

Then Outlet renders: 

<Products />

If the user visits: 

https://site.com/products/101

Then Outlet renders: 

<ProductDetail />

Meanwhile ScrollRestoration keeps tracking scroll positions for all routes.

Manual Scroll-To-Top Alternative

Sometimes developers do not want restoration. They always want a new page to start at the top.

ScrollToTop.jsx
import {useEffect} from 'react';
import {useLocation} from 'react-router-dom';

export default function ScrollToTop(){

  const {pathname}=useLocation();

  useEffect(()=>{
    window.scrollTo(0,0);
  },[pathname]);

  return null;

}

Behavior

Navigate To Route
↓ Always Scroll To Top
↓ Old Position Lost

ScrollRestoration vs ScrollToTop

Feature ScrollRestoration ScrollToTop
Back Button Support ✅ Yes ❌ No
Restore Old Position ✅ Yes ❌ No
Always Go Top ❌ No ✅ Yes
Best For E-commerce, blogs Traditional page navigation

Real-World Example

Amazon Product List
↓ Scroll To Item #200
↓ Open Product
↓ Read Details
↓ Press Back
↓ Return To Item #200
This is exactly the user experience that ScrollRestoration provides.

Interview Answer

React Router v6.4+ handles scroll restoration through the <ScrollRestoration /> component. It is typically placed inside the root layout route and automatically saves and restores scroll positions during browser back and forward navigation. For applications that always need to start at the top after navigation, developers can instead use a custom component with useLocation() and window.scrollTo(0,0).
