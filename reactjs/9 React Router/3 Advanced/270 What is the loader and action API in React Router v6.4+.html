270. What is the loader and action API in React Router v6.4+?

One-Line Answer

React Router v6.4+ introduced loaders for reading data before rendering a route and actions for handling form submissions and data mutations.

Problem Before v6.4

Traditionally we fetched data inside useEffect().

Old Way
function Users(){

  const [users,setUsers]=useState([]);

  useEffect(()=>{
    fetch('/api/users')
      .then(res=>res.json())
      .then(data=>setUsers(data));
  },[]);

  return <UsersList users={users}/>;

}
Route Loads
↓ Component Renders
↓ useEffect Runs
↓ Fetch Starts
↓ Data Arrives
↓ UI Updates
The component renders before data is available.

Loader API

A loader runs before the route component renders.

users-loader.js
export async function usersLoader(){

  const res=await fetch('/api/users');

  return res.json();

}
router.js
createBrowserRouter([
  {
    path:'/users',
    element:<Users/>,
    loader:usersLoader
  }
]);
Users.jsx
import {useLoaderData} from 'react-router-dom';

export default function Users(){

  const users=useLoaderData();

  return (
    <ul>
      {users.map(user=>
        <li key={user.id}>{user.name}</li>
      )}
    </ul>
  );

}

Loader Flow

Navigate To Route
↓ Loader Runs
↓ Fetch Data
↓ Data Ready
↓ Render Component
Very similar to React Server Components where data is fetched before rendering.

Action API

Actions handle form submissions, POST requests, updates and deletes.

users-action.js
export async function createUserAction({request}){

  const formData=await request.formData();

  await fetch('/api/users',{
    method:'POST',
    body:formData
  });

  return null;

}
router.js
createBrowserRouter([
  {
    path:'/users',
    element:<Users/>,
    loader:usersLoader,
    action:createUserAction
  }
]);
Users.jsx
import {Form} from 'react-router-dom';

export default function Users(){

  return (
    <Form method="post">
      <input name="name"/>
      <button>Save</button>
    </Form>
  );

}

Action Flow

User Submits Form
↓ Action Runs
↓ POST/PUT/DELETE Request
↓ Data Updated
↓ Route Revalidates
↓ Loader Runs Again
↓ Fresh Data Appears

Real Example

Open Users Page
↓ Loader Fetches Users
↓ Users Displayed

User Adds New User
↓ Action Executes
↓ Database Updated
↓ Loader Re-runs
↓ Updated User List Shown

Why Loader + Action Are Powerful

Feature Benefit
Loader Fetch before render
Action Handle mutations
Automatic Revalidation Fresh data after actions
No useEffect Fetching Cleaner code
Route-Based Data Data belongs to route

Easy Way To Remember

Loader
↓ GET Data
↓ Read Data

Action
↓ POST / PUT / DELETE
↓ Change Data

Comparison With React Server Components

React Router Loader React Server Component
Runs before route render Runs on server before render

useLoaderData() await fetch()

Interview Answer

In React Router v6.4+, a loader is a route-level function used to fetch data before a route component renders. An action is a route-level function used to handle form submissions and data mutations such as POST, PUT, PATCH, and DELETE requests. After an action completes, React Router can automatically re-run loaders to keep the UI synchronized with the latest data.
