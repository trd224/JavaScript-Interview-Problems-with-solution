New Way of Router in React Router v6.4+

One-Line Answer

React Router v6.4+ uses createBrowserRouter() and <RouterProvider /> instead of wrapping the app with <BrowserRouter>. This new way supports loader, action, useLoaderData, and ScrollRestoration.

Old Way

main.jsx
import React from 'react';
import ReactDOM from 'react-dom/client';
import {BrowserRouter} from 'react-router-dom';
import App from './App';

ReactDOM.createRoot(document.getElementById('root')).render(
  <BrowserRouter>
    <App />
  </BrowserRouter>
);
App.jsx
import {Routes,Route} from 'react-router-dom';

export default function App(){

  return (
    <Routes>
      <Route path="/" element={<Home />} />
      <Route path="/products" element={<Products />} />
      <Route path="/products/:id" element={<ProductDetail />} />
    </Routes>
  );

}
This old way is fine for normal routing, but it does not support route-level loader, action, and ScrollRestoration.

New Way

main.jsx
import React from 'react';
import ReactDOM from 'react-dom/client';
import {RouterProvider} from 'react-router-dom';
import {router} from './router';

ReactDOM.createRoot(document.getElementById('root')).render(
  <RouterProvider router={router} />
);
router.jsx
import {createBrowserRouter} from 'react-router-dom';
import AppLayout from './AppLayout';
import Home from './Home';
import Products from './Products';
import ProductDetail from './ProductDetail';

export const router=createBrowserRouter([
  {
    path:'/',
    element:<AppLayout />,
    children:[
      {index:true,element:<Home />},
      {path:'products',element:<Products />},
      {path:'products/:id',element:<ProductDetail />}
    ]
  }
]);

Where Is App.jsx In The New Router?

In the Data Router API, App.jsx is usually not rendered directly inside main.jsx.

Instead, RouterProvider renders the route tree defined in createBrowserRouter().

↓ App
↓ Routes
↓ Pages

↓ RouterProvider
↓ router
↓ Route Components
With the Data Router API, App.jsx is commonly used as a Layout Route instead of being rendered directly from main.jsx.
App.jsx
import { Outlet } from 'react-router-dom';

function App() {
  return (
    <>
      <h1>My App</h1>
      <Outlet />
    </>
  );
}

export default App;
router.jsx
import { createBrowserRouter } from 'react-router-dom';
import App from './App';
import Home from './pages/Home';
import Login from './pages/Login';

export const router = createBrowserRouter([
  {
    path: '/',
    element: <App />,
    children: [
      { index: true, element: <Home /> },
      { path: 'login', element: <Login /> }
    ]
  }
]);

How Rendering Works

root
↓ RouterProvider
↓ App (Layout Route)
↓ Outlet
↓ Home / Login / Products
In React Router v6.4+, App.jsx is typically a layout component rendered by the router configuration, while RouterProvider replaces BrowserRouter as the application's router entry point.

AppLayout Means Main Layout

AppLayout.jsx
import {Outlet,ScrollRestoration} from 'react-router-dom';

export default function AppLayout(){

  return (
    <>
      <Header />
      <Outlet />
      <Footer />
      <ScrollRestoration />
    </>
  );

}
<Outlet /> is the place where child pages render.

Why New Router Was Created?

Because React Router v6.4+ can load data before rendering the page.

router.jsx with loader
import {createBrowserRouter} from 'react-router-dom';

async function productsLoader(){

  const res=await fetch('/api/products');

  return res.json();

}

export const router=createBrowserRouter([
  {
    path:'/',
    element:<AppLayout />,
    children:[
      {path:'products',element:<Products />,loader:productsLoader}
    ]
  }
]);
Products.jsx
import {useLoaderData} from 'react-router-dom';

export default function Products(){

  const products=useLoaderData();

  return (
    <ul>
      {products.map(product=>
        <li key={product.id}>{product.name}</li>
      )}
    </ul>
  );

}

New Router Flow

User Visits /products
↓ React Router Finds Route
↓ Runs productsLoader()
↓ Data Comes
↓ Products Component Renders
↓ useLoaderData() Gives Data

Old vs New

Old Router New Router
<BrowserRouter> createBrowserRouter()
<Routes> and <Route> Route object array
Data usually fetched in useEffect Data can be fetched in loader
No route action API Supports action API
Basic scroll handling Supports ScrollRestoration

Easy Way To Remember

Old Way
BrowserRouter + Routes + Route

New Way
createBrowserRouter + RouterProvider + Route Objects

Interview Answer

The new React Router v6.4+ routing style uses createBrowserRouter() to define routes as objects and <RouterProvider /> to provide the router to the app. This new data router system supports route-level features like loaders for data fetching, actions for form submissions, useLoaderData for reading loader data, and ScrollRestoration for restoring scroll positions.
