422. What are the document metadata improvements in React 19?

One-Line Answer

React 19 allows you to render document metadata such as <title>, <meta>, and <link> directly inside React components. React automatically moves them to the document <head>.

The Problem Before React 19

Before React 19, metadata was usually managed using:

  • document.title
  • useEffect()
  • react-helmet
  • react-helmet-async
ProductPage.jsx
import {useEffect} from 'react';

function ProductPage(){

  useEffect(()=>{
    document.title='iPhone 17';
  },[]);

  return <h1>iPhone 17</h1>;

}
Page Render
↓ useEffect Runs
↓ document.title Updated
Managing metadata required extra code or third-party libraries.

React 19 Solution

Now metadata can be written directly inside JSX.

ProductPage.jsx
function ProductPage(){

  return (
    <>
      <title>iPhone 17</title>

      <meta
        name="description"
        content="Buy iPhone 17 Online"
      />

      <h1>iPhone 17</h1>
    </>
  );

}
React automatically detects these tags and places them inside the document head.

What Happens Internally?

Component Renders
↓ React Finds title/meta/link
↓ React Hoists Metadata
↓ Document Head Updated

Real-World Blog Example

BlogPost.jsx
function BlogPost({post}){

  return (
    <>

      <title>
        {post.title}
      </title>

      <meta
        name="description"
        content={post.description}
      />

      <meta
        name="keywords"
        content={post.keywords}
      />

      <h1>
        {post.title}
      </h1>

    </>
  );

}
Blog Data Loaded
↓ Metadata Generated
↓ Head Updated
↓ SEO Improved

Supported Metadata Tags

Tag Purpose
<title> Browser Tab Title
<meta> Description, Keywords, Author
<link> Canonical URL, Favicon, Stylesheets

Using Title, Meta and Link Together

CoursePage.jsx
function CoursePage(){

  return (
    <>

      <title>
        React Course
      </title>

      <meta
        name="description"
        content="Learn React From Scratch"
      />

      <link
        rel="canonical"
        href="https://example.com/react-course"
      />

      <h1>
        React Course
      </h1>

    </>
  );

}

Relation To Server Components

You recently learned React Server Components.

Server Component Example
export default async function ProductPage(){

  const product=await fetch(
    '/api/product/1'
  ).then(res=>res.json());

  return (
    <>
      <title>
        {product.name}
      </title>

      <meta
        name="description"
        content={product.description}
      />

      <h1>
        {product.name}

    </>
  );

}
Metadata can now be generated directly from server-fetched data.

Interview Answer

React 19 introduces native document metadata support, allowing developers to render title, meta, and link tags directly inside React components. React automatically hoists these tags into the document head, reducing the need for document.title, useEffect, or libraries such as react-helmet. This improves SEO, simplifies metadata management, and works well with Server Components and SSR.
