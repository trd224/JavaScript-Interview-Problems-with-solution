414. What are the major breaking changes and new features in React 19?

One-Line Answer

React 19 adds new features like Actions, Form Actions, useActionState, useFormStatus, useOptimistic, use(), Server Components support, ref as prop, and metadata support. It also removes or changes some old APIs like string refs, module pattern factories, ReactDOM.render, hydrate, unmountComponentAtNode, findDOMNode, and old testing APIs.

1. Actions

Actions are async functions used for updates like save, delete, login, signup, add comment, and place order.

Action Example
async function saveUser(formData){
  const name=formData.get('name');
  await fetch('/api/users',{method:'POST',body:JSON.stringify({name})});
}

export default function UserForm(){
  return (
    <form action={saveUser}>
      <input name="name"/>
      <button>Save</button>
    </form>
  );
}
User Clicks Save
↓ Action Runs
↓ Server Update
↓ UI Updates

2. useActionState()

useActionState Example
import {useActionState} from 'react';

async function createUser(prevState,formData){
  const name=formData.get('name');
  if(!name) return {message:'Name is required'};
  return {message:'User created'};
}

export default function Signup(){
  const [state,formAction]=useActionState(createUser,{message:''});

  return (
    <form action={formAction}>
      <input name="name"/>
      <button>Create</button>
      <p>{state.message}</p>
    </form>
  );
}

3. useFormStatus()

Correct Child Component Example
import {useFormStatus} from 'react-dom';

function SubmitButton(){
  const {pending}=useFormStatus();

  return (
    <button disabled={pending}>
      {pending?'Saving...':'Save'}
    </button>
  );
}

function UserForm(){
  return (
    <form action={saveUser}>
      <input name="name"/>
      <SubmitButton />
    </form>
  );
}
Do not use useFormStatus inside the same component that creates the form. Use it inside a child component inside the form.
Wrong Same Component Example
import {useFormStatus} from 'react-dom';

function UserForm(){
  const {pending}=useFormStatus();

  return (
    <form action={saveUser}>
      <input name="name"/>
      <button disabled={pending}>Save</button>
    </form>
  );
}

4. useOptimistic()

Like Button Example
import {useState,useOptimistic} from 'react';

export default function LikeButton(){
  const [likes,setLikes]=useState(10);
  const [optimisticLikes,addLike]=useOptimistic(likes,(current,value)=>current+value);

  async function handleLike(){
    addLike(1);
    const updatedLikes=await saveLike();
    setLikes(updatedLikes);
  }

  return <button onClick={handleLike}>❤️ {optimisticLikes}</button>;
}

5. use() API

use() With Suspense
import {use,Suspense} from 'react';

const usersPromise=fetch('/api/users').then(res=>res.json());

function Users(){
  const users=use(usersPromise);
  return <ul>{users.map(user=><li key={user.id}>{user.name}</li>)}</ul>;
}

export default function Page(){
  return (
    <Suspense fallback={<h2>Loading...</h2>}>
      <Users />
    </Suspense>
  );
}

6. Server Components

Server Component Example
export default async function Users(){
  const users=await fetch('https://api.example.com/users').then(res=>res.json());

  return (
    <ul>
      {users.map(user=><li key={user.id}>{user.name}</li>)}
    </ul>
  );
}
If there is no 'use client', it is inside a framework that supports Server Components, and it fetches data directly with async/await, it is usually a Server Component.

7. ref as Prop

Before React 19
import {forwardRef} from 'react';

const Input=forwardRef(function Input(props,ref){
  return <input ref={ref} {...props}/>;
});
React 19
function Input({ref,...props}){
  return <input ref={ref} {...props}/>;
}

Before

Major Breaking Changes In React 19

Breaking ChangeWhat To Use Instead
New JSX Transform requiredUpdate Babel/Vite/Next config
ReactDOM.render removedcreateRoot()
ReactDOM.hydrate removedhydrateRoot()
unmountComponentAtNode removedroot.unmount()
findDOMNode removedUse refs
String refs removedUse createRef/useRef
Module pattern factories removedUse normal function/class components
react-test-renderer deprecatedUse Testing Library
propTypes/defaultProps for functions removedUse TypeScript/default parameters

Breaking Change 1: ReactDOM.render Removed

Before
import ReactDOM from 'react-dom';

ReactDOM.render(<App />,document.getElementById('root'));
React 19
import {createRoot} from 'react-dom/client';

createRoot(document.getElementById('root')).render(<App />);

Breaking Change 2: hydrate Removed

Before
import ReactDOM from 'react-dom';

ReactDOM.hydrate(<App />,document.getElementById('root'));
React 19
import {hydrateRoot} from 'react-dom/client';

hydrateRoot(document.getElementById('root'),<App />);

Breaking Change 3: unmountComponentAtNode Removed

Before
ReactDOM.unmountComponentAtNode(document.getElementById('root'));
React 19
const root=createRoot(document.getElementById('root'));

root.render(<App />);

root.unmount();

Breaking Change 4: findDOMNode Removed

Before
import {findDOMNode} from 'react-dom';

const node=findDOMNode(component);
React 19
import {useRef} from 'react';

function App(){
  const inputRef=useRef(null);

  return <input ref={inputRef}/>;
}

Breaking Change 5: String Refs Removed

Before
class App extends React.Component{
  render(){
    return <input ref="nameInput"/>;
  }
}
React 19
function App(){
  const nameInput=useRef(null);

  return <input ref={nameInput}/>;
}

Breaking Change 6: Function defaultProps Removed

Before
function Button(props){
  return <button>{props.label}</button>;
}

Button.defaultProps={
  label:'Save'
};
React 19
function Button({label='Save'}){
  return <button>{label}</button>;
}

Interview Answer

React 19 introduces major new features such as Actions, Form Actions, useActionState, useFormStatus, useOptimistic, the use() API, Server Components support, ref as prop, simpler Context providers, metadata support, and better hydration errors. Major breaking changes include requiring the new JSX transform, removing ReactDOM.render, hydrate, unmountComponentAtNode, findDOMNode, string refs, module pattern factories, and old testing APIs. Most changes modernize React and reduce boilerplate.
