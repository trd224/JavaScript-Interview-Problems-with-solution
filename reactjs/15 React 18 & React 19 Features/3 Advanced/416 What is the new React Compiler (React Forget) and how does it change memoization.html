416. What is the new React Compiler (React Forget) and how does it change memoization?

One-Line Answer

React Compiler, previously called React Forget, is an optional build-time optimization tool that analyzes React components and automatically applies many memoization optimizations, reducing the need for manual useMemo(), useCallback(), and React.memo().

First Clear The Biggest Confusion

React Compiler is not the same thing as Babel/SWC, and it does not replace them.
Babel / SWC
↓ Converts JSX To JavaScript

React Compiler
↓ Optimizes React Rendering

Normal React Build Without React Compiler

In a normal React project, you write JSX, and your build tool converts it into browser-understandable JavaScript.

React Code
↓ npm run build
↓ Vite / Next.js / Webpack
↓ Babel / SWC
↓ JavaScript Bundle
↓ Browser

Example

Your JSX Code
function App(){
  return <h1>Hello</h1>;
}

The browser does not understand JSX directly, so Babel/SWC converts it.

Converted JavaScript Idea
function App(){
  return React.createElement('h1',null,'Hello');
}
Babel/SWC is required because JSX cannot run directly in the browser.

Build With React Compiler

When React Compiler is configured, you still run the same build command.

React Code
↓ npm run build
↓ React Compiler
↓ Babel / SWC
↓ Optimized JavaScript Bundle
↓ Browser
You do not run another command manually. After setup, npm run build does everything.

Why Do We Need React Compiler?

You already learned that React can re-render components unnecessarily when new object or function references are created.

Without Optimization
function Parent(){
  const settings={theme:'dark'};

  function handleClick(){
    console.log('Clicked');
  }

  return <Child settings={settings} onClick={handleClick}/>;
}

Every render creates:

  • New settings object
  • New handleClick function
  • Possible unnecessary child re-render
Parent Re-renders
↓ New Object Created
↓ New Function Created
↓ Child Receives New Props
↓ Child May Re-render

Old Manual Optimization

Before React Compiler, developers often used useMemo, useCallback, and React.memo.

Parent.jsx
import {useMemo,useCallback} from 'react';
import Child from './Child';

function Parent(){
  const settings=useMemo(()=>({theme:'dark'}),[]);

  const handleClick=useCallback(()=>{
    console.log('Clicked');
  },[]);

  return <Child settings={settings} onClick={handleClick}/>;
}
Child.jsx
import React from 'react';

const Child=React.memo(function Child({settings,onClick}){
  return <button onClick={onClick}>{settings.theme}</button>;
});

export default Child;
This works, but developers often forget it, overuse it, or use wrong dependencies.

With React Compiler

The goal is to let you write normal React code.

Parent.jsx
function Parent(){
  const settings={theme:'dark'};

  function handleClick(){
    console.log('Clicked');
  }

  return <Child settings={settings} onClick={handleClick}/>;
}
Developer Writes Normal Code
↓ React Compiler Analyzes Code
↓ Finds Stable Values
↓ Adds Memoization Internally
↓ Fewer Unnecessary Re-renders
React Compiler tries to give optimized behavior without forcing you to manually write memoization everywhere.

Does React Compiler Replace Babel?

No. React Compiler does not replace Babel or SWC.
ToolJob
Babel / SWCConvert JSX and modern JavaScript syntax
React CompilerAnalyze React code and optimize rendering
Wrong Thinking
↓ React Compiler OR Babel

Correct Thinking
↓ React Compiler + Babel/SWC

Why Is It Not Added By Default With Babel?

Because Babel is required for JSX, but React Compiler is an optional optimization.

ReasonExplanation
Babel is requiredBrowser cannot understand JSX without compilation
React Compiler is optionalYour app works without it
It changes optimization behaviorReact team wants gradual adoption
Existing apps need testingTurning it on automatically could cause unexpected issues
Build tools differVite, Next.js, Webpack, CRA all configure builds differently
Simple reason: Babel is necessary to run React JSX. React Compiler is not necessary to run React; it improves performance.

Is React Compiler A React 19 Feature?

Not exactly. React Compiler is not a core React 19 runtime feature. It is a separate optional build-time compiler that works with React 19.
React 19 FeatureReact Compiler
Built into React APIsSeparate build tool
Example: Actions, useActionState, useOptimisticExample: babel-plugin-react-compiler
Runtime/API featureBuild-time optimization

It is often discussed with React 19 because it became popular around the React 19 era.

Does It Mean I Do Not Need useMemo/useCallback?

In many cases, React Compiler reduces the need for useMemo and useCallback.

Before

const filteredProducts=useMemo(()=>products.filter(product=>product.price>1000),[products]);

With Compiler

const filteredProducts=products.filter(product=>product.price>1000);

Before

const handleClick=useCallback(()=>{
  console.log('Clicked');
},[]);

With Compiler

function handleClick(){
  console.log('Clicked');
}
But do not think you can ignore useMemo, useCallback, or React.memo. Many projects do not use the compiler, and the compiler cannot optimize every possible pattern.

How To Configure React Compiler

Install

Terminal
npm install -D babel-plugin-react-compiler

Vite Example

vite.config.js
import {defineConfig} from 'vite';
import react from '@vitejs/plugin-react';

export default defineConfig({
  plugins:[
    react({
      babel:{
        plugins:['babel-plugin-react-compiler']
      }
    })
  ]
});

Babel Example

babel.config.js
module.exports={
  plugins:[
    'babel-plugin-react-compiler'
  ]
};

Create React App

Create React App hides Babel config, so React Compiler setup is not clean there. You usually need eject, CRACO, or react-app-rewired. For new apps, Vite is easier.
CRA
↓ Babel Config Hidden
↓ Hard To Configure Compiler

Vite / Custom Babel
↓ Easy To Configure Compiler

Interview Answer

React Compiler, formerly called React Forget, is an optional build-time compiler for React. It analyzes React components during the build process and automatically applies many memoization optimizations that developers previously handled using useMemo, useCallback, and React.memo. It does not replace Babel or SWC; instead, it works alongside them. Babel/SWC converts JSX to JavaScript, while React Compiler optimizes React rendering behavior. Once configured, it runs automatically during npm run build.

Final Summary

Use React Compiler as an optimization layer, not as a Babel replacement. Your app can run without it, but when configured and tested, it can reduce unnecessary re-renders and make your React code cleaner.
