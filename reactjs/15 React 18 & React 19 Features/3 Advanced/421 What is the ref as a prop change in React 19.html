421. What is the ref as a prop change in React 19?

One-Line Answer

In React 19, function components can receive ref directly as a normal prop, reducing the need for forwardRef() in many cases.

Problem Before React 19

Before React 19, function components could not receive ref directly.

App.jsx
function Input(props){
  return <input {...props}/>;
}

function App(){
  const inputRef=useRef(null);

  return <Input ref={inputRef}/>;
}
This does not work because React does not pass ref as a normal prop.

Old Solution: forwardRef()

Input.jsx
import {forwardRef} from 'react';

const Input=forwardRef(function Input(props,ref){
  return <input ref={ref} {...props}/>;
});

export default Input;
App.jsx
import {useRef} from 'react';
import Input from './Input';

export default function App(){

  const inputRef=useRef(null);

  function focusInput(){
    inputRef.current.focus();
  }

  return (
    <>
      <Input ref={inputRef} placeholder="Name"/>
      <button onClick={focusInput}>Focus</button>
    </>
  );

}
Parent Creates Ref
↓ Passes Ref
↓ forwardRef Receives Ref
↓ Attach To DOM Element
↓ inputRef.current Works

React 19

Now ref can be received directly as a prop.

Input.jsx
function Input({ref,...props}){
  return <input ref={ref} {...props}/>;
}

export default Input;
App.jsx
import {useRef} from 'react';
import Input from './Input';

export default function App(){

  const inputRef=useRef(null);

  function focusInput(){
    inputRef.current.focus();
  }

  return (
    <>
      <Input ref={inputRef} placeholder="Name"/>
      <button onClick={focusInput}>Focus</button>
    </>
  );

}
No forwardRef() needed in many simple cases.

Real-World Example

Suppose we have a reusable search component.

SearchInput.jsx
function SearchInput({ref,...props}){
  return <input ref={ref} {...props}/>;
}

export default SearchInput;
Products.jsx
import {useRef} from 'react';
import SearchInput from './SearchInput';

export default function Products(){

  const searchRef=useRef(null);

  function focusSearch(){
    searchRef.current.focus();
  }

  return (
    <>
      <SearchInput ref={searchRef} placeholder="Search Products"/>
      <button onClick={focusSearch}>Focus Search</button>
    </>
  );

}
Products Page Loads
↓ User Clicks Focus Search
↓ searchRef.current.focus()
↓ Cursor Goes To Search Box

Why Did React Add This?

A huge amount of React code looked like this: 

const Input=forwardRef(function Input(props,ref){
  return <input ref={ref} {...props}/>;
});

The only purpose was forwarding the ref.

Create Component
↓ Need Ref
↓ Write forwardRef
↓ Extra Boilerplate

React 19 reduces that boilerplate.

Before vs After

Before React 19 React 19
forwardRef required Often not required
Special API ref behaves more like normal prop
More boilerplate Cleaner code

Important Note

You still need to understand forwardRef because many existing React applications and interview questions use it.
React 19 does not magically remove all forwardRef use cases. It simply makes many common ref-forwarding scenarios easier.

Easy Way To Remember

Before React 19
↓ ref
↓ forwardRef
↓ DOM Element

React 19
↓ ref
↓ Component Prop
↓ DOM Element

Interview Answer

The ref as a prop change in React 19 allows function components to receive ref directly as a prop, reducing the need for forwardRef in many common scenarios. This makes component code simpler and removes boilerplate that was previously required only to forward refs to child elements.
