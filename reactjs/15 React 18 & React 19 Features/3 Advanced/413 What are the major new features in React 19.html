413. What are the major new features in React 19?

One-Line Answer

React 19 adds important features like Actions, Form Actions, useActionState, useFormStatus, useOptimistic, use(), React Server Components support, ref as prop, simpler Context provider syntax, and better SSR / metadata / hydration error handling.

1. Actions

Actions are async functions used for updates like save, delete, login, signup, place order, add comment, etc.

User Clicks Save
↓ Action Runs
↓ Pending State
↓ Server Update
↓ UI Updates
Example: Save User Action
async function saveUser(formData){
  const name=formData.get('name');

  await fetch('/api/users',{
    method:'POST',
    body:JSON.stringify({name})
  });
}

export default function UserForm(){
  return (
    <form action={saveUser}>
      <input name="name" placeholder="Enter name"/>
      <button>Save</button>
    </form>
  );
}
Here saveUser is the Action. It receives formData automatically when the form is submitted.

2. Form Actions

Before React 19, form submit usually needed onSubmit, preventDefault(), manual FormData, loading state, and API call.

Before React 19
function UserForm(){
  async function handleSubmit(e){
    e.preventDefault();
    const formData=new FormData(e.target);
    await saveUser(formData);
  }

  return (
    <form onSubmit={handleSubmit}>
      <input name="name"/>
      <button>Save</button>
    </form>
  );
}
React 19 Form Action
function UserForm(){
  return (
    <form action={saveUser}>
      <input name="name"/>
      <button>Save</button>
    </form>
  );
}
React collects the form data and calls the action function automatically.

3. useActionState()

useActionState() is used when you want your Action to return a message or state.

SignupForm.jsx
import {useActionState} from 'react';

async function createUser(prevState,formData){
  const name=formData.get('name');

  if(!name){
    return {message:'Name is required'};
  }

  await fetch('/api/users',{
    method:'POST',
    body:JSON.stringify({name})
  });

  return {message:'User created successfully'};
}

export default function SignupForm(){
  const [state,formAction]=useActionState(createUser,{message:''});

  return (
    <form action={formAction}>
      <input name="name" placeholder="Name"/>
      <button>Create User</button>
      <p>{state.message}</p>
    </form>
  );
}
Submit Form
↓ createUser Runs
↓ Returns Message
↓ state.message Updates

4. useFormStatus()

useFormStatus() is used inside a child component of a form to know if the form is currently submitting.

SubmitButton.jsx
import {useFormStatus} from 'react-dom';

function SubmitButton(){
  const {pending}=useFormStatus();

  return (
    <button disabled={pending}>
      {pending?'Saving...':'Save'}
    </button>
  );
}
UserForm.jsx
function UserForm(){
  return (
    <form action={saveUser}>
      <input name="name"/>
      <SubmitButton />
    </form>
  );
}
This avoids manually creating const [loading,setLoading]=useState(false).

Can We Use useFormStatus() In The Same Component?

No. The form component itself cannot read its own status using useFormStatus().
❌ Doesn't Work
import {useFormStatus} from 'react-dom';

function UserForm(){

  const {pending}=useFormStatus();

  return (
    <form action={saveUser}>
      <input name="name"/>
      <button disabled={pending}>
        Save
      </button>
    </form>
  );

}
When useFormStatus() runs, React has not entered the form context yet, so it cannot access the form status.
UserForm
↓ form
↓ button
✅ Correct Way
import {useFormStatus} from 'react-dom';

function SubmitButton(){

  const {pending}=useFormStatus();

  return (
    <button disabled={pending}>
      {pending?'Saving...':'Save'}
    </button>
  );

}

function UserForm(){

  return (
    <form action={saveUser}>
      <input name="name"/>
      <SubmitButton />
    </form>
  );

}
UserForm
↓ form
↓ SubmitButton
↓ useFormStatus()
Easy Rule:

❌ Form Component Itself
✅ Child Component Inside Form

5. useOptimistic()

useOptimistic() updates UI immediately before the server confirms the update.

User Clicks Like
↓ Like Count Increases Immediately
↓ API Request Runs
↓ Success = Keep Change
↓ Failure = Rollback
LikeButton.jsx
import {useState,useOptimistic} from 'react';

export default function LikeButton(){
  const [likes,setLikes]=useState(10);
  const [optimisticLikes,addLike]=useOptimistic(likes,(current,value)=>current+value);

  async function handleLike(){
    addLike(1);

    try{
      const updatedLikes=await saveLike();
      setLikes(updatedLikes);
    }catch(error){
      console.log(error);
    }
  }

  return <button onClick={handleLike}>❤️ {optimisticLikes}</button>;
}

function saveLike(){
  return new Promise(resolve=>setTimeout(()=>resolve(11),1000));
}

6. use() API

The use() API can read a Promise during render. It works with Suspense.

Users.jsx
import {use,Suspense} from 'react';

const usersPromise=fetch('/api/users').then(res=>res.json());

function Users(){
  const users=use(usersPromise);

  return (
    <ul>
      {users.map(user=><li key={user.id}>{user.name}</li>)}
    </ul>
  );
}

export default function Page(){
  return (
    <Suspense fallback={<h2>Loading users...</h2>}>
      <Users />
    </Suspense>
  );
}
use() is not like useEffect(). useEffect() runs after render, but use() reads async data during render.

7. React Server Components Support

Server Components run only on the server. They can fetch data directly and do not send their component JavaScript to the browser.

Users.jsx
export default async function Users(){
  const users=await fetch('https://api.example.com/users').then(res=>res.json());

  return (
    <ul>
      {users.map(user=><li key={user.id}>{user.name}</li>)}
    </ul>
  );
}
Server Component Runs
↓ Fetch Data On Server
↓ Render UI
↓ Send Result To Browser
↓ Less Client JavaScript
If a component has no 'use client', can be async, fetches data directly with await, and doesn't use useState/useEffect, it's typically a Server Component.

8. ref as Prop

React 19 allows ref to be passed as a normal prop, reducing the need for forwardRef().

Before React 19
Input.jsx
import {forwardRef} from 'react';

const Input=forwardRef(function Input(props,ref){
  return <input ref={ref} {...props}/>;
});

export default Input;
App.jsx
import {useRef} from 'react';
import Input from './Input';

export default function App(){

  const inputRef=useRef(null);

  function focusInput(){
    inputRef.current.focus();
  }

  return (
    <>
      <Input
        ref={inputRef}
        placeholder="Enter name"
      />

      <button onClick={focusInput}>
        Focus Input
      </button>
    </>
  );

}
React 19
function Input({ref,...props}){
  return <input ref={ref} {...props}/>;
}

9. Context Provider Shortcut

React 19 allows you to use context directly as a provider.

</ThemeContext.Provider>
React 19

  <App />
</ThemeContext>

Interview Answer

React 19 introduced Actions, Form Actions, useActionState, useFormStatus, useOptimistic, the use() API, React Server Components support, ref as a prop, simplified Context providers, document metadata support, better SSR support, and improved hydration error messages. These features simplify forms, async updates, optimistic UI, server rendering, and reduce common boilerplate in React applications.

Final Summary

React 19 mainly improves real-world app development by making forms, async actions, optimistic UI, server components, refs, metadata, and SSR easier to handle.
