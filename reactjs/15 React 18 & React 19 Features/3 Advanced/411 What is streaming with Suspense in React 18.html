411. What is Streaming with Suspense in React 18?

One-Line Answer

Streaming with Suspense allows React 18 SSR to send completed HTML parts immediately instead of waiting for the entire page to finish rendering.

First Recall Traditional SSR

From our previous SSR discussion:

User Requests Page
↓ Server Renders Everything
↓ Wait For All Data
↓ Generate Full HTML
↓ Send HTML
↓ Hydration Starts
One slow component can block the entire page.

Real Example

Imagine an e-commerce product page:

Header
Product Details
Buy Button
Reviews

Assume:

ComponentTime
Header100ms
Product Details200ms
Reviews5000ms

Traditional SSR

Wait Header
↓ Wait Product Details
↓ Wait Reviews
↓ Send HTML

User waits 5 seconds before seeing anything.

Streaming SSR

Header Ready
↓ Product Details Ready
↓ Send HTML Immediately
↓ Show Reviews Loading UI
↓ Reviews Finish Later
↓ Stream Reviews HTML
User can start reading the page while Reviews are still loading.

How Suspense Makes This Possible

ProductPage.jsx
import {Suspense} from 'react';

export default function ProductPage(){

  return (
    <>

      <Header />

      <ProductInfo />

      <Suspense fallback={<ReviewsSkeleton />}>
        <Reviews />
      </Suspense>

    </>
  );

}

What Does Suspense Mean Here?

React treats the Reviews section as a separate boundary.

Page
↓ Suspense Boundary
↓ Reviews

If Reviews is slow:

Render Header
↓ Render ProductInfo
↓ Render ReviewsSkeleton
↓ Send HTML
↓ Wait For Reviews
↓ Replace Skeleton With Reviews

What User Sees

At 0.3 Seconds


Product Name

₹19,999

Buy Now

Loading Reviews...

At 5 Seconds


Product Name

₹19,999

Buy Now

⭐⭐⭐⭐⭐
Great Product
Fast Delivery
...

Why This Is Called Streaming

Because HTML is sent in multiple chunks.

Without Streaming

Server
↓ One Big HTML Response
↓ Browser

With Streaming

Server
↓ Chunk 1 → Header
↓ Chunk 2 → ProductInfo
↓ Chunk 3 → Reviews
↓ Browser
HTML arrives gradually like a video stream instead of one big download.

Relation To React Server Components

Streaming becomes even more powerful with React Server Components.

Server Component Fetches Data
↓ Suspense Boundary
↓ Ready Content Streams First

Traditional SSR vs Streaming SSR

Traditional SSR Streaming SSR
Wait for everything Send ready parts immediately
One HTML response Multiple HTML chunks
Slow components block page Slow components isolated
Long blank screen Content appears quickly

Easy Way To Remember

↓ Cook Entire Meal

The customer starts eating immediately instead of waiting for every dish to be ready.

Interview Answer

Streaming with Suspense in React 18 is an SSR feature that allows the server to send rendered HTML in chunks rather than waiting for the entire page to finish rendering. Components wrapped in Suspense can display fallback content immediately while slow data loads. Once the data becomes available, React streams the remaining HTML to the browser. This improves perceived performance and reduces blank-screen waiting time.
