423. What are stylesheets and scripts improvements in React 19?

One-Line Answer

React 19 improves how React handles stylesheets and scripts. You can declare resources like <link rel="stylesheet"> and <script> inside components, and React can coordinate loading, deduplicate repeated resources, and work better with SSR and streaming.

Why Was This Needed?

Before React 19, developers commonly managed stylesheets and scripts manually in index.html or by creating script tags inside useEffect().

index.html
<link rel="stylesheet" href="/product.css"/>
<script src="/analytics.js"></script>
Old Dynamic Script Loading
import {useEffect} from 'react';

function ProductPage(){
  useEffect(()=>{
    const script=document.createElement('script');
    script.src='/analytics.js';
    document.body.appendChild(script);
  },[]);

  return <h1>Products</h1>;
}
This works, but loading order, duplicate files, and timing can become difficult, especially with SSR and streaming.

React 19 Stylesheet Support

React 19 allows stylesheet links to be declared directly inside components.

ProductPage.jsx
function ProductPage(){
  return (
    <>
      <link rel="stylesheet" href="/product.css"/>
      <h1>Products</h1>
    </>
  );
}
Component Renders
↓ React Finds Stylesheet
↓ React Loads CSS
↓ Content Appears With Correct Styling
This helps reduce problems like content appearing before important CSS is ready.

React 19 Script Support

Scripts can also be declared from components.

Analytics.jsx
function Analytics(){
  return <script async src="/analytics.js"/>;
}
React can understand the script as a resource instead of you manually appending it with DOM APIs.

Real-World Product Page Example

ProductPage.jsx
function ProductPage(){
  return (
    <>
      <title>iPhone 17</title>
      <meta name="description" content="Buy iPhone 17"/>
      <link rel="stylesheet" href="/product.css"/>
      <script async src="/analytics.js"/>
      <h1>iPhone 17</h1>
    </>
  );
}
Page Component Renders
↓ React Finds Metadata
↓ React Finds Stylesheet
↓ React Finds Script
↓ Resources Are Managed By React

Deduplication

If multiple components request the same stylesheet or script, React can avoid loading it multiple times.

Header.jsx
function Header(){
  return <link rel="stylesheet" href="/app.css"/>;
}
Footer.jsx
function Footer(){
  return <link rel="stylesheet" href="/app.css"/>;
}
Header Requests app.css
↓ Footer Requests app.css
↓ React Detects Same href
↓ Loads app.css Once

Relation To Streaming SSR

You recently learned streaming with Suspense. Stylesheet and script improvements are useful because React can coordinate resources while HTML is being streamed.

Server Component Example
export default async function ProductPage(){
  const product=await fetch('/api/product/1').then(res=>res.json());

  return (
    <>
      <title>{product.name}</title>
      <meta name="description" content={product.description}/>
      <link rel="stylesheet" href="/product.css"/>
      <script async src="/analytics.js"/>
      <h1>{product.name}</h1>
    </>
  );
}
Server Starts Rendering
↓ React Knows Required CSS
↓ HTML Streams
↓ Styles Are Coordinated
↓ Page Appears Correctly Styled

Interview Answer

React 19 improves stylesheet and script handling by treating them as resources that can be declared directly inside React components. React can coordinate their loading, deduplicate repeated resources, and integrate better with SSR and streaming. This reduces manual DOM code, improves loading behavior, and helps avoid issues like duplicate scripts or unstyled content appearing too early.
