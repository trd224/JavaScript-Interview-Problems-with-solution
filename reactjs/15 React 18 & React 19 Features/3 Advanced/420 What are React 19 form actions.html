420. What are React 19 Form Actions?

One-Line Answer

React 19 Form Actions allow a form to call a function directly using the action prop. React automatically passes FormData to that function, so we need less onSubmit, preventDefault(), and manual form-handling code. React’s official docs describe this as passing a function to the form action prop, where formData is passed automatically. :contentReference[oaicite:0]{index=0}

Before React 19

Before React 19, we normally handled forms using onSubmit.

OldForm.jsx
function OldForm(){
  async function handleSubmit(e){
    e.preventDefault();
    const formData=new FormData(e.target);
    const name=formData.get('name');
    await fetch('/api/users',{method:'POST',body:JSON.stringify({name})});
  }

  return (
    <form onSubmit={handleSubmit}>
      <input name="name"/>
      <button>Save</button>
    </form>
  );
}
Form Submit
↓ onSubmit Runs
↓ preventDefault()
↓ new FormData()
↓ API Call
This works, but it creates extra boilerplate code.

React 19 Form Action

In React 19, we can pass an async function directly to action.

UserForm.jsx
async function saveUser(formData){
  const name=formData.get('name');
  await fetch('/api/users',{method:'POST',body:JSON.stringify({name})});
}

export default function UserForm(){
  return (
    <form action={saveUser}>
      <input name="name"/>
      <button>Save</button>
    </form>
  );
}
Here saveUser is the Form Action. React automatically gives it formData.

What Happens Internally?

User Types Name
↓ Clicks Save
↓ React Creates FormData
↓ saveUser(formData) Runs
↓ API Call Happens
↓ Form Action Completes

Real-World Example

CreateUserForm.jsx
async function createUser(formData){
  const name=formData.get('name');
  const email=formData.get('email');

  await fetch('/api/users',{
    method:'POST',
    headers:{'Content-Type':'application/json'},
    body:JSON.stringify({name,email})
  });
}

export default function CreateUserForm(){
  return (
    <form action={createUser}>
      <input name="name" placeholder="Name"/>
      <input name="email" placeholder="Email"/>
      <button>Create User</button>
    </form>
  );
}

Form Actions With useActionState()

useActionState() is useful when the action should return a message, error, or result. React’s docs define useActionState as a hook for updating state with side effects using Actions. :contentReference[oaicite:1]{index=1}

SignupForm.jsx
import {useActionState} from 'react';

async function createUser(prevState,formData){
  const name=formData.get('name');

  if(!name){
    return {message:'Name is required'};
  }

  await fetch('/api/users',{method:'POST',body:JSON.stringify({name})});

  return {message:'User created successfully'};
}

export default function SignupForm(){
  const [state,formAction]=useActionState(createUser,{message:''});

  return (
    <form action={formAction}>
      <input name="name" placeholder="Name"/>
      <button>Create</button>
      <p>{state.message}</p>
    </form>
  );
}
Submit Form
↓ Action Runs
↓ Action Returns Message
↓ state.message Updates
↓ UI Shows Result

Form Actions With useFormStatus()

useFormStatus() gives information about the parent form submission, like pending. It must be used in a component rendered inside the form. :contentReference[oaicite:2]{index=2}

SubmitButton.jsx
import {useFormStatus} from 'react-dom';

function SubmitButton(){
  const {pending}=useFormStatus();

  return (
    <button disabled={pending}>
      {pending?'Saving...':'Save'}
    </button>
  );
}
UserForm.jsx
function UserForm(){
  return (
    <form action={saveUser}>
      <input name="name"/>
      <SubmitButton />
    </form>
  );
}
Do not use useFormStatus() in the same component that creates the form. Use it inside a child component inside the form.

Wrong Example: Same Component

WrongUserForm.jsx
import {useFormStatus} from 'react-dom';

function UserForm(){
  const {pending}=useFormStatus();

  return (
    <form action={saveUser}>
      <input name="name"/>
      <button disabled={pending}>Save</button>
    </form>
  );
}
This is wrong because UserForm itself is not inside the form context. The child component SubmitButton is inside the form context.

Benefits

  • Less boilerplate code
  • No manual preventDefault()
  • No manual new FormData(e.target)
  • Cleaner async form submission
  • Works well with useActionState()
  • Works well with useFormStatus()

Interview Answer

React 19 Form Actions let us pass a function directly to a form’s action prop. When the form is submitted, React automatically creates and passes FormData to that function. This simplifies form handling by reducing onSubmit, preventDefault(), and manual FormData boilerplate, and it works naturally with hooks like useActionState() and useFormStatus().
