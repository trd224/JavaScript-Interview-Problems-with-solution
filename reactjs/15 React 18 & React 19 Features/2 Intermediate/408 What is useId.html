408. What is useId?

One-Line Definition

useId() is a React 18 hook that generates unique and stable IDs for components, especially useful for form accessibility and Server-Side Rendering (SSR).

Why Do We Need useId?

Consider a form: 

<label htmlFor="email">
  Email
</label>

<input id="email" />

This works for a single input.

But imagine rendering multiple forms: 

<LoginForm />
<LoginForm />
<LoginForm />

Now all inputs may use the same id: 

<input id="email" />
<input id="email" />
<input id="email" />
HTML IDs must be unique.

Solution: useId()

Component Render
↓
useId()
↓
Generate Unique ID
↓
Attach To Label + Input
↓
No ID Conflicts

Basic Example

File: src/components/LoginForm.jsx
import { useId } from 'react';

function LoginForm() {
  const id = useId();

  return (
    <>
      <label htmlFor={id}>
        Email
      </label>

      <input id={id} />
    </>
  );
}

export default LoginForm;

Use Component

File: src/App.jsx
import LoginForm from './components/LoginForm';

function App() {
  return (
    <>
      <LoginForm />
      <LoginForm />
      <LoginForm />
    </>
  );
}

export default App;

Generated IDs

React automatically creates unique IDs: 

:r0:
:r1:
:r2:
Every component gets its own unique ID.

Real World Form Example

File: src/components/RegisterForm.jsx
import { useId } from 'react';

function RegisterForm() {
  const emailId = useId();
  const passwordId = useId();

  return (
    <form>

      <label htmlFor={emailId}>
        Email
      </label>

      <input id={emailId} type="email" />

      <label htmlFor={passwordId}>
        Password
      </label>

      <input id={passwordId} type="password" />

    </form>
  );
}

export default RegisterForm;

Label
↓

This improves accessibility (a11y).

Without useId()

function LoginForm() {
  return (
    <>
      <label htmlFor="email">Email</label>
      <input id="email" />
    </>
  );
}

Multiple forms create duplicate IDs.

With useId()

const id = useId();

<label htmlFor={id}>Email</label>
<input id={id} />

Every form gets a unique ID.

Why Not Use Math.random()?

Some developers do: 

const id = Math.random();
Bad idea for React SSR.

Because:

Server Generates ID
↓
Client Generates Different ID
↓
Hydration Mismatch

Why useId Is Better?

Server Render
↓
Generate ID
↓
Client Hydration
↓
Same ID
↓
No Mismatch
useId was specifically designed to work with SSR.

Common Interview Question

Can useId Be Used As React Keys?

No.

❌ Wrong: 
items.map(item => (
  <li key={useId()}>
    {item}
  </li>
))
✅ Correct: 
items.map(item => (
  <li key={item.id}>
    {item.name}
  </li>
))
useId is for accessibility IDs, not list keys.

React 17 vs React 18

React 17 React 18
No useId useId available
Manual IDs Automatic IDs
SSR issues possible SSR-safe IDs
More boilerplate Simpler forms

When Should You Use useId?

  • Form labels and inputs
  • Accessibility attributes
  • ARIA relationships
  • SSR applications
  • Reusable form components

When Should You NOT Use useId?

  • List keys
  • Database IDs
  • API IDs
  • Persistent IDs

Interview Answer

useId is a React 18 hook that generates unique, stable IDs for components. It is primarily used for linking labels and form controls, improving accessibility, and avoiding hydration mismatches in Server-Side Rendered applications.

One-Line Interview Answer

useId is a React 18 hook that generates unique SSR-safe IDs for accessibility and form elements.
