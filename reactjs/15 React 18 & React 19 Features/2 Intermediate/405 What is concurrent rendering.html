405. What is Concurrent Rendering?

One-Line Definition

Concurrent Rendering is React 18's new rendering engine that allows React to pause, interrupt, resume, or abandon rendering work to keep the UI responsive.

Why Was Concurrent Rendering Introduced?

Before React 18, rendering was synchronous.

Start Rendering
↓
Finish Entire Render
↓
Only Then Handle User Actions

If rendering was expensive, the UI could freeze.

Large renders could block typing, clicking, scrolling, and animations.

React 17 (Synchronous Rendering)

User Types
↓
Huge List Re-renders
↓
CPU Busy
↓
UI Freezes
↓
User Waits

React 18 (Concurrent Rendering)

User Types
↓
Huge List Starts Rendering
↓
User Clicks Button
↓
React Pauses Render
↓
Handles Click First
↓
Resumes Render
The UI remains responsive even during expensive rendering work.

Important Interview Point

Concurrent Rendering does NOT mean React renders components in parallel threads.

JavaScript is still single-threaded.

It means React can break rendering work into smaller chunks and prioritize important updates.

Traditional Rendering Example

File: src/components/Search.jsx
function Search() {
  const [text, setText] = useState('');

  const items = Array.from(
    { length: 50000 },
    (_, i) => `Item ${i}`
  );

  return (
    <>
      <input
        value={text}
        onChange={e => setText(e.target.value)}
      />

      {items.map(item => (
        <div key={item}>{item}</div>
      ))}
    </>
  );
}

Typing may feel slow because React must finish rendering thousands of elements.

How Concurrent Rendering Helps

Large Render Task
↓
Break Into Chunks
↓
Check For User Input
↓
Handle Important Updates
↓
Continue Rendering

Priority Levels

React now treats updates differently.

Priority Example
High Typing in input
High Button clicks
High Scrolling
Low Large search results
Low Filtering huge lists
Low Background updates

Example With startTransition()

File: src/components/Search.jsx
import { useState, startTransition } from 'react';

function Search() {
  const [text, setText] = useState('');
  const [results, setResults] = useState([]);

  function handleChange(e) {
    const value = e.target.value;

    setText(value);

    startTransition(() => {
      setResults(expensiveSearch(value));
    });
  }

  return (
    <input
      value={text}
      onChange={handleChange}
    />
  );
}

What Happens Here?

↓

↓

↓

setResults()
↓

Typing stays smooth even when searching thousands of records.

Can React Cancel Rendering?

Yes.

Start Rendering List A
↓
User Types Again
↓
List A Becomes Outdated
↓
React Discards Work
↓
Starts Rendering List B
React avoids wasting time rendering outdated UI.

Does Concurrent Rendering Require Code Changes?

Usually no.

React 18 enables Concurrent Rendering internally when using:

File: src/main.jsx
import ReactDOM from 'react-dom/client';

const root = ReactDOM.createRoot(
  document.getElementById('root')
);

root.render(<App />);

However, APIs like startTransition(), useTransition(), and useDeferredValue() allow you to take advantage of it.

Interview Answer

Concurrent Rendering is React 18's rendering engine that allows React to interrupt, pause, resume, or discard rendering work. It helps React prioritize urgent updates such as typing and clicking while delaying less important updates, keeping the UI responsive during expensive rendering operations.

One-Line Interview Answer

Concurrent Rendering allows React to pause and prioritize rendering work so that the UI remains responsive during expensive updates.
