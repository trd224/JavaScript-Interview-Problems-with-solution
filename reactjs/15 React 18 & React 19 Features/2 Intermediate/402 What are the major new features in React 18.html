402. What are the Major New Features in React 18?

One-Line Definition

React 18 introduced Concurrent Rendering, Automatic Batching, Transitions, Suspense improvements, Streaming SSR, new hooks, and the new createRoot API to improve performance and user experience.

Why React 18 Was Released?

More Responsive UI
↓
Faster Rendering
↓
Better SSR
↓
Future React Features

Major React 18 Features

1. Concurrent Rendering

React can now pause, resume, or abandon rendering work when necessary.

Large Render
↓
User Clicks Button
↓
React Pauses Render
↓
Handles Click First
↓
Resumes Render
UI stays responsive during heavy work.

2. Automatic Batching

Before React 18: 

setCount(c => c + 1);
setLoading(true);

Two state updates could cause two renders.

React 18 automatically batches them into a single render. :contentReference[oaicite:0]{index=0}

Multiple State Updates
↓
One Render
↓
Better Performance

3. startTransition()

File: App.jsx
import { startTransition } from 'react';

function handleSearch(value) {
  startTransition(() => {
    setResults(value);
  });
}

Marks updates as non-urgent so React can prioritize important interactions. :contentReference[oaicite:1]{index=1}

4. useTransition()

File: App.jsx
const [isPending, startTransition] = useTransition();

Allows showing loading indicators while a transition is running. 

{
  isPending && <p>Loading...</p>
}

5. useDeferredValue()

File: Search.jsx
const deferredSearch = useDeferredValue(searchText);
User Types
↓
Input Updates Immediately
↓
Expensive UI Updates Later

Useful for large search results and filtering operations. :contentReference[oaicite:2]{index=2}

6. Suspense Improvements

React 18 greatly improved Suspense and enabled better integration with server rendering. :contentReference[oaicite:3]{index=3}

File: App.jsx
import React, { Suspense } from 'react';

<Suspense fallback={<h2>Loading...</h2>}>
  <Products />
</Suspense>

7. Streaming Server Side Rendering (SSR)

Before React 18, the server waited for the whole page before sending HTML.

React 18 can stream HTML in chunks. :contentReference[oaicite:4]{index=4}

Server Starts Rendering
↓
Send First HTML Chunk
↓
User Sees Content Earlier
↓
Remaining HTML Streams Later

8. New createRoot() API

React 17:

File: index.js
import ReactDOM from 'react-dom';

ReactDOM.render(<App />, document.getElementById('root'));

React 18:

File: main.jsx
import ReactDOM from 'react-dom/client';

const root = ReactDOM.createRoot(document.getElementById('root'));

root.render(<App />);
Required to enable React 18 features. :contentReference[oaicite:5]{index=5}

9. useId()

File: Login.jsx
import { useId } from 'react';

function Login() {
  const id = useId();

  return (
    <>
      <label htmlFor={id}>Email</label>
      <input id={id} />
    </>
  );
}

Generates stable IDs for forms and SSR. :contentReference[oaicite:6]{index=6}

React 18 Strict Mode intentionally mounts components twice in development.

Mount Component
↓
Unmount
↓
Mount Again
↓
Detect Side Effects
