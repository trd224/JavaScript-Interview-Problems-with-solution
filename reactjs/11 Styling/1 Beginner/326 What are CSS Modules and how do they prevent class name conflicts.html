326. What are CSS Modules and How Do They Prevent Class Name Conflicts?

One-Line Definition

CSS Modules are CSS files that automatically generate unique class names for each component, preventing class name conflicts across the application.

What Is the Problem with Normal CSS?

Normal CSS is global.

If two components use the same class name, their styles can overwrite each other.

Button.css

.button {

  background: blue;

  color: white;

}

Card.css

.button {

  background: green;

  color: black;

}
Both files contain a class named button. Because CSS is global, one style can override the other.

How CSS Modules Solve This Problem

Create CSS Module
↓
React Build Process
↓
Generate Unique Class Name
↓
button__x7A92
↓
No Class Name Conflict

Step 1 - Create a CSS Module File

Button.module.css
CSS Module files must end with .module.css.

Step 2 - Add CSS Classes

/* Button.module.css */

.button {

  background: blue;

  color: white;

  padding: 10px 20px;

  border-radius: 5px;

}

Step 3 - Import the CSS Module

import styles from './Button.module.css';

Step 4 - Use the Class

<button className={styles.button}>

  Click Me

</button>

What React Actually Generates

We write: 

styles.button

During build, React generates something similar to: 

Button_button__aX93k
React automatically creates a unique class name so that no other component can accidentally use the same style.

Real Conflict Example

Without CSS Modules

/* Button.css */

.button {

  background: blue;

}
/* Card.css */

.button {

  background: green;

}
The final color depends on which CSS file loads last.

With CSS Modules

Button_button__aX93k

Card_button__pL82m
Both classes become different, so there is no collision.

Complete Example

Button.module.css

.button {

  background: blue;

  color: white;

  padding: 10px 20px;

}

Button.jsx

import styles from './Button.module.css';

function Button() {

  return (

    <button className={styles.button}>

      Click Me

    </button>

  );

}

export default Button;

CSS File vs CSS Module

Feature Normal CSS CSS Module
Scope Global Component
Class Name Conflicts Possible Prevented
Import Style import './App.css' import styles from './App.module.css'
Usage className="button" className={styles.button}

When Should We Use CSS Modules?

  • Large React applications.
  • Reusable component libraries.
  • Projects with many developers.
  • Applications where style conflicts must be avoided.

Real Project Example

In our Aadhaar Authentication application, popup components, authentication forms, loaders, dashboards, and reusable widgets can use CSS Modules to ensure their styles do not affect other parts of the application.

Important Interview Points

  • CSS Modules create locally scoped CSS.
  • Files must end with .module.css.
  • React generates unique class names automatically.
  • Class name collisions are prevented.
  • Classes are accessed through an imported styles object.

Short Interview Answer

CSS Modules are locally scoped CSS files that automatically generate unique class names, preventing styles from one component from affecting another component.

One-Line Interview Answer

CSS Modules prevent class name conflicts by generating unique, component-scoped class names automatically.
