325. What are CSS Modules and How Are They Used in React?

One-Line Definition

CSS Modules are CSS files whose class names are automatically scoped to a specific React component, preventing style conflicts.

Why Do We Need CSS Modules?

With normal CSS files, class names are global.

If two components use the same class name, their styles can conflict.

Problem with Normal CSS

/* Button.css */

.button {

  background: blue;

}
/* Card.css */

.button {

  background: green;

}
Both files use the same class name. One style may overwrite the other.

How CSS Modules Solve This Problem

Button.module.css
↓
React Builds Project
↓
Generates Unique Class Name
↓
button__aX93k
↓
No Style Conflict

Step 1 - Create a CSS Module File

CSS Module files must end with: 

.module.css

Example: 

Button.module.css

Step 2 - Add Styles

/* Button.module.css */

.button {

  background: blue;

  color: white;

  padding: 10px 20px;

  border: none;

  border-radius: 5px;

}

Step 3 - Import the Module

import styles from './Button.module.css';
The imported object contains all CSS class names.

Step 4 - Apply the Class

<button className={styles.button}>

  Click Me

</button>

Complete Example

Button.module.css

.button {

  background: blue;

  color: white;

  padding: 10px 20px;

  border: none;

  border-radius: 5px;

}

Button.jsx

import styles from './Button.module.css';

function Button() {

  return (

    <button className={styles.button}>

      Click Me

    </button>

  );

}

export default Button;

What React Actually Generates

We write: 

styles.button

React generates something similar to: 

Button_button__X8YkP
Every component gets unique class names automatically.

Multiple Classes Example

Card.module.css

.card {

  padding: 20px;

  background: white;

}

.title {

  color: blue;

}

Card.jsx

import styles from './Card.module.css';

function Card() {

  return (

    <div className={styles.card}>

      <h2 className={styles.title}>

        User Profile

      </h2>

    </div>

  );

}

CSS File vs CSS Module

Normal CSS CSS Module
Global scope Component scope
Class conflicts possible No class conflicts
Imported normally Imported as object
className="button" className={styles.button}

When Should We Use CSS Modules?

  • Medium and large React projects.
  • Reusable components.
  • Projects with many developers.
  • When avoiding style conflicts is important.

React Style Options Comparison

Real Project Example

In our Aadhaar Authentication application, CSS Modules can be used for popup components, authentication forms, consent screens, and reusable widgets to ensure that styles from one component do not affect another component.

Important Interview Points

  • CSS Modules provide component-scoped CSS.
  • Files must end with .module.css.
  • Classes are imported as a JavaScript object.
  • React generates unique class names automatically.
  • CSS Modules prevent style conflicts.

Short Interview Answer

CSS Modules are CSS files with locally scoped class names. They are imported as objects and help prevent class name collisions between React components.

One-Line Interview Answer

CSS Modules provide component-scoped CSS by generating unique class names automatically.
