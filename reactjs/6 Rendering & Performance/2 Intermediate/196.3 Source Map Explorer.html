Source Map Explorer (Complete Beginner Guide)

One-Line Definition

Source Map Explorer is a tool that analyzes JavaScript bundles using source maps and shows which original files contribute to the final bundle size.

Why Do We Need It?

After creating a production build, we may know the bundle size but not which files created it. Source Map Explorer reads the bundle and its source map to identify the size contribution of every file.

Without source maps, the tool cannot accurately map bundled code back to the original source files.

How It Works

Create Production Build → Generate Source Maps → Read bundle.js + bundle.js.map → Analyze Original Files → Display Visual Report

Step 1 - Install

npm install --save-dev source-map-explorer

Step 2 - Build

npm run build

Step 3 - Run Explorer

npx source-map-explorer 'build/static/js/*.js'

Example

import axios from 'axios';
import _ from 'lodash';
import moment from 'moment';
import Chart from 'chart.js';

Report:

Chart.js - 520 KB
Moment - 330 KB
Lodash - 480 KB
Axios - 32 KB

How Source Maps Help

The bundle contains minified code, while the .map file stores information linking minified code back to the original files. Source Map Explorer uses this mapping to calculate the size of each source file.

Optimization Tips

  • Remove unused dependencies.
  • Use tree shaking.
  • Lazy load heavy components.
  • Import only required functions.
  • Replace heavy libraries.
