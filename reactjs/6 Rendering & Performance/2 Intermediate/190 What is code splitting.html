190. What is Code Splitting?

One-Line Definition

Code Splitting is a technique that divides a large JavaScript bundle into smaller chunks so users download only the code they currently need.

Why Is It Needed?

Suppose our React application contains:

  • Dashboard
  • Products Page
  • Reports Page
  • Settings Page
  • Admin Panel

Without code splitting, React builds: 

bundle.js = 5 MB

When the user opens: 

/dashboard

the browser still downloads: 


Dashboard Code
+
Products Code
+
Reports Code
+
Settings Code
+
Admin Code
The user downloads code for pages they may never visit.

The Problem

Open Dashboard
↓
Download Entire Application
↓
Large Bundle
↓
Slow Initial Load

How Code Splitting Helps

Instead of one large bundle: 


bundle.js

React creates: 


dashboard.chunk.js

products.chunk.js

reports.chunk.js

settings.chunk.js

admin.chunk.js

Now when the user opens: 

/dashboard

the browser downloads only: 


dashboard.chunk.js
Users download only the code needed for the current screen.

How To Implement Code Splitting

React provides: 

React.lazy()

and: 

Suspense

Normal Import

import Dashboard from './Dashboard';

Dashboard becomes part of the main bundle.

Lazy Import

const Dashboard = React.lazy(() =>
  import('./Dashboard')
);

Dashboard is loaded only when needed.

This creates a separate chunk automatically.

Using Suspense

import React, { Suspense } from 'react';

const Dashboard = React.lazy(() =>
  import('./Dashboard')
);

function App() {

  return (

    <Suspense fallback={<p>Loading...</p>}>

      <Dashboard />

    </Suspense>

  );

}

While the chunk downloads: 

Loading...

is shown.

Route-Based Code Splitting

Most applications split code by routes. 

import { BrowserRouter, Routes, Route } from 'react-router-dom';
import React, { Suspense } from 'react';

const Dashboard = React.lazy(() =>
  import('./pages/Dashboard')
);

const Products = React.lazy(() =>
  import('./pages/Products')
);

const Reports = React.lazy(() =>
  import('./pages/Reports')
);

function App() {

  return (

    <BrowserRouter>

      <Suspense fallback={<h2>Loading...</h2>}>

        <Routes>

          <Route
            path="/"
            element={<Dashboard />}
          />

          <Route
            path="/products"
            element={<Products />}
          />

          <Route
            path="/reports"
            element={<Reports />}
          />

        </Routes>

      </Suspense>

    </BrowserRouter>

  );

}

export default App;

Each page becomes its own chunk.

Code Splitting vs Lazy Loading

Code Splitting Lazy Loading
Creates separate chunks. Loads chunks when needed.
Build-time optimization. Runtime optimization.
Uses dynamic imports. Uses React.lazy.

Usually both work together.

What Does Webpack/Vite Generate?

Instead of: 


main.js

The build may create: 


main.js

dashboard.abc123.js

products.xyz456.js

reports.pqr789.js

These are called chunks.

Benefits of Code Splitting

  • Smaller initial bundle.
  • Faster page load.
  • Less bandwidth usage.
  • Better user experience.
  • Improved Core Web Vitals.
  • Better performance on slow networks.

When Should You Use It?

  • Large React applications.
  • Admin dashboards.
  • E-commerce websites.
  • Applications with many routes.
  • Pages containing heavy libraries.

Important Interview Points

  • Code splitting divides large bundles into smaller chunks.
  • Implemented using dynamic import().
  • React.lazy enables component-level code splitting.
  • Suspense shows loading UI while chunks load.
  • Commonly used with route-based splitting.
  • Improves initial load performance.

Short Interview Answer

Code Splitting is a performance optimization technique that divides a large JavaScript bundle into smaller chunks. React loads only the required chunks when needed, reducing initial load time and improving application performance.

One-Line Interview Answer

Code Splitting loads only the code needed for the current page instead of downloading the entire application upfront.
