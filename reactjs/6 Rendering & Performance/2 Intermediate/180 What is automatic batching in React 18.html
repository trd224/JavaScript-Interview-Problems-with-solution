180. What is Automatic Batching in React 18?

One-Line Definition

Automatic Batching is a React 18 feature that groups multiple state updates into a single re-render to improve performance.

What Problem Does It Solve?

Every time state changes, React normally re-renders the component.

Consider: 

setCount(c => c + 1);

setName('Rizwan');

Without batching:

setCount()
↓ Render #1

setName()
↓ Render #2

Two state updates cause two renders.

This is inefficient.

Multiple renders mean extra work for React.

What Is Batching?

Batching means React groups multiple state updates together and performs only one render.

setCount()
+
setName()

Grouped Together
↓
Single Render
Fewer renders = better performance.

Example

function App() {

  const [count, setCount] = useState(0);

  const [name, setName] = useState('');

  function handleClick() {

    setCount(c => c + 1);

    setName('Rizwan');

  }

}

React combines both updates: 

setCount(...)

setName(...)

into: 

One Render
before React 18 batching mainly happened when updates occurred in the same React event.

Before React 18

React only batched updates inside React event handlers.

React Event Handler

function handleClick() {

  setCount(c => c + 1);

  setName('Rizwan');

}

This was batched.

React 17: One render.

But Inside setTimeout

setTimeout(() => {

  setCount(c => c + 1);

  setName('Rizwan');

}, 1000);

React 17 did NOT batch these updates.

setCount()
↓ Render #1

setName()
↓ Render #2
React 17 only batched inside React event handlers.

React 18 Automatic Batching

React 18 automatically batches updates almost everywhere.

  • Click handlers
  • setTimeout
  • setInterval
  • Promises
  • fetch callbacks
  • Async functions
  • Native event listeners

Example

setTimeout(() => {

  setCount(c => c + 1);

  setName('Rizwan');

}, 1000);

React 18 batches them:

setCount()
+
setName()

Single Render

Promise Example

fetch('/users')

  .then(() => {

    setCount(c => c + 1);

    setName('Rizwan');

  });

React 17: 

Two Renders

React 18: 

One Render

How Does React Know?

React waits until the current task finishes.

Then it combines all pending state updates and performs one render.

Can We Disable Batching?

Yes.

React provides: 

flushSync()

Example: 

import { flushSync } from 'react-dom';

flushSync(() => {
  setCount(c => c + 1);
});

flushSync(() => {
  setName('Rizwan');
});

Now React renders immediately after each update.

Use flushSync only when absolutely necessary.

Before vs After React 18

React 17 React 18
Batched only in React events. Batched almost everywhere.

Promises not batched. Promises batched.
More renders. Fewer renders.
Lower performance. Better performance.

Relationship With Concurrent Rendering

Concurrent Rendering and Automatic Batching were both introduced in React 18.

Important Interview Points

  • Introduced in React 18.
  • Groups multiple state updates into one render.
  • Works in timeouts, promises, async functions, and events.
  • Reduces unnecessary renders.
  • Improves performance automatically.
  • Can be bypassed using flushSync().

Short Interview Answer

Automatic Batching is a React 18 optimization that groups multiple state updates into a single render, reducing unnecessary rendering and improving performance.
