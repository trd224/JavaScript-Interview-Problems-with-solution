178. What causes unnecessary re-renders and how do you avoid them?

One-Line Definition

Unnecessary re-render means a component renders again even though its visible output did not actually need to change.

When Does React Re-render?

A React component re-renders mainly when:

  • Its state changes.
  • Its props change.
  • Its parent component re-renders.
  • Context value changes.
Re-render is not always bad. It becomes a problem only when expensive components render again without need.

Main Causes of Unnecessary Re-renders

  • Parent component re-renders.
  • Passing new object or array props every render.
  • Passing new function props every render.
  • Updating state unnecessarily.
  • Putting too much state in one parent component.
  • Context value changes too often.
  • Expensive calculations inside render.

Cause 1 - Parent Re-renders

When a parent component re-renders, its child components also re-render by default. 

function Parent() {

  const [count, setCount] = useState(0);

  return (
    <div>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>

      <Child />
    </div>
  );

}

function Child() {

  console.log('Child rendered');

  return <h2>Child Component</h2>;

}

Even if Child does not use count, Child still renders when Parent renders.

How To Avoid It?

Use React.memo for child components that receive the same props. 

const Child = React.memo(function Child() {

  console.log('Child rendered');

  return <h2>Child Component</h2>;

});
React.memo prevents child re-render when props are unchanged.

Cause 2 - New Function Prop Every Render

In JavaScript, a new function is created every time the parent renders. 

function Parent() {

  const [count, setCount] = useState(0);

  function handleClick() {
    console.log('Clicked');
  }

  return (
    <div>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>

      <Child onClick={handleClick} />
    </div>
  );

}

Even with React.memo, Child may re-render because handleClick is a new function reference.

How To Avoid It?

Use useCallback to keep the same function reference. 

const handleClick = useCallback(() => {
  console.log('Clicked');
}, []);

Complete example: 

function Parent() {

  const [count, setCount] = useState(0);

  const handleClick = useCallback(() => {
    console.log('Clicked');
  }, []);

  return (
    <div>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>

      <Child onClick={handleClick} />
    </div>
  );

}

const Child = React.memo(function Child({ onClick }) {

  console.log('Child rendered');

  return <button onClick={onClick}>Child Button</button>;

});

Cause 3 - New Object or Array Prop Every Render

Objects and arrays are compared by reference. 

<Child user={{ name: 'Rizwan' }} />

This creates a new object on every render.

So React sees it as a changed prop.

How To Avoid It?

Use useMemo to keep the same object reference. 

const user = useMemo(() => {
  return { name: 'Rizwan' };
}, []);

return <Child user={user} />;
useMemo keeps object or array reference stable.

Cause 4 - Expensive Calculations Inside Render

If expensive logic runs on every render, the UI may become slow. 

const total = products.reduce((sum, item) => sum + item.price, 0);

This calculation runs every time the component renders.

How To Avoid It?

Use useMemo for expensive calculations. 

const total = useMemo(() => {
  return products.reduce((sum, item) => sum + item.price, 0);
}, [products]);

Cause 5 - State Kept Too High

If state is placed in a top parent, many child components may re-render when that state changes.

Parent State Changes
↓
Parent Re-renders
↓
All Children Re-render

How To Avoid It?

Keep state as close as possible to where it is used.

Do not keep every state in the top parent. Move local state into the component that actually needs it.

Cause 6 - Context Value Changes Too Often

When context value changes, all components consuming that context may re-render. 

<UserContext.Provider value={{ user, logout }}>
  {children}
</UserContext.Provider>

Here value is a new object on every render.

How To Avoid It?

Memoize the context value. 

const value = useMemo(() => {
  return { user, logout };
}, [user, logout]);

<UserContext.Provider value={value}>
  {children}
</UserContext.Provider>

Important Note

Do not blindly use React.memo, useMemo, and useCallback everywhere. Use them when there is an actual performance problem or stable reference is required.

Best Practices

  • Keep component state local when possible.
  • Use React.memo for expensive child components.
  • Use useCallback for function props passed to memoized children.
  • Use useMemo for expensive calculations and stable objects.
  • Avoid creating objects and arrays directly in JSX props.
  • Split large components into smaller components.
  • Use React DevTools Profiler to find real performance problems.

Quick Table

Cause Solution
Parent re-renders Use React.memo if child props are same.

New object or array prop Use useMemo.
Expensive calculation Use useMemo.
State too high Move state closer to where it is used.
Context value changes Memoize context value or split context.

Short Interview Answer

Unnecessary re-renders are caused by parent re-renders, new object or function references, unnecessary state updates, broad context updates, and expensive calculations during render. We avoid them using React.memo, useCallback, useMemo, local state placement, and context optimization.

One-Line Interview Answer

Avoid unnecessary re-renders by keeping props stable, memoizing expensive parts, placing state correctly, and preventing needless parent or context updates.
