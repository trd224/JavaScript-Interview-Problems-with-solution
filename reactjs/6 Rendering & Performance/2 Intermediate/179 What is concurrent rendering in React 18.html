179. What is Concurrent Rendering in React 18?

One-Line Definition

Concurrent Rendering is React 18's new rendering system that allows React to pause, interrupt, restart, and prioritize rendering work to keep the UI responsive.

The Problem Before React 18

Before React 18, rendering was synchronous.

Once React started rendering, it had to finish everything before responding to user interactions.

React Starts Rendering
↓
Must Finish Entire Render
↓
User Waits
↓
UI Updates

For large applications, this could make the UI feel slow or frozen.

Example Problem

Suppose we have:

  • A search box.
  • A list containing 10,000 items.

When the user types: 

R
Re
Rea
Reac
React

React must: 

Update Input
+
Filter 10,000 Items
+
Render 10,000 Items

In older React versions, everything happened immediately.

The input could feel laggy.

Heavy rendering work could block user interactions.

How Concurrent Rendering Helps

React can now split rendering work into smaller pieces.

Start Rendering
↓
User Types
↓
Pause Current Work
↓
Handle User Interaction
↓
Continue Rendering Later

The UI stays responsive because urgent work gets priority.

The Most Important Thing To Understand

Concurrent Rendering does NOT mean React renders multiple components at the same time.

"Concurrent" does NOT mean parallel rendering.

JavaScript still runs on a single thread.

React simply becomes smarter about scheduling work.

What React Can Do Now

  • Pause rendering.
  • Resume rendering.
  • Restart rendering.
  • Discard outdated renders.
  • Prioritize important updates.
React can interrupt low-priority work when more important work arrives.

Real Example

User types in a search box: 

setSearch('React');

At the same time: 

renderLargeList();

React gives higher priority to: 

User Typing

and lower priority to: 

Large List Rendering

So the typing stays smooth.

How Do We Use Concurrent Rendering?

Most of the time we do not use it directly.

React automatically uses the concurrent renderer when we create our app with: 

import ReactDOM from 'react-dom/client';

const root = ReactDOM.createRoot(
  document.getElementById('root')
);

root.render(<App />);
Using createRoot enables React 18's concurrent features.

Features Built On Top Of Concurrent Rendering

Feature Purpose
useTransition Mark updates as low priority.
useDeferredValue Delay expensive updates.
Suspense Show fallback UI while waiting.
Automatic Batching Reduce unnecessary renders.

Example Using useTransition

const [isPending, startTransition] = useTransition();

function handleChange(e) {

  const value = e.target.value;

  setInput(value);

  startTransition(() => {

    setSearch(value);

  });

}

React treats: 

setInput(value);

as urgent work.

And: 

setSearch(value);

as lower priority work.

Before vs After React 18

Before React 18 React 18
Rendering was blocking. Rendering can be interrupted.
No prioritization. Priority-based scheduling.
UI could become unresponsive. UI stays responsive.
All updates treated equally. Urgent and non-urgent updates separated.

Important Interview Points

  • Introduced in React 18.
  • Allows interruptible rendering.
  • Improves responsiveness.
  • Does not mean parallel rendering.
  • Works behind useTransition and useDeferredValue.
  • Enabled through createRoot().
  • Allows React to prioritize updates.

Short Interview Answer

Concurrent Rendering is React 18's rendering system that allows React to pause, interrupt, and prioritize rendering work so the UI remains responsive during expensive updates.

One-Line Interview Answer

Concurrent Rendering lets React prioritize important updates and interrupt less important rendering work to keep the UI responsive.
