202. How do you prevent memory leaks in React components?

One-Line Definition

We prevent memory leaks in React components by cleaning up async tasks, timers, subscriptions, event listeners, and connections when the component unmounts.

Simple Meaning

A memory leak happens when something keeps running even after the component is removed from the UI.

For example:

  • Timer keeps running.
  • API request finishes after component unmount.
  • Event listener remains attached.
  • WebSocket or SignalR connection remains active.
  • Subscription keeps sending updates.
If these things are not cleaned up, they can waste memory, update removed components, and slow down the application.

Visual Flow

Component Mounted
↓
Effect Starts Timer / API / Listener
↓
Component Unmounted
↓

↓
Memory Leak Prevented

Main Rule

Anything started inside useEffect should usually be stopped inside the cleanup function.

Basic Cleanup Syntax

useEffect(() => {
  // start something

  return () => {
    // clean it here
  };
}, []);

1. Clear setTimeout

Problem

useEffect(() => {
  setTimeout(() => setMessage('Done'), 3000);
}, []);

If the component unmounts before 3 seconds, timeout still runs.

Solution

useEffect(() => {
  const timeoutId = setTimeout(() => setMessage('Done'), 3000);

  return () => clearTimeout(timeoutId);
}, []);
clearTimeout stops the pending timeout when the component unmounts.

2. Clear setInterval

Problem

useEffect(() => {
  setInterval(() => setCount(c => c + 1), 1000);
}, []);

Interval continues forever if we do not clear it.

Solution

useEffect(() => {
  const intervalId = setInterval(() => setCount(c => c + 1), 1000);

  return () => clearInterval(intervalId);
}, []);

3. Remove Event Listeners

Problem

useEffect(() => {
  window.addEventListener('resize', handleResize);
}, []);

The resize listener remains attached after component unmount.

Solution

useEffect(() => {
  function handleResize() {
    setWidth(window.innerWidth);
  }

  window.addEventListener('resize', handleResize);

  return () => window.removeEventListener('resize', handleResize);
}, []);
The same function reference must be used for addEventListener and removeEventListener.

4. Cancel API Requests

If a component unmounts before an API response comes back, the response may still try to update state.

Solution With AbortController

useEffect(() => {
  const controller = new AbortController();

  fetch('/api/user', { signal: controller.signal })
    .then(res => res.json())
    .then(data => setUser(data))
    .catch(error => {
      if (error.name !== 'AbortError') console.log(error);
    });

  return () => controller.abort();
}, []);
AbortController cancels the fetch request when the component unmounts.

5. Cleanup WebSocket / SignalR Connections

Example

useEffect(() => {
  const connection = new signalR.HubConnectionBuilder()
    .withUrl('/faceAuthHub')
    .build();

  connection.start();

  connection.on('ReceiveMsg', message => {
    setMessage(message);
  });

  return () => {
    connection.off('ReceiveMsg');
    connection.stop();
  };
}, []);
Connections should be stopped when the component is removed.

6. Cleanup Subscriptions

useEffect(() => {
  const subscription = userService.subscribe(data => {
    setUser(data);
  });

  return () => subscription.unsubscribe();
}, []);

This is common in observable-based APIs or custom subscription services.

7. Avoid State Updates After Unmount

Sometimes we cannot cancel an async task directly.

In that case, we can ignore the result after unmount. 

useEffect(() => {
  let isMounted = true;

  fetch('/api/user')
    .then(res => res.json())
    .then(data => {
      if (isMounted) setUser(data);
    });

  return () => {
    isMounted = false;
  };
}, []);
Prefer AbortController for fetch. Use this pattern only when the async operation cannot be cancelled.

Real Project Example

Suppose our Aadhaar Authentication modal starts:

  • OTP API request
  • Face authentication SignalR connection
  • Device capture callback
  • Popup timer

If the user closes the modal, all these operations should be cleaned. 

useEffect(() => {
  const timerId = setTimeout(() => setShowError(false), 30000);

  return () => clearTimeout(timerId);
}, []);
When modal closes or component unmounts, timers, connections, and pending async operations should not continue unnecessarily.

Common Memory Leak Sources

Important Interview Points

  • Memory leaks usually happen when async work continues after unmount.
  • useEffect cleanup function is the main solution.
  • Clear timers and intervals.
  • Remove event listeners.
  • Cancel API requests using AbortController.
  • Unsubscribe from subscriptions.
  • Close WebSocket or SignalR connections.
  • Avoid calling setState after component unmount.

Short Interview Answer

To prevent memory leaks in React, we clean up side effects in the cleanup function returned from useEffect. This includes clearing timers, removing event listeners, aborting API requests, unsubscribing from subscriptions, and closing WebSocket or SignalR connections when the component unmounts.

One-Line Interview Answer

Prevent memory leaks by cleaning up all side effects in useEffect cleanup before the component unmounts.
