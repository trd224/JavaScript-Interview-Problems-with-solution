182. What is Concurrent Mode / Concurrent Features in React 18?

One-Line Definition

Concurrent Features are React 18 capabilities that allow React to pause, interrupt, prioritize, and resume rendering work to keep the UI responsive.

Important Interview Update

The old term "Concurrent Mode" is mostly outdated. React 18 does not ask us to enable a special Concurrent Mode. Instead, React provides Concurrent Features automatically through the new rendering engine.

Today, interviewers usually mean: 

Concurrent Rendering
+
useTransition
+
useDeferredValue
+
Suspense
+
Automatic Batching

The Problem Before React 18

Before React 18, rendering was synchronous.

Start Rendering
↓
Must Finish Everything
↓
User Waits
↓
UI Updates

If rendering took a long time, the UI could freeze temporarily.

Example Problem

Suppose we have:

  • Search box
  • 10,000 products
  • Expensive filtering

User types: 

R
Re
Rea
Reac
React

React must: 

Update Input
+
Filter 10,000 Items
+
Render 10,000 Items

The typing may feel slow.

Large rendering work can block user interactions.

How Concurrent Features Help

React can now split rendering work into smaller pieces.

Start Rendering
↓
User Types
↓
Pause Current Work
↓
Handle User Input First
↓
Continue Rendering Later
The UI stays responsive because important updates get higher priority.

The Biggest Misunderstanding

Many developers think: 

Concurrent = Parallel

This is NOT true.

JavaScript still runs on a single thread.

React is not rendering multiple components simultaneously.

React is simply smarter about scheduling work.

What React Can Do Now

  • Pause rendering.
  • Resume rendering.
  • Interrupt rendering.
  • Restart rendering.
  • Discard outdated renders.
  • Prioritize urgent updates.

Feature 1 - useTransition

Allows React to mark updates as low priority. 

const [isPending, startTransition] = useTransition();

function handleChange(e) {

  const value = e.target.value;

  setInput(value);

  startTransition(() => {

    setSearch(value);

  });

}

React prioritizes: 

setInput()

over: 

setSearch()

Feature 2 - useDeferredValue

Delays expensive updates while keeping important updates responsive. 

const deferredSearch = useDeferredValue(search);

The input updates immediately while expensive filtering waits.

Feature 3 - Suspense

Allows React to show fallback UI while waiting. 

<Suspense fallback={<Loading />}>

  <ProductList />

</Suspense>

Instead of blocking the whole page, React shows loading UI.

Feature 4 - Automatic Batching

React automatically groups multiple state updates into one render. 

setCount(c => c + 1);

setName('Rizwan');

React performs: 

One Render

instead of multiple renders.

How Are Concurrent Features Enabled?

React 18 concurrent features require: 

import ReactDOM from 'react-dom/client';

const root = ReactDOM.createRoot(
  document.getElementById('root')
);

root.render(<App />);
createRoot() enables the React 18 rendering engine.

Important Interview Points

  • Concurrent Mode is an older term.
  • React 18 uses Concurrent Features.
  • Concurrent does not mean parallel rendering.
  • React can interrupt and prioritize rendering.
  • Works with useTransition and useDeferredValue.
  • Improves UI responsiveness.
  • Enabled through createRoot().

Short Interview Answer

Concurrent Features in React 18 allow React to pause, interrupt, resume, and prioritize rendering work. This keeps the UI responsive during expensive rendering operations and powers features like useTransition, useDeferredValue, Suspense, and Automatic Batching.

One-Line Interview Answer

Concurrent Features let React prioritize important updates and interrupt less important rendering work to keep the UI responsive.
