192. How do you implement code splitting with React.lazy and Suspense?

One-Line Definition

We implement code splitting in React by using React.lazy() to load components dynamically and Suspense to show fallback UI while the component is loading.

Why Do We Need It?

Without code splitting, all components are included in the main JavaScript bundle.

Dashboard
+
Products
+
Reports
+
Admin
↓
One Large Bundle

This can make the initial page load slow.

The user may download code for pages they have not visited yet.

Step 1 - Normal Import

In normal import, the component becomes part of the main bundle. 

import Dashboard from './pages/Dashboard';

This means Dashboard code loads immediately.

Step 2 - Lazy Import

With React.lazy, the component is loaded only when it is rendered. 

const Dashboard = React.lazy(() =>
  import('./pages/Dashboard')
);
This creates a separate JavaScript chunk for Dashboard.

Step 3 - Wrap Lazy Component With Suspense

React.lazy components must be rendered inside Suspense. 

import React, { Suspense } from 'react';

const Dashboard = React.lazy(() =>
  import('./pages/Dashboard')
);

function App() {

  return (

    <Suspense fallback={<p>Loading...</p>}>

      <Dashboard />

    </Suspense>

  );

}

export default App;

While Dashboard is downloading, React shows: 

Loading...

Route-Based Code Splitting

In real projects, code splitting is usually done with routes. 

import { BrowserRouter, Routes, Route } from 'react-router-dom';
import React, { Suspense } from 'react';

const Dashboard = React.lazy(() => import('./pages/Dashboard'));
const Products = React.lazy(() => import('./pages/Products'));
const Reports = React.lazy(() => import('./pages/Reports'));

export default function App() {
  return (
    <BrowserRouter>

      <Suspense fallback={<h2>Loading...</h2>}>

        <Routes>
          <Route path="/" element={<Dashboard />} />
          <Route path="/products" element={<Products />} />
          <Route path="/reports" element={<Reports />} />
        </Routes>

      </Suspense>

    </BrowserRouter>
  );
}

What Happens Here?

User Opens Dashboard

/

React downloads: 

dashboard.chunk.js

User Opens Products Page

/products

React downloads: 

products.chunk.js

for the first time.

User Opens Reports Page

/reports

React downloads: 

reports.chunk.js

for the first time.

Each route loads its own JavaScript chunk only when needed.

Important Point About Suspense

Suspense does not split code by itself.

React.lazy creates the lazy component.

Suspense only handles the loading state.

API Role
React.lazy() Loads component dynamically and creates code splitting.
Suspense Shows fallback UI while lazy component is loading.

Code Splitting vs Lazy Loading

Code Splitting Lazy Loading
Breaks bundle into smaller chunks. Loads chunk only when needed.
Build output optimization. Runtime loading behavior.
Done by dynamic import(). Done using React.lazy().

Common Mistake

React.lazy only supports default exports directly.

This works: 

export default Dashboard;

Then: 

const Dashboard = React.lazy(() =>
  import('./pages/Dashboard')
);

If Component Uses Named Export

If the component is exported like: 

export function Dashboard() {

  return <h1>Dashboard</h1>;

}

Then use: 

const Dashboard = React.lazy(() =>
  import('./pages/Dashboard').then(module => ({
    default: module.Dashboard
  }))
);

When Should You Use React.lazy?

  • Route components.
  • Admin pages.
  • Heavy dashboard pages.
  • Chart components.
  • Large modals opened rarely.
  • Feature modules not needed initially.

When Should You Avoid It?

  • Small components used immediately.
  • Components above the fold that are needed instantly.
  • Very tiny components where extra network request is not worth it.

Important Interview Points

  • Use React.lazy with dynamic import().
  • Wrap lazy components in Suspense.
  • Suspense fallback shows loading UI.
  • Commonly used for route-based code splitting.
  • Improves initial load performance.
  • React.lazy expects default exports.

Short Interview Answer

We implement code splitting by importing components with React.lazy and wrapping them with Suspense. React.lazy loads the component chunk only when needed, and Suspense displays fallback UI while that chunk is loading.

One-Line Interview Answer

Use React.lazy() for dynamic component import and Suspense for loading UI while the split chunk downloads.
