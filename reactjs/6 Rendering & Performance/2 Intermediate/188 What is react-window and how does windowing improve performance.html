188. What is react-window and how does windowing improve performance?

One-Line Definition

react-window is a lightweight React library that implements list virtualization (windowing) by rendering only the visible items instead of the entire list.

What is react-window?

react-window is a performance optimization library created by:

:contentReference[oaicite:0]{index=0}

It is used when we need to display:

  • Thousands of products
  • Large tables
  • Chat messages
  • Transaction history
  • Log files
  • Admin dashboards

Without rendering thousands of DOM elements.

Install

npm install react-window

The Problem

Suppose we have: 

50,000 Products

and render them normally: 

products.map(product => (

  <ProductCard
    key={product.id}
    product={product}
  />

));

React creates: 


50,000 Components
50,000 DOM Elements

Even though the user can only see a few products on the screen.

Most of the rendered DOM elements are invisible but still consume memory and rendering time.

What is Windowing?

Windowing means rendering only the visible part (window) of a large list.

Instead of: 


1
2
3
4
...
50000

React renders only: 


101
102
103
104
105
106
107
108

which are currently visible.

Only visible items are rendered into the DOM.

How Windowing Improves Performance

50,000 Items
↓
User Can See Only 10 Items
↓
Render Only 10 Items
↓
Smaller DOM
↓
Faster Application

Real-World Example

import { FixedSizeList } from 'react-window';

const products = Array.from(
  { length: 50000 },
  (_, index) => ({
    id: index + 1,
    name: `Product ${index + 1}`,
    price: Math.floor(Math.random() * 10000)
  })
);

function ProductRow({ index, style }) {

  const product = products[index];

  return (

    <div
      style={{
        ...style,
        padding: '10px',
        borderBottom: '1px solid #ddd',
        display: 'flex',
        justifyContent: 'space-between',
        alignItems: 'center'
      }}
    >

      <span>{product.name}</span>

      <strong>₹{product.price}</strong>

    </div>

  );

}

function App() {

  return (

    <FixedSizeList
      height={500}
      width={800}
      itemCount={products.length}
      itemSize={60}
    >

      {ProductRow}

    </FixedSizeList>

  );

}

What Happens Internally?

Suppose: 

height={500}

itemSize={60}

react-window calculates: 


500 / 60
≈ 8 Rows Visible

Instead of rendering: 


50,000 Rows

it renders only: 


8 - 15 Rows
The DOM stays small no matter how large the dataset becomes.

How Can The Scrollbar Represent 50,000 Rows?

This is the trick many interviewers ask about.

react-window creates a huge virtual container.

Example: 


itemCount = 50000

itemSize = 60

Virtual height: 


50000 × 60

=

3,000,000px

The browser sees: 


height: 3,000,000px

and displays a huge scrollbar.

However, only visible rows are actually rendered.

Why Is style Required?

react-window passes a style prop: 

function ProductRow({ index, style })

Example style: 

{
  position: 'absolute',
  top: 7200,
  height: 60,
  width: '100%'
}

This positions rows correctly inside the virtual container.

Without: 

style={style}

all rows would overlap.

The style prop is mandatory when using react-window.

Important Interview Points

  • react-window implements list virtualization.
  • Also called windowing.
  • Renders only visible items.
  • Creates a large virtual container.
  • Uses absolute positioning.
  • Keeps DOM size small.
  • Improves rendering and scrolling performance.
  • Common alternatives: react-virtualized, TanStack Virtual.

Short Interview Answer

react-window is a virtualization library that renders only the visible portion of a large list. Windowing improves performance by reducing the number of DOM elements, lowering memory usage, and keeping scrolling smooth even with very large datasets.

One-Line Interview Answer

react-window improves performance by rendering only visible list items instead of the entire dataset.
