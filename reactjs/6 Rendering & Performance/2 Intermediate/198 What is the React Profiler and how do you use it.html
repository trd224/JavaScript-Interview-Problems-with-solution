198. What is the React Profiler and how do you use it?

One-Line Definition

React Profiler is a performance analysis tool that helps us identify which components are rendering, how often they render, and how long each render takes.

Why Is React Profiler Needed?

Sometimes a React application feels slow.

The problem is usually not React itself.

The problem is often:

  • Unnecessary re-renders.
  • Heavy calculations during rendering.
  • Large component trees.
  • Incorrect use of state.
  • Missing memoization.
Without profiling, it is difficult to know which component is causing the slowdown.

What Does React Profiler Show?

Information Description
Render Count How many times a component rendered.
Render Duration How long the render took.
Component Tree Which components rendered.
Commit Time How long React took to update the UI.
Render Reason Why the component rendered.

Method 1 - Using React DevTools Profiler

Step 1 - Install React DevTools

Install the React Developer Tools browser extension.

Step 2 - Open DevTools

Open browser DevTools and click the Profiler tab.

Step 3 - Start Recording

Click: 

Start Profiling

Then interact with your application.

  • Click buttons.
  • Type in forms.
  • Navigate pages.
  • Open modals.

Step 4 - Stop Recording

Click: 

Stop Profiling

React DevTools will display render information.

Now you can see exactly which components rendered and how much time they consumed.

Example Problem

function Parent() {

  const [count, setCount] = useState(0);

  return (
    <>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>

      <Child />
    </>
  );
}

Every time Parent renders: 

Child also renders.

Profiler helps us detect this behavior.

Profiler Result

Component Render Count
Parent 10
Child 10

Even if Child does not use count, it still re-renders.

This is a common performance issue found using Profiler.

Optimization Using React.memo

const Child = React.memo(function Child() {

  console.log('Child Rendered');

  return <h2>Child Component</h2>;

});

After optimization:

Component Render Count
Parent 10
Child 1
Profiler confirms that the optimization worked.

Method 2 - Using the Profiler Component

React also provides a special Profiler component. 

import { Profiler } from 'react';

function onRender(id, phase, actualDuration) {
  console.log(id, phase, actualDuration);
}

function App() {
  return (
    <Profiler id="Dashboard" onRender={onRender}>
      <Dashboard />
    </Profiler>
  );
}

How It Works

Parameter Meaning
id Profiler identifier.
phase Mount or update.
actualDuration Time spent rendering.

Example Output

Dashboard mount 8.3
Dashboard update 2.1
Dashboard update 1.7

This shows exactly how long each render took.

Flamegraph View

React DevTools provides a Flamegraph.

The Flamegraph helps us see:

  • Which components rendered.
  • Which components were slow.
  • Where rendering time was spent.
Large bars usually indicate components that consume more rendering time.

Ranked View

Profiler also provides a Ranked View.

This sorts components by render cost.

Component Render Time
ChartDashboard 42ms
UserTable 18ms
Header 1ms

This makes performance bottlenecks easy to identify.

Real Project Example

Suppose our Aadhaar Authentication page feels slow while typing.

Using React Profiler we may discover:

  • Typing in Aadhaar input causes 25 components to re-render.
  • Consent section re-renders unnecessarily.
  • Device information panel re-renders unnecessarily.
  • Large validation functions run on every keystroke.

After applying:

  • React.memo
  • useMemo
  • useCallback
  • Component splitting

Profiler can verify the improvement.

When Should You Use React Profiler?

  • Application feels slow.
  • Large component trees.
  • Frequent re-renders.
  • Performance optimization work.
  • Before and after performance comparison.

Important Interview Points

  • React Profiler is a performance debugging tool.
  • It shows render counts and render duration.
  • Available inside React DevTools.
  • Can identify unnecessary re-renders.
  • Helps validate React.memo, useMemo, and useCallback optimizations.
  • Provides Flamegraph and Ranked views.
  • React also provides a <Profiler> component API.

Short Interview Answer

React Profiler is a performance analysis tool that measures component render times and render frequency. It is commonly used through React DevTools to identify slow or unnecessary re-renders and verify performance optimizations such as React.memo, useMemo, and useCallback.

One-Line Interview Answer

React Profiler helps identify slow and unnecessary component renders by measuring render frequency and render duration.
