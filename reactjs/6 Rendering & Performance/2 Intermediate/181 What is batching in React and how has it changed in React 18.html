181. What is batching in React and how has it changed in React 18?

One-Line Definition

Batching means React groups multiple state updates together and performs only one re-render instead of rendering after every state update.

Why Is Batching Needed?

Every state update can cause a component to re-render.

Example: 

setCount(c => c + 1);

setName('Rizwan');

If React renders after every state update, the UI does extra work.

Without batching, multiple state updates can cause multiple unnecessary renders.

What Batching Does

Instead of rendering like this:

setCount()
↓
Render #1

setName()
↓
Render #2

React batches the updates like this:

setCount()
+
setName()
↓
One Render
Batching improves performance because React renders fewer times.

Batching Before React 18

Before React 18, React already supported batching, but only inside React event handlers.

React Event Example

function handleClick() {

  setCount(c => c + 1);

  setName('Rizwan');

}

In React 17, this already produced:

Click Event
↓
setCount()
+
setName()
↓
1 Render
React 17 already batched state updates inside React events like onClick and onChange.

Where React 17 Did Not Batch

React 17 did not automatically batch updates inside asynchronous code.

Examples:

  • setTimeout
  • Promise
  • fetch callback
  • async / await
  • native event listener

setTimeout Example in React 17

setTimeout(() => {

  setCount(c => c + 1);

  setName('Rizwan');

}, 1000);

React 17 result:

setCount()
↓
Render #1

setName()
↓
Render #2
In React 17, updates inside setTimeout caused separate renders.

Batching In React 18

React 18 introduced automatic batching almost everywhere.

Now React batches updates inside:

  • React event handlers
  • setTimeout
  • Promises
  • fetch callbacks
  • async / await
  • native event listeners

Same setTimeout Example in React 18

setTimeout(() => {

  setCount(c => c + 1);

  setName('Rizwan');

}, 1000);

React 18 result:

setCount()
+
setName()
↓
1 Render
React 18 automatically batches these updates, even though they are inside setTimeout.

Promise / Fetch Example

fetch('/api/users')

  .then(() => {

    setCount(c => c + 1);

    setName('Rizwan');

  });
Version Result
React 17 2 renders
React 18 1 render

Simple Difference

Where State Updates Happen React 17 React 18
React click event 1 render 1 render
React onChange event 1 render 1 render
setTimeout 2 renders 1 render
Promise / fetch 2 renders 1 render
async / await 2 renders 1 render
Native event listener 2 renders 1 render

Important Correction

React 18 did not introduce batching for React events. React events were already batched before React 18. React 18 expanded batching to async code like setTimeout, Promise, fetch, async/await, and native event listeners.

Can We Force React To Render Immediately?

Yes. React provides: 

flushSync()

Example: 

import { flushSync } from 'react-dom';

flushSync(() => {
  setCount(c => c + 1);
});

flushSync(() => {
  setName('Rizwan');
});

This forces React to update immediately instead of batching.

flushSync should be used rarely because it disables batching for that update.

Easy Way To Remember

React 17
↓
Batching only inside React events

React 18
↓
Batching almost everywhere

Important Interview Points

  • Batching means grouping multiple state updates into one render.
  • React 17 batched updates inside React event handlers.
  • React 18 batches updates almost everywhere.
  • React 18 supports batching inside timers, promises, fetch, async/await, and native events.
  • Automatic batching reduces unnecessary renders.
  • flushSync can be used to force immediate rendering.

Short Interview Answer

Batching in React means grouping multiple state updates into a single render. Before React 18, batching mainly worked inside React event handlers. In React 18, automatic batching works almost everywhere, including setTimeout, promises, fetch callbacks, async functions, and native event listeners.

One-Line Interview Answer

React 18 expanded batching from only React events to almost all update sources, reducing unnecessary renders automatically.
