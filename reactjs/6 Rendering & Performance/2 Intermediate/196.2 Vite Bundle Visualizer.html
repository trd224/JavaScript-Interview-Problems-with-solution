Vite Bundle Visualizer (Complete Beginner Guide)

One-Line Definition

Vite Bundle Visualizer is a plugin that visually displays what is included in our Vite production bundle and how much space each file or dependency occupies.

Why Do We Need It?

When our Vite application grows, the production bundle may become large. The visualizer helps us identify large libraries, duplicate code, and optimization opportunities.

A large bundle increases download, parsing, and execution time.

How It Works

Create Production Build
↓
Plugin Reads Bundle
↓
Generate Interactive Treemap
↓

↓
Optimize Bundle

Step 1 - Install

npm install --save-dev rollup-plugin-visualizer

Step 2 - Configure Vite

import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import { visualizer } from 'rollup-plugin-visualizer';

export default defineConfig({
  plugins: [
    react(),
    visualizer({
      open: true,
      filename: 'stats.html',
      gzipSize: true,
      brotliSize: true
    })
  ]
});

Step 3 - Build

npm run build

After the build finishes, a stats.html report is generated and automatically opens in the browser.

Example

import axios from 'axios';
import _ from 'lodash';
import Chart from 'chart.js';
import moment from 'moment';

The visualizer may show:

Chart.js (520 KB)
Moment (330 KB)
Lodash (480 KB)
Axios (32 KB)

How to Reduce Bundle Size

  • Remove unused dependencies.
  • Use dynamic imports.
  • Use React.lazy().
  • Replace heavy libraries.
  • Enable tree shaking.
  • Import only required functions.

Dynamic Import Example

const Reports = React.lazy(() => import('./Reports'));
The Reports component is downloaded only when required, reducing the initial bundle size.
