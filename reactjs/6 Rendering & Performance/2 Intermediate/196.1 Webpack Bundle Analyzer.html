Webpack Bundle Analyzer (Complete Beginner Guide)

One-Line Definition

Webpack Bundle Analyzer is a tool that visually shows what is inside our JavaScript bundle and helps identify large files and dependencies.

Why Do We Need It?

After running a production build, we may get a large bundle.js file but not know what makes it large. Bundle Analyzer displays every package as a rectangle, where bigger rectangles represent larger bundle size.

Example Problem

bundle.js (3.1 MB)
Without Bundle Analyzer we cannot easily identify which library is increasing the bundle size.

Step 1 - Install

npm install --save-dev webpack-bundle-analyzer

Step 2 - Configure Webpack

Where do we configure it?

If you have a webpack.config.js file, add the Bundle Analyzer plugin there. If you are using Create React App (CRA), Webpack configuration is hidden by default, so this code cannot be added directly unless you eject or use tools like CRACO/react-app-rewired.
Project TypeConfiguration FileCan Use This Plugin?
Custom Webpackwebpack.config.js✅ Yes
Create React App (CRA)Webpack hidden⚠️ Not directly
Vitevite.config.js❌ Use Vite Bundle Visualizer
Next.jsnext.config.js⚠️ Configure through Next.js

Example: webpack.config.js

const path = require('path');
const { BundleAnalyzerPlugin } = require('webpack-bundle-analyzer');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist')
  },
  plugins: [
    new BundleAnalyzerPlugin()
  ]
};

Step 3 - Build

npm run build

The browser automatically opens a report (usually on port 8888) showing a treemap of the bundle.

How It Helps

  • Find large third-party libraries.
  • Verify tree shaking removed unused code.
  • Check code splitting and lazy loading.
  • Detect duplicate packages.
  • Optimize bundle size for faster loading.

Tree Shaking Example

import { add } from './math';

If only add() is used, unused exports can be removed in the production build.

Lazy Loading Example

const Dashboard = React.lazy(() => import('./Dashboard'));
Bundle Analyzer will show separate chunks such as main.js and dashboard.chunk.js, confirming code splitting works.
