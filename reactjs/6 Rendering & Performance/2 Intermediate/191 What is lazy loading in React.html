191. What is Lazy Loading in React?

One-Line Definition

Lazy Loading is a technique where React loads a component only when it is actually needed instead of loading it during the initial page load.

Why Is It Needed?

Suppose our application contains:

  • Dashboard
  • Products Page
  • Reports Page
  • Settings Page
  • Admin Panel

Without lazy loading: 


All Components
↓
Loaded Immediately
↓
Large Bundle
↓
Slow Initial Load
The user downloads code for pages they may never visit.

How Lazy Loading Helps

With lazy loading:

Open Dashboard
↓
Load Dashboard Code Only

Visit Products Page
↓
Load Products Code

Visit Reports Page
↓
Load Reports Code
Components are downloaded only when they are needed.

Normal Import

Without lazy loading: 

import Dashboard from './Dashboard';

Dashboard becomes part of the main bundle.

Lazy Import

With lazy loading: 

const Dashboard = React.lazy(() =>
  import('./Dashboard')
);

Dashboard is downloaded only when React renders it.

Using Suspense

React.lazy must be wrapped inside Suspense. 

import React, { Suspense } from 'react';

const Dashboard = React.lazy(() =>
  import('./Dashboard')
);

function App() {

  return (

    <Suspense fallback={<p>Loading...</p>}>

      <Dashboard />

    </Suspense>

  );

}

While Dashboard is downloading: 


Loading...

is displayed.

How React.lazy Works Internally

User Requests Component
↓
Dynamic import()
↓
Download Component Chunk
↓
Render Component

Real Route-Based Example

import { BrowserRouter, Routes, Route } from 'react-router-dom';
import React, { Suspense } from 'react';

const Dashboard = React.lazy(() =>
  import('./pages/Dashboard')
);

const Products = React.lazy(() =>
  import('./pages/Products')
);

function App() {

  return (

    <BrowserRouter>

      <Suspense fallback={<h2>Loading...</h2>}>

        <Routes>

          <Route
            path="/"
            element={<Dashboard />}
          />

          <Route
            path="/products"
            element={<Products />}
          />

        </Routes>

      </Suspense>

    </BrowserRouter>

  );

}

What Happens Here?

User Opens


/

React downloads: 


Dashboard Chunk

User Navigates To


/products

React downloads: 


Products Chunk

for the first time.

Only required page code is downloaded.

Lazy Loading vs Code Splitting

Code Splitting Lazy Loading
Creates separate chunks. Loads chunks when needed.
Build-time optimization. Runtime optimization.
Uses dynamic import(). Uses React.lazy().
Splits bundle. Delays downloading.

They usually work together.

Real-World Example

Imagine: 


Dashboard = 200 KB

Products = 300 KB

Reports = 500 KB

Admin = 700 KB

Without lazy loading: 


1.7 MB Downloaded Immediately

With lazy loading: 


Dashboard Only

=

200 KB Initially
Much faster initial page load.

Common Use Cases

  • Route-based pages.
  • Admin panels.
  • Charts and dashboards.
  • Heavy third-party libraries.
  • Modals opened rarely.
  • Settings screens.

Easy Way To Remember

Code Splitting
↓
Create Multiple Chunks

Lazy Loading
↓
Load Chunk Only When Needed

Important Interview Points

  • Lazy loading delays component loading.
  • Implemented using React.lazy().
  • Requires Suspense.
  • Uses dynamic import().
  • Improves initial page load performance.
  • Often used with route-based code splitting.
  • Reduces initial bundle size.

Short Interview Answer

Lazy Loading is a technique where React loads components only when they are needed. It is implemented using React.lazy and Suspense, helping reduce initial bundle size and improve application performance.

One-Line Interview Answer

Lazy Loading delays downloading a component until it is actually required by the user.
