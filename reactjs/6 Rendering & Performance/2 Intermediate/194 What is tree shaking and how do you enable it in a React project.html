194. What is tree shaking and how do you enable it in a React project?

One-Line Definition

Tree shaking is a build optimization technique that removes unused JavaScript code from the final production bundle.

Simple Meaning

In our React project, we may create or import many functions, components, or library features.

But sometimes we use only a few of them.

Tree shaking removes the code that is imported nowhere or used nowhere.

Simple Example

utils.js

export function add(a, b) {
  return a + b;
}

export function subtract(a, b) {
  return a - b;
}

export function multiply(a, b) {
  return a * b;
}

App.js

import { add } from './utils';

function App() {
  return <h1>{add(10, 5)}</h1>;
}

export default App;

Here our application uses only add().

During production build, unused functions like subtract() and multiply() can be removed from the final bundle.

Why Is Tree Shaking Needed?

  • It reduces JavaScript bundle size.
  • It removes unused code.
  • It improves page loading speed.
  • It improves application performance.
  • It helps avoid sending unnecessary JavaScript to the browser.

How Tree Shaking Works

Source Code
↓
Bundler Checks Imports and Exports
↓
Finds Unused Code
↓
Removes Unused Code
↓
Creates Smaller Production Bundle

Important Requirement

Tree shaking works best with ES Modules, meaning import and export syntax.

Good Syntax

import { add } from './utils';

export function add(a, b) {
  return a + b;
}

Avoid CommonJS For Tree Shaking

const utils = require('./utils');

CommonJS is harder for bundlers to analyze statically, so tree shaking may not work properly.

How to Enable Tree Shaking in React?

1. Use Production Build

npm run build

Tree shaking usually runs automatically during the production build.

2. Use ES Module Import Export

import { add } from './utils';

export function add(a, b) {
  return a + b;
}

3. Import Only What You Need

Good:

import { debounce } from 'lodash-es';

Avoid:

import _ from 'lodash';

4. Use Tree-Shakable Libraries

Some libraries are better for tree shaking because they provide ES module builds.

Library Style Tree Shaking Support
ES Module library Good
CommonJS library Limited
Large full import Bad for bundle size

5. Avoid Unnecessary Side Effects

A side effect means code that runs automatically when a file is imported. 

console.log('This file runs immediately');

Bundlers may keep such files because removing them can change application behavior.

React Project Example

icons.js

export function UserIcon() {
  return <span>👤</span>;
}

export function CartIcon() {
  return <span>🛒</span>;
}

export function SearchIcon() {
  return <span>🔍</span>;
}

App.js

import { UserIcon } from './icons';

function App() {
  return <UserIcon />;
}

export default App;
Only UserIcon is used. CartIcon and SearchIcon can be removed from the final production bundle.

Tree Shaking in Vite or Create React App

Modern React build tools like Vite and Create React App already support tree shaking in production builds. 

npm run build

The important thing is to write code in a tree-shakable way.

Wrong Practice

import * as utils from './utils';

console.log(utils.add(10, 5));

This may make optimization harder in some cases because the bundler has to keep more information about the imported module.

Better Practice

import { add } from './utils';

console.log(add(10, 5));
Import only what our application actually needs.

Real Project Use Case

Suppose our React application uses an icon library with thousands of icons.

But our page uses only one icon. 

import { FaUser } from 'react-icons/fa';

function Profile() {
  return <FaUser />;
}

export default Profile;

Tree shaking helps keep unused icons out of the final bundle.

Important Interview Points

  • Tree shaking removes unused JavaScript code.
  • It works during production build.
  • It requires static ES module syntax.
  • It reduces bundle size.
  • It improves loading performance.
  • Modern bundlers like Webpack, Rollup, Parcel, and Vite support it.
  • CommonJS is less tree-shakable than ES Modules.

Short Interview Answer

Tree shaking is a bundling optimization that removes unused code from the final JavaScript bundle. In React, it is enabled by using ES module import/export syntax and creating a production build with a modern bundler like Webpack or Vite.

One-Line Interview Answer

Tree shaking removes unused JavaScript from the production bundle and is enabled by using ES modules with a production build.
