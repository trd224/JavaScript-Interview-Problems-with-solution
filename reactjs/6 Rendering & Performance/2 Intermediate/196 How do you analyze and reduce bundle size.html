196. How do you analyze and reduce bundle size?

One-Line Definition

Bundle size analysis means checking which files, libraries, and code are making our final JavaScript bundle large, then removing or optimizing unnecessary code.

Simple Meaning

When we build a React application, all our JavaScript, components, libraries, CSS, and assets are bundled into production files.

If the bundle becomes too large, the browser needs more time to download, parse, and execute it.

Large bundle size can make our React app slow, especially on mobile devices or slow internet.

Why Do We Analyze Bundle Size?

  • To find large libraries.
  • To find unused code.
  • To detect duplicate packages.
  • To improve loading speed.
  • To reduce JavaScript sent to the browser.
  • To improve performance and user experience.

How To Analyze Bundle Size

1. Create a Production Build

npm run build

Always analyze the production build, not the development build.

2. Use Bundle Analyzer Tools

Tool Use
Webpack Bundle Analyzer Shows bundle contents visually
Source Map Explorer Shows which files take space in the bundle
Rollup Visualizer Useful for Vite/Rollup projects
Vite Bundle Visualizer Analyzes Vite production bundles

Example: Using Source Map Explorer

Install

npm install --save-dev source-map-explorer

Add Script

{
  "scripts": {
    "build": "react-scripts build",
    "analyze": "source-map-explorer 'build/static/js/*.js'"
  }
}

Run

npm run build
npm run analyze

This shows which packages and files are taking the most space.

Example: Using Webpack Bundle Analyzer

Install

npm install --save-dev webpack-bundle-analyzer

Use With Custom Webpack Setup

const { BundleAnalyzerPlugin } = require('webpack-bundle-analyzer');

module.exports = {
  plugins: [
    new BundleAnalyzerPlugin()
  ]
};

This creates a visual report of our bundle.

How To Reduce Bundle Size

1. Remove Unused Dependencies

npm uninstall unused-package

Check package.json and remove libraries that are no longer used.

2. Import Only What You Need

Bad:

import _ from 'lodash';

_.debounce(search, 300);

Better:

import debounce from 'lodash/debounce';

debounce(search, 300);

Or:

import { debounce } from 'lodash-es';
Smaller imports help reduce final bundle size.

3. Use Code Splitting

Code splitting means loading some code only when it is needed. 

import React, { Suspense } from 'react';

const Reports = React.lazy(() => import('./Reports'));

function App() {
  return (
    <Suspense fallback={<h2>Loading...</h2>}>
      <Reports />
    </Suspense>
  );
}

export default App;

Now the Reports component is loaded only when required.

4. Route-Based Code Splitting

import React, { Suspense } from 'react';
import { Routes, Route } from 'react-router-dom';

const Dashboard = React.lazy(() => import('./pages/Dashboard'));
const Products = React.lazy(() => import('./pages/Products'));
const Reports = React.lazy(() => import('./pages/Reports'));

function App() {
  return (
    <Suspense fallback={<h2>Loading...</h2>}>
      <Routes>
        <Route path="/" element={<Dashboard />} />
        <Route path="/products" element={<Products />} />
        <Route path="/reports" element={<Reports />} />
      </Routes>
    </Suspense>
  );
}

export default App;
Each route can get its own JavaScript chunk.

5. Enable Tree Shaking

Use ES module import/export syntax. 

import { formatDate } from './utils';

export function formatDate(date) {
  return date.toDateString();
}

Avoid: 

const utils = require('./utils');
Tree shaking removes unused exports from the final production bundle.

6. Replace Heavy Libraries

Heavy Library Possible Alternative
moment date-fns or dayjs
lodash full import lodash-es or single-function imports
large chart libraries load only chart pages lazily
large rich text editors load editor only when user opens editor page

7. Compress and Optimize Assets

  • Compress images.
  • Use modern formats like WebP or AVIF.
  • Lazy load images.
  • Use SVG icons carefully.
  • Avoid importing very large images directly into JavaScript.

8. Avoid Duplicate Packages

npm ls package-name

Sometimes two versions of the same package are installed.

Duplicate dependencies can increase bundle size unnecessarily.

9. Use Dynamic Import for Heavy Features

async function openEditor() {
  const editor = await import('./RichTextEditor');
  editor.showEditor();
}

The editor code loads only when the user needs it.

10. Remove Console Logs in Production

Console logs do not usually create a huge bundle, but removing them keeps production code cleaner. 

console.log('debug data');

Real Project Example

Suppose our dashboard page uses a large chart library.

Instead of loading the chart library in the main bundle: 

import ChartDashboard from './ChartDashboard';

We can lazy load it: 

import React, { Suspense } from 'react';

const ChartDashboard = React.lazy(() => import('./ChartDashboard'));

function Dashboard() {
  return (
    <Suspense fallback={<p>Loading charts...</p>}>
      <ChartDashboard />
    </Suspense>
  );
}

export default Dashboard;
Now chart-related JavaScript loads only when the dashboard chart component is needed.

Common Bundle Size Problems

Problem Solution
Large library imported fully Import only required functions
All routes loaded initially Use React.lazy route splitting
Unused dependencies Remove from package.json
Heavy chart/editor libraries Lazy load those features
Large images Compress and lazy load images
Duplicate packages Deduplicate dependencies

Important Interview Points

  • Bundle size should be analyzed using production builds.
  • Bundle analyzer tools show what is inside the final bundle.
  • Code splitting reduces initial JavaScript load.
  • Tree shaking removes unused exports.
  • Lazy loading loads code only when needed.
  • Heavy libraries should be replaced or loaded lazily.
  • Unused and duplicate dependencies should be removed.

Short Interview Answer

We analyze bundle size by creating a production build and using tools like Webpack Bundle Analyzer or Source Map Explorer. We reduce bundle size by removing unused dependencies, using tree shaking, importing only required code, applying route-based code splitting, lazy loading heavy components, and optimizing assets.

One-Line Interview Answer

Analyze bundle size with bundle analyzer tools and reduce it using tree shaking, code splitting, lazy loading, optimized imports, and removing unused dependencies.
