184. What is the startTransition API?

One-Line Definition

startTransition is a React 18 API that marks state updates as low-priority so urgent updates can happen first.

Why Was It Introduced?

Sometimes a user action triggers:

  • A fast update that should happen immediately.
  • A slow update that can wait a little.

Example:

  • User types in a search box.
  • React filters 10,000 products.

Updating the input is important.

Filtering thousands of products is less important.

Without startTransition, expensive rendering can make typing feel slow.

Example Without startTransition

function handleChange(e) {

  const value = e.target.value;

  setInput(value);

  setSearch(value);

}

React treats both updates with the same priority.

User Types
↓
Update Input
+
Filter Large List
↓
Possible UI Lag

Using startTransition

import { startTransition } from 'react';

function handleChange(e) {

  const value = e.target.value;

  setInput(value);

  startTransition(() => {

    setSearch(value);

  });

}

Now React treats: 

setInput(value)

as high priority.

And: 

setSearch(value)

as low priority.

The input stays responsive even if filtering is expensive.

How React Thinks

Urgent Update
↓
Update Input Immediately

Low Priority Update
↓
Filter List Later

The Most Important Interview Point

startTransition does NOT make code faster.

It changes priority, not speed.

The expensive work still happens.

React simply delays less important work so the UI remains responsive.

Real Example

function Search() {

  const [input, setInput] = useState('');

  const [search, setSearch] = useState('');

  function handleChange(e) {

    const value = e.target.value;

    setInput(value);

    startTransition(() => {

      setSearch(value);

    });

  }

}

Typing updates: 

input

immediately.

Filtering uses: 

search

which updates with lower priority.

Relationship With useTransition

There are two related APIs:

startTransition

import { startTransition } from 'react';

Only marks updates as low priority.

useTransition

const [isPending, startTransition] =
  useTransition();

Provides:

  • startTransition
  • isPending loading state

Example Using useTransition

const [isPending, startTransition] =
  useTransition();

startTransition(() => {

  setSearch(value);

});

Now we can show: 

{isPending && <p>Loading...</p>}

When Should You Use startTransition?

  • Large list filtering.
  • Heavy sorting operations.
  • Expensive rendering work.
  • Search interfaces.
  • Charts and dashboards.
  • Large table updates.

When Should You NOT Use It?

  • Button clicks that must update immediately.
  • Form validation.
  • Critical state updates.
  • Small lightweight renders.
Do not wrap every state update in startTransition.

Important Interview Points

  • Introduced in React 18.
  • Marks updates as low priority.
  • Improves UI responsiveness.
  • Works with Concurrent Features.
  • Does not make code faster.
  • Helps with expensive rendering.
  • Related to useTransition.

Short Interview Answer

startTransition is a React 18 API that marks state updates as non-urgent. React prioritizes user interactions first and processes transition updates later to keep the UI responsive.

One-Line Interview Answer

startTransition tells React that a state update is low priority and can be delayed in favor of more important updates.
