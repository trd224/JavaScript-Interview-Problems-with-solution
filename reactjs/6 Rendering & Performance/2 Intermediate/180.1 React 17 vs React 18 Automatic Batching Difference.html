180. React 17 vs React 18 Automatic Batching Difference

One-Line Definition

Automatic batching means React groups multiple state updates together and performs only one re-render.

Your Main Confusion

You asked:

If batching already happened before React 18, then what is the difference between React 17 and React 18?

The answer is simple:

The difference is WHERE batching works.

React 17 Batching

In React 17, batching worked only inside React event handlers.

Example - React Click Event

function handleClick() {

  setCount(c => c + 1);

  setName('Rizwan');

}

Both state updates happen inside the same React click event.

React 17 batches them.

Click Event
↓
setCount()
+
setName()
↓
1 Render
React 17 already batched updates inside React events.

Where React 17 Did NOT Batch

React 17 did not automatically batch updates inside asynchronous code.

Examples:

  • setTimeout
  • Promise
  • fetch callback
  • async / await
  • native event listener

Example - setTimeout in React 17

setTimeout(() => {

  setCount(c => c + 1);

  setName('Rizwan');

}, 1000);

React 17 result:

setCount()
↓
Render #1

setName()
↓
Render #2
In React 17, this caused 2 renders because it was outside a React event handler.

React 18 Automatic Batching

In React 18, batching works almost everywhere.

The same setTimeout example: 

setTimeout(() => {

  setCount(c => c + 1);

  setName('Rizwan');

}, 1000);

React 18 result:

setCount()
+
setName()
↓
1 Render
React 18 automatically batches state updates even inside setTimeout, Promise, async functions, and native events.

Promise Example

fetch('/api/users')

  .then(() => {

    setCount(c => c + 1);

    setName('Rizwan');

  });

React 17

setCount()
↓
Render #1

setName()
↓
Render #2

React 18

setCount()
+
setName()
↓
1 Render

Async / Await Example

async function loadData() {

  await fetch('/api/users');

  setCount(c => c + 1);

  setName('Rizwan');

}

React 17

After await, React 17 treats the updates separately, so it may render twice.

React 18

React 18 batches both updates and renders once.

Simple Comparison Table

Place Where State Updates Happen React 17 React 18
React click event 1 render 1 render
React onChange event 1 render 1 render
setTimeout 2 renders 1 render
Promise / fetch 2 renders 1 render
async / await 2 renders 1 render
Native event listener 2 renders 1 render

Important Correction

React 18 did not introduce batching for React events. React events were already batched before React 18. React 18 expanded batching to asynchronous code like setTimeout, promises, fetch, async/await, and native events.

Short Interview Answer

In React 17, batching happened only inside React event handlers. In React 18, automatic batching also works inside setTimeout, promises, async functions, fetch callbacks, and native event listeners.

One-Line Interview Answer

React 17 batches updates only in React events, while React 18 automatically batches updates almost everywhere.
