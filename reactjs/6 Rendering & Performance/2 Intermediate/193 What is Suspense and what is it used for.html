193. What is Suspense and what is it used for?

One-Line Definition

Suspense is a React component that displays a fallback UI while waiting for something to finish loading.

Why Was Suspense Introduced?

Sometimes React cannot render a component immediately because it is still waiting for:

  • Lazy-loaded components
  • Data fetching (modern React)
  • Server Components (Next.js App Router)

Without Suspense:

Component Needed
↓
Still Loading
↓
Blank Screen

React would have nothing to show.

Users may see an empty page while content is loading.

How Suspense Helps

Suspense allows us to show temporary UI.

Component Loading
↓
Show Fallback UI
↓
Component Ready
↓
Show Actual Component
Users see a loading state instead of a blank screen.

Basic Syntax

import { Suspense } from 'react';

<Suspense fallback={<p>Loading...</p>}>

  <MyComponent />

</Suspense>

While: 

<MyComponent />

is loading, React shows: 

Loading...

Most Common Use Case - React.lazy

Suspense is most commonly used with: 

React.lazy()

Lazy Component

const Dashboard = React.lazy(() =>
  import('./Dashboard')
);

Using Suspense

import { Suspense } from 'react';

<Suspense fallback={<h2>Loading Dashboard...</h2>}>

  <Dashboard />

</Suspense>

React shows: 

Loading Dashboard...

while downloading the Dashboard chunk.

What Happens Internally?

React Tries To Render Dashboard
↓
Dashboard Chunk Not Downloaded Yet
↓
Suspense Shows Fallback
↓
Chunk Downloads
↓
Dashboard Renders

Real Route Example

const Products = React.lazy(() =>
  import('./pages/Products')
);

<Suspense fallback={<p>Loading Products...</p>}>

  <Products />

</Suspense>

When the user visits: 

/products

React downloads: 

products.chunk.js

and displays the fallback while downloading.

Suspense Does NOT Load Anything

Many developers think: 

Suspense loads components.

This is incorrect.

Suspense only shows fallback UI.

The actual loading is done by:

  • React.lazy()
  • Data fetching APIs
  • Server Components

Easy Analogy

Imagine ordering food in a restaurant.

Order Food
↓
Food Not Ready Yet
↓
Waiter Gives Water
↓
Food Arrives

The waiter is like: 

Suspense

The water is: 

fallback UI

The food is: 

actual component

Custom Fallback UI

Fallback can be anything.

Text

fallback={<p>Loading...</p>}

Spinner

fallback={<Spinner />}

Skeleton Loader

fallback={<ProductSkeleton />}

Nested Suspense

We can have multiple Suspense boundaries. 

<Suspense fallback={<PageLoader />}>

  <Dashboard>

    <Suspense fallback={<ChartLoader />}>
      <SalesChart />
    </Suspense>

  </Dashboard>

</Suspense>

Different parts of the page can load independently.

React 18 and Concurrent Features

Suspense became much more powerful in React 18.

It now works closely with:

  • Concurrent Rendering
  • Transitions
  • Streaming SSR
  • Server Components

Next.js App Router

In Next.js App Router, Suspense is heavily used. 

<Suspense fallback={<Loading />}>

  <ProductsPage />

</Suspense>

This allows parts of the page to stream to the browser as soon as they are ready.

Suspense vs React.lazy

React.lazy Suspense
Loads component dynamically. Shows fallback UI.
Creates code splitting. Handles waiting state.
Uses import(). Uses fallback prop.

Easy Way To Remember

React.lazy
↓
Downloads Component

Suspense
↓
Shows Loading UI

Component Ready
↓
Render Component

Important Interview Points

  • Suspense displays fallback UI while waiting.
  • Most commonly used with React.lazy.
  • Does not load components itself.
  • Uses the fallback prop.
  • Supports nested boundaries.
  • Important in React 18 and Next.js App Router.
  • Improves user experience during loading.

Short Interview Answer

Suspense is a React component that displays fallback UI while React waits for a lazy-loaded component, fetched data, or streamed content to become available. It improves the user experience by preventing blank screens during loading.

One-Line Interview Answer

Suspense shows a fallback UI while React waits for content to finish loading.
