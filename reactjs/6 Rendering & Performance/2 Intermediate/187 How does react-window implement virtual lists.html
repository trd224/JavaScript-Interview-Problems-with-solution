187. How does react-window implement virtual lists?

One-Line Definition

react-window implements virtual lists by rendering only the visible items and reusing DOM elements as the user scrolls.

The Main Problem

Suppose we have: 

50,000 Products

Without virtualization: 

products.map(product => (

  <ProductCard
    key={product.id}
    product={product}
  />

));

React creates: 

50,000 Components
50,000 DOM Elements

Even though the user can only see a few rows.

Most of those DOM elements are invisible but still consume memory and rendering time.

How react-window Solves It

Instead of rendering all items: 


1
2
3
4
...
50000

It renders only visible items.

For example: 


101
102
103
104
105
106
107
108
Only visible rows exist in the DOM.

The Secret Trick

Many developers ask:

If only 8 rows are rendered, how can the scrollbar represent 50,000 rows?

The answer is:

react-window creates a huge invisible container.

Example

<FixedSizeList
  height={500}
  itemSize={50}
  itemCount={50000}
/>

react-window calculates: 


50000 × 50
=
2,500,000px

Then it creates a container approximately: 


height: 2,500,000px

The browser thinks: 


Wow, this list is huge.

So it shows a huge scrollbar.

Visual Structure

Scrollable Container
↓
Height = 2,500,000px
↓
Huge Scrollbar
↓
Only Visible Rows Rendered

Suppose currently visible rows are:

User scrolls down.

Now visible rows become:

react-window:

  • Removes old rows.
  • Creates new rows.
  • Updates their positions.
Scroll
↓

↓
Create New Rows
↓
Keep DOM Small

The Row Component

function Row({ index, style }) {

  return (
    <div style={style}>
      Product {index}
    </div>
  );

}

Notice: 

style={style}

This is extremely important.

Why Does style Matter?

react-window sends: 

{
  position: 'absolute',
  top: 5000,
  left: 0,
  width: '100%',
  height: 50
}

for each row.

This tells the row where it should appear.

Without: 

style={style}

all rows would overlap each other.

The style prop is mandatory for virtualization to work correctly.

How Visible Rows Are Calculated

Suppose: 

height={500}

itemSize={50}

react-window calculates: 


500 / 50
=
10 rows

So roughly: 


10-15 rows

are rendered instead of: 


50,000 rows

Real Example

<FixedSizeList
  height={500}
  width={800}
  itemCount={50000}
  itemSize={50}
>

  {Row}

</FixedSizeList>

Internally:

50,000 Records
↓

↓

↓
Reuse DOM During Scroll

Important Interview Points

  • react-window uses list virtualization.
  • Only visible items are rendered.
  • Creates a large virtual container.
  • Uses absolute positioning.
  • Provides a style prop for each row.
  • Reuses DOM elements during scrolling.
  • Greatly improves performance for large lists.

Short Interview Answer

react-window implements virtual lists by creating a large virtual scrollable area while rendering only the rows visible on screen. As the user scrolls, old rows are removed and new rows are rendered, keeping the DOM small and performant.

One-Line Interview Answer

react-window renders only visible rows and reuses DOM elements while maintaining a large virtual scroll area.
