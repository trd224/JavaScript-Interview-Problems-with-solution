183. What is flushSync and when is it needed?

One-Line Definition

flushSync is a React API that forces React to immediately process state updates and update the DOM synchronously.

How Does React 18 work?

React waits until the current task finishes.

Then it combines all pending state updates and performs one render.

+
State Update
+

↓

Why Does flushSync Exist?

In React 18, automatic batching groups multiple state updates together.

React often delays rendering until the current task finishes.

Normally this is good because it improves performance.

Most of the time we want React to batch updates automatically.

However, sometimes we need the DOM updated immediately before running some code.

That is where flushSync is useful.

Importing flushSync

import { flushSync } from 'react-dom';

Normal React Behavior

function handleClick() {

  setCount(c => c + 1);

  console.log(count);

}

Many developers expect: 

1

But often see: 

0

because React schedules the update and renders later.

State updates are not applied immediately.

How flushSync Works

flushSync(() => {

  setCount(c => c + 1);

});

React immediately:

Run State Update
↓
Render Component
↓
Update DOM
↓
Continue Remaining Code

Simple Example

import { flushSync } from 'react-dom';

function handleClick() {

  flushSync(() => {

    setCount(c => c + 1);

  });

  console.log('DOM Updated');

}

The component is rendered before the code continues.

The Most Common Real-World Example

Suppose we open a chat message: 

setMessages(newMessages);

Then we want to scroll to the bottom. 

listRef.current.scrollTop = listRef.current.scrollHeight;

Problem:

The DOM may not have updated yet.

So the scroll height can be wrong.

Without flushSync

function addMessage() {

  setMessages(messages => [
    ...messages,
    'Hello'
  ]);

  listRef.current.scrollTop = listRef.current.scrollHeight;

}

Sometimes scrolling happens before React updates the DOM.

The new message may not be included in the height calculation.

With flushSync

function addMessage() {

  flushSync(() => {

    setMessages(messages => [
      ...messages,
      'Hello'
    ]);

  });

  listRef.current.scrollTop = listRef.current.scrollHeight;

}

Now React updates the DOM first.

Then scrolling happens.

The scroll calculation uses the latest DOM.

Understanding scrollTop and scrollHeight

The following line is what actually scrolls the container to the bottom: 

listRef.current.scrollTop = listRef.current.scrollHeight;

What does each part mean?

Property Description
listRef.current References the actual DOM element (for example, the messages container).
scrollTop The current vertical scroll position of the element.
scrollHeight The total height of all the content inside the element, including the part that is not currently visible.

When we write: 

listRef.current.scrollTop = listRef.current.scrollHeight;

we set the current scroll position equal to the total content height, causing the container to scroll all the way to the bottom.

Example

<div
  ref={listRef}
  style={{
    height: '150px',
    overflowY: 'auto'
  }}
>
  {/* Messages */}
</div>

Suppose:

  • Visible height (clientHeight) = 150px
  • Total content height (scrollHeight) = 800px

Initially: 

scrollTop = 0

After executing: 

listRef.current.scrollTop = listRef.current.scrollHeight;

The scroll position moves to the bottom of the container.

This is why chat applications automatically show the newest message whenever one is added.
Without flushSync, React may not have rendered the new message yet, so scrollHeight may still represent the old content height. Using flushSync ensures the DOM is updated before the scroll calculation runs.

Another Example - Measuring DOM Size

flushSync(() => {

  setOpen(true);

});

const height = modalRef.current.offsetHeight;

Without flushSync:

The modal may not exist in the DOM yet.

With flushSync:

The modal is guaranteed to be rendered before measuring.

React 18 automatically batches updates:

Multiple flushSync Calls

flushSync(() => {
  setCount(c => c + 1);
});

flushSync(() => {
  setName('Rizwan');
});

This causes:

flushSync #1
↓
Render #1

flushSync #2
↓

Batching is bypassed.

Too many flushSync calls can hurt performance.

When Should You Use flushSync?

  • Before measuring DOM size.
  • Before scrolling calculations.
  • Before focusing newly created elements.
  • When third-party libraries need the updated DOM immediately.
  • When code must run after React has already updated the DOM.

When Should You NOT Use flushSync?

  • For normal state updates.
  • To fix logic errors.
  • As a replacement for useEffect.
  • For routine rendering.
flushSync is an escape hatch and should be used rarely.

Important Interview Points

  • Imported from react-dom.
  • Forces synchronous rendering.
  • Bypasses automatic batching.
  • Useful for DOM measurements and scrolling.
  • Should be used sparingly.
  • Can reduce performance if overused.

Short Interview Answer

flushSync is a React API that forces React to immediately process state updates and update the DOM. It is mainly used when code depends on the DOM being updated right away, such as measuring element size or scrolling.

One-Line Interview Answer

flushSync forces React to render immediately instead of waiting for automatic batching.
