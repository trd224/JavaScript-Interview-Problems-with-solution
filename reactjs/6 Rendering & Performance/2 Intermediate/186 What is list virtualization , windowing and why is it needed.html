186. What is List Virtualization / Windowing and Why is it Needed?

One-Line Definition

List Virtualization (or Windowing) is a technique where React renders only the visible items on the screen instead of rendering the entire list.

Why Is It Needed?

Imagine we have: 

100,000 Products

and we render them like: 

products.map(product => (

  <ProductCard
    key={product.id}
    product={product}
  />

));

React creates: 

100,000 DOM Elements

even though the user can only see a small portion of the list.

Rendering thousands of DOM elements can make the application slow and consume a lot of memory.

The Real Problem

Suppose your screen shows only: 

20 Products

But React renders: 

100,000 Products

That means:

User Sees 20 Items
↓
React Renders 100,000 Items
↓
Huge DOM
↓
Slow Performance

How Virtualization Solves This

Instead of rendering all items:

...
100000

React renders only visible items.

For example:

Old items are removed and new items are rendered.

The important thing is that a single scroll does NOT replace the entire visible list. It only updates the items affected by the scroll.
The DOM stays small regardless of how many items exist.

Without Virtualization

products.map(product => (

With Virtualization

Only visible rows are rendered.

100,000 Items
↓

↓
Only 20 Components Rendered
↓
Fast UI

Popular Library - react-window

The most common virtualization library is: 

react-window

Install

npm install react-window

Example

import { FixedSizeList } from 'react-window';

const products = Array.from({ length: 50000 }, (_, index) => ({
  id: index + 1,
  name: `Product ${index + 1}`,
  price: Math.floor(Math.random() * 10000)
}));

function ProductRow({ index, style }) {
  const product = products[index];

  return (
    <div style={{ ...style, padding: '10px', borderBottom: '1px solid #ddd', display: 'flex', justifyContent: 'space-between', alignItems: 'center' }}>
      <span>{product.name}</span>
      <strong>₹{product.price}</strong>
    </div>
  );
}

export default function App() {
  return (
    <div>
      <h2>Products ({products.length})</h2>

      <FixedSizeList height={500} width={800} itemCount={products.length} itemSize={60}>
        {ProductRow}
      </FixedSizeList>

    </div>
  );
}

The style prop is provided automatically by react-window. You never create it yourself.

What Happens Internally?

Even though: 

itemCount={100000}

React does NOT render all 100,000 rows.

It renders only what fits inside: 

height={400}

Usually around: 

10 - 20 Rows
Only visible rows exist in the DOM.

Real-World Example

  • E-commerce product lists.
  • Large tables.
  • Data grids.
  • Chat applications.
  • Log viewers.
  • Admin dashboards.
  • Excel-like interfaces.

Virtualization vs Pagination

Pagination Virtualization
Loads data page by page. Loads visible items only.
User changes page. User scrolls naturally.
Good for APIs. Good for huge UI lists.
50 items/page. 100,000+ items possible.

Virtualization vs Infinite Scroll

Infinite Scroll Virtualization
Keeps adding items. Reuses DOM elements.
DOM grows continuously. DOM stays small.
Memory usage increases. Memory usage remains stable.

Important Interview Points

  • Also called Windowing.
  • Renders only visible items.
  • Reduces DOM size.
  • Improves performance.
  • Reduces memory usage.
  • Useful for large lists and tables.
  • Common libraries: react-window, react-virtualized.

Short Interview Answer

List Virtualization (Windowing) is a performance optimization technique where only the visible items in a large list are rendered. As the user scrolls, React removes old items and renders new ones, keeping the DOM small and fast.

One-Line Interview Answer

List Virtualization renders only visible list items instead of the entire list, improving performance for large datasets.
