189. How do you implement a virtualized list for rendering thousands of rows?

One-Line Definition

A virtualized list renders only the visible rows on the screen instead of rendering all rows at once.

Why Do We Need It?

Suppose we have 50,000 products.

If we render normally: 

products.map(product => (
  <div key={product.id}>
    {product.name}
  </div>
));

React creates 50,000 DOM elements.

This can make the first render slow, increase memory usage, and make scrolling laggy.

Install react-window

npm install react-window

Step 1 - Import FixedSizeList

import { FixedSizeList } from 'react-window';

We use FixedSizeList when every row has the same height.

Step 2 - Create Large Data

const products = Array.from(
  { length: 50000 },
  (_, index) => ({
    id: index + 1,
    name: `Product ${index + 1}`,
    price: Math.floor(Math.random() * 10000)
  })
);

Step 3 - Create Row Component

function ProductRow({ index, style }) {

  const product = products[index];

  return (

    <div
      style={{
        ...style,
        padding: '10px',
        borderBottom: '1px solid #ddd',
        display: 'flex',
        justifyContent: 'space-between',
        alignItems: 'center'
      }}
    >

      <span>{product.name}</span>

      <strong>₹{product.price}</strong>

    </div>

  );

}
The style prop is very important. react-window uses it to position each row correctly.

Step 4 - Render FixedSizeList

function App() {

  return (

    <div>

      <h2>Products ({products.length})</h2>

      <FixedSizeList
        height={500}
        width={800}
        itemCount={products.length}
        itemSize={60}
      >

        {ProductRow}

      </FixedSizeList>

    </div>

  );

}

export default App;

Complete Example

import { FixedSizeList } from 'react-window';

const products = Array.from(
  { length: 50000 },
  (_, index) => ({
    id: index + 1,
    name: `Product ${index + 1}`,

  const product = products[index];

  return (

    <div
      style={{

Meaning of FixedSizeList Props

Prop Meaning
height Visible height of the list container.
width Width of the list container.
itemCount Total number of rows in the list.
itemSize Height of each row.

How Many Rows Render?

In this example: 

height={500}
itemSize={60}

Visible rows: 

500 / 60 = around 8 rows

So React renders only around 8 to 15 rows instead of 50,000 rows.

This keeps the DOM very small and fast.

How Scrollbar Still Works?

react-window internally creates a large virtual space.

Example: 

itemCount = 50000
itemSize = 60

Total virtual height: 

50000 × 60 = 3,000,000px

Because of this huge virtual height, the browser shows a scrollbar for the full list.

3,000,000px Virtual Height
↓
Large Scrollbar
↓
Only Visible Rows Rendered

Why style={style} Is Required?

react-window passes a style object to every row. 

function ProductRow({ index, style })

Example: 

{
  position: 'absolute',
  top: 1200,
  height: 60,
  width: '100%'
}

This tells each row where it should appear.

So we must apply: 

style={style}
If we do not apply style={style}, rows can overlap or appear incorrectly.

What Happens While Scrolling?

User Scrolls
↓
react-window Calculates Visible Indexes
↓
Old Rows Removed
↓
New Rows Rendered
↓
DOM Remains Small

Without Virtualization vs With Virtualization

Without Virtualization With Virtualization
50,000 DOM elements. Only visible DOM elements.
Slow initial render. Fast initial render.
High memory usage. Low memory usage.
Laggy scrolling. Smooth scrolling.

Real-World Use Cases

  • Product catalog with thousands of products.
  • Admin dashboard tables.
  • Bank transaction history.
  • Chat messages.
  • Log viewer.
  • CRM customer list.
  • Excel-like grid interfaces.

Important Interview Points

  • Use react-window or react-virtualized.
  • Use FixedSizeList when rows have equal height.
  • Use VariableSizeList when rows have different heights.
  • Only visible rows are rendered.
  • The list still has full scroll height virtually.
  • The style prop from react-window must be applied to each row.
  • It improves performance by reducing DOM nodes.

Short Interview Answer

We can implement a virtualized list using react-window. It renders only the visible rows and creates a virtual scrollable area for the full list. This keeps the DOM small and improves performance when rendering thousands of rows.

One-Line Interview Answer

Use react-window's FixedSizeList to render only visible rows instead of thousands of DOM elements.
