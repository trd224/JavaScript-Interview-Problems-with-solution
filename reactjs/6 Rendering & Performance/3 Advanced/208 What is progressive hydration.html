208. What is Progressive Hydration?

One-Line Answer

Progressive Hydration means hydrating a server-rendered page gradually in smaller parts instead of hydrating the whole page at once.

First Understand Hydration

In SSR, the server sends HTML to the browser. The user can see the page quickly, but React events are not active until hydration happens.

Server Sends HTML
↓ Browser Shows Page
↓ JavaScript Downloads
↓ React Hydrates HTML
↓ Page Becomes Interactive
Hydration means React attaches JavaScript behavior like onClick, onChange, state updates, and effects to already-rendered HTML.

Problem With Hydrating Everything At Once

Imagine a large ecommerce page:

Navbar
Search
Filters
ProductList
Reviews
Comments
Footer

If React tries to hydrate everything together, the browser may become busy.

Big hydration work can delay interactivity. The page is visible, but buttons, inputs, and filters may not work immediately.

Progressive Hydration Idea

Hydrate Navbar
↓ Hydrate Search
↓ Hydrate Filters
↓ Hydrate ProductList
↓ Hydrate Comments
↓ Hydrate Footer
Instead of one big hydration job, the page becomes interactive step by step.

Simple Analogy

Without progressive hydration:

Build Whole House
↓ Then Allow Anyone Inside

With progressive hydration:

Finish Main Door
↓ People Can Enter
↓ Finish Living Room
↓ Finish Kitchen
↓ Finish Other Rooms

Real Example

Page.jsx
function Page(){
  return (
    <>
      <Navbar />
      <SearchBox />
      <ProductList />
      <Comments />
    </>
  );
}

On a big page, some parts are more important than others.

ComponentNeedPriority
NavbarUser may click immediatelyHigh
SearchBoxUser may type immediatelyHigh
ProductListImportant contentMedium/High
CommentsUsually used laterLow
FooterRarely urgentLow

Progressive Hydration Flow

HTML Visible
↓ JS Downloads
↓ Navbar Hydrates
↓ SearchBox Hydrates
↓ ProductList Hydrates
↓ Comments Hydrate Later

Relationship With Selective Hydration

Progressive hydration and selective hydration are related, but not exactly the same.

FeatureMeaning
Progressive HydrationHydrate page gradually in stages
Selective HydrationHydrate important or user-interacted parts first
Progressive Hydration = step-by-step hydration.

Selective Hydration = priority-based hydration.

Example Difference

Progressive Hydration

Navbar Hydrates
↓ Search Hydrates
↓ Products Hydrate
↓ Comments Hydrate

Selective Hydration

Hydration Already Running
↓ User Clicks Search
↓ React Prioritizes Search
↓ Search Hydrates First
↓ Comments Later
Progressive hydration is about gradual hydration. Selective hydration is about choosing what should hydrate first.

Important Correction From Previous Topic

Hydration starts automatically after JavaScript downloads. User interaction does not start hydration. User interaction can change priority in selective hydration.

Progressive Hydration And Suspense

Suspense boundaries help React split the page into smaller parts.

Suspense Example
function Page(){
  return (
    <>
      <Navbar />

      <Suspense fallback={<p>Loading products...</p>}>
        <ProductList />
      </Suspense>

      <Suspense fallback={<p>Loading comments...</p>}>
        <Comments />
      </Suspense>
    </>
  );
}
Suspense Boundary
↓ Separate Page Section
↓ Can Load/Hydrate Independently

Progressive Hydration And Streaming SSR

Progressive hydration is commonly discussed with Streaming SSR.

Server Streams HTML In Pieces
↓ Browser Shows Pieces Early
↓ JS Loads
↓ React Hydrates Available Parts
↓ More Parts Stream In
↓ More Parts Hydrate Later
Streaming SSR sends HTML progressively. Progressive hydration makes those parts interactive progressively.

Without Progressive Hydration

Full Page HTML
↓ Full JS
↓ Hydrate Everything
↓ Then Page Fully Interactive

With Progressive Hydration

Important HTML Visible
↓ Important JS Ready
↓ Important Area Hydrates
↓ User Can Interact
↓ Less Important Areas Hydrate Later

Why It Improves User Experience

  • Important areas become interactive earlier
  • Browser does not need to process one huge hydration task
  • Large SSR pages feel faster
  • Less important sections can wait
  • Works well with Suspense and streaming

Progressive Hydration vs Lazy Loading

ConceptMeaning
Lazy LoadingLoad code or data later
HydrationAttach React behavior to SSR HTML
Progressive HydrationAttach behavior gradually section by section
Lazy loading is about when code/data loads. Progressive hydration is about when server-rendered HTML becomes interactive.

Where Is It Useful?

  • Large ecommerce pages
  • News websites
  • Dashboards
  • Pages with comments/reviews below the fold
  • SSR apps using Suspense and streaming

Interview Answer

Progressive Hydration is an SSR performance technique where React hydrates a server-rendered page gradually in smaller parts instead of hydrating the whole page at once. The browser can show HTML first, then React attaches interactivity section by section. This improves perceived performance because important parts like navbar, search, or buttons can become interactive earlier, while less important sections like comments or footer hydrate later.

Easy Way To Remember

SSR HTML Visible
↓ JS Downloads
↓ Hydration Happens Step By Step
↓ Important Parts Become Interactive Earlier
↓ Less Important Parts Later
↓ Progressive Hydration

Final Difference

Progressive Hydration = hydrate gradually.

Selective Hydration = hydrate based on priority or user interaction.
