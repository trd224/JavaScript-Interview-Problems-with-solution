204. What is Time Slicing?

One-Line Answer

Time Slicing is a React Concurrent Rendering feature that allows React to break large rendering work into small chunks and pause between them so the browser can stay responsive.

Problem Before Time Slicing

Imagine React must render 10,000 items.

User Types
↓ React Starts Rendering
↓ Render 10,000 Items
↓ CPU Busy
↓ UI Freezes
↓ User Waits
In older React rendering was mostly synchronous. Once rendering started, React continued until all work finished.

With Time Slicing

Render 10,000 Items
↓ Do Small Chunk
↓ Pause
↓ Give Browser Time
↓ Continue Next Chunk
↓ Pause Again
↓ Continue Until Complete
The browser gets opportunities to handle clicks, typing, scrolling, and animations while React is rendering.

Simple Analogy

Without Time Slicing:

Read Entire 500 Page Book
↓ Then Answer Phone

With Time Slicing:

Read 10 Pages
↓ Check Phone
↓ Read 10 More Pages
↓ Check Phone Again

What React Actually Does

Update Starts
↓ React Creates Work
↓ Process Small Part
↓ Should Browser Run?
↓ Yes → Pause
↓ Browser Handles Input
↓ React Resumes Later

Real Example

Heavy Search Result Rendering
function Search() {
  const [query, setQuery] = useState('');

  const results = hugeList.filter(item => item.includes(query));

  return (
    <>
      <input
        value={query}
        onChange={e => setQuery(e.target.value)}
      />

      {results.map(item => (
        <div key={item}>{item}</div>
      ))}
    </>
  );
}

If thousands of results must render, React may split the rendering work into smaller pieces instead of blocking the browser.

Relationship With Concurrent Rendering

Concurrent Rendering
↓ Includes
↓ Time Slicing
↓ React Can Pause Work
↓ React Can Resume Work
↓ React Can Prioritize Important Updates

Relationship With useTransition

Example
const [isPending, startTransition] = useTransition();

function handleChange(e) {
  setInput(e.target.value);

  startTransition(() => {
    setResults(filterHugeList(e.target.value));
  });
}
When you use startTransition(), React may use Time Slicing to keep typing responsive while rendering large result lists.

Can Users See Partial UI?

Usually No.

React Builds Work
↓ May Pause Internally
↓ Continue Rendering
↓ Finally Commit UI
↓ User Sees Complete Update

Time Slicing happens during the render phase, not during the commit phase.

Without vs With Time Slicing

Without With
Long blocking render Work split into chunks
UI may freeze UI stays responsive
Cannot pause work Can pause and resume
User input delayed User input handled quickly

Interview Answer

Time Slicing is a React Concurrent Rendering capability that breaks large rendering work into smaller chunks. React can pause rendering, allow the browser to process user interactions such as typing or clicking, and then continue rendering later. This prevents the UI from freezing and makes applications feel more responsive.

Easy Way To Remember

Big Render Work
↓ Split Into Small Pieces
↓ Pause Between Pieces
↓ Browser Stays Responsive
↓ = Time Slicing
