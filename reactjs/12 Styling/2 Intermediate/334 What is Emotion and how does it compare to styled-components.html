334. What is Emotion and How Does it Compare to styled-components?

One-Line Definition

Emotion is a CSS-in-JS library for React that allows you to write styles inside JavaScript. It is very similar to styled-components but offers additional styling approaches and slightly better performance.

What is Emotion?

Emotion is a modern CSS-in-JS library that supports:

  • Styled Components API
  • css Prop API
  • Dynamic Styling
  • Theming
  • Server Side Rendering
  • TypeScript Support
If you know styled-components, learning Emotion takes only a few minutes because the syntax is almost identical.

Install Emotion

npm install @emotion/react @emotion/styled

Project Structure

src/
│
├── App.jsx
└── components/
    └── Button.jsx

Emotion Styled Component Example

File: src/components/Button.jsx
import styled from '@emotion/styled';

const Button = styled.button`
  background: royalblue;
  color: white;
  border: none;
  padding: 12px 20px;
  border-radius: 8px;
`;

export default Button;
File: src/App.jsx
import Button from './components/Button';

function App() {
  return <Button>Save</Button>;
}

export default App;

styled-components Version

File: src/components/Button.jsx
import styled from 'styled-components';

const Button = styled.button`
  background: royalblue;
  color: white;
`;

export default Button;
The syntax is almost identical.

↓

Unique Feature - css Prop

Emotion supports the css prop.

File: src/App.jsx
/** @jsxImportSource @emotion/react */

function App() {
  return (
    <button
      css={{
        background: 'royalblue',
        color: 'white',
        padding: '12px'
      }}
    >
      Save
    </button>
  );
}

export default App;
styled-components does not provide a built-in css prop like Emotion.

Dynamic Styling

File: src/components/Button.jsx
import styled from '@emotion/styled';

const Button = styled.button`
  background: ${props => props.primary ? 'royalblue' : 'gray'};
  color: white;
`;

export default Button;

Usage: 

<Button primary>Primary</Button>
<Button>Secondary</Button>

Theming Example

File: src/theme.js
export const theme = {
  colors: {
    primary: 'royalblue'
  }
};
File: src/App.jsx
import { ThemeProvider } from '@emotion/react';
import { theme } from './theme';

function App() {
  return (
    <ThemeProvider theme={theme}>
      <Button>Save</Button>
    </ThemeProvider>
  );
}

export default App;

.css-1abcde {

}

Advantages of Emotion

  • Supports styled API
  • Supports css prop
  • Smaller bundle size
  • Excellent TypeScript support
  • Good SSR support
  • Dynamic styling with props

Disadvantages of Emotion

  • Runtime styling cost
  • More JavaScript execution
  • Larger bundle than plain CSS
  • Still CSS-in-JS overhead

Real World Usage

Material UI (MUI)
↓
Uses Emotion By Default
If you work with MUI v5+, you are already using Emotion internally.

Current Industry Trend

Emotion and styled-components are still popular, but many modern projects now prefer Tailwind CSS, CSS Modules, or build-time CSS solutions because they avoid runtime styling costs.

When Should You Use Emotion?

  • Using Material UI.
  • Need css prop support.
  • Want styled-components style syntax.
  • Need excellent TypeScript support.

Interview Answer

Emotion is a CSS-in-JS library that allows developers to write styles directly in JavaScript. It provides a styled API similar to styled-components but also includes a powerful css prop API. Compared to styled-components, Emotion generally offers a smaller bundle size, slightly better performance, and tighter integration with libraries such as Material UI.

One-Line Interview Answer

Emotion is a CSS-in-JS library similar to styled-components, but it also supports a css prop API and is used internally by Material UI.
