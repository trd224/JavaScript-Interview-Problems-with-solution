333. What are the Pros and Cons of CSS-in-JS?

One-Line Definition

CSS-in-JS is a styling approach where CSS is written inside JavaScript files. It provides component-scoped styling and dynamic styles, but adds runtime overhead and larger bundle sizes.

Popular CSS-in-JS Libraries

  • styled-components
  • Emotion
  • JSS
  • Stitches
  • Linaria

How CSS-in-JS Works

Write CSS In JavaScript
↓
Library Generates CSS
↓
Creates Unique Class Names
↓
Injects CSS Into DOM
↓
Component Gets Styles

Example

File: src/components/Button.jsx
import styled from 'styled-components';

const Button = styled.button`
  background: royalblue;
  color: white;
  padding: 12px;
`;

export default Button;

Pros of CSS-in-JS

1. No Class Name Conflicts

Traditional CSS: 

.button {
  background: red;
}

Another developer: 

.button {
  background: green;
}

Conflict occurs.

CSS-in-JS generates unique class names automatically.

No naming collisions.

2. Component Scoped Styles

File: src/components/UserCard.jsx
const Card = styled.div`
  padding: 20px;
`;

Styles belong only to that component.

UserCard
↓
Own CSS
↓
No Global Pollution

3. Dynamic Styling With Props

File: src/components/Button.jsx
const Button = styled.button`
  background: ${props => props.primary ? 'royalblue' : 'gray'};
`;

Usage: 

<Button primary>Save</Button>
<Button>Cancel</Button>
Styles can change based on component props.

4. Easier Reusability

File: src/components/Button.jsx
export const Button = styled.button`
  padding: 12px;
`;

export const DangerButton = styled(Button)`
  background: crimson;
`;

Reuse styles by extending components.

5. Better Component Encapsulation

Component
↓
JS Logic
+
Styles
↓
Single File

Everything related to a component stays together.

Cons of CSS-in-JS

1. Runtime Performance Cost

Many CSS-in-JS libraries generate styles in the browser.

Render Component
↓
Generate CSS
↓
Inject Into DOM
↓
Extra Work
This can be slower than static CSS.

2. Larger Bundle Size

Extra styling library must be downloaded. 

React App
+
styled-components
=
Larger Bundle

3. More JavaScript Execution

Styles are now part of JavaScript.

Load JS
↓
Execute JS
↓
Generate CSS
↓
Render UI

4. Harder Browser DevTools Debugging

Generated class names may look like: 

sc-a1b2c3
sc-k8x4m9
sc-p7q9r2

Not as readable as: 

.user-card
.login-button

5. Less Popular Today

Many modern projects prefer:

  • Tailwind CSS
  • CSS Modules
  • Vanilla Extract
  • CSS Variables

Because they avoid runtime styling costs.

Current Industry Trend

Around 2018-2021, styled-components and Emotion were extremely popular. Today many teams prefer Tailwind CSS, CSS Modules, or build-time CSS solutions because they provide better performance and smaller bundles.

When Should You Use CSS-in-JS?

  • Need dynamic styling based on props.
  • Want component-scoped styles.
  • Already using styled-components or Emotion.
  • Medium-sized applications.

When Should You Avoid CSS-in-JS?

  • Performance-critical applications.
  • Very large applications.
  • Need smallest bundle size.
  • Using Tailwind or CSS Modules already.

Interview Answer

The main advantages of CSS-in-JS are component-scoped styles, automatic class name generation, dynamic styling through props, and improved reusability. The disadvantages are runtime performance overhead, larger bundle sizes, additional JavaScript execution, and increased complexity compared to static CSS solutions such as CSS Modules or Tailwind.

One-Line Interview Answer

CSS-in-JS provides scoped and dynamic styling, but it adds runtime overhead and larger bundle sizes.
