451. What is Web Accessibility and Why Does It Matter?

One-Line Definition

Web Accessibility (a11y) is the practice of designing and developing websites so that all people, including those with disabilities, can use them effectively.

What is Web Accessibility?

Accessibility means making our websites and web applications usable for everyone, regardless of physical, visual, hearing, cognitive, or motor limitations.

An accessible website allows users to:

  • Navigate using a keyboard.
  • Use screen readers.
  • Read content with sufficient color contrast.
  • Understand forms and error messages.
  • Access content without relying on a mouse.

Why is it Called a11y?

Accessibility → a11y

There are 11 letters between "a" and "y", so accessibility is often abbreviated as a11y.

User Type Benefit

Common Accessibility Problems

  • Images without alt text.
  • Buttons without labels.
  • Poor color contrast.
  • Forms without labels.
  • Keyboard-inaccessible menus.
  • Videos without captions.

Example: Image Accessibility

Bad

<img src="profile.jpg" />

Good

<img src="profile.jpg" alt="User profile picture" />
Screen readers read the alt text to visually impaired users.

Example: Form Accessibility

Bad

<input type="email" />

Good

<label htmlFor="email">

  Email Address

</label>

<input id="email" type="email" />

Example: Button Accessibility

Bad

<div onClick={saveData}>

  Save

</div>

Good

<button onClick={saveData}>

  Save

</button>
Native HTML elements provide built-in accessibility support.

Keyboard Accessibility

Many users cannot use a mouse and rely entirely on keyboard navigation.

Important Keys

Key Purpose
Tab Move to next element
Shift + Tab Move to previous element
Enter Activate button or link
Space Activate buttons and checkboxes

ARIA Attributes

ARIA (Accessible Rich Internet Applications) attributes provide additional accessibility information when native HTML is not enough. 

<button

  aria-label="Close modal"

>

  ×

</button>

Accessibility and React

React supports accessibility through standard HTML, ARIA attributes, keyboard handling, and semantic elements. 

<button

  aria-label="Submit Form"

>

  Submit

</button>

Real Project Example

In our Aadhaar Authentication application, accessibility can be improved by providing labels for form fields, keyboard-accessible modal popups, proper focus management, screen-reader-friendly error messages, and sufficient color contrast for validation messages.

Important Interview Points

  • Accessibility is often abbreviated as a11y.
  • It ensures websites are usable by people with disabilities.
  • Use semantic HTML whenever possible.
  • Provide alt text for images.
  • Ensure keyboard accessibility.
  • Use ARIA attributes when necessary.

Short Interview Answer

Web accessibility is the practice of building websites that can be used by everyone, including people with disabilities, through semantic HTML, keyboard support, screen reader compatibility, and proper design.

One-Line Interview Answer

Web accessibility ensures that websites and web applications can be used effectively by all users, including those with disabilities.
