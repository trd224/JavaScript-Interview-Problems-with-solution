224. How Do You Show a Loading State While Fetching Data?

One-Line Definition

A loading state is usually displayed by storing a loading flag in useState() and showing a loading message or spinner until the API response arrives.

Why Do We Need a Loading State?

API requests take time to complete.

Without a loading state, users may see an empty screen and think the application is broken.

A loading state informs users that data is currently being fetched.

How It Works

Component Mounts
↓
loading = true
↓
API Request Sent
↓
Response Received
↓
loading = false
↓
Show Data

Step 1 - Create a Loading State

const [loading, setLoading] = useState(true);
We initialize loading with true because the API request has not completed yet.

Step 2 - Update Loading During Fetch

import { useEffect, useState } from 'react';

function Users() {

  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {

    fetch('https://jsonplaceholder.typicode.com/users')
      .then(res => res.json())
      .then(data => {

        setUsers(data);

        setLoading(false);

      });

  }, []);

}

Step 3 - Display Loading UI

if (loading) {

  return (

    <p>

      Loading...

    </p>

  );

}

Complete Example

import { useEffect, useState } from 'react';

function Users() {

  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {

    fetch('https://jsonplaceholder.typicode.com/users')
      .then(res => res.json())
      .then(data => {

        setUsers(data);

        setLoading(false);

      });

  }, []);

  if (loading) {

    return (

      <p>

        Loading...

      </p>

    );

  }

  return (

    <ul>

      {users.map(user => (

        <li key={user.id}>

          {user.name}

        </li>

      ))}

    </ul>

  );

}

Using async/await

useEffect(() => {

  async function fetchUsers() {

    const response = await fetch(url);

    const data = await response.json();

    setUsers(data);

    setLoading(false);

  }

  fetchUsers();

}, []);

Loading + Error Handling

const [users, setUsers] = useState([]);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);

useEffect(() => {

  fetch(url)
    .then(res => res.json())
    .then(data => {

      setUsers(data);

      setLoading(false);

    })
    .catch(err => {

      setError(err.message);

      setLoading(false);

    });

}, []);

Why Do We Set Loading to False in Both Cases?

Whether the request succeeds or fails, loading must become false because the request has finished.

↓

↓

Important Interview Points

  • Loading state is usually managed using useState().
  • Initialize loading with true.
  • Set loading to false after the API request finishes.
  • Loading improves user experience.
  • Loading and error states are often handled together.

Short Interview Answer

We create a loading state using useState(true), display a loading UI while the request is running, and set loading to false when the API call completes.

One-Line Interview Answer

A loading state is typically shown using useState() and displayed until the API request finishes.
