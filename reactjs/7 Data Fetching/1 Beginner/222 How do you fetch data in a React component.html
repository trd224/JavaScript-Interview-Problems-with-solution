222. How Do You Fetch Data in a React Component?

One-Line Definition

Data is typically fetched in a React component using the fetch() API inside the useEffect() hook and stored using useState().

Why Do We Use useEffect for Fetching?

Fetching data is a side effect because it communicates with an external server.

Side effects should be handled inside useEffect().

Component Mounts
↓
useEffect Runs
↓
API Request Sent
↓
Response Received
↓
State Updated
↓
Component Re-renders

Basic Example

import { useEffect, useState } from 'react';

function Users() {

  const [users, setUsers] = useState([]);

  useEffect(() => {

    fetch('https://jsonplaceholder.typicode.com/users')
      .then(res => res.json())
      .then(data => setUsers(data));

  }, []);

  return (

    <ul>

      {users.map(user => (

        <li key={user.id}>

          {user.name}

        </li>

      ))}

    </ul>

  );

}

Why is the Dependency Array Empty?

useEffect(() => {

  fetchData();

}, []);
An empty dependency array means the API call runs only once when the component mounts.

Using async/await

import { useEffect, useState } from 'react';

function Users() {

  const [users, setUsers] = useState([]);

  useEffect(() => {

    async function fetchUsers() {

      const response = await fetch(
        'https://jsonplaceholder.typicode.com/users'
      );

      const data = await response.json();

      setUsers(data);

    }

    fetchUsers();

  }, []);

}

Error Handling Example

const [error, setError] = useState(null);

useEffect(() => {

  fetch(url)
    .then(res => res.json())
    .then(data => setUsers(data))
    .catch(err => setError(err.message));

}, []);
if (error) {

  return <p>{error}</p>;

}

Complete Real-World Example

import { useEffect, useState } from 'react';

function Users() {

  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {

    fetch('https://jsonplaceholder.typicode.com/users')
      .then(res => res.json())
      .then(data => {

        setUsers(data);

        setLoading(false);

      })
      .catch(err => {

        setError(err.message);

        setLoading(false);

      });

  }, []);

  if (loading) {

    return <p>Loading...</p>;

  }

  if (error) {

    return <p>{error}</p>;

  }

  return (

    <ul>

      {users.map(user => (

        <li key={user.id}>

          {user.name}

        </li>

      ))}

    </ul>

  );

}

Can We Fetch Data Without useEffect?

Yes, but in most client-side React applications, useEffect is the standard way to fetch data when a component loads. 
// Event-driven API call

const handleSearch = () => {

  fetch(url);

};
When an API call is triggered by a user action such as a button click, useEffect is usually not needed.

Real Project Example

In our Aadhaar Authentication application, useEffect can be used to fetch user details, audit records, department lists, service configurations, and dashboard statistics when a page loads. The returned data is stored in state and displayed in the UI.

Important Interview Points

  • Data fetching is a side effect.
  • useEffect is commonly used for API calls on component mount.
  • useState stores the fetched data.
  • Loading and error states should be handled.
  • API calls triggered by user actions usually do not require useEffect.

Short Interview Answer

Data is usually fetched inside useEffect() and stored using useState(). After the API response arrives, updating state causes React to re-render the component with the new data.

One-Line Interview Answer

React components typically fetch data using fetch() inside useEffect() and store the result in useState().
