What is the .next Folder and What Does It Contain?

The .next folder is the build output directory created by Next.js. It contains compiled application code, generated pages, cached files, server bundles, and other assets required to run the application.

The .next folder is automatically generated when we run npm run dev or npm run build.

When is the .next Folder Created?

Commands
npm run dev

npm run build

After running either command, Next.js creates the .next directory automatically.

Typical .next Structure

.next Folder
.next
├── cache
├── server
├── static
├── traces
├── build-manifest.json
├── prerender-manifest.json
├── routes-manifest.json
└── BUILD_ID

cache Folder

Stores cached compilation data to speed up future builds and development server startup.

Example
.next
└── cache

Deleting it is safe because Next.js can recreate it.

server Folder

Contains server-side code used for SSR, Route Handlers, Server Components, and API routes.

Example
.next
└── server
    ├── app
    ├── pages
    └── chunks

static Folder

Contains optimized JavaScript, CSS, and static assets sent to the browser.

Example
.next
└── static
    ├── chunks
    ├── css
    └── media

BUILD_ID File

Contains a unique identifier for the current build.

Example
BUILD_ID

3KjD8LpRr6Qh2xY

Next.js uses this value to identify a specific deployment build.

build-manifest.json

Maps routes to their generated JavaScript and CSS files.

Example
{
  "/": [
    "static/chunks/home.js"
  ]
}

prerender-manifest.json

Contains information about SSG and ISR pages.

Example
{
  "/blog": {
    "initialRevalidateSeconds": 60
  }
}

routes-manifest.json

Contains route information used by Next.js.

Example
{
  "dynamicRoutes": [],
  "staticRoutes": []
}

Can We Delete the .next Folder?

Yes.

Delete Folder
rm -rf .next

or on Windows: 

rmdir /s /q .next

The folder will be recreated automatically the next time we run: 

npm run dev

or

npm run build

Should We Commit .next to Git?

No.

.gitignore
.next

Since the folder is generated automatically, it should not be stored in Git.

.next in Development vs Production

Mode Purpose
Development Stores compilation cache and dev assets
Production Stores optimized build output

Important Note

The .next folder is the most important build output directory in Next.js. The production server runs directly from the files generated inside this folder.

Interview Tip

The .next folder contains the compiled output of a Next.js application, including server bundles, static assets, generated pages, build manifests, and cache files used during development and production.
One-Line Definition:
The .next folder is the build output directory generated by Next.js that contains everything needed to run the application, including compiled code, server bundles, static assets, and build metadata.
