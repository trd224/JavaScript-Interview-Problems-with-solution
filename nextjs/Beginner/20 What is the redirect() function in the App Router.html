What is the redirect() Function in the App Router?

The redirect() function is used in the Next.js App Router to immediately redirect users from one route to another.

The redirect() function stops the current rendering process and sends the user to a different route.

Importing redirect()

Import
import { redirect } from 'next/navigation';

Basic Example

Redirect to Login Page
import { redirect } from 'next/navigation';

export default function Dashboard() {

  redirect('/login');

}

Result: 

Current Route
      ↓
/dashboard

Redirected To
      ↓
/login

Why Do We Use redirect()?

We often need to prevent users from accessing certain pages.

Examples:
  • User is not logged in.
  • User does not have permission.
  • Required data does not exist.
  • A page has moved to a new location.

Authentication Example

Protected Route
import { redirect } from 'next/navigation';

export default async function Dashboard() {

  const user = null;

  if (!user) {
    redirect('/login');
  }

  return <h1>Dashboard</h1>;

}

Flow: 

User Opens Dashboard
          │
          ▼
Check Authentication
          │
          ▼
User Not Logged In
          │
          ▼
redirect('/login')
          │
          ▼
Login Page

Redirect Based on Data

Product Example
import { redirect } from 'next/navigation';

export default async function Product() {

  const product = null;

  if (!product) {
    redirect('/products');
  }

  return <h1>Product Details</h1>;

}

If the product does not exist, users are redirected to the products page.

         │

Import next/navigation useRouter()

Server Component Example

Server Component
import { redirect } from 'next/navigation';

export default async function Page() {

  const isLoggedIn = false;

  if (!isLoggedIn) {
    redirect('/login');
  }

  return <h1>Welcome</h1>;

}

This works because Server Components can use redirect().

Client Component Navigation

For button clicks and user actions, we normally use router.push().

Client Component
'use client';

import { useRouter } from 'next/navigation';

export default function Home() {

  const router = useRouter();

  return (
    <button onClick={() => router.push('/about')}>
      Go To About
    </button>
  );

}

redirect() vs notFound()

Function Purpose
redirect() Send user to another page
notFound() Show 404 page
Redirect

Common Use Cases

  • Authentication checks.
  • Authorization checks.
  • Redirecting old URLs.
  • Handling missing data.
  • Multi-step workflows.
One-Line Definition:
The redirect() function in the Next.js App Router immediately stops rendering and redirects users to another route, commonly used for authentication, authorization, and conditional navigation.
