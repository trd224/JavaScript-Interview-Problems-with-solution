53. How do you generate dynamic metadata using generateMetadata()?

The generateMetadata() function is used when metadata depends on dynamic data such as products, blog posts, user profiles, CMS content, or database records.

Unlike the static metadata export, generateMetadata() can fetch data and generate metadata dynamically for each page.

Why Use generateMetadata()?

Sometimes metadata cannot be known ahead of time because it depends on the current route or fetched data.

For example:

  • Product Detail Pages
  • Blog Detail Pages
  • User Profile Pages
  • CMS Content
  • Database Records

Complete Example

app/products/[id]/page.js
async function getProduct(id) {
  const res = await fetch(`https://dummyjson.com/products/${id}`);
  return res.json();
}

export async function generateMetadata({ params }) {
  const { id } = await params;

  const product = await getProduct(id);

  return {
    title: product.title,
    description: product.description
  };
}

export default async function Page({ params }) {
  const { id } = await params;

  const product = await getProduct(id);

  return (
    <div>
      <h1>{product.title}</h1>
      <p>{product.description}</p>
      <p>Price: ${product.price}</p>
    </div>
  );
}

Do We Really Fetch Data Twice?

At first glance, it appears that getProduct() is called twice.

Calls
generateMetadata()
      │
      ▼
getProduct()

Page()
      │
      ▼
getProduct()

However, Next.js automatically deduplicates identical fetch requests.

Although the code calls getProduct() twice, Next.js usually performs only one network request when both fetch calls are identical.
Actual Behavior
Code Calls    = 2

Network Calls = 1

When Will It Fetch Twice?

If the requests are different, deduplication cannot occur.

Different Fetch Options
fetch(`/products/${id}`);

fetch(`/products/${id}`, {
  cache: 'no-store'
});
Different URLs
fetch(`/products/${id}`);

fetch(`/products/${id}?details=true`);

When Should We Use generateMetadata()?

  • Product detail pages.
  • Blog detail pages.
  • User profile pages.
  • CMS-driven pages.
  • Database-driven pages.
  • Any route where metadata depends on fetched data.

Interview Tip

The generateMetadata() function runs before the page is rendered and allows us to fetch data and dynamically generate SEO metadata such as title and description.
One-Line Definition:
generateMetadata() is an App Router function that dynamically generates page metadata by fetching data before the page is rendered.
