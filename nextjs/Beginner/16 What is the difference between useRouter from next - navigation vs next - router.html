What is the Difference Between useRouter from next/navigation vs next/router?

Both hooks allow us to perform programmatic navigation in Next.js, but they belong to different routing systems.

next/router is used with the Pages Router, while next/navigation is used with the App Router introduced in Next.js 13.

Import Difference

Router Import
Pages Router import { useRouter } from 'next/router';
App Router import { useRouter } from 'next/navigation';

Pages Router Example

next/router
import { useRouter } from 'next/router';

export default function Home() {

  const router = useRouter();

  return (
    <button onClick={() => router.push('/about')}>
      About
    </button>
  );

}

App Router Example

next/navigation
'use client';

import { useRouter } from 'next/navigation';

export default function Home() {

  const router = useRouter();

  return (
    <button onClick={() => router.push('/about')}>
      About
    </button>
  );

}

Query Parameters

In the Pages Router, query parameters are available through router.query.

Pages Router
const router = useRouter();

console.log(router.query.id);

URL: 

/products?id=100

Output: 

100

In the App Router, router.query does not exist.

App Router
'use client';

import { useSearchParams } from 'next/navigation';

export default function Product() {

  const searchParams = useSearchParams();
  const id = searchParams.get('id');

  return <h1>{id}</h1>;

}

Dynamic Route Parameters

Pages Router
const router = useRouter();

console.log(router.query.id);

Route: 

pages/products/[id].js
App Router
export default function Product({ params }) {

  return <h1>{params.id}</h1>;

}

Route: 

app/products/[id]/page.js

Current Path

Pages Router
const router = useRouter();

console.log(router.pathname);
App Router
'use client';

import { usePathname } from 'next/navigation';

const pathname = usePathname();

New App Router Methods

router.refresh()
router.refresh();

Refreshes the current route and re-fetches server data.

router.forward()
router.forward();

Moves forward in browser history.

When Should We Use Each One?

  • Use next/router when working with the Pages Router.
  • Use next/navigation when working with the App Router.
  • For new Next.js applications, the App Router and next/navigation are recommended.
One-Line Definition:
useRouter from next/router is designed for the Pages Router and provides features such as query, pathname, and router events, while useRouter from next/navigation is designed for the App Router and works with modern hooks such as useSearchParams(), usePathname(), and router.refresh().
