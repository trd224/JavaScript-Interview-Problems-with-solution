67. How does metadata title.template work in nested layouts?

The title.template property allows a layout to define a title pattern that is automatically applied to all child pages and nested layouts.

acts like a wrapper around child page titles. The %s placeholder is replaced with the child page's title.

Basic Example

app/layout.js
export const metadata = {
  title: {
    template: '%s | My Website',
    default: 'My Website'
  }
};

export default function RootLayout({ children }) {
  return (
    <html>
      <body>{children}</body>
    </html>
  );
}
app/about/page.js
export const metadata = {
  title: 'About'
};

export default function About() {
  return <h1>About</h1>;
}

Generated Title

Result
<title>About | My Website</title>

How %s Works

Template
%s | My Website
Page Title
About
Final Title
About | My Website

Using default Title

If a page does not define its own title, Next.js uses the default value.

app/layout.js
export const metadata = {
  title: {
    template: '%s | My Website',
    default: 'My Website'
  }
};
app/contact/page.js
export default function Contact() {
  return <h1>Contact</h1>;
}
Generated Title
<title>My Website</title>

Nested Layout Example

Project Structure
app
├── layout.js
├── page.js
└── dashboard
    ├── layout.js
    ├── page.js
    └── settings
        └── page.js

Root Layout

app/layout.js
export const metadata = {
  title: {
    template: '%s | My Website',
    default: 'My Website'
  }
};

Dashboard Layout

app/dashboard/layout.js
export const metadata = {
  title: {
    template: '%s | Dashboard',
    default: 'Dashboard'
  }
};

export default function DashboardLayout({
  children
}) {
  return children;
}

Dashboard Page

app/dashboard/page.js
export const metadata = {
  title: 'Home'
};

export default function DashboardPage() {
  return <h1>Dashboard</h1>;
}

Generated Title

Result
<title>Home | Dashboard</title>

Important Rule

The nearest layout's title.template wins.

In the previous example:

Hierarchy
Root Layout
     │
     ▼
Dashboard Layout
     │
     ▼
Dashboard Page

The Dashboard layout overrides the Root layout template.

Nested Layout Resolution

Example
Root Template:
%s | My Website

Dashboard Template:
%s | Dashboard

Page Title:
Home
Result
Home | Dashboard

NOT: 

Home | Dashboard | My Website

Multiple Nested Layouts

Project Structure
app
└── dashboard
    └── settings
        ├── layout.js
        └── page.js
app/dashboard/settings/layout.js
export const metadata = {
  title: {

    default: 'Settings'
  }
};
app/dashboard/settings/page.js

How Title Resolution Works

Flow
Page Title
     │
     ▼
Find Nearest Layout
     │
     ▼
Apply title.template
     │
     ▼
Generate Final Title

title.template vs title.default

Property Purpose
template Formats child page titles
default Used when child page has no title

Real-World Example

Root Layout
title: {
  template: '%s | Amazon',
  default: 'Amazon'
}
Product Page
title: 'iPhone 15'
Generated Title
<title>iPhone 15 | Amazon</title>

Important Note

only affects child routes. The layout's own title comes from title.default.

Interview Tip

In nested layouts, Next.js uses the closest parent layout's title.template when generating the final page title.
One-Line Definition:
metadata.title.template defines a title format for child pages, and in nested layouts the nearest parent layout's template is applied to the page title.
