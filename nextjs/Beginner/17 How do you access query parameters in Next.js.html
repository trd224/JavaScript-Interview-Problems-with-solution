How do you Access Query Parameters in Next.js?

Query parameters are values passed in the URL after a question mark (?).

Query parameters allow us to pass additional data through the URL without changing the route structure.

Example URL

URL
/products?id=100&category=mobile

Here:

Parameter Value
id 100
category mobile

Pages Router Method

In the Pages Router, query parameters are accessed using router.query.

pages/products.js
import { useRouter } from 'next/router';

export default function Products() {

  const router = useRouter();

  return (
    <>
      <h2>ID: {router.query.id}</h2>
      <h2>Category: {router.query.category}</h2>
    </>
  );

}

URL: 

/products?id=100&category=mobile

Output: 

ID: 100
Category: mobile

App Router Method

In the App Router, router.query does not exist.

Instead, we use useSearchParams().

app/products/page.js
'use client';

import { useSearchParams } from 'next/navigation';

export default function Products() {

  const searchParams = useSearchParams();

  const id = searchParams.get('id');
  const category = searchParams.get('category');

  return (
    <>
      <h2>ID: {id}</h2>
      <h2>Category: {category}</h2>
    </>
  );

}

URL: 

/products?id=100&category=mobile

Output: 

ID: 100
Category: mobile

Reading a Single Parameter

App Router
const searchParams = useSearchParams();

const id = searchParams.get('id');

console.log(id);

URL: 

/products?id=100

Output: 

100

Reading Multiple Parameters

App Router
const searchParams = useSearchParams();

const id = searchParams.get('id');
const page = searchParams.get('page');
const sort = searchParams.get('sort');

URL: 

/products?id=100&page=2&sort=price

Output: 

id   = 100
page = 2
sort = price

Checking if a Query Parameter Exists

App Router
const searchParams = useSearchParams();

const id = searchParams.get('id');

if (id) {

  console.log('Parameter Found');

}

Getting All Query Parameters

App Router
const searchParams = useSearchParams();

console.log(searchParams.toString());

URL: 

/products?id=100&page=2

Output: 

id=100&page=2

Pages Router vs App Router

Feature Pages Router App Router
Hook useRouter() useSearchParams()
Access Query router.query searchParams.get()
Import next/router next/navigation
Requires 'use client' No Yes

Query Parameters vs Dynamic Route Parameters

Type Example URL Access Method
Query Parameter /products?id=100 searchParams.get('id')
Dynamic Route Parameter /products/100 params.id
Important:
Query parameters come after ?, while dynamic route parameters are part of the URL path.

Real-World Example

Suppose our application supports filtering products.

URL
/products?category=mobile&sort=price&page=2

We can access all filter values using:

App Router
const searchParams = useSearchParams();

const category = searchParams.get('category');
const sort = searchParams.get('sort');
const page = searchParams.get('page');
One-Line Definition:
In Next.js, query parameters are accessed using router.query in the Pages Router and useSearchParams() in the App Router.
