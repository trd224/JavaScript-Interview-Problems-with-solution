What are CSS Modules and how do you use them in Next.js?

CSS Modules are CSS files whose styles are scoped locally to a specific component.

CSS Modules allow us to write component-specific CSS without worrying about class name conflicts in our application.

Why Do We Need CSS Modules?

Suppose we have two components:

Header.css
.title {
  color: blue;
}
Footer.css
.title {
  color: red;
}

Both files use the same class name: 

title

This can cause style conflicts when using normal CSS.

How CSS Modules Solve This

Next.js automatically generates unique class names.

Generated Internally
title
      │
      ▼

Header_title__a12bc

title
      │
      ▼

Footer_title__x78yz

Now both classes are unique and cannot conflict.

Creating a CSS Module

CSS Module files must end with .module.css.

Button.module.css
.button {
  background: blue;
  color: white;
  padding: 10px 20px;
  border: none;
}

Using a CSS Module

Button.js
import styles from './Button.module.css';

export default function Button() {

  return (
    <button className={styles.button}>
      Click Me
    </button>
  );

}

components/

Example with Multiple Classes

Card.module.css
.card {
  padding: 20px;
  border: 1px solid gray;
}

.title {
  color: blue;
}
Card.js
import styles from './Card.module.css';

export default function Card() {

  return (
    <div className={styles.card}>
      <h2 className={styles.title}>
        Product
      </h2>
    </div>
  );

}

Using Multiple Module Classes

Button.module.css
.button {
  padding: 10px;
}

.primary {
  background: blue;
}
Button.js
import styles from './Button.module.css';

export default function Button() {

  return (
    <button
      className={`${styles.button} ${styles.primary}`}
    >
      Save
    </button>
  );

}

Pages Router Example

pages/index.js
import styles from '../styles/Home.module.css';

export default function Home() {

  return (

  );

}

CSS Modules vs Global CSS

Feature CSS Modules Global CSS
Scope Single Component Entire Application
Class Name Conflicts No Possible
Import Location Any Component Root Files Only
Best For Component Styles App-Wide Styles

Real-World Example

Suppose our application contains:

Components
Navbar
Sidebar
ProductCard
Footer

Each component can have its own CSS Module:

Structure
Navbar.module.css
Sidebar.module.css
ProductCard.module.css
Footer.module.css

This keeps styles isolated and easy to maintain.

Advantages of CSS Modules

  • No class name collisions.
  • Component-level styling.
  • Easy maintenance.
  • Automatic unique class generation.
  • Works in both Pages Router and App Router.

Important Note

A common misconception is that CSS Modules are global. In reality, CSS Modules are locally scoped to the component that imports them.

Interview Tip

When we write styles.button, Next.js automatically maps it to a unique generated class name behind the scenes.
One-Line Definition:
CSS Modules are locally scoped CSS files that allow us to create component-specific styles without class name conflicts by using files that end with .module.css.
