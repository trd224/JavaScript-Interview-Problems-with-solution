What changed with cookies(), headers(), and params becoming async in Next.js 15?

In Next.js 15, cookies(), headers(), params, and searchParams became asynchronous APIs.

Before Next.js 15, these APIs could be accessed directly. In Next.js 15, we must use await because they now return Promises.

Why Did Next.js Make Them Async?

This change helps Next.js support:

  • Streaming
  • Partial Prerendering (PPR)
  • Server Components
  • Future rendering optimizations

Making these APIs asynchronous gives Next.js more flexibility when preparing and rendering pages.

cookies() Before Next.js 15

Old Version
import { cookies } from 'next/headers';

export default function Page() {

  const cookieStore = cookies();

  const token = cookieStore.get('token');

  return <h1>{token?.value}</h1>;

}

cookies() In Next.js 15

Next.js 15
import { cookies } from 'next/headers';

export default async function Page() {

  const cookieStore = await cookies();

  const token = cookieStore.get('token');

  return <h1>{token?.value}</h1>;

}

Or more commonly:

Compact Version
const token = (await cookies()).get('token')?.value;

headers() Before Next.js 15

Old Version
import { headers } from 'next/headers';

export default function Page() {

  const headerList = headers();

  const userAgent = headerList.get('user-agent');

  return <h1>{userAgent}</h1>;

}

headers() In Next.js 15

Next.js 15
import { headers } from 'next/headers';

export default async function Page() {

  const headerList = await headers();

  const userAgent = headerList.get('user-agent');

  return <h1>{userAgent}</h1>;

}

params Before Next.js 15

Old Version
export default function Product({ params }) {

  return <h1>{params.id}</h1>;

}

params In Next.js 15

Next.js 15
export default async function Product({ params }) {

  const { id } = await params;

  return <h1>{id}</h1>;

}

searchParams In Next.js 15

Example
export default async function Products({ searchParams }) {

  const { page } = await searchParams;

  return <h1>Page {page}</h1>;

}

Real-World Example

Authentication Check
import { cookies } from 'next/headers';

export default async function Dashboard() {

  const token = (await cookies()).get('token')?.value;

  if (!token) {
    return <h1>Unauthorized</h1>;
  }

  return <h1>Dashboard</h1>;

}

Important Note

After upgrading to Next.js 15, one of the most common migration issues is forgetting to add await when accessing cookies(), headers(), params, or searchParams.

Interview Tip

A common misconception is that only cookies() and headers() became async. In reality, params and searchParams also became asynchronous and should be awaited in Next.js 15.
One-Line Definition:
In Next.js 15, cookies(), headers(), params, and searchParams became asynchronous APIs, requiring us to use await before accessing their values.
