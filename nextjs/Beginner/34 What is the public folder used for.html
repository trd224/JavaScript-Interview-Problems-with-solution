What is the public folder used for?

The public folder is a special folder in Next.js used to store static files that should be served directly to the browser without processing by Next.js.

The public folder allows us to store images, icons, videos, PDFs, robots.txt, sitemap.xml, and other static assets that can be accessed directly through a URL.

Why Do We Need the public Folder?

Some files do not need bundling, compilation, or optimization.

Examples:

  • Images
  • Favicons
  • PDF Files
  • Videos
  • robots.txt
  • sitemap.xml

These files can simply be served as-is.

How Files Are Accessed

Files inside the public folder are automatically available from the root URL.

Browser URL
/logo.png

Notice that the word public is not included in the URL.

Using Images from public

Using img

Example
export default function Home() {

  return (
    <img
      src="/logo.png"
      alt="Logo"
    />
  );

}

Using next/image

Example
import Image from 'next/image';

export default function Home() {

  return (
    <Image
      src="/logo.png"
      alt="Logo"
      width={200}
      height={100}
    />
  );

}

public Folder vs Importing Assets

Feature public Folder Import Asset
Accessed Via URL ✅ ❌
Bundled by Next.js ❌ ✅
Can Be Referenced Dynamically ✅ Limited
Good For Large Static Files ✅ ❌

Example: Importing an Asset

Local Import
import logo from './logo.png';

export default function Home() {

  return (
    <img
      src={logo.src}
      alt="Logo"
    />
  );

}

Here the image becomes part of the application bundle.

When Should We Use the public Folder?

  • Images shared across the application.
  • Favicons.
  • PDF downloads.
  • Videos.
  • robots.txt.
  • sitemap.xml.
  • Static assets referenced by URL.

When Should We Avoid It?

  • Component-specific assets.
  • Assets that should be bundled.
  • Assets imported directly into React components.

Pages Router vs App Router

The public folder works exactly the same in both routers.

Interview Tip

A common misconception is that files inside the public folder are imported into the JavaScript bundle. In reality, files in the public folder are served directly by Next.js and accessed using URLs.
One-Line Definition:
The public folder is a special Next.js folder used to store static assets that are served directly to the browser and accessed through root-based URLs such as /logo.png.
