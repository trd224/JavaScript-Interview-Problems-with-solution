What is Automatic Code Splitting in Next.js?

Automatic Code Splitting is a Next.js optimization that automatically splits JavaScript into smaller chunks so users download only the code needed for the current page.

Instead of sending the entire application bundle to the browser, Next.js sends only the JavaScript required for the requested route.

Why Do We Need Code Splitting?

Imagine an application with many pages:

Project Structure
app
├── page.js
├── about
│   └── page.js
├── products
│   └── page.js
├── dashboard
│   └── page.js
└── settings
    └── page.js

If all page code were bundled together, every user would download JavaScript for every page, even pages they never visit.

Without Code Splitting

Single Bundle
Home Code
About Code
Products Code
Dashboard Code
Settings Code
      │
      ▼
One Large Bundle
      │
      ▼
Download Everything

Even users visiting only the Home page must download code for all pages.

With Automatic Code Splitting

Route-Based Splitting
Home Route
    │
    ▼
Download Home Chunk

About Route
    │
    ▼
Download About Chunk

Products Route
    │
    ▼
Download Products Chunk

Users download only the JavaScript needed for the current page.

Example

app/page.js
export default function Home() {
  return <h1>Home Page</h1>;
}
app/about/page.js
export default function About() {
  return <h1>About Page</h1>;
}

When a user visits: 

/

Next.js sends only the Home page JavaScript.

When a user visits: 

/about

Next.js sends only the About page JavaScript.

Build Output Concept

Generated Chunks
.next
└── static
    └── chunks
        ├── home.js
        ├── about.js
        ├── products.js
        └── dashboard.js

Each route gets its own JavaScript chunk.

How It Works

Flow
User Visits /about
        │
        ▼
Next.js Detects Route
        │
        ▼
Load About Chunk
        │
        ▼
Do Not Load
Home/Dashboard Chunks
        │
        ▼
Faster Page Load

Automatic Route Splitting

No configuration is required.

Example
app/page.js
app/about/page.js
app/products/page.js

Next.js automatically creates separate bundles for these routes.

Dynamic Imports for Additional Splitting

We can split large components even further using dynamic imports.

Example
import dynamic from 'next/dynamic';

const Chart = dynamic(() =>
  import('./Chart')
);

export default function Dashboard() {
  return <Chart />;
}

The Chart component is loaded only when needed.

Benefits of Automatic Code Splitting

  • Smaller JavaScript bundles.
  • Faster initial page load.
  • Better performance.
  • Reduced bandwidth usage.
  • Improved user experience.
  • No manual configuration required.

Automatic Code Splitting vs Traditional SPA

Feature Next.js Traditional SPA
Route-Based Splitting ✅ Automatic ⚠️ Manual Setup
Smaller Initial Bundle ✅ ❌ Often Large
Performance ✅ Better ⚠️ Depends
Configuration None Usually Required

Real-World Example

E-Commerce Website
Home Page
    │
    ▼
Download Home Code

Product Page
    │
    ▼
Download Product Code

Admin Dashboard
    │
    ▼
Download Dashboard Code

Customers visiting the Home page do not download Dashboard code.

Important Note

Automatic code splitting happens at the route level by default. Every page gets its own JavaScript bundle automatically.

Interview Tip
