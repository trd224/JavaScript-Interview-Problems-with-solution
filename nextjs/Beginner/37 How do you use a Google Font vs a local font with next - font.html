How do you use a Google Font vs a local font with next/font?

The next/font package allows us to use both Google Fonts and local fonts with automatic optimization, self-hosting, and layout shift prevention.

Google Fonts are imported from next/font/google, while local fonts are imported from next/font/local.

Google Font

Use Google Fonts when the font is available in the Google Fonts library.

Import a Google Font

app/layout.js
import { Inter } from 'next/font/google';

const inter = Inter({
  subsets: ['latin']
});

export default function RootLayout({ children }) {

  return (
    <html lang="en">
      <body className={inter.className}>
        {children}
      </body>
    </html>
  );

}

Using Multiple Weights

Example
import { Inter } from 'next/font/google';

const inter = Inter({
  subsets: ['latin'],
  weight: ['400', '700']
});

Using Multiple Styles

Example
import { Roboto } from 'next/font/google';

const roboto = Roboto({
  subsets: ['latin'],
  weight: ['400', '700'],
  style: ['normal', 'italic']
});

Local Font

Use a local font when the font file exists inside our project.

Project Structure

Example
app/
├── fonts/
│   └── MyFont.woff2
├── layout.js
└── page.js

Import a Local Font

app/layout.js
import localFont from 'next/font/local';

const myFont = localFont({
  src: './fonts/MyFont.woff2'
});

export default function RootLayout({ children }) {

  return (
    <html lang="en">
      <body className={myFont.className}>
        {children}
      </body>
    </html>
  );

}

Local Font with Multiple Files

Many fonts have separate files for different weights.

Example
import localFont from 'next/font/local';

const myFont = localFont({
  src: [
    {
      path: './fonts/MyFont-Regular.woff2',
      weight: '400',
      style: 'normal'
    },
    {
      path: './fonts/MyFont-Bold.woff2',
      weight: '700',
      style: 'normal'
    }
  ]
});

Using CSS Variables

Fonts can be exposed as CSS variables.

Example
import { Inter } from 'next/font/google';

const inter = Inter({
  subsets: ['latin'],
  variable: '--font-inter'
});

export default function RootLayout({ children }) {

  return (
    <html
      lang="en"
      className={inter.variable}
    >
      <body>{children}</body>
    </html>
  );

}
globals.css
body {
  font-family: var(--font-inter);
}

Recommended

Suppose we want a popular font like Inter or Roboto.

Recommended

Pages Router Example

pages/_app.js
import { Inter } from 'next/font/google';

const inter = Inter({
  subsets: ['latin']
});

export default function MyApp({ Component, pageProps }) {

  return (
    <main className={inter.className}>
      <Component {...pageProps} />
    </main>
  );

}

Interview Tip

A common misconception is that Google Fonts loaded through next/font are fetched from Google every time a user visits the site. In reality, Next.js downloads and self-hosts the fonts during the build process.
One-Line Definition:
Google Fonts are used through next/font/google, local fonts are used through next/font/local, and both are automatically optimized and self-hosted by Next.js to improve performance and eliminate layout shift.
