What is the useRouter Hook and What Can We Do With It?

The useRouter hook allows us to access and control the Next.js router programmatically from our components.

The useRouter hook allows us to navigate between pages, access route parameters, read query strings, reload pages, and move backward or forward in browser history.

Important Note

The usage depends on which routing system our application uses.

Router Import
Pages Router import { useRouter } from 'next/router'
App Router import { useRouter } from 'next/navigation'

Pages Router Example

Import useRouter
import { useRouter }
from 'next/router';

export default function Home() {

  const router = useRouter();

}

Use Case 1: Navigate Programmatically

Instead of clicking a Link component, we can navigate using JavaScript.

router.push()
import { useRouter }
from 'next/router';

export default function Home() {

  const router = useRouter();

  const goToAbout = () => {

    router.push('/about');

  };

  return (
    <button
      onClick={goToAbout}
    >
      About
    </button>
  );

}

Clicking the button navigates to:

URL
/about

Use Case 2: Navigate to Dynamic Routes

Dynamic Route Navigation
router.push('/products/100');

Navigates to:

URL
/products/100

Use Case 3: Read Route Parameters

Suppose we have:

Dynamic Route
pages/
└── products/
    └── [id].js
Access Route Parameter
import { useRouter }
from 'next/router';

export default function Product() {

  const router = useRouter();

  return (
    <h1>
      {router.query.id}
    </h1>
  );

}

URL:

Example
/products/100

Output:

Result
100

Use Case 4: Read Query Parameters

URL
/products?id=100
Read Query String
const router = useRouter();

console.log(
  router.query.id
);

Output:

Console
100

Use Case 5: Replace Current Route

The replace() method navigates without creating a new browser history entry.

router.replace()
router.replace('/login');

Useful after login or logout pages.

Use Case 6: Go Back

router.back()
router.back();

Equivalent to:

Browser Action
Browser Back Button

Use Case 7: Reload Current Page

router.reload()
router.reload();

Reloads the current page.

Common Router Properties

Property Description
router.pathname Current route path
router.query Query and route parameters
router.push() Navigate to another page
router.replace() Replace current route
router.back() Go back
router.reload() Reload page

App Router Version

In the App Router, useRouter is imported from next/navigation.

App Router Example
'use client';

import { useRouter }
from 'next/navigation';

export default function Home() {

  const router = useRouter();

  const handleClick = () => {

    router.push('/about');

  };

  return (
    <button
      onClick={handleClick}
    >
      About
    </button>
  );

}

Pages Router vs App Router

Feature Pages Router App Router
Import next/router next/navigation
router.query Available Not Available
Dynamic Params router.query.id params.id
push() Available Available
replace() Available Available
back() Available Available

When Should We Use useRouter?

  • Navigate after form submission.
  • Redirect after login.
  • Read dynamic route parameters.
  • Read query string values.
  • Move backward or forward in history.
  • Perform programmatic navigation.
One-Line Definition:
The useRouter hook allows us to access and control the Next.js router programmatically, enabling navigation, route parameter access, query string handling, and browser history management.
