What is the Difference Between [[...slug]] and [...slug]?

Both [...slug] and [[...slug]] are Catch-All Routes in Next.js, but they differ in whether the route parameter is required or optional.

[...slug] requires at least one URL segment, while [[...slug]] can match zero or more URL segments.

1. Catch-All Route: [...slug]

The [...slug] route matches one or more URL segments.

Folder Structure
app/
└── docs/
    └── [...slug]/
        └── page.js

Valid URLs

Matches
/docs/react
/docs/react/hooks
/docs/react/hooks/useState

Invalid URL

Does NOT Match
/docs

Because at least one segment is required after /docs.

Example

URL
/docs/react/hooks

Next.js provides:

params
{
  slug: [
    "react",
    "hooks"
  ]
}

2. Optional Catch-All Route: [[...slug]]

The [[...slug]] route matches zero or more URL segments.

Folder Structure
app/
└── docs/
    └── [[...slug]]/
        └── page.js

Valid URLs

Matches
/docs
/docs/react
/docs/react/hooks
/docs/react/hooks/useState

Notice that /docs is also matched.

Example 1

URL
/docs

Next.js provides:

params
{
  slug: undefined
}

Example 2

URL
/docs/react/hooks

Next.js provides:

params
{
  slug: [
    "react",
    "hooks"
  ]
}

Visual Comparison

[...slug]
/docs/react            ✅
/docs/react/hooks      ✅
/docs                  ❌
[[...slug]]
/docs                  ✅
/docs/react            ✅
/docs/react/hooks      ✅

Route Matching Flow

[...slug]
Requires:

/docs/
      │
      ▼
 At Least One Segment
[[...slug]]
Requires:

/docs/
      │
      ▼
 Zero or More Segments

Real-World Example

Documentation Site

Suppose our documentation homepage is:

Homepage
/docs

And sub-pages are:

Sub Pages
/docs/react
/docs/react/hooks
/docs/react/hooks/useState

In this situation we should use:

Recommended Route
[[...slug]]

because it can handle both the homepage and nested pages.

Main Differences

Feature [...slug] [[...slug]]
Matches One Segment Yes Yes
Matches Multiple Segments Yes Yes
Matches Zero Segments No Yes
Matches /docs No Yes
Parameter Required Yes No
Parameter Type Array Array or undefined

Quick Memory Trick

The extra square brackets in [[...slug]] make the entire parameter optional.
One-Line Definition:
[...slug] is a catch-all route that requires at least one URL segment, while [[...slug]] is an optional catch-all route that can match zero or more URL segments, including the parent route itself.
