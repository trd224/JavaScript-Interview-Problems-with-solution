What is Static Site Generation (SSG) in Next.js?

Static Site Generation (SSG) is a rendering technique where Next.js generates HTML pages during the build process and stores them as static files.

SSG allows us to generate pages once at build time and serve the same pre-generated HTML to all users.

Why Do We Need SSG?

Many pages do not change frequently.

Examples:
  • Blog Posts
  • Documentation Websites
  • Marketing Pages
  • Landing Pages
  • Company Information Pages

Generating these pages on every request would be unnecessary.

Instead, we can generate them once during the build process.

SSG in the Pages Router

In the Pages Router, SSG is implemented using getStaticProps().

pages/blog.js
export async function getStaticProps() {

  const res = await fetch(
    'https://api.example.com/posts'
  );

  const posts = await res.json();

  return {
    props: { posts }
  };

}

export default function Blog({ posts }) {

  return (
    <h1>
      Total Posts: {posts.length}
    </h1>
  );

}

Dynamic SSG Pages

For dynamic routes, we use getStaticPaths().

pages/posts/[id].js
export async function getStaticPaths() {

  return {
    paths: [
      { params: { id: '1' } },
      { params: { id: '2' } }
    ],
    fallback: false
  };

}

export async function getStaticProps({ params }) {

  return {
    props: {
      id: params.id
    }
  };

}

SSG in the App Router

The App Router does not use getStaticProps().

By default, many App Router pages are statically generated when possible.

app/blog/page.js
async function getPosts() {

  const res = await fetch(
    'https://api.example.com/posts'
  );

  return res.json();

}

export default async function Blog() {

  const posts = await getPosts();

  return (
    <h1>
      Total Posts: {posts.length}
    </h1>
  );

}

If the data is static and cacheable, Next.js may generate the page during the build process.

Advantages of SSG

  • Very fast page loading.
  • Excellent SEO.
  • Low server cost.
  • Can be served from a CDN.
  • Highly scalable.

Disadvantages of SSG

  • Data can become outdated.
  • Requires rebuilding for content updates.
  • Large sites may have longer build times.

When Should We Use SSG?

  • Blogs
  • Documentation Sites
  • Landing Pages
  • Portfolio Websites
  • Marketing Pages
  • Company Websites

Important Difference Between SSR and SSG

SSR:
HTML is generated every time a user requests the page.
SSG:
HTML is generated once during the build process and the same HTML is served to all users.
