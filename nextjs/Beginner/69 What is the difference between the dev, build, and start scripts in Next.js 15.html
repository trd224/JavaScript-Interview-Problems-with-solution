What is the Difference Between the dev, build, and start Scripts in Next.js 15?

The dev, build, and start scripts serve different purposes in the Next.js development and deployment lifecycle.

dev is used during development, build creates the optimized production build, and start runs the production build.

Default Scripts

package.json
{
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start"
  }
}

1. dev Script

Command
npm run dev

Starts the development server.

What Happens?

  • Starts local development server.
  • Enables Fast Refresh.
  • Rebuilds automatically on file changes.
  • Shows detailed error overlays.
  • Optimized for developer experience, not performance.

2. build Script

Command
npm run build

Creates the production build.

What Happens?

  • Compiles the application.
  • Optimizes JavaScript and CSS.
  • Generates SSG pages.
  • Creates server bundles for SSR.
  • Creates the .next folder.
  • Analyzes routes and rendering strategies.
Flow
Source Code
     │
     ▼
next build
     │
     ▼
Optimization
     │
     ▼
Generate Assets
     │
     ▼

○ /                        SSG

Next.js determines how each route will be rendered.

3. start Script

Command
npm start

Runs the production build generated by next build.

What Happens?

  • Reads files from the .next folder.
  • Starts the production server.
  • Runs optimized SSR pages.
  • Serves generated SSG pages.
  • Uses production caching.
Flow
next build
     │
     ▼
.next Folder
     │
     ▼
next start
     │
     ▼
Production Server

Important Relationship

npm start requires a completed npm run build. Without a build, the start command will fail.
Wrong
npm start

Without building first, Next.js may show: 

Could not find a production build
in the '.next' directory.
Correct
npm run build

npm start

Real-World Development Workflow

Development
npm run dev

Develop and test locally.

Production
npm run build

npm start

Create and run the production build.

Example Scenario

Developer Changes app/page.js
export default function Page() {
  return <h1>Home Page</h1>;
}

Using:

npm run dev

The browser updates immediately using Fast Refresh.

Using:

npm run build

Next.js generates optimized production assets.

Using:

npm start

The production version is served to users.

Important Note

Users never interact with next dev. Real users access the application through the production server started by next start (or a hosting platform such as Vercel).

Interview Tip

The most common interview answer is:
  • next dev → Development server.
  • next build → Creates optimized production build.
  • next start → Runs the production build.
One-Line Definition:
In Next.js 15, dev starts the development server, build creates the optimized production build, and start runs that production build from the .next folder.
