How do you add and configure ESLint in a Next.js project?

ESLint is used in a Next.js project to find code issues, enforce coding rules, and improve code quality during development.

Next.js supports ESLint out of the box, and we can configure it using an ESLint configuration file such as eslint.config.mjs.

What is ESLint?

ESLint is a linting tool that checks JavaScript, TypeScript, React, and Next.js code for mistakes and style issues.

Install ESLint Packages

Terminal
npm install -D eslint eslint-config-next

Create ESLint Config File

eslint.config.mjs
import next from 'eslint-config-next';

export default [
  ...next
];

Alternative Config for CommonJS Projects

.eslintrc.json
{
  "extends": "next/core-web-vitals"
}
In newer ESLint versions, eslint.config.mjs is the modern flat config style. Older projects commonly use .eslintrc.json.

Add Lint Script

package.json
{
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start",
    "lint": "eslint ."
  }
}

Run ESLint

Terminal
npm run lint

This checks our project files and reports linting errors or warnings.

Example ESLint Warning

Bad Code
export default function Page() {
  const name = 'Rizwan';

  return <h1>Hello</h1>;
}

Here, name is declared but not used, so ESLint can show a warning.

Possible Warning
'name' is assigned a value but never used.

Correct Code

app/page.js
export default function Page() {
  const name = 'Rizwan';

  return <h1>Hello {name}</h1>;
}

Next.js ESLint Rules

The eslint-config-next package includes rules for:

  • React best practices.
  • Next.js best practices.
  • Accessibility issues.
  • Image optimization issues.
  • Link usage issues.
  • Core Web Vitals related rules.

Example: Next.js Link Rule

Bad Code
export default function Page() {
  return <a href="/about">About</a>;
}

For internal navigation in Next.js, we should use Link.

Correct Code
import Link from 'next/link';

export default function Page() {
  return <Link href="/about">About</Link>;
}

Example: Image Rule

Bad Code
export default function Page() {
  return <img src="/logo.png" alt="Logo" />;
}

Next.js recommends using the Image component for image optimization.

Correct Code
import Image from 'next/image';

export default function Page() {
  return <Image src="/logo.png" alt="Logo" width={200} height={100} />;
}

Custom ESLint Rules

We can also add our own rules.

eslint.config.mjs
import next from 'eslint-config-next';

export default [
  ...next,
  {
    rules: {
      'no-console': 'warn',
      'no-unused-vars': 'warn'
    }
  }
];

Ignore Files

We can ignore generated folders like .next and node_modules.

eslint.config.mjs
import next from 'eslint-config-next';

export default [
  {
    ignores: ['.next/**', 'node_modules/**', 'out/**']
  },
  ...next
];

ESLint with TypeScript

For TypeScript projects, ESLint can check .ts and .tsx files also.

Example Files
app/page.tsx
app/layout.tsx
components/Header.tsx

Old Next.js Command

In older Next.js projects, we commonly used:

Terminal
next lint

In newer projects, using the normal ESLint command is preferred:

Terminal
eslint .

ESLint Flow

Flow
Write Code
    │
    ▼
Run npm run lint
    │
    ▼
ESLint Checks Rules
    │
    ▼
Show Errors / Warnings
    │
    ▼
Fix Code Quality Issues

ESLint vs Prettier

Tool Purpose
ESLint Finds code problems and enforces rules
Prettier Formats code style automatically

Common Commands

Terminal
npm run lint

npx eslint .

npx eslint . --fix

Important Note

ESLint helps catch problems before production. It does not replace testing, but it improves code quality and prevents common mistakes.

Interview Tip

In a Next.js project, ESLint is configured using eslint-config-next. We add a lint script such as eslint . and run it to check code quality, React rules, and Next.js best practices.
One-Line Definition:
ESLint in Next.js is used to detect code issues and enforce best practices, and it can be configured with eslint-config-next and run using a script like npm run lint.
