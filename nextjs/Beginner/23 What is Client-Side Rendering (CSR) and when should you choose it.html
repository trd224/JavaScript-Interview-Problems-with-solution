23. What is Client-Side Rendering (CSR) and When Should We Choose It?

Client-Side Rendering (CSR) is a rendering technique where the browser downloads JavaScript and generates the page UI on the client side.

In CSR, the server sends a minimal HTML page and JavaScript bundle, and the browser renders the application after the JavaScript executes.

Why Do We Need CSR?

Some applications are highly interactive and depend heavily on user actions.

Examples:
  • Admin Dashboards
  • Project Management Tools
  • Chat Applications
  • Internal Business Applications
  • Analytics Dashboards

For these applications, SEO is usually less important than user interaction.

CSR Example in React

Client-Side Data Fetching
import { useEffect, useState } from 'react';

export default function Users() {

  const [users, setUsers] = useState([]);

  useEffect(() => {

    fetch('/api/users')
      .then(res => res.json())
      .then(data => setUsers(data));

  }, []);

  return (
    <h1>
      Users: {users.length}
    </h1>
  );

}

Advantages of CSR

  • Highly interactive UI.
  • Less server work.
  • Fast page transitions after initial load.
  • Good for authenticated applications.
  • Works well with real-time updates.

Disadvantages of CSR

  • Poorer SEO.
  • Slower first page load.
  • Requires JavaScript.
  • Users may initially see loading states.

When Should We Choose CSR?

  • Admin Dashboards
  • Internal Company Tools
  • Chat Applications
  • Project Management Systems
  • Analytics Dashboards
  • User-Specific Applications

When Should We Avoid CSR?

  • Marketing Pages
  • Blogs
  • News Websites
  • Documentation Sites
  • SEO-Focused Pages

Important Difference Between SSR and CSR

SSR:
The server generates HTML before sending it to the browser.
CSR:
The browser downloads JavaScript and generates the UI after the page loads.
