What is next.config.js / next.config.ts and what can you configure in it?

next.config.js or next.config.ts is the main configuration file for a Next.js application.

It allows us to customize how Next.js builds, serves, optimizes, and runs our application.

Why Do We Need next.config?

By default, Next.js works with sensible defaults.

However, sometimes we need to customize behavior such as:

  • Remote image domains
  • Redirects
  • Rewrites
  • Environment variables
  • Webpack configuration
  • Headers
  • Build settings
  • Output configuration

These settings are configured in next.config.js or next.config.ts.

Project Structure

Example
my-app/

├── app/
├── public/
├── package.json
├── next.config.js
└── tsconfig.json

Basic Example

next.config.js
/** @type {import('next').NextConfig} */

const nextConfig = {};

module.exports = nextConfig;

TypeScript Version

next.config.ts
import type {
  NextConfig
} from 'next';

const nextConfig: NextConfig = {};

export default nextConfig;

Both files serve the same purpose.

The TypeScript version provides type safety and IntelliSense.

Configure Remote Image Domains

Example
const nextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'images.example.com'
      }
    ]
  }
};

module.exports = nextConfig;

This allows next/image to optimize images from external domains.

Configure Redirects

Example
const nextConfig = {

  async redirects() {

    return [
      {
        source: '/old-page',
        destination: '/new-page',
        permanent: true
      }
    ];

  }

};

module.exports = nextConfig;

Visitors to /old-page are redirected to /new-page.

Configure Rewrites

Example
const nextConfig = {

  async rewrites() {

    return [
      {
        source: '/api/users',
        destination: 'https://api.example.com/users'
      }
    ];

  }

};

module.exports = nextConfig;

The browser sees: 

/api/users

But the request actually goes to: 

https://api.example.com/users

Configure HTTP Headers

Example
const nextConfig = {

  async headers() {

    return [
      {
        source: '/(.*)',
        headers: [
          {
            key: 'X-App',
            value: 'NextJS'
          }
        ]
      }
    ];

  }

};

module.exports = nextConfig;

Environment Variables

Example
const nextConfig = {
  env: {
    API_URL: 'https://api.example.com'
  }
};

module.exports = nextConfig;

Usage:

Component
console.log(
  process.env.API_URL
);

Custom Webpack Configuration

Example
const nextConfig = {

  webpack(config) {

    config.resolve.alias = {
      ...config.resolve.alias,
      '@': __dirname
    };

    return config;

  }

};

module.exports = nextConfig;

This allows: 

import Header from '@/components/Header';

Standalone Deployment Output

Example
const nextConfig = {
  output: 'standalone'
};

module.exports = nextConfig;

Useful for Docker deployments because Next.js generates a minimal production build.

Enable React Strict Mode

Example
const nextConfig = {
  reactStrictMode: true
};

module.exports = nextConfig;

Helps detect potential React issues during development.

What Happens Internally?

Flow
npm run dev
      │
      ▼
Read next.config.js
      │
      ▼
Apply Configuration
      │
      ▼
Start Next.js Server

Next.js reads the configuration during startup.

Important Note

Whenever we change next.config.js or next.config.ts, we must restart the development server because configuration files are loaded only during startup.

Common Configuration Options

Option Purpose
images Configure image optimization
redirects() Create redirects
rewrites() Rewrite routes
headers() Add HTTP headers
env Define environment variables
webpack() Customize Webpack
output Configure build output
reactStrictMode Enable React Strict Mode

Real-World Example

next.config.js
const nextConfig = {

  reactStrictMode: true,

  output: 'standalone',

  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'cdn.example.com'
      }
    ]
  }

};

module.exports = nextConfig;

This configuration:

  • Enables React Strict Mode.
  • Allows external image optimization.
  • Creates a standalone production build.

next.config.js vs next.config.ts

Feature next.config.js next.config.ts
JavaScript ✅ ❌
Type Safety ❌ ✅
IntelliSense Limited Better
Functionality Same Same

Interview Tip

A common misconception is that next.config.js runs in the browser. In reality, it runs only during the build and server startup process and is never sent to the client.
One-Line Definition:
next.config.js or next.config.ts is the central Next.js configuration file that allows us to customize features such as images, redirects, rewrites, headers, environment variables, build output, and Webpack behavior.
