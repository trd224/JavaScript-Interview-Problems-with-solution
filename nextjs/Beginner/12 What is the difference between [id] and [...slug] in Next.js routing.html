What is the Difference Between [id] and [...slug] in Next.js Routing?

Both [id] and [...slug] are used for dynamic routing in Next.js, but they behave differently.

[id] captures a single URL segment, while [...slug] captures multiple URL segments as an array.

1. Single Dynamic Route: [id]

The [id] syntax matches exactly one dynamic segment.

App Router Structure
app/
└── products/
    └── [id]/
        └── page.js

Supported URLs:

URLs
/products/1
/products/10
/products/500

Not Supported:

Invalid URLs
/products
/products/10/reviews
Accessing the Parameter
export default function Product({
  params
}) {

  return (
    <h1>
      {params.id}
    </h1>
  );

}

URL:

Example
/products/100

Output:

params
{
  id: "100"
}

2. Catch-All Route: [...slug]

The [...slug] syntax matches one or more URL segments.

App Router Structure
app/
└── docs/
    └── [...slug]/
        └── page.js

Supported URLs:

URLs
/docs/react
/docs/react/hooks
/docs/react/hooks/useState
Accessing Parameters
export default function Docs({
  params
}) {

  return (
    <pre>
      {JSON.stringify(params.slug)}
    </pre>
  );

}

URL:

Example
/docs/react/hooks

Output:

params
{
  slug: ["react", "hooks"]
}

Visual Comparison

[id]
/products/100

id = "100"
[...slug]
/docs/react/hooks/useState

slug = [
  "react",
  "hooks",
  "useState"
]

Real-World Example

Product Page

A product usually needs only one ID.

Product Route
app/
└── products/
    └── [id]/
        └── page.js

URLs:

Examples
/products/1
/products/25
/products/100

Use [id].

Documentation Website

Documentation often contains multiple nested paths.

Docs Route
app/
└── docs/
    └── [...slug]/
        └── page.js

URLs:

Examples
/docs/react
/docs/react/hooks
/docs/react/hooks/useState

Use [...slug].

Optional Catch-All Route

Next.js also supports [[...slug]].

Optional Catch-All
app/
└── docs/
    └── [[...slug]]/
        └── page.js

Supported URLs:

URLs
/docs
/docs/react
/docs/react/hooks

Unlike [...slug], this route also matches /docs.

Main Differences

Feature [id] [...slug]
Matches One segment One or more segments
Returned Value String Array
Example URL /products/100 /docs/react/hooks
Parameter Value "100" ["react","hooks"]
Use Case Product ID, User ID Documentation, Categories

Quick Memory Trick

Think of [id] as capturing a single value and [...slug] as capturing the rest of the URL path.
One-Line Definition:
[id] captures a single dynamic URL segment and returns a string, while [...slug] captures multiple URL segments and returns them as an array.
