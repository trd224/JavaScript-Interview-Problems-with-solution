How do you fetch data on the client side in a Next.js app?

Client-side data fetching means fetching data in the browser after the page has loaded.

In Next.js, we fetch data on the client side by using a Client Component with 'use client', then calling an API inside useEffect() or using a library like SWR or React Query.

When Do We Use Client-Side Fetching?

Client-side fetching is useful when data depends on browser interaction or user-specific state.

  • Dashboard data
  • User profile data
  • Search results
  • Filters
  • Pagination
  • Live updates
  • Data after button click

Basic Client-Side Fetching

In the App Router, we must use 'use client' because useEffect() runs only in the browser.

app/users/page.js
'use client';

import { useEffect, useState } from 'react';

export default function UsersPage() {

  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    async function fetchUsers() {
      const res = await fetch('/api/users');
      const data = await res.json();

      setUsers(data);
      setLoading(false);
    }

    fetchUsers();
  }, []);

  if (loading) {
    return <h2>Loading...</h2>;
  }

  return (
    <div>
      <h1>Users</h1>

      {users.map(user => (
        <p key={user.id}>{user.name}</p>
      ))}
    </div>
  );

}

Fetching Data After Button Click

Sometimes we do not want to fetch data immediately. We may fetch data only after a user action.

Button Click Fetch
'use client';

import { useState } from 'react';

export default function ProductsPage() {

  const [products, setProducts] = useState([]);

  const loadProducts = async () => {
    const res = await fetch('/api/products');
    const data = await res.json();

    setProducts(data);
  };

  return (
    <div>
      <button onClick={loadProducts}>Load Products</button>

      {products.map(product => (
        <p key={product.id}>{product.name}</p>
      ))}
    </div>
  );

}

Using SWR

SWR is a popular data fetching library from Vercel.

It handles caching, loading states, revalidation, and refetching automatically.

Install SWR
npm install swr
SWR Example
'use client';

import useSWR from 'swr';

const fetcher = url => fetch(url).then(res => res.json());

export default function UsersPage() {

  const { data, error, isLoading } = useSWR('/api/users', fetcher);

  if (isLoading) {
    return <h2>Loading...</h2>;
  }

  if (error) {
    return <h2>Failed to load users</h2>;
  }

  return (
    <div>
      {data.map(user => (
        <p key={user.id}>{user.name}</p>
      ))}
    </div>
  );

}

Client-Side Fetching in Pages Router

In the Pages Router, client-side fetching is also done using useEffect().

Client-Side Fetching vs Server-Side Fetching

Feature Client-Side Fetching Server-Side Fetching
Where Data Fetches Browser Server
SEO Weaker Better
Initial HTML May not include data Includes data
Best For Interactive/user-specific data SEO/fresh server-rendered data
Loading State Usually needed Often not needed initially

When Should We Choose Client-Side Fetching?

  • When data is not important for SEO.
  • When data depends on user interaction.
  • When data is private or user-specific.
  • When we need live updates.
  • When fetching should happen after the page loads.

When Should We Avoid Client-Side Fetching?

  • SEO-focused pages
  • Blogs
  • Marketing pages
  • Product pages that need search engine indexing
  • Pages where content should be visible immediately in HTML

Important Note

Client-side fetching means users may initially see a loading state because the data is fetched after JavaScript runs in the browser.

Interview Tip

A common misconception is that all data fetching in Next.js should happen on the server. In reality, client-side fetching is still useful for interactive, user-specific, and frequently changing data.
One-Line Definition:
Client-side data fetching in Next.js means fetching data in the browser using a Client Component with useEffect(), fetch(), or a library like SWR, usually for interactive or user-specific data.
