What is getStaticPaths and What Does It Return?

getStaticPaths() is a special Next.js function used with dynamic routes in the Pages Router to tell Next.js which pages should be generated at build time.

getStaticPaths() runs during the build process and returns a list of dynamic route parameters that Next.js should pre-generate.

Why Do We Need getStaticPaths()?

Suppose our application has dynamic product pages.

Dynamic Route
pages/products/[id].js

Next.js does not know which product IDs exist.

We must tell Next.js which pages should be generated.

Example Problem

Suppose our database contains:

Products
Product 1
Product 2
Product 3

We want these pages:

Generated Routes
/products/1
/products/2
/products/3

This is exactly what getStaticPaths() provides.

Basic Example

pages/products/[id].js
export async function getStaticPaths() {

  return {
    paths: [
      { params: { id: '1' } },
      { params: { id: '2' } },
      { params: { id: '3' } }
    ],
    fallback: false
  };

}

What Does It Return?

getStaticPaths() returns an object containing:

Return Structure
return {
  paths: [],
  fallback: false
};
Property Purpose
paths List of dynamic pages to generate
fallback Controls behavior for non-generated pages

Understanding paths

Example
paths: [
  { params: { id: '1' } },
  { params: { id: '2' } },
  { params: { id: '3' } }
]

Next.js generates:

Generated Pages
/products/1
/products/2
/products/3

Using getStaticProps with getStaticPaths

Complete Example
export async function getStaticPaths() {

  return {
    paths: [
      { params: { id: '1' } },
      { params: { id: '2' } }
    ],
    fallback: false
  };

}

export async function getStaticProps({ params }) {

  return {
    props: {
      id: params.id
    }
  };

}

export default function Product({ id }) {

  return <h1>Product {id}</h1>;

}

getStaticPaths vs getStaticProps

Feature getStaticPaths getStaticProps
Purpose Provides Route Parameters Provides Page Data
Used For Dynamic Routes Static Generation
Runs Build Time Build Time
Returns paths + fallback props

Important Limitations

  • Only works in the Pages Router.
  • Only used with dynamic routes.
  • Runs only during build.
  • Usually used together with getStaticProps().

App Router Equivalent

The App Router does not use getStaticPaths().

Instead, it uses generateStaticParams().

App Router
export async function generateStaticParams() {

  return [
    { id: '1' },
    { id: '2' },
    { id: '3' }
  ];

}

This serves the same purpose as getStaticPaths().

Interview Tip

A common misconception is that getStaticPaths() fetches page data. In reality, it only provides route parameters. The actual page data is usually fetched inside getStaticProps().
One-Line Definition:
getStaticPaths() is a Pages Router function that returns a list of dynamic route parameters and tells Next.js which pages should be statically generated during the build process.
