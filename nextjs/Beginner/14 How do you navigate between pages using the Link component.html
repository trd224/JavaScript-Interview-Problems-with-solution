How do you Navigate Between Pages Using the Link Component?

The Link component in Next.js allows us to navigate between pages without performing a full page refresh.

The Link component enables client-side navigation, making page transitions faster and providing a smoother user experience.

Why Not Use a Normal <a> Tag?

A normal anchor tag reloads the entire page.

Normal HTML Link
<a href="/about">
  About
</a>

Navigation Flow:

Page Reload
Current Page
      │
      ▼
Browser Request
      │
      ▼
Server
      │
      ▼
Full Page Reload

This is slower because the browser reloads the entire page.

Using the Link Component

Next.js provides the Link component for client-side navigation.

Basic Example
import Link from 'next/link';

export default function Home() {

  return (
    <Link href="/about">
      About
    </Link>
  );

}

When we click the link, Next.js updates only the necessary content without reloading the entire page.

Navigation Flow with Link

Client-Side Navigation
Current Page
      │
      ▼
Link Click
      │
      ▼
Next.js Router
      │
      ▼
Update UI

app/page.js
import Link from 'next/link';

export default function Home() {

Navigation Menu Example

Navigation Component
import Link from 'next/link';

export default function Navbar() {

  return (
    <nav>

      <Link href="/">
        Home
      </Link>

      <Link href="/about">
        About
      </Link>

      <Link href="/contact">
        Contact
      </Link>

    </nav>
  );

}

Dynamic Route Navigation

We can navigate to dynamic routes by including route parameters in the URL.

Dynamic Route Link
import Link from 'next/link';

export default function Products() {

  return (
    <Link href="/products/100">
      Product 100
    </Link>
  );

}

This navigates to:

URL
/products/100

Passing Query Parameters

Query Parameters
import Link from 'next/link';

export default function Home() {

  return (
    <Link
      href="/products?id=100"
    >
      Product
    </Link>
  );

}

Generated URL:

URL
/products?id=100

Prefetching

One of the biggest advantages of the Link component is automatic prefetching.

When a link becomes visible in the viewport, Next.js may preload the target page in the background.

Prefetch Flow
Page Loads
      │
      ▼
Link Visible
      │
      ▼
Next.js Prefetches Page
      │
      ▼
User Clicks Link
      │
      ▼
Instant Navigation

Opening External URLs

For external websites, we usually use a normal anchor tag.

External Link
<a
  href="https://example.com"
  target="_blank"
  rel="noopener noreferrer"
>
  Visit Website
</a>

Link vs Anchor Tag

Feature Link <a>
Internal Navigation Yes Yes
Client-Side Navigation Yes No
Page Reload No Yes
Prefetching Yes No
Recommended for Internal Pages Yes No

Pages Router and App Router

The Link component works in both routing systems.

Router Usage
Pages Router import Link from 'next/link'
App Router import Link from 'next/link'
One-Line Definition:
The Link component in Next.js allows us to navigate between pages using client-side routing, providing faster navigation, automatic prefetching, and avoiding full page reloads.
