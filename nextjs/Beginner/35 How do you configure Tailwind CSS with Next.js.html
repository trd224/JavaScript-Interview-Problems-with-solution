35. How do you configure Tailwind CSS with Next.js?

Tailwind CSS is a utility-first CSS framework that allows us to style our Next.js application using utility classes directly inside JSX.

In modern Next.js projects, Tailwind CSS can be added during project creation, or manually installed using Tailwind CSS packages and PostCSS configuration.

Method 1: Configure Tailwind During Project Creation

The easiest way is to select Tailwind CSS while creating a new Next.js project.

Create Next.js App
npx create-next-app@latest my-app

During setup, choose:

CLI Option
Would you like to use Tailwind CSS? Yes

Next.js will automatically create the required Tailwind setup.

Method 2: Manual Setup

If our application already exists, we can install Tailwind CSS manually.

Step 1: Install Tailwind CSS

Terminal
npm install tailwindcss @tailwindcss/postcss postcss

In the latest Tailwind setup, @tailwindcss/postcss is used as the PostCSS plugin.

Step 2: Create PostCSS Config

postcss.config.mjs
const config = {
  plugins: {
    '@tailwindcss/postcss': {}
  }
};

export default config;

Step 3: Import Tailwind in Global CSS

app/globals.css
@import 'tailwindcss';

Step 4: Import Global CSS in Root Layout

app/layout.js
import './globals.css';

export default function RootLayout({ children }) {

  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );

}

Step 5: Use Tailwind Classes

app/page.js
export default function Home() {

  return (
    <main className="min-h-screen bg-slate-900 p-10 text-white">
      <h1 className="text-4xl font-bold text-sky-400">
        Hello Tailwind
      </h1>
    </main>
  );

}

Tailwind with Pages Router

In the Pages Router, global CSS is imported inside pages/_app.js.

styles/globals.css
@import 'tailwindcss';
pages/_app.js
import '../styles/globals.css';

export default function MyApp({ Component, pageProps }) {

  return <Component {...pageProps} />;

}

Tailwind v3 Setup Note

Older projects using Tailwind CSS v3 commonly use tailwind.config.js and these CSS directives:

Tailwind v3 globals.css
@tailwind base;
@tailwind components;
@tailwind utilities;
tailwind.config.js
module.exports = {
  content: [
    './app/**/*.{js,ts,jsx,tsx,mdx}',
    './pages/**/*.{js,ts,jsx,tsx,mdx}',
    './components/**/*.{js,ts,jsx,tsx,mdx}'
  ],
  theme: {
    extend: {}
  },
  plugins: []
};

For new projects, the latest Tailwind setup uses @import 'tailwindcss' and @tailwindcss/postcss. :contentReference[oaicite:0]{index=0}

Common Mistakes

Mistake Fix
Tailwind classes not working Make sure globals.css is imported in layout.js or _app.js
Using old Tailwind directives in new setup Use @import 'tailwindcss' for latest Tailwind
Missing PostCSS config Create postcss.config.mjs
Wrong file location Use app/globals.css for App Router or styles/globals.css for Pages Router

When Should We Use Tailwind CSS?

  • When we want fast UI development.
  • When we want utility-first styling.
  • When we want consistent spacing, colors, and typography.
  • When we want to avoid writing many custom CSS files.
  • When our application needs responsive design quickly.

Interview Tip

A common misconception is that Tailwind replaces CSS completely. In reality, Tailwind provides utility classes, but it still generates normal CSS behind the scenes.
One-Line Definition:
Tailwind CSS is configured in Next.js by installing Tailwind packages, adding the PostCSS plugin, importing Tailwind in global CSS, and then using utility classes directly inside our JSX.
