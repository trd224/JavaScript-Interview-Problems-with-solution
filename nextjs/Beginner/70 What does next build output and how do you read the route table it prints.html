What does next build output and how do you read the route table it prints?

When we run next build, Next.js compiles the application, analyzes every route, generates optimized assets, and prints a route table showing how each route will be rendered in production.

The route table is one of the most important parts of the build output because it tells us whether a route is Static, Dynamic, ISR, or a Route Handler.

Build Command

Terminal
npm run build

or 

next build

What Happens During Build?

Route Symbols

The symbol at the beginning of each route tells us how Next.js will render that route.

Symbol Meaning
○ Static Route (SSG)
● SSG with generateStaticParams()
ƒ Dynamic Route (SSR)

○ Static Route

Example
export default function About() {
  return <h1>About</h1>;
}
Build Output
○ /about

This means the page is generated during build time and served as static HTML.

● Static Dynamic Route

app/blog/[slug]/page.js
export async function generateStaticParams() {

  return [
    { slug: 'nextjs' },
    { slug: 'react' }
  ];

}
Build Output
● /blog/[slug]

Next.js generates static pages for all values returned by generateStaticParams().

ƒ Dynamic Route

app/products/[id]/page.js
export default async function Page({
  params
}) {

  const { id } = await params;

  const product = await fetch(
    `https://api.example.com/products/${id}`
  ).then(res => res.json());

  return <h1>{product.title}</h1>;

}
Build Output
ƒ /products/[id]

This route is rendered dynamically on each request.

Route Handler Example

app/api/users/route.js
export async function GET() {

  return Response.json({
    users: []
  });

}
Build Output
ƒ /api/users

Route Handlers also appear as dynamic functions.

Understanding the Size Column

Example
○ /about      180 B

This represents the size of the route-specific JavaScript.

Understanding First Load JS

Example
○ /about      180 B      98 kB

The First Load JS value includes:

  • Route JavaScript.
  • Shared React runtime.
  • Shared Next.js runtime.
  • Shared framework code.

Shared Bundle

Example
+ First Load JS shared by all
  98 kB

This bundle is downloaded by every route.

How to Read This Table

Analysis
/            → Static Page
/about       → Static Page
/contact     → Static Page

/dashboard   → Dynamic SSR
/products    → Dynamic SSR

Shared JS    → 98 kB

What Should We Look For?

  • Unexpected SSR routes.
  • Large First Load JS sizes.
  • Routes that should be static but became dynamic.
  • Large shared bundles.

Example of Unexpected Dynamic Route

Code
import { cookies } from 'next/headers';

export default async function Page() {

  const cookieStore = await cookies();

  return <h1>Dashboard</h1>;

}
Build Output
ƒ /dashboard

Using cookies() makes the page dynamic.

Route Table Summary

Symbol Meaning
○ Static Route (SSG)
● Static Dynamic Route (generateStaticParams)
ƒ Dynamic Route (SSR)

Important Note

The route table is the easiest way to verify whether Next.js is generating routes as Static (SSG) or Dynamic (SSR). Many performance issues can be discovered simply by inspecting this table after a build.

Interview Tip

After running next build, always inspect the route table. It shows how each route will be rendered, its bundle size, and the amount of JavaScript downloaded by users.
One-Line Definition:
The next build route table shows each route's rendering strategy (SSG or SSR), route size, and first-load JavaScript size, helping us understand how the application will behave in production.
