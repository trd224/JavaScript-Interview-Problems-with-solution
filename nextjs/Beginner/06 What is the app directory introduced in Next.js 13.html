What is the app Directory Introduced in Next.js 13?

The app directory is a new routing system introduced in Next.js 13 that uses the App Router.

The app directory is used to create routes, layouts, loading states, error pages, and Server Components using a folder-based structure.

Why Was the app Directory Introduced?

The older pages directory worked well, but it had limitations when building large applications.

The app directory was introduced to provide:

  • Server Components
  • Nested Layouts
  • Streaming
  • Loading UI
  • Error UI
  • Better Data Fetching
  • Improved Performance

Basic app Directory Structure

App Router Structure
app/
├── page.js
├── layout.js
├── about/
│   └── page.js
└── contact/
    └── page.js

Generated Routes

File Route
app/page.js /
app/about/page.js /about
app/contact/page.js /contact

page.js

Every route must contain a page.js file.

app/about/page.js
export default function About() {

  return (
    <h1>About Page</h1>
  );

}

This creates:

Generated Route
/about

layout.js

A layout is shared UI that appears across multiple pages.

app/layout.js
export default function RootLayout({
  children
}) {

  return (
    <html>
      <body>
        {children}
      </body>
    </html>
  );

}

The layout wraps every page inside the application.

Server Components

Components inside the app directory are Server Components by default.
Server Component Example
export default async function Users() {

  const response = await fetch(
    'https://api.example.com/users'
  );

  const users = await response.json();

  return (
    <div>{users.length}</div>
  );

}

This code runs on the server, not in the browser.

Client Components

To use state, effects, or browser APIs, add:

Client Component Example
'use client';

import { useState } from 'react';

export default function Counter() {

  const [count, setCount] = useState(0);

  return (
    <button
      onClick={() => setCount(count + 1)}
    >
      {count}
    </button>
  );

}

loading.js

Displays a loading UI while a route is loading.

app/loading.js
export default function Loading() {

  return (
    <h2>Loading...</h2>
  );

}

error.js

Displays an error UI when an error occurs.

app/error.js
'use client';

export default function Error({
  error
}) {

  return (
    <h2>Something went wrong</h2>
  );

}

Nested Layouts

Different sections can have their own layouts.

Nested Layout Example
app/
├── layout.js
├── dashboard/
│   ├── layout.js
│   └── page.js

Dashboard pages will use the dashboard layout automatically.

One-Line Definition:
The app directory introduced in Next.js 13 is the new App Router system that uses folder-based routing and provides built-in support for layouts, Server Components, loading states, error handling, and improved performance.
