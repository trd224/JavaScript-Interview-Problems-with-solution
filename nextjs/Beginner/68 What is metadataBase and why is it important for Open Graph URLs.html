68. What is metadataBase and why is it important for Open Graph URLs?

metadataBase is a Metadata API property that defines the base URL for all relative metadata URLs such as Open Graph images, canonical URLs, Twitter images, and alternates.

Without metadataBase, relative URLs like /og-image.jpg cannot be converted into complete URLs such as https://example.com/og-image.jpg.

Why Do We Need metadataBase?

Social media platforms and search engines require absolute URLs.

Invalid Open Graph URL
openGraph: {
  images: ['/og-image.jpg']
}

This only contains a relative path.

Result
/og-image.jpg

Facebook, LinkedIn, WhatsApp, and Twitter cannot reliably use this path because the domain is missing.

Using metadataBase

app/layout.js
export const metadata = {

  metadataBase: new URL('https://example.com'),

  openGraph: {
    images: ['/og-image.jpg']
  }

};

export default function RootLayout({
  children
}) {
  return (
    <html>
      <body>{children}</body>
    </html>
  );
}

Generated Open Graph URL

Result
https://example.com/og-image.jpg

Dynamic Product Example

export async function generateMetadata({
  params
}) {

  return {
    title: `Product ${id}`,

    openGraph: {
      images: [`/products/${id}.jpg`]
    }
  };

}

Important Note

Open Graph images, Twitter images, canonical links, and alternate URLs should generally be absolute URLs. metadataBase allows us to use simple relative paths while Next.js automatically generates complete URLs.

Interview Tip

metadataBase is commonly placed in the root layout because all child pages inherit it, allowing relative Open Graph and canonical URLs to be converted into fully qualified URLs automatically.
One-Line Definition:
metadataBase defines the site's base URL and allows Next.js to automatically convert relative Open Graph, Twitter, and canonical URLs into absolute URLs required by search engines and social platforms.
