How do you Create a Custom 404 Page in Next.js?

A custom 404 page allows us to show our own page when a user visits a route that does not exist.

In the Pages Router, we create pages/404.js. In the App Router, we create app/not-found.js.

Pages Router: Custom 404 Page

In the Pages Router, create a file named 404.js inside the pages directory.

Folder Structure
pages/
├── index.js
├── about.js
└── 404.js
pages/404.js
import Link from 'next/link';

export default function Custom404() {

  return (
    <div>
      <h1>404 - Page Not Found</h1>
      <p>The page we are looking for does not exist.</p>
      <Link href="/">Go Home</Link>
    </div>
  );

}

App Router: Custom 404 Page

In the App Router, create a file named not-found.js inside the app directory.

Folder Structure
app/
├── page.js
├── layout.js
└── not-found.js
app/not-found.js
import Link from 'next/link';

export default function NotFound() {

  return (
    <div>
      <h1>404 - Page Not Found</h1>
      <p>The page we are looking for does not exist.</p>
      <Link href="/">Go Home</Link>
    </div>
  );

}

Show 404 Manually in App Router

If data is not found, we can manually show the 404 page using notFound().

app/products/[id]/page.js
import { notFound } from 'next/navigation';

export default async function Product({ params }) {

  const product = null;

  if (!product) {
    notFound();
  }

  return <h1>Product Details</h1>;

}

Pages Router vs App Router

Router Custom 404 File Location
Pages Router 404.js pages/404.js
App Router not-found.js app/not-found.js

When Do We Use a Custom 404 Page?

  • When a route does not exist.
  • When requested data is not found.
  • When we want to show a branded error page.
  • When we want to provide a link back to the home page.
One-Line Definition:
A custom 404 page in Next.js allows us to display our own Not Found page when a user visits a missing route, using pages/404.js in the Pages Router or app/not-found.js in the App Router.
