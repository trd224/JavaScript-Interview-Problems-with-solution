19. What is Shallow Routing and When Would We Use It?

Shallow Routing is a feature in Next.js that allows us to change the URL without re-running data fetching methods or fully reloading the current page.

Shallow Routing updates the URL and router state while keeping the current page component mounted and avoiding unnecessary data fetching.

Why Do We Need Shallow Routing?

Suppose our application displays products with pagination, sorting, or filters.

When a user changes a filter or page number, we may want to update the URL so it can be bookmarked, refreshed, or shared with others.

Without Shallow Routing, changing the URL may trigger data fetching methods again.

Note:
Shallow Routing is most useful when our application data has already been loaded and we only want to update the URL or UI state. In such cases, re-fetching the same data from the server would be unnecessary.

We may only want the URL to change without causing a complete page refresh cycle.

Basic Example

Pages Router
import { useRouter } from 'next/router';

export default function Products() {

  const router = useRouter();

  const nextPage = () => {

    router.push('/products?page=2', undefined, { shallow: true });  //router.push(url, as, options);

  };

  return (
    <button onClick={nextPage}>
      Next Page
    </button>
  );

}
| Parameter | Purpose                                                               |
| --------- | --------------------------------------------------------------------- |
| `url`     | Actual route to navigate to                                           |
| `as`      | URL shown in the browser (mainly for older dynamic routing use cases) |
| `options` | Extra options such as `shallow`, `scroll`, `locale`                   |

What Happens Internally?

Normal Routing

Flow
URL Change
    │
    ▼
Next.js Navigation
    │
    ▼
getServerSideProps()
or
getStaticProps()
Runs Again
    │
    ▼
Page Re-renders

Shallow Routing

Flow
URL Change
    │
    ▼
Router State Updates
    │
    ▼
Current Component Stays Mounted
    │
    ▼
No Data Fetching Re-execution
    │
    ▼
UI Updates Using New Query Values

Reading the Updated Query Parameter

Example
import { useRouter } from 'next/router';

export default function Products() {

  const router = useRouter();

  return (
    <h2>Page: {router.query.page}</h2>
  );

}

URL: 

/products?page=2

Output: 

Page: 2

Using replace() with Shallow Routing

router.replace()
router.replace('/products?page=3', undefined, { shallow: true });

Unlike push(), replace() does not create a new browser history entry.

Real-World Use Cases

  • Pagination
  • Search Filters
  • Sorting
  • Dashboard State
  • Wizard Steps
  • Tab Navigation

URL Changes

Example: Tab Navigation

URL Changes
/profile?tab=posts
/profile?tab=photos
/profile?tab=settings

Using Shallow Routing, we can switch tabs without re-running page data fetching methods.

Important Limitation

Shallow Routing only works when navigating within the same page.

Valid: 

/products?page=1
       ↓
/products?page=2

Not Valid: 

/products?page=1
       ↓
/about

Navigating to a different page always performs normal routing.

Data Fetching Methods Affected

When using Shallow Routing, these methods are not re-executed:

  • getServerSideProps()
  • getStaticProps()
  • getInitialProps()

Pages Router vs App Router

Feature Pages Router App Router
shallow option ✅ Supported ❌ Not Available
router.push(..., { shallow: true }) ✅ ❌
Main Purpose Update URL without re-fetching data Uses a different routing architecture

App Router Note

The App Router does not support the shallow option.

Instead, App Router uses:

  • useSearchParams()
  • Server Components
  • Client Components
  • router.push()
  • router.replace()

Because of the new architecture, traditional shallow routing is generally unnecessary in most App Router scenarios.

One-Line Definition:
Shallow Routing is a Pages Router feature that allows us to update the URL and query parameters without re-running data fetching methods or fully reloading the current page.

Good Use Case for Shallow Routing

Shallow Routing is useful when all required data is already available in the browser and we only want to update the URL to reflect UI state.

Example Data
const products = allProducts;

Suppose our application supports different views.

Current URL
/products?view=grid

User switches to List View:

Updated URL
/products?view=list

In this case, no new server data is needed because all products are already loaded.

What We Want
✔ Update URL
✔ Update UI
✔ Update router.query

✘ Don't run getServerSideProps()
✘ Don't run getStaticProps()
✘ Don't make API calls again

This is an ideal use case for Shallow Routing because only the UI state changes while the underlying data remains the same.

Bad Use Case for Shallow Routing

Suppose each page requires new data from the server.

Current URL
/products?page=1

User navigates to:

Updated URL
/products?page=2

Page 2 contains completely different products:

Page Data
Page 1:
Product 1
Product 2
Product 3

Page 2:
Product 4
Product 5
Product 6

In this scenario, we must fetch new data from the server.

What We Need
✔ Run getServerSideProps()
✔ Fetch new data
✔ Render new products

This is a poor use case for Shallow Routing because the page content depends on new server data.
