38. How do you configure remote image domains in next.config.js?

By default, the next/image component only optimizes local images and trusted remote sources.

To use external images with next/image, we must explicitly allow the image domain in next.config.js.

Why Is Configuration Required?

Suppose our application displays an image from:

Remote Image URL
https://images.example.com/product.jpg

Using it directly:

Image Component
import Image from 'next/image';

export default function Product() {

  return (
    <Image
      src="https://images.example.com/product.jpg"
      alt="Product"
      width={500}
      height={300}
    />
  );

}

Will produce an error because Next.js does not trust unknown domains.

Error Example

Runtime Error
Invalid src prop

Hostname "images.example.com"
is not configured under images
in your next.config.js

Solution: Configure Remote Domains

next.config.js
/** @type {import('next').NextConfig} */

const nextConfig = {
  images: {
    domains: [
      'images.example.com'
    ]
  }
};

module.exports = nextConfig;

Now Next.js allows images from: 

https://images.example.com

Complete Example

next.config.js
const nextConfig = {
  images: {
    domains: [
      'images.example.com'
    ]
  }
};

module.exports = nextConfig;
app/page.js
import Image from 'next/image';

export default function Home() {

  return (
    <Image
      src="https://images.example.com/product.jpg"
      alt="Product"
      width={600}
      height={400}
    />
  );

}

Modern Approach: remotePatterns

In newer versions of Next.js, remotePatterns is preferred because it provides more control.

next.config.js
const nextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'images.example.com',
        pathname: '/**'
      }
    ]
  }
};

module.exports = nextConfig;

Why remotePatterns Is Better?

With domains, all images from the domain are allowed.

With remotePatterns, we can restrict:

  • Protocol
  • Hostname
  • Port
  • Pathname

Example

Specific Folder Only
const nextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'images.example.com',
        pathname: '/products/**'
      }
    ]
  }
};

module.exports = nextConfig;

Allowed: 

https://images.example.com/products/shirt.jpg
https://images.example.com/products/shoes.jpg

Blocked: 

https://images.example.com/users/profile.jpg

Multiple Domains

next.config.js
const nextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'images.example.com'
      },
      {
        protocol: 'https',
        hostname: 'cdn.example.com'
      }
    ]
  }
};

module.exports = nextConfig;

Why Does Next.js Restrict Domains?

  • Security
  • Performance
  • Prevent Abuse
  • Control Optimization Sources

Next.js needs to know which remote images it is allowed to download and optimize.

Important Note

After changing next.config.js, we must restart the Next.js development server because configuration files are loaded only during startup.

Interview Tip

A common misconception is that next/image can load any remote image automatically. In reality, remote image domains must be explicitly configured in next.config.js before Next.js will optimize and serve them.
One-Line Definition:
Remote image domains are configured in next.config.js using images.domains or the recommended images.remotePatterns option, allowing Next.js to securely optimize external images.
