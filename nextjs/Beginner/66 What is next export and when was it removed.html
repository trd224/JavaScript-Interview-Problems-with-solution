What is next export and when was it removed?

next export was a Next.js command used to generate a fully static website consisting of HTML, CSS, and JavaScript files that could be hosted on any static hosting service.

The next export command was deprecated in Next.js 13 and removed in Next.js 14. It was replaced by the output: 'export' configuration.

What Did next export Do?

It converted a Next.js application into static files.

Old Commands
npm run build

next export

or 

next build
next export

The generated files could be hosted on any static web server.

Limitations of next export

Since everything became static files, many Next.js features could not work.

Feature Supported?
SSG ✅
Static Assets ✅
SSR ❌
ISR ❌
API Routes ❌
Middleware ❌
Server Actions ❌

Why Was next export Removed?

The App Router introduced a new static export system that made the separate next export command unnecessary.

Instead of running a separate export command, Next.js now uses the output: 'export' option in next.config.js.

Modern Replacement

next.config.js
const nextConfig = {
  output: 'export'
};

module.exports = nextConfig;

Build Command

Terminal
npm run build

No separate export command is needed.

Generated Output

Output Folder
out
├── index.html
├── about
│   └── index.html
└── _next

When Should We Use output: 'export'?

  • Static websites.
  • Documentation sites.
  • Portfolio websites.
  • Landing pages.
  • Sites hosted on static hosting providers.

When Should We Not Use It?

  • SSR applications.
  • ISR applications.
  • API Routes.
  • Server Actions.
  • Middleware.
  • Authentication requiring server rendering.

Important Note

If you see old tutorials using next export, the modern equivalent is setting output: 'export' in next.config.js and running a normal production build.
