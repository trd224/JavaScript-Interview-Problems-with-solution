29. What is the 'use server' directive and where can you place it?

'use server' is a Next.js directive that marks a function or file to run only on the server.

Simple Meaning:
'use server' tells Next.js, "This code must execute on the server and never in the browser."

Why Do We Need 'use server'?

Some code should never be sent to the browser.

Examples:

  • Database queries
  • Secret API keys
  • Password hashing
  • File system access
  • Server-side business logic
Example
await db.user.create({
  data: {
    name: 'Rizwan'
  }
});

The browser cannot access the database directly, so this code must run on the server.

Function-Level use server

We can place 'use server' inside an individual function.

Example
export default function Page() {

  async function createUser(formData: FormData) {
    'use server';

    await db.user.create({
      data: {
        name: formData.get('name')
      }
    });
  }

  return (
    <form action={createUser}>
      <input name="name" />
      <button>Save</button>
    </form>
  );

}

Only the createUser() function runs on the server.

File-Level use server

We can place 'use server' at the top of a file.

actions.ts
'use server';

export async function createUser() {

  await db.user.create({
    data: {
      name: 'Rizwan'
    }
  });

}

export async function deleteUser(id: string) {

  await db.user.delete({
    where: { id }
  });

}

Now every exported function in this file becomes a Server Action.

How Next.js Treats It

Flow
Client Component
       ↓
Calls Server Action
       ↓
Next.js Internal Request
       ↓
Server
       ↓
Execute Function
       ↓
Return Result

The browser never executes the server code directly.

Using use server with Forms

Server Action
export async function createUser(
  formData: FormData
) {
  'use server';

  await db.user.create({
    data: {
      name: formData.get('name')
    }
  });
}
Form
<form action={createUser}>
  <input name="name" />
  <button>Save</button>
</form>

When the form is submitted: 

Submit Form
      ↓
Server Action
      ↓
Database
      ↓
Response

Using use server from a Client Component

actions.ts
'use server';

export async function createUser() {

  await db.user.create({
    data: {
      name: 'Rizwan'
    }
  });

}
Client Component
'use client';

import { createUser } from './actions';

export default function Button() {

  return (
    <button
      onClick={() => createUser()}
    >
      Create User
    </button>
  );

}

Although the button runs in the browser, the action executes on the server.

Where Can We Place use server?

Location Allowed?
Inside a Function ✅ Yes
Top of actions.ts File ✅ Yes
Inside Client Component File ❌ No
Inside Event Handler ❌ No
Inside useEffect() ❌ No

use server vs use client

Feature use server use client
Runs on Server ✅ ❌
Runs in Browser ❌ ✅
Database Access ✅ ❌
Access Secret Keys ✅ ❌
useState/useEffect ❌ ✅

Easy Memory Trick

Remember
use server
      ↓
Backend Code


use client
      ↓
Browser Code
One-Line Definition:
'use server' is a Next.js directive that marks functions or files to execute only on the server, allowing secure access to databases, secrets, and backend logic.
