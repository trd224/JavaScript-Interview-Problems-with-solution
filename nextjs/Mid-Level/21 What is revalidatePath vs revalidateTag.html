21. What is revalidatePath vs revalidateTag?

Both revalidatePath() and revalidateTag() are used for on-demand revalidation in Next.js.

The main difference is:

revalidatePath() refreshes a specific route.
revalidateTag() refreshes cached data associated with a tag.

revalidatePath()

Used when we want to invalidate a specific page or route.

Example
revalidatePath('/products');

This tells Next.js: 

Clear Cache For

/products

Flow

Product Updated
       ↓
revalidatePath('/products')
       ↓
Products Page Cache Cleared
       ↓
Next Visit Regenerates Page

Real Example

Product Page
export default async function ProductsPage() {
  const res = await fetch('https://api.myshop.com/products');
  const products = await res.json();

  return <h1>Products</h1>;
}

After product update:

API Route
import { revalidatePath } from 'next/cache';

revalidatePath('/products');

Only the /products page is refreshed.

Problem With revalidatePath()

Suppose product data appears on multiple pages: 

/products

/home

/cart

/search

If we only run: 

revalidatePath('/products');

Then: 

/products      ✅ Updated

/home          ❌ Old Data

/cart          ❌ Old Data

/search        ❌ Old Data
revalidatePath() only affects the specified route.

revalidateTag()

Used when multiple pages share the same cached data.

Step 1: Tag Fetches

Products Page
const res = await fetch('https://api.myshop.com/products', {
  next: { tags: ['products'] }
});

Now this fetch belongs to: 

products

tag.

Step 2: Revalidate Tag

API Route
import { revalidateTag } from 'next/cache';

revalidateTag('products', 'max');

Flow

Product Updated
       ↓
revalidateTag('products')
       ↓
All Product Cache Cleared
       ↓
Next Requests Get Fresh Data

Real World Example

Product data is used in: 

Home Page

Products Page

Cart Page

Search Page

All pages fetch using:

Tagged Fetch
await fetch('https://api.myshop.com/products', {
  next: { tags: ['products'] }
});

After update:

Revalidate
revalidateTag('products', 'max');

Result: 

Home Page      ✅ Updated

Products Page  ✅ Updated

Cart Page      ✅ Updated

Search Page    ✅ Updated
One tag refreshes every page using that data.

Visual Comparison

revalidatePath()

Product Updated
       ↓
revalidatePath('/products')
       ↓
Only /products Refreshed

revalidateTag()

Product Updated
       ↓
revalidateTag('products')
       ↓
Every Fetch Using
products Tag Refreshed

Complete Example

Home Page

app/page.tsx
const res = await fetch('https://api.myshop.com/products', {
  next: { tags: ['products'] }
});

Products Page

app/products/page.tsx
const res = await fetch('https://api.myshop.com/products', {
  next: { tags: ['products'] }
});

Cart Page

app/cart/page.tsx
const res = await fetch('https://api.myshop.com/products', {
  next: { tags: ['products'] }
});

After Product Update

Revalidation
revalidateTag('products', 'max');

All three pages automatically get fresh data.

When To Use Which?

Situation Best Choice
Refresh one page revalidatePath()
Refresh many pages using same data revalidateTag()
Blog post updated revalidatePath('/blog/post-1')
Product data updated revalidateTag('products')
User profile page updated revalidatePath('/profile')
Shared catalog data updated revalidateTag('catalog')

One-Line Definition:
revalidatePath() invalidates a specific route's cache, while revalidateTag() invalidates all cached fetches associated with a specific tag, regardless of which pages use them.
