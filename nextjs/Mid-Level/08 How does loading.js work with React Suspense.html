8. How does loading.js work with React Suspense?

The loading.js file is a special App Router file that automatically creates a React Suspense boundary around a route segment and displays a loading UI while that segment is being rendered.

In simple words, loading.js is Next.js's automatic Suspense fallback for a route.

Why Do We Need loading.js?

Sometimes a page takes time to render because it is waiting for:

  • Database Queries
  • API Requests
  • Server Components
  • Dynamic Data

Instead of showing a blank screen, we can show a loading UI.

Basic Example

Folder Structure
app/
│
├── dashboard/
│   ├── page.tsx
│   └── loading.tsx
loading.tsx
export default function Loading() {

  return <h2>Loading Dashboard...</h2>;

}
page.tsx
export default async function Page() {

  await new Promise(resolve =>
    setTimeout(resolve, 3000)
  );

  return <h1>Dashboard</h1>;

}

What Happens?

Flow
User Opens Route
        │
        ▼
Page Starts Rendering
        │
        ▼
Page Is Slow
        │
        ▼
Show loading.js
        │
        ▼
Page Ready
        │
        ▼
Replace Loading UI

The user sees the loading UI while the page is rendering.

How Does Suspense Fit In?

Internally, Next.js automatically wraps the route in a Suspense boundary.

Conceptual Implementation
<Suspense fallback={<Loading />}>
  <Page />
</Suspense>

Next.js creates this Suspense boundary automatically.

You do not manually write Suspense when using loading.js. Next.js creates it for you.

Equivalent Manual Suspense

Without loading.js
import { Suspense } from 'react';

export default function Dashboard() {
  return (
    <Suspense fallback={<h2>Loading...</h2>}>
      <UserStats />
    </Suspense>
  );
}

This is very similar to what loading.js does automatically.

Important Difference

The scope is different.

Component-Level Example

page.tsx
import { Suspense } from 'react';
import UserStats from './UserStats';

export default function Dashboard() {
  return (
    <>
      <Header />
      <Suspense fallback={<h2>Loading Stats...</h2>}>
        <UserStats />
      </Suspense>
      <Footer />
    </>
  );
}

Now only the UserStats section shows a loading state.

Result
Header Appears
Footer Appears

Loading Stats...

UserStats Arrives Later

This is more granular than loading.js.

loading.js and PPR

When using Partial Prerendering (PPR), loading.js often acts as the fallback content for dynamic sections.

PPR Flow
Static Shell
      │
      ▼
Show Loading UI
      │
      ▼
Dynamic Content Ready
      │
      ▼
Replace Loading UI

This helps users see content immediately instead of waiting for the entire page.

loading.js vs Template vs Layout

File Purpose
layout.js Persistent Shared UI
template.js Remount Shared Wrapper
loading.js Loading Fallback UI

One-Line Definition:
The loading.js file automatically creates a React Suspense fallback for a route segment and displays loading UI while the page or route content is being rendered.
