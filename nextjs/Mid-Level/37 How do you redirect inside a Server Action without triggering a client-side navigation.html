37. How do you redirect inside a Server Action without triggering a client-side navigation?

Inside a Server Action, we use redirect() from next/navigation.

Simple Meaning:
After the Server Action completes its work, Next.js can redirect the user from the server without using router.push().

Import redirect()

Import
import { redirect } from 'next/navigation';

Basic Example

Suppose we create a user and then want to redirect to the users page.

Server Action
'use server';

import { redirect } from 'next/navigation';

export async function createUser(formData: FormData) {
  const name = formData.get('name') as string;

  await db.user.create({ data: { name } });

  redirect('/users');
}
Page
import { createUser } from './actions';

export default function Page() {
  return (
    <form action={createUser}>
      <input name="name" />
      <button>Create User</button>
    </form>
  );
}

Flow

What Happens?
Submit Form
      ↓
Server Action Runs
      ↓
Database Updated
      ↓
redirect('/users')
      ↓
User Lands On /users

Important: redirect() Stops Execution

Once redirect() is called, no code below it will execute.

Example
'use server';

import { redirect } from 'next/navigation';

export async function createUser() {
  await db.user.create({ data: { name: 'Rizwan' } });

  redirect('/users');

  console.log('Never runs'); // ❌
}

Think of redirect() like throwing a special redirect exception.

Real World Login Example

Login Action
'use server';

import { redirect } from 'next/navigation';

export async function login(formData: FormData) {
  const email = formData.get('email') as string;
  const password = formData.get('password') as string;

  const user = await validateUser(email, password);

  if (!user) {
    return { success: false, message: 'Invalid credentials' };
  }

  await createSession(user.id);

  redirect('/dashboard');
}

After successful login, the user is redirected to the dashboard directly from the server.

redirect() vs router.push()

Feature redirect() router.push()
Runs On Server Client
Used In Server Actions ✅ ❌
Used In Client Components ❌ ✅
Navigation Type Server Driven Client Driven
Client Navigation Example
'use client';

import { useRouter } from 'next/navigation';

export default function Page() {
  const router = useRouter();

  return (
    <button onClick={() => router.push('/users')}>
      Go To Users
    </button>
  );
}

Can We Return Data And Redirect?

No. A single execution path can either return data or redirect.

Wrong
'use server';

import { redirect } from 'next/navigation';

export async function createUser() {
  redirect('/users');

  return { success: true }; // ❌ Never reached
}
Correct
'use server';

import { redirect } from 'next/navigation';

export async function createUser() {
  await db.user.create({ data: { name: 'Rizwan' } });

  redirect('/users');
}

redirect() With Validation

Example
'use server';

import { redirect } from 'next/navigation';

export async function createUser(formData: FormData) {
  const name = formData.get('name') as string;

  if (!name) {
    return { success: false, message: 'Name is required' };
  }

  await db.user.create({ data: { name } });

  redirect('/users');
}
Flow
Validation Fails
       ↓
Return Error


Validation Passes
       ↓
Save User
       ↓
Redirect

Why Is This Not Client-Side Navigation?

The redirect decision is made on the server while the Server Action is executing.

Server Redirect Flow
Browser
    ↓
Submit Form
    ↓
Server Action
    ↓
redirect('/users')
    ↓
Server Sends Redirect Response
    ↓
Browser Navigates

No router.push() is executed in the browser.

Server Component vs Client Component

Scenario Redirect API
Server Action redirect()
Server Component redirect()
Client Component router.push()

One-Line Definition:
Inside a Server Action, we redirect by calling redirect() from next/navigation, which performs a server-driven redirect and immediately stops further execution of the action.
