What is the priority prop on next/image and when must you set it?

The priority prop tells Next.js that an image is important and should be loaded immediately instead of being lazy-loaded.

Simple Meaning:
Use priority for the most important image visible when the page first loads, usually the Hero or LCP image.

Why Does next/image Lazy Load By Default?

Next.js automatically lazy-loads most images to improve performance.

Default Image
import Image from 'next/image';

export default function Page() {
  return (
    <Image
      src="/product.jpg"
      alt="Product"
      width={600}
      height={400}
    />
  );
}
Flow
Page Loads
      ↓
Visible Content Loads
      ↓
Image Loads When Needed

This is good for most images.

What Does priority Do?

The priority prop tells Next.js to preload the image and fetch it as soon as possible.

Priority Image
import Image from 'next/image';

export default function Page() {
  return (
    <Image
      src="/hero.jpg"
      alt="Hero"
      width={1200}
      height={600}
      priority
    />
  );
}
Flow
Page Starts Loading
      ↓
Image Request Starts Immediately
      ↓
Hero Image Appears Faster

What Problem Does It Solve?

Many websites have a large hero image at the top of the page.

Without priority
HTML Loads
      ↓
CSS Loads
      ↓
JavaScript Loads
      ↓
Hero Image Loads Later
      ↓
Slow LCP Score
With priority
HTML Loads
      ↓
Hero Image Preloaded
      ↓
Hero Image Appears Faster
      ↓
Better LCP Score

What Is LCP?

LCP (Largest Contentful Paint) measures how quickly the largest visible content appears on the screen.

Often the LCP element is:

  • Hero image
  • Banner image
  • Large product image
  • Landing page image
Important:
If an image becomes the LCP element, Next.js usually recommends adding the priority prop.

Real World Example

E-Commerce Hero Banner
import Image from 'next/image';

export default function HomePage() {
  return (
    <Image
      src="/banner.jpg"
      alt="Summer Sale"
      width={1600}
      height={700}
      priority
    />
  );
}

Because the banner is visible immediately after page load, it should use priority.

When Must You Use priority?

Image Type Use priority?
Main Hero Image ✅ Yes
LCP Image ✅ Yes
Above-the-Fold Banner ✅ Yes
Logo (Large & Visible) Sometimes
Gallery Images ❌ No
Below-the-Fold Images ❌ No
Product Images Far Down Page ❌ No

When Should You Avoid priority?

Do not add priority to every image.

Bad Practice:
Using priority on many images forces the browser to download all of them immediately, which can slow down the page.
Bad Example
<Image src="/img1.jpg" alt="" width={400} height={300} priority />
<Image src="/img2.jpg" alt="" width={400} height={300} priority />
<Image src="/img3.jpg" alt="" width={400} height={300} priority />
<Image src="/img4.jpg" alt="" width={400} height={300} priority />

Usually only one or two important images should use it.

What Happens Internally?

When priority is used, Next.js automatically inserts preload hints.

Conceptual Output
<link
  rel="preload"
  as="image"
  href="/hero.jpg"
/>

This tells the browser to fetch the image earlier.

priority vs Lazy Loading

Feature priority Default Image
Preloaded ✅ ❌
Lazy Loaded ❌ ✅
Good For LCP ✅ ❌
Good For Below Fold ❌ ✅
