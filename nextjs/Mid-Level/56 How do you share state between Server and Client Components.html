How do you share state between Server and Client Components?

Server Components and Client Components do not share live React state directly. Instead, we pass server data to Client Components through props, and Client Components manage interactive state in the browser.

Simple Meaning:
Server Components can send initial data to Client Components, but live state like useState() belongs only inside Client Components.

Important Rule

Component Type Can Use useState? Main Role
Server Component ❌ No Fetch data and render HTML/RSC payload
Client Component ✅ Yes Handle interaction and browser state

1. Pass Server Data To Client Component Using Props

This is the most common way.

Server Component - page.tsx
import Counter from './Counter';

export default async function Page() {
  const initialCount = await getCartCount();

  return <Counter initialCount={initialCount} />;
}
Client Component - Counter.tsx
'use client';

import { useState } from 'react';

export default function Counter({ initialCount }: { initialCount: number }) {
  const [count, setCount] = useState(initialCount);

  return (
    <button onClick={() => setCount(count + 1)}>
      Cart: {count}
    </button>
  );
}
Flow
Server Component
      ↓
Fetch initialCount
      ↓
Pass initialCount as prop
      ↓
Client Component
      ↓
useState(initialCount)
      ↓
User interaction updates client state

2. Server State vs Client State

State Type Where It Lives Example
Server State Database / API / Server Component Products, user profile, cart count from DB
Client State Browser / Client Component Modal open, selected tab, input value
Best Practice:
Fetch real data on the server, then pass it as props to the Client Component only when interaction is needed.

3. Client Component Can Update Server State Using Server Action

If the user changes data, call a Server Action and then update UI.

actions.ts
'use server';

export async function updateCart(productId: string) {
  await db.cart.update({ where: { productId }, data: { quantity: 1 } });

  return { success: true };
}
AddToCartButton.tsx
'use client';

import { useState } from 'react';
import { updateCart } from './actions';

export default function AddToCartButton({ productId }: { productId: string }) {
  const [added, setAdded] = useState(false);

  async function handleClick() {
    const result = await updateCart(productId);
    if (result.success) setAdded(true);
  }

  return <button onClick={handleClick}>{added ? 'Added' : 'Add To Cart'}</button>;
}
Flow
Client Button Click
        ↓
Server Action Runs
        ↓
Database Updated
        ↓
Client State Updates

4. Refresh Server Components After Mutation

After updating data, we can refresh the route to get fresh Server Component data.

Client Component
'use client';

import { useRouter } from 'next/navigation';
import { updateCart } from './actions';

export default function AddToCartButton({ productId }: { productId: string }) {
  const router = useRouter();

  async function handleClick() {
    await updateCart(productId);
    router.refresh();
  }

  return <button onClick={handleClick}>Add To Cart</button>;
}

router.refresh() asks Next.js to re-fetch Server Components for the current route.

Flow
Server Action Updates DB
        ↓
router.refresh()
        ↓
Server Component Re-renders
        ↓
Fresh Data Sent To Client

5. Use URL Search Params For Shareable State

For filters, sorting, pagination, and tabs that should be shareable, use URL search params.

Client Component
'use client';

import { useRouter, useSearchParams } from 'next/navigation';

export default function ProductFilter() {
  const router = useRouter();
  const searchParams = useSearchParams();

  function changeCategory(category: string) {
    router.push(`/products?category=${category}`);
  }

  return <button onClick={() => changeCategory('mobile')}>Mobiles</button>;
}
Server Component
export default async function ProductsPage({ searchParams }: { searchParams: Promise<{ category?: string }> }) {
  const { category } = await searchParams;
  const products = await getProducts(category);

  return <ProductList products={products} />;
}
Use URL State For:
Filters, sorting, pagination, search query, selected category, and shareable tabs.

6. Use Context Only Inside Client Components

React Context with state works only in Client Components.

ThemeProvider.tsx
'use client';

import { createContext, useContext, useState } from 'react';

const ThemeContext = createContext(null);

export function ThemeProvider({ children }: { children: React.ReactNode }) {
  const [theme, setTheme] = useState('dark');

  return <ThemeContext.Provider value={{ theme, setTheme }}>{children}</ThemeContext.Provider>;
}

export function useTheme() {
  return useContext(ThemeContext);
}

Server Components can be rendered inside a Client Provider as children, but they cannot read client context directly.

What We Cannot Do

Important:
We cannot directly share live useState() between Server Components and Client Components.
Wrong Idea
Server Component useState
        ↓
Client Component useState
        ↓
Shared live state

This does not work because Server Components run on the server, while Client Components run in the browser.

Best Patterns

Need Best Pattern
Initial data from database Server Component fetches and passes props
Button click / modal / tabs Client Component useState
Update database Server Action
Refresh server data after update router.refresh() or revalidatePath()
Share filter/search state URL search params
Global browser state Client Context / Zustand / Redux

One-Line Definition:
We share state between Server and Client Components by passing server data as props to Client Components, managing interactive state on the client, and syncing changes back to the server using Server Actions, URL params, or route refreshes.
