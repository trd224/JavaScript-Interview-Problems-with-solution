54. What is the staleTimes config option introduced in Next.js 14.2 for Router Cache?

staleTimes is an experimental Next.js configuration that allows us to control how long page segments remain reusable inside the client-side Router Cache before Next.js considers them stale and fetches fresh data. :contentReference[oaicite:0]{index=0}

Simple Meaning:
staleTimes controls how long Next.js keeps previously visited or prefetched pages in the browser's Router Cache.

Why Was staleTimes Introduced?

Before Next.js 14.2, Router Cache durations were mostly fixed and controlled internally by Next.js.

Next.js 14.2 introduced staleTimes so we can decide how long cached route segments should remain reusable on the client. :contentReference[oaicite:1]{index=1}

Without staleTimes
Visit Page
      ↓
Page Cached
      ↓
Next.js Uses Internal Cache Rules
With staleTimes
Visit Page
      ↓
Page Cached
      ↓
Custom Expiration Time
      ↓
Fresh Data When Needed

Basic Configuration

next.config.js
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180
    }
  }
};

module.exports = nextConfig;

Values are specified in seconds. :contentReference[oaicite:2]{index=2}

The Two staleTimes Options

Option Purpose
dynamic Cache duration for dynamic route navigations
static Cache duration for static or fully prefetched routes

The dynamic and static settings correspond to different Router Cache scenarios. :contentReference[oaicite:3]{index=3}

dynamic

Used when a page is neither statically generated nor fully prefetched. :contentReference[oaicite:4]{index=4}

Example
experimental: {
  staleTimes: {
    dynamic: 30
  }
}
Flow
Visit Dynamic Page
        ↓
Store In Router Cache
        ↓
30 Seconds Pass
        ↓
Cache Becomes Stale
        ↓
Fresh Navigation Fetches Again

In Next.js 15, the default value became 0 seconds. :contentReference[oaicite:5]{index=5}

static

Used for statically generated pages, fully prefetched routes, or routes loaded using router.prefetch(). :contentReference[oaicite:6]{index=6}

Example
experimental: {
  staleTimes: {
    static: 300
  }
}
Flow
Prefetch Route
       ↓
Store In Router Cache
       ↓
5 Minutes Pass
       ↓
Cache Becomes Stale
       ↓
Fresh Data Retrieved

The default static value is 5 minutes. :contentReference[oaicite:7]{index=7}

What Cache Does staleTimes Affect?

A very common confusion is thinking it affects:

  • Data Cache
  • Full Route Cache
  • ISR Cache

It does not.

Important:
staleTimes only affects the Client-side Router Cache (the cache used during navigation in the browser). It does not control ISR revalidation, fetch caching, Data Cache, or Full Route Cache behavior. :contentReference[oaicite:8]{index=8}

staleTimes vs revalidate

Feature staleTimes revalidate
Affects Router Cache ✅ ❌
Affects ISR ❌ ✅
Runs In Browser ✅ ❌
Runs On Server ❌ ✅
Controls Navigation Cache ✅ ❌
