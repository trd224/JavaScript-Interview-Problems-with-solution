What strategies does the next/script component offer and when do you use each?

next/script allows us to control when third-party JavaScript files are loaded so they do not negatively impact page performance.

Simple Meaning:
The strategy prop decides when a script should be downloaded and executed.

Why Use next/script?

Normally, a script tag can block page rendering.

Traditional Script
<script src="https://example.com/script.js"></script>
Problem
Browser Reads HTML
        ↓
Downloads Script
        ↓
Executes Script
        ↓
Continues Rendering

Large scripts can slow down page loading.

next/script provides optimized loading strategies.

Basic Usage

Example
import Script from 'next/script';

export default function Page() {
  return (
    <>
      <Script
        src="https://example.com/script.js"
        strategy="afterInteractive"
      />
    </>
  );
}

Available Strategies

Strategy When It Runs
beforeInteractive Before Next.js hydration
afterInteractive After hydration begins
lazyOnload During browser idle time
worker Inside a web worker

1. beforeInteractive

Loads and executes before Next.js hydrates the page.

Example
<Script
  src="https://example.com/script.js"
  strategy="beforeInteractive"
/>
Flow
HTML Arrives
      ↓
Script Loads
      ↓
Script Executes
      ↓
Hydration Starts
Use For:
Critical scripts that must run before the application becomes interactive.

Examples:

  • Bot detection
  • Security scripts
  • Consent management
  • Critical monitoring scripts
Warning:
Use sparingly because it can delay page interactivity.

2. afterInteractive (Default)

This is the default strategy.

Example
<Script
  src="https://example.com/script.js"
  strategy="afterInteractive"
/>
Flow
HTML Loads
      ↓
Hydration Starts
      ↓
Script Downloads
      ↓
Script Executes
Use For:
Scripts needed soon after page interaction begins.

Examples:

  • Google Analytics
  • Tag Manager
  • Customer chat widgets
  • A/B testing tools

3. lazyOnload

Loads only when the browser is idle.

Example
<Script
  src="https://example.com/script.js"
  strategy="lazyOnload"
/>
Flow
Page Loads
      ↓
Hydration Completes
      ↓
Browser Becomes Idle
      ↓
Script Downloads
      ↓
Script Executes
Use For:
Low-priority scripts that are not immediately needed.

Examples:

  • Social sharing widgets
  • Heatmaps
  • Feedback widgets
  • Marketing tools

4. worker

Loads the script inside a Web Worker instead of the main thread.

Example
<Script
  src="https://example.com/script.js"
  strategy="worker"
/>
Flow
Script Loads
      ↓
Runs In Worker
      ↓
Main Thread Remains Free
Use For:
Heavy third-party scripts that support running in a worker environment.
Important:
Not all third-party scripts work inside a Web Worker.

Real World Examples

Script Recommended Strategy
Cookie Consent beforeInteractive
Google Analytics afterInteractive
Google Tag Manager afterInteractive
Hotjar lazyOnload
Feedback Widget lazyOnload
Heavy Analytics SDK worker (if supported)

Decision Flow

Complete Example

Layout
import Script from 'next/script';

export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        {children}

        <Script
          src="https://www.googletagmanager.com/gtag/js"
          strategy="afterInteractive"
        />

        <Script
          src="https://example.com/feedback.js"
          strategy="lazyOnload"
        />
      </body>
    </html>
  );
}

One-Line Definition:
The strategy prop of next/script controls when a script loads and executes, with beforeInteractive, afterInteractive, lazyOnload, and worker providing different performance and loading behaviors.
