3. What is the Difference Between Server Components and Client Components?

Server Components run only on the server and do not send their component JavaScript to the browser, while Client Components run in the browser and support interactivity such as state, effects, and event handlers.

In the App Router, components are Server Components by default. A component becomes a Client Component only when we add the 'use client' directive.

Server Component Example

app/page.js
export default async function Page() {

  const products =
    await fetch(
      'https://dummyjson.com/products'
    ).then(res => res.json());

  return (
    <ul>
      {products.products.map(product => (
        <li key={product.id}>
          {product.title}
        </li>
      ))}
    </ul>
  );

}

This is a Server Component because there is no 'use client'.

Client Component Example

app/Counter.js
'use client';

import { useState } from 'react';

export default function Counter() {

  const [count, setCount] = useState(0);

  return (
    <button
      onClick={() => setCount(count + 1)}
    >
      {count}
    </button>
  );

}

This is a Client Component because it contains the 'use client' directive.

What Can Server Components Do?

Allowed
await fetch(...)

await cookies()

await headers()

await database.query(...)
  • Fetch data directly.
  • Access databases.
  • Access cookies.
  • Access request headers.
  • Keep secrets on the server.

What Server Components Cannot Do?

Not Allowed
useState()

useEffect()

onClick={...}

window.localStorage

Server Components cannot use browser APIs or React client hooks.

What Can Client Components Do?

  • useState
  • useEffect
  • useRef
  • Event handlers
  • Browser APIs
  • Animations
  • Forms
Example
'use client';

import { useState } from 'react';

export default function Login() {

  const [email, setEmail] = useState('');

  return (
    <input
      value={email}
      onChange={e => setEmail(e.target.value)}
    />
  );

}

Mixing Server and Client Components

app/page.js
import Counter from './Counter';

export default function Page() {

  return (
    <>
      <h1>Dashboard</h1>
      <Counter />
    </>
  );

}

import { useState } from 'react';

Why Use Server Components?

Because they reduce the amount of JavaScript sent to the browser.

Without Server Components
Header.js
Sidebar.js
Products.js
Footer.js

↓ Sent To Browser
With Server Components
Header.js   ❌
Sidebar.js  ❌
Products.js ❌
Footer.js   ❌

Only HTML Sent

This improves performance and reduces bundle size.

Interview Tip

Server Components are used for rendering content and fetching data on the server, while Client Components are used for browser interactivity such as state management, effects, and event handlers.
One-Line Definition:
Server Components execute only on the server and do not send component JavaScript to the browser, while Client Components execute in the browser and support interactivity using React hooks and event handlers.
