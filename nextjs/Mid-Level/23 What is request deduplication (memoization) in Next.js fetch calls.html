23. What is Request Deduplication (Memoization) in Next.js Fetch Calls?

Request Deduplication means that if multiple Server Components make the same fetch() request during a single render, Next.js automatically performs only one network request and shares the result with all components.

Request Deduplication = Multiple identical fetch calls → One network request.

The Problem Without Deduplication

Suppose three components need the same user data.

Header.tsx
export default async function Header() {
  const user = await fetch('https://api.myshop.com/user').then(res => res.json());

  return <h1>{user.name}</h1>;
}
Sidebar.tsx
export default async function Sidebar() {
  const user = await fetch('https://api.myshop.com/user').then(res => res.json());

  return <p>{user.email}</p>;
}
Profile.tsx
export default async function Profile() {
  const user = await fetch('https://api.myshop.com/user').then(res => res.json());

  return <p>{user.role}</p>;
}

Without deduplication: 

Header
   ↓
API Call

Sidebar
   ↓
API Call

Profile
   ↓
API Call

Result: 

3 Network Requests

What Next.js Does

Next.js notices that all fetch calls are identical:

Same URL
fetch('https://api.myshop.com/user')

fetch('https://api.myshop.com/user')

fetch('https://api.myshop.com/user')

So Next.js performs: 

1 Network Request
       ↓
Result Shared
       ↓
Header
Sidebar
Profile
This is called Request Deduplication (Memoization).

Real Example

Page
import Header from './Header';
import Sidebar from './Sidebar';
import Profile from './Profile';

export default function Page() {
  return (
    <>
      <Header />
      <Sidebar />
      <Profile />
    </>
  );
}

All three components call:

Identical Fetch
await fetch('https://api.myshop.com/user');

Next.js internally does: 

First Fetch
      ↓
Call API

Second Fetch
      ↓
Reuse Result

Third Fetch
      ↓
Reuse Result

Important Rule

Deduplication only works for identical requests.

Deduplicated
fetch('https://api.myshop.com/user')

fetch('https://api.myshop.com/user')

fetch('https://api.myshop.com/user')

Result: 

1 Network Request
Not Deduplicated
fetch('https://api.myshop.com/user')

fetch('https://api.myshop.com/user/1')

fetch('https://api.myshop.com/user/2')

Result: 

3 Network Requests

How Is It Different From Cache?

Many developers confuse memoization with caching.

Feature Memoization Cache
Purpose Avoid duplicate requests in same render Reuse data across requests
Lifetime Current request only Multiple requests
Uses Data Cache ❌ ✅
Network Calls Saved Same render Future renders

Memoization + Cache Together

In Next.js both can work together.

Example
const res = await fetch('https://api.myshop.com/products', {
  next: { revalidate: 60 }
});

During one render: 

Deduplication
      ↓
One Network Request

Across future requests: 

Cache
      ↓
Reuse Data For 60 Seconds

One-Line Definition:
Request Deduplication (Memoization) is a Next.js optimization that automatically combines identical fetch requests made during the same render into a single network request and shares the result with all components.
