Can Server Actions be called from Client Components? How?

Yes, Server Actions can be called from Client Components, but the Server Action must be defined in a separate file with 'use server' at the top.

Simple Meaning:
A Client Component can import a Server Action and call it from a form, button click, or event handler. The function still runs on the server.

1. Create Server Action In Separate File

First, create a separate server-only file.

app/actions.ts
'use server';

export async function createUser(name: string) {
  await db.user.create({ data: { name } });

  return { success: true, message: 'User created successfully' };
}

Because this file starts with 'use server', the exported function becomes a Server Action.

2. Call Server Action From Client Component

Now import that Server Action inside a Client Component.

app/UserButton.tsx
'use client';

import { createUser } from './actions';

export default function UserButton() {
  async function handleClick() {
    const result = await createUser('Rizwan');
    console.log(result.message);
  }

  return <button onClick={handleClick}>Create User</button>;
}

The button click happens in the browser, but createUser() executes on the server.

Flow

Internal Flow
User Clicks Button
        ↓
Client Component Calls createUser()
        ↓
Next.js Sends Internal Request
        ↓
Server Action Runs On Server
        ↓
Database Updated
        ↓
Result Returned To Client

3. Calling Server Action From Client Form

We can also use a Server Action with a form inside a Client Component.

app/actions.ts
'use server';

export async function saveUser(formData: FormData) {
  const name = formData.get('name') as string;

  await db.user.create({ data: { name } });

  return { success: true, message: 'Saved successfully' };
}
app/UserForm.tsx
'use client';

import { saveUser } from './actions';

export default function UserForm() {
  return (
    <form action={saveUser}>
      <input name="name" placeholder="Enter name" />
      <button>Save</button>
    </form>
  );
}

4. With useActionState

For showing success or error messages, use useActionState().

app/actions.ts
'use server';

type State = { success: boolean; message: string };

export async function createUser(previousState: State, formData: FormData): Promise<State> {
  const name = formData.get('name') as string;

  if (!name) return { success: false, message: 'Name is required' };

  await db.user.create({ data: { name } });

  return { success: true, message: 'User created successfully' };
}
app/UserForm.tsx
'use client';

import { useActionState } from 'react';
import { createUser } from './actions';

const initialState = { success: false, message: '' };

export default function UserForm() {
  const [state, formAction] = useActionState(createUser, initialState);

  return (
    <form action={formAction}>
      <input name="name" placeholder="Enter name" />
      <button>Save</button>
      {state.message && <p>{state.message}</p>}
    </form>
  );
}

Important Rule

Important:
Do not define inline Server Actions directly inside a Client Component.

❌ Wrong

Client Component
'use client';

export default function UserForm() {
  async function createUser(formData: FormData) {
    'use server';

    await db.user.create({ data: { name: formData.get('name') as string } });
  }

  return (
    <form action={createUser}>
      <input name="name" />
      <button>Save</button>
    </form>
  );
}

This is wrong because a Client Component file cannot contain an inline Server Action. In Server Component, inline Server Action is allowed.

✅ Correct

actions.ts
'use server';

export async function createUser(formData: FormData) {
  await db.user.create({ data: { name: formData.get('name') as string } });
}
UserForm.tsx
'use client';

import { createUser } from './actions';

export default function UserForm() {
  return (
    <form action={createUser}>
      <input name="name" />
      <button>Save</button>
    </form>
  );
}

Server Component vs Client Component Usage

Location How To Use Server Action
Server Component Can define inline Server Action or import it
Client Component Must import Server Action from a separate 'use server' file

When Should Client Components Call Server Actions?

  • When button click should update database
  • When form submit should create/update/delete data
  • When we need success/error message in UI
  • When we need client-side interactivity plus server-side mutation

Remember
Client Component
       ↓
Import Server Action
       ↓
Call Action
       ↓

One-Line Definition:
Yes, Client Components can call Server Actions by importing them from a separate file marked with 'use server'; the call starts in the browser, but the action executes on the server.
