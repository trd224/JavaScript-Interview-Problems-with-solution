50. How does prefetching work in the Link component and how did it change in v13?

Prefetching allows Next.js to download a route before the user clicks the link, making navigation much faster.

Simple Meaning:
When a link becomes visible on the screen, Next.js can quietly prepare the destination page in the background.

What Is Prefetching?

Without prefetching:

Normal Navigation
User Clicks Link
        ↓
Download Route Code
        ↓
Fetch Route Data
        ↓
Render New Page

There is a delay because everything starts after the click.

With prefetching:

Prefetch Navigation
Page Loads
      ↓
Link Becomes Visible
      ↓
Route Prefetched
      ↓
User Clicks Link
      ↓
Instant Navigation

Basic Link Example

Link
import Link from 'next/link';

export default function Page() {
  return (
    <Link href="/products">
      Products
    </Link>
  );
}

In production mode, Next.js automatically prefetches this route when the link enters the viewport.

How Prefetching Works

Next.js uses an Intersection Observer internally.

Flow
Link Appears In Viewport
          ↓
Intersection Observer Detects It
          ↓
Next.js Starts Prefetching
          ↓
Route Stored In Cache
          ↓
User Clicks Link
          ↓
Fast Navigation

Automatic Prefetching

By default:

Automatic Prefetch
<Link href="/products">
  Products
</Link>
Environment Prefetch
Development ❌ Disabled
Production ✅ Enabled

This prevents unnecessary network requests during development.

Disable Prefetching

Sometimes we do not want prefetching.

Disable
<Link href="/products" prefetch={false}>
  Products
</Link>

Now the route loads only after the user clicks.

What Changed In Next.js 13?

1. No More <a> Tag Required

Before v13:

Next.js 12
<Link href="/products">
  <a>Products</a>
</Link>

Starting with v13:

Next.js 13+
<Link href="/products">
  Products
</Link>

The Link component automatically renders the anchor element.

Version Syntax
v12 <Link><a>...</a></Link>
v13+ <Link>...</Link>

2. App Router Prefetching Became Smarter

In the App Router, Next.js prefetches React Server Component (RSC) payloads instead of just JavaScript bundles.

Pages Router
Prefetch JavaScript Bundle
          ↓
User Click
          ↓
Render Page
App Router
Prefetch Route Segments
          ↓
Prefetch RSC Payload
          ↓
User Click
          ↓
Faster Render
Main v13 Improvement:
The App Router can prefetch route segments and React Server Component payloads, making navigation more efficient.

Real World Example

Navigation Menu
import Link from 'next/link';

export default function Navigation() {
  return (
    <nav>
      <Link href="/">Home</Link>
      <Link href="/products">Products</Link>
      <Link href="/about">About</Link>
    </nav>
  );
}
Flow
Home Page Loads
        ↓
Links Become Visible
        ↓
Routes Prefetched
        ↓
User Clicks Products
        ↓
Products Page Opens Quickly

When Should We Disable Prefetch?

Scenario Disable Prefetch?
Main Navigation ❌ No
Frequently Visited Pages ❌ No
Thousands Of Links ✅ Sometimes
Rarely Used Admin Pages ✅ Sometimes
Huge Dynamic Routes ✅ Sometimes

Link vs Traditional Anchor

Feature Link <a>
Client Navigation ✅ ❌
Prefetching ✅ ❌
SPA Experience ✅ ❌
Full Page Reload ❌ ✅

Important Interview Point

Important:
Prefetching works only in production builds. During development (next dev), automatic prefetching is disabled.

One-Line Definition:
The Link component automatically prefetches visible routes in production, and in Next.js 13 it became smarter by prefetching App Router data and no longer requiring a nested <a> element.
