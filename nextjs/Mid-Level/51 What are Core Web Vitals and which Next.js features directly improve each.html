What are Core Web Vitals and which Next.js features directly improve each?

Core Web Vitals are Google's user experience metrics that measure loading speed, responsiveness, and visual stability.

Simple Meaning:
Core Web Vitals tell us how fast, responsive, and stable our application feels to real users.

The Three Core Web Vitals

Metric Measures Good Score
LCP Loading Performance ≤ 2.5 seconds
INP User Interaction Responsiveness ≤ 200 ms
CLS Visual Stability ≤ 0.1

1. LCP (Largest Contentful Paint)

LCP measures how quickly the largest visible content appears on the screen.

Example
User Opens Page
       ↓
Hero Image Appears
       ↓
LCP Recorded

Usually the LCP element is:

  • Hero image
  • Banner image
  • Large heading
  • Featured product image

Next.js Features That Improve LCP

Feature How It Helps
next/image Image optimization
priority Preloads LCP image
SSG HTML already generated
ISR Cached HTML served quickly
Server Components Less JavaScript sent
next/font Eliminates font delays
LCP Example
import Image from 'next/image';

export default function Home() {
  return (
    <Image
      src="/hero.jpg"
      alt="Hero"
      width={1200}
      height={600}
      priority
    />
  );
}
Why?
The priority prop preloads the hero image, helping the largest visible content appear faster.

2. INP (Interaction to Next Paint)

INP measures how quickly the page responds after a user interaction.

Example
User Clicks Button
        ↓
Browser Processes Event
        ↓
UI Updates
        ↓
INP Recorded

Slow JavaScript usually causes poor INP.

Next.js Features That Improve INP

Feature How It Helps
Server Components Less client JS
next/dynamic Lazy loads heavy code
Code Splitting Smaller bundles
next/script Optimized third-party scripts
Suspense Avoids blocking UI
Partial Prerendering (PPR) Interactive shell loads faster
Dynamic Import Example
import dynamic from 'next/dynamic';

const Chart = dynamic(() => import('./Chart'));

export default function Page() {
  return <Chart />;
}
Why?
Heavy chart code is downloaded later, reducing the amount of JavaScript that blocks interaction.

3. CLS (Cumulative Layout Shift)

CLS measures unexpected movement of page elements while the page is loading.

Bad Experience
Page Loads
      ↓
Button Appears
      ↓
Image Loads
      ↓
Button Moves Down
      ↓
CLS Increases

Users may click the wrong thing because the layout shifts.

Next.js Features That Improve CLS

Feature How It Helps
next/image Reserves image space
width / height Prevents image shifts
fill Maintains image layout
next/font Prevents font swapping shifts
Layouts Keeps stable UI structure
Good Example
import Image from 'next/image';

export default function Page() {
  return (
    <Image
      src="/product.jpg"
      alt="Product"
      width={600}
      height={400}
    />
  );
}
Why?
Because dimensions are known before loading, space is reserved and no layout shift occurs.

One-Line Definition:
Core Web Vitals measure loading speed (LCP), interaction responsiveness (INP), and visual stability (CLS), and Next.js improves them through features such as next/image, priority, next/font, Server Components, dynamic imports, and static rendering.
