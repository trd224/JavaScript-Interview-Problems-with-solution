18. What is the difference between cache: 'force-cache' and cache: 'no-store'?

These are two caching modes provided by the Next.js extended fetch API.

force-cache tells Next.js to reuse cached data, while no-store tells Next.js to fetch fresh data on every request.

cache: 'force-cache'

Next.js fetches the data once and stores it in the Data Cache.

Example
const res = await fetch('https://api.example.com/products', {
  cache: 'force-cache'
});

const products = await res.json();

Flow

First Request
      ↓
Fetch API Called
      ↓
Data Stored In Cache


Second Request
      ↓
Use Cached Data


Third Request
      ↓
Use Cached Data

Result

Same Data Returned
Without Calling API Again
This is similar to SSG behavior because cached data is reused.

cache: 'no-store'

Next.js skips the cache completely.

Example
const res = await fetch('https://api.example.com/products', {
  cache: 'no-store'
});

const products = await res.json();

Flow

Request 1
     ↓
Call API


Request 2
     ↓
Call API Again


Request 3
     ↓
Call API Again

Result

Fresh Data
Every Request
This makes the route dynamic and behaves like SSR.

Real World Example

Product Catalog

Products rarely change.

Good Candidate For force-cache
const res = await fetch('/api/products', {
  cache: 'force-cache'
});

Users can safely see cached data.

User Dashboard

Data changes constantly.

Good Candidate For no-store
const res = await fetch('/api/dashboard', {
  cache: 'no-store'
});

Every user should receive fresh data.

The Biggest Difference

Suppose the API currently returns:

API Response
{
  "price": 100
}

Now the database price changes:

New Database Value
{
  "price": 120
}

Using force-cache

Database Price
      ↓
120

Cache Still Contains
      ↓
100

User Sees
      ↓
100

Until cache is cleared or revalidated.

Using no-store

Database Price
      ↓
120

Request Arrives
      ↓
API Called Again
      ↓
User Sees
      ↓
120

Fresh data is always returned.

What Happens To Rendering?

force-cache

Example
export default async function ProductsPage() {

  const res = await fetch('/api/products', {
    cache: 'force-cache'
  });

  return <h1>Products</h1>;

}

Next.js can cache the result and treat the page like static content.

Result

SSG Style Rendering

no-store

Example
export default async function DashboardPage() {

  const res = await fetch('/api/dashboard', {
    cache: 'no-store'
  });

  return <h1>Dashboard</h1>;

}

Result

SSR Style Rendering

Fetch Runs
Every Request

One-Line Definition:
cache: 'force-cache' reuses cached data and behaves like static rendering, while cache: 'no-store' bypasses the cache and fetches fresh data on every request, resulting in dynamic SSR rendering.
