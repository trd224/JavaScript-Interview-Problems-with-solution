How do you lazy-load a heavy Client Component to avoid hydration cost?

We lazy-load a heavy Client Component by using next/dynamic, so that the heavy component's JavaScript is not included in the initial page bundle.

Simple Meaning:
Load the heavy Client Component only when needed, instead of downloading and hydrating it immediately on page load.

Problem: Heavy Client Component

A Client Component needs JavaScript in the browser and must be hydrated.

Heavy Component
'use client';

export default function Chart() {
  return <div>Heavy Chart Component</div>;
}

If we import it normally, it becomes part of the initial JavaScript bundle.

Normal Import
import Chart from './Chart';

export default function Page() {
  return <Chart />;
}
Flow
Page Loads
      ↓
Chart JavaScript Downloads Immediately
      ↓
Chart Hydrates Immediately
      ↓
Initial Load Becomes Heavier

Solution: Lazy Load With next/dynamic

Use dynamic() from next/dynamic.

Lazy Loaded Component
import dynamic from 'next/dynamic';

const Chart = dynamic(() => import('./Chart'), {
  loading: () => <p>Loading chart...</p>
});

export default function Page() {
  return <Chart />;
}
Flow
Page Loads
      ↓
Main Page Hydrates
      ↓
Chart Bundle Loads Separately
      ↓
Chart Hydrates Later

Disable SSR For Browser-Only Heavy Component

If the component uses browser-only APIs like window, document, localStorage, maps, charts, or editors, use ssr: false.

Client Only Dynamic Import
import dynamic from 'next/dynamic';

const RichEditor = dynamic(() => import('./RichEditor'), {
  ssr: false,
  loading: () => <p>Loading editor...</p>
});

export default function Page() {
  return <RichEditor />;
}
Meaning:
ssr: false means this component will not render on the server. It will only load and render in the browser.

Real World Example: Dashboard Chart

Suppose our dashboard has normal static content plus one heavy chart.

HeavyChart.tsx
'use client';

import Chart from 'chart.js/auto';

export default function HeavyChart() {
  return <canvas id="sales-chart"></canvas>;
}
page.tsx
import dynamic from 'next/dynamic';

const HeavyChart = dynamic(() => import('./HeavyChart'), {
  ssr: false,
  loading: () => <p>Loading sales chart...</p>
});

export default function DashboardPage() {
  return (
    <main>
      <h1>Dashboard</h1>
      <p>This content loads fast.</p>
      <HeavyChart />
    </main>
  );
}

Now the dashboard content can load first, and the chart JavaScript loads separately.

Lazy Load Only After User Click

For maximum saving, we can load the heavy component only after user interaction.

ReportSection.tsx
'use client';

import { useState } from 'react';
import dynamic from 'next/dynamic';

const HeavyReport = dynamic(() => import('./HeavyReport'), {
  loading: () => <p>Loading report...</p>
});

export default function ReportSection() {
  const [showReport, setShowReport] = useState(false);

  return (
    <div>
      <button onClick={() => setShowReport(true)}>Show Report</button>
      {showReport && <HeavyReport />}
    </div>
  );
}
Flow
Page Loads
      ↓
HeavyReport Not Downloaded
      ↓
User Clicks Show Report
      ↓
HeavyReport Downloads
      ↓
HeavyReport Hydrates

When Should We Lazy Load?

Component Lazy Load?
Chart component ✅ Yes
Map component ✅ Yes
Rich text editor ✅ Yes
Video player ✅ Yes
Payment widget ✅ Yes
Small button ❌ Usually No
Header navigation ❌ Usually No

Important: Lazy Loading Reduces Initial JS

Lazy loading does not remove hydration completely. It delays hydration until the component is loaded and rendered.

Normal Import Dynamic Import
Component JS loads immediately Component JS loads later
Hydrates immediately Hydrates later
Higher initial cost Lower initial cost

Best Practice

Best Pattern
import dynamic from 'next/dynamic';

const HeavyClientComponent = dynamic(() => import('./HeavyClientComponent'), {
  ssr: false,
  loading: () => <p>Loading...</p>
});

export default function Page() {
  return <HeavyClientComponent />;
}
Important:
Use ssr: false only for components that do not need server rendering or depend on browser-only APIs.
