Difference Between cacheLife and cacheRevalidate in Next.js 15

In Next.js 15, both cacheLife() and cacheRevalidate() are used with the 'use cache' directive, but they are not exactly the same.

Simple meaning:
cacheRevalidate() controls when cache should refresh.
cacheLife() controls the full cache lifecycle: Fresh, Stale, and Expire.

Basic use cache Example

Cached Function
import { cacheLife } from 'next/cache';

async function getProducts() {
  'use cache';

  cacheLife('hours');

  return db.product.findMany();
}

Here, Next.js stores the result of getProducts() in cache.

What is cacheRevalidate?

cacheRevalidate() tells Next.js when cached data should become stale and refresh.

cacheRevalidate Example
import { cacheRevalidate } from 'next/cache';

async function getProducts() {
  'use cache';

  cacheRevalidate({ seconds: 3600 });

  return db.product.findMany();
}

This means: 

Cache is fresh for 1 hour.

After 1 hour:
Cache becomes stale.
Old cache can still be returned.
Refresh happens in background.

cacheRevalidate Timeline

10:00 Cache created
      Data = ₹100

10:30 Request
      ↓
      Fresh cache returned
      User sees ₹100

11:00 Revalidate time reached

11:05 Request
      ↓
      User gets OLD cache ₹100 immediately
      ↓
      Background DB/API fetch starts
      ↓
      DB has latest data ₹120
      ↓
      Cache updated to ₹120

11:06 Request
      ↓
      User gets NEW cache ₹120
In cacheRevalidate(), old cache is still allowed after revalidate time. It refreshes in background.

What is Expire in cacheLife?

cacheLife() gives one extra important stage: Expire.

cacheLife Example
import { cacheLife } from 'next/cache';

async function getProducts() {
  'use cache';

  cacheLife('hours');

  return db.product.findMany();
}

Conceptually, cacheLife creates this flow: 

Fresh → Stale → Expired

cacheLife Timeline

10:00 Cache created
      Data = ₹100

10:30 Request
      ↓
      Fresh cache returned
      User sees ₹100

11:00 Fresh time finished

11:05 Request
      ↓
      User gets OLD cache ₹100 immediately
      ↓
      Background DB/API fetch starts
      ↓
      DB has latest data ₹120
      ↓
      Cache updated to ₹120

11:06 Request
      ↓
      User gets NEW cache ₹120

Until this point, cacheLife() looks similar to cacheRevalidate().

The Real Difference Appears at Expire

24:00 Cache expired completely

24:05 Request
      ↓
      OLD cache is NOT allowed
      ↓
      Must fetch DB/API first
      ↓
      DB has latest data ₹150
      ↓
      New cache created
      ↓
      User gets ₹150
Important:
After cache expires, old cache cannot be returned to the user. Next.js must fetch fresh data first.

Main Difference

cacheRevalidate

Revalidate time reached
      ↓
Old cache is still allowed
      ↓
User gets old data immediately
      ↓
Refresh happens in background

cacheLife Expire

Expire time reached
      ↓
Old cache is NOT allowed
      ↓
User must wait
      ↓
Fresh DB/API data is fetched
      ↓
New cache is created

Example with Slow DB

Suppose the database takes 5 seconds.

cacheRevalidate

11:05 Request
      ↓
      User gets OLD cache immediately
      Response time = almost 0 second
      ↓
      DB fetch runs in background for 5 seconds
      ↓
      Cache updated

cacheLife Expire

24:05 Request
      ↓
      Old cache is expired
      ↓
      User waits 5 seconds
      ↓
      DB fetch completes
      ↓
      New cache created
      ↓
      User gets response

Visual Comparison

Point cacheRevalidate cacheLife Expire
Old cache after time finished Allowed Not allowed
User waits for DB/API? No Yes
Refresh type Background refresh Blocking refresh
Response speed Fast Depends on DB/API speed
Purpose Refresh cache after time Stop trusting old cache after expiry

Why Use cacheLife?

We use cacheLife() when we want more control than only revalidation.

  • How long cache should be fresh.
  • How long stale cache can still be used.
  • When old cache must be completely rejected.
cacheRevalidate() answers: When should cache refresh?
cacheLife() answers: When should cache refresh, and when should old cache completely expire?

Easy Memory Trick

cacheRevalidate
      ↓
Old cache allowed
Refresh in background


cacheLife Expire
      ↓
Old cache not allowed
Fetch fresh data first

Final One-Line Definition

cacheRevalidate means old cache can still be served while Next.js refreshes it in the background, but cacheLife expire means old cache is no longer trusted and Next.js must fetch fresh data before returning the response.
