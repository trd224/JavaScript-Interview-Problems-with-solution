6. What is the root layout and why is it required in the App Router?

The Root Layout is the top-level layout of a Next.js App Router application. It is defined in app/layout.tsx and wraps every page and layout in the application.

The primary purpose of the Root Layout is to provide the application's global HTML structure and shared UI that should exist across the entire application.

Why Is the Root Layout Required?

Every App Router application must have exactly one Root Layout because Next.js needs a top-level component to define:

  • HTML Structure
  • Body Element
  • Global UI
  • Shared Providers
  • Fonts
  • Application Shell

Without a Root Layout, Next.js would not know how to construct the application's HTML document.

Basic Root Layout Example

app/layout.tsx
export default function RootLayout({ children }) {

  return (
    <html lang="en">
      <body>
        {children}
      </body>
    </html>
  );

}

This is the minimum required Root Layout.

Why Must It Contain html and body?

The Root Layout is responsible for generating the application's document structure.

Required Structure
<html lang="en">
  <body>
    Application Content
  </body>
</html>

Unlike nested layouts, the Root Layout must contain both the html and body tags.

Where Is the head Tag?

The Root Layout must contain the html and body tags, but you might notice that it does not contain a head tag.

Root Layout
export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );
}

In the App Router, Next.js automatically generates the head element. We usually use the Metadata API instead of manually creating a head tag.

Metadata API
export const metadata = {
  title: 'My App',
  description: 'Learning Next.js'
};

Next.js converts this metadata into HTML inside the head element.

Generated HTML
<html>
  <head>
    <title>My App</title>
  </head>
  <body>...</body>
</html>

Although uncommon, a head tag can also be added manually.

Manual head Tag
export default function RootLayout({ children }) {
  return (
    <html>
      <head>
        <meta name="author" content="Rizwan" />
      </head>
      <body>{children}</body>
    </html>
  );
}
The Root Layout must define the html and body tags. The head tag is usually generated automatically by Next.js from the Metadata API.

Adding Global UI

Root Layout Example
import Header from './Header';
import Footer from './Footer';

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>
        <Header />
        {children}
        <Footer />
      </body>
    </html>
  );
}

The Header and Footer automatically appear on every page.

What If We Do Not Want Header and Footer Everywhere?

If the Header and Footer are placed inside the Root Layout, they automatically appear on every route.

Nested layouts do not replace the Root Layout. They are rendered inside the Root Layout. Route Groups allow different sections of the application to have different layouts while still sharing the same Root Layout.

Using Providers

The Root Layout is commonly used for global providers.

Theme Provider
import ThemeProvider from './ThemeProvider';

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>
        <ThemeProvider>
          {children}
        </ThemeProvider>
      </body>
    </html>
  );
}

This makes the provider available throughout the application.

What Happens If We Remove It?

Next.js will fail to build the application because every App Router project must have a Root Layout.

How Does It Persist Across Navigations?

The Root Layout remains mounted while users navigate between pages.

Navigation

    │
Navigate
    ▼

    │

The Root Layout is reused and only the page content changes.

When Should We Use the Root Layout?

  • Global Header.
  • Global Footer.
  • Theme Providers.
  • Authentication Providers.
  • Font Configuration.
  • Application-wide UI.

Important Note

There can be only one Root Layout in an App Router application, and it must define the application's html and body elements.

Interview Tip

A common misconception is that the Root Layout is just another layout. In reality, it is a mandatory top-level layout that creates the application's HTML document structure and wraps every route.
One-Line Definition:
The Root Layout is the mandatory top-level layout in the App Router that defines the application's html and body structure and wraps every page and layout in the application.
