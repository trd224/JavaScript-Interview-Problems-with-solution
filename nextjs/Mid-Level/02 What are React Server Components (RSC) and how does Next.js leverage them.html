2. What are React Server Components (RSC) and how does Next.js leverage them?

React Server Components (RSC) are React components that execute on the server instead of the browser. They can fetch data, access backend resources, and generate UI without sending their JavaScript code to the client.

In the App Router, all components are React Server Components by default unless we add the 'use client' directive.

Why Were Server Components Introduced?

Traditional React applications send component JavaScript to the browser.

Large applications can send a lot of JavaScript to the browser.

React Server Components reduce the amount of JavaScript sent to users.

Server Component Example

app/products/page.js
export default async function Page() {

  const products =
    await fetch(
      'https://dummyjson.com/products'
    ).then(res => res.json());

  return (
    <ul>
      {products.products.map(product => (
        <li key={product.id}>
          {product.title}
        </li>
      ))}
    </ul>
  );

}

This component runs on the server.

Client Component Example

app/components/Counter.js
'use client';

import { useState } from 'react';

export default function Counter() {

  const [count, setCount] = useState(0);

  return (
    <button
      onClick={() => setCount(count + 1)}
    >
      {count}
    </button>
  );

}

This component runs in the browser because it uses:

  • useState
  • useEffect
  • Event handlers
  • Browser APIs

Server Component Rules

Allowed
await fetch(...)

await database.query(...)

await cookies()

await headers()
Not Allowed
useState()

useEffect()

onClick={...}

window.localStorage

Server Components cannot use browser-only features.

How Next.js Leverages RSC

Next.js App Router is built around React Server Components.

app/page.js
export default function Page() {
  return <h1>Home Page</h1>;
}

Even this simple component is a Server Component because there is no 'use client'.

Mixing Server and Client Components

app/page.js
import Counter from './Counter';

export default function Page() {

  return (
    <>
      <h1>Dashboard</h1>
      <Counter />
    </>
  );

}
app/Counter.js
'use client';

import { useState } from 'react';

export default function Counter() {

  const [count, setCount] = useState(0);

  return (
    <button
      onClick={() => setCount(count + 1)}
    >
      {count}
    </button>
  );

}

Here:

  • Page → Server Component.
  • Counter → Client Component.

Benefits of RSC

  • Smaller JavaScript bundles.
  • Faster page loads.
  • Direct server-side data fetching.
  • Reduced hydration cost.
  • Better performance.
  • Improved SEO.

Important Note

React Server Components are not the same as SSR. SSR determines where HTML is generated, while RSC determines where the React component executes.

Interview Tip

The App Router uses React Server Components by default. A component becomes a Client Component only when we add the 'use client' directive.
One-Line Definition:
React Server Components (RSC) are React components that execute on the server and send rendered UI instead of component JavaScript to the browser, and Next.js App Router uses them by default to reduce bundle size and improve performance.
