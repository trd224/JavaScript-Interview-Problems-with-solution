What is the error.js file and what is its Error Boundary contract?

The error.js file is a special App Router file that automatically creates a React Error Boundary for a route segment and displays a fallback UI when an error occurs.

In simple words, error.js is the error version of loading.js. Instead of showing loading UI, it shows error UI.

Why Do We Need error.js?

Sometimes pages fail because of:

  • Database Errors
  • API Errors
  • Server Component Errors
  • Unexpected Exceptions
  • Runtime Errors

Without an Error Boundary, the entire route could crash.

With error.js, Next.js catches the error and displays a fallback UI instead.

Basic Example

Folder Structure
app/
│
├── dashboard/
│   ├── page.tsx
│   └── error.tsx
page.tsx
export default function Page() {

  throw new Error(
    'Database Connection Failed'
  );

}
error.tsx
'use client';

export default function Error() {

  return (
    <h2>
      Something Went Wrong
    </h2>
  );

}

What Happens?

Flow
User Request
      │
      ▼
Page Starts Rendering
      │
      ▼
Error Occurs
      │
      ▼
error.js Rendered
      │
      ▼
User Sees Error UI

The route does not crash completely.

How Does React Error Boundary Fit In?

Internally, Next.js automatically wraps the route segment in a React Error Boundary.

Conceptual Implementation
<ErrorBoundary
  fallback={<Error />}
>
  <Page />
</ErrorBoundary>

Next.js creates this Error Boundary automatically.

Just like loading.js automatically creates Suspense, error.js automatically creates an Error Boundary.

Error Boundary Contract

The Error Component must follow a specific contract.

Required Signature
'use client';

export default function Error({
  error,
  reset
}) {

  return (
    <div>
      Error
    </div>
  );

}

Why Must error.js Be a Client Component?

The error component receives:

  • error
  • reset()

and handles user interaction.

Therefore it must start with:

Required
'use client';

The error Parameter

error Object
'use client';

export default function Error({
  error
}) {

  return (
    <p>
      {error.message}
    </p>
  );

}

The error object contains information about the exception that occurred.

The reset Function

The reset function retries rendering the route segment.

Retry Example
'use client';

export default function Error({
  error,
  reset
}) {

  return (
    <>
      <p>
        {error.message}
      </p>

      <button
        onClick={() => reset()}
      >
        Try Again
      </button>
    </>
  );

}

When the user clicks the button: 

reset()
     │
     ▼
Render Route Again

Real Example

page.tsx
let count = 0;

export default function Page() {

  count++;

  if (count < 2) {

    throw new Error(
      'Temporary Error'
    );

  }

  return <h1>Success</h1>;

}
error.tsx
'use client';

export default function Error({
  error,
  reset
}) {

  return (
    <button
      onClick={() => reset()}
    >
      Retry
    </button>
  );

}

First render fails.

After clicking Retry: 

reset()
     │
     ▼
Render Again
     │
     ▼
Success

Error Scope

Like loading.js, error.js only affects its route segment.

Folder Structure
app/
│
├── dashboard/
│   ├── error.tsx
│   ├── page.tsx
│   │
│   └── settings/
│       └── page.tsx

If an error occurs inside dashboard: 

Dashboard Error Boundary
      │
      ▼
Handles Error

The rest of the application continues working.

Error Boundary Contract:
The error.js file must be a Client Component and receives two props: error (the thrown error) and reset() (a function that retries rendering the route segment).
One-Line Definition:
The error.js file automatically creates a React Error Boundary for a route segment and displays fallback UI when rendering fails, while providing error and reset() through its Error Boundary contract.
