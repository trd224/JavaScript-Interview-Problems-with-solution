What is useActionState (React 19) and how does it replace useFormState in Next.js 15?

useActionState() is a React 19 hook that manages state returned from Server Actions.

Simple Meaning:
useActionState() allows us to receive success messages, validation errors, and other state updates directly from a Server Action.

Why Was useFormState Replaced?

Before React 19, Next.js used useFormState() from React DOM.

Old API
import { useFormState } from 'react-dom';

React 19 introduced a more general solution called useActionState().

New API
import { useActionState } from 'react';

As a result, Next.js 15 uses useActionState() instead of useFormState().

What Problem Does useActionState Solve?

Suppose a form submits data and the Server Action returns a success or error message.

Without useActionState
Submit Form
      ↓
Server Action
      ↓
Success/Error
      ↓
Manually Manage State

We would need extra state variables and custom handling.

With useActionState(): 

Submit Form
      ↓
Server Action
      ↓
Return State
      ↓
UI Updates Automatically

Basic Example

Server Action
'use server';

export async function createUser(
  previousState: string,
  formData: FormData
) {

  const name = formData.get('name');

  if (!name) {
    return 'Name is required';
  }

  return 'User created successfully';

}
Client Component
'use client';

import { useActionState }
from 'react';

import { createUser }
from './actions';

export default function Form() {

  const [
    message,
    formAction
  ] = useActionState(
    createUser,
    ''
  );

  return (
    <form action={formAction}>

      <input name="name" />

      <button>
        Save
      </button>

      <p>{message}</p>

    </form>
  );

}

Understanding the Parameters

Syntax
const [
  state,
  formAction
] = useActionState(
  action,
  initialState
);
Value Description
state Current state returned from action
formAction Action attached to form
action Server Action function
initialState Starting value

How State Flows

Flow
User Submits Form
          ↓
Server Action Executes
          ↓
Returns State
          ↓
useActionState Receives State
          ↓
Component Re-renders
          ↓
UI Updated

Real World Validation Example

Server Action
'use server';

export async function register(
  previousState: any,
  formData: FormData
) {

  const email =
    formData.get('email');

  if (!email) {

    return {
      success: false,
      error: 'Email is required'
    };

  }

  return {
    success: true,
    error: null
  };

}
Client Component
'use client';

import { useActionState }
from 'react';

const [
  state,
  formAction
] = useActionState(
  register,
  {
    success: false,
    error: null
  }
);

return (
  <form action={formAction}>

    <input
      name="email"
    />

    <button>
      Register
    </button>

    {state.error && (
      <p>
        {state.error}
      </p>
    )}

  </form>
);

useFormState vs useActionState

Feature useFormState useActionState
Library react-dom react
React Version React 18 React 19
Status Older API Recommended
Works with Server Actions ✅ ✅
Used in Next.js 15 ❌ ✅

Relationship with useFormStatus()

These hooks solve different problems.

Hook Purpose
useActionState() Manage returned state
useFormStatus() Track pending/loading state

Easy Memory Trick

Remember
useActionState()
        ↓
Returns Data


useFormStatus()
        ↓
Returns Loading Status
One-Line Definition:
useActionState() is a React 19 hook that manages state returned from Server Actions and replaces the older useFormState() API used in earlier Next.js versions.
