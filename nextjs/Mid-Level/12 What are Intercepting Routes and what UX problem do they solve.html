12. What are Intercepting Routes and what UX problem do they solve?

Intercepting Routes are a Next.js App Router feature that allows us to load one route inside another route's UI instead of performing a full page navigation.

Intercepting Routes are primarily used to show a route as a modal, drawer, popup, or overlay while keeping the current page visible underneath.

The UX Problem

Suppose we have an Instagram-style gallery.

Gallery Page
/photos

User sees: 

Photo 1
Photo 2
Photo 3
Photo 4

User clicks: 

Photo 3

Without Intercepting Routes

Normal Navigation
/photos
     │
     ▼
/photos/3

What happens? 

Gallery Page Disappears

Photo Detail Page Appears

Now the user loses:

  • Scroll Position
  • Gallery Context
  • Current UI State

When the user presses Back: 

Reload Gallery
Find Previous Position Again
This creates a poor user experience because the user leaves the gallery completely.

With Intercepting Routes

Intercept Navigation
/photos
     │
Click Photo
     │
     ▼
Intercept /photos/3

Instead of replacing the page: 

Gallery Stays Visible

Modal Opens

Photo Details Appear

User sees: 

Gallery Page

┌──────────────┐
│ Photo Detail │
└──────────────┘

The gallery remains in the background.

Why Is This Better?

Close Modal
User Clicks Close
       │
       ▼
Return To Gallery

Because the gallery never disappeared:

  • Scroll Position Remains
  • State Remains
  • Filters Remain
  • Search Remains

Real World Examples

  • Instagram Photo Modal.
  • Twitter/X Tweet Modal.
  • Facebook Post Preview.
  • E-Commerce Product Quick View.
  • Email Preview Modal.

│       └── page.tsx

Notice: 

(.)[id]

This is an intercepting route.

Why Do We Need Both [id] and (.)[id]?

A very common question is:

Folder Structure
photos/
│
├── [id]/
│   └── page.tsx

└── (.)[id]/
    └── page.tsx

Why do we need both folders?

The answer is that they serve different purposes.

[id] = Full Page Version

Normal Route
photos/
└── [id]/
    └── page.tsx

This route handles: 

/photos/1
/photos/2
/photos/3

when the user:

  • Opens the URL directly.
  • Refreshes the page.
  • Shares the URL with someone.
  • Bookmarks the URL.
Example
https://site.com/photos/3

There is no gallery page underneath, so Next.js must render a complete page.

Result
Full Photo Detail Page

(.)[id] = Modal Version

Intercepting Route
photos/
└── (.)[id]/
    └── page.tsx

This route handles navigation that starts from inside the application.

Flow
/photos
     │
Click Photo
     │
     ▼
(.)[id]

Instead of replacing the gallery page, Next.js opens the photo inside a modal.

Result
Gallery Page

│ Photo Detail │
└──────────────┘

Can We Use Only (.)[id]?

Short Answer
No

If someone directly visits: 

/photos/3

Next.js needs a real page to render.

Without: 

[id]/page.tsx

there would be no standalone page for the route.

Easy Rule

Remember
[id]
   = Real Page

(.)[id]
   = Modal Version Of Same Page
Most Intercepting Route implementations contain both folders because one provides the full-page experience and the other provides the modal experience for the same URL.

What Does (.) Mean?

Intercept Current Level
(.)segment

Meaning: 

Intercept Route
At Current Level

Intercepting Route Rendering

When user navigates inside the app: 

/photos
     │
     ▼
Click Photo 3

Next.js loads: 

(.)[id]/page.tsx

instead of: 

[id]/page.tsx

The result is a modal experience.

Direct URL Access

Suppose user directly opens: 

https://site.com/photos/3

Now there is no gallery page underneath.

Next.js automatically renders: 

[id]/page.tsx

Result: 

Full Photo Detail Page
Intercepting Routes provide a modal experience during navigation but still support direct URL access.

Why Is This Powerful?

We get:

  • Shareable URLs.
  • Modal UX.
  • No State Loss.
  • No Scroll Loss.
  • Deep Linking.
  • Browser Back Support.

How It Works With Parallel Routes

Intercepting Routes are commonly used together with Parallel Routes.

Example
@modal
   │
   └── (.)photo

The modal becomes a slot.

Layout
Page Content

+ Modal Slot

This is how many modern applications implement modal routing.

Intercepting Route Syntax

Syntax Meaning
(.)segment Current Level
(..)segment One Level Up
(..)(..)segment Two Levels Up
(...)segment Root Level

UX Problem Solved:
Intercepting Routes solve the problem of losing page context, scroll position, and UI state when navigating to detail pages by displaying the destination route as an overlay instead of replacing the current page.
One-Line Definition:
Intercepting Routes allow one route to be rendered inside another route's UI (typically as a modal) so users can view route content without leaving the current page, while still preserving shareable URLs and browser navigation.
