7. What is a template.js file and how is it different from a layout?

A template.js file is a special file in the Next.js App Router that works like a wrapper around a page, but unlike a layout, it is recreated on every navigation.

In simple words, layout.js keeps the same wrapper during navigation, but template.js creates a fresh wrapper every time the route changes.

Basic Difference

Layout Template
Persists across navigation Remounts on navigation
Preserves state Resets state
Good for Header, Footer, Sidebar Good for animations, reset state, re-running effects
Same component instance is reused New component instance is created

Important Flow

In the App Router, the rendering flow is:

Hierarchy
Root Layout
      │
      ▼
Nested Layout
      │
      ▼
Template
      │
      ▼
Page
The Root Layout and Nested Layout persist, but the Template and Page are recreated during navigation.

      ▼

Notice what happened: 

Root Layout       → Same
Dashboard Layout  → Same

Without template, only the page changes. The layouts remain mounted.

Main Confusion: Page Already Remounts, Then Why Need Template?

Yes, the page already remounts during navigation.

Page Behavior
Page
  → Always recreated on navigation

So the question is:

If page already remounts, why do we need template.js?

The answer is:

Answer
Page remounts itself.

Template remounts the wrapper around the page.

So template.js is useful when we want something around the page to also restart.

Example: Animation Wrapper

Suppose we have an animation wrapper around pages.

Animation Wrapper
export default function AnimationWrapper({
  children
}) {

  return (
    <div className="fade-in">
      {children}
    </div>
  );

}

If We Put Animation Wrapper Inside Layout

dashboard/layout.tsx
export default function Layout({
  children
}) {

  return (
    <>
      <Sidebar />

      <AnimationWrapper>
        {children}
      </AnimationWrapper>
    </>
  );

}

When navigating: 

/dashboard
      ↓
/dashboard/settings

The animation wrapper stays alive because layout persists.

Result
AnimationWrapper stays mounted
Animation runs once
Animation does not restart on every navigation

If We Put Animation Wrapper Inside Template

dashboard/template.tsx
export default function Template({
  children
}) {

  return (
    <AnimationWrapper>
      {children}
    </AnimationWrapper>
  );

}

When navigating: 

/dashboard
      ↓
/dashboard/settings

The old template is destroyed and a new template is created.

Result
Old AnimationWrapper destroyed
New AnimationWrapper created
Animation starts again
This is one of the main reasons to use template.js: page transition animations.

Example: Reset State

Suppose our wrapper has state.

dashboard/template.tsx
'use client';

import { useState } from 'react';

export default function Template({
  children
}) {

  const [step, setStep] = useState(1);

  return (
    <>
      <h2>Step {step}</h2>

      <button onClick={() => setStep(step + 1)}>
        Next Step
      </button>

      {children}
    </>
  );

}

User opens: 

/product/1

Then user changes the step: 

Step = 5

Now user navigates: 

/product/2

Because template remounts, the state resets: 

Step = 1
Template is useful when we want wrapper state to reset automatically on route change.

useEffect Example

The useEffect example helps us see whether a component mounted again or not.

Client Component
'use client';

import { useEffect } from 'react';

export default function Template({
  children
}) {

  useEffect(() => {

    console.log('Template Mounted');

  }, []);

  return children;

}

The empty dependency array means this effect runs only when the component mounts.

When Using Layout
Open /dashboard

Layout Mounted
Dashboard Page Mounted

Navigate to /dashboard/settings

Settings Page Mounted
Layout Mounted does not run again
When Using Template
Open /dashboard

Template Mounted
Dashboard Page Mounted

Navigate to /dashboard/settings

Template Mounted
Settings Page Mounted
With template, useEffect runs again because the template component is recreated on navigation.

Where Do html and body Tags Go?

A template cannot replace the Root Layout.

The html and body tags always belong inside the Root Layout, not inside template.js.
app/layout.tsx
export default function RootLayout({
  children
}) {

  return (
    <html lang="en">
      <body>
        {children}
      </body>
    </html>
  );

}
app/template.tsx
export default function Template({
  children
}) {

  return (
    <>
      {children}
    </>
  );

}

The template only wraps content. It does not create the HTML document.

Should Header and Footer Be Inside Template?

Technically, we can put Header and Footer inside template, but usually we should not.

Recommended: Header and Footer Inside Layout

layout.tsx
export default function Layout({
  children
}) {

  return (
    <>
      <Header />

      {children}

      <Footer />
    </>
  );

}

This keeps Header and Footer mounted across navigations.

Result
Header stays mounted
Footer stays mounted
Only page content changes

Not Recommended Usually: Header and Footer Inside Template

template.tsx
export default function Template({
  children
}) {

  return (
    <>
      <Header />

      {children}

      <Footer />
    </>
  );

}
Result
Old Header destroyed
Old Footer destroyed

New Header created
New Footer created

This is only useful if we intentionally want Header or Footer to reset on every navigation.

Real World Rule

Use Layout For
Header
Footer
Sidebar
Navbar
Persistent UI
Shared State
Use Template For
Page transition animation
Resetting wrapper state
Re-running useEffect
Fresh wrapper instance
Route-specific initialization

One-Line Definition:
A template.js file is a route wrapper that behaves like a layout but remounts on every navigation, allowing shared wrappers to reset state, rerun effects, and restart animations, while layouts persist across navigations.
