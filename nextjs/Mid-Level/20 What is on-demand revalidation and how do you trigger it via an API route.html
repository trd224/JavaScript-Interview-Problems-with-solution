What is on-demand revalidation and how do we trigger it via an API route?

On-demand revalidation means manually clearing cached data when something changes, instead of waiting for revalidate: 60, revalidate: 3600, or another time-based interval.

In simple words, on-demand revalidation lets us refresh cached pages or cached data immediately after an event, such as product update, blog publish, CMS webhook, or admin action.

Problem Without On-Demand Revalidation

Suppose our product page uses ISR.

app/products/page.tsx
export default async function ProductsPage() {
  const res = await fetch('https://api.example.com/products', { next: { revalidate: 3600 } });
  const products = await res.json();

  return <h1>Products</h1>;
}

This means cached data can remain old for 1 hour.

Problem
Product Updated Now
        ↓
Cache Still Valid For 1 Hour
        ↓
User May See Old Data

Solution: On-Demand Revalidation

When product data changes, we manually tell Next.js:

Meaning
Clear Cache Now
        ↓
Next Request Gets Fresh Data

This is called on-demand revalidation.

Two Main Functions

Function Purpose
revalidatePath() Refresh a specific page or route path
revalidateTag() Refresh all cached fetches using a specific tag

1. Revalidate By Path

Use revalidatePath() when we want to refresh a specific route.

Example
revalidatePath('/products');

This tells Next.js to invalidate the cache for the /products page.

API Route Example

app/api/revalidate/route.ts
import { revalidatePath } from 'next/cache';
import { NextRequest } from 'next/server';

export async function POST(request: NextRequest) {
  const secret = request.headers.get('x-revalidate-secret');

  if (secret !== process.env.REVALIDATE_SECRET) {
    return Response.json({ message: 'Invalid secret' }, { status: 401 });
  }

  revalidatePath('/products');

  return Response.json({ revalidated: true, path: '/products' });
}

How To Trigger This API Route

POST Request
await fetch('https://example.com/api/revalidate', {
  method: 'POST',
  headers: { 'x-revalidate-secret': 'my-secret-key' }
});

Flow

On-Demand Revalidation Flow
Admin Updates Product
        ↓
Call /api/revalidate
        ↓
Check Secret Key
        ↓
revalidatePath('/products')
        ↓
Products Cache Invalidated
        ↓
Next User Gets Fresh Data

2. Revalidate By Tag

Use revalidateTag() when multiple pages use the same cached data.

Tagged Fetch
export default async function ProductsPage() {
  const res = await fetch('https://api.example.com/products', { next: { tags: ['products'] } });
  const products = await res.json();

  return <h1>Products</h1>;
}

Now this fetch is connected to the products tag.

API Route Using revalidateTag()

app/api/revalidate-products/route.ts
import { revalidateTag } from 'next/cache';
import { NextRequest } from 'next/server';

export async function POST(request: NextRequest) {
  const secret = request.headers.get('x-revalidate-secret');

  if (secret !== process.env.REVALIDATE_SECRET) {
    return Response.json({ message: 'Invalid secret' }, { status: 401 });
  }

  revalidateTag('products', 'max');

  return Response.json({ revalidated: true, tag: 'products' });
}

When To Use Path vs Tag?

Use Case Best Choice
Refresh one page revalidatePath('/products')
Refresh many pages using same data revalidateTag('products', 'max')
CMS updated one blog page revalidatePath('/blog/my-post')
Product data used on home, products, cart revalidateTag('products', 'max')

Real World Example

Suppose an admin changes a product price.

Before Update
{
  "price": 100
}
After Update
{
  "price": 120
}

Without on-demand revalidation, users may still see old cached price.

Without Revalidation
Database Price = 120
Cache Price    = 100
User Sees      = 100

With on-demand revalidation:

With Revalidation
Product Updated
        ↓
API Route Called
        ↓
Cache Invalidated
        ↓
Next User Sees 120

Environment Variable

.env.local
REVALIDATE_SECRET=my-secret-key

The secret prevents anyone from publicly calling our revalidation API.

Important Note

On-demand revalidation does not mean the browser automatically refreshes instantly. It invalidates the server cache, so the next request or navigation gets fresh data.

One-Line Definition:
On-demand revalidation means manually invalidating cached pages or cached data using revalidatePath() or revalidateTag(), usually from a Route Handler API after data changes.
