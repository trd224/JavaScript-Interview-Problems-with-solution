1. What is the Difference Between the Pages Router and the App Router?

The Pages Router and App Router are two routing systems in Next.js. The Pages Router uses the pages directory and traditional data-fetching methods, while the App Router uses the app directory and introduces Server Components, nested layouts, streaming, and the Metadata API.

The App Router is the modern routing system introduced in Next.js 13 and is now the recommended approach for new applications.

Pages Router

The Pages Router uses the pages folder to create routes.

Project Structure
pages
├── index.js
├── about.js
└── products
    └── [id].js
pages/about.js
export default function About() {
  return <h1>About Page</h1>;
}

This creates: 

/about

App Router

The App Router uses the app folder.

Project Structure
app
├── page.js
├── about
│   └── page.js
└── products
    └── [id]
        └── page.js
app/about/page.js
export default function About() {
  return <h1>About Page</h1>;
}

This also creates: 

/about

Data Fetching

Pages Router

pages/products.js
export async function getServerSideProps() {

  const products =
    await fetch('https://api.example.com/products')
      .then(res => res.json());

  return {
    props: { products: products }
  };

}

export default function Products({
  products
}) {

  return (
    <div>
      {products.length}
    </div>
  );

}

App Router

app/products/page.js
export default async function Page() {

  const products =
    await fetch('https://api.example.com/products')
      .then(res => res.json());

  return (
    <div>
      {products.length}
    </div>
  );

}

No getServerSideProps() is required.

Layouts

Pages Router

Uses a shared layout pattern manually.

pages/_app.js
import Layout from '../components/Layout';

export default function App({ Component, pageProps }) {

  return (
    <Layout>
      <Component {...pageProps} />
    </Layout>
  );

}
Layout 
export default function Layout({ children }) {
  return (
    <>
      <header>
        Header
      </header>

      <main>
        {children}
      </main>

      <footer>
        Footer
      </footer>
    </>
  );
}
or 
import Header from '../components/Header';
import Footer from '../components/Footer';

export default function MyApp({ Component, pageProps }) {

  return (
    <>
      <Header />

      <Component {...pageProps} />

      <Footer />
    </>
  );

}

App Router

Has built-in nested layouts.

app/layout.js
export default function RootLayout({ children }) {

  return (
    <html>
      <body>
        {children}
      </body>
    </html>
  );

}

Metadata

Pages Router

pages/about.js
import Head from 'next/head';

export default function About() {

  return (
    <>
      <Head>
        <title>About</title>
      </Head>

      <h1>About</h1>
    </>
  );

}

App Router

app/about/page.js
export const metadata = {
  title: 'About'
};

export default function About() {
  return <h1>About</h1>;
}

Loading UI

Pages Router

Manual loading state management is required.

App Router

app/loading.js
export default function Loading() {
  return <p>Loading...</p>;
}

Automatic loading UI support.

Error Handling

Pages Router

Requires custom error handling.

App Router

app/error.js
'use client';

export default function Error({
  error
}) {

  return (
    <h1>
      Something went wrong
    </h1>
  );

}

Server Components

Pages Router

❌ Not Supported

App Router

✅ Supported By Default
Server Component
export default async function Page() {

  const data =
    await fetch('https://api.example.com');

  return <h1>Hello</h1>;

}

Streaming

Pages Router

❌ Not Supported

App Router

✅ Supported
Example
import { Suspense } from 'react';

export default function Page() {

  return (
    <Suspense fallback="Loading...">
      <Products />
    </Suspense>
  );

}

Pages Router

Important Note

Both routers can coexist in the same project. Next.js allows gradual migration from the Pages Router to the App Router.

Interview Tip

The biggest differences are that the App Router provides Server Components, nested layouts, streaming, Route Handlers, and the Metadata API, while the Pages Router relies on getServerSideProps, getStaticProps, and the Head component.
One-Line Definition:
The Pages Router is the traditional routing system based on the pages directory, while the App Router is the modern routing system based on the app directory that adds Server Components, nested layouts, streaming, and the Metadata API.
