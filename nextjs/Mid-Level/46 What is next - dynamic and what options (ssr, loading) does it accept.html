46. What is next/dynamic and what options (ssr, loading) does it accept?

next/dynamic is a Next.js utility that allows us to dynamically import components instead of including them in the initial JavaScript bundle.

Simple Meaning:
next/dynamic lets us load a component only when it is needed, reducing the initial bundle size and improving page performance.

Why Do We Need next/dynamic?

Normally, imported components become part of the page bundle.

Normal Import
import Chart from './Chart';

export default function Page() {
  return <Chart />;
}

Flow: 

Page Loads
      ↓
Chart Downloaded
      ↓
Page Rendered

Even if the user never views the chart, its JavaScript is still downloaded.

Using next/dynamic

Dynamic Import
import dynamic from 'next/dynamic';

const Chart = dynamic(() => import('./Chart'));

export default function Page() {
  return <Chart />;
}

Flow: 

Page Loads
      ↓
Chart Not Downloaded Yet
      ↓
Chart Needed
      ↓
Download Chart
      ↓
Render Chart

Basic Syntax

Syntax
const Component = dynamic(
  () => import('./Component'),
  options
);

Option 1: ssr

The ssr option controls whether the component should be rendered on the server.

Disable SSR
import dynamic from 'next/dynamic';

const Map = dynamic(() => import('./Map'), {
  ssr: false
});

Flow: 

Server Render
      ↓
Map Skipped
      ↓
Browser Loads
      ↓
Map Rendered
Use Case:
Browser-only libraries such as Google Maps, Leaflet, Chart.js, or components that access window, document, or localStorage.

Without ssr: false

Problem Example
const Map = dynamic(() => import('./Map'));

If the component contains: 

window.location.href

The server will throw an error because window does not exist on the server.

Option 2: loading

The loading option displays a fallback component while the dynamic component is being downloaded.

Loading Example
import dynamic from 'next/dynamic';

const Chart = dynamic(() => import('./Chart'), {
  loading: () => <p>Loading Chart...</p>
});

Flow: 

Page Loads
      ↓
Loading Chart...
      ↓
Chart Bundle Downloaded
      ↓
Chart Rendered

Using Both Options Together

Example
import dynamic from 'next/dynamic';

const Map = dynamic(() => import('./Map'), {
  ssr: false,
  loading: () => <p>Loading Map...</p>
});

Meaning:

Option Effect
ssr: false Do not render on server
loading Show fallback while downloading

Real World Example

Suppose our dashboard contains a heavy chart library.

Without Dynamic Import
Dashboard Page
      ↓
Download Chart Library
      ↓
Download Table
      ↓
Download Filters
      ↓
Render

The page becomes slower.

With Dynamic Import
Dashboard Page
      ↓
Render Immediately
      ↓
Chart Downloads Separately
      ↓
Chart Appears Later

The dashboard becomes faster because the chart code is split into a separate chunk.

Use Case Use next/dynamic?

Small Header Component ❌ Usually No

One-Line Definition:
next/dynamic dynamically loads components to reduce initial bundle size, while ssr controls server rendering and loading provides a temporary fallback UI during download.
