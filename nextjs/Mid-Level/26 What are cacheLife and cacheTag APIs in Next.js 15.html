26. What are cacheLife and cacheTag APIs in Next.js 15?

When using the new 'use cache' directive, Next.js provides two helper APIs:

  • cacheLife() → Controls how long cached data lives.
  • cacheTag() → Assigns tags to cached data for later invalidation.
Think of it like this:

cacheLife = When should cache expire?
cacheTag = Which cache should be cleared?

Both APIs are designed to work together with the 'use cache' directive. :contentReference[oaicite:0]{index=0}

1. cacheLife()

cacheLife() controls the lifetime of a cached function or component. :contentReference[oaicite:1]{index=1}

Basic Example
import { cacheLife } from 'next/cache';

async function getProducts() {
  'use cache';

  cacheLife('hours');

  return db.product.findMany();
}

Meaning: 

First Request
      ↓
Query Database
      ↓
Store In Cache

Next Requests
      ↓
Use Cache

After Cache Life Ends
      ↓
Generate Fresh Data

What Problem Does cacheLife Solve?

Suppose we have:

Database Query
return db.product.findMany();

Without caching: 

Request 1
   ↓
Database Query

Request 2
   ↓
Database Query

Request 3
   ↓
Database Query

With:

cacheLife()
cacheLife('hours');

The query result can be reused instead of hitting the database every time. :contentReference[oaicite:2]{index=2}

2. cacheTag()

cacheTag() attaches one or more tags to cached data. Later, we can clear that cache using revalidateTag(). :contentReference[oaicite:3]{index=3}

Basic Example
import { cacheTag } from 'next/cache';

async function getProducts() {
  'use cache';

  cacheTag('products');

  return db.product.findMany();
}

Now this cached data belongs to: 

products

tag.

Clearing Tagged Cache

Revalidate
import { revalidateTag } from 'next/cache';

revalidateTag('products', 'max');

Flow: 

Product Updated
      ↓
revalidateTag('products')
      ↓
Products Cache Cleared
      ↓
Next Request Gets Fresh Data

Real World Example

Suppose we have an e-commerce site.

Products Function
import { cacheLife, cacheTag } from 'next/cache';

async function getProducts() {
  'use cache';

  cacheLife('hours');

  cacheTag('products');

  return db.product.findMany();
}

Here:

API Purpose

cacheTag('products') Identifies the cache entry

  "price": 120

Backend then triggers:

Invalidate Cache
revalidateTag('products', 'max');

Result:

Deleted

       ↓

       ↓
New Cache Created
