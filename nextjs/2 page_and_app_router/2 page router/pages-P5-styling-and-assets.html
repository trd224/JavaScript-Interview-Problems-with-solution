P5-01How do you add global CSS to a Next.js application?

Global CSS is CSS that applies to the entire application instead of a single component.

Global CSS allows us to define styles such as fonts, resets, colors, layouts, and utility classes that can be used throughout our application.

Why Do We Need Global CSS?

Some styles should be available everywhere.

Examples:
  • Font Family
  • CSS Reset
  • Body Background
  • Text Colors
  • Global Utility Classes

Pages Router

In the Pages Router, global CSS is imported inside pages/_app.js.

Step 1: Create Global CSS File

styles/globals.css
body {
  font-family: Arial, sans-serif;
  background: #f5f5f5;
}
h1 {
  color: blue;
}

Step 2: Import Global CSS

pages/_app.js
import '../styles/globals.css';
export default function MyApp({ Component, pageProps }) {
  return (
    <Component {...pageProps} />
  );
}

Project Structure

Pages Router
pages/
├── _app.js
├── index.js
styles/
└── globals.css

App Router

In the App Router, global CSS is imported inside app/layout.js.

Step 1: Create Global CSS File

app/globals.css
body {
  font-family: Arial, sans-serif;
  background: #f5f5f5;
}
h1 {
  color: blue;
}

Step 2: Import Global CSS

app/layout.js
import './globals.css';
export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>
        {children}
      </body>
    </html>
  );
}

Project Structure

App Router
app/
├── layout.js
├── page.js
└── globals.css

What Happens Internally?

Pages Router
globals.css
      │
      ▼
Imported In _app.js
      │
      ▼
Available To All Pages
App Router
globals.css
      │
      ▼
Imported In layout.js
      │
      ▼
Available To Entire App

Can We Import Global CSS Anywhere?

No.

Global CSS can only be imported in special root files.

Router Import Location
Pages Router pages/_app.js
App Router app/layout.js

Incorrect Example

components/Header.js
import './globals.css';

This will cause a build error.

Global CSS vs CSS Modules

Feature Global CSS CSS Modules
Scope Entire Application Single Component
Class Name Collision Possible Prevented
Use Case Global Styles Component Styles

Real Example

app/globals.css
body {
  margin: 0;
  font-family: Arial, sans-serif;
}
.container {
  max-width: 1200px;
  margin: auto;
}
app/page.js
export default function Home() {
  return (
    <div className="container">
      <h1>Home Page</h1>
    </div>
  );
}

The container class is available because it was defined globally.

Interview Tip

A common misconception is that global CSS can be imported into any component. In reality, Next.js only allows global CSS imports in pages/_app.js (Pages Router) or app/layout.js (App Router).
One-Line Definition:
Global CSS in Next.js is added by creating a global stylesheet and importing it in pages/_app.js (Pages Router) or app/layout.js (App Router), making the styles available throughout our application.

P5-02What are CSS Modules and how do you use them in Next.js?

CSS Modules are CSS files whose styles are scoped locally to a specific component.

CSS Modules allow us to write component-specific CSS without worrying about class name conflicts in our application.

Why Do We Need CSS Modules?

Suppose we have two components:

Header.css
.title {
  color: blue;
}
Footer.css
.title {
  color: red;
}

Both files use the same class name: 

title

This can cause style conflicts when using normal CSS.

How CSS Modules Solve This

Next.js automatically generates unique class names.

Generated Internally
title
      │
      ▼
Header_title__a12bc
title
      │
      ▼
Footer_title__x78yz

Now both classes are unique and cannot conflict.

Creating a CSS Module

CSS Module files must end with .module.css.

Button.module.css
.button {
  background: blue;
  color: white;
  padding: 10px 20px;
  border: none;
}

Using a CSS Module

Button.js
import styles from './Button.module.css';
export default function Button() {
  return (
    <button className={styles.button}>
      Click Me
    </button>
  );
}

Project Structure

Example
components/
├── Button.js
└── Button.module.css

What Happens Internally?

        │
        ▼
Import styles Object
        │
        ▼
styles.button
        │
        ▼
Generated Class Name
Button_button__a1b2c

Example with Multiple Classes

Card.module.css

When we write styles.button, Next.js automatically maps it to a unique generated class name behind the scenes.

P5-03What built-in CSS-in-JS solution does Next.js support in the App Router?

Next.js supports styled-jsx as its built-in CSS-in-JS solution.

styled-jsx allows us to write component-scoped CSS directly inside our JSX without needing separate CSS files.

What is CSS-in-JS?

CSS-in-JS means writing CSS directly inside JavaScript or React components.

Instead of creating:

Separate CSS File
Button.js
Button.css

We can keep styles inside the component itself.

Built-in Solution: styled-jsx

Example
export default function Home() {
  return (
    <div>
      <h1>Hello Next.js</h1>
      <style jsx>{`
        h1 {
          color: blue;
        }
      `}</style>
    </div>
  );
}

The CSS is scoped only to this component.

What Happens Internally?

Flow
Component
      │
      ▼
styled-jsx
      │
      ▼
Generate Unique Class
      │
      ▼
Apply CSS Only
To This Component

Example of Style Isolation

Component A
export default function A() {
  return (
    <>
      <h1>A</h1>
      <style jsx>{`
        h1 {
          color: red;
        }
      `}</style>
    </>
  );
}
Component B
export default function B() {
  return (
    <>
      <h1>B</h1>
      <style jsx>{`
        h1 {
          color: blue;
        }
      `}</style>
    </>
  );
}

Both components can style h1 differently without conflicts.

Global Styles with styled-jsx

We can also create global styles.

Global Style
<style jsx global>{`
  body {
    margin: 0;
    background: #111827;
  }
`}</style>

Using global removes style scoping.

App Router Recommendation

Although styled-jsx is supported, the App Router documentation generally recommends:

  • CSS Modules
  • Tailwind CSS
  • Global CSS

for most styling needs.

CSS Modules Example

Button.module.css
.button {
  background: blue;
  color: white;
}
Button.js
import styles from './Button.module.css';
export default function Button() {
  return (
    <button className={styles.button}>
      Save
    </button>
  );
}

Does Next.js Include styled-components?

No.

Next.js does not include styled-components by default.

We must install it separately.

Install
npm install styled-components

Does Next.js Include Emotion?

No.

Emotion must also be installed separately.

Install
npm install @emotion/react @emotion/styled

Built-in vs External CSS-in-JS

Library Built Into Next.js
styled-jsx ✅
styled-components ❌
Emotion ❌
Stitches ❌

Important App Router Note

A common misconception is that styled-jsx is the recommended styling solution for the App Router. In reality, while styled-jsx is still supported, Next.js generally recommends CSS Modules, Tailwind CSS, and Global CSS for most App Router applications.

When Should We Use styled-jsx?

  • Small component-specific styles.
  • Quick prototypes.
  • Simple isolated styling.
  • When we want styles inside the component file.

When Should We Prefer CSS Modules?

  • Large applications.
  • Reusable components.
  • Better maintainability.
  • Team projects.

Interview Tip

The built-in CSS-in-JS solution in Next.js is styled-jsx, but in modern App Router projects, CSS Modules and Tailwind CSS are more commonly used.
One-Line Definition:
Next.js supports styled-jsx as its built-in CSS-in-JS solution, allowing us to write component-scoped CSS directly inside JSX without class name conflicts.

P5-04How does the next/image component improve performance over a plain ?

The next/image component is a built-in Next.js component that automatically optimizes images for better performance, faster loading, and improved Core Web Vitals.

Unlike a normal <img> tag, next/image automatically resizes, compresses, lazy loads, and serves modern image formats when possible.

Plain img Example

Regular HTML Image
<img
  src="/product.jpg"
  alt="Product"
/>

The browser downloads exactly the image file that exists on the server.

No optimization happens automatically.

next/image Example

Next.js Image Component
import Image from 'next/image';
export default function Product() {
  return (
    <Image
      src="/product.jpg"
      alt="Product"
      width={500}
      height={300}
    />
  );
}

Next.js automatically optimizes the image before sending it to the browser.

Why Do We Need Image Optimization?

Suppose our original image is:

Original Image
4000 × 3000 Pixels
5 MB

But our page displays it at:

Displayed Size
400 × 300 Pixels

Using a normal img: 

Download Full 5 MB Image

Most of those pixels are never used.

This wastes bandwidth and slows page loading.

Automatic Image Resizing

The Image component generates properly sized images.

Images below the viewport are not loaded immediately.

Hero Image

This is useful for:

  • Hero Images
  • Above-the-fold Content
  • Landing Pages

Why Do We Need priority If Above-the-Fold Images Already Load?

A common misconception is that priority is required to load above-the-fold images.

Even without priority, an image that is visible in the viewport is not lazy loaded and will load automatically.

Without priority
HTML Received
      │
      ▼
Browser Parses HTML
      │
      ▼
Finds Image
      │
      ▼
Starts Downloading Image

The browser discovers the image while parsing the page and then starts downloading it.

With priority
HTML Received
      │
      ▼
Preload Image
(rel="preload")
      │
      ▼
Start Downloading Image ASAP
      │
      ▼
Render Page

When priority is used, Next.js adds a preload hint so the browser can begin downloading the image earlier.

Internally
<link
  rel="preload"
  as="image"
  href="/hero.jpg"
/>

This can improve Largest Contentful Paint (LCP) when the image is a major part of the page.

Behavior Above-the-Fold Image Below-the-Fold Image
next/image Loads Automatically Lazy Loaded
next/image + priority Preloaded and Higher Priority Usually Not Needed
Important:
priority does not make an above-the-fold image load instead of lazy loading. The image would already load automatically. It simply tells the browser that the image is critical and should be fetched as early as possible.

P5-05What is the public folder used for?

The public folder is a special folder in Next.js used to store static files that should be served directly to the browser without processing by Next.js.

The public folder allows us to store images, icons, videos, PDFs, robots.txt, sitemap.xml, and other static assets that can be accessed directly through a URL.

Why Do We Need the public Folder?

Some files do not need bundling, compilation, or optimization.

Examples:

  • Images
  • Favicons
  • PDF Files
  • Videos
  • robots.txt
  • sitemap.xml

These files can simply be served as-is.

Project Structure

│   ├── banner.jpg
│   ├── video.mp4
│   ├── brochure.pdf
│   └── favicon.ico
├── package.json
└── next.config.js

How Files Are Accessed

Files inside the public folder are automatically available from the root URL.

  • Favicons.
  • PDF downloads.

P5-06How do you configure Tailwind CSS with Next.js?

Tailwind CSS is a utility-first CSS framework that allows us to style our Next.js application using utility classes directly inside JSX.

In modern Next.js projects, Tailwind CSS can be added during project creation, or manually installed using Tailwind CSS packages and PostCSS configuration.

Method 1: Configure Tailwind During Project Creation

The easiest way is to select Tailwind CSS while creating a new Next.js project.

Create Next.js App
npx create-next-app@latest my-app

During setup, choose:

CLI Option
Would you like to use Tailwind CSS? Yes

Next.js will automatically create the required Tailwind setup.

Method 2: Manual Setup

If our application already exists, we can install Tailwind CSS manually.

Step 1: Install Tailwind CSS

Terminal
npm install tailwindcss @tailwindcss/postcss postcss

In the latest Tailwind setup, @tailwindcss/postcss is used as the PostCSS plugin.

Step 2: Create PostCSS Config

postcss.config.mjs
const config = {
  plugins: {
    '@tailwindcss/postcss': {}
  }
};
export default config;

Step 3: Import Tailwind in Global CSS

app/globals.css
@import 'tailwindcss';

Step 4: Import Global CSS in Root Layout

app/layout.js
import './globals.css';
export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>

P5-07What is next/font and how does it eliminate layout shift from font loading?

next/font is a built-in Next.js feature that optimizes font loading by downloading, self-hosting, and serving fonts directly from our application.

next/font helps eliminate layout shift caused by font loading and improves Core Web Vitals such as CLS (Cumulative Layout Shift).

Why Do We Need next/font?

Traditionally, web fonts are loaded from external servers such as Google Fonts.

Traditional Method
<link
  href="https://fonts.googleapis.com/css2?family=Roboto"
  rel="stylesheet"
/>

This causes an extra network request before the font becomes available.

The Layout Shift Problem

Suppose our page contains:

HTML
<h1>Welcome To Next.js</h1>

When the page loads:

Step 1
Font Not Loaded Yet
Browser Uses
Fallback Font
Welcome To Next.js

After the font downloads:

Step 2
Custom Font Loaded
Welcome To Next.js

The text width and height may change.

This causes content movement on the screen.

What is Layout Shift?

Without next/font
Page Loads
      │
      ▼
Fallback Font
Welcome To Next.js
      │
      ▼
Font Downloads
      │
      ▼
Text Resizes
      │
      ▼
Content Moves

Users see the page jump unexpectedly.

How next/font Solves This

Next.js downloads the font during the build process and serves it directly from our application.

next/font Flow
Build Application
      │

P5-09How do you configure remote image domains in next.config.js?

By default, the next/image component only optimizes local images and trusted remote sources.

To use external images with next/image, we must explicitly allow the image domain in next.config.js.

Why Is Configuration Required?

Suppose our application displays an image from:

Remote Image URL
https://images.example.com/product.jpg

Using it directly:

Image Component
import Image from 'next/image';
export default function Product() {
  return (
    <Image
      src="https://images.example.com/product.jpg"
      alt="Product"
      width={500}
      height={300}
    />
  );
}

Will produce an error because Next.js does not trust unknown domains.

Error Example

Runtime Error
Invalid src prop
Hostname "images.example.com"
is not configured under images
in your next.config.js

Solution: Configure Remote Domains

next.config.js
/** @type {import('next').NextConfig} */
const nextConfig = {
  images: {
    domains: [
      'images.example.com'
    ]
  }
};
module.exports = nextConfig;

Now Next.js allows images from: 

https://images.example.com

Complete Example

next.config.js
const nextConfig = {
  images: {
    domains: [
      'images.example.com'
    ]
  }
};
module.exports = nextConfig;
app/page.js
import Image from 'next/image';
export default function Home() {
  return (
    <Image
      src="https://images.example.com/product.jpg"
      alt="Product"
      width={600}
      height={400}
    />
  );
}

Modern Approach: remotePatterns

In newer versions of Next.js, remotePatterns is preferred because it provides more control.

next.config.js
const nextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'images.example.com',
        pathname: '/**'
      }
    ]
  }
};
module.exports = nextConfig;

Why remotePatterns Is Better?

With domains, all images from the domain are allowed.

With remotePatterns, we can restrict:

  • Protocol
  • Hostname
  • Port
  • Pathname

Example

Specific Folder Only
const nextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'images.example.com',
        pathname: '/products/**'
      }
    ]
  }
};
module.exports = nextConfig;

Allowed: 

https://images.example.com/products/shirt.jpg
https://images.example.com/products/shoes.jpg

Blocked: 

https://images.example.com/users/profile.jpg

Multiple Domains

const nextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'images.example.com'
      },
      {
        protocol: 'https',
        hostname: 'cdn.example.com'
      }
    ]
  }
};
module.exports = nextConfig;

P5-10How do you import and use SVG files as React components in Next.js?

SVG files can be used in Next.js in multiple ways, but to use an SVG as a React component we typically use SVGR.

Using SVG as a React component allows us to pass props, apply CSS classes, change colors, control width and height, and reuse SVGs like normal React components.

Method 1: Use SVG from the public Folder

The simplest method is storing SVG files in the public folder.

Project Structure
public/
└── logo.svg
Usage
export default function Home() {
  return (
    <img
      src="/logo.svg"
      alt="Logo"
    />
  );
}

This works, but the SVG behaves like a normal image.

We cannot easily change SVG properties using React props.

Method 2: Use next/image

Example
import Image from 'next/image';
export default function Home() {
  return (
    <Image
      src="/logo.svg"
      alt="Logo"
      width={100}
      height={100}
    />
  );
}

This is useful when SVG behaves like a static image.

Method 3: Import SVG as a React Component (Recommended)

To use SVG files as React components, we need SVGR.

Step 1: Install SVGR

Terminal
npm install -D @svgr/webpack

Step 2: Configure next.config.js

next.config.js
const nextConfig = {
  webpack(config) {
    config.module.rules.push({
      test: /\.svg$/i,
      issuer: /\.[jt]sx?$/,
      use: ['@svgr/webpack']
    });
    return config;
  }
};
module.exports = nextConfig;

Step 3: Create SVG File

assets/logo.svg
<svg
  xmlns="http://www.w3.org/2000/svg"
  viewBox="0 0 100 100"
>
  <circle
    cx="50"
    cy="50"
    r="40"
    fill="green"
  />
</svg>

Step 4: Import SVG as Component

app/page.js
import Logo from '../assets/logo.svg';
export default function Home() {
  return (
    <Logo />
  );
}
