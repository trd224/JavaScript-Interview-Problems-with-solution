P6-01How do you add meta tags using the Head component ? (Pages Router)

In the Pages Router, we use the Head component from next/head to add meta tags, title, description, Open Graph tags, and other <head> elements for a page.

Example

pages/about.js
import Head from 'next/head';
export default function About() {
  return (
    <>
      <Head>
        <title>About Us</title>
        <meta name="description" content="Learn more about our company." />
        <meta name="keywords" content="Next.js, React, Web Development" />
        <meta property="og:title" content="About Us" />
      </Head>
      <h1>About Page</h1>
    </>
  );
}

Generated HTML

HTML Output
<html>
<head>
  <title>About Us</title>
  <meta name="description" content="Learn more about our company." />
  <meta name="keywords" content="Next.js, React, Web Development" />
  <meta property="og:title" content="About Us" />
</head>
<body>
  <h1>About Page</h1>
</body>
</html>

Key Points

  • Head is imported from next/head.
  • Meta tags added inside Head are rendered inside the page's <head> section.
  • Each page can have its own title and SEO metadata.
  • Common tags include title, description, keywords, Open Graph og:*, and Twitter Card tags.
One-Line Definition:
Head is a Next.js component used in the Pages Router to add page-specific metadata and elements inside the HTML <head> section.

P6-02Why is SSR/SSG Better for SEO than Pure CSR?

SSR (Server-Side Rendering) and SSG (Static Site Generation) are generally better for SEO because search engines receive fully rendered HTML immediately, whereas pure CSR (Client-Side Rendering) initially sends very little content.

Search engines can index SSR and SSG pages more easily because the content already exists in the HTML response.

Pure CSR Example

React SPA
<body>
  <div id="root"></div>
  <script src="bundle.js"></script>
</body>

When the browser first receives this page, there is almost no content.

CSR Rendering Flow

CSR Flow
Request
   │
   ▼
Empty HTML
   │
   ▼
Download JavaScript
   │
   ▼
Execute JavaScript
   │
   ▼
Fetch Data
   │
   ▼

The crawler may need to execute JavaScript before it can see the page content.

SSR Example

Server-Side Rendering
<body>
  <h1>iPhone 15</h1>
  <p>Latest Apple smartphone</p>
</body>

The content already exists in the HTML response sent by the server.

SSR Rendering Flow

SSR Flow
Request
   │
   ▼
Server Renders HTML
   │
   ▼
Send Complete HTML
   │
   ▼
Crawler Reads Content

SSG Example

Static Site Generation
Build Time
    │
    ▼
Generate HTML
    │
    ▼
Store HTML File
    │
    ▼
Serve HTML Directly

The page is generated during the build and can be served instantly.

What Does Google See?

CSR
<body>
  <div id="root"></div>
</body>
SSR / SSG
<body>
  <h1>iPhone 15</h1>
  <p>Latest Apple smartphone</p>
</body>

SSR and SSG provide actual content immediately, making crawling and indexing easier.

SEO Advantages of SSR/SSG

  • Content is available immediately.
  • Faster indexing.
  • Better metadata visibility.
  • Better Open Graph previews.
  • Improved crawl efficiency.
  • Faster initial page load.

Metadata Example

SSR / SSG
<head>
  <title>iPhone 15</title>
  <meta
    name="description"
    content="Latest Apple smartphone"
  />
</head>

Search engines can immediately read the title and description.

Can Google Index CSR?

Yes.

Modern Google crawlers can execute JavaScript and index CSR applications.

The problem is not that Google cannot index CSR. The problem is that CSR requires an extra rendering step, which can delay indexing and make crawling less efficient.

Real-World Example

Website Type Recommended

Documentation SSG
Marketing Website SSG

Dashboard CSR

SSR vs SSG vs CSR

Feature SSR SSG CSR
SEO ✅ ✅ ⚠️
Initial HTML Content ✅ ✅ ❌
Fast First Paint ✅ ✅ ❌
Requires JS To See Content ❌ ❌ ✅

Important Note

The biggest SEO advantage of SSR and SSG is that search engines receive meaningful HTML immediately instead of needing to execute JavaScript before seeing page content.

Interview Tip

Google can index CSR applications, but SSR and SSG are generally preferred for SEO because content and metadata are already present in the initial HTML response.
One-Line Definition:
SSR and SSG are better for SEO than pure CSR because search engines receive fully rendered HTML and metadata immediately, making crawling and indexing faster and more reliable.
