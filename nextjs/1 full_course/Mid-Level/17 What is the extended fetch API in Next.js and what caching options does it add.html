17. What is the Extended fetch API in Next.js and what caching options does it add?

Next.js extends the standard JavaScript fetch() API by adding built-in caching, revalidation, deduplication, and data cache controls.

The browser's fetch API only downloads data. Next.js fetch can also control how long data is cached and when it should be refreshed.

Normal JavaScript fetch()

In plain React or Node.js:

Standard Fetch
const res = await fetch('https://api.example.com/products');
const products = await res.json();

This simply requests data.

There is no built-in:

  • ISR
  • Data Cache
  • Automatic Revalidation
  • Cache Tags

Next.js Extended fetch()

Next.js adds additional options:

Extended Fetch
const res = await fetch('https://api.example.com/products', {
  cache: 'force-cache',
  next: {
    revalidate: 60,
    tags: ['products']
  }
});

These options do not exist in the normal browser fetch API.

1. cache: 'force-cache'

This tells Next.js to cache the result.

Example
const res = await fetch('https://api.example.com/products', {
  cache: 'force-cache'
});

Flow

First Request
      ↓
Fetch API
      ↓
Save In Cache

Second Request
      ↓
Read From Cache

Good for:

  • Blog Posts
  • Documentation
  • Static Content
This is similar to SSG-style caching.

2. cache: 'no-store'

This disables caching completely.

Example
const res = await fetch('https://api.example.com/stats', {
  cache: 'no-store'
});

Flow

Request
   ↓
Fetch API
   ↓
Return Data

Next Request
   ↓
Fetch Again

Every request gets fresh data.

Good for:

  • User Dashboard
  • Live Statistics
  • Stock Prices
  • User Session Data
This makes the route dynamic (SSR).

3. next.revalidate

This enables ISR-style revalidation.

Example
const res = await fetch('https://api.example.com/products', {
  next: {
    revalidate: 60
  }
});

Meaning: 

Cache Data
      ↓
Keep For 60 Seconds
      ↓
Refresh On Next Request

Timeline

0 sec
   ↓
Fetch Product Data

30 sec
   ↓
Use Cache

60 sec
   ↓
Cache Expired

61 sec
   ↓
Fetch New Data

Good for:

  • Product Catalog
  • News Articles
  • Public Content
This is the most common ISR pattern in the App Router.

4. next.tags

Allows data to be grouped using tags.

Example
const res = await fetch('https://api.example.com/products', {
  next: {
    tags: ['products']
  }
});

Now this cache belongs to: 

products

tag.

Revalidating By Tag

Suppose an admin updates a product.

Server Action
import { revalidateTag } from 'next/cache';

await updateProduct();

revalidateTag('products');

Flow

Product Updated
       ↓
revalidateTag('products')
       ↓
Products Cache Cleared
       ↓
Next Request Gets Fresh Data
Instead of waiting 60 seconds, we can refresh immediately.

5. Request Deduplication

Next.js automatically deduplicates identical fetch requests.

Example
const users1 = fetch('https://api.example.com/users');
const users2 = fetch('https://api.example.com/users');
const users3 = fetch('https://api.example.com/users');

Next.js performs: 

1 Network Request

instead of: 

3 Network Requests

This improves performance automatically.

Real World Examples

Product Catalog

ISR
await fetch(url, {
  next: {
    revalidate: 60
  }
});

Products update every minute.

User Dashboard

SSR
await fetch(url, {
  cache: 'no-store'
});

Fresh data every request.

Blog Posts

Static Cache
await fetch(url, {
  cache: 'force-cache'
});

Cache forever until rebuild or revalidation.

One-Line Definition:
Next.js extends the standard fetch() API by adding caching, ISR revalidation, cache tags, and request deduplication, allowing us to control when data is cached and refreshed.
