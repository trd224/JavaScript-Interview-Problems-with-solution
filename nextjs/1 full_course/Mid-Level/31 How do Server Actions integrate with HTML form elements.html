31. How do Server Actions integrate with HTML <form> elements?

Server Actions integrate directly with HTML forms using the form's action attribute.

Simple Meaning:
Instead of submitting a form to an API endpoint, we can submit it directly to a Server Action.

Traditional HTML Form

Before Server Actions, forms were submitted to a URL.

Traditional Form
<form action="/api/users" method="POST">

  <input name="name" />

  <button>Save</button>

</form>

Flow: 

Form Submit
      ↓
/api/users
      ↓
API Route
      ↓
Database
      ↓
Response

Server Actions Form

With Server Actions, the action attribute receives a function instead of a URL.

Server Action
'use server';

export async function createUser(
  formData: FormData
) {

  console.log(
    formData.get('name')
  );

}
Form
import { createUser } from './actions';

export default function Page() {

  return (
    <form action={createUser}>

      <input
        name="name"
        placeholder="Enter Name"
      />

      <button>
        Save
      </button>

    </form>
  );

}

Flow: 

Submit Form
      ↓
Server Action
      ↓
Database
      ↓
Response

No API route is required.

How Does formData Work?

When the form is submitted, Next.js automatically collects all form fields and passes them to the Server Action as a FormData object.

Form
<input name="name" />
<input name="email" />
Server Action
'use server';

export async function createUser(
  formData: FormData
) {

  const name = formData.get('name');

  const email = formData.get('email');

}

Values are automatically available inside formData.

Real World Example

Server Action
'use server';

export async function addProduct(
  formData: FormData
) {

  await db.product.create({
    data: {
      name: formData.get('name'),
      price: Number(
        formData.get('price')
      )
    }
  });

}
Form
<form action={addProduct}>

  <input
    name="name"
    placeholder="Product Name"
  />

  <input
    name="price"
    placeholder="Price"
  />

  <button>
    Add Product
  </button>

</form>

Flow: 

Submit Product Form
         ↓
Server Action
         ↓
Insert Into Database
         ↓
Success

What Happens Internally?

Next.js automatically creates an internal request behind the scenes.

Internal Flow
Browser
    ↓
Submit Form
    ↓
Next.js Internal Request
    ↓
Server Action
    ↓
Database
    ↓
Response

We do not need:

  • fetch()
  • API Route
  • JSON.stringify()
  • req.json()

Form + Revalidation

Server Actions often refresh cached pages after data changes.

Example
'use server';

import { revalidatePath } from 'next/cache';

export async function addProduct(
  formData: FormData
) {

  await db.product.create({
    data: {
      name: formData.get('name')
    }
  });

  revalidatePath('/products');

}

Flow: 

Submit Form
      ↓
Insert Product
      ↓
revalidatePath()
      ↓
Cache Cleared
      ↓
Fresh Data

Feature Traditional Form Server Action Form

Less Boilerplate ❌ ✅

One-Line Definition:
Server Actions integrate with HTML forms by allowing a Server Action function to be assigned directly to the form's action attribute, eliminating the need for separate API routes.
