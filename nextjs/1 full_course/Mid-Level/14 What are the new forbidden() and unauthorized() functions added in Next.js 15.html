14. What are the new forbidden() and unauthorized() functions added in Next.js 15?

Next.js 15 introduced unauthorized() and forbidden(), which work similarly to notFound().

Instead of manually returning error pages, we can call unauthorized() or forbidden() and Next.js automatically stops rendering and displays a 401 or 403 page. :contentReference[oaicite:0]{index=0}

Why Were These Added?

Before Next.js 15, developers usually did something like:

Old Way
if (!session) {
  return <LoginPage />;
}

if (session.role !== 'admin') {
  return <AccessDenied />;
}

Now we can use built-in auth interrupts:

New Way
if (!session) {
  unauthorized();
}

if (session.role !== 'admin') {
  forbidden();
}

This makes authentication and authorization handling much cleaner. :contentReference[oaicite:1]{index=1}

unauthorized()

unauthorized() is used when the user is not authenticated.

401 = User is not logged in.

Example

app/dashboard/page.tsx
import { unauthorized } from 'next/navigation';

export default async function DashboardPage() {

  const session = await verifySession();

  if (!session) {
    unauthorized();
  }

  return <h1>Dashboard</h1>;

}

What Happens?

Flow
User Requests Dashboard
           │
           ▼
No Session Found
           │
           ▼
unauthorized()
           │
           ▼
401 Page Rendered

Next.js stops rendering the page and shows the unauthorized.js UI. :contentReference[oaicite:2]{index=2}

forbidden()

forbidden() is used when the user is authenticated but lacks permission.

403 = User is logged in but does not have access.

Example

app/admin/page.tsx
import { forbidden } from 'next/navigation';

export default async function AdminPage() {

  const session = await verifySession();

  if (session.role !== 'admin') {
    forbidden();
  }

  return <h1>Admin Panel</h1>;

}

What Happens?

Flow
User Requests Admin Page
           │
           ▼
User Logged In
           │
           ▼
Role = User
           │
           ▼
forbidden()
           │
           ▼
403 Page Rendered

Next.js stops rendering and displays the forbidden.js UI. :contentReference[oaicite:3]{index=3}

401 vs 403

Status Meaning Function
401 User is not logged in unauthorized()
403 User is logged in but lacks permission forbidden()

Custom unauthorized.js Page

app/unauthorized.tsx
export default function Unauthorized() {

  return (
    <div>
      <h2>Please Login</h2>
      <p>You must be logged in to access this page.</p>
    </div>
  );

}

Whenever unauthorized() is called, this page is rendered. :contentReference[oaicite:4]{index=4}

Custom forbidden.js Page

app/forbidden.tsx
export default function Forbidden() {

  return (
    <div>
      <h2>Access Denied</h2>
      <p>You do not have permission to access this page.</p>
    </div>
  );

}

Whenever forbidden() is called, this page is rendered. :contentReference[oaicite:5]{index=5}

How Are They Similar to notFound()?

Function Status Code Purpose
notFound() 404 Page does not exist
unauthorized() 401 User not authenticated
forbidden() 403 User lacks permission

All three functions immediately stop rendering and let Next.js display a special page. :contentReference[oaicite:6]{index=6}

Configuration Required

To use these APIs, Next.js currently requires the experimental authInterrupts option. :contentReference[oaicite:7]{index=7}

next.config.ts
import type { NextConfig } from 'next';

const nextConfig: NextConfig = {
  experimental: {
    authInterrupts: true
  }
};

export default nextConfig;
