27. How does next/after allow post-response work in Next.js 15?

after() allows us to run code after Next.js has finished sending the response to the browser.

Simple meaning:
Response first, background work after that.

1. Without after()

Without after(), if we run logging, analytics, or audit code before returning the UI, the user has to wait.

Without after()
export default async function Page() {

  await saveAnalytics();

  return <h1>Products</h1>;

}

Flow: 

Request
   ↓
saveAnalytics()
   ↓
Render Page
   ↓
Send Response

Here, the page response is delayed because saveAnalytics() runs before the response.

2. With after()

With after(), we can schedule non-critical work after the response is finished.

With after()
import { after } from 'next/server';

export default function Page() {

  after(async () => {
    await saveAnalytics();
  });

  return <h1>Products</h1>;

}

Flow: 

Request
   ↓
Render Page
   ↓
Send Response
   ↓
saveAnalytics()
Now the user gets the page first, and analytics work runs after the response.

Real World Example

Suppose our product page needs to save a page-view record.

Product Page Visit Tracking
import { after } from 'next/server';

export default async function ProductPage() {

  const products = await getProducts();

  after(async () => {
    await db.pageViews.create({
      page: '/products',
      viewedAt: new Date()
    });
  });

  return <ProductList products={products} />;

}

Here:

Work When It Happens
Get products Before response, because UI needs products
Render product page Before response
Save page view After response

Common Use Cases

  • Analytics tracking
  • Logging
  • Audit records
  • Sending webhooks
  • Updating counters
  • Background notifications

Important Point

after() does not make a page dynamic by itself.

Static Page With after()
import { after } from 'next/server';

export default function Page() {

  after(() => {
    console.log('Logged after response');
  });

  return <h1>Static Page</h1>;

}

This page can still be static. Using after() only schedules post-response work.

Where Can We Use after()?

Place Can Use after()?
Server Components ✅ Yes
Server Actions / Server Functions ✅ Yes
Route Handlers ✅ Yes
Client Components ❌ No

Easy Memory Trick

Remember
Normal Code
     ↓
Runs before response


after()
     ↓

One-Line Definition:
after() is a Next.js API that allows us to execute non-critical work after the response is sent, without delaying page rendering.
