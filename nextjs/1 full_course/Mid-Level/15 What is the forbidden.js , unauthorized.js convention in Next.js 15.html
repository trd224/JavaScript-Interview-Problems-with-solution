15. What is the forbidden.js / unauthorized.js Convention in Next.js 15?

Next.js 15 introduced two new special files: 

forbidden.tsx

unauthorized.tsx

These files are similar to: 

not-found.tsx

error.tsx

loading.tsx

They provide custom UI for authentication and authorization failures.

When unauthorized() or forbidden() is called, Next.js automatically renders unauthorized.tsx or forbidden.tsx.

Why Were These Files Added?

Before Next.js 15, developers typically wrote:

Old Approach
if (!session) {
  return <LoginPage />;
}

if (session.role !== 'admin') {
  return <AccessDenied />;
}

Now Next.js provides dedicated authentication interruption APIs.

New Approach
if (!session) {
  unauthorized();
}

if (session.role !== 'admin') {
  forbidden();
}

Next.js automatically renders the correct page.

What is unauthorized.tsx?

This file is shown when: 

unauthorized()

is called.

401 Unauthorized = User is not logged in.

Folder Structure

App Router
app/
├── unauthorized.tsx
├── dashboard/
│   └── page.tsx

dashboard/page.tsx

Protected Page
import { unauthorized } from 'next/navigation';

export default async function DashboardPage() {

  const session = await verifySession();

  if (!session) {
    unauthorized();
  }

  return <h1>Dashboard</h1>;

}

unauthorized.tsx

Custom 401 Page
export default function Unauthorized() {

  return (
    <div>
      <h1>401</h1>
      <p>Please login to continue.</p>
    </div>
  );

}

Flow

What Happens?
User Opens /dashboard
           │
           ▼
verifySession()
           │
           ▼
No Session Found
           │
           ▼
unauthorized()
           │
           ▼
unauthorized.tsx Rendered

What is forbidden.tsx?

This file is shown when: 

forbidden()

is called.

403 Forbidden = User is logged in but does not have permission.

Folder Structure

App Router
app/
├── forbidden.tsx
├── admin/
│   └── page.tsx

admin/page.tsx

Admin Page
import { forbidden } from 'next/navigation';

export default async function AdminPage() {

  const session = await verifySession();

  if (session.role !== 'admin') {
    forbidden();
  }

  return <h1>Admin Panel</h1>;

}

forbidden.tsx

Custom 403 Page
export default function Forbidden() {

  return (
    <div>
      <h1>403</h1>
      <p>Access denied.</p>
    </div>
  );

}

Flow

What Happens?
User Opens /admin
           │
           ▼
verifySession()
           │
           ▼
User Logged In
           │
           ▼
Role = user
           │
           ▼
forbidden()
           │
           ▼
forbidden.tsx Rendered

Difference Between unauthorized.tsx and forbidden.tsx

File Status Meaning
unauthorized.tsx 401 User is not logged in
forbidden.tsx 403 User lacks permission

How Are They Similar to not-found.tsx?

Next.js now has special UI files for common situations:

Function File Rendered Status
notFound() not-found.tsx 404
unauthorized() unauthorized.tsx 401
forbidden() forbidden.tsx 403

Do These Files Need 'use client'?

No.

Like most App Router pages, they are Server Components by default.

Server Component
export default function Unauthorized() {
  return <h1>Please Login</h1>;
}

Add 'use client' only if browser features are required.

Important Configuration

To use these features, enable auth interrupts.

next.config.ts
import type { NextConfig } from 'next';

const nextConfig: NextConfig = {
  experimental: {
    authInterrupts: true
  }
};

export default nextConfig;
