10. What are Route Groups (folder) and when do you use them?

Route Groups are special folders in the Next.js App Router that allow us to organize routes without affecting the URL structure.

A Route Group is created by wrapping a folder name in parentheses ().

Basic Syntax

Route Group Folder
(marketing)
(auth)
(dashboard)

The folder name is used only for organization and layout purposes.

Why Do We Need Route Groups?

Sometimes we want:

  • Different layouts for different sections.
  • Better folder organization.
  • Separate application areas.
  • Shared layouts for specific routes.
  • Cleaner project structure.

Without Route Groups, all routes would need to share the same folder hierarchy.

Basic Example

Folder Structure
app/
│
├── (marketing)/
│   ├── page.tsx
│   └── about/
│       └── page.tsx
│
├── (dashboard)/
│   ├── users/
│   │   └── page.tsx
│   │
│   └── reports/
│       └── page.tsx

Generated URLs

URLs
/                  ← page.tsx
/about             ← about/page.tsx

/users             ← users/page.tsx
/reports           ← reports/page.tsx

Notice that: 

(marketing)
(dashboard)

do not appear in the URL.

Route Group names never become part of the route URL.

Without Route Groups

Folder Structure
app/
│
├── marketing/
│   └── about/
│       └── page.tsx

Generated URL: 

/marketing/about

With Route Groups

Folder Structure
app/
│
├── (marketing)/
│   └── about/
│       └── page.tsx

Generated URL: 

/about

The Route Group disappears from the URL.

Most Common Use Case: Different Layouts

Suppose we want:

  • Marketing pages → Marketing Layout
  • Dashboard pages → Dashboard Layout
Folder Structure
app/
│
├── (marketing)/
│   ├── layout.tsx
│   ├── page.tsx
│   └── about/
│       └── page.tsx
│
├── (dashboard)/
│   ├── layout.tsx
│   ├── users/
│   │   └── page.tsx
│   │
│   └── reports/
│       └── page.tsx

Marketing Layout

(marketing)/layout.tsx
export default function Layout({
  children
}) {

  return (
    <>
      <MarketingHeader />

      {children}

      <MarketingFooter />
    </>
  );

}

Dashboard Layout

(dashboard)/layout.tsx
export default function Layout({
  children
}) {

  return (
    <>
      <Sidebar />

      {children}
    </>
  );

}

Now different sections of the application have completely different layouts.

How Rendering Works

User visits: 

/about

Next.js builds: 

Root Layout
      │
      ▼
Marketing Layout
      │
      ▼
About Page

User visits: 

/users

Next.js builds: 

Root Layout
      │
      ▼
Dashboard Layout
      │
      ▼
Users Page

Different layouts are used even though the Route Group name is not in the URL.

Route Groups and Layout Persistence

Remember: 

Root Layout
   Always Persists

Nested Layout
   Persists

Template
   Recreated

Page
   Recreated

A Route Group simply decides which layout tree Next.js should use.

Route Groups do not change layout behavior. They only help organize and separate route trees.

Real World Example

Application Structure
app/
│
├── (public)/
│   ├── layout.tsx
│   ├── page.tsx
│   ├── about/
│   └── contact/
│
├── (auth)/
│   ├── layout.tsx
│   ├── login/
│   └── register/
│
├── (dashboard)/
│   ├── layout.tsx
│   ├── users/
│   ├── reports/
│   └── settings/

This keeps the application organized while keeping URLs clean.

Route Groups vs Normal Folders

Feature Normal Folder Route Group
Appears In URL ✅ ❌
Can Have Layout ✅ ✅
Organizes Routes ✅ ✅
Used For Layout Separation ❌ ✅

Easy Memory Trick

Remember
users/
   → URL becomes /users

(users)/
   → URL does NOT change

When Should We Use Route Groups?

  • Different layouts for different sections.
  • Marketing pages.
  • Authentication pages.
  • Dashboard pages.
  • Large application organization.
  • Separating route trees without changing URLs.

When Should We Avoid Route Groups?

  • Simple applications.
  • When normal folders are sufficient.
  • When URL structure should reflect folder structure.

One-Line Definition:
Route Groups are folders wrapped in parentheses that help organize routes and create separate layout trees without affecting the URL structure.
