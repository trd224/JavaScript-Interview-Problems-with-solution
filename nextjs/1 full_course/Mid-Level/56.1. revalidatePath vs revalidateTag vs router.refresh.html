56.1. revalidatePath() vs revalidateTag() vs router.refresh()

All three are related to getting fresh data, but they work at different levels.

API Runs Where? Purpose
router.refresh() Client Component Re-fetch current route Server Components
revalidatePath() Server Action / Route Handler Invalidate cache for a specific route
revalidateTag() Server Action / Route Handler Invalidate tagged data across routes

1. router.refresh()

Client Component
'use client';

import { useRouter } from 'next/navigation';

export default function AddToCartButton() {
  const router = useRouter();

  async function handleClick() {
    await updateCart();
    router.refresh();
  }

  return <button onClick={handleClick}>Add To Cart</button>;
}
What it does:
Requests the current route again and re-runs Server Components.
Flow
Client Component
        ↓
router.refresh()
        ↓
Current Route Requested Again
        ↓
Server Components Re-run
        ↓
Fresh UI
Important:
router.refresh() does not invalidate cache. If data is still cached, the refreshed route may receive the same cached data.

2. revalidatePath()

Server Action
'use server';

import { revalidatePath } from 'next/cache';

export async function updateCart(productId: string) {
  await db.cart.add(productId);

  revalidatePath('/cart');
}
What it does:
Marks a specific route cache as stale so the next render gets fresh data.
Flow
Database Updated
        ↓
revalidatePath('/cart')
        ↓
Cart Route Cache Invalidated
        ↓
Fresh Data Generated

3. revalidateTag()

Tag Data
await fetch('/api/products', {
  next: {
    tags: ['products']
  }
});
Invalidate Tag
'use server';

import { revalidateTag } from 'next/cache';

export async function addProduct() {
  await db.products.create(...);

  revalidateTag('products');
}
What it does:
Invalidates all cached data that uses the same tag.
Flow
Tagged Data
      ↓
revalidateTag('products')
      ↓
All Matching Cached Data Invalidated
      ↓
Fresh Data Generated

Which One Should You Use?

Situation Best Choice
Refresh current page UI router.refresh()
Invalidate one route revalidatePath()
Invalidate shared data across pages revalidateTag()
Server Action updates database revalidatePath() or revalidateTag()
Easy Memory Trick:
router.refresh() = Refresh Current Route
revalidatePath() = Refresh One Route's Cache
revalidateTag() = Refresh Shared Data Cache
Important:
router.refresh() does not invalidate cache.

If the data is still cached, the refreshed route may receive the same cached data. 
router.refresh()
       ↓
Request Current Route Again
       ↓
Cached Data Still Exists
       ↓
Same Data Returned

Think of it like this: 
router.refresh()
       =
Reload Current Route

revalidatePath()
       =
Clear Route Cache

revalidateTag()
       =
Clear Tagged Data Cache

If you refresh without invalidating cache: 
Refresh
   ↓
Cached Data
   ↓
Same Result

If you invalidate cache: 
Database Updated
      ↓
revalidatePath('/cart')
      ↓
Cache Cleared
      ↓
Next Render Gets Fresh Data

For Server Actions, the recommended pattern is usually: 
'use server';

import { revalidatePath } from 'next/cache';

export async function updateCart(productId: string) {
  await db.cart.add(productId);

  revalidatePath('/cart');
}
In many cases, revalidatePath() or revalidateTag() is enough and you do not need router.refresh(). Use router.refresh() when you specifically need to refresh the current route from a Client Component.
