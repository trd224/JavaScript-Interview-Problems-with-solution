33. What is useFormStatus and what does it provide during a Server Action submission?

useFormStatus() is a React hook that provides information about the status of a form submission while a Server Action is running.

Simple Meaning:
useFormStatus() tells us whether a form is currently submitting so we can show loading indicators, disable buttons, and prevent duplicate submissions.

What Problem Does useFormStatus Solve?

Suppose a Server Action takes 3 seconds to complete.

Without useFormStatus
User Clicks Submit
         ↓
Nothing Changes
         ↓
User Clicks Again
         ↓
Duplicate Requests

The user cannot tell whether the form is still processing.

With useFormStatus(): 

User Clicks Submit
         ↓
pending = true
         ↓
Button Disabled
         ↓
Loading Text Shown
         ↓
Request Completes
         ↓
pending = false
         ↓
Button Enabled

Import

Import Hook
import { useFormStatus } from 'react-dom';

Unlike useActionState(), this hook is imported from react-dom.

Basic Example

1. Server Action

actions.ts
'use server';

export async function createUser(formData: FormData) {
  await new Promise(resolve => setTimeout(resolve, 3000));

  console.log(formData.get('name'));
}

2. Submit Button

SubmitButton.tsx
'use client';

import { useFormStatus } from 'react-dom';

export default function SubmitButton() {
  const { pending } = useFormStatus();

  return (
    <button disabled={pending}>
      {pending ? 'Saving...' : 'Save'}
    </button>
  );
}

3. Form

UserForm.tsx
import { createUser } from './actions';
import SubmitButton from './SubmitButton';

export default function UserForm() {
  return (
    <form action={createUser}>
      <input name="name" />
      <SubmitButton />
    </form>
  );
}

How It Works

Flow
Submit Form
      ↓
pending = true
      ↓
Button Disabled
      ↓
Server Action Runs
      ↓
pending = false
      ↓
Button Enabled

What Does useFormStatus Return?

The hook returns an object describing the current form submission.

Example
const status = useFormStatus();
Property Description
pending Whether the form is currently submitting
data The submitted FormData
method The HTTP method used by the form
action The Server Action being executed

Using FormData During Submission

Example
'use client';

import { useFormStatus } from 'react-dom';

export default function Status() {
  const { pending, data } = useFormStatus();

  return (
    <>
      {pending && (
        <p>Saving {data?.get('name')}...</p>
      )}
    
  );
}

While the form is submitting, we can access the values currently being submitted.

Important Rule

Important:
useFormStatus() only works inside a component rendered somewhere inside the form.

❌ Wrong

Will Not Work
'use client';

import { useFormStatus } from 'react-dom';

export default function Page() {
  const { pending } = useFormStatus();

  return (
    <form>
      <button disabled={pending}>
        Save
      </button>
    </form>
  );
}

The hook is being called in the same component that creates the form, so it cannot subscribe to that form's status.

✅ Correct

Works
'use client';

import { useFormStatus } from 'react-dom';

function SubmitButton() {
  const { pending } = useFormStatus();

  return (
    <button disabled={pending}>
      {pending ? 'Saving...' : 'Save'}
    </button>
  );
}
}

export default function UserForm() {
  return (
    <form action={createUser}>
      <input name="name" />
      <SubmitButton />
    </form>
  );
}

Here, SubmitButton is rendered inside the form, so it can access the form status.

useActionState vs useFormStatus

Hook Purpose
useActionState() Stores the value returned from the Server Action
useFormStatus() Provides loading/submission status
Relationship
useActionState()
         ↓
Success / Error Message


useFormStatus()
         ↓
Loading State

Imagine an e-commerce checkout form.

Flow

      ↓

      ↓

      ↓

      ↓

One-Line Definition:
useFormStatus() is a React hook that provides information about an active form submission, including whether a Server Action is currently pending.
