25.1. React.cache() vs unstable_cache() vs 'use cache'

Next.js provides multiple caching mechanisms. Each solves a different problem.

Quick Rule
React.cache() → Request-level memoization
unstable_cache() → Persistent data cache
'use cache' → Modern replacement for unstable_cache()

1. React.cache()

React.cache() memoizes function results during a single server request.

It prevents duplicate database queries, session checks, and custom async functions from running multiple times during the same render.

Without React.cache()

lib/user.ts
export async function getUser() {
  console.log('DB Query');
  return db.user.findFirst();
}
Usage
await getUser();
await getUser();
await getUser();
Result
DB Query
DB Query
DB Query

With React.cache()

lib/user.ts
import { cache } from 'react';

export const getUser = cache(
  async () => {
    console.log('DB Query');
    return db.user.findFirst();
  }
);
Usage
await getUser();
await getUser();
await getUser();
Result
DB Query

Request Flow

Request 1
   ↓
getUser()
   ↓
DB Query
   ↓
Memory Cache

getUser()
   ↓
Cached Result

getUser()
   ↓
Cached Result

New Request

Request 2
   ↓
DB Query Again

React.cache() cache is cleared after every request.

Example With Arguments

lib/product.ts
import { cache } from 'react';

export const getProduct = cache(
  async (id) => {
    return db.product.findUnique({
      where:{id}
    });
  }
);
Usage
await getProduct('1');
await getProduct('1');
await getProduct('2');
Result
getProduct('1') → Query
getProduct('1') → Cached
getProduct('2') → Query

Best Use Cases

  • Database Queries
  • Session Checks
  • Prisma Queries
  • Mongo Queries
  • Authentication Logic
Interview Answer:
React.cache() memoizes custom function results during a single server request, preventing duplicate executions and reusing the same result within that request.

2. unstable_cache()

unstable_cache() is a Next.js Data Cache API that stores function results across requests.

Unlike React.cache(), the cache survives between requests and can be shared across users.

Why Do We Need unstable_cache()?

Database queries can be expensive. If the same data is requested repeatedly, we can cache it and avoid hitting the database every time.

Without unstable_cache()
async function getProducts() {
  return db.product.findMany();
}
Request Flow
Request 1
   ↓
Database Query

Request 2
   ↓
Database Query

Request 3
   ↓
Database Query

Using unstable_cache()

lib/products.ts
import { unstable_cache } from 'next/cache';

export const getProducts = unstable_cache(
    async () => {
      return db.product.findMany();
    },
    ['products']
  );
Usage
const products = await getProducts();
Request Flow
Request 1
   ↓
Database Query
   ↓
Store In Cache

Request 2
   ↓
Cached Result

Request 3
   ↓
Cached Result

Using Revalidation

Cache can automatically expire after a specified time.

Revalidate Every Hour
import { unstable_cache } from 'next/cache';

export const getProducts = unstable_cache(
    async () => {
      return db.product.findMany();
    },
    ['products'],
    {
      revalidate: 3600
    }
  );
Flow
Hour 1
   ↓
Database Query

Next Requests
   ↓
Cached Result

After 3600 Seconds
   ↓
Cache Expires

Next Request
   ↓
Database Query Again

Using Tags

Tags allow manual cache invalidation.

Tagged Cache
import { unstable_cache } from 'next/cache';

export const getProducts = unstable_cache(
    async () => {
      return db.product.findMany();
    },
    ['products'],
    {
      tags:['products']
    }
  );
Revalidate Tag
import { revalidateTag } from 'next/cache';

revalidateTag( 'products' );
Flow
Request 1
   ↓
Database Query
   ↓
Cache Stored

Request 2
   ↓
Cached Result

revalidateTag('products')
   ↓
Cache Cleared

Request 3
   ↓
Database Query Again

Example With Parameters

Product By ID
import { unstable_cache } from 'next/cache';

export const getProduct = unstable_cache(
    async (id) => {
      return db.product.findUnique({
        where:{id}
      });
    },
    ['product']
  );

Be careful when using parameters. Cache keys must uniquely identify data.

When Should We Use unstable_cache()?

  • Product Lists
  • Category Lists
  • Blog Posts
  • CMS Data
  • Frequently Read Database Data

When Not To Use It?

  • User Sessions
  • Authentication Data
  • User-Specific Information
  • Request-Specific Data
Interview Answer:
unstable_cache() caches server function results in Next.js Data Cache, allowing data to be reused across requests with support for revalidation and tag-based invalidation.

3. 'use cache'

'use cache' is the modern Next.js caching directive that replaces many unstable_cache() use cases.

Think of 'use cache' as the new, cleaner, and more powerful way to create persistent Data Cache entries in Next.js.

Basic Example

lib/products.ts
async function getProducts() {
  'use cache';

  return db.product.findMany();
}
Usage
const products = await getProducts();
Request Flow
Request 1
   ↓
Database Query
   ↓
Store In Cache

Request 2
   ↓
Cached Result

Request 3
   ↓
Cached Result

Adding Cache Lifetime

cacheLife() controls how long the cached data remains valid.

Cache For Minutes
import { cacheLife } from 'next/cache';

async function getProducts() {
  'use cache';

  cacheLife('minutes');

  return db.product.findMany();
}
Cache For Hours
import { cacheLife } from 'next/cache';

async function getProducts() {
  'use cache';

  cacheLife('hours');

  return db.product.findMany();
}

All options

import { cacheLife } from 'next/cache';

cacheLife('seconds');
cacheLife('minutes');
cacheLife('hours');
cacheLife('days');
cacheLife('weeks');
cacheLife('max');
Profile Revalidate Expire

Adding Cache Tags

Tags allow selective cache invalidation.

Tagged Cache
import { cacheTag } from 'next/cache';

async function getProducts() {
  'use cache';

  cacheTag('products');

  return db.product.findMany();
}
Invalidate Tag
import { revalidateTag } from 'next/cache';

revalidateTag(
  'products'
);
Flow
Request 1
   ↓
Database Query
   ↓
Cache Stored

Request 2
   ↓
Cached Result

revalidateTag('products')
   ↓
Cache Cleared

Request 3
   ↓
Database Query Again

Using cacheLife + cacheTag Together

Recommended Pattern
import { cacheLife, cacheTag } from 'next/cache';

async function getProducts() {
  'use cache';

  cacheLife('hours');
  cacheTag('products');

  return db.product.findMany();
}

Real World Example

Blog Posts
import { cacheLife, cacheTag } from 'next/cache';

async function getPosts() {
  'use cache';

  cacheLife('hours');
  cacheTag('posts');

  return db.post.findMany();
}
After Creating New Post
revalidateTag('posts');

The next request automatically gets fresh data.

Easy Rule:

React.cache()
= Prevent duplicate function calls during one request.

unstable_cache()
= Old Next.js persistent Data Cache.

'use cache'
= Modern Next.js persistent Data Cache with cacheLife() and cacheTag().
Final Interview Answer:
React.cache() provides request-level memoization, unstable_cache() provides persistent Data Cache across requests, and 'use cache' is the modern Next.js caching API that offers persistent caching with cacheLife() and cacheTag() support.
