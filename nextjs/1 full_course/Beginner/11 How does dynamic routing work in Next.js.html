11. How does Dynamic Routing work in Next.js?

Dynamic Routing in Next.js allows us to create routes whose URL segments are determined at runtime instead of being fixed at development time.

Dynamic routes are created using square brackets []. The value inside the URL is captured as a route parameter.

Why Do We Need Dynamic Routing?

Suppose our application displays products.

Creating a separate file for every product would be impossible:

Bad Approach
products-1.js
products-2.js
products-3.js
products-100.js
products-1000.js

Instead, we create a single dynamic route that handles all product IDs.

Pages Router Dynamic Route

Folder Structure
pages/
└── products/
    └── [id].js

The [id] segment is dynamic.

This single file can handle:

URLs
/products/1
/products/5
/products/100
/products/999

Accessing Route Parameters (Pages Router)

pages/products/[id].js
import { useRouter } from 'next/router';

export default function Product() {

  const router = useRouter();

  return (
    <h1>
      Product ID: {router.query.id}
    </h1>
  );

}

If the URL is:

URL
/products/100

Output:

Output
Product ID: 100

App Router Dynamic Route

In modern Next.js applications, dynamic routes are created inside the app directory.

Folder Structure
app/
└── products/
    └── [id]/
        └── page.js

Accessing Route Parameters (App Router)

app/products/[id]/page.js
export default function Product({ params }) {

  return (
    <h1>
      Product ID: {params.id}
    </h1>
  );

}

For the URL:

URL
/products/100

Output:

Output
Product ID: 100

Multiple Dynamic Parameters

We can create multiple dynamic segments.

Folder Structure
app/
└── users/
    └── [userId]/
        └── orders/
            └── [orderId]/
                └── page.js

Supported URL:

URL
/users/10/orders/500
Access Parameters
export default function Order({ params }) {

  return (
    <>
      User: {params.userId}
      Order: {params.orderId}
    </>
  );

}

Catch-All Routes

Catch-all routes match multiple URL segments.

Folder Structure
app/
└── docs/
    └── [...slug]/
        └── page.js

Can match:

URLs
/docs/react
/docs/react/hooks
/docs/react/hooks/useState
Access Values
export default function Docs({ params }) {

  return (
    <pre>
      {JSON.stringify(params.slug)}
    </pre>
  );

}

For:

URL
/docs/react/hooks

Output:

Output
["react", "hooks"]

Optional Catch-All Routes

Optional catch-all routes can match zero or more segments.

Folder Structure
app/
└── docs/
    └── [[...slug]]/
        └── page.js

Can match:

URLs
/docs
/docs/react
/docs/react/hooks

Real-World Example

Suppose our e-commerce application has thousands of products.

Dynamic Product Route
app/
└── products/
    └── [id]/
        └── page.js

URLs:

Examples
/products/1
/products/25
/products/1000
/products/5000

A single page handles all product IDs.

Dynamic Routing vs Static Routing

Static Route Dynamic Route
/about /products/[id]
Fixed URL Variable URL
One page per route One page handles many routes
No route parameters Supports route parameters

Pages Router vs App Router

Pages Router App Router
pages/products/[id].js app/products/[id]/page.js
useRouter() params
next/router next/navigation
One-Line Definition:
Dynamic Routing in Next.js allows us to create routes with variable URL segments using square brackets ([]), enabling a single page to handle multiple URLs by accessing route parameters.
