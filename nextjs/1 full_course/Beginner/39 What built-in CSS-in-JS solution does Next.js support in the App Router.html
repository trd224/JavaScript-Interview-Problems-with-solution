39. What built-in CSS-in-JS solution does Next.js support in the App Router?

Next.js supports styled-jsx as its built-in CSS-in-JS solution.

styled-jsx allows us to write component-scoped CSS directly inside our JSX without needing separate CSS files.

What is CSS-in-JS?

CSS-in-JS means writing CSS directly inside JavaScript or React components.

Instead of creating:

Separate CSS File
Button.js
Button.css

We can keep styles inside the component itself.

Built-in Solution: styled-jsx

Example
export default function Home() {

  return (
    <div>
      <h1>Hello Next.js</h1>

      <style jsx>{`
        h1 {
          color: blue;
        }
      `}</style>
    </div>
  );

}

The CSS is scoped only to this component.

Example of Style Isolation

Component A
export default function A() {

  return (
    <>
      <h1>A</h1>

      <style jsx>{`
        h1 {
          color: red;
        }
      `}</style>
    </>
  );

}
Component B
export default function B() {

  return (
    <>
      <h1>B</h1>

      <style jsx>{`
        h1 {
          color: blue;
        }
      `}</style>
    </>
  );

}

Both components can style h1 differently without conflicts.

Global Styles with styled-jsx

We can also create global styles.

Global Style
<style jsx global>{`
  body {
    margin: 0;
    background: #111827;
  }
`}</style>

Using global removes style scoping.

App Router Recommendation

Although styled-jsx is supported, the App Router documentation generally recommends:

  • CSS Modules
  • Tailwind CSS
  • Global CSS

for most styling needs.

CSS Modules Example

Button.module.css
.button {
  background: blue;
  color: white;
}
Button.js
import styles from './Button.module.css';

export default function Button() {

  return (
    <button className={styles.button}>
      Save
    </button>
  );

}

Does Next.js Include styled-components?

No.

Next.js does not include styled-components by default.

We must install it separately.

Install
npm install styled-components

Does Next.js Include Emotion?

No.

Emotion must also be installed separately.

Install
npm install @emotion/react @emotion/styled

Important App Router Note

A common misconception is that styled-jsx is the recommended styling solution for the App Router. In reality, while styled-jsx is still supported, Next.js generally recommends CSS Modules, Tailwind CSS, and Global CSS for most App Router applications.

When Should We Use styled-jsx?

  • Small component-specific styles.
  • Quick prototypes.
  • Simple isolated styling.
  • When we want styles inside the component file.

When Should We Prefer CSS Modules?

  • Large applications.
  • Reusable components.
  • Better maintainability.
  • Team projects.

Interview Tip

The built-in CSS-in-JS solution in Next.js is styled-jsx, but in modern App Router projects, CSS Modules and Tailwind CSS are more commonly used.
One-Line Definition:
Next.js supports styled-jsx as its built-in CSS-in-JS solution, allowing us to write component-scoped CSS directly inside JSX without class name conflicts.
