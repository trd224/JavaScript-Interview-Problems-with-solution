25. What is getServerSideProps and How Does It Differ from getStaticProps?

getServerSideProps() is a special Next.js function used in the Pages Router to fetch data on every request and generate fresh HTML for each user request.

getServerSideProps() runs on every request, while getStaticProps() runs during the build process.

Why Do We Need getServerSideProps()?

Some pages require fresh data every time users visit them.

Examples:
  • Stock Prices
  • Weather Information
  • User Dashboards
  • Live Reports
  • Real-Time Analytics

In these cases, build-time data may become outdated.

Basic Example

pages/products.js
export async function getServerSideProps() {

  const res = await fetch(
    'https://api.example.com/products'
  );

  const products = await res.json();

  return {
    props: { products }
  };

}

export default function Products({ products }) {

  return (
    <h1>
      Total Products: {products.length}
    </h1>
  );

}

Accessing Request Information

One major advantage of getServerSideProps() is access to request-specific data.

Context Object
export async function getServerSideProps(context) {

  console.log(context.req);
  console.log(context.res);
  console.log(context.query);

}

This allows us to access:

  • Cookies
  • Headers
  • Query Parameters
  • User Information
  • Authentication Tokens

Example: Authentication

Protected Page
export async function getServerSideProps(context) {

  const token = context.req.cookies.token;

  if (!token) {

    return {
      redirect: {
        destination: '/login',
        permanent: false
      }
    };

  }

  return {
    props: {}
  };

}

This is impossible with getStaticProps() because build-time code has no access to user requests.

When Should We Use getServerSideProps()?

  • User-Specific Pages
  • Authentication
  • Real-Time Data
  • Stock Prices
  • Weather Information
  • Live Dashboards

When Should We Avoid It?

  • Blogs
  • Documentation Sites
  • Marketing Pages
  • Landing Pages
  • Static Content

Important Note

A common misconception is that getServerSideProps() runs once when the application starts. In reality, it executes for every incoming request and generates fresh HTML each time.

App Router Equivalent

The App Router does not use getServerSideProps().

Instead, SSR is achieved by fetching data inside a Server Component using:

App Router SSR
async function getProducts() {

  const res = await fetch(
    'https://api.example.com/products',
    { cache: 'no-store' }
  );

  return res.json();

}

export default async function Products() {

  const products = await getProducts();

  return (
    <h1>
      {products.length}
    </h1>
  );

}

Using cache: 'no-store' ensures fresh data on every request.

One-Line Definition:
getServerSideProps() is a Pages Router function that runs on every request, fetches fresh data, and generates HTML on the server before sending it to the browser.
