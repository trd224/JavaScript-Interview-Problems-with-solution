33. How does the next/image component improve performance over a plain <img>?

The next/image component is a built-in Next.js component that automatically optimizes images for better performance, faster loading, and improved Core Web Vitals.

Unlike a normal <img> tag, next/image automatically resizes, compresses, lazy loads, and serves modern image formats when possible.

Plain img Example

Regular HTML Image
<img
  src="/product.jpg"
  alt="Product"
/>

The browser downloads exactly the image file that exists on the server.

No optimization happens automatically.

next/image Example

Next.js Image Component
import Image from 'next/image';

export default function Product() {

  return (
    <Image
      src="/product.jpg"
      alt="Product"
      width={500}
      height={300}
    />
  );

}

Next.js automatically optimizes the image before sending it to the browser.

Preventing Layout Shift

One of the biggest benefits of next/image is preventing layout shifts.

Without Width and Height

Plain img
<img
  src="/banner.jpg"
  alt="Banner"
/>

The browser does not know the image size initially.

When the image loads: 

Content Moves Down
Layout Shifts

With next/image

Image Component
<Image
  src="/banner.jpg"
  alt="Banner"
  width={1200}
  height={500}
/>

The browser reserves space before loading the image.

No layout shift occurs.

Priority Loading

Important images can be loaded immediately.

Hero Image
<Image
  src="/hero.jpg"
  alt="Hero"
  width={1200}
  height={600}
  priority
/>

This is useful for:

  • Hero Images
  • Above-the-fold Content
  • Landing Pages

Why Do We Need priority If Above-the-Fold Images Already Load?

A common misconception is that priority is required to load above-the-fold images.

Even without priority, an image that is visible in the viewport is not lazy loaded and will load automatically.

Without priority
HTML Received
      │
      ▼
Browser Parses HTML
      │
      ▼
Finds Image
      │
      ▼
Starts Downloading Image

The browser discovers the image while parsing the page and then starts downloading it.

With priority
HTML Received
      │
      ▼
Preload Image
(rel="preload")
      │
      ▼
Start Downloading Image ASAP
      │
      ▼
Render Page

When priority is used, Next.js adds a preload hint so the browser can begin downloading the image earlier.

Internally
<link
  rel="preload"
  as="image"
  href="/hero.jpg"
/>

This can improve Largest Contentful Paint (LCP) when the image is a major part of the page.

Behavior Above-the-Fold Image Below-the-Fold Image
next/image Loads Automatically Lazy Loaded
next/image + priority Preloaded and Higher Priority Usually Not Needed
Important:
priority does not make an above-the-fold image load instead of lazy loading. The image would already load automatically. It simply tells the browser that the image is critical and should be fetched as early as possible.

When Should We Use next/image?

  • Product Images
  • Blog Images
  • Hero Banners
  • Gallery Images
  • Marketing Websites

When Might We Use img?

  • Very small icons.
  • External content where optimization is unnecessary.
  • Special edge cases requiring raw image behavior.

Interview Tip

A common misconception is that next/image is just a React wrapper around img. In reality, it includes automatic resizing, compression, lazy loading, responsive image generation, caching, and modern format conversion.
One-Line Definition:
The next/image component improves performance by automatically optimizing images through resizing, compression, lazy loading, responsive image generation, caching, and layout shift prevention, whereas a plain <img> tag does none of these automatically.
