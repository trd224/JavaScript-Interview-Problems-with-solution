52. What is the metadata export in the App Router?

The metadata export is a special App Router feature that allows us to define page metadata such as title, description, Open Graph tags, Twitter cards, icons, and other SEO-related information without using the Head component.

The metadata export automatically generates HTML head tags and is the recommended way to manage SEO metadata in the App Router.

Static Metadata Example

Use metadata when metadata is known in advance and does not depend on fetched data.

app/about/page.js
export const metadata = {
  title: 'About Us',
  description: 'Learn more about our company.'
};

export default function About() {
  return <h1>About Page</h1>;
}

Generated HTML

HTML Output
<head>
  <title>About Us</title>
  <meta name="description" content="Learn more about our company." />
</head>

Layout Metadata Example

Metadata can also be defined inside layout.js. This metadata is inherited by all child pages unless a page overrides it.

app/layout.js
export const metadata = {
  title: 'My Website',
  description: 'Welcome to my website'
};

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );
}

Open Graph Example

Open Graph Metadata
export const metadata = {
  title: 'About Us',
  description: 'Learn more about our company.',
  openGraph: {
    title: 'About Us',
    description: 'Learn more about our company.',
    type: 'website'
  }
};

This automatically generates Open Graph tags used by Facebook, LinkedIn, WhatsApp, and other social media platforms.

Dynamic Metadata Example

When metadata depends on API data, database data, or CMS content, we use generateMetadata().

app/products/[id]/page.js
async function getProduct(id) {
  const res = await fetch(`https://dummyjson.com/products/${id}`);
  return res.json();
}

export async function generateMetadata({ params }) {
  const { id } = await params;
  const product = await getProduct(id);

  return {
    title: product.title,
    description: product.description
  };
}

export default async function Page({ params }) {
  const { id } = await params;
  const product = await getProduct(id);

  return (
    <div>
      <h1>{product.title}</h1>
      <p>{product.description}</p>
      <p>Price: ${product.price}</p>
    </div>
  );
}

Do We Really Fetch Data Twice?

At first glance, it looks like getProduct() is called twice:

  • Once inside generateMetadata().
  • Once inside the page component.
Example
export async function generateMetadata({ params }) {
  const { id } = await params;

  const product = await getProduct(id);

  return {
    title: product.title,
    description: product.description
  };
}

export default async function Page({ params }) {
  const { id } = await params;

  const product = await getProduct(id);

  return (
    <div>
      <h1>{product.title}</h1>
    </div>
  );
}

This appears to call getProduct() twice, but Next.js automatically deduplicates identical fetch requests.

Although the code calls getProduct() twice, Next.js usually performs only one network request when both fetch calls are identical.
Summary
Code Calls    = 2

Network Calls = 1

When Will It Fetch Twice?

If the requests are different, Next.js cannot deduplicate them.

Different Fetch Options
fetch(`/products/${id}`);

fetch(`/products/${id}`, {
  cache: 'no-store'
});
Different URLs
fetch(`/products/${id}`);

fetch(`/products/${id}?details=true`);
Deduplication works only when the requests are identical.

Important Note

Although getProduct() appears to be called twice, once by generateMetadata() and once by Page(), Next.js automatically deduplicates identical fetch requests, so only one network request is typically made.

When to Use metadata?

  • Home pages.
  • About pages.
  • Contact pages.
  • Documentation pages.
  • Marketing pages.
  • Static SEO content.

When to Use generateMetadata()?

  • Product pages.
  • Blog detail pages.
  • User profile pages.
  • CMS-driven pages.
  • Database-driven pages.
  • Any page whose metadata depends on fetched data.

Interview Tip

In the Pages Router we use Head from next/head. In the App Router we use metadata for static metadata and generateMetadata() for dynamic metadata.
One-Line Definition:
The metadata export is a special App Router feature that declaratively defines SEO and HTML head metadata, while generateMetadata() dynamically generates metadata using fetched data.
