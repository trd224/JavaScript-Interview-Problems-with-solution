24. What is getStaticProps and When is it Called?

getStaticProps() is a special Next.js function used in the Pages Router to fetch data at build time for Static Site Generation (SSG).

getStaticProps() runs during the build process, generates static HTML, and provides the fetched data as props to the page component.

Why Do We Need getStaticProps()?

Some pages contain data that does not change frequently.

Examples:
  • Blog Posts
  • Documentation Pages
  • Landing Pages
  • Company Information Pages
  • Marketing Websites

Instead of fetching data for every request, we can fetch it once during the build process.

Basic Example

pages/posts.js
export async function getStaticProps() {

  const res = await fetch(
    'https://api.example.com/posts'
  );

  const posts = await res.json();

  return {
    props: { posts }
  };

}

export default function Posts({ posts }) {

  return (
    <h1>
      Total Posts: {posts.length}
    </h1>
  );

}

When is getStaticProps() Called?

The most important thing to remember is that getStaticProps() is NOT called when users visit the page.

What Does getStaticProps() Return?

Return Object
return {
  props: {
    posts
  }
};

The returned props are passed directly to the page component.

Page Component
export default function Posts({ posts }) {

  return (
    <h1>
      {posts.length}
    </h1>
  );

}

Using revalidate (ISR)

We can regenerate the page periodically using Incremental Static Regeneration (ISR).

ISR Example
export async function getStaticProps() {

  const posts = await fetchPosts();

  return {
    props: { posts },
    revalidate: 60
  };

}

getStaticProps vs getServerSideProps

Important Limitations

  • Only works in the Pages Router.
  • Cannot access request-specific data.
  • Cannot access cookies or headers.
  • Cannot access authenticated user information.

App Router Equivalent

The App Router does not use getStaticProps().

Instead, static generation happens automatically when data is cacheable.

App Router
async function getPosts() {

  const res = await fetch(
    'https://api.example.com/posts'
  );

  return res.json();

}

export default async function Blog() {

  const posts = await getPosts();

  return (
    <h1>
      {posts.length}
    </h1>
  );

}

Interview Tip

A common misconception is that getStaticProps() runs when users visit the page. In reality, it runs during the build process and the generated HTML is reused for all users.
One-Line Definition:
getStaticProps() is a Pages Router function that fetches data during the build process and provides it to a statically generated page.
