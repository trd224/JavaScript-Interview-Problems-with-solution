48. What is SWR and how does it integrate with Next.js?

SWR is a React data fetching library created by Vercel that provides caching, automatic revalidation, request deduplication, polling, and real-time data synchronization.

SWR stands for Stale-While-Revalidate. It immediately shows cached data (stale data), then fetches fresh data in the background and updates the UI automatically.

Why Do We Need SWR?

Without SWR, we often write repetitive code for:

  • Loading states
  • Error handling
  • Caching
  • Refetching
  • Revalidation
Without SWR
'use client';

import { useEffect, useState } from 'react';

export default function Users() {

  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {

    async function loadUsers() {

      const res = await fetch('/api/users');
      const data = await res.json();

      setUsers(data);
      setLoading(false);

    }

    loadUsers();

  }, []);

  return loading
    ? <h2>Loading...</h2>
    : <div>{users.length}</div>;

}

SWR handles most of this automatically.

Install SWR

Terminal
npm install swr

Basic SWR Example

Client Component
'use client';

import useSWR from 'swr';

const fetcher = (url) => fetch(url).then(res=>res.json());

export default function Users(){

  const {data, error, isLoading} = useSWR(
    '/api/users',
    fetcher
  );

  if(isLoading){
    return <h2>Loading...</h2>;
  }

  if(error){
    return <h2>Error Loading Users</h2>;
  }

  return(
    <div>
      {data.map(user=>(
        <p key={user.id}>
          {user.name}
        </p>
      ))}
    </div>
  );

}

Why Is SWR Fast?

Because cached data can be displayed instantly.

Users do not always need to wait for a new API request.

Automatic Revalidation

SWR automatically refetches data when:

  • User returns to the tab.
  • Network reconnects.
  • Component remounts.
Example
const {data}=useSWR(
  '/api/users',
  fetcher
);

These behaviors work automatically.

Polling Data

SWR can automatically refresh data at a specified interval.

Auto Refresh Every 5 Seconds
const {data}=useSWR(
  '/api/users',
  fetcher,
  {refreshInterval:5000}
);

Manual Revalidation

We can force SWR to refetch data.

Example
import useSWR,{mutate} from 'swr';

const refresh=()=>{
  mutate('/api/users');
};

How Does SWR Integrate With Next.js?

SWR works especially well with Next.js because both are maintained by Vercel.

  • Works in Client Components.
  • Works with API Routes.
  • Works with Route Handlers.
  • Works with Server Components + Client Components.
  • Supports Next.js caching strategies.

App Router Example

app/users/page.js
'use client';

import useSWR from 'swr';

const fetcher=url=>
  fetch(url).then(res=>res.json());

export default function UsersPage(){

  const {data,isLoading}=useSWR(
    '/api/users',
    fetcher
  );

  if(isLoading){
    return <h2>Loading...</h2>;
  }

  return(
    <div>
      Total Users: {data.length}
    </div>
  );

}

Pages Router Example

pages/users.js
import useSWR from 'swr';

When Should We Use SWR?

  • User dashboards.
  • Profile pages.
  • Live statistics.
  • Search results.
  • Admin panels.
  • Frequently changing data.

When Should We Avoid SWR?

  • SEO-critical pages.
  • Static marketing pages.
  • Pages that should be rendered fully on the server.

Important Note

SWR runs in Client Components. If we need SEO-friendly data fetching, Server Components or server-side data fetching should usually be preferred.

Interview Tip

A common misconception is that SWR replaces server-side rendering. In reality, SWR is primarily for client-side data fetching, caching, and revalidation.
One-Line Definition:
SWR is a React data fetching library from Vercel that integrates seamlessly with Next.js by providing client-side caching, automatic revalidation, polling, and stale-while-revalidate data fetching.
