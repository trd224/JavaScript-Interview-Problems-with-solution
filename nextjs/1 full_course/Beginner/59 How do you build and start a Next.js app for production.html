59. How do you build and start a Next.js app for production?

To run a Next.js app in production, we first build the application using npm run build, then start the optimized production server using npm start.

The build command creates an optimized production version of our Next.js application, and the start command runs that production build.

Step 1: Install Dependencies

Terminal
npm install

This installs all dependencies from package.json.

Step 2: Build the Application

Terminal
npm run build

This creates an optimized production build inside the .next folder.

Step 3: Start the Production Server

Terminal
npm start

This starts the Next.js production server.

package.json Scripts

package.json
{
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start"
  }
}

Production Server Runs

What Happens During npm run build?

  • Next.js compiles our application.
  • JavaScript and CSS are optimized.
  • Static pages are generated.
  • Server-rendered pages are prepared.
  • Route information is created.
  • Production assets are generated.

Default Production Port

By default, next start runs the app on port 3000.

Output
Local: http://localhost:3000

Run on a Custom Port

Terminal
npm start -- -p 4000

This starts the production server on port 4000.

Alternative Command

Terminal
npx next start -p 4000

Using Environment Variables

Production environment variables should be available before building and starting the app.

.env.production
NEXT_PUBLIC_API_URL=https://api.example.com
DATABASE_URL=mongodb://localhost:27017/mydb
Variables used in the browser with NEXT_PUBLIC_ are included during build time, so rebuild the app after changing them.

Development vs Production

Command Purpose
npm run dev Runs development server with hot reload
npm run build Creates optimized production build
npm start Runs the production build

Common Production Commands

Commands
npm install
npm run build
npm start

Using PM2 in Production

On a real server, we often use a process manager like PM2 to keep the Next.js app running.

Install PM2
npm install -g pm2
Start Next.js with PM2
pm2 start npm --name "next-app" -- start
Useful PM2 Commands
pm2 status
pm2 logs next-app
pm2 restart next-app
pm2 save

Important Note

next start should be used only after running next build. It does not build the app automatically.

Interview Tip

In production, we run npm run build to create an optimized build and then run npm start to start the production server.
One-Line Definition:
A Next.js app is built for production using npm run build and started using npm start, which runs the optimized production build from the .next folder.
